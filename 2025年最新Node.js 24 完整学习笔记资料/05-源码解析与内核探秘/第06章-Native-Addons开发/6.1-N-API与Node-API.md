# 6.1 N-API 与 Node-API：C++ 扩展最佳实践

> 深入理解 Node.js 稳定 ABI 的原生扩展开发

---

## 📋 核心概念

**N-API（Node-API）** 是 Node.js 提供的**稳定 C API**，用于构建原生扩展模块。

**核心问题：**
- 什么是 N-API？与 NAN 有什么区别？
- 如何进行 JavaScript ↔ C++ 类型转换？
- 如何处理异步操作？
- 如何管理对象生命周期？

**核心优势：**
```
1. ABI 稳定：跨 Node.js 版本兼容
2. 性能优异：接近纯 C++ 性能
3. 内存安全：自动垃圾回收集成
4. 异步支持：线程池和 Promise 集成
```

---

## 💻 N-API vs NAN

### 1. 对比

| 特性 | N-API | NAN |
|------|-------|-----|
| **ABI 稳定性** | ✅ 稳定 | ❌ 不稳定 |
| **跨版本兼容** | ✅ 无需重新编译 | ❌ 需要重新编译 |
| **API 复杂度** | 中等 | 较高 |
| **性能** | 接近原生 | 接近原生 |
| **推荐度** | ✅ 强烈推荐 | ❌ 已过时 |

---

### 2. N-API 架构

```
JavaScript 层
     ↓
N-API (napi.h)  ← 稳定 ABI
     ↓
Node.js 内部
     ↓
V8 引擎
```

## 💻 基础示例

### 1. Hello World

```cpp
// hello.cc
#include <node_api.h>

// 导出函数
napi_value Hello(napi_env env, napi_callback_info info) {
  napi_value result;
  
  // 创建字符串
  napi_create_string_utf8(env, "Hello from C++!", NAPI_AUTO_LENGTH, &result);
  
  return result;
}

// 模块初始化
NAPI_MODULE_INIT() {
  napi_value hello_fn;
  
  // 创建函数
  napi_create_function(env, "hello", NAPI_AUTO_LENGTH, Hello, nullptr, &hello_fn);
  
  // 设置导出属性
  napi_set_named_property(env, exports, "hello", hello_fn);
  
  return exports;
}
```

```python
  }]
```

### 2. 加法函数

```cpp
// add.cc
#include <node_api.h>

napi_value Add(napi_env env, napi_callback_info info) {
  napi_status status;
  
  // 🔥 获取参数
  size_t argc = 2;
  napi_value args[2];
  status = napi_get_cb_info(env, info, &argc, args, nullptr, nullptr);
  
  // 🔥 检查参数数量
  if (argc < 2) {
    napi_throw_error(env, nullptr, "Expected 2 arguments");
    return nullptr;
  }
  
  // 🔥 转换为数字
  double a, b;
  status = napi_get_value_double(env, args[0], &a);
  if (status != napi_ok) {
    napi_throw_type_error(env, nullptr, "Argument 0 must be a number");
    return nullptr;
  }
  
  status = napi_get_value_double(env, args[1], &b);
  if (status != napi_ok) {
    napi_throw_type_error(env, nullptr, "Argument 1 must be a number");
    return nullptr;
  }
  
  // 🔥 计算结果
  double result = a + b;
  
  // 🔥 创建返回值
  napi_value result_value;
  napi_create_double(env, result, &result_value);
  
  return result_value;
}

NAPI_MODULE_INIT() {
  napi_value add_fn;
  napi_create_function(env, "add", NAPI_AUTO_LENGTH, Add, nullptr, &add_fn);
  napi_set_named_property(env, exports, "add", add_fn);
  return exports;
}
```

## 💻 异步操作

### 1. AsyncWork（线程池）

```cpp
// async_work.cc
#include <node_api.h>
#include <unistd.h>  // sleep

struct AsyncData {
  napi_async_work work;
  napi_deferred deferred;
  int input;
  int result;
};

// 🔥 后台线程执行（不能调用 N-API）
void ExecuteWork(napi_env env, void* data) {
  AsyncData* async_data = (AsyncData*)data;
  
  // 模拟耗时操作
  sleep(2);
  
  // 计算结果
  async_data->result = async_data->input * 2;
}

// 🔥 完成回调（主线程，可调用 N-API）
void CompleteWork(napi_env env, napi_status status, void* data) {
  AsyncData* async_data = (AsyncData*)data;
  
  if (status == napi_ok) {
    // 🔥 Resolve Promise
    napi_value result;
    napi_create_int32(env, async_data->result, &result);
    napi_resolve_deferred(env, async_data->deferred, result);
  } else {
    // 🔥 Reject Promise
    napi_value error;
    napi_create_string_utf8(env, "Work failed", NAPI_AUTO_LENGTH, &error);
    napi_reject_deferred(env, async_data->deferred, error);
  }
  
  // 🔥 清理
  napi_delete_async_work(env, async_data->work);
  delete async_data;
}

napi_value AsyncFunction(napi_env env, napi_callback_info info) {
  // 🔥 获取参数
  size_t argc = 1;
  napi_value args[1];
  napi_get_cb_info(env, info, &argc, args, nullptr, nullptr);
  
  int32_t input;
  napi_get_value_int32(env, args[0], &input);
  
  // 🔥 创建 Promise
  napi_value promise;
  napi_deferred deferred;
  napi_create_promise(env, &deferred, &promise);
  
  // 🔥 创建异步数据
  AsyncData* async_data = new AsyncData();
  async_data->input = input;
  async_data->deferred = deferred;
  
  // 🔥 创建异步工作
  napi_value work_name;
  napi_create_string_utf8(env, "AsyncWork", NAPI_AUTO_LENGTH, &work_name);
  
  napi_create_async_work(
    env,
    nullptr,
    work_name,
    ExecuteWork,
    CompleteWork,
    async_data,
    &async_data->work
  );
  
  // 🔥 加入队列
  napi_queue_async_work(env, async_data->work);
  
  return promise;
}

NAPI_MODULE_INIT() {
  napi_value fn;
  napi_create_function(env, "asyncFunction", NAPI_AUTO_LENGTH, AsyncFunction, nullptr, &fn);
  napi_set_named_property(env, exports, "asyncFunction", fn);
  return exports;
}
```

**使用：**
```javascript
const addon = require('./build/Release/async_work.node');

addon.asyncFunction(10).then(result => {
  console.log('Result:', result);  // 20 (2秒后)
});
```

### 2. ThreadSafe Function（回调）

```cpp
// threadsafe_function.cc
#include <node_api.h>
#include <thread>

struct TsfData {
  napi_threadsafe_function tsfn;
  std::thread worker_thread;
};

// 🔥 后台线程调用
void CallJavaScript(napi_env env, napi_value js_callback, void* context, void* data) {
  int* value = (int*)data;
  
  napi_value argv[1];
  napi_create_int32(env, *value, &argv[0]);
  
  napi_value global;
  napi_get_global(env, &global);
  
  napi_call_function(env, global, js_callback, 1, argv, nullptr);
  
  delete value;
}

// 🔥 工作线程
void WorkerThread(napi_threadsafe_function tsfn) {
  for (int i = 0; i < 10; i++) {
    int* value = new int(i);
    
    // 🔥 安全调用 JavaScript
    napi_call_threadsafe_function(tsfn, value, napi_tsfn_blocking);
    
    std::this_thread::sleep_for(std::chrono::milliseconds(500));
  }
  
  // 🔥 释放
  napi_release_threadsafe_function(tsfn, napi_tsfn_release);
}

napi_value StartWorker(napi_env env, napi_callback_info info) {
  // 🔥 获取回调函数
  size_t argc = 1;
  napi_value args[1];
  napi_get_cb_info(env, info, &argc, args, nullptr, nullptr);
  
  // 🔥 创建 ThreadSafe Function
  napi_value work_name;
  napi_create_string_utf8(env, "Worker", NAPI_AUTO_LENGTH, &work_name);
  
  napi_threadsafe_function tsfn;
  napi_create_threadsafe_function(
    env,
    args[0],             // JS callback
    nullptr,
    work_name,
    0,                   // max_queue_size (0 = unlimited)
    1,                   // initial_thread_count
    nullptr,
    nullptr,
    nullptr,
    CallJavaScript,
    &tsfn
  );
  
  // 🔥 启动工作线程
  std::thread worker(WorkerThread, tsfn);
  worker.detach();
  
  return nullptr;
}

NAPI_MODULE_INIT() {
  napi_value fn;
  napi_create_function(env, "startWorker", NAPI_AUTO_LENGTH, StartWorker, nullptr, &fn);
  napi_set_named_property(env, exports, "startWorker", fn);
  return exports;
}
```

**使用：**
```javascript
const addon = require('./build/Release/threadsafe_function.node');

addon.startWorker(value => {
  console.log('Received:', value);
});

// 输出（每500ms）：
// Received: 0
// Received: 1
// ...
// Received: 9
```

### 2. Finalizer（析构函数）

```cpp
struct MyData {
  int value;
  ~MyData() {
    printf("MyData destroyed\n");
  }
};

void Finalize(napi_env env, void* finalize_data, void* finalize_hint) {
  MyData* data = (MyData*)finalize_data;
  delete data;
}

napi_value CreateObject(napi_env env, napi_callback_info info) {
  napi_value obj;
  napi_create_object(env, &obj);

  napi_wrap(env, obj, data, Finalize, nullptr, nullptr);
  
  return obj;
```

### 案例1：SHA256 哈希

```cpp
// sha256.cc
#include <node_api.h>
#include <openssl/sha.h>

napi_value Sha256(napi_env env, napi_callback_info info) {
  size_t argc = 1;
  napi_value args[1];
  napi_get_cb_info(env, info, &argc, args, nullptr, nullptr);
  
  // 获取输入
  size_t str_len;
  napi_get_value_string_utf8(env, args[0], nullptr, 0, &str_len);
  
  char* input = new char[str_len + 1];
  napi_get_value_string_utf8(env, args[0], input, str_len + 1, &str_len);
  
  // SHA256
  unsigned char hash[SHA256_DIGEST_LENGTH];
  SHA256((unsigned char*)input, str_len, hash);
  
  // 转为 hex
  char hex[SHA256_DIGEST_LENGTH * 2 + 1];
  for (int i = 0; i < SHA256_DIGEST_LENGTH; i++) {
    sprintf(hex + i * 2, "%02x", hash[i]);
  }
  hex[SHA256_DIGEST_LENGTH * 2] = '\0';
  
  delete[] input;
  
  napi_value result;
  napi_create_string_utf8(env, hex, NAPI_AUTO_LENGTH, &result);
  return result;
}

NAPI_MODULE_INIT() {
  napi_value fn;
  napi_create_function(env, "sha256", NAPI_AUTO_LENGTH, Sha256, nullptr, &fn);
  napi_set_named_property(env, exports, "sha256", fn);
  return exports;
}
```

### 案例2：图像处理（灰度化）

```cpp
// grayscale.cc
#include <node_api.h>

napi_value Grayscale(napi_env env, napi_callback_info info) {
  size_t argc = 1;
  napi_value args[1];
  napi_get_cb_info(env, info, &argc, args, nullptr, nullptr);
  
  // 获取 Buffer
  void* data;
  size_t length;
  napi_get_buffer_info(env, args[0], &data, &length);
  
  unsigned char* pixels = (unsigned char*)data;
  
  // 灰度化（RGBA）
  for (size_t i = 0; i < length; i += 4) {
    unsigned char r = pixels[i];
    unsigned char g = pixels[i + 1];
    unsigned char b = pixels[i + 2];
    
    // 灰度值
    unsigned char gray = (unsigned char)(0.299 * r + 0.587 * g + 0.114 * b);
    
    pixels[i] = gray;
    pixels[i + 1] = gray;
    pixels[i + 2] = gray;
    // alpha 通道不变
  }
  
  return args[0];  // 返回修改后的 Buffer
}

NAPI_MODULE_INIT() {
  napi_value fn;
  napi_create_function(env, "grayscale", NAPI_AUTO_LENGTH, Grayscale, nullptr, &fn);
  napi_set_named_property(env, exports, "grayscale", fn);
  return exports;
}
```

## 📝 总结

### 核心要点

1. **N-API**：稳定 ABI，跨版本兼容
2. **类型转换**：JavaScript ↔ C++ 数据交换
3. **异步操作**：AsyncWork、ThreadSafe Function
4. **生命周期**：Reference、Finalizer

### 实现关键

1. **napi_create_function**：导出函数
2. **napi_get_cb_info**：获取参数
3. **napi_create_async_work**：异步任务
4. **napi_create_threadsafe_function**：线程安全回调

### 应用价值

1. **高性能**：CPU 密集型任务加速
2. **系统集成**：调用系统 API
3. **跨版本**：无需重新编译
4. **生态兼容**：与 Node.js 无缝集成
