## 💻 为什么选择 Rust？

### 1. 内存安全

```rust
// Rust：编译时检查
fn process_data(data: Vec<i32>) -> i32 {
    data[0]  // ✅ 编译器保证 data 有效
}

// C++：运行时风险
int process_data(int* data, int len) {
    return data[0];  // ❌ 可能空指针、越界
}
```

---

### 2. 并发安全

```rust
// Rust：编译时检查数据竞争
use std::sync::Arc;
use std::sync::Mutex;

let counter = Arc::new(Mutex::new(0));
let counter2 = Arc::clone(&counter);

std::thread::spawn(move || {
    let mut num = counter2.lock().unwrap();
    *num += 1;
});

// C++：运行时竞争
int counter = 0;
std::thread t([&counter]() {
    counter++;  // ❌ 数据竞争！
});
```

use napi_derive::napi;

  })

  if flag { Some(42) } else { None }

  }
}

## 💻 结构体与类

### 1. 导出结构体

```rust
use napi_derive::napi;

#[napi]
pub struct Point {
  pub x: f64,
  pub y: f64,
}

#[napi]
impl Point {
  #[napi(constructor)]
  pub fn new(x: f64, y: f64) -> Self {
    Point { x, y }
  }

  #[napi]
  pub fn distance(&self, other: &Point) -> f64 {
    let dx = self.x - other.x;
    let dy = self.y - other.y;
    (dx * dx + dy * dy).sqrt()
  }

  #[napi(getter)]
  pub fn get_x(&self) -> f64 {
    self.x
  }

  #[napi(setter)]
  pub fn set_x(&mut self, x: f64) {
    self.x = x;
  }
}
```

**使用：**
```javascript
const { Point } = require('./index.js');

const p1 = new Point(0, 0);
const p2 = new Point(3, 4);

console.log(p1.distance(p2));  // 5

p1.x = 1;
console.log(p1.x);  // 1
```

### 2. Tokio 集成

```rust
use napi_derive::napi;
use tokio::fs;

#[napi]
async fn read_file_async(path: String) -> Result<String> {
  let content = fs::read_to_string(path)
    .await
    .map_err(|e| Error::from_reason(e.to_string()))?;
  
  Ok(content)
}

#[napi]
async fn fetch_url(url: String) -> Result<String> {
  let response = reqwest::get(&url)
    .await
    .map_err(|e| Error::from_reason(e.to_string()))?;
  
  let body = response.text()
    .await
    .map_err(|e| Error::from_reason(e.to_string()))?;
  
  Ok(body)
}
```

### 3. 并行任务

```rust
use napi_derive::napi;
use tokio::task;

#[napi]
async fn parallel_tasks() -> Result<Vec<i32>> {
  let task1 = task::spawn(async { compute(1) });
  let task2 = task::spawn(async { compute(2) });
  let task3 = task::spawn(async { compute(3) });

  let results = tokio::try_join!(task1, task2, task3)
    .map_err(|e| Error::from_reason(e.to_string()))?;

  Ok(vec![results.0, results.1, results.2])
}

async fn compute(n: i32) -> i32 {
  tokio::time::sleep(tokio::time::Duration::from_secs(1)).await;
  n * 2
}
```

## 💻 Buffer 处理

### 1. 读取 Buffer

```rust
use napi::bindgen_prelude::*;
use napi_derive::napi;

#[napi]
fn process_buffer(buffer: Buffer) -> Result<u32> {
  let data: &[u8] = &buffer;
  
  // 计算校验和
  let checksum: u32 = data.iter().map(|&b| b as u32).sum();
  
  Ok(checksum)
}
```

---

### 2. 创建 Buffer

```rust
use napi::bindgen_prelude::*;
use napi_derive::napi;

#[napi]
fn create_buffer(size: u32) -> Result<Buffer> {
  let mut data = vec![0u8; size as usize];
  
  // 填充数据
  for (i, byte) in data.iter_mut().enumerate() {
    *byte = (i % 256) as u8;
  }
  
  Ok(Buffer::from(data))
}
```

## 💻 实战案例

### 案例1：图像处理

```rust
// Cargo.toml
[dependencies]
image = "0.24"
napi = "2"
napi-derive = "2"

// src/lib.rs
use image::{DynamicImage, GenericImageView, ImageFormat};
use napi_derive::napi;
use napi::bindgen_prelude::*;

#[napi]
async fn resize_image(
  input_path: String,
  output_path: String,
  width: u32,
  height: u32
) -> Result<()> {
  // 读取图像
  let img = image::open(&input_path)
    .map_err(|e| Error::from_reason(e.to_string()))?;

  // 调整大小
  let resized = img.resize(
    width,
    height,
    image::imageops::FilterType::Lanczos3
  );

  // 保存
  resized.save(&output_path)
    .map_err(|e| Error::from_reason(e.to_string()))?;

  Ok(())
}

#[napi]
fn grayscale(input_path: String, output_path: String) -> Result<()> {
  let img = image::open(&input_path)
    .map_err(|e| Error::from_reason(e.to_string()))?;

  let gray = img.grayscale();

  gray.save(&output_path)
    .map_err(|e| Error::from_reason(e.to_string()))?;

  Ok(())
}
```

### 案例2：JSON 解析

```rust
// Cargo.toml
[dependencies]
serde = { version = "1.0", features = ["derive"] }
serde_json = "1.0"
napi = "2"
napi-derive = "2"

// src/lib.rs
use serde::{Deserialize, Serialize};
use napi_derive::napi;
use napi::bindgen_prelude::*;

#[derive(Serialize, Deserialize)]
#[napi(object)]
pub struct User {
  pub id: u32,
  pub name: String,
  pub email: String,
}

#[napi]
fn parse_users(json_str: String) -> Result<Vec<User>> {
  let users: Vec<User> = serde_json::from_str(&json_str)
    .map_err(|e| Error::from_reason(e.to_string()))?;
  
  Ok(users)
}

#[napi]
fn filter_users(users: Vec<User>, min_id: u32) -> Vec<User> {
  users.into_iter()
    .filter(|u| u.id >= min_id)
    .collect()
}
```

### 案例3：加密

```rust
// Cargo.toml
[dependencies]
sha2 = "0.10"
hex = "0.4"
napi = "2"
napi-derive = "2"

// src/lib.rs
use sha2::{Sha256, Digest};
use napi_derive::napi;

#[napi]
fn sha256_hash(input: String) -> String {
  let mut hasher = Sha256::new();
  hasher.update(input.as_bytes());
  let result = hasher.finalize();
  hex::encode(result)
}

#[napi]
fn batch_hash(inputs: Vec<String>) -> Vec<String> {
  inputs.iter()
    .map(|input| sha256_hash(input.clone()))
    .collect()
}
```

## 📝 总结

### 核心要点

1. **Rust**：内存安全、并发安全、现代语法
2. **napi-rs**：推荐框架，易用、高效
3. **异步**：Tokio 无缝集成
4. **类型安全**：编译时保证

### 实现关键

1. **#[napi]**：导出函数/结构体
2. **async fn**：异步操作
3. **Result<T>**：错误处理
4. **Buffer**：二进制数据传递

### 应用价值

1. **性能**：接近原生 C++ 性能
2. **安全**：无内存泄漏、无数据竞争
3. **效率**：现代语法、丰富生态
4. **维护**：编译器友好、易重构
