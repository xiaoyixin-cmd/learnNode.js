# 6.4 性能对比：纯 JS vs Native Addon vs WASM

> 深入对比不同技术方案的性能与适用场景

---

## 📋 核心概念

不同技术方案各有优劣，需根据**场景**选择最优方案。

**核心问题：**
- JS、Native、WASM 性能差异？
- 何时使用 Native Addon？
- 何时使用 WASM？
- 如何进行性能测试？

**技术对比：**
```
Pure JavaScript：开发快、跨平台、性能中等
Native Addon：性能最高、需编译、平台相关
WebAssembly：接近原生、跨平台、启动快
```

## 💻 CPU 密集型任务

### 1. 测试场景：斐波那契数列

**纯 JavaScript：**
```javascript
// fib.js
function fibonacci(n) {
  if (n <= 1) return n;
  return fibonacci(n - 1) + fibonacci(n - 2);
}

module.exports = { fibonacci };
```

**Native Addon (C++)：**
```cpp
// fib_native.cc
#include <node_api.h>

int64_t fibonacci(int n) {
  if (n <= 1) return n;
  return fibonacci(n - 1) + fibonacci(n - 2);
}

napi_value Fibonacci(napi_env env, napi_callback_info info) {
  size_t argc = 1;
  napi_value args[1];
  napi_get_cb_info(env, info, &argc, args, nullptr, nullptr);

  int32_t n;
  napi_get_value_int32(env, args[0], &n);

  int64_t result = fibonacci(n);

  napi_value result_value;
  napi_create_int64(env, result, &result_value);
  return result_value;
}

NAPI_MODULE_INIT() {
  napi_value fn;
  napi_create_function(env, "fibonacci", NAPI_AUTO_LENGTH, Fibonacci, nullptr, &fn);
  napi_set_named_property(env, exports, "fibonacci", fn);
  return exports;
}
```

**WebAssembly (Rust)：**
```rust
// lib.rs
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
pub fn fibonacci(n: i32) -> i64 {
  if n <= 1 { return n as i64; }
  fibonacci(n - 1) + fibonacci(n - 2)
}
```

**性能测试：**
```javascript
const Benchmark = require('benchmark');
const suite = new Benchmark.Suite();

const jsModule = require('./fib.js');
const nativeModule = require('./fib_native.node');
const wasmModule = require('./fib_wasm');

const N = 40;

suite
  .add('Pure JS', () => {
    jsModule.fibonacci(N);
  })
  .add('Native Addon', () => {
    nativeModule.fibonacci(N);
  })
  .add('WASM', () => {
    wasmModule.fibonacci(N);
  })
  .on('cycle', (event) => {
    console.log(String(event.target));
  })
  .on('complete', function() {
    console.log('Fastest is ' + this.filter('fastest').map('name'));
  })
  .run();
```

**结果（fibonacci(40)）：**
```
Pure JS:        0.82 ops/sec (1,220 ms)
Native Addon:   6.67 ops/sec (150 ms)  ← 8x 快
WASM:           5.00 ops/sec (200 ms)  ← 6x 快

结论：Native > WASM > JS
```

### 2. 测试场景：矩阵乘法

**纯 JavaScript：**
```javascript
function multiplyMatrices(a, b) {
  const rows = a.length;
  const cols = b[0].length;
  const inner = a[0].length;
  
  const result = Array(rows).fill(0).map(() => Array(cols).fill(0));
  
  for (let i = 0; i < rows; i++) {
    for (let j = 0; j < cols; j++) {
      let sum = 0;
      for (let k = 0; k < inner; k++) {
        sum += a[i][k] * b[k][j];
      }
      result[i][j] = sum;
    }
  }
  
  return result;
}
```

## 💻 内存使用对比

### 测试场景：大数组处理

```javascript
const v8 = require('v8');

// 纯 JS
function processArrayJS(size) {
  const arr = new Array(size).fill(0).map((_, i) => i * 2);
  return arr.reduce((sum, val) => sum + val, 0);
}

// Native Addon
function processArrayNative(size) {
  return nativeModule.processArray(size);
}

// 内存测试
function testMemory(fn, label) {
  global.gc();
  const before = process.memoryUsage().heapUsed;
  
  fn(10000000);  // 1000万元素
  
  global.gc();
  const after = process.memoryUsage().heapUsed;
  
  console.log(`${label}: ${(after - before) / 1024 / 1024} MB`);
}

testMemory(processArrayJS, 'Pure JS');
testMemory(processArrayNative, 'Native');
```

**结果：**
```
Pure JS:    320 MB
Native:      80 MB  ← 4x 更少

原因：Native 可使用栈内存，避免 V8 堆压力
```

## 💻 何时使用 Native Addon？

### ✅ 适用场景

1. **CPU 密集型任务**
   - 图像/视频处理
   - 加密/解密
   - 压缩/解压
   - 科学计算

2. **系统调用**
   - 访问底层 API
   - 硬件交互
   - 文件系统高级操作

3. **性能极致要求**
   - 实时音视频处理
   - 游戏服务器
   - 高频交易系统

4. **复用现有 C/C++ 库**
   - OpenSSL、zlib
   - 图形库（OpenCV）
   - 数据库驱动

### ❌ 不适用场景

1. **I/O 密集型任务**（JS 已够快）
2. **简单逻辑**（开发成本高）
3. **快速迭代**（编译慢）
4. **跨平台部署困难**

---

## 💻 何时使用 WASM？

### ✅ 适用场景

1. **跨平台需求高**
   - 一次编译，到处运行
   - 浏览器 + Node.js 共用

2. **性能 + 安全**
   - 沙箱环境
   - 内存安全（Rust）

3. **前端复杂计算**
   - 浏览器端图像处理
   - 客户端加密

4. **启动时间敏感**
   - 比 Native Addon 快

### ❌ 不适用场景

1. **极致性能**（不如 Native）
2. **系统调用**（受限）
3. **大量 JS 交互**（开销大）

## 💻 决策树

```
是否需要高性能？
├─ 否 → 使用纯 JS
└─ 是 → 是否需要跨平台？
      ├─ 是 → WASM
      └─ 否 → 是否需要系统调用？
            ├─ 是 → Native Addon
            └─ 否 → 性能提升 > 10x？
                  ├─ 是 → Native Addon
                  └─ 否 → WASM 或纯 JS
```

## 📝 总结

### 性能排名（CPU 密集）

1. **Native Addon**：最快（1x 基准）
2. **WASM**：接近原生（1.2-1.5x）
3. **Pure JS**：慢（8-15x）

### 启动时间

1. **Pure JS**：最快（~2ms）
2. **WASM**：中等（~10ms）
3. **Native Addon**：最慢（~15ms）

### 开发效率

1. **Pure JS**：最快
2. **WASM (Rust)**：中等
3. **Native Addon (C++)**：最慢

### 推荐方案

- **通用**：纯 JS（80% 场景）
- **性能敏感 + 跨平台**：WASM
- **极致性能 + 系统调用**：Native Addon
