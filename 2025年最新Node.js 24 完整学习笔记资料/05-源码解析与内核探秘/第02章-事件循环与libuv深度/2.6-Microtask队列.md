# 2.6 Microtask 队列与 Promise 执行时机

> 深入理解微任务队列在 Node.js 和 V8 中的协作机制

---

## 📋 核心概念

Microtask（微任务）是 JavaScript 事件循环中**优先级最高的异步任务**。

**微任务来源：**
- `process.nextTick()` - Node.js 独有
- `Promise.then/catch/finally`
- `queueMicrotask()`
- `MutationObserver`（浏览器）

**执行时机：**
```
每个宏任务执行后 → 清空微任务队列 → 下一个宏任务
```

## 💻 微任务队列架构

### 1. 队列结构

```
┌─────────────────────────────┐
│   nextTick Queue            │ ← 最高优先级
│   (Node.js 独有)            │
└──────────────┬──────────────┘
               ↓
┌─────────────────────────────┐
│   Microtask Queue           │ ← 次优先级
│   (V8 引擎管理)              │
│   - Promise.then            │
│   - queueMicrotask          │
└──────────────┬──────────────┘
               ↓
┌─────────────────────────────┐
│   Macrotask (宏任务)         │
│   - setTimeout              │
│   - setImmediate            │
│   - I/O 回调                │
└─────────────────────────────┘
```

---

### 2. 执行流程

```c
// Node.js 事件循环伪代码
while (true) {
  // 1. 执行一个宏任务
  executeNextMacrotask();
  
  // 2. 清空 nextTick 队列（Node.js）
  while (nextTickQueue.length > 0) {
    const callback = nextTickQueue.shift();
    callback();
  }
  
  // 3. 清空微任务队列（V8）
  while (microtaskQueue.length > 0) {
    const microtask = microtaskQueue.shift();
    microtask();
  }
  
  // 4. 继续下一个宏任务
}
```

## 💻 process.nextTick 实现

### 1. JavaScript 层

```javascript
// lib/internal/process/task_queues.js
const queue = {
  head: null,
  tail: null
};

function nextTick(callback, ...args) {
  validateFunction(callback, 'callback');

  // 🔥 创建 TickObject
  const tickObject = {
    callback,
    args,
    next: null
  };

  // 🔥 加入队列
  if (queue.tail !== null) {
    queue.tail.next = tickObject;
  } else {
    queue.head = tickObject;
  }
  queue.tail = tickObject;

  // 🔥 设置标志位（通知 C++ 层）
  setHasTickScheduled(true);
}
```

```cpp
void InternalCallbackScope::Close() {
  Environment* env = env_;
  
  if (!tick_info->has_tick_scheduled()) {
    env->isolate()->RunMicrotasks();
  }
  
  // 🔥 执行 nextTick 队列
  if (tick_info->has_tick_scheduled()) {
    HandleScope scope(env->isolate());
    
    do {
      // 🔥 调用 JavaScript 层的 processTicksAndRejections
      env->tick_callback_function()->Call(
        env->context(),
        Undefined(env->isolate()),
        0,
        nullptr
      ).ToLocalChecked();
      
      // 🔥 执行 V8 微任务
      env->isolate()->RunMicrotasks();
      
    } while (tick_info->has_tick_scheduled());
  }
}
```

### 3. 队列处理

```javascript
// lib/internal/process/task_queues.js
function processTicksAndRejections() {
  let tock;
  
  // 🔥 处理 nextTick 队列
  do {
    while (tock = queue.head) {
      queue.head = tock.next;
      
      const callback = tock.callback;
      const args = tock.args;
      
      // 🔥 执行回调
      if (args === undefined) {
        callback();
      } else {
        callback(...args);
      }
    }
    
    // 🔥 执行 Promise rejections
    processPromiseRejections();
    
    // 🔥 重置队列
    queue.head = queue.tail = null;
    
  } while (queue.head !== null);
  
  setHasTickScheduled(false);
}
```

### 2. V8 内部实现

```cpp
// deps/v8/src/execution/microtask-queue.cc
void MicrotaskQueue::EnqueueMicrotask(Microtask microtask) {
  // 🔥 加入微任务队列
  if (size_ == capacity_) {
    // 扩容
    Grow();
  }
  
  ring_buffer_[start_ + size_] = microtask;
  size_++;
}

void MicrotaskQueue::RunMicrotasks(Isolate* isolate) {
  HandleScope scope(isolate);
  
  // 🔥 执行所有微任务
  while (size_ > 0) {
    Microtask microtask = ring_buffer_[start_];
    start_ = (start_ + 1) % capacity_;
    size_--;
    
    // 🔥 执行微任务
    microtask.Run(isolate);
  }
}
```

### 1. Promise.then 实现

```cpp

transitioning javascript builtin PromisePrototypeThen(
    js-implicit context: NativeContext, receiver: JSAny
)(onFulfilled: JSAny, onRejected: JSAny): JSAny {
  // 🔥 创建 PromiseReaction
  const promise = Cast<JSPromise>(receiver) otherwise ThrowTypeError;
  
  const resultPromise = NewJSPromise(promise);
  
  // 🔥 注册 reaction
  PerformPromiseThen(
    promise,
    onFulfilled,
    onRejected,
    resultPromise
  );
  
  return resultPromise;
}
```

### 2. Promise 状态转换

```cpp
// deps/v8/src/builtins/promise-resolve.tq
transitioning builtin ResolvePromise(
    implicit context: Context
)(promise: JSPromise, resolution: JSAny): JSAny {
  // 🔥 Promise 状态转换
  if (IsPromise(resolution)) {
    return PromiseResolveThenableJob(promise, resolution);
  }
  
  // 🔥 Fulfill promise
  FulfillPromise(promise, resolution);
  
  // 🔥 触发 reactions（加入微任务队列）
  TriggerPromiseReactions(promise.reactions, resolution);
  
  return Undefined;
}

transitioning builtin TriggerPromiseReactions(
    implicit context: Context
)(reactions: PromiseReaction, argument: JSAny): JSAny {
  // 🔥 遍历所有 reactions
  let current = reactions;
  
  while (current != Undefined) {
    const reaction = Cast<PromiseReaction>(current);
    
    // 🔥 创建微任务
    EnqueueMicrotask(
      PromiseReactionJob(reaction, argument)
    );
    
    current = reaction.next;
  }
  
  return Undefined;
}
```

## 💻 执行顺序详解

### 1. nextTick vs Promise

```javascript
// order-test.js
console.log('1. sync start');

setTimeout(() => console.log('2. setTimeout'), 0);

Promise.resolve().then(() => console.log('3. Promise'));

process.nextTick(() => console.log('4. nextTick'));

console.log('5. sync end');

// 输出顺序：
// 1. sync start
// 5. sync end
// 4. nextTick        ← 最高优先级
// 3. Promise         ← 次优先级
// 2. setTimeout      ← 宏任务
```

**执行流程：**
```
1. 执行同步代码：1 → 5
2. 清空 nextTick 队列：4
3. 清空微任务队列：3
4. 执行下一个宏任务：2
```

```javascript
// nested-test.js
Promise.resolve().then(() => {
```

```
3. 清空微任务：Promise 1（加入新的 nextTick 和 Promise）
4. 清空 nextTick：nextTick in Promise
5. 清空微任务：Promise 2
```

## 💻 常见陷阱

### 1. 微任务饥饿

```javascript
// starvation.js
let tick = 0;

function infiniteNextTick() {
  tick++;
  
  if (tick % 1000 === 0) {
    console.log(`Tick: ${tick}`);
  }
  
  process.nextTick(infiniteNextTick);
}

process.nextTick(infiniteNextTick);

// 🔥 定时器永远不会执行！
setTimeout(() => {
  console.log('This will never run');
}, 100);

// 进程会卡死在 nextTick 循环中
```

**解决方案：**
```javascript
// 使用 setImmediate 让出控制权
function infiniteImmediate() {
  tick++;
  
  if (tick % 1000 === 0) {
    console.log(`Tick: ${tick}`);
  }
  
  setImmediate(infiniteImmediate);  // ✅ 允许其他任务执行
}

setImmediate(infiniteImmediate);

setTimeout(() => {
  console.log('This will run');  // ✅ 能够执行
}, 100);
```

---

### 2. Promise 链过长

```javascript
// long-promise-chain.js
let chain = Promise.resolve();

for (let i = 0; i < 10000; i++) {
  chain = chain.then(() => {
    // 每个 then 都会创建一个微任务
  });
}

// ⚠️ 10000 个微任务会阻塞事件循环
```

**优化：**
```javascript
// 使用 async/await + setImmediate
async function processChain() {
  for (let i = 0; i < 10000; i++) {
    // 处理逻辑
    
    if (i % 100 === 0) {
      // 每 100 次让出控制权
      await new Promise(resolve => setImmediate(resolve));
    }
  }
}
```

### 2. 检测微任务饥饿

```javascript
// starvation-detector.js
const MAX_MICROTASKS = 1000;
let microtaskCount = 0;
let lastCheck = Date.now();

const originalNextTick = process.nextTick;

process.nextTick = function(...args) {
  microtaskCount++;
  
  if (microtaskCount >= MAX_MICROTASKS) {
    const duration = Date.now() - lastCheck;
    
    console.warn(`⚠️  Microtask starvation detected!`);
    console.warn(`   ${microtaskCount} nextTicks in ${duration}ms`);
    console.warn(`   Macrotasks may be blocked`);
    
    microtaskCount = 0;
    lastCheck = Date.now();
  }
  
  originalNextTick(...args);
};

setInterval(() => {
  microtaskCount = 0;
  lastCheck = Date.now();
}, 1000);
```

## 📝 总结

### 核心要点

1. **优先级**：nextTick > 微任务 > 宏任务
2. **执行时机**：每个宏任务后清空
3. **V8 协作**：Node.js nextTick + V8 微任务队列
4. **性能影响**：微任务会阻塞宏任务

### 实现关键

1. **nextTick 队列**：JavaScript 层链表管理
2. **微任务队列**：V8 引擎环形缓冲区
3. **Promise reaction**：状态转换触发微任务
4. **执行流程**：nextTick → 微任务 → 宏任务

### 应用价值

1. **理解异步**：掌握微任务执行机制
2. **性能优化**：避免微任务饥饿
3. **问题排查**：定位微任务相关问题
4. **最佳实践**：合理选择异步方式
