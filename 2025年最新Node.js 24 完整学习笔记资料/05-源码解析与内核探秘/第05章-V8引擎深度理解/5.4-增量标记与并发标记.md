# 5.4 增量标记（Incremental Marking）与并发标记（Concurrent Marking）

> 深入理解 V8 的低延迟垃圾回收技术

---

## 📋 核心概念

增量标记和并发标记是 V8 为了**减少 GC 停顿时间**而引入的优化技术。

**核心问题：**
- 为什么需要增量标记？
- 并发标记如何工作？
- 写屏障（Write Barrier）是什么？
- Orinoco GC 有什么改进？

**GC 停顿问题：**
```
传统 Mark-Sweep：
  应用执行 → [停止世界 GC 200ms] → 应用执行

增量/并发 GC：
  应用执行 ⇄ [GC 10ms] ⇄ 应用执行 ⇄ [GC 10ms] ⇄ ...
  或者并发执行（不停止应用）
```

---

## 💻 增量标记（Incremental Marking）

### 1. 核心思想

**目标：** 将 GC 工作分散到多个小步骤，与应用执行交替进行。

```
完整 GC（200ms 停顿）：
━━━━━━━━━━━━━━━━━━━━━━━ 标记

增量 GC（10ms x 20 = 200ms 总时间，但分散）：
━━ 标记 ⇄ 应用 ⇄ ━━ 标记 ⇄ 应用 ⇄ ... ⇄ ━━ 标记
```

```cpp
class IncrementalMarking {

  // 🔥 增量标记状态
  enum State {

  bool ShouldStart();

  State state_;
  Heap* heap_;
  MarkingWorklist* marking_worklist_;
  IncrementalMarkingJob* incremental_marking_job_;
};
```

### 3. 增量标记启动

```cpp
// deps/v8/src/heap/incremental-marking.cc

/**
 * 启动增量标记
 */
void IncrementalMarking::Start(GarbageCollectionReason reason) {
  // 🔥 检查前置条件
  if (!CanStartIncrementalMarking()) {
    return;
  }

  // 🔥 初始化标记状态
  state_ = MARKING;

  // 🔥 重置标记位图
  heap_->mark_compact_collector()->ClearMarkbits();

  // 🔥 标记根对象
  MarkRoots();

  // 🔥 启动增量标记任务
  if (FLAG_incremental_marking_task) {
    incremental_marking_job_->ScheduleTask();
  }
}

bool IncrementalMarking::CanStartIncrementalMarking() {
  // 🔥 不在 GC 中
  if (heap_->gc_state() != Heap::NOT_IN_GC) {
    return false;
  }

  // 🔥 老生代接近限制
  if (heap_->OldGenerationSpaceAvailable() < heap_->PromotedSpaceSizeOfObjects()) {
    return true;
  }

  return false;
}
```

### 4. 增量标记步骤

```cpp
// deps/v8/src/heap/incremental-marking.cc

/**
 * 执行一步增量标记
 */
void IncrementalMarking::Step(
    intptr_t bytes_to_process,
    CompletionAction action) {
  
  if (state_ != MARKING) {
    return;
  }

  // 🔥 处理标记队列
  intptr_t bytes_processed = 0;

  while (bytes_processed < bytes_to_process) {
    if (marking_worklist_->IsEmpty()) {
      // 🔥 标记完成
      if (action == FORCE_COMPLETION) {
        Finish();
      }
      break;
    }

    // 🔥 处理一个对象
    HeapObject* object = marking_worklist_->Pop();
    bytes_processed += ProcessMarkingObject(object);
  }

  // 🔥 更新进度
  bytes_marked_ += bytes_processed;
}

intptr_t IncrementalMarking::ProcessMarkingObject(HeapObject* object) {
  // 🔥 访问对象的指针字段
  MarkingVisitor visitor(heap_);
  int size = object->Size();
  object->Iterate(&visitor);

  return size;
}
```

## 💻 并发标记（Concurrent Marking）

### 1. 核心思想

**目标：** 在后台线程执行标记，与应用线程并发运行。

```
应用线程：     ━━━━━━━━━━━━━━━━━━━━━━━
标记线程1：        ━━━━━━━━━━━━━━━━
标记线程2：          ━━━━━━━━━━━━━━
标记线程3：            ━━━━━━━━━━━━
```

---

### 3. 并发标记启动

```cpp
// deps/v8/src/heap/concurrent-marking.cc

/**
 * 调度并发标记任务
 */
void ConcurrentMarking::ScheduleTasks() {
  // 🔥 确定任务数量（通常 = CPU 核心数）
  int num_tasks = std::min(
    NumberOfAvailableProcessors(),
    kMaxConcurrentMarkingTasks
  );

  // 🔥 启动后台任务
  for (int i = 0; i < num_tasks; i++) {
    auto task = std::make_unique<ConcurrentMarkingTask>(
      heap_->isolate(),
      this,
      i
    );

    tasks_.push_back(std::move(task));

    // 🔥 提交到任务队列
    V8::GetCurrentPlatform()->CallOnWorkerThread(tasks_.back().get());
  }
}
```

### 4. 并发标记执行

```cpp
// deps/v8/src/heap/concurrent-marking.cc

/**
 * 后台线程执行标记
 */
void ConcurrentMarking::Run(int task_id) {
  // 🔥 获取工作队列
  MarkingWorklist::Local local_worklist(marking_worklist_);

  size_t marked_bytes = 0;

  // 🔥 处理标记队列
  while (true) {
    HeapObject* object;

    // 🔥 从队列获取对象
    if (!local_worklist.Pop(&object)) {
      break;  // 队列空
    }

    // 🔥 标记对象
    marked_bytes += MarkObject(object, &local_worklist);
  }

  // 🔥 更新总标记字节数
  total_marked_bytes_.fetch_add(marked_bytes, std::memory_order_relaxed);
}

size_t ConcurrentMarking::MarkObject(
    HeapObject* object,
    MarkingWorklist::Local* local_worklist) {
  
  // 🔥 访问对象的指针字段
  ConcurrentMarkingVisitor visitor(heap_, local_worklist);
  
  int size = object->Size();
  object->Iterate(&visitor);

  return size;
}
```

## 💻 写屏障（Write Barrier）

### 1. 问题：并发修改

```javascript
// 应用线程执行：
const obj1 = { ref: null };
const obj2 = { data: 'hello' };

// 标记线程并发执行，标记 obj1（黑色）

// 应用线程修改引用：
obj1.ref = obj2;  // 🔥 问题：obj2 可能未被标记！

// 如果没有写屏障，obj2 会被错误回收
```

---

### 2. 三色标记算法

```
白色（White）：未标记，可能是垃圾
灰色（Grey）：已标记，但子对象未标记
黑色（Black）：已标记，子对象也已标记

不变式：黑色对象不能直接引用白色对象
```

---

### 3. 写屏障实现

```cpp
// deps/v8/src/heap/incremental-marking-inl.h

/**
 * 写屏障（记录黑→白引用）
 */
void IncrementalMarking::RecordWrite(
    HeapObject* host,
    ObjectSlot slot,
    Object* value) {
  
  // 🔥 检查是否在增量标记中
  if (!IsMarking()) {
    return;
  }

  // 🔥 检查 value 是否为堆对象
  if (!value->IsHeapObject()) {
    return;
  }

  HeapObject* target = HeapObject::cast(value);

  // 🔥 检查 host 是否为黑色
  if (!marking_state_->IsBlack(host)) {
    return;  // host 不是黑色，无需处理
  }

  // 🔥 检查 target 是否为白色
  if (marking_state_->IsBlack(target) || marking_state_->IsGrey(target)) {
    return;  // target 已标记，无需处理
  }

  // 🔥 违反不变式！将 target 标记为灰色
  marking_state_->WhiteToGrey(target);
  marking_worklist_->Push(target);
}
```

## 💻 Orinoco GC（2025 最新优化）

### 1. Orinoco 改进

**Orinoco** 是 V8 的 GC 项目名称，引入了多项并发优化：

| 阶段 | 传统 | Orinoco |
|------|------|---------|
| **标记** | 停止世界 | 并发标记 |
| **清除** | 停止世界 | 并发清除 |
| **压缩** | 停止世界 | 并发压缩（部分） |
| **新生代回收** | 串行 | 并行 Scavenge |

### 2. 并发清除

```cpp
// deps/v8/src/heap/concurrent-sweeper.cc

/**
 * 并发清除
 */
class ConcurrentSweeper {
 public:
  void ScheduleTasks() {
    // 🔥 为每个空间启动清除任务
    for (PagedSpace* space : spaces_to_sweep_) {
      auto task = std::make_unique<SweeperTask>(space);
      V8::GetCurrentPlatform()->CallOnWorkerThread(task.get());
    }
  }

  void SweepPage(Page* page) {
    // 🔥 清除页中的死对象
    Address current = page->area_start();
    Address limit = page->area_end();

    while (current < limit) {
      HeapObject* object = HeapObject::FromAddress(current);
      
      if (marking_state_->IsBlack(object)) {
        // 存活对象
        current += object->Size();
      } else {
        // 死对象：添加到空闲列表
        int size = object->Size();
        page->heap()->CreateFillerObjectAt(current, size);
        current += size;
      }
    }
  }
};
```

```cpp
// deps/v8/src/heap/scavenger.cc

/**
 * 并行 Scavenge（新生代 GC）
 */
class ParallelScavenger {
 public:
  void Run() {
    // 🔥 启动多个 Scavenger 任务
    int num_tasks = NumberOfScavengerTasks();

    std::vector<std::unique_ptr<ScavengerTask>> tasks;
    
    for (int i = 0; i < num_tasks; i++) {
      tasks.push_back(std::make_unique<ScavengerTask>(i));
    }

    // 🔥 并行执行
    platform::ExecuteJobs(tasks);
  }

  void ScavengeTask(int task_id) {
    // 🔥 处理根对象子集
    Scavenger scavenger(heap_, task_id);

    // 🔥 从工作队列获取对象
    while (!work_queue_.IsEmpty()) {
      HeapObject* object = work_queue_.Pop();
      scavenger.ScavengeObject(object);
    }
  }
};
```

## 💻 性能优化实战

### 1. 监控 GC 暂停时间

```javascript
const { PerformanceObserver } = require('perf_hooks');

const gcStats = {
  scavenge: { count: 0, totalTime: 0, maxTime: 0 },
  markSweepCompact: { count: 0, totalTime: 0, maxTime: 0 }
};

const obs = new PerformanceObserver((items) => {
  items.getEntries().forEach((entry) => {
    if (entry.entryType === 'gc') {
      const stats = entry.kind === 1 ? gcStats.scavenge : gcStats.markSweepCompact;
      
      stats.count++;
      stats.totalTime += entry.duration;
      stats.maxTime = Math.max(stats.maxTime, entry.duration);
    }
  });
});

obs.observe({ entryTypes: ['gc'] });

// 定期打印统计
setInterval(() => {
  console.log('Scavenge:', gcStats.scavenge);
  console.log('Mark-Sweep-Compact:', gcStats.markSweepCompact);
}, 10000);
```

### 4. 使用 Worker Threads 分担 GC 压力

```javascript
const { Worker } = require('worker_threads');

// 主线程
function distributeWork(tasks) {
  const numWorkers = 4;
  const workers = [];

  for (let i = 0; i < numWorkers; i++) {
    const worker = new Worker('./worker.js');
    workers.push(worker);
    
    // 🔥 每个 Worker 有独立的堆，独立 GC
    worker.postMessage(tasks.slice(i * tasks.length / numWorkers, (i + 1) * tasks.length / numWorkers));
  }

  // 等待所有 Worker 完成
  Promise.all(workers.map(w => new Promise(resolve => w.on('message', resolve))));
}
```

## 📝 总结

### 核心要点

1. **增量标记**：GC 工作分散到多个小步骤，减少单次停顿
2. **并发标记**：后台线程执行标记，与应用并发
3. **写屏障**：维护三色标记不变式，保证并发正确性
4. **Orinoco**：并发清除、并行 Scavenge，全面降低延迟

### 实现关键

1. **IncrementalMarking**：Step() 执行增量标记
2. **ConcurrentMarking**：后台线程并发标记
3. **WriteBarrier**：记录黑→白引用
4. **三色标记**：白（未标记）→ 灰（部分标记）→ 黑（已标记）

### 应用价值

1. **低延迟**：减少 GC 停顿，提升响应速度
2. **高吞吐**：并发 GC 充分利用多核
3. **内存优化**：合理分配，减少 GC 压力
4. **调试技巧**：监控 GC，优化性能
