# 5.2 隐藏类（Hidden Class）与内联缓存（Inline Cache）

> 深入理解 V8 的属性访问优化机制

---

## 📋 核心概念

隐藏类（Hidden Class，也称 Map）是 V8 的核心优化技术，用于**快速访问对象属性**。

**核心问题：**
- 为什么 JavaScript 对象访问这么快？
- 什么是隐藏类（Map）？
- 内联缓存（IC）如何工作？
- 如何编写对隐藏类友好的代码？

**基本原理：**
```javascript
// JavaScript 是动态语言
const obj = {};
obj.x = 1;  // 动态添加属性

// V8 内部：为 obj 创建隐藏类
// 隐藏类记录：属性 x 在偏移量 0
```

### 2. 隐藏类创建过程

```cpp
// deps/v8/src/objects/map.cc

/**
 * 创建新的 Map（添加属性时）
 */
Handle<Map> Map::CopyWithField(
    Isolate* isolate,
    Handle<Map> map,
    Handle<Name> name,
    Handle<FieldType> type,
    PropertyAttributes attributes,
    PropertyConstness constness,
    Representation representation,
    TransitionFlag flag) {
  
  // 🔥 检查是否可以复用现有转换
  Handle<Map> maybe_transition = TransitionsAccessor(isolate, map).SearchTransition(*name, kData, attributes);
  
  if (!maybe_transition.is_null()) {
    return maybe_transition;
  }

  // 🔥 创建新 Map
  Handle<Map> new_map = Map::Copy(isolate, map, "CopyWithField");

  // 🔥 添加属性描述符
  Descriptor d = Descriptor::DataField(name, new_map->NumberOfOwnDescriptors(), attributes, constness, representation, type);
  
  new_map->AppendDescriptor(isolate, &d);

  // 🔥 记录转换
  if (flag == INSERT_TRANSITION) {
    TransitionsAccessor(isolate, map).Insert(name, new_map, PROPERTY_TRANSITION);
  }

  return new_map;
}
```

### 2. 转换树示例

```javascript
// 不同的属性添加顺序 → 不同的 Map
function Point1(x, y) {
  this.x = x;  // Map0 → Map1
  this.y = y;  // Map1 → Map2
}

function Point2(x, y) {
  this.y = y;  // Map0 → Map3
  this.x = x;  // Map3 → Map4
}

// 结果：Map2 ≠ Map4
// 转换树：
//        Map0
//       /    \
//      x      y
//     /        \
//  Map1        Map3
//    |           |
//    y           x
//    |           |
//  Map2        Map4
```

### 3. 避免隐藏类分裂

```javascript
// ❌ 坏：隐藏类分裂
function Point(x, y, z) {
  this.x = x;
  this.y = y;
  if (z) {
    this.z = z;  // 🔥 条件添加属性！
  }
}

const p1 = new Point(1, 2);       // Map1: { x, y }
const p2 = new Point(3, 4, 5);    // Map2: { x, y, z }
// p1 和 p2 的 Map 不同！

// ✅ 好：统一隐藏类
function Point(x, y, z) {
  this.x = x;
  this.y = y;
  this.z = z;  // 🔥 总是初始化
}

const p1 = new Point(1, 2, undefined);  // Map1: { x, y, z }
const p2 = new Point(3, 4, 5);          // Map1: { x, y, z }
// p1 和 p2 共享 Map1！
```

## 💻 Inline Cache（内联缓存）

### 1. IC 状态

```cpp
// deps/v8/src/ic/ic.h

/**
 * InlineCacheState：IC 状态
 */
enum InlineCacheState {
  UNINITIALIZED,  // 🔥 未初始化
  PREMONOMORPHIC, // 预单态
  MONOMORPHIC,    // 🔥 单态（1种类型）
  RECOMPUTE_HANDLER, // 重新计算处理器
  POLYMORPHIC,    // 🔥 多态（2-4种类型）
  MEGAMORPHIC,    // 🔥 超多态（>4种类型）
  GENERIC,        // 通用
};

/**
 * InlineCache 基类
 */
class IC {
 public:
  // 🔥 更新 IC 状态
  void UpdateState(Handle<Object> object, Handle<Object> name);

  // 🔥 获取当前状态
  InlineCacheState state() const;

  // 🔥 加载 IC
  static void Load(Isolate* isolate, Handle<Object> object, Handle<Name> name);

  // 🔥 存储 IC
  static void Store(Isolate* isolate, Handle<Object> object, Handle<Name> name, Handle<Object> value);
};
```

---

### 2. 单态 IC（Monomorphic）

```javascript
// 单态：所有对象具有相同的隐藏类
function getX(obj) {
  return obj.x;  // 🔥 IC 点
}

const p1 = { x: 1, y: 2 };
const p2 = { x: 3, y: 4 };

// 首次调用：UNINITIALIZED
getX(p1);  // IC 记录：Map1, 偏移量 0

// 第二次调用：MONOMORPHIC
getX(p2);  // IC 检查：Map1? ✅ 快速访问偏移量 0

// 伪机器码：
// cmp [obj], Map1      ; 检查 Map
// je fast_path         ; 相同 → 快速路径
// call slow_path       ; 不同 → 慢速路径
// fast_path:
// mov rax, [obj+0]     ; 直接访问偏移量 0
// ret
```

---

### 3. 多态 IC（Polymorphic）

```javascript
// 多态：2-4 种不同的隐藏类
function getX(obj) {
  return obj.x;
}

const p1 = { x: 1, y: 2 };       // Map1
const p2 = { x: 3, z: 4 };       // Map2 (不同顺序)
const p3 = { x: 5, y: 6, z: 7 }; // Map3

getX(p1);  // IC: [Map1 → offset 0]
getX(p2);  // IC: [Map1 → offset 0, Map2 → offset 0]
getX(p3);  // IC: [Map1 → offset 0, Map2 → offset 0, Map3 → offset 0]

// 伪机器码：
// cmp [obj], Map1
// je offset_0
// cmp [obj], Map2
// je offset_0
// cmp [obj], Map3
// je offset_0
// call slow_path
// offset_0:
// mov rax, [obj+0]
// ret
```

---

### 4. 超多态 IC（Megamorphic）

```javascript
// 超多态：> 4 种隐藏类
function getX(obj) {
  return obj.x;
}

// 5+ 种不同的 Map
getX({ x: 1 });
getX({ x: 2, a: 1 });
getX({ x: 3, b: 1 });
getX({ x: 4, c: 1 });
getX({ x: 5, d: 1 });  // 🔥 超多态！

// IC 放弃优化，回退到通用查找
// 伪代码：
// call runtime_get_property(obj, "x")
```

### 5. IC 源码实现

```cpp
// deps/v8/src/ic/ic.cc

/**
 * 加载 IC
 */
MaybeHandle<Object> LoadIC::Load(
    Isolate* isolate,
    Handle<Object> object,
    Handle<Name> name) {
  
  // 🔥 快速路径：检查 IC 缓存
  if (state() == MONOMORPHIC) {
    Handle<Map> receiver_map(HeapObject::cast(*object)->map(), isolate);
    
    if (*receiver_map == *cached_map()) {
      // 🔥 Map 匹配！直接访问
      int offset = cached_offset();
      return handle(HeapObject::cast(*object)->RawFastPropertyAt(offset), isolate);
    }
  }

  // 🔥 慢速路径：通用属性查找
  LookupIterator it(isolate, object, name);
  Handle<Object> result;
  ASSIGN_RETURN_ON_EXCEPTION(isolate, result, Object::GetProperty(&it), Object);

  // 🔥 更新 IC
  UpdateCaches(&it);

  return result;
}

/**
 * 更新 IC 缓存
 */
void LoadIC::UpdateCaches(LookupIterator* lookup) {
  Handle<Object> object = lookup->GetReceiver();
  Handle<Name> name = lookup->name();

  // 🔥 获取对象的 Map
  Handle<Map> map(HeapObject::cast(*object)->map(), isolate());

  // 🔥 计算新状态
  InlineCacheState new_state = ComputeNewState(map);

  switch (new_state) {
    case MONOMORPHIC:
      // 🔥 单态：缓存 Map 和偏移量
      set_cached_map(map);
      set_cached_offset(lookup->GetFieldIndex().offset());
      break;

    case POLYMORPHIC:
      // 🔥 多态：添加到多态列表
      AddToPolymorphicCache(map, lookup->GetFieldIndex().offset());
      break;

    case MEGAMORPHIC:
      // 🔥 超多态：放弃优化
      set_state(MEGAMORPHIC);
      break;
  }
}
```

## 💻 性能优化实战

### 1. 保持隐藏类稳定

```javascript
// ❌ 坏：动态添加属性
class Point {
  constructor(x, y, z) {
    this.x = x;
    this.y = y;
    if (z !== undefined) {
      this.z = z;  // 🔥 条件属性！
    }
  }
}

// ✅ 好：固定属性
class Point {
  constructor(x, y, z) {
    this.x = x;
    this.y = y;
    this.z = z ?? null;  // 🔥 总是初始化
  }
}
```

### 2. 固定属性添加顺序

```javascript
// ❌ 坏：不同顺序
function createPoint1(x, y) {
  return { x, y };  // Map1: x, y
}

function createPoint2(x, y) {
  return { y, x };  // Map2: y, x  (不同！)
}

// ✅ 好：统一顺序

  return { x, y };  // Map1: x, y

function createPoint2(x, y) {
  return { x, y };  // Map1: x, y  (相同！)
```

### 3. 避免删除属性

```javascript
// ❌ 坏：删除属性
```

### 4. 使用类而非对象字面量

```javascript
// ❌ 坏：对象字面量（难以优化）
function createPoints(n) {

    points.push({ x: i, y: i * 2 });
  }
  return points;

    this.y = y;
  }

  const points = [];
  for (let i = 0; i < n; i++) {
    points.push(new Point(i, i * 2));
  }
  return points;
}
```

### 5. 避免超多态


## 📝 总结

### 核心要点

1. **隐藏类（Map）**：V8 为对象创建的内部类型描述
2. **转换树**：通过属性添加顺序构建的 Map 转换图
3. **内联缓存（IC）**：缓存属性访问路径，加速访问
4. **IC 状态**：单态（快）→ 多态（中）→ 超多态（慢）

### 实现关键

1. **Map 结构**：instance_descriptors、transitions、prototype
2. **Transition Tree**：记录 Map 之间的转换关系
3. **IC 机制**：MONOMORPHIC → POLYMORPHIC → MEGAMORPHIC
4. **快速路径**：Map 检查 → 偏移量访问

### 应用价值

1. **性能优化**：保持隐藏类稳定，避免超多态
2. **代码设计**：使用类、固定属性顺序
3. **调试技巧**：使用 --trace-ic 分析
4. **架构指导**：设计对 V8 友好的数据结构
