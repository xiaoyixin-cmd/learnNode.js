# 1.4 JavaScript 与 C++ 通信桥梁：N-API 与 Internal Binding

> 深入理解 Node.js 跨语言通信机制

---

## 📋 知识点概述

### 核心概念

Node.js 的强大之处在于 JavaScript 和 C++ 的无缝集成，这依赖于完善的跨语言通信机制。

**通信方式：**
1. **Internal Binding** - Node.js 内部使用
2. **N-API (Node-API)** - 用户扩展标准接口
3. **V8 API** - 直接使用 V8 引擎 API

**通信流程：**
```
JavaScript                    C++
    ↓                          ↑
require('module')       NODE_MODULE_CONTEXT_AWARE
    ↓                          ↑
process.binding()       GetLinkedBinding()
    ↓                          ↑
V8::FunctionTemplate    C++ Implementation
    ↓                          ↑
调用 C++ 函数            执行并返回结果
```

## 💻 Internal Binding 机制

### 1. C++ 侧：注册内置模块

```cpp
// src/node_file.cc（文件系统模块示例）
#include "node.h"
#include "v8.h"
#include "uv.h"

namespace node {
namespace fs {

using v8::Context;
using v8::FunctionCallbackInfo;
using v8::FunctionTemplate;
using v8::Isolate;
using v8::Local;
using v8::Object;
using v8::String;
using v8::Value;

// C++ 函数：读取文件
void ReadFile(const FunctionCallbackInfo<Value>& args) {
  Isolate* isolate = args.GetIsolate();
  Environment* env = Environment::GetCurrent(args);
  
  // 1. 获取参数
  if (args.Length() < 2) {
    return THROW_ERR_MISSING_ARGS(env, "path and callback are required");
  }
  
  // 文件路径
  BufferValue path(isolate, args[0]);
  if (!path.length()) {
    return THROW_ERR_INVALID_ARG_TYPE(env, "path must be a string");
  }
  
  // 回调函数
  Local<Value> callback = args[1];
  if (!callback->IsFunction()) {
    return THROW_ERR_INVALID_ARG_TYPE(env, "callback must be a function");
  }
  
  // 2. 创建异步请求
  FSReqCallback* req_wrap = new FSReqCallback(env, callback.As<Object>());
  
  // 3. 提交到 libuv 线程池
  int err = uv_fs_open(
    env->event_loop(),
    req_wrap->req(),
    *path,
    O_RDONLY,
    0,
    AfterOpen  // 完成回调
  );
  
  if (err < 0) {
    delete req_wrap;
    return args.GetReturnValue().Set(err);
  }
  
  req_wrap->Dispatched();
}

// 模块初始化函数
void Initialize(
  Local<Object> exports,
  Local<Value> unused,
  Local<Context> context,
  void* priv
) {
  Isolate* isolate = context->GetIsolate();
  Environment* env = Environment::GetCurrent(context);
  
  // 导出 readFile 函数
  env->SetMethod(exports, "readFile", ReadFile);
  
  // 导出更多函数...
  env->SetMethod(exports, "writeFile", WriteFile);
  env->SetMethod(exports, "stat", Stat);
  env->SetMethod(exports, "mkdir", MkDir);
}

} // namespace fs
} // namespace node

// 注册模块
NODE_MODULE_CONTEXT_AWARE_INTERNAL(fs, node::fs::Initialize)
```

**关键宏定义：**
```cpp
// src/node_binding.h
#define NODE_MODULE_CONTEXT_AWARE_INTERNAL(modname, regfunc)  \
  NODE_MODULE_CONTEXT_AWARE_CPP(modname, regfunc, nullptr, NM_F_INTERNAL)

#define NODE_MODULE_CONTEXT_AWARE_CPP(modname, regfunc, priv, flags)  \
  static node::node_module _module = {                                \
    NODE_MODULE_VERSION,                                              \
    flags,                                                            \
    nullptr,                                                          \
    __FILE__,                                                         \
    nullptr,                                                          \
    (node::addon_context_register_func)(regfunc),                    \
    NODE_STRINGIFY(modname),                                          \
    priv,                                                             \
    nullptr                                                           \
  };                                                                  \
  void _register_##modname() {                                        \
    node_module_register(&_module);                                   \
  }
```

```javascript
// lib/fs.js
'use strict';

const { 
  readFile: _readFile 
} = internalBinding('fs');

function readFile(path, options, callback) {
  // 参数处理
  callback = maybeCallback(callback || options);
  options = getOptions(options, { flag: 'r' });
  
  // 调用 C++ binding
  _readFile(
    pathModule.toNamespacedPath(path),
    options.flag || 'r',
    options.encoding,
    (err, data) => {
      if (err) {
        return callback(err);
      }
      callback(null, data);
    }
  );
}
```

### 3. Binding 注册机制

```cpp
// src/node_binding.cc
namespace node {

// 内置模块注册表
typedef std::map<std::string, node_module*> BindingMap;
static BindingMap* binding_map = nullptr;

// 注册模块
void node_module_register(void* m) {
  struct node_module* mp = reinterpret_cast<struct node_module*>(m);
  
  if (binding_map == nullptr) {
    binding_map = new BindingMap();
  }
  
  (*binding_map)[mp->nm_modname] = mp;
}

// 获取模块
node_module* get_linked_module(const char* name) {
  if (binding_map == nullptr) {
    return nullptr;
  }
  
  BindingMap::iterator it = binding_map->find(name);
  if (it == binding_map->end()) {
    return nullptr;
  }
  
  return it->second;
}

// internalBinding() 实现
void GetInternalBinding(
  const FunctionCallbackInfo<Value>& args
) {
  Environment* env = Environment::GetCurrent(args);
  Isolate* isolate = env->isolate();
  
  // 获取模块名
  Local<String> module = args[0].As<String>();
  Utf8Value module_v(isolate, module);
  
  // 查找模块
  node_module* mod = get_linked_module(*module_v);
  
  if (mod == nullptr) {
    char errmsg[1024];
    snprintf(errmsg, sizeof(errmsg), 
      "No such module: %s", *module_v);
    return THROW_ERR_INVALID_MODULE(env, errmsg);
  }
  
  // 检查权限
  if (!(mod->nm_flags & NM_F_INTERNAL)) {
    return THROW_ERR_INVALID_MODULE(env, 
      "Cannot access internal module");
  }
  
  // 初始化模块
  Local<Object> exports = Object::New(isolate);
  mod->nm_context_register_func(
    exports,
    Undefined(isolate),
    env->context(),
    mod->nm_priv
  );
  
  // 返回 exports
  args.GetReturnValue().Set(exports);
}

} // namespace node
```

## 💻 N-API (Node-API) 标准接口

### 1. N-API 概述

N-API 是 Node.js 提供的稳定的 ABI（Application Binary Interface），允许原生模块在不同 Node.js 版本间兼容。

**优势：**
- **ABI 稳定**：跨 Node.js 版本兼容
- **C 接口**：不依赖 C++ 和 V8
- **官方支持**：Node.js 核心团队维护

**架构：**
```
User Addon (C/C++)
    ↓
N-API Layer (C Interface)
    ↓
V8 Implementation
    ↓
JavaScript Runtime
```

### 2. N-API 示例：Hello World

```c
// hello.c
#include <node_api.h>
#include <string.h>

// C 函数
napi_value Hello(napi_env env, napi_callback_info info) {
  napi_value greeting;
  
  // 创建 JavaScript 字符串
  napi_status status = napi_create_string_utf8(
    env,
    "Hello from C!",
    NAPI_AUTO_LENGTH,
    &greeting
  );
  
  if (status != napi_ok) {
    return NULL;
  }
  
  return greeting;
}

// 模块初始化
napi_value Init(napi_env env, napi_value exports) {
  napi_value fn;
  
  // 创建函数
  napi_status status = napi_create_function(
    env,
    NULL,
    0,
    Hello,
    NULL,
    &fn
  );
  
  if (status != napi_ok) {
    return NULL;
  }
  
  // 导出函数
  status = napi_set_named_property(env, exports, "hello", fn);
  if (status != napi_ok) {
    return NULL;
  }
  
  return exports;
}

// 注册模块
NAPI_MODULE(NODE_GYP_MODULE_NAME, Init)
```

```python

    {
      "target_name": "hello",
      "sources": [ "hello.c" ]
    }
  ]
}
```

**使用：**

### 4. N-API 异步示例

```c
// async.c
#include <node_api.h>
#include <unistd.h>

typedef struct {
  napi_async_work work;
  napi_deferred deferred;
  int input;
  int result;
} AddonData;

// 后台执行（在线程池）
void Execute(napi_env env, void* data) {
  AddonData* addon_data = (AddonData*)data;
  
  // 模拟耗时操作
  sleep(2);
  
  // 计算结果
  addon_data->result = addon_data->input * 2;
}

// 完成回调（在主线程）
void Complete(napi_env env, napi_status status, void* data) {
  AddonData* addon_data = (AddonData*)data;
  
  napi_value result;
  napi_create_int32(env, addon_data->result, &result);
  
  // Resolve Promise
  napi_resolve_deferred(env, addon_data->deferred, result);
  
  // 清理
  napi_delete_async_work(env, addon_data->work);
  free(addon_data);
}

// JavaScript 调用的函数
napi_value AsyncDouble(napi_env env, napi_callback_info info) {
  size_t argc = 1;
  napi_value args[1];
  napi_get_cb_info(env, info, &argc, args, NULL, NULL);
  
  // 获取输入
  int input;
  napi_get_value_int32(env, args[0], &input);
  
  // 创建 Promise
  napi_value promise;
  napi_deferred deferred;
  napi_create_promise(env, &deferred, &promise);
  
  // 准备数据
  AddonData* addon_data = malloc(sizeof(AddonData));
  addon_data->deferred = deferred;
  addon_data->input = input;
  
  // 创建异步工作
  napi_value resource_name;
  napi_create_string_utf8(env, "AsyncDouble", NAPI_AUTO_LENGTH, &resource_name);
  
  napi_create_async_work(
    env,
    NULL,
    resource_name,
    Execute,
    Complete,
    addon_data,
    &addon_data->work
  );
  
  // 提交到队列
  napi_queue_async_work(env, addon_data->work);
  
  return promise;
}

// 模块初始化
napi_value Init(napi_env env, napi_value exports) {
  napi_value fn;
  napi_create_function(env, NULL, 0, AsyncDouble, NULL, &fn);
  napi_set_named_property(env, exports, "asyncDouble", fn);
  return exports;
}

NAPI_MODULE(NODE_GYP_MODULE_NAME, Init)
```

### 1. V8 类型系统

```cpp
// src/addon.cc
#include <node.h>
#include <v8.h>

namespace demo {

using v8::Context;
using v8::FunctionCallbackInfo;
using v8::Isolate;
using v8::Local;
using v8::Object;
using v8::String;
using v8::Value;
using v8::Number;
using v8::Array;

// V8 值类型
void TypeDemo(const FunctionCallbackInfo<Value>& args) {
  Isolate* isolate = args.GetIsolate();
  Local<Context> context = isolate->GetCurrentContext();
  
  // 1. String
  Local<String> str = String::NewFromUtf8(
    isolate, 
    "Hello V8"
  ).ToLocalChecked();
  
  // 2. Number
  Local<Number> num = Number::New(isolate, 42);
  
  // 3. Object
  Local<Object> obj = Object::New(isolate);
  obj->Set(
    context,
    String::NewFromUtf8(isolate, "key").ToLocalChecked(),
    String::NewFromUtf8(isolate, "value").ToLocalChecked()
  ).Check();
  
  // 4. Array
  Local<Array> arr = Array::New(isolate, 3);
  arr->Set(context, 0, Number::New(isolate, 1)).Check();
  arr->Set(context, 1, Number::New(isolate, 2)).Check();
  arr->Set(context, 2, Number::New(isolate, 3)).Check();
  
  // 返回对象
  args.GetReturnValue().Set(obj);
}

void Init(Local<Object> exports) {
  NODE_SET_METHOD(exports, "typeDemo", TypeDemo);
}

NODE_MODULE(NODE_GYP_MODULE_NAME, Init)

} // namespace demo
```

### 2. V8 函数模板

```cpp
// function_template.cc
#include <node.h>
#include <v8.h>

namespace demo {

using v8::Context;
using v8::Function;
using v8::FunctionCallback;
using v8::FunctionCallbackInfo;
using v8::FunctionTemplate;
using v8::Isolate;
using v8::Local;
using v8::Object;
using v8::String;
using v8::Value;

// JavaScript 调用的 C++ 函数
void Add(const FunctionCallbackInfo<Value>& args) {
  Isolate* isolate = args.GetIsolate();
  
  if (args.Length() < 2) {
    isolate->ThrowException(String::NewFromUtf8(
      isolate, "Wrong number of arguments"
    ).ToLocalChecked());
    return;
  }
  
  double value = args[0].As<Number>()->Value() + 
                 args[1].As<Number>()->Value();
  
  args.GetReturnValue().Set(Number::New(isolate, value));
}

void Init(Local<Object> exports) {
  Isolate* isolate = exports->GetIsolate();
  Local<Context> context = isolate->GetCurrentContext();
  
  // 创建函数模板
  Local<FunctionTemplate> tpl = FunctionTemplate::New(isolate, Add);
  
  // 获取函数
  Local<Function> fn = tpl->GetFunction(context).ToLocalChecked();
  
  // 导出
  exports->Set(
    context,
    String::NewFromUtf8(isolate, "add").ToLocalChecked(),
    fn
  ).Check();
}

NODE_MODULE(NODE_GYP_MODULE_NAME, Init)

} // namespace demo
```

## 📝 总结

### 核心要点

1. **Internal Binding**：`NODE_MODULE_CONTEXT_AWARE_INTERNAL` 注册
2. **N-API**：稳定的 ABI，跨版本兼容
3. **V8 API**：直接使用 V8 引擎功能
4. **异步支持**：`napi_async_work` 线程池

### 实现关键

1. **类型转换**：JavaScript ↔ C++ 类型映射
2. **内存管理**：HandleScope、引用计数
3. **错误处理**：异常传递机制
4. **生命周期**：对象创建、使用、销毁

### 应用价值

1. **性能优化**：用 C++ 实现性能关键路径
2. **系统集成**：调用操作系统 API
3. **扩展开发**：开发 Node.js 原生模块
4. **理解原理**：深入 Node.js 内核
