# 1.1 JavaScript 运行时演进史：从浏览器到服务端

> 了解 JavaScript 如何从浏览器脚本语言演变为全栈开发语言

---

## 📚 JavaScript 的诞生（1995）

### 背景故事
- **创始人**：Brendan Eich（网景公司）
- **开发时间**：仅用 10 天设计出原型
- **初衷**：为 Netscape Navigator 浏览器添加简单的交互功能
- **原名**：Mocha → LiveScript → JavaScript（营销策略，蹭 Java 热度）

### 早期的局限性
```
浏览器 JavaScript 的限制：
✗ 只能在浏览器中运行
✗ 无法访问文件系统
✗ 无法访问操作系统 API
✗ 无法进行网络通信（除了浏览器提供的 XMLHttpRequest）
✗ 性能较差，主要用于简单的表单验证和动画效果
```

### 第一个 JavaScript 程序
```html
<!-- 1995年的 JavaScript -->
<html>
<body>
  <script>
    alert('Hello World!');
    document.write('当前时间：' + new Date());
  </script>
</body>
</html>
```

## 🌟 为什么 V8 引擎是转折点？

### V8 引擎的革命性创新

#### 1. JIT（即时编译）技术
```
传统解释器：
源代码 → 逐行解释执行（慢）

V8 的 JIT 编译：
源代码 → 编译为机器码 → 直接执行（快 10 倍+）
```

**性能对比：**
```javascript
// 计算斐波那契数列（第40项）
function fibonacci(n) {
  if (n <= 1) return n;
  return fibonacci(n - 1) + fibonacci(n - 2);
}

// 性能测试
console.time('fibonacci');
const result = fibonacci(40);
console.timeEnd('fibonacci');

// 不同引擎性能对比：
// SpiderMonkey (2008): ~3000ms
// V8 (2008):          ~300ms ⚡
// V8 (2025):          ~30ms  ⚡⚡⚡
```

#### 2. 隐藏类（Hidden Classes）
```javascript
// V8 优化对象访问
class Point {
  constructor(x, y) {
    this.x = x; // 创建隐藏类 C0
    this.y = y; // 转换为隐藏类 C1
  }
}

// 相同结构的对象共享隐藏类，访问速度快
const p1 = new Point(1, 2); // 使用 C1
const p2 = new Point(3, 4); // 复用 C1
```

#### 3. 内联缓存（Inline Caching）
```javascript
// V8 会缓存频繁调用的函数
function add(a, b) {
  return a + b;
}

// 第一次调用：检查类型，执行
add(1, 2);

// 后续调用：直接使用缓存的机器码（快100倍）
add(3, 4);
add(5, 6);
```

#### 4. 垃圾回收优化
```
V8 的分代垃圾回收：

新生代（Young Generation）:
  - 小对象，生命周期短
  - 使用 Scavenge 算法（快速）
  - 垃圾回收耗时 ~1ms

老生代（Old Generation）:
  - 大对象，生命周期长
  - 使用 Mark-Sweep-Compact 算法
  - 增量标记，减少停顿时间
```

## 💡 Node.js 的诞生（2009）

### Ryan Dahl 的创新理念

**创始人**：Ryan Dahl（美国程序员）  
**发布时间**：2009年11月8日  
**初衷**：解决 Web 应用的高并发问题

### 核心设计思想
```
传统服务器（Apache/PHP）：
每个请求 = 一个线程
1000 个并发请求 = 1000 个线程
→ 内存爆炸，性能崩溃

Node.js 设计：
所有请求 = 一个线程 + 事件循环
1000 个并发请求 = 异步 I/O 处理
→ 低内存占用，高性能
```

### Node.js 让 JavaScript 可以做什么？

#### 1. 访问文件系统
```javascript
// Node.js 可以读写文件
const fs = require('fs');

// 读取文件
const content = fs.readFileSync('data.txt', 'utf-8');
console.log(content);

// 写入文件
fs.writeFileSync('output.txt', 'Hello Node.js!');
```

#### 2. 创建 HTTP 服务器
```javascript
// Node.js 可以创建 Web 服务器
const http = require('http');

const server = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'text/plain' });
  res.end('Hello from Node.js Server!');
});

server.listen(3000, () => {
  console.log('Server running at http://localhost:3000');
});
```

#### 3. 访问数据库
```javascript
// Node.js 可以连接数据库
const mysql = require('mysql2');

const connection = mysql.createConnection({
  host: 'localhost',
  user: 'root',
  password: 'password',
  database: 'mydb'
});

connection.query('SELECT * FROM users', (err, results) => {
  console.log(results);
});
```

#### 4. 执行系统命令
```javascript
// Node.js 可以执行命令行
const { exec } = require('child_process');

exec('ls -la', (error, stdout, stderr) => {
  console.log(stdout);
});
```

## 💡 常见问题（FAQ）

**Q1：为什么 V8 引擎这么快？**  
A：主要是 JIT 即时编译技术，将 JavaScript 代码直接编译为机器码执行，而不是逐行解释。

**Q2：Node.js 和浏览器 JavaScript 有什么区别？**  
A：语法相同，但 API 不同。浏览器有 DOM/BOM API，Node.js 有文件系统、网络等系统 API。

**Q3：为什么 JavaScript 能成为全栈语言？**  
A：V8 引擎 + Node.js 让 JavaScript 可以运行在服务端，前后端使用同一种语言，降低学习成本。
