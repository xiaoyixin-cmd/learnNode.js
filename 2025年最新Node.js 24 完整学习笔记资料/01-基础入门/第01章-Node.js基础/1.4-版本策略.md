## 🏷️ 版本类型详解

### LTS（Long Term Support）长期支持版本

#### 特点

```
发布规律：
└─ 每年 4 月发布偶数版本
   ├─ v18 (2022.04)
   ├─ v20 (2023.04)
   ├─ v22 (2024.04)
   └─ v24 (2025.04 预计)

支持周期：30 个月
├─ Active LTS：18 个月
│  └─ 新功能、Bug 修复、安全更新
│
└─ Maintenance：12 个月
   └─ 仅重要 Bug 和安全更新

稳定性：⭐⭐⭐⭐⭐
└─ 生产环境首选
```

#### LTS 版本代号

```
Node.js LTS 版本使用元素周期表命名：

v18 LTS - Hydrogen (氢)
v20 LTS - Iron (铁)
v22 LTS - ???? (待公布)
v24 LTS - ???? (待公布)
```

#### LTS 适合人群

```
✅ 生产环境部署
✅ 企业级应用
✅ 长期维护项目
✅ 追求稳定性
✅ 不想频繁升级
```

### Current 版本

#### 特点

```
发布规律：
└─ 每年 10 月发布奇数版本
   ├─ v19 (2022.10)
   ├─ v21 (2023.10)
   ├─ v23 (2024.10)
   └─ v25 (2025.10 预计)

支持周期：6-8 个月
└─ 包含最新实验性特性
└─ 可能有 Breaking Changes
└─ 到期后停止维护

稳定性：⭐⭐⭐
└─ 不适合生产环境
```

#### Current 适合人群

```
✅ 开发者尝鲜
✅ 测试新特性
✅ 开源项目探索
```

## 🔄 版本升级指南

### 升级前准备

```bash
# 1. 查看当前版本
node -v

# 2. 查看项目使用的包
npm list --depth=0

# 3. 查看哪些包可能有问题
npm outdated

# 4. 备份项目
git checkout -b upgrade-node-22

# 5. 查看 Node.js 更新日志
# https://github.com/nodejs/node/blob/main/doc/changelogs/CHANGELOG_V22.md
```

### 升级步骤

```bash
# 1. 安装新版本
nvm install 22
nvm use 22

# 2. 重新安装依赖
rm -rf node_modules package-lock.json
npm install

# 3. 运行测试
npm test

# 4. 检查警告和弃用信息
node --trace-warnings app.js

# 5. 本地测试
npm start

# 6. 部署到测试环境
# ... 测试完整功能 ...

# 7. 逐步灰度发布到生产环境
```

### 常见升级问题

#### 问题1：某些包不兼容

```bash
# 解决方案1：更新包
npm update package-name

# 解决方案2：使用兼容版本
npm install package-name@compatible-version

# 解决方案3：寻找替代包
npm search alternative-package
```

#### 问题2：Breaking Changes

```javascript
// 例如：Node.js 22 移除了某些废弃 API

// ❌ 旧代码
const { _extend } = require('util');
const obj = _extend({}, { a: 1 });

// ✅ 新代码
const obj = Object.assign({}, { a: 1 });
// 或
const obj = { ...{ a: 1 } };
```

## 💡 常见问题（FAQ）

**Q1：一定要用 LTS 版本吗？**  
A：生产环境强烈建议使用 LTS。开发环境和个人项目可以使用 Current 体验新特性。

**Q2：应该什么时候升级 Node.js？**  
A：
- LTS 版本进入 Maintenance 阶段时（18 个月后）
- 当前版本即将停止支持时
- 新 LTS 版本发布 3-6 个月后（稳定性验证期）

**Q3：升级 Node.js 会破坏现有项目吗？**  
A：可能会。建议：
1. 先在开发环境测试
2. 查看更新日志中的 Breaking Changes
3. 运行完整测试套件
4. 在生产环境灰度发布

**Q4：如何知道某个包支持哪些 Node.js 版本？**  
A：查看包的 `package.json` 中的 `engines` 字段：
```json
{
  "engines": {
    "node": ">=18.0.0"
  }
}
```
