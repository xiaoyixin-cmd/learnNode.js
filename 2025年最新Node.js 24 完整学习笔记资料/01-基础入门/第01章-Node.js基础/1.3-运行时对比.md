## 🎯 核心特性对比

### 1. 包管理

```
Node.js：
├─ npm (默认，最大生态)
├─ pnpm (快速，节省空间)
└─ yarn (成熟，企业级)

Deno：
├─ 内置包管理器
├─ 支持 npm 包（兼容模式）
└─ URL 导入（去中心化）

Bun：
├─ 内置包管理器
├─ 完全兼容 npm
└─ 比 npm 快 20+ 倍
```

**实战对比：**
```bash
# Node.js (npm)
npm install express
# 耗时：~5秒，下载 52个包

# Deno
import express from "npm:express";
# 首次：~3秒，缓存后秒开

# Bun
bun add express
# 耗时：~0.2秒 ⚡
```

### 2. 模块系统

```javascript
// ===== Node.js =====
// CommonJS (传统)
const fs = require('fs');
module.exports = { foo };

// ES Modules (现代，需配置 type: "module")
import fs from 'fs';
export { foo };

// ===== Deno =====
// 仅支持 ES Modules
import { serve } from "https://deno.land/std/http/server.ts";
import express from "npm:express"; // 兼容 npm

// ===== Bun =====
// 两种都支持（自动识别）
import fs from 'fs';
const path = require('path'); // 两种混用也可以
```

### 3. TypeScript 支持

```typescript
// ===== Node.js =====
// 需要额外配置
// 1. 安装依赖
npm install -D typescript ts-node

// 2. 创建 tsconfig.json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "commonjs"
  }
}

// 3. 运行
ts-node app.ts

// ===== Deno =====
// 原生支持，零配置
deno run app.ts

// ===== Bun =====
// 原生支持，零配置
bun run app.ts
```

### 4. 权限模型

```bash
# ===== Node.js =====
# 无权限控制，完全信任代码
node app.js

# ===== Deno =====
# 默认沙箱，需要显式授权
deno run app.js                    # ❌ 无权限
deno run --allow-read app.js       # ✅ 允许读取文件
deno run --allow-net app.js        # ✅ 允许网络访问
deno run --allow-all app.js        # ✅ 允许所有权限

# ===== Bun =====
# 无权限控制
bun run app.js
```

### 5. Web API 兼容性

```javascript
// ===== Node.js v22 =====
// 部分支持 Web API
fetch('https://api.example.com/data')    // ✅ 原生支持
new WebSocket('ws://localhost:8080')     // ✅ 原生支持
new ReadableStream()                     // ✅ 原生支持
navigator.userAgent                      // ❌ 不支持

// ===== Deno =====
// 完全遵循 Web 标准
fetch()           // ✅
WebSocket         // ✅
ReadableStream    // ✅
navigator         // ✅
localStorage      // ✅ (实验性)

// ===== Bun =====
// 完全兼容 Web API
fetch()           // ✅
WebSocket         // ✅
ReadableStream    // ✅
navigator         // ✅
```

## ⚡ 性能对比（2025 最新数据）

### 启动速度

```bash
# 测试：运行 console.log('Hello')

Node.js:  ~50ms
Deno:     ~30ms   ⚡
Bun:      ~5ms    ⚡⚡⚡ (快 10 倍)
```

### HTTP 服务器性能（请求/秒）

```
基准测试：处理 10万个 Hello World 请求

Bun:      ~260,000 req/s  ⚡⚡⚡⚡⚡
Node.js:  ~50,000 req/s   ⚡⚡⚡
Deno:     ~45,000 req/s   ⚡⚡⚡
```

### 文件 I/O 性能

```javascript
// 测试：读取 1000 个文件

const files = Array.from({ length: 1000 }, (_, i) => `file${i}.txt`);

// Node.js
console.time('Node.js');
for (const file of files) {
  await fs.promises.readFile(file);
}
console.timeEnd('Node.js');
// 结果：~850ms

// Deno
console.time('Deno');
for (const file of files) {
  await Deno.readTextFile(file);
}
console.timeEnd('Deno');
// 结果：~900ms

// Bun
console.time('Bun');
for (const file of files) {
  await Bun.file(file).text();
}
console.timeEnd('Bun');
// 结果：~320ms ⚡ (快 2.7 倍)
```

## 💡 常见问题（FAQ）

**Q1：Bun 这么快，为什么不直接用 Bun？**  
A：Bun 还很年轻（2021年），生态和稳定性不如 Node.js。生产环境建议等待 1-2 年再考虑。

**Q2：Deno 是 Ryan Dahl 的新作品，会取代 Node.js 吗？**  
A：短期内不会。Node.js 生态太强大，Deno 更像是补充而非替代。两者会长期共存。

**Q3：我应该从哪个开始学？**  
A：建议从 Node.js 开始，因为：
- 生态最大，学习资源最多
- 企业需求最高，就业机会最多
- 掌握 Node.js 后，学习 Deno/Bun 会很容易

**Q4：三个运行时的代码能互相迁移吗？**  
A：大部分可以，但需要注意：
- Node.js → Deno：需要改用 Deno API
- Node.js → Bun：大部分兼容
- Deno/Bun → Node.js：需要改用 Node.js API
