# 10.2 依赖安全审计

> 检测和修复依赖中的安全漏洞

---

## 🎯 为什么依赖安全重要？

### 核心概念

```
依赖安全 = 确保使用的npm包没有已知漏洞

常见安全问题：
• 已知漏洞（CVE）
• 恶意代码注入
• 供应链攻击
• 过时的依赖
• 许可证问题

真实案例：
• event-stream 事件（2018）- 窃取比特币
• ua-parser-js 事件（2021）- 挖矿木马
• colors/faker 事件（2022）- 作者恶意破坏

后果：
❌ 数据泄露
❌ 服务中断
❌ 恶意代码执行
❌ 声誉损失
```

## 📦 npm audit - 内置审计

### 1. 基础使用

```bash
# ===== 审计依赖 =====

npm audit

# 输出示例：
# found 3 vulnerabilities (1 moderate, 2 high)
# 
# High - Prototype Pollution
# Package: lodash
# Dependency of: express
# Path: express > lodash
# More info: https://npmjs.com/advisories/1523

# ===== 查看详细信息 =====

npm audit --json
npm audit --parseable

# ===== 自动修复 =====

npm audit fix

# 强制修复（可能破坏性更新）
npm audit fix --force

# 仅修复生产依赖
npm audit fix --production

# ===== 审计报告 =====

npm audit --audit-level=moderate  # 只显示moderate以上
npm audit --audit-level=high      # 只显示high以上
```

### 2. package.json 配置

```json
{
  "scripts": {
    "audit": "npm audit",
    "audit:fix": "npm audit fix",
    "audit:check": "npm audit --audit-level=high"
  }
}
```

### 2. GitHub 集成

```yaml
# ===== .github/workflows/security.yml =====

name: Security Check

on: [push, pull_request]

jobs:
  security:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Run Snyk
        uses: snyk/actions/node@master
        env:
          SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
        with:
          args: --severity-threshold=high
```

## 🎯 实战案例

### 案例1：完整的安全审计流程

```javascript
// ===== scripts/security-audit.js =====

import { execSync } from 'child_process';
import chalk from 'chalk';

console.log(chalk.cyan('\n🔒 开始安全审计...\n'));

// 1. npm audit
console.log(chalk.yellow('步骤1: npm audit'));
try {
  execSync('npm audit --audit-level=moderate', { stdio: 'inherit' });
  console.log(chalk.green('✅ npm audit 通过\n'));
} catch (error) {
  console.log(chalk.red('❌ 发现安全漏洞\n'));
  
  // 尝试自动修复
  console.log(chalk.yellow('尝试自动修复...'));
  try {
    execSync('npm audit fix', { stdio: 'inherit' });
    console.log(chalk.green('✅ 已修复\n'));
  } catch (fixError) {
    console.log(chalk.red('❌ 无法自动修复，需要手动处理\n'));
  }
}

// 2. 检查过时的包
console.log(chalk.yellow('步骤2: 检查过时的包'));
try {
  execSync('npm outdated', { stdio: 'inherit' });
} catch (error) {
  // npm outdated 在有过时包时会返回非0退出码
}

// 3. Snyk测试（如果已安装）
console.log(chalk.yellow('\n步骤3: Snyk测试'));
try {
  execSync('snyk test', { stdio: 'inherit' });
  console.log(chalk.green('✅ Snyk测试通过\n'));
} catch (error) {
  console.log(chalk.red('❌ Snyk发现问题（或未安装Snyk）\n'));
}

console.log(chalk.cyan('安全审计完成！\n'));
```

```json
// ===== package.json =====

{
  "scripts": {
    "security": "node scripts/security-audit.js",
    "pre-commit": "npm run security"
  }
}
```

```json
// ===== package.json - 锁定版本 =====

{
  "dependencies": {
    // ❌ 不推荐：允许任意升级

    "lodash": "^4.17.0",

    "express": "4.18.2",
    "lodash": "4.17.21"
  },
  "devDependencies": {
    // 开发依赖可以稍微宽松
    "vitest": "^0.34.0"
  }
}
```

```bash
# ===== 使用 package-lock.json =====

# 安装依赖时使用确切版本
npm ci  # 而不是 npm install

# 更新依赖
npm update lodash  # 更新特定包
npm outdated      # 查看可更新的包
```

### 案例3：自定义安全策略

```javascript
// ===== scripts/check-dependencies.js =====

import fs from 'fs-extra';
import semver from 'semver';

// 读取 package.json
const pkg = await fs.readJson('package.json');

// 黑名单（已知有问题的包）
const blacklist = [
  'event-stream@3.3.6',  // 已知恶意版本
  'colors@1.4.1',         // 作者恶意破坏
  'faker@5.5.4'           // 作者恶意破坏
];

// 检查依赖
function checkDependencies(deps) {
  const issues = [];
  
  for (const [name, version] of Object.entries(deps)) {
    // 检查黑名单
    const fullName = `${name}@${version}`;
    if (blacklist.includes(fullName)) {
      issues.push({
        type: 'blacklist',
        package: name,
        version,
        message: '此版本已知存在安全问题'
      });
    }
    
    // 检查通配符
    if (version === '*' || version === 'latest') {
      issues.push({
        type: 'wildcard',
        package: name,
        version,
        message: '不应使用通配符版本'
      });
    }
    
    // 检查过旧的版本
    const match = version.match(/[\d.]+/);
    if (match && semver.valid(match[0])) {
      const age = getPackageAge(name, match[0]);
      if (age > 365) {  // 超过1年
        issues.push({
          type: 'outdated',
          package: name,
          version,
          message: `此版本已过时（${Math.floor(age / 30)}个月前）`
        });
      }
    }
  }
  
  return issues;
}

// 检查所有依赖
const allIssues = [
  ...checkDependencies(pkg.dependencies || {}),
  ...checkDependencies(pkg.devDependencies || {})
];

// 输出结果
if (allIssues.length === 0) {
  console.log('✅ 依赖检查通过');
} else {
  console.log('❌ 发现以下问题：\n');
  allIssues.forEach(issue => {
    console.log(`${issue.package}@${issue.version}`);
    console.log(`  类型: ${issue.type}`);
    console.log(`  说明: ${issue.message}\n`);
  });
  process.exit(1);
}
```

## 💡 最佳实践

```bash
# ✅ 1. 定期审计
npm audit  # 至少每周一次

# ✅ 2. 锁定版本
# 使用 package-lock.json
npm ci  # 不要用 npm install

# ✅ 3. 最小化依赖
# 只安装真正需要的包
npm uninstall unused-package

# ✅ 4. 审查新依赖
# 安装前检查：
# • npm页面
# • GitHub星标
# • 最近更新时间
# • 维护者信誉
npm view package-name

# ✅ 5. CI中运行审计
# 在CI pipeline中添加安全检查

# ✅ 6. 使用.nvmrc锁定Node版本
echo "20.10.0" > .nvmrc

# ✅ 7. 监控依赖更新
npm outdated  # 定期检查

# ✅ 8. 自动化更新
# 使用 Dependabot 或 Renovate
```
