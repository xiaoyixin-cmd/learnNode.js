# 2.1 安装 Node.js 的三种方式对比

> 选择最适合你的 Node.js 安装方式

---

## 📦 方式一：官网安装包（推荐新手）

### 安装步骤

#### 1. 访问官网
```
官方网站：https://nodejs.org/

中文网站：https://nodejs.cn/
```

#### 2. 选择版本

```
首页会显示两个版本：

┌─────────────────────────────┐
│  v22.x.x LTS (推荐)         │
│  长期支持版，生产环境首选    │
└─────────────────────────────┘

┌─────────────────────────────┐
│  v23.x.x Current            │
│  最新特性，实验性功能        │
└─────────────────────────────┘
```

#### 3. 下载安装包

**Windows：**
```
下载 node-v22.x.x-x64.msi
双击安装，一路 Next
```

**macOS：**
```
下载 node-v22.x.x.pkg
双击安装，按提示操作
```

**Linux：**
```bash
# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

# CentOS/RHEL
curl -fsSL https://rpm.nodesource.com/setup_22.x | sudo bash -
sudo yum install -y nodejs

# 通用方式（下载二进制包）
wget https://nodejs.org/dist/v22.0.0/node-v22.0.0-linux-x64.tar.xz
tar -xJvf node-v22.0.0-linux-x64.tar.xz
sudo mv node-v22.0.0-linux-x64 /usr/local/node
echo 'export PATH=/usr/local/node/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

#### 4. 验证安装

```bash
# 查看 Node.js 版本
node -v
# 输出：v22.x.x

# 查看 npm 版本
npm -v
# 输出：10.x.x

# 测试运行
node -e "console.log('Hello Node.js!')"
# 输出：Hello Node.js!
```

### 优点与缺点

| 优点 | 缺点 |
|------|------|
| ✅ 安装简单，一键完成 | ❌ 只能安装一个版本 |
| ✅ 自动配置环境变量 | ❌ 切换版本需要重新安装 |
| ✅ 适合新手快速上手 | ❌ 不适合管理多个项目 |
| ✅ 稳定可靠 | ❌ 升级需要卸载重装 |

### 适用场景

```
✅ 第一次接触 Node.js
✅ 只维护一个项目
✅ 追求稳定，不频繁升级
✅ 不需要测试不同版本
```

## 🔄 方式二：nvm（Node Version Manager）推荐

### nvm 是什么？

```
nvm = Node Version Manager
├─ 管理多个 Node.js 版本
├─ 快速切换版本
├─ 项目独立版本配置
└─ 团队版本统一
```

### 安装 nvm

#### macOS / Linux

```bash
# 方式1：使用 curl 安装
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 方式2：使用 wget 安装
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 配置环境变量（安装脚本会自动添加，如未生效手动执行）
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"
[ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"

# 重新加载配置
source ~/.bashrc  # 或 source ~/.zshrc

# 验证安装
nvm --version
# 输出：0.39.7
```

#### Windows

```bash
# 方式1：使用 nvm-windows（推荐）
# 1. 访问：https://github.com/coreybutler/nvm-windows/releases
# 2. 下载：nvm-setup.exe
# 3. 双击安装

# 方式2：使用 Chocolatey
choco install nvm

# 方式3：使用 Scoop
scoop install nvm

# 验证安装
nvm version
# 输出：1.1.12
```

### nvm 常用命令

```bash
# ===== 查看版本 =====

# 查看所有可安装的版本
nvm ls-remote
# 输出：大量版本列表

# 查看 LTS 版本
nvm ls-remote --lts

# 查看已安装的版本
nvm ls
# 输出：
# ->     v22.0.0
#        v20.0.0
#        v18.0.0

# ===== 安装版本 =====

# 安装最新 LTS 版本
nvm install --lts
# 输出：Installing latest LTS version...

# 安装最新版本
nvm install node

# 安装指定版本
nvm install 22.0.0
nvm install 20.0.0

# 安装指定 LTS 代号
nvm install lts/iron  # Node.js 20

# ===== 使用版本 =====

# 切换到指定版本
nvm use 22.0.0
# 输出：Now using node v22.0.0

# 使用最新 LTS
nvm use --lts

# 使用 .nvmrc 指定的版本
nvm use

# ===== 设置默认版本 =====

# 设置默认版本
nvm alias default 22.0.0
# 输出：default -> 22.0.0

# 设置 LTS 为默认
nvm alias default lts/iron

# ===== 查看当前版本 =====

# 查看当前使用的版本
nvm current
# 输出：v22.0.0

# 查看指定版本的安装路径
nvm which 22.0.0
# 输出：/Users/username/.nvm/versions/node/v22.0.0/bin/node

# ===== 卸载版本 =====

# 卸载指定版本
nvm uninstall 20.0.0
# 输出：Uninstalled node v20.0.0

# 卸载多个版本
nvm uninstall 18.0.0 20.0.0

# ===== 其他命令 =====

# 运行指定版本的 node
nvm run 22.0.0 app.js
# 等同于：nvm use 22.0.0 && node app.js

# 执行指定版本的命令
nvm exec 22.0.0 node -v

# 查看帮助
nvm --help
```

### 优点与缺点

| 优点 | 缺点 |
|------|------|
| ✅ 管理多个版本 | ❌ 安装稍复杂（需要脚本） |
| ✅ 快速切换版本 | ❌ Windows 需要单独版本 |
| ✅ 项目独立配置 | ❌ 需要学习命令 |
| ✅ 团队版本统一 | ❌ 占用磁盘空间较多 |
| ✅ 社区成熟 | |

### 适用场景

```
✅ 维护多个项目
✅ 需要测试不同版本
✅ 团队协作开发
✅ 需要频繁切换版本
✅ 前端+后端开发者
```

## ⚡ 方式三：fnm（Fast Node Manager）新选择

### fnm 是什么？

```
fnm = Fast Node Manager
├─ 用 Rust 编写
├─ 比 nvm 快 40 倍
├─ 跨平台统一
└─ 配置简单
```

### 安装 fnm

#### macOS / Linux

```bash
# 方式1：使用 curl 安装
curl -fsSL https://fnm.vercel.app/install | bash

# 方式2：使用 Homebrew（macOS）
brew install fnm

# 方式3：使用 Cargo（Rust）
cargo install fnm

# 配置环境变量
# 添加到 ~/.bashrc 或 ~/.zshrc
eval "$(fnm env --use-on-cd)"

# 重新加载
source ~/.bashrc  # 或 source ~/.zshrc

# 验证安装
fnm --version
# 输出：fnm 1.35.1
```

#### Windows

```bash
# 方式1：使用 Chocolatey
choco install fnm

# 方式2：使用 Scoop
scoop install fnm

# 方式3：使用 winget
winget install Schniz.fnm

# 配置环境变量（PowerShell）
fnm env --use-on-cd | Out-String | Invoke-Expression

# 验证安装
fnm --version
```

### fnm 常用命令

```bash
# ===== 安装版本 =====

# 安装最新 LTS
fnm install --lts
# 或
fnm install lts-latest

# 安装指定版本
fnm install 22.0.0

# 安装最新版本
fnm install latest

# ===== 使用版本 =====

# 切换版本
fnm use 22.0.0

# 使用 LTS
fnm use lts-latest

# 自动使用（检测 .nvmrc 或 .node-version）
fnm use

# ===== 设置默认版本 =====

# 设置默认版本
fnm default 22.0.0

# 设置 LTS 为默认
fnm default lts-latest

# ===== 查看版本 =====

# 列出已安装的版本
fnm list
# 或简写
fnm ls

# 列出所有远程版本
fnm list-remote
# 或
fnm ls-remote

# 查看当前版本
fnm current

# ===== 卸载版本 =====

# 卸载指定版本
fnm uninstall 20.0.0

# ===== 其他命令 =====

# 执行命令（使用指定版本）
fnm exec --using=22.0.0 node -v

# 查看帮助
fnm --help
```

### fnm vs nvm 性能对比

```bash
# 测试场景：切换 Node.js 版本

nvm use 22.0.0
# 耗时：~400ms

fnm use 22.0.0
# 耗时：~10ms ⚡⚡⚡

# 性能提升：40 倍！
```

### 优点与缺点

| 优点 | 缺点 |
|------|------|
| ✅ 速度极快（Rust 编写） | ❌ 相对较新（2019年） |
| ✅ 跨平台统一 | ❌ 社区相对较小 |
| ✅ 配置简单 | ❌ 部分功能不如 nvm 丰富 |
| ✅ 支持 .nvmrc | ❌ 文档相对较少 |
| ✅ 内存占用小 | |

### 适用场景

```
✅ 追求极致性能
✅ 需要跨平台统一体验
✅ 喜欢现代化工具
✅ 频繁切换版本
✅ 项目启动速度敏感
```

✅ 团队统一使用 .nvmrc

操作：

## 💡 常见问题（FAQ）

**Q1：我应该选择哪种安装方式？**  
A：新手选官网安装包；需要多版本管理选 nvm；追求性能选 fnm。

**Q2：nvm 和 fnm 可以共存吗？**  
A：不建议。会造成路径冲突。选择其中一个使用。

**Q3：安装多个版本会占用很多空间吗？**  
A：会的。每个版本约 50-100MB。建议只保留常用版本。

**Q4：如何卸载 Node.js？**  
A：
- 官网安装：使用系统卸载工具
- nvm：`nvm uninstall <version>`
- fnm：`fnm uninstall <version>`

**Q5：Windows 上应该用哪个？**  
A：nvm-windows 或 fnm 都可以。fnm 配置更简单，性能更好。
