# 2.4 项目初始化规范：package.json 字段详解

> 深入理解 package.json 的每个字段，掌握项目配置规范

---

## 📋 package.json 是什么？

```
package.json = Node.js 项目的"身份证"
├─ 描述项目信息（名称、版本、作者）
├─ 声明依赖关系（dependencies）
├─ 定义脚本命令（scripts）
├─ 配置构建工具
└─ 发布到 npm 的元数据
```

---

## 🚀 快速初始化

### 方式1：交互式初始化

```bash
npm init

# 会依次询问：
# package name: (my-app)           ← 项目名称
# version: (1.0.0)                 ← 版本号
# description:                     ← 项目描述
# entry point: (index.js)          ← 入口文件
# test command:                    ← 测试命令
# git repository:                  ← Git 仓库
# keywords:                        ← 关键词
# author:                          ← 作者
# license: (ISC)                   ← 许可证

# 最后确认：
# Is this OK? (yes)
```

### 方式2：快速初始化（推荐）

```bash
npm init -y
# 或
npm init --yes

# 使用默认值生成 package.json
```

### 方式3：使用模板

```bash
# 创建 React 项目
npm create vite@latest my-app -- --template react

# 创建 Next.js 项目
npx create-next-app@latest my-app

# 创建 Express 项目
npx express-generator my-app
```

## 📖 核心字段详解

### 1. name（项目名称）

```json
{
  "name": "my-awesome-app"
}
```

**规则：**

```
✅ 只能包含小写字母、数字、- 和 _
✅ 不能以 . 或 _ 开头
✅ 长度 ≤ 214 字符
✅ 不能包含非 URL 安全字符

❌ 错误示例：
"My-App"        ← 大写字母
"my app"        ← 空格
"@my/app"       ← 需要作用域（见下方）

✅ 正确示例：
"my-app"
"my_app"
"my-app-v2"
```

**Scoped Packages（作用域包）：**

```json
{
  "name": "@myorg/my-app"
  //      └─────┘ └─────┘
  //       作用域   包名
}
```

```
作用域包的优势：
✅ 避免命名冲突（包名可重复）
✅ 团队/组织包管理
✅ 私有包发布

示例：
@vue/cli
@angular/core
@babel/preset-env
```

### 2. version（版本号）

```json
{
  "version": "1.2.3"
  //         └─┬─┘
  //           └─ 主版本.次版本.补丁版本
}
```

**语义化版本（SemVer）：**

```
版本格式：主版本号.次版本号.修订号

1.2.3
│ │ └─ PATCH：修复 bug，向后兼容
│ └─── MINOR：新功能，向后兼容
└───── MAJOR：破坏性更新，不兼容

版本演进示例：
1.0.0  ← 初始发布
1.0.1  ← 修复 bug
1.1.0  ← 新增功能
1.1.1  ← 又修复 bug
2.0.0  ← 破坏性更新
```

**预发布版本：**

```json
{
  "version": "1.0.0-alpha.1",    // 内部测试
  "version": "1.0.0-beta.2",     // 公开测试
  "version": "1.0.0-rc.1",       // 候选版本
  "version": "1.0.0"             // 正式版本
}
```

### 3. type（模块类型）★ 2025 重点

```json
{
  "type": "module"
  // 或 "commonjs"（默认）
}
```

**区别：**

```javascript
// ===== type: "commonjs"（默认） =====

// 使用 CommonJS 语法
const express = require('express');
module.exports = app;

// .js 文件 → CommonJS
// .mjs 文件 → ES Module
// .cjs 文件 → CommonJS

// ===== type: "module" =====

// 使用 ES Module 语法
import express from 'express';
export default app;

// .js 文件 → ES Module
// .mjs 文件 → ES Module
// .cjs 文件 → CommonJS
```

**2025 推荐：**

```json
{
  "type": "module"
}
```

```
原因：
✅ ES Module 是未来标准
✅ 更好的 Tree Shaking
✅ 静态分析优化
✅ 浏览器原生支持
✅ Deno/Bun 默认使用
```

### 4. description（项目描述）

```json
{
  "description": "A modern web application built with Express and React"
}
```

```
用途：
✅ npm 搜索展示
✅ GitHub 项目说明
✅ 团队成员快速了解项目
```

### 5. main / exports（入口文件）★ 2025 重点

#### 传统方式：main

```json
{
  "main": "./index.js"
}
```

```javascript
// 别人引用你的包时
import myPackage from 'my-package';
// 实际加载：my-package/index.js
```

#### 现代方式：exports（推荐）

```json
{
  "exports": {
    ".": "./index.js",
    "./utils": "./src/utils.js",
    "./config": "./src/config.js"
  }
}
```

```javascript
// 精确控制导出路径

import pkg from 'my-package';            // → index.js
import utils from 'my-package/utils';    // → src/utils.js
import config from 'my-package/config';  // → src/config.js

// ❌ 禁止访问未声明的路径
import secret from 'my-package/src/secret.js';  // Error!
```

**条件导出（Conditional Exports）：**

```json
{
  "exports": {
    ".": {
      "import": "./esm/index.js",      // ES Module
      "require": "./cjs/index.js",     // CommonJS
      "types": "./types/index.d.ts"    // TypeScript 类型
    },
    "./package.json": "./package.json"
  }
}
```

```
优势：
✅ 同时支持 ESM 和 CJS
✅ 自动选择合适的版本
✅ 更好的封装性
```

### 6. scripts（脚本命令）

```json
{
  "scripts": {
    "dev": "node server.js",
    "start": "node dist/index.js",
    "build": "tsc",
    "test": "jest",
    "lint": "eslint ."
  }
}
```

**运行方式：**

```bash
npm run dev
npm run build

# 特殊脚本（可省略 run）
npm start   # = npm run start
npm test    # = npm run test
npm stop    # = npm run stop
```

**生命周期钩子：**

```json
{
  "scripts": {
    "pretest": "echo 'Before test'",
    "test": "jest",
    "posttest": "echo 'After test'",
    
    "prebuild": "rm -rf dist",
    "build": "tsc",
    "postbuild": "cp package.json dist/",
    
    "preinstall": "echo 'Before install'",
    "postinstall": "node setup.js"
  }
}
```

```bash
# 执行 npm test 时，实际运行顺序：
npm run pretest   → npm run test   → npm run posttest
```

**传递参数：**

```bash
# 使用 -- 传递参数
npm run dev -- --port=3000

# scripts 中获取
"dev": "node server.js"
# 实际执行：node server.js --port=3000
```

**复杂脚本示例：**

```json
{
  "scripts": {
    "dev": "nodemon src/index.js",
    "dev:debug": "nodemon --inspect src/index.js",
    
    "build": "npm run build:clean && npm run build:compile",
    "build:clean": "rm -rf dist",
    "build:compile": "tsc",
    
    "test": "NODE_ENV=test jest",
    "test:watch": "jest --watch",
    "test:coverage": "jest --coverage",
    
    "lint": "npm run lint:js && npm run lint:style",
    "lint:js": "eslint 'src/**/*.{js,ts}'",
    "lint:style": "prettier --check 'src/**/*.{js,ts}'",
    "lint:fix": "eslint --fix 'src/**/*.{js,ts}' && prettier --write 'src/**/*.{js,ts}'",
    
    "prepare": "husky install",
    "prepublishOnly": "npm run build && npm test"
  }
}
```

### 7. dependencies / devDependencies

```json
{
  "dependencies": {
    "express": "^4.18.0",
    "mongoose": "~8.0.0"
  },
  "devDependencies": {
    "eslint": "^8.50.0",
    "jest": "^29.0.0"
  }
}
```

**版本约束符号：**

```
^4.18.0  ← 插入符（Caret）
    └─ 允许：4.18.0 ≤ 版本 < 5.0.0
    └─ 兼容次版本和补丁版本更新

~8.0.0   ← 波浪符（Tilde）
    └─ 允许：8.0.0 ≤ 版本 < 8.1.0
    └─ 只兼容补丁版本更新

*        ← 星号
    └─ 最新版本（不推荐）

1.2.3    ← 精确版本
    └─ 只允许 1.2.3

>=1.0.0  ← 范围
<2.0.0

1.2.x    ← X 范围
    └─ 等同于 ~1.2.0

latest   ← 最新版本（不推荐）
```

**dependencies vs devDependencies：**

```
dependencies（生产依赖）：
✅ 运行时需要的包
✅ 会被打包到生产环境
✅ npm install 会安装

示例：
express, mongoose, react, axios

devDependencies（开发依赖）：
✅ 开发时需要的包
✅ 不会被打包到生产环境
✅ npm install --production 不会安装

示例：
eslint, jest, nodemon, @types/node
```

### 8. peerDependencies（对等依赖）

```json
{
  "name": "my-react-plugin",
  "peerDependencies": {
    "react": "^18.0.0"
  }
}
```

```
用途：插件/扩展包声明宿主依赖

含义：
"我的包需要 react 18.x，但不会安装它，
请使用者自己安装 react。"

示例：
- eslint-plugin-* 需要 eslint
- @vue/cli-plugin-* 需要 @vue/cli
- jest-* 需要 jest
```

### 9. optionalDependencies（可选依赖）

```json
{
  "optionalDependencies": {
    "fsevents": "^2.3.0"  // macOS 文件监听优化
  }
}
```

```
特点：
✅ 安装失败不会中断 npm install
✅ 需要在代码中检查是否存在

用例：
- 平台特定的优化包
- 可选的性能增强
```

```javascript
// 使用可选依赖
let fsevents;
try {
  fsevents = require('fsevents');
} catch (err) {
  // 降级方案
  console.log('fsevents not available, using fallback');
}
```

### 10. engines（环境约束）

```json
{
  "engines": {
    "node": ">=20.0.0",
    "npm": ">=10.0.0"
  }
}
```

```bash
# 启用严格模式（.npmrc）
engine-strict=true

# 如果环境不匹配，npm install 会报错
npm ERR! Unsupported engine
npm ERR! Required: {"node":">=20.0.0"}
npm ERR! Actual: {"node":"18.0.0"}
```

**推荐配置（2025）：**

```json
{
  "engines": {
    "node": ">=20.0.0 <23.0.0",
    "npm": ">=10.0.0"
  },
  "packageManager": "pnpm@9.0.0"
}
```

## 🔧 其他重要字段

### 11. files（发布文件）

```json
{
  "files": [
    "dist",
    "README.md",
    "LICENSE"
  ]
}
```

```
作用：指定发布到 npm 时包含的文件

默认排除：
node_modules/
.git/
.DS_Store

默认包含：
package.json
README.md
LICENSE
```

### 12. bin（命令行工具）

```json
{
  "name": "my-cli",
  "bin": {
    "my-cli": "./bin/cli.js"
  }
}
```

```bash
# 全局安装后
npm install -g my-cli

# 可直接运行
my-cli --help

# cli.js 必须添加 shebang
#!/usr/bin/env node
console.log('Hello CLI');
```

### 13. workspaces（Monorepo）

```json
{
  "name": "monorepo-root",
  "workspaces": [
    "packages/*",
    "apps/*"
  ]
}
```

```
目录结构：
monorepo/
├── package.json
├── packages/
│   ├── ui/
│   │   └── package.json
│   └── utils/
│       └── package.json
└── apps/
    └── web/
        └── package.json
```

### 14. private（私有包）

```json
{
  "private": true
}
```

```
作用：
✅ 防止意外发布到 npm
✅ Monorepo 根包必须设置

如果尝试发布：
npm publish
npm ERR! This package is marked as private
```

## 💡 最佳实践

### ✅ 推荐做法

```json
{
  "name": "@myorg/my-app",
  "version": "1.0.0",
  "type": "module",  // ✅ 使用 ESM
  
  "engines": {
    "node": ">=20.0.0"  // ✅ 指定 Node.js 版本
  },
  
  "packageManager": "pnpm@9.0.0",  // ✅ 锁定包管理器

    "dev": "tsx watch src/index.ts",
    "build": "tsc",
    "test": "vitest",
    "lint": "eslint src",
    "format": "prettier --write src"
  },
  
  "dependencies": {
    "express": "^4.18.0"  // ✅ 使用 ^ 符号
  }
}
```

### ❌ 避免的做法

```json
{
  "name": "My-App",  // ❌ 不要使用大写
  
  "dependencies": {
    "express": "*",  // ❌ 不要使用 *
    "lodash": "latest"  // ❌ 不要使用 latest
  },
  
  "scripts": {
    "start": "node index.js || echo failed"  // ❌ 不要吞掉错误
  }
}
```
