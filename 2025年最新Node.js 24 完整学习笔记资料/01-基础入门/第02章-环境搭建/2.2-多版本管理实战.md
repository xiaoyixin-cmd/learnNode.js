# 2.2 多版本管理实战：nvm 切换 LTS 与 Current 版本

> 掌握 nvm 高级用法，实现项目自动版本切换

---

## 🎯 版本管理核心概念

### Node.js 版本策略

```
Node.js 版本发布规律：

偶数版本（LTS）          奇数版本（Current）
    ↓                        ↓
v20.x.x ─────────►     v21.x.x
    │                        │
    │ 长期支持 30 个月        │ 短期支持 6-8 个月
    │ 稳定性优先              │ 新特性实验
    │ 生产环境首选            │ 学习新功能
    │                        │
    ↓                        ↓
v22.x.x ─────────►     v23.x.x
```

### 版本选择策略

```
┌─────────────────┬──────────┬──────────┬────────────┐
│   场景          │   版本    │  稳定性   │   推荐度    │
├─────────────────┼──────────┼──────────┼────────────┤
│ 生产环境         │ LTS      │ ⭐⭐⭐⭐⭐  │ ✅ 强烈推荐  │
│ 新项目开发       │ LTS      │ ⭐⭐⭐⭐⭐  │ ✅ 推荐     │
│ 学习新特性       │ Current  │ ⭐⭐⭐⭐   │ ⚠️  可尝试   │
│ 旧项目维护       │ 旧 LTS   │ ⭐⭐⭐⭐⭐  │ ✅ 必须     │
│ 性能测试         │ Current  │ ⭐⭐⭐    │ ⚠️  谨慎    │
└─────────────────┴──────────┴──────────┴────────────┘
```

## 🔧 nvm 高级命令实战

### 1. 查看和安装版本

```bash
# ===== 查看所有可安装的 LTS 版本 =====

nvm ls-remote --lts

# 输出：
#         v18.19.0   (LTS: Hydrogen)
#         v18.19.1   (LTS: Hydrogen)
#         v20.10.0   (LTS: Iron)
#         v20.11.0   (LTS: Iron)
#         v22.0.0    (LTS: Jod)  ← 假设是最新 LTS
#        ->v22.1.0   (Latest LTS: Jod)

# ===== 查看所有 Current 版本 =====

nvm ls-remote | grep "v23"

# 输出：
#         v23.0.0
#         v23.1.0
#        ->v23.2.0

# ===== 安装最新 LTS =====

nvm install --lts
# 或指定代号
nvm install lts/iron      # Node.js 20
nvm install lts/hydrogen  # Node.js 18

# ===== 安装最新 Current =====

nvm install node
# 或指定版本
nvm install 23.2.0

# ===== 安装指定 LTS 的最新补丁版本 =====

nvm install 20
# 会自动安装 20.x.x 的最新版本

# ===== 列出本地已安装的版本 =====

nvm ls

# 输出：
#       v18.19.1
#       v20.11.0
#   ->  v22.1.0    ← 当前使用的版本
#       v23.2.0
#       default -> 22.1.0
#       lts/* -> v22.1.0
#       lts/hydrogen -> v18.19.1
#       lts/iron -> v20.11.0
```

### 2. 版本切换实战

```bash
# ===== 切换到指定版本 =====

nvm use 20.11.0
# 输出：Now using node v20.11.0 (npm v10.2.4)

# 验证当前版本
node -v  # v20.11.0
npm -v   # 10.2.4

# ===== 切换到最新 LTS =====

nvm use --lts
# 输出：Now using node v22.1.0 (npm v10.5.0)

# ===== 切换到最新版本 =====

nvm use node
# 输出：Now using node v23.2.0 (npm v10.6.0)

# ===== 切换到默认版本 =====

nvm use default
# 输出：Now using node v22.1.0

# ===== 查看当前版本 =====

nvm current
# 输出：v22.1.0
```

### 3. 设置默认版本

```bash
# ===== 设置默认版本（打开新终端时自动使用） =====

nvm alias default 22.1.0
# 输出：default -> 22.1.0

# 验证：关闭终端，重新打开
node -v  # v22.1.0 ✅ 自动使用默认版本

# ===== 设置 LTS 为默认 =====

nvm alias default lts/iron
# 输出：default -> lts/iron (-> v20.11.0)

# ===== 设置别名 =====

nvm alias production 20.11.0
nvm alias development 23.2.0

# 使用别名
nvm use production     # 切换到生产版本
nvm use development    # 切换到开发版本

# 查看所有别名
nvm alias
# 输出：
#   default -> 22.1.0
#   production -> 20.11.0
#   development -> 23.2.0
```

### 4. 运行特定版本的命令

```bash
# ===== 不切换版本，直接运行 =====

# 方式1：nvm run
nvm run 20.11.0 app.js
# 等同于：nvm use 20.11.0 && node app.js

# 方式2：nvm exec
nvm exec 20.11.0 node -v
# 输出：v20.11.0

# 一次性执行多个命令
nvm exec 22.1.0 sh -c "node -v && npm -v"
# 输出：
#   v22.1.0
#   10.5.0

# ===== 测试不同版本的兼容性 =====

# 创建测试脚本 test.js
echo "console.log('Node:', process.version)" > test.js

# 在多个版本中运行
nvm run 18.19.1 test.js  # Node: v18.19.1
nvm run 20.11.0 test.js  # Node: v20.11.0
nvm run 22.1.0 test.js   # Node: v22.1.0
```

## 📄 .nvmrc 文件：项目级版本锁定

### 什么是 .nvmrc？

```
.nvmrc = Node Version Manager Run Commands
├─ 锁定项目 Node.js 版本
├─ 团队统一开发环境
├─ 自动切换版本
└─ CI/CD 环境标准化
```

### 创建 .nvmrc 文件

```bash
# ===== 方式1：手动创建 =====

echo "22.1.0" > .nvmrc

# ===== 方式2：使用当前版本 =====

node -v > .nvmrc

# ===== 方式3：指定 LTS =====

echo "lts/iron" > .nvmrc

# ===== 方式4：只指定主版本号 =====

echo "20" > .nvmrc
# nvm 会自动使用 20.x.x 的最新版本
```

### .nvmrc 内容格式

```bash
# ===== 格式1：精确版本号 =====
# .nvmrc
22.1.0

# ===== 格式2：主版本号 =====
# .nvmrc
20

# ===== 格式3：LTS 代号 =====
# .nvmrc
lts/iron

# ===== 格式4：特殊标记 =====
# .nvmrc
node        # 最新版本
lts/*       # 最新 LTS
--lts       # 最新 LTS（另一种写法）

# ===== 格式5：带注释（推荐） =====
# .nvmrc
# Node.js LTS Iron (20.11.0)
# 用于生产环境部署
20.11.0
```

### 使用 .nvmrc

```bash
# ===== 进入项目目录 =====

cd /path/to/project

# ===== 手动切换到 .nvmrc 指定的版本 =====

nvm use
# 输出：Found '/path/to/project/.nvmrc' with version <20.11.0>
# 输出：Now using node v20.11.0 (npm v10.2.4)

# 如果版本未安装，会提示
# 输出：N/A: version "20.11.0 -> N/A" is not yet installed.
# 输出：You need to run "nvm install 20.11.0" to install it before using it.

# 安装并使用
nvm install
# 输出：Found '/path/to/project/.nvmrc' with version <20.11.0>
# 输出：Downloading and installing node v20.11.0...
# 输出：Now using node v20.11.0
```

### 自动切换版本

#### 方式1：手动配置 shell（推荐）

**Bash / Zsh：**

```bash
# 编辑 ~/.bashrc 或 ~/.zshrc
vim ~/.zshrc

# 添加以下代码：
# ===== nvm 自动切换 =====
autoload -U add-zsh-hook

load-nvmrc() {
  local node_version="$(nvm version)"
  local nvmrc_path="$(nvm_find_nvmrc)"

  if [ -n "$nvmrc_path" ]; then
    local nvmrc_node_version=$(nvm version "$(cat "${nvmrc_path}")")

    if [ "$nvmrc_node_version" = "N/A" ]; then
      nvm install
    elif [ "$nvmrc_node_version" != "$node_version" ]; then
      nvm use
    fi
  elif [ "$node_version" != "$(nvm version default)" ]; then
    echo "Reverting to nvm default version"
    nvm use default
  fi
}

add-zsh-hook chpwd load-nvmrc
load-nvmrc

# 保存退出，重新加载
source ~/.zshrc
```

**Fish Shell：**

```fish
# 编辑 ~/.config/fish/config.fish
# 添加以下代码：

function load_nvmrc --on-variable PWD
  set -l node_version (nvm version)
  set -l nvmrc_path (nvm_find_nvmrc)

  if test -n "$nvmrc_path"
    set -l nvmrc_node_version (nvm version (cat $nvmrc_path))

    if test "$nvmrc_node_version" = "N/A"
      nvm install
    else if test "$nvmrc_node_version" != "$node_version"
      nvm use
    end
  else if test "$node_version" != (nvm version default)
    echo "Reverting to nvm default version"
    nvm use default
  end
end

load_nvmrc
```

#### 方式2：使用 direnv（推荐进阶用户）

```bash
# ===== 安装 direnv =====

# macOS
brew install direnv

# Ubuntu/Debian
sudo apt install direnv

# ===== 配置 direnv =====

# 添加到 ~/.bashrc 或 ~/.zshrc
eval "$(direnv hook bash)"   # Bash
eval "$(direnv hook zsh)"    # Zsh

# ===== 在项目中创建 .envrc =====

echo "use node" > .envrc

# 允许 direnv 执行
direnv allow

# ===== 测试自动切换 =====

cd /path/to/project
# 输出：direnv: loading ~/project/.envrc
# 输出：direnv: using node
# 输出：Found '/path/to/project/.nvmrc' with version <20.11.0>
# 输出：Now using node v20.11.0

cd ..
# 输出：direnv: unloading
# 输出：Reverting to nvm default version
```

```bash
# ===== 项目结构 =====

~/Projects/

cd ~/Projects/frontend-app
node -v  # v20.11.0 ✅ 自动切换

cd ~/Projects/backend-api
node -v  # v22.1.0 ✅ 自动切换
```

### 场景2：旧项目维护

```bash
# ===== 维护 2020 年的老项目 =====

cd ~/Projects/legacy-app

cat .nvmrc
# 18.19.1  ← 老版本

nvm install  # 安装 18.19.1
nvm use      # 切换到 18.19.1

npm install  # 使用 18.x 运行
npm start
```

### 场景3：CI/CD 环境

```yaml
# ===== GitHub Actions 示例 =====

# .github/workflows/test.yml
name: Test

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
      
      # 读取 .nvmrc 文件
      - name: Read .nvmrc
        run: echo "NODE_VERSION=$(cat .nvmrc)" >> $GITHUB_OUTPUT
        id: nvm
      
      # 使用 .nvmrc 指定的版本
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ steps.nvm.outputs.NODE_VERSION }}
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run tests
        run: npm test
```

```yaml
# ===== GitLab CI 示例 =====

# .gitlab-ci.yml
image: node:20  # 默认镜像

before_script:
  # 如果存在 .nvmrc，使用指定版本
  - |
    if [ -f .nvmrc ]; then
      NODE_VERSION=$(cat .nvmrc)
      echo "Using Node.js $NODE_VERSION from .nvmrc"
      # 切换到对应的 Docker 镜像
      docker pull node:$NODE_VERSION
    fi

test:
  script:
    - npm ci
    - npm test
```

## 🔍 版本兼容性检查

### 检查项目兼容性

```bash
# ===== 使用 npx 检查 =====

npx node-version-checker
# 会读取 package.json 中的 engines 字段

# package.json
{
  "engines": {
    "node": ">=20.0.0 <23.0.0",
    "npm": ">=10.0.0"
  }
}

# ===== 手动测试多版本兼容性 =====

# 创建测试脚本
cat > test-versions.sh << 'EOF'
#!/bin/bash

echo "Testing multiple Node.js versions..."

for version in 18.19.1 20.11.0 22.1.0; do
  echo "\n==== Testing Node.js $version ===="
  nvm use $version
  npm test
done
EOF

chmod +x test-versions.sh
./test-versions.sh
```

### package.json 版本约束

```json
{
  "name": "my-app",
  "version": "1.0.0",
  "engines": {
    "node": ">=20.0.0",    // 最低版本
    "npm": ">=10.0.0"
  },
  "scripts": {
    "preinstall": "node -e \"if(!/^20/.test(process.version)){throw new Error('Required Node.js 20.x')}\"",
    "start": "node server.js"
  }
}

## 💡 最佳实践

### ✅ 推荐做法

```bash
# 1. 项目根目录必须有 .nvmrc
echo "20.11.0" > .nvmrc

# 2. package.json 指定 engines
{
  "engines": {
    "node": "20.11.0",
    "npm": "10.2.4"
  }
}

# 3. 团队成员统一版本
git add .nvmrc
git commit -m "chore: add .nvmrc to lock Node.js version"

# 4. 文档化版本要求
# README.md
## 开发环境要求
- Node.js: 20.11.0 (LTS)
- npm: 10.2.4

## 快速开始
```bash
nvm use         # 自动切换到项目指定版本
npm install
npm run dev
```

# 5. CI/CD 使用 .nvmrc
# 确保开发环境与生产环境版本一致
```

### ❌ 避免的做法

```bash
# ❌ 不要频繁安装删除版本
nvm install 20.11.0
nvm uninstall 20.11.0  # 第二天又安装回来

# ❌ 不要使用 Latest 标签
echo "node" > .nvmrc  # 不稳定，会变化

# ❌ 不要混用多个版本管理工具
# 同时安装 nvm + fnm + 官方安装包 ← 会冲突

# ❌ 不要忽略 .nvmrc
# 团队成员各自使用不同版本 ← 会有兼容性问题
```

## 🚀 进阶技巧

### 1. 快速切换常用版本

```bash
# 创建快捷别名（添加到 ~/.bashrc 或 ~/.zshrc）
alias n18="nvm use 18"
alias n20="nvm use 20"
alias n22="nvm use 22"
alias nlts="nvm use --lts"

# 使用
n20  # 快速切换到 Node.js 20
```

### 2. 查看版本性能对比

```bash
# 创建性能测试脚本
cat > benchmark.js << 'EOF'
const { performance } = require('perf_hooks');

const start = performance.now();

// 测试代码
for (let i = 0; i < 1000000; i++) {
  JSON.parse('{"key":"value"}');
}

const end = performance.now();
console.log(`${process.version}: ${(end - start).toFixed(2)}ms`);
EOF

# 在多个版本中运行
nvm run 18 benchmark.js  # v18.19.1: 245.67ms
nvm run 20 benchmark.js  # v20.11.0: 198.34ms
nvm run 22 benchmark.js  # v22.1.0: 175.23ms ⚡ 最快
```

### 3. 一键升级所有全局包

```bash
# 切换版本后，自动安装常用全局包
cat > ~/.nvm-global-packages << 'EOF'
yarn
pnpm
typescript
ts-node
nodemon
pm2
eslint
prettier
EOF

# 安装全局包到新版本
nvm install 22 --reinstall-packages-from=20
# 会自动从 Node.js 20 复制全局包到 Node.js 22
```
