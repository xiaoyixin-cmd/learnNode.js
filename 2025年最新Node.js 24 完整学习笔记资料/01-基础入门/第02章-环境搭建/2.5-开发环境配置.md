## 🔌 必装扩展推荐（2025 版本）

### 1. ESLint（代码质量）

```
扩展 ID：dbaeumer.vscode-eslint
作用：实时检查 JavaScript/TypeScript 代码错误
```

**安装：**

```bash
# VS Code 扩展商店搜索 "ESLint"
# 或命令行安装
code --install-extension dbaeumer.vscode-eslint
```

**配置：**

```json
// .vscode/settings.json
{
  "eslint.enable": true,
  "eslint.validate": [
    "javascript",
    "javascriptreact",
    "typescript",
    "typescriptreact"
  ],
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": "explicit"  // 保存时自动修复
  }
}
```

**项目配置：**

```bash
# 安装 ESLint
npm install -D eslint

# 初始化配置
npx eslint --init

# 选择：
# ✔ How would you like to use ESLint? · problems
# ✔ What type of modules does your project use? · esm
# ✔ Which framework does your project use? · none
# ✔ Does your project use TypeScript? · Yes
# ✔ Where does your code run? · node
```

```javascript
// eslint.config.js（ESLint 9+ 新格式）
export default [
  {
    files: ['**/*.{js,mjs,cjs,ts}'],
    rules: {
      'no-unused-vars': 'warn',
      'no-console': 'off',
      'semi': ['error', 'always'],
      'quotes': ['error', 'single']
    }
  }
];
```

## 🔐 环境变量管理

### 使用 .env 文件

**安装 dotenv：**

```bash
npm install dotenv
```

**创建 .env 文件：**

```bash
# .env（项目根目录）

# 服务器配置
PORT=3000
HOST=localhost

# 数据库配置
DB_HOST=localhost
DB_PORT=5432
DB_NAME=myapp
DB_USER=admin
DB_PASSWORD=secret123

# JWT 配置
JWT_SECRET=your-secret-key-here
JWT_EXPIRES_IN=7d

# API 密钥
API_KEY=your-api-key
OPENAI_API_KEY=sk-...

# 环境
NODE_ENV=development
```

**加载环境变量：**

```javascript
// src/index.js
import 'dotenv/config';  // ESM
// 或
// require('dotenv').config();  // CommonJS

console.log(process.env.PORT);  // 3000
console.log(process.env.DB_HOST);  // localhost

const server = app.listen(process.env.PORT, () => {
  console.log(`Server running on port ${process.env.PORT}`);
});
```

**类型安全（TypeScript）：**

```typescript
// src/env.ts
interface Env {
  PORT: number;
  DB_HOST: string;
  DB_PORT: number;
  JWT_SECRET: string;
  NODE_ENV: 'development' | 'production' | 'test';
}

function loadEnv(): Env {
  const port = parseInt(process.env.PORT || '3000', 10);
  const dbPort = parseInt(process.env.DB_PORT || '5432', 10);
  
  if (!process.env.JWT_SECRET) {
    throw new Error('JWT_SECRET is required');
  }
  
  return {
    PORT: port,
    DB_HOST: process.env.DB_HOST || 'localhost',
    DB_PORT: dbPort,
    JWT_SECRET: process.env.JWT_SECRET,
    NODE_ENV: process.env.NODE_ENV as Env['NODE_ENV'] || 'development'
  };
}

export const env = loadEnv();
```

```typescript
// src/index.ts
import { env } from './env.js';

console.log(env.PORT);  // 类型安全：number
console.log(env.NODE_ENV);  // 类型安全：'development' | 'production' | 'test'
```

### 多环境配置

```bash
# 项目结构
.
├── .env                # 默认（所有环境）
├── .env.development    # 开发环境
├── .env.production     # 生产环境
├── .env.test           # 测试环境
└── .env.local          # 本地覆盖（不提交到 Git）
```

```bash
# .env.development
NODE_ENV=development
PORT=3000
DB_HOST=localhost
DEBUG=true

# .env.production
NODE_ENV=production
PORT=8080
DB_HOST=prod-db.example.com
DEBUG=false

# .env.test
NODE_ENV=test
PORT=3001
DB_HOST=localhost
DB_NAME=myapp_test
```

**加载特定环境：**

```javascript
// src/config.js
import dotenv from 'dotenv';
import path from 'path';

const env = process.env.NODE_ENV || 'development';

// 按优先级加载
dotenv.config({ path: '.env' });  // 默认
dotenv.config({ path: `.env.${env}` });  // 环境特定
dotenv.config({ path: '.env.local' });  // 本地覆盖
```

```json
// package.json
{
  "scripts": {
    "dev": "NODE_ENV=development node src/index.js",
    "prod": "NODE_ENV=production node src/index.js",
    "test": "NODE_ENV=test jest"
  }
}
```

### 安全注意事项

```bash
# .gitignore（必须添加）
.env
.env.local
.env.*.local

# ✅ 可以提交的（模板）
.env.example
```

```bash
# .env.example（提交到 Git，作为模板）
PORT=3000
DB_HOST=localhost
DB_PORT=5432
DB_NAME=myapp
DB_USER=
DB_PASSWORD=
JWT_SECRET=
```

```
⚠️ 安全规则：
❌ 不要提交 .env 文件到 Git
❌ 不要在代码中硬编码密钥
❌ 不要在日志中打印敏感信息
✅ 使用 .env.example 作为模板
✅ 定期轮换密钥
✅ 使用密钥管理服务（生产环境）
```

## 🔄 自动版本切换

### 使用 .nvmrc

```bash
# 项目根目录创建 .nvmrc
echo "20.11.0" > .nvmrc
```

**配置自动切换（见 2.2 节）：**

```bash
# ~/.zshrc 或 ~/.bashrc
autoload -U add-zsh-hook
load-nvmrc() {
  if [ -f .nvmrc ]; then
    nvm use
  fi
}
add-zsh-hook chpwd load-nvmrc
load-nvmrc
```

**效果：**

```bash
cd /path/to/project
# 输出：Found '.nvmrc' with version <20.11.0>
# 输出：Now using node v20.11.0
```

### 使用 .node-version（fnm）

```bash
# 项目根目录创建 .node-version
echo "20.11.0" > .node-version
```

```bash
# fnm 会自动检测并切换
cd /path/to/project
# 自动切换到 Node.js 20.11.0
```

## 💡 最佳实践

### ✅ 推荐做法

```
1. 统一团队开发环境
   ├─ 提交 .nvmrc / .node-version
   ├─ 提交 .vscode/settings.json
   └─ 提交 .env.example

2. 代码格式化自动化
   ├─ ESLint + Prettier
   ├─ 保存时自动格式化
   └─ Git 提交前检查（husky）

3. 环境变量管理
   ├─ 使用 .env 文件
   ├─ 不提交敏感信息
   └─ 提供 .env.example 模板

4. 调试配置
   ├─ 配置 launch.json
   ├─ 使用 VS Code 调试器
   └─ 利用 sourcemap
```

### ❌ 避免的做法

```
❌ 不提交 .env 文件
❌ 不硬编码密钥
❌ 不混用不同的格式化工具
❌ 不忽略 ESLint 警告
❌ 不在生产环境启用 DEBUG 模式
```
