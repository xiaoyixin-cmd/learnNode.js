### npm 10+ 新特性详解

#### 1. overrides 字段（覆盖依赖版本）

```json
{
  "name": "my-app",
  "dependencies": {
    "express": "^4.18.0",
    "some-package": "^1.0.0"
  },
  "overrides": {
    // 强制所有依赖使用 express 4.18.2
    "express": "4.18.2",
    
    // 只覆盖 some-package 下的 lodash
    "some-package": {
      "lodash": "4.17.21"
    },
    
    // 覆盖所有深层依赖的 axios
    "*": {
      "axios": "1.6.0"
    }
  }
}
```

#### 2. Workspaces 增强（Monorepo 支持）

```json
{
  "name": "monorepo-root",
  "workspaces": [
    "packages/*"
  ]
}
```

```bash
# 项目结构
monorepo/
├── package.json
└── packages/
    ├── app-web/
    │   └── package.json
    └── app-server/
        └── package.json

# 为特定工作区安装依赖
npm install express --workspace=app-server

# 在所有工作区运行脚本
npm run test --workspaces

# 只在特定工作区运行
npm run dev --workspace=app-web
```

#### 3. 性能优化

```bash
# npm 10+ 默认启用并行安装
# 同时下载多个包

npm install  # 比 npm 8 快 40%

# 查看安装耗时
npm install --timing

# 使用本地缓存（更快）
npm cache verify
```

### pnpm 原理：内容寻址存储

#### 传统 npm/yarn 的问题

```
项目结构（npm/yarn）：

project-a/
└── node_modules/
    └── lodash/         ← 100 MB
        └── ...

project-b/
└── node_modules/
    └── lodash/         ← 100 MB（重复）
        └── ...

project-c/
└── node_modules/
    └── lodash/         ← 100 MB（重复）
        └── ...

总占用：300 MB ❌ 严重浪费磁盘空间
```

#### pnpm 的解决方案

```
pnpm 结构：

全局存储（~/.pnpm-store）：
└── lodash@4.17.21/     ← 只存一份（100 MB）
    └── ...

project-a/
└── node_modules/
    └── .pnpm/
        └── lodash@4.17.21/ → 符号链接到全局存储

project-b/
└── node_modules/
    └── .pnpm/
        └── lodash@4.17.21/ → 符号链接到全局存储

project-c/
└── node_modules/
    └── .pnpm/
        └── lodash@4.17.21/ → 符号链接到全局存储

总占用：100 MB ✅ 节省 66% 空间
```

#### 符号链接（Symlink）机制

```bash
# 查看 pnpm 的 node_modules 结构

ls -la node_modules/

# 输出：
# drwxr-xr-x  .pnpm/
# lrwxr-xr-x  express -> .pnpm/express@4.18.0/node_modules/express
# lrwxr-xr-x  lodash -> .pnpm/lodash@4.17.21/node_modules/lodash

# express 实际指向：
node_modules/.pnpm/express@4.18.0/node_modules/express
    ↓
~/.pnpm-store/v3/files/00/12345678...（硬链接）
```

### pnpm 工作区（Workspaces）

### PnP 模式详解

#### 传统 node_modules 的问题

```
传统方式：

node_modules/
├── package-a/
│   ├── node_modules/
│   │   └── package-b/  ← 嵌套深层目录
│   └── index.js
└── package-c/
    └── index.js

问题：
❌ 目录层级深（Windows 路径限制）
❌ 文件数量多（几十万个文件）
❌ 安装慢（需要解压写入）
❌ 幽灵依赖（可访问未声明的包）
```

#### PnP 的革命性方案

```
PnP 模式：

.yarn/
├── cache/
│   ├── express-npm-4.18.0.zip    ← 压缩包，不解压
│   └── lodash-npm-4.17.21.zip
└── unplugged/  ← 只解压必须的原生模块

.pnp.cjs  ← 依赖映射表（JavaScript 文件）

node_modules/  ← 不存在！

优势：
✅ 无 node_modules 目录
✅ 安装速度快 3-5 倍
✅ 零安装（可提交缓存到 Git）
✅ 绝对无幽灵依赖
```

#### 启用 PnP 模式

```yaml
# .yarnrc.yml

nodeLinker: pnp  # 默认就是 pnp

# 或使用传统 node_modules
# nodeLinker: node-modules
```

```bash

# 运行项目
yarn node server.js
# 或
yarn run dev

# VS Code 集成
yarn dlx @yarnpkg/sdks vscode
# 会生成 .vscode/settings.json 配置
```

```bash
# package.json
{
  "packageManager": "pnpm@9.0.0"
  // 或 "npm@10.5.0"
  // 或 "yarn@4.1.0"
}

# 2. 提交 lock 文件到 Git
git add package-lock.json    # npm
git add pnpm-lock.yaml       # pnpm
git add yarn.lock            # yarn

# 3. 团队统一工具
# .npmrc（禁止使用其他工具）
engine-strict=true

# package.json
{
  "engines": {
    "pnpm": "9.x"
  }
}

# 4. CI/CD 使用 lock 文件
npm ci              # npm
pnpm install --frozen-lockfile  # pnpm
yarn install --immutable        # yarn
```

### ❌ 避免的做法

```bash
# ❌ 不要混用多个包管理器
npm install && pnpm install  # 会造成冲突

# ❌ 不要忽略 lock 文件
echo "package-lock.json" >> .gitignore  # 不要这样做

# ❌ 不要手动修改 lock 文件
vim package-lock.json  # 会导致不一致

# ❌ 不要在生产环境使用 npm install
npm install  # 可能安装不同版本
npm ci       # ✅ 正确做法
```
