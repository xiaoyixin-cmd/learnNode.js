# 9.3 测试框架选型：Vitest vs Jest

> 2025年主流测试框架对比与选择

---

## 🎯 为什么需要测试框架？

### 核心概念

```
测试框架 = 提供完整测试功能的工具集

Node.js 内置测试 vs 测试框架：

内置 node:test：
✅ 零依赖
✅ 快速启动
❌ 功能简单
❌ 生态较小

测试框架（Vitest/Jest）：
✅ 功能强大
✅ 丰富的生态
✅ 更好的DX（开发体验）
✅ Mock、Snapshot等高级功能
❌ 需要安装

何时使用测试框架：
• 中大型项目
• 需要 Mock 功能
• 需要快照测试
• 需要覆盖率报告
• 团队协作项目
```

## 🚀 Vitest - 现代化测试框架

### 1. 为什么选择 Vitest？

```
Vitest = Vite 驱动的测试框架

优势：
✅ 极快的速度（基于 Vite）
✅ 原生 ES Module 支持
✅ 兼容 Jest API
✅ 开箱即用 TypeScript
✅ 智能监听模式
✅ 并发测试
✅ UI 界面
✅ 2025 最流行的新选择

适合场景：
• Vite 项目（天然集成）
• 现代化项目（ESM first）
• 追求速度
• TypeScript 项目
```

### 2. 安装与配置

```bash
# ===== 安装 =====
npm install --save-dev vitest
```

```javascript
// ===== vitest.config.js =====

import { defineConfig } from 'vitest/config';

export default defineConfig({
  test: {
    globals: true,              // 全局 API（describe, test, expect）
    environment: 'node',        // 测试环境（node/jsdom/happy-dom）
    coverage: {
      provider: 'v8',           // 覆盖率工具
      reporter: ['text', 'json', 'html']
    },
    include: ['**/*.{test,spec}.{js,ts}'],  // 测试文件
    exclude: ['**/node_modules/**', '**/dist/**']
  }
});
```

```json
// ===== package.json =====

{
  "scripts": {
    "test": "vitest",
    "test:ui": "vitest --ui",
    "test:coverage": "vitest --coverage"
  }
}
```

### 3. 基础使用

```javascript
// ===== math.test.js =====

import { describe, test, expect } from 'vitest';

// 💡 如果配置了 globals: true，可以省略import

describe('数学运算', () => {
  test('加法', () => {
    expect(1 + 1).toBe(2);
  });
  
  test('减法', () => {
    expect(5 - 3).toBe(2);
  });
  
  test('乘法', () => {
    expect(3 * 4).toBe(12);
  });
  
  test('除法', () => {
    expect(10 / 2).toBe(5);
  });
});

// 运行：npm test
```

### 4. Vitest 特色功能

```javascript
// ===== 1. 智能监听 =====

// 运行：vitest --watch
// 自动检测文件变化并重新运行相关测试

// ===== 2. UI 界面 =====

// 运行：vitest --ui
// 打开浏览器查看测试结果

// ===== 3. 并发测试 =====

import { describe, test } from 'vitest';

describe.concurrent('并发测试套件', () => {
  test('测试1', async () => {
    // 这些测试会并发运行
    await someAsyncOperation();
  });
  
  test('测试2', async () => {
    await someAsyncOperation();
  });
});

// ===== 4. 基准测试 =====

import { bench, describe } from 'vitest';

describe('性能测试', () => {
  bench('方法A', () => {
    // 测试代码
    methodA();
  });
  
  bench('方法B', () => {
    methodB();
  });
});

// ===== 5. 内联快照 =====

test('快照测试', () => {
  expect({ name: 'Alice', age: 25 }).toMatchInlineSnapshot(`
    {
      "name": "Alice",
      "age": 25
    }
  `);
});
```

## 🎨 Jest - 成熟的测试框架

### 1. 为什么选择 Jest？

```
Jest = Facebook 开发的测试框架

优势：
✅ 功能完整
✅ 生态成熟
✅ 文档详尽
✅ 社区庞大
✅ 零配置（开箱即用）
✅ 快照测试
✅ Mock 强大

适合场景：
• React 项目（官方推荐）
• 成熟的生态需求
• 需要强大的 Mock
• 团队熟悉 Jest
```

### 2. 安装与配置

```bash
# ===== 安装 =====
npm install --save-dev jest

# 对于 ESM 项目
npm install --save-dev jest @types/jest
```

```javascript
// ===== jest.config.js =====

export default {
  testEnvironment: 'node',
  transform: {},                    // 禁用转换（原生ESM）
  testMatch: ['**/*.test.js'],
  collectCoverageFrom: [
    'src/**/*.js',
    '!src/**/*.test.js'
  ],
  coverageDirectory: 'coverage',
  coverageReporters: ['text', 'lcov', 'html']
};
```

```json
// ===== package.json（ESM支持）=====

{
  "type": "module",
  "scripts": {
    "test": "node --experimental-vm-modules node_modules/jest/bin/jest.js"
  }
}
```

### 3. Jest 特色功能

```javascript
// ===== 1. 快照测试 =====

test('组件快照', () => {
  const tree = renderer.create(<Button>Click me</Button>).toJSON();
  expect(tree).toMatchSnapshot();
  // 第一次运行会生成快照文件
  // 后续运行会与快照对比
});

// ===== 2. Mock 函数 =====

import { jest } from '@jest/globals';

test('Mock 函数', () => {
  const mockFn = jest.fn();
  mockFn('hello');
  mockFn('world');
  
  expect(mockFn).toHaveBeenCalledTimes(2);
  expect(mockFn).toHaveBeenCalledWith('hello');
  expect(mockFn).toHaveBeenLastCalledWith('world');
});

// ===== 3. Mock 模块 =====

// api.js
export async function fetchUser(id) {
  const response = await fetch(`/api/users/${id}`);
  return response.json();
}

// api.test.js
import { jest } from '@jest/globals';

jest.unstable_mockModule('./api.js', () => ({
  fetchUser: jest.fn(() => Promise.resolve({ id: 1, name: 'Alice' }))
}));

const { fetchUser } = await import('./api.js');

test('Mock API', async () => {
  const user = await fetchUser(1);
  expect(user.name).toBe('Alice');
});

// ===== 4. Timer Mock =====

jest.useFakeTimers();

test('setTimeout', () => {
  const callback = jest.fn();
  setTimeout(callback, 1000);
  
  // 快进1秒
  jest.advanceTimersByTime(1000);
  
  expect(callback).toHaveBeenCalled();
});

jest.useRealTimers();
```

## ⚖️ Vitest vs Jest 对比

### 功能对比表

| 特性 | Vitest | Jest | 说明 |
|------|--------|------|------|
| **速度** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | Vitest 更快 |
| **ESM 支持** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | Vitest 原生支持 |
| **配置复杂度** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | Vitest 更简单 |
| **生态成熟度** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | Jest 更成熟 |
| **文档质量** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | Jest 更详细 |
| **社区规模** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | Jest 更大 |
| **UI 界面** | ✅ | ❌ | Vitest 独有 |
| **Watch 模式** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | Vitest 更智能 |
| **TypeScript** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | Vitest 原生支持 |

### API 对比

```javascript
// ===== 两者API高度相似 =====

// Vitest
import { describe, test, expect, vi } from 'vitest';

// Jest
import { describe, test, expect, jest } from '@jest/globals';

// 💡 主要区别：
// • Vitest 使用 vi，Jest 使用 jest
// • 其他API几乎完全兼容
```

### 性能对比

```
启动速度：
• Vitest: ~100ms
• Jest: ~2000ms

测试运行速度（1000个测试）：
• Vitest: ~5s
• Jest: ~15s

监听模式响应：
• Vitest: <100ms
• Jest: ~500ms

💡 Vitest 通常快 2-3 倍
```

## 🎯 如何选择？

### 决策树

```
开始
  │
  ├─ 使用 Vite？
  │   ├─ 是 → Vitest ✅
  │   └─ 否 ↓
  │
  ├─ 追求极致速度？
  │   ├─ 是 → Vitest ✅
  │   └─ 否 ↓
  │
  ├─ 需要成熟生态？
  │   ├─ 是 → Jest ✅
  │   └─ 否 ↓
  │
  ├─ React 项目？
  │   ├─ 是 → Jest ✅
  │   └─ 否 ↓
  │
  ├─ 团队熟悉 Jest？
  │   ├─ 是 → Jest ✅
  │   └─ 否 ↓
  │
  └─ 新项目 → Vitest ✅
```

### 推荐方案

```
2025 年推荐：

1. Vite 项目 → Vitest
2. Next.js 项目 → Jest
3. React 项目 → Jest
4. Vue 项目 → Vitest
5. Node.js 库 → Vitest
6. 简单项目 → node:test
7. 追求速度 → Vitest
8. 需要生态 → Jest
```

## 📝 迁移指南

### Jest → Vitest

```bash
# 1. 卸载 Jest
npm uninstall jest @types/jest

# 2. 安装 Vitest
npm install --save-dev vitest

# 3. 更新 package.json
{
  "scripts": {
    "test": "vitest"  // 替换 jest
  }
}

# 4. 创建 vitest.config.js
# 5. 更新测试文件的import

# 从：
import { jest } from '@jest/globals';

# 改为：
import { vi } from 'vitest';

# 6. 运行测试
npm test
```

## 💡 最佳实践

```javascript
// ✅ 1. 使用 describe 组织测试
describe('用户模块', () => {
  test('创建用户', () => { });
  test('删除用户', () => { });
});

// ✅ 2. 使用 beforeEach/afterEach
let db;

beforeEach(async () => {
  db = await setupDatabase();
});

afterEach(async () => {
  await db.close();
});

// ✅ 3. 使用有意义的测试名
test('add() 应该返回两数之和', () => { });  // ✅ 清晰
test('test1', () => { });                    // ❌ 模糊

// ✅ 4. 一个测试一个断言（尽量）
test('用户名应该是 Alice', () => {
  expect(user.name).toBe('Alice');
});

// ✅ 5. 使用 .only 调试
test.only('只运行这个测试', () => {
  // 调试时很有用
});

// ✅ 6. 使用 .skip 跳过
test.skip('暂时跳过这个测试', () => {
  // 稍后修复
});
```
