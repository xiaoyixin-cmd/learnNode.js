# 9.5 代码覆盖率

> 使用c8和istanbul生成覆盖率报告

---

## 🎯 什么是代码覆盖率？

### 核心概念

```
代码覆盖率 = 测试覆盖了多少代码

覆盖率类型：
• 行覆盖率（Line Coverage）- 多少行被执行
• 分支覆盖率（Branch Coverage）- 多少分支被执行
• 函数覆盖率（Function Coverage）- 多少函数被调用
• 语句覆盖率（Statement Coverage）- 多少语句被执行

示例：
function divide(a, b) {
  if (b === 0) {     // 分支1
    return 0;        // ← 未测试
  }
  return a / b;      // 分支2 ← 已测试
}

// 只测试了分支2
test('divide', () => {
  expect(divide(10, 2)).toBe(5);
});

覆盖率：
• 行覆盖率: 75% (3/4行)
• 分支覆盖率: 50% (1/2分支)
• 函数覆盖率: 100% (1/1函数)

覆盖率指标：
• 90%+ = 优秀 ✅
• 80-90% = 良好 👍
• 70-80% = 及格 ⚠️
• <70% = 需要改进 ❌
```

## 📦 c8 - V8 原生覆盖率

### 1. 为什么选择 c8？

```
c8 = 基于 V8 的覆盖率工具

优势：
✅ 快速（使用 V8 原生功能）
✅ 准确（不需要代码插桩）
✅ 支持 ESM
✅ 零配置
✅ Node.js 内置测试的推荐工具

适合场景：
• Node.js 项目
• ESM 项目
• 追求速度和准确性
```

### 2. 安装与使用

```bash
# ===== 安装 =====
npm install --save-dev c8
```

```json
// ===== package.json =====

{
  "scripts": {
    "test": "node --test",
    "test:coverage": "c8 --reporter=text --reporter=html npm test"
  }
}
```

```bash
# ===== 运行覆盖率 =====

npm run test:coverage

# 输出：
# -----------|---------|----------|---------|---------|
# File       | % Stmts | % Branch | % Funcs | % Lines |
# -----------|---------|----------|---------|---------|
# All files  |   85.71 |       75 |     100 |   85.71 |
#  math.js   |   85.71 |       75 |     100 |   85.71 |
# -----------|---------|----------|---------|---------|
#
# HTML 报告: coverage/index.html
```

### 3. c8 配置

```javascript
// ===== .c8rc.json =====

{
  "all": true,
  "include": [
    "src/**/*.js",
    "lib/**/*.js"
  ],
  "exclude": [
    "**/*.test.js",
    "**/*.spec.js",
    "**/node_modules/**",
    "**/tests/**"
  ],
  "reporter": [
    "text",
    "html",
    "lcov"
  ],
  "check-coverage": true,
  "lines": 80,
  "functions": 80,
  "branches": 80,
  "statements": 80
}
```

```bash
# ===== 覆盖率阈值检查 =====

# 如果覆盖率低于阈值，测试会失败
c8 --check-coverage --lines 80 --functions 80 npm test

# 成功输出：
# ✅ Coverage meets thresholds

# 失败输出：
# ❌ Coverage for lines (75%) does not meet threshold (80%)
```

## 🎨 Vitest 内置覆盖率

### 1. 配置

```bash
# ===== 安装覆盖率工具 =====
npm install --save-dev @vitest/coverage-v8
# 或
npm install --save-dev @vitest/coverage-istanbul
```

```javascript
// ===== vitest.config.js =====

import { defineConfig } from 'vitest/config';

export default defineConfig({
  test: {
    coverage: {
      provider: 'v8',  // 或 'istanbul'
      reporter: ['text', 'json', 'html'],
      include: ['src/**/*.js'],
      exclude: [
        '**/*.test.js',
        '**/*.spec.js',
        '**/node_modules/**'
      ],
      // 覆盖率阈值
      thresholds: {
        lines: 80,
        functions: 80,
        branches: 80,
        statements: 80
      }
    }
  }
});
```

```json
// ===== package.json =====

{
  "scripts": {
    "test": "vitest",
    "test:coverage": "vitest --coverage"
  }
}
```

## 📊 解读覆盖率报告

### 1. HTML报告

```
coverage/index.html

点击文件可以看到：
• 绿色 = 已覆盖的代码
• 红色 = 未覆盖的代码
• 黄色 = 部分覆盖的分支

示例：
1  function divide(a, b) {
2 ✅ if (b === 0) {
3 ❌   return 0;        // 未测试
4    }
5 ✅ return a / b;
6  }
```

### 2. 指标解释

```javascript
// ===== 示例代码 =====

export function calculate(a, b, operation) {
  if (operation === 'add') {          // 分支 1
    return a + b;                     // 语句 1
  } else if (operation === 'subtract') { // 分支 2
    return a - b;                     // 语句 2
  } else {                            // 分支 3
    throw new Error('Invalid');      // 语句 3
  }
}

// ===== 测试 =====

test('add', () => {
  expect(calculate(2, 3, 'add')).toBe(5);
});

// 覆盖率：
// • 函数覆盖率: 100% (1/1 函数被调用)
// • 行覆盖率: 66.67% (4/6 行被执行)
// • 分支覆盖率: 33.33% (1/3 分支被测试)
// • 语句覆盖率: 33.33% (1/3 语句被执行)

// ===== 提高覆盖率 =====

test('add', () => {
  expect(calculate(2, 3, 'add')).toBe(5);
});

test('subtract', () => {
  expect(calculate(5, 3, 'subtract')).toBe(2);
});

test('invalid operation', () => {
  expect(() => calculate(1, 2, 'invalid')).toThrow('Invalid');
});

// 现在覆盖率：
// • 函数覆盖率: 100%
// • 行覆盖率: 100%
// • 分支覆盖率: 100%
// • 语句覆盖率: 100%
```

## 🎯 实战案例

### 案例1：提高覆盖率

```javascript
// ===== user.js =====

export class User {
  constructor(name, age) {
    if (!name) {
      throw new Error('Name required');
    }
    
    if (age < 0) {
      throw new Error('Invalid age');
    }
    
    this.name = name;
    this.age = age;
  }
  
  isAdult() {
    return this.age >= 18;
  }
  
  greet() {
    if (this.age < 18) {
      return `Hi, I'm ${this.name}`;
    } else {
      return `Hello, I'm ${this.name}`;
    }
  }
}

// ===== user.test.js（初始版本 - 覆盖率50%）=====

import { test, expect } from 'vitest';
import { User } from './user.js';

test('创建用户', () => {
  const user = new User('Alice', 25);
  expect(user.name).toBe('Alice');
  expect(user.age).toBe(25);
});

// 运行覆盖率：
// user.js: 50% (很多代码未测试)

// ===== user.test.js（改进版本 - 覆盖率100%）=====

import { describe, test, expect } from 'vitest';
import { User } from './user.js';

describe('User 类', () => {
  describe('构造函数', () => {
    test('应该创建有效用户', () => {
      const user = new User('Alice', 25);
      expect(user.name).toBe('Alice');
      expect(user.age).toBe('25);
    });
    
    test('应该拒绝空名字', () => {
      expect(() => new User('', 25)).toThrow('Name required');
    });
    
    test('应该拒绝负数年龄', () => {
      expect(() => new User('Alice', -1)).toThrow('Invalid age');
    });
  });
  
  describe('isAdult()', () => {
    test('18岁以上应该返回true', () => {
      const user = new User('Alice', 25);
      expect(user.isAdult()).toBe(true);
    });
    
    test('18岁以下应该返回false', () => {
      const user = new User('Bob', 15);
      expect(user.isAdult()).toBe(false);
    });
  });
  
  describe('greet()', () => {
    test('成人应该返回Hello', () => {
      const user = new User('Alice', 25);
      expect(user.greet()).toBe("Hello, I'm Alice");
    });
    
    test('未成年应该返回Hi', () => {
      const user = new User('Bob', 15);
      expect(user.greet()).toBe("Hi, I'm Bob");
    });
  });
});

// 运行覆盖率：
// user.js: 100% ✅
```

### 案例2：CI中的覆盖率检查

```yaml
# ===== .github/workflows/test.yml =====

name: Test

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '20'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run tests with coverage
        run: npm run test:coverage
      
      - name: Check coverage thresholds
        run: |
          if [ $(jq -r '.total.lines.pct' coverage/coverage-summary.json | cut -d. -f1) -lt 80 ]; then
            echo "Coverage below 80%"
            exit 1
          fi
      
      - name: Upload coverage to Codecov
        uses: codecov/codecov-action@v3
        with:
          files: ./coverage/coverage-final.json
```

## 💡 最佳实践

```javascript
// ✅ 1. 设置合理的覆盖率目标
{
  "coverage": {
    "thresholds": {
      "lines": 80,      // 80%以上为优秀
      "functions": 80,
      "branches": 75,   // 分支可以稍低
      "statements": 80
    }
  }
}

// ✅ 2. 排除不需要测试的文件
{
  "coverage": {
    "exclude": [
      "**/*.test.js",
      "**/*.config.js",
      "**/types/**",
      "**/mocks/**"
    ]
  }
}

// ✅ 3. 测试边界情况
test('边界情况', () => {
  expect(func(0)).toBe(...);    // 最小值
  expect(func(100)).toBe(...);  // 最大值
  expect(func(-1)).toThrow();   // 无效值
});

// ✅ 4. 不要为了覆盖率而测试
// ❌ 不好：只为了100%覆盖率
test('getter', () => {
  const obj = new MyClass();
  obj.name;  // 什么都没测试
});

// ✅ 好：测试实际行为
test('getter应该返回正确的值', () => {
  const obj = new MyClass('Alice');
  expect(obj.name).toBe('Alice');
});

// ✅ 5. 关注未覆盖的代码
// 查看HTML报告，找出红色的代码
// 思考：为什么没有测试到？
// • 边界情况？
// • 错误处理？
// • 罕见分支？

// ✅ 6. 定期审查覆盖率
// 在CI中设置覆盖率阈值
// PR时检查覆盖率变化
// 不让覆盖率下降

// ✅ 7. 覆盖率不是唯一指标
// 100%覆盖率 ≠ 没有bug
// 重点是测试质量，不是数量
// 测试应该测试行为，不是实现
```
