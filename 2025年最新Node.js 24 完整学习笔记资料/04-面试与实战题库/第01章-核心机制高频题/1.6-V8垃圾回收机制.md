## 📚 核心知识点

### 1. 分代垃圾回收

V8 使用**分代垃圾回收**，将内存分为：
- **新生代（Young Generation）**：存放生命周期短的对象
- **老生代（Old Generation）**：存放生命周期长的对象

```
V8 堆内存
│
├── 新生代（1-8MB，默认）
│   ├── From Space（活跃区）
│   └── To Space（空闲区）
│
└── 老生代（约 1.4GB，64位系统）
    ├── Old Pointer Space（包含指针）
    └── Old Data Space（不包含指针）
```

---

### 2. 新生代垃圾回收（Scavenge）

**算法：** Cheney 的半空间复制算法（Semispace Copying）

**步骤：**
1. 将新生代分为 **From Space** 和 **To Space**
2. 对象先分配到 From Space
3. GC 时，将 From Space 中存活的对象复制到 To Space
4. 清空 From Space
5. 交换 From Space 和 To Space

```
Before GC:
From Space: [A][B][C][D]  （D 已死）
To Space:   [空闲]

After GC:
From Space: [空闲]
To Space:   [A][B][C]  （只复制存活对象）
```

**晋升条件（Promotion）：**
- 对象经历过**两次 GC** 仍存活
- To Space 使用超过 **25%**

---

### 3. 老生代垃圾回收

#### **算法 1：标记-清除（Mark-Sweep）**

**步骤：**
1. **标记阶段**：从 GC Roots 开始遍历，标记所有可达对象
2. **清除阶段**：清除未标记的对象

```
Before:
[A][dead][B][dead][C]

After Mark:
[A✓][dead][B✓][dead][C✓]

After Sweep:
[A✓][ ][][B✓][ ][C✓]  （留下内存碎片）
```

**缺点：** 产生内存碎片

---

#### **算法 2：标记-整理（Mark-Compact）**

**步骤：**
1. **标记阶段**：标记存活对象
2. **整理阶段**：将存活对象向一端移动
3. **清理阶段**：清除边界外的内存

```
Before:
[A][dead][B][dead][C]

After Mark:
[A✓][dead][B✓][dead][C✓]

After Compact:
[A✓][B✓][C✓][ ][ ]  （无碎片）
```

**优点：** 解决内存碎片  
**缺点：** 速度较慢（需要移动对象）

---

#### **算法 3：增量标记（Incremental Marking）**

**目的：** 减少 GC 停顿时间（Stop-The-World）

**原理：**
- 将标记工作分散到多个小步骤
- 每个步骤只标记一部分对象
- 与 JavaScript 执行交替进行

```
JavaScript 执行 → 标记 5ms → JavaScript 执行 → 标记 5ms → ...
```

**三色标记法：**
- **白色**：未访问的对象（默认）
- **灰色**：已访问但子对象未访问
- **黑色**：已访问且子对象已访问

```
初始: 所有对象为白色
  ↓
从 GC Roots 开始，标记为灰色
  ↓
遍历灰色对象的引用，子对象标记为灰色，自身标记为黑色
  ↓
直到没有灰色对象
  ↓
清除所有白色对象
```

---

#### **算法 4：并发标记（Concurrent Marking）**

**特点：**
- 主线程继续执行 JavaScript
- GC 线程在后台并发标记
- 减少主线程停顿

```
主线程:  [JS执行]     [JS执行]     [JS执行]
         ↓            ↓            ↓
GC线程:  [标记]────────[标记]────────[标记]
```

---

#### **算法 5：并行标记（Parallel Marking）**

**特点：**
- 使用多个 GC 线程并行标记
- 加快标记速度
- 主线程仍然暂停

```
主线程:    [暂停]
           ↓
GC线程1:   [标记]
GC线程2:   [标记]
GC线程3:   [标记]
GC线程4:   [标记]
```

---

### 4. GC 触发时机

**新生代 GC（Minor GC）：**
- From Space 满了

**老生代 GC（Major GC）：**
- 老生代空间不足
- 晋升对象过多
- 手动调用 `global.gc()`（需开启 --expose-gc）

## 🎓 面试标准答题

### 问题 1："V8 的垃圾回收分为哪几代？"

**标准答案：**
> V8 使用**分代垃圾回收**，将堆内存分为两代：
> 
> **1. 新生代（Young Generation）：**
> - **大小**：1-8MB（默认）
> - **存储**：生命周期短的对象
> - **算法**：Scavenge（Cheney 半空间复制）
> - **特点**：
>   - 分为 From Space 和 To Space
>   - 存活对象复制到 To Space
>   - 速度快（1-5ms）
>   - 晋升条件：经历两次 GC 或 To Space 使用超过 25%
> 
> **2. 老生代（Old Generation）：**
> - **大小**：约 1.4GB（64 位系统）
> - **存储**：生命周期长的对象
> - **算法**：
>   - Mark-Sweep（标记清除）
>   - Mark-Compact（标记整理）
>   - Incremental Marking（增量标记）
>   - Concurrent Marking（并发标记）
> - **特点**：
>   - 速度较慢（50-200ms）
>   - 使用增量标记减少停顿
>   - 使用并发标记提高性能
> 
> **分代原因：**
> - 大部分对象生命周期短（98%）
> - 针对不同生命周期使用不同策略
> - 提高 GC 效率

**加分项：**
> V8 还有**大对象空间（Large Object Space）**，存放超过新生代限制的大对象，直接分配到老生代，使用标记清除算法。

---

### 问题 2："新生代和老生代使用的回收算法是什么？"

**标准答案：**
> **新生代：Scavenge 算法**
> - **原理**：Cheney 的半空间复制算法
> - **步骤**：
>   1. 将新生代分为 From Space 和 To Space
>   2. 对象分配到 From Space
>   3. GC 时，将存活对象复制到 To Space
>   4. 清空 From Space
>   5. 交换 From/To Space
> - **优点**：
>   - 速度快（1-5ms）
>   - 无内存碎片
> - **缺点**：
>   - 空间利用率低（只用 50%）
>   - 只适合小内存
> 
> **老生代：Mark-Sweep + Mark-Compact**
> - **Mark-Sweep（标记清除）：**
>   1. 标记阶段：从 GC Roots 遍历，标记可达对象
>   2. 清除阶段：清除未标记对象
>   - 优点：速度较快
>   - 缺点：产生内存碎片
> 
> - **Mark-Compact（标记整理）：**
>   1. 标记阶段：同 Mark-Sweep
>   2. 整理阶段：将存活对象移动到一端
>   3. 清理阶段：清除边界外内存
>   - 优点：无内存碎片
>   - 缺点：速度较慢
> 
> - **选择策略**：
>   - 内存碎片少时使用 Mark-Sweep
>   - 内存碎片多时使用 Mark-Compact
> 
> **优化技术：**
> - **增量标记**：将标记分散到多个步骤
> - **并发标记**：后台线程并发标记
> - **并行标记**：多线程并行标记

---

### 问题 3："什么是增量标记？"

**标准答案：**
> 增量标记（Incremental Marking）是一种**减少 GC 停顿时间**的优化技术。
> 
> **原理：**
> - 将标记工作**分散到多个小步骤**
> - 每个步骤只标记一部分对象（如 5ms）
> - 与 JavaScript 执行**交替进行**
> 
> **执行流程：**
> ```
> JavaScript 执行 → 标记 5ms → JavaScript 执行 → 标记 5ms → ...
> ```
> 
> **三色标记法：**
> 1. **白色**：未访问（默认）
> 2. **灰色**：已访问但子对象未访问
> 3. **黑色**：已访问且子对象已访问
> 
> **步骤：**
> 1. 所有对象初始为白色
> 2. 从 GC Roots 开始，标记为灰色
> 3. 遍历灰色对象的引用：
>    - 子对象标记为灰色
>    - 自身标记为黑色
> 4. 直到没有灰色对象
> 5. 清除所有白色对象
> 
> **优点：**
> - 减少单次停顿时间（从 50-200ms 降到 5-10ms）
> - 提高应用响应性
> 
> **缺点：**
> - 总耗时可能增加（频繁切换）
> - 需要写屏障（Write Barrier）处理并发修改
> 
> **写屏障：**
> - 监控对象引用的修改
> - 黑色对象引用白色对象时，将白色对象标记为灰色
> - 防止漏标记

## ⚠️ 常见内存泄漏

### 泄漏 1：全局变量

```javascript
// ❌ 内存泄漏
function leak() {
  global.data = new Array(1000000);  // 全局变量不会被回收
}

leak();

// ✅ 正确
function noLeak() {
  const data = new Array(1000000);  // 函数结束后回收
}

noLeak();
```

---

### 泄漏 2：闭包引用

```javascript
// ❌ 内存泄漏
function createClosure() {
  const largeData = new Array(1000000);
  
  return function() {
    console.log(largeData.length);  // 闭包持有 largeData
  };
}

const fn = createClosure();  // largeData 不会被回收

// ✅ 正确
function createClosure() {
  const largeData = new Array(1000000);
  const length = largeData.length;  // 只保存需要的值
  
  return function() {
    console.log(length);
  };
}
```

---

### 泄漏 3：定时器未清除

```javascript
// ❌ 内存泄漏
function startTimer() {
  const data = new Array(1000000);
  
  setInterval(() => {
    console.log(data.length);  // data 不会被回收
  }, 1000);
}

startTimer();

// ✅ 正确
function startTimer() {
  const data = new Array(1000000);
  
  const timer = setInterval(() => {
    console.log(data.length);
  }, 1000);
  
  // 清除定时器
  setTimeout(() => {
    clearInterval(timer);  // data 可以被回收
  }, 10000);
}
```

---

### 泄漏 4：事件监听器未移除

```javascript
// ❌ 内存泄漏
const EventEmitter = require('events');
const emitter = new EventEmitter();

function addListener() {
  const data = new Array(1000000);
  
  emitter.on('event', () => {
    console.log(data.length);  // data 不会被回收
  });
}

addListener();

// ✅ 正确
function addListener() {
  const data = new Array(1000000);
  
  const listener = () => {
    console.log(data.length);
  };
  
  emitter.on('event', listener);
  
  // 移除监听器
  setTimeout(() => {
    emitter.off('event', listener);  // data 可以被回收
  }, 10000);
}
```

## 🔧 内存优化技巧

### 技巧 1：手动触发 GC（开发环境）

```javascript
// node --expose-gc app.js

if (global.gc) {
  console.log('Before GC:', process.memoryUsage().heapUsed);
  global.gc();  // 手动触发 Full GC
  console.log('After GC:', process.memoryUsage().heapUsed);
}
```

---

### 技巧 2：增加堆内存

```bash
# 增加老生代内存（默认 1.4GB）
node --max-old-space-size=4096 app.js  # 4GB

# 增加新生代内存（默认 16MB）
node --max-semi-space-size=32 app.js  # 32MB
```

---

### 技巧 3：使用 WeakMap/WeakSet

```javascript
// ❌ 内存泄漏
const cache = new Map();

function cacheData(obj, data) {
  cache.set(obj, data);  // obj 不会被回收
}

// ✅ 正确
const cache = new WeakMap();

function cacheData(obj, data) {
  cache.set(obj, data);  // obj 可以被回收
}
```

---

### 技巧 4：及时释放大对象

```javascript
// ❌ 内存泄漏
let largeData = new Array(1000000);

setTimeout(() => {
  processData(largeData);
}, 10000);

// ✅ 正确
let largeData = new Array(1000000);

setTimeout(() => {
  processData(largeData);
  largeData = null;  // 及时释放
}, 10000);
```
