## 📚 核心知识点

### 1. 内存泄漏的表现

**症状：**
- 内存持续增长，不回落
- 应用变慢，响应时间增加
- 最终触发 OOM（Out of Memory）崩溃

**监控内存使用：**
```javascript
setInterval(() => {
  const usage = process.memoryUsage();
  console.log({
    rss: `${(usage.rss / 1024 / 1024).toFixed(2)} MB`,       // 常驻内存
    heapTotal: `${(usage.heapTotal / 1024 / 1024).toFixed(2)} MB`,  // 堆总量
    heapUsed: `${(usage.heapUsed / 1024 / 1024).toFixed(2)} MB`,   // 堆使用
    external: `${(usage.external / 1024 / 1024).toFixed(2)} MB`,   // C++ 对象
  });
}, 5000);
```

```bash

```

**打开 DevTools：**
1. 访问 `chrome://inspect`
2. 点击 "Open dedicated DevTools for Node"
3. 或点击 "inspect" 链接

---

### 2. 堆快照分析

**步骤 1：生成快照**
```javascript
// app.js
const leaks = [];

setInterval(() => {
  // 模拟内存泄漏
  leaks.push(new Array(1000000).fill('leak'));
  console.log('Leaks count:', leaks.length);
}, 1000);
```

**步骤 2：对比快照**
1. 点击 "Memory" 标签
2. 选择 "Heap snapshot"
3. 点击 "Take snapshot"（快照 1）
4. 等待 10 秒，再次点击（快照 2）
5. 选择快照 2，Comparison 选择快照 1
6. 按 "Size Delta" 排序，找出增长的对象

**步骤 3：定位代码**
```
(array)  +9.6 MB  +10
  → 点击展开
  → 查看 Retainers（保持引用的对象）
  → 找到 leaks 数组
  → 点击定位到源代码
```

---

### 3. 实时内存监控

**Timeline 录制：**
1. 切换到 "Memory" 标签
2. 选择 "Allocation instrumentation on timeline"
3. 点击 "Start"
4. 执行操作（触发内存泄漏）
5. 点击 "Stop"
6. 分析蓝色柱状图（内存分配）

**特征：**
- 蓝色柱子持续增长 = 内存泄漏
- 蓝色柱子有起有落 = 正常 GC

## 🎯 方案 2：heapdump

### 1. 安装与使用

```bash
npm install heapdump
```

```javascript
const heapdump = require('heapdump');
const fs = require('fs');

// 方法 1：手动触发
heapdump.writeSnapshot('./heap-' + Date.now() + '.heapsnapshot', (err, filename) => {
  console.log('Heap dump written to', filename);
});

// 方法 2：监听信号触发（生产环境）
process.on('SIGUSR2', () => {
  heapdump.writeSnapshot((err, filename) => {
    console.log('Heap dump written to', filename);
  });
});

// 触发：kill -USR2 <pid>
```

---

### 2. 定时快照

```javascript
const heapdump = require('heapdump');

// 每 30 秒生成一次快照
setInterval(() => {
  const filename = `./heap-${Date.now()}.heapsnapshot`;
  heapdump.writeSnapshot(filename, (err) => {
    if (err) {
      console.error('Heap dump error:', err);
    } else {
      console.log('Heap dump written to', filename);
    }
  });
}, 30000);
```

### 3. 对比分析

**步骤：**
1. 生成多个快照（间隔 10-30 秒）

4. 在快照 2 中选择 Comparison，对比快照 1
5. 查看 Size Delta，找出增长的对象

## 💻 实战案例

### 案例 1：全局变量泄漏

```javascript
// leak-global.js
let count = 0;

setInterval(() => {
  global.leaks = global.leaks || [];
  global.leaks.push(new Array(1000000).fill('data'));
  console.log('Leaks:', global.leaks.length);
}, 1000);

// 检测步骤：
// 1. node --inspect leak-global.js
// 2. chrome://inspect
// 3. Take heap snapshot
// 4. 搜索 "leaks"
// 5. 找到 global 对象引用
```

**定位结果：**
```
global → leaks (Array) → [0]: Array(1000000)
  → @1234567 (源代码位置)
```

---

### 案例 2：闭包泄漏

```javascript
// leak-closure.js
function createClosure() {
  const largeData = new Array(1000000).fill('data');
  
  return function() {
    return largeData.length;  // 闭包持有 largeData
  };
}

const closures = [];
setInterval(() => {
  closures.push(createClosure());
  console.log('Closures:', closures.length);
}, 1000);

// 检测：堆快照中搜索 "largeData"
// 定位：Retainers 显示被 closures 数组引用
```

---

### 案例 3：定时器未清除

```javascript
// leak-timer.js
function startTimer() {
  const data = new Array(1000000).fill('data');
  
  setInterval(() => {
    console.log(data.length);  // data 不会被回收
  }, 1000);
}

for (let i = 0; i < 10; i++) {
  startTimer();
}

// 检测：堆快照显示 10 个 Array(1000000)
// 修复：清除定时器
function startTimer() {
  const data = new Array(1000000).fill('data');
  
  const timer = setInterval(() => {
    console.log(data.length);
  }, 1000);
  
  setTimeout(() => {
    clearInterval(timer);  // ✅ 清除定时器
  }, 10000);
}
```

---

### 案例 4：事件监听器泄漏

```javascript
// leak-listener.js
const EventEmitter = require('events');
const emitter = new EventEmitter();

function addListener() {
  const data = new Array(1000000).fill('data');
  
  emitter.on('event', () => {
    console.log(data.length);  // data 不会被回收
  });
}

for (let i = 0; i < 10; i++) {
  addListener();
}

// 检测：EventEmitter._events.event 有 10 个监听器
// 修复：移除监听器
function addListener() {
  const data = new Array(1000000).fill('data');
  
  const listener = () => {
    console.log(data.length);
  };
  
  emitter.on('event', listener);
  
  setTimeout(() => {
    emitter.off('event', listener);  // ✅ 移除监听器
  }, 10000);
}
```

## 🎓 面试标准答题

### 问题 1："如何检测 Node.js 内存泄漏？请给出三种工具方案"

**标准答案：**
> 我会根据不同场景选择三种工具：
> 
> **1. Chrome DevTools（开发环境）**
> - **启动**：`node --inspect app.js`，访问 `chrome://inspect`
> - **优势**：
>   - 可视化界面，易于分析
>   - 实时堆快照对比
>   - 查看 Retainers（引用链）
>   - Timeline 录制，查看分配趋势
> - **步骤**：
>   1. Take heap snapshot（快照 1）
>   2. 触发内存泄漏操作
>   3. Take heap snapshot（快照 2）
>   4. Comparison 对比，按 Size Delta 排序
>   5. 查看增长对象，点击 Retainers 定位代码
> 
> **2. heapdump（生产环境）**
> - **安装**：`npm install heapdump`
> - **优势**：
>   - 生产环境可用
>   - 导出快照文件，离线分析
>   - 支持信号触发（`kill -USR2 <pid>`）
> - **使用**：
>   ```javascript
>   const heapdump = require('heapdump');
>   heapdump.writeSnapshot('./heap.heapsnapshot');
>   ```
> - **分析**：快照导入 Chrome DevTools
> 
> **3. node --heap-prof（性能分析）**
> - **启动**：`node --heap-prof app.js`
> - **优势**：
>   - 自动生成 .heapprofile 文件
>   - CPU + 内存综合分析
>   - 无需额外依赖
> - **分析**：Chrome DevTools → Profiler → Load

**加分项：**
> 在实际项目中，我结合使用这三种工具：
> - **开发阶段**：Chrome DevTools 实时调试
> - **测试阶段**：定时 heapdump，压测时对比快照
> - **生产环境**：监听 SIGUSR2 信号，按需导出快照
> 
> 通过这套方案，我定位并修复了多个内存泄漏问题，包括未清除的定时器、闭包持有大对象、EventEmitter 监听器未移除等。

### 问题 2："Chrome DevTools 如何分析堆快照？"

**标准答案：**
> **分析步骤：**
> 
> **1. 生成快照对比**
> - 快照 1（初始状态）
> - 触发内存泄漏操作
> - 快照 2（泄漏后）
> - Comparison 对比，查看 Size Delta
> 
> **2. 定位泄漏对象**
> - 按 "Size Delta" 排序，找出增长最大的对象
> - 常见泄漏类型：
>   - `(array)`：数组泄漏
>   - `(closure)`：闭包泄漏
>   - `(system)`：系统对象泄漏
> 
> **3. 查看引用链（Retainers）**
> - 点击对象，查看 Retainers 面板
> - 从下往上看引用链：
>   ```
>   Window → global → leaks (Array) → [0]: Array(1000000)
>   ```
> - 找到 GC Root（通常是 global、closure、timer）
> 
> **4. 定位源代码**
> - 点击 Retainers 中的函数名
> - 跳转到源代码位置
> - 修复泄漏（清除引用、移除监听器等）
> 
> **5. 验证修复**
> - 重新生成快照对比
> - 确认 Size Delta 不再增长

## ⚠️ 常见内存泄漏场景

### 场景 1：未清除的全局变量

```javascript
// ❌ 泄漏
global.cache = [];
setInterval(() => {
  global.cache.push(new Array(1000000));
}, 1000);

// ✅ 修复：及时清理
const cache = [];
setInterval(() => {
  cache.push(new Array(1000000));
  if (cache.length > 10) {
    cache.shift();  // 移除旧数据
  }
}, 1000);
```

---

### 场景 2：闭包持有大对象

```javascript
// ❌ 泄漏
function createHandler() {
  const largeData = new Array(1000000);
  return () => largeData.length;
}

// ✅ 修复：只保存需要的值
function createHandler() {
  const largeData = new Array(1000000);
  const length = largeData.length;
  return () => length;
}
```

## 🔧 生产环境实战

### 1. 监控内存增长

```javascript
const os = require('os');

let lastHeapUsed = 0;

setInterval(() => {
  const usage = process.memoryUsage();
  const heapUsed = usage.heapUsed / 1024 / 1024;
  
  if (heapUsed > lastHeapUsed + 50) {  // 增长超过 50MB
    console.warn('⚠️ Memory leak detected:', heapUsed.toFixed(2), 'MB');
    
    // 生成堆快照
    require('heapdump').writeSnapshot((err, filename) => {
      console.log('Heap dump written to', filename);
    });
  }
  
  lastHeapUsed = heapUsed;
}, 60000);  // 每分钟检查
```

### 2. 自动重启机制

```javascript
const MAX_MEMORY = 500;  // 500MB

setInterval(() => {
  const usage = process.memoryUsage();
  const heapUsed = usage.heapUsed / 1024 / 1024;
  
  if (heapUsed > MAX_MEMORY) {
    console.error('💀 Memory limit exceeded, exiting...');
    
    // 导出快照
    require('heapdump').writeSnapshot((err, filename) => {
      console.log('Heap dump written to', filename);
      process.exit(1);  // 退出，让 PM2 重启
    });
  }
}, 30000);
```

## 💪 课后练习

### 练习 1：定位内存泄漏

```javascript
// 以下代码有内存泄漏，请使用 Chrome DevTools 定位

const EventEmitter = require('events');
const emitter = new EventEmitter();

function processData() {
  const data = new Array(1000000).fill('data');
  
  emitter.on('process', () => {
    console.log(data.length);
  });
  
  emitter.emit('process');
}

setInterval(processData, 1000);

// 提示：每次 processData 都会添加新的监听器
```

<details>
<summary>点击查看答案</summary>

**泄漏原因：**
- 每次调用 `processData` 都添加新监听器
- 监听器持有 `data` 引用
- `data` 不会被回收

**修复方案：**
```javascript
function processData() {
  const data = new Array(1000000).fill('data');
  
  const listener = () => {
    console.log(data.length);
  };
  
  emitter.once('process', listener);  // ✅ 使用 once
  emitter.emit('process');
}
```
</details>

### 练习 2：heapdump 实战

```javascript
// 请编写代码：
// 1. 每 10 秒生成一次堆快照
// 2. 保留最近 5 个快照
// 3. 超过 5 个时删除最旧的
```

<details>
<summary>点击查看答案</summary>

```javascript
const heapdump = require('heapdump');
const fs = require('fs');
const path = require('path');

const snapshots = [];
const MAX_SNAPSHOTS = 5;

setInterval(() => {
  const filename = path.join(__dirname, `heap-${Date.now()}.heapsnapshot`);
  
  heapdump.writeSnapshot(filename, (err) => {
    if (err) {
      console.error('Heap dump error:', err);
      return;
    }
    
    console.log('Heap dump written to', filename);
    snapshots.push(filename);
    
    // 删除最旧的快照
    if (snapshots.length > MAX_SNAPSHOTS) {
      const oldFile = snapshots.shift();
      fs.unlinkSync(oldFile);
      console.log('Deleted old snapshot:', oldFile);
    }
  });
}, 10000);
```
</details>
