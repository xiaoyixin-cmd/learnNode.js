## 📚 核心知识点

### 1. 什么是背压（Backpressure）？

**定义：**
> 当**数据生产速度 > 消费速度**时，导致内存中积压大量数据，最终可能内存溢出。

**类比：**
```
水龙头（生产者）→ 水管（缓冲区）→ 水桶（消费者）
           快速出水         积水         慢速排水
                              ↓
                          背压产生
```

**Node.js 中的背压：**
```javascript
// 生产者：快速读取文件（1 GB/s）
const readable = fs.createReadStream('./large-file.txt');

// 消费者：慢速写入（100 MB/s）
const writable = fs.createWriteStream('./output.txt');

// 问题：内存中积压 900 MB/s 的数据
readable.on('data', (chunk) => {
  writable.write(chunk);  // ⚠️ 可能导致内存溢出
});
```

---

### 2. 背压的表现

```javascript
const writable = fs.createWriteStream('./output.txt');

const canContinue = writable.write('data');

if (!canContinue) {
  console.log('背压产生：缓冲区已满');
  // 需要等待 'drain' 事件
}
```

**关键信号：**
- `writable.write()` 返回 `false`：缓冲区已满
- `writable` 触发 `'drain'` 事件：缓冲区已清空，可以继续写入

## 💻 背压处理方案

### 方案 1：手动处理（监听 drain）

```javascript
const fs = require('fs');

const readable = fs.createReadStream('./input.txt');
const writable = fs.createWriteStream('./output.txt');

readable.on('data', (chunk) => {
  const canContinue = writable.write(chunk);
  
  if (!canContinue) {
    console.log('背压产生，暂停读取');
    readable.pause();  // 暂停读取
  }
});

writable.on('drain', () => {
  console.log('缓冲区已清空，恢复读取');
  readable.resume();  // 恢复读取
});

readable.on('end', () => {
  writable.end();
});
```

**执行流程：**
```
1. readable 读取数据 → writable.write(chunk)
2. write() 返回 false → 缓冲区满
3. readable.pause() → 暂停读取
4. writable 触发 'drain' → 缓冲区清空
5. readable.resume() → 恢复读取
```

---

### 方案 2：使用 pipe()（自动处理）

```javascript
const fs = require('fs');

const readable = fs.createReadStream('./input.txt');
const writable = fs.createWriteStream('./output.txt');

readable.pipe(writable);  // 自动处理背压

// pipe() 内部实现（简化版）：
function pipe(readable, writable) {
  readable.on('data', (chunk) => {
    const canContinue = writable.write(chunk);
    if (!canContinue) {
      readable.pause();
    }
  });
  
  writable.on('drain', () => {
    readable.resume();
  });
}
```

**优势：**
- ✅ 自动暂停/恢复
- ✅ 自动处理错误
- ✅ 自动清理资源

---

### 方案 3：Transform Stream（数据转换）

```javascript
const { Transform } = require('stream');

const transformStream = new Transform({
  highWaterMark: 16 * 1024,  // 16 KB 缓冲区
  
  transform(chunk, encoding, callback) {
    // 数据转换（如：大写）
    const transformed = chunk.toString().toUpperCase();
    
    // 推送数据
    this.push(transformed);
    
    // 完成
    callback();
  }
});

fs.createReadStream('./input.txt')
  .pipe(transformStream)
  .pipe(fs.createWriteStream('./output.txt'));
```

## 💻 实战案例

### 案例 1：复制大文件（避免内存溢出）

```javascript
const fs = require('fs');

// ❌ 错误：一次性读取整个文件（内存溢出）
const data = fs.readFileSync('./large-file.txt');  // 10 GB 文件
fs.writeFileSync('./copy.txt', data);  // OOM（内存溢出）

// ✅ 正确：使用 Stream（分块读取）
const readable = fs.createReadStream('./large-file.txt', {
  highWaterMark: 64 * 1024  // 64 KB 缓冲区
});

const writable = fs.createWriteStream('./copy.txt');

readable.pipe(writable);

writable.on('finish', () => {
  console.log('Copy complete');
});

// 内存占用：< 1 MB（仅缓冲区）
```

### 案例 3：数据库批量写入

```javascript
const { Writable } = require('stream');
const db = require('./db');

class DatabaseWriter extends Writable {
  constructor() {
    super({
      highWaterMark: 1000,  // 最多缓存 1000 条记录
      objectMode: true
    });
    
    this.batch = [];
    this.batchSize = 100;
  }
  
  _write(record, encoding, callback) {
    this.batch.push(record);
    
    if (this.batch.length >= this.batchSize) {
      this.flush(callback);
    } else {
      callback();
    }
  }
  
  async flush(callback) {
    if (this.batch.length === 0) {
      return callback();
    }
    
    try {
      await db.insert(this.batch);
      console.log(`Inserted ${this.batch.length} records`);
      this.batch = [];
      callback();
    } catch (error) {
      callback(error);
    }
  }
  
  _final(callback) {
    this.flush(callback);  // 写入剩余数据
  }
}

// 使用
const dbWriter = new DatabaseWriter();

const readable = fs.createReadStream('./data.json')
  .pipe(JSONStream.parse('*'))
  .pipe(dbWriter);

dbWriter.on('finish', () => {
  console.log('All data inserted');
});
```

### 案例 4：限速下载（控制速率）

```javascript
const { Transform } = require('stream');

class RateLimiter extends Transform {
  constructor(bytesPerSecond) {
    super();
    this.bytesPerSecond = bytesPerSecond;
    this.chunks = [];
    this.timer = null;
  }
  
  _transform(chunk, encoding, callback) {
    this.chunks.push(chunk);
    
    if (!this.timer) {
      this.timer = setInterval(() => {
        if (this.chunks.length === 0) {
          clearInterval(this.timer);
          this.timer = null;
          return;
        }
        
        const chunk = this.chunks.shift();
        this.push(chunk);
      }, 1000 / (this.bytesPerSecond / chunk.length));
    }
    
    callback();
  }
}

// 限速 1 MB/s
const limiter = new RateLimiter(1024 * 1024);

fs.createReadStream('./large-file.txt')
  .pipe(limiter)
  .pipe(fs.createWriteStream('./output.txt'));
```

## 🎯 highWaterMark 详解

### 1. 什么是 highWaterMark？

> **高水位线**：Stream 内部缓冲区的最大字节数

```javascript
const readable = fs.createReadStream('./file.txt', {
  highWaterMark: 16 * 1024  // 16 KB
});

// 当内部缓冲区超过 16 KB 时，停止从文件读取
```

---

### 2. 不同 Stream 的默认值

| Stream 类型 | 默认 highWaterMark |
|------------|-------------------|
| Readable | 16 KB |
| Writable | 16 KB |
| Transform | 16 KB |
| objectMode | 16 个对象 |

---

### 3. 调优 highWaterMark

```javascript
// 场景 1：大文件复制（增加缓冲区）
const readable = fs.createReadStream('./large-file.txt', {
  highWaterMark: 1024 * 1024  // 1 MB
});

// 场景 2：低内存环境（减少缓冲区）
const readable = fs.createReadStream('./file.txt', {
  highWaterMark: 1024  // 1 KB
});

// 场景 3：对象模式（限制对象数量）
const readable = new Readable({
  objectMode: true,
  highWaterMark: 100  // 最多 100 个对象
});
```

## 🎓 面试标准答题

### 问题 1："什么是背压？如何处理背压问题？"

**标准答案：**
> **背压（Backpressure）** 是指数据生产速度超过消费速度，导致内存中积压大量数据的现象。
> 
> **表现：**
> - `writable.write()` 返回 `false`
> - 内部缓冲区超过 `highWaterMark`
> - 内存占用持续增长
> 
> **处理方案：**
> 
> **1. 手动处理**
> ```javascript
> readable.on('data', (chunk) => {
>   const canContinue = writable.write(chunk);
>   if (!canContinue) {
>     readable.pause();  // 暂停读取
>   }
> });
> 
> writable.on('drain', () => {
>   readable.resume();  // 恢复读取
> });
> ```
> 
> **2. 使用 pipe()（推荐）**
> ```javascript
> readable.pipe(writable);  // 自动处理背压
> ```
> 
> **3. 调整 highWaterMark**
> ```javascript
> const writable = fs.createWriteStream('./file.txt', {
>   highWaterMark: 1024 * 1024  // 增大缓冲区
> });
> ```

**加分项：**
> 在实际项目中，我处理过视频流上传，客户端上传速度（10 MB/s）远大于服务器写入磁盘速度（2 MB/s），通过监听 `drain` 事件暂停客户端上传，避免了服务器内存溢出。

---

### 问题 2："Stream 的 `write()` 返回 `false` 意味着什么？"

**标准答案：**
> `writable.write()` 返回 `false` 表示：
> 
> **1. 内部缓冲区已满**
> - 缓冲区大小超过 `highWaterMark`
> - 数据尚未完全写入底层资源（文件、网络）
> 
> **2. 应该做什么？**
> - ✅ 暂停数据生产（`readable.pause()`）
> - ✅ 等待 `'drain'` 事件
> - ❌ 不要继续 `write()`（会导致内存持续增长）
> 
> **3. 示例**
> ```javascript
> const canContinue = writable.write(chunk);
> 
> if (!canContinue) {
>   console.log('缓冲区满，暂停写入');
>   readable.pause();
>   
>   writable.once('drain', () => {
>     console.log('缓冲区已清空，恢复写入');
>     readable.resume();
>   });
> }
> ```
> 
> **4. pipe() 的自动处理**
> ```javascript
> readable.pipe(writable);
> // pipe() 内部自动监听 write() 返回值和 drain 事件
> ```

## ⚠️ 常见误区

### 误区 1：忽略 write() 返回值

```javascript
// ❌ 错误：忽略返回值，持续写入
readable.on('data', (chunk) => {
  writable.write(chunk);  // 可能返回 false，但继续写入
});

// ✅ 正确：检查返回值
readable.on('data', (chunk) => {
  const canContinue = writable.write(chunk);
  if (!canContinue) {
    readable.pause();
  }
});
```

---

### 误区 2：不使用 pipe()

```javascript
// ❌ 错误：手动实现（容易出错）
readable.on('data', (chunk) => {
  writable.write(chunk);
});

// ✅ 正确：使用 pipe()
readable.pipe(writable);
```

---

### 误区 3：highWaterMark 设置过大

```javascript
// ❌ 错误：设置过大（占用过多内存）
const readable = fs.createReadStream('./file.txt', {
  highWaterMark: 100 * 1024 * 1024  // 100 MB（内存占用过高）
});

// ✅ 正确：合理设置（16 KB - 1 MB）
const readable = fs.createReadStream('./file.txt', {
  highWaterMark: 64 * 1024  // 64 KB
});
```
