## 📚 核心知识点

### 1. JSON.stringify() 的性能问题

**性能瓶颈：**
1. **动态类型检测**：每个属性都要检查类型
2. **递归遍历**：深层对象遍历慢
3. **内存分配**：频繁创建字符串
4. **循环引用检测**：需要维护访问记录

**性能测试：**
```javascript
const data = {
  id: 1,
  name: 'John',
  email: 'john@example.com',
  age: 30,
  active: true
};

console.time('JSON.stringify');
for (let i = 0; i < 1e6; i++) {
  JSON.stringify(data);
}
console.timeEnd('JSON.stringify');  // ~400ms
```

---

### 2. fast-json-stringify 原理

**核心思想：**
> 使用 **JSON Schema** 预定义数据结构，生成**优化的序列化函数**

**性能对比：**
```javascript
const fastJson = require('fast-json-stringify');

// 定义 Schema
const stringify = fastJson({
  type: 'object',
  properties: {
    id: { type: 'integer' },
    name: { type: 'string' },
    email: { type: 'string' },
    age: { type: 'integer' },
    active: { type: 'boolean' }
  }
});

console.time('fast-json-stringify');
for (let i = 0; i < 1e6; i++) {
  stringify(data);
}
console.timeEnd('fast-json-stringify');  // ~150ms（快 2.7 倍）
```

**为什么快？**
- ✅ 跳过类型检测（Schema 已定义）
- ✅ 无递归（生成平铺的代码）
- ✅ 字符串拼接优化
- ✅ 无循环引用检测（信任 Schema）

## 💻 基础用法

### 1. 简单对象序列化

```javascript
const fastJson = require('fast-json-stringify');

// 定义 Schema
const stringify = fastJson({
  type: 'object',
  properties: {
    id: { type: 'integer' },
    name: { type: 'string' }
  }
});

const result = stringify({ id: 1, name: 'John' });
console.log(result);  // {"id":1,"name":"John"}
```

---

### 2. 数组序列化

```javascript
const stringify = fastJson({
  type: 'array',
  items: {
    type: 'object',

    }
  }
});

const users = [
  { id: 1, name: 'Alice' },
  { id: 2, name: 'Bob' }

```

### 1. Date 类型

const stringify = fastJson({

### 2. BigInt 类型

```javascript
// ❌ JSON.stringify 不支持 BigInt
const data = { value: 9007199254740991n };
JSON.stringify(data);  // TypeError: Do not know how to serialize a BigInt

// ✅ 使用 fast-json-stringify
const stringify = fastJson({
  type: 'object',
  properties: {
    value: { type: 'integer' }
  }
});

console.log(stringify({ value: 9007199254740991n }));
// {"value":9007199254740991}
```

---

### 3. 循环引用

```javascript
// ❌ JSON.stringify 报错
const obj = { name: 'Alice' };
obj.self = obj;
JSON.stringify(obj);  // TypeError: Converting circular structure to JSON

// ✅ 使用 flatted 库
const { stringify } = require('flatted');
console.log(stringify(obj));
// [{"name":"Alice","self":"0"}]

// 或手动处理
const seen = new WeakSet();
const replacer = (key, value) => {
  if (typeof value === 'object' && value !== null) {
    if (seen.has(value)) return '[Circular]';
    seen.add(value);
  }
  return value;
};

console.log(JSON.stringify(obj, replacer));
// {"name":"Alice","self":"[Circular]"}
```

## 🎓 面试标准答题

### 问题 1："JSON.stringify() 的性能瓶颈在哪？如何优化？"

**标准答案：**
> **性能瓶颈：**
> 
> **1. 动态类型检测**
> ```javascript
> // 每个属性都要检查类型
> if (typeof value === 'string') { ... }
> else if (typeof value === 'number') { ... }
> else if (Array.isArray(value)) { ... }
> ```
> 
> **2. 递归遍历**
> - 深层对象需要递归调用
> - 函数调用栈开销大
> 
> **3. 内存分配**
> - 频繁创建字符串
> - 字符串拼接需要新分配内存
> 
> **4. 循环引用检测**
> - 需要维护 `WeakSet` 记录访问过的对象
> - 额外的内存和时间开销
> 
> **优化方案：**
> 
> **方案 1：使用 fast-json-stringify**
> ```javascript
> const fastJson = require('fast-json-stringify');
> const stringify = fastJson({
>   type: 'object',
>   properties: {
>     id: { type: 'integer' },
>     name: { type: 'string' }
>   }
> });
> ```
> 
> **方案 2：自定义 replacer**
> ```javascript
> JSON.stringify(obj, (key, value) => {
>   if (key === 'password') return undefined;  // 跳过敏感字段
>   return value;
> });
> ```
> 
> **方案 3：使用 JSON Schema**
> - 预定义数据结构
> - 生成优化的序列化代码
> 
> **性能对比：**
> - JSON.stringify()：~400ms/100万次
> - fast-json-stringify：~150ms/100万次（快 2.7 倍）

**加分项：**
> 在实际项目中，我们的 API 返回大型数组（10000+ 条记录），使用 fast-json-stringify 后，响应时间从 50ms 降到 15ms，API 吞吐量提升 3 倍。

---

### 问题 2："fast-json-stringify 为什么比原生快 2-5 倍？"

**标准答案：**
> **核心原理：使用 JSON Schema 预编译序列化函数**
> 
> **1. 跳过类型检测**
> ```javascript
> // 原生 JSON.stringify（需要类型检测）
> if (typeof obj.name === 'string') { ... }
> 
> // fast-json-stringify（已知类型）
> str += '"name":"' + obj.name + '"';  // 直接拼接
> ```
> 
> **2. 无递归（平铺代码）**
> ```javascript
> // 原生：递归调用
> function stringify(obj) {
>   for (let key in obj) {
>     stringify(obj[key]);  // 递归
>   }
> }
> 
> // fast-json-stringify：生成平铺代码
> str = '{"id":' + obj.id + ',"name":"' + obj.name + '"}';
> ```
> 
> **3. 字符串拼接优化**
> - 原生：频繁创建临时字符串
> - fast-json-stringify：预分配 Buffer，减少内存分配
> 
> **4. 无循环引用检测**
> - 信任 Schema，不维护 WeakSet
> - 节省内存和时间
> 
> **5. 示例对比**
> ```javascript
> // 原生（动态）
> JSON.stringify({ id: 1, name: 'Alice' });
> // 内部：类型检测 + 递归 + 循环引用检测
> 
> // fast-json-stringify（预编译）
> const stringify = fastJson({
>   type: 'object',
>   properties: {
>     id: { type: 'integer' },
>     name: { type: 'string' }
>   }
> });
> stringify({ id: 1, name: 'Alice' });
> // 内部：直接字符串拼接（无检测）
> ```

## ⚠️ 常见误区

### 误区 1：Schema 定义不完整

```javascript
// ❌ 错误：Schema 缺少字段
const stringify = fastJson({
  type: 'object',
  properties: {
    id: { type: 'integer' }
    // 缺少 name 字段
  }
});

const result = stringify({ id: 1, name: 'Alice' });
console.log(result);  // {"id":1}（name 被忽略）

// ✅ 正确：定义所有字段
const stringify = fastJson({
  type: 'object',
  properties: {
    id: { type: 'integer' },
    name: { type: 'string' }
  }
});
```

---

### 误区 2：类型不匹配

```javascript
// ❌ 错误：类型不匹配
const stringify = fastJson({
  type: 'object',
  properties: {
    age: { type: 'integer' }
  }
});

stringify({ age: '25' });  // 字符串 '25'，不是整数
// 结果：{"age":"25"}（未做类型转换）

// ✅ 正确：确保数据类型正确
stringify({ age: 25 });
```

---

### 误区 3：忘记处理 null/undefined

```javascript
// ❌ 错误：未处理 null
const stringify = fastJson({
  type: 'object',
  properties: {
    name: { type: 'string' }
  }
});

stringify({ name: null });  // {"name":null}

// ✅ 正确：定义 nullable
const stringify = fastJson({
  type: 'object',
  properties: {
    name: { type: ['string', 'null'] }
  }
});
```
