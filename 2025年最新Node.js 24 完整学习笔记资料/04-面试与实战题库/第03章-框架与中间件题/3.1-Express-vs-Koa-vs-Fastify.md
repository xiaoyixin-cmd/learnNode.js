## 💻 核心差异详解

### 差异 1：中间件机制

#### Express（传统回调）
```javascript
const express = require('express');
const app = express();

// 中间件 1
app.use((req, res, next) => {
  console.log('Middleware 1 - before');
  next();  // 调用下一个中间件
  console.log('Middleware 1 - after');  // ❌ 不会等待下一个中间件完成
});

// 中间件 2
app.use((req, res, next) => {
  setTimeout(() => {
    console.log('Middleware 2');
    next();
  }, 100);
});

app.get('/', (req, res) => {
  res.json({ msg: 'Hello' });
});

// 输出顺序：
// Middleware 1 - before
// Middleware 1 - after  ← 立即执行，不等待 Middleware 2
// Middleware 2
```

**问题：**
- `next()` 不等待异步操作完成
- 无法保证中间件执行顺序
- 错误处理复杂

---

#### Koa（洋葱模型）
```javascript
const Koa = require('koa');
const app = new Koa();

// 中间件 1
app.use(async (ctx, next) => {
  console.log('Middleware 1 - before');
  await next();  // ✅ 等待下一个中间件完成
  console.log('Middleware 1 - after');
});

// 中间件 2
app.use(async (ctx, next) => {
  await new Promise(resolve => setTimeout(resolve, 100));
  console.log('Middleware 2');
  await next();
});

app.use(async ctx => {
  ctx.body = { msg: 'Hello' };
});

// 输出顺序：
// Middleware 1 - before
// Middleware 2
// Middleware 1 - after  ← 等待所有中间件完成后执行
```

**优势：**
- ✅ `await next()` 等待异步完成
- ✅ 洋葱模型保证执行顺序
- ✅ 错误处理简洁（try/catch）

---

#### Fastify（Schema 驱动）
```javascript
const fastify = require('fastify')();

// 定义 Schema（类型校验 + 序列化优化）
const schema = {
  response: {
    200: {
      type: 'object',
      properties: {
        hello: { type: 'string' }
      }
    }
  }
};

// 路由（自动使用 Schema 优化序列化）
fastify.get('/', { schema }, async (request, reply) => {
  return { hello: 'world' };
});

fastify.listen({ port: 3000 });
```

**优势：**
- ✅ Schema 自动验证请求参数
- ✅ 序列化性能提升 2-5 倍（fast-json-stringify）
- ✅ 自动生成 API 文档

### 差异 2：性能优化原理

#### Express 的性能瓶颈
```javascript
// 1. 动态路由匹配（慢）
app.get('/users/:id', (req, res) => {
  // 每次请求都要解析路由参数
  const userId = req.params.id;
  res.json({ userId });
});

// 2. JSON 序列化（慢）
app.get('/data', (req, res) => {
  const data = { /* 大对象 */ };
  res.json(data);  // 使用原生 JSON.stringify（慢）
});

// 3. 中间件链（慢）
app.use(middleware1);
app.use(middleware2);
// ... 10个中间件
// 每个请求都要遍历所有中间件
```

---

#### Koa 的性能优化
```javascript
// 1. 轻量级核心
const Koa = require('koa');
const app = new Koa();

// Koa 核心只有 4 个文件（Express 有 30+）
// 更少的代码 = 更快的启动 + 更小的内存占用

// 2. 异步流程控制
app.use(async (ctx, next) => {
  const start = Date.now();
  await next();
  const ms = Date.now() - start;
  ctx.set('X-Response-Time', `${ms}ms`);
});

// 3. Context 对象（减少对象创建）
app.use(async ctx => {
  ctx.body = { msg: 'Hello' };  // ctx 封装了 req 和 res
});
```

---

#### Fastify 的性能优化
```javascript
// 1. Schema 驱动的序列化
const fastify = require('fastify')();

const schema = {
  response: {
    200: {
      type: 'object',
      properties: {
        id: { type: 'integer' },
        name: { type: 'string' }
      }
    }
  }
};

fastify.get('/user', { schema }, async () => {
  return { id: 1, name: 'Alice' };
  // 内部使用 fast-json-stringify（比原生快 2-5 倍）
});

// 2. 高性能路由（find-my-way）
// 使用 Radix Tree 优化路由匹配（O(log n) vs O(n)）

// 3. 请求/响应复用
// 复用对象，减少 GC 压力

// 4. 插件系统（封装复用）
fastify.register(require('@fastify/jwt'), {
  secret: 'supersecret'
});
```

## 💻 实战案例

### 案例 1：同一个 API 用三个框架实现

#### Express 版本
```javascript
const express = require('express');
const app = express();

app.use(express.json());

// 日志中间件
app.use((req, res, next) => {
  console.log(`${req.method} ${req.url}`);
  next();
});

// 用户路由
app.get('/users/:id', async (req, res) => {
  try {
    const user = await db.users.findById(req.params.id);
    res.json(user);
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});

app.listen(3000);
```

---

#### Koa 版本
```javascript
const Koa = require('koa');
const Router = require('@koa/router');
const bodyParser = require('koa-bodyparser');

const app = new Koa();
const router = new Router();

app.use(bodyParser());

// 日志中间件（洋葱模型）
app.use(async (ctx, next) => {
  console.log(`${ctx.method} ${ctx.url}`);
  await next();
});

// 用户路由
router.get('/users/:id', async ctx => {
  const user = await db.users.findById(ctx.params.id);
  ctx.body = user;
});

// 全局错误处理
app.use(async (ctx, next) => {
  try {
    await next();
  } catch (error) {
    ctx.status = 500;
    ctx.body = { error: error.message };
  }
});

app.use(router.routes());
app.listen(3000);
```

---

#### Fastify 版本
```javascript
const fastify = require('fastify')({ logger: true });

// 定义 Schema
const userSchema = {
  response: {
    200: {
      type: 'object',
      properties: {
        id: { type: 'integer' },
        name: { type: 'string' },
        email: { type: 'string' }
      }
    }
  }
};

// 用户路由（自动验证 + 优化序列化）
fastify.get('/users/:id', { schema: userSchema }, async (request) => {
  const user = await db.users.findById(request.params.id);
  return user;
});

// 全局错误处理
fastify.setErrorHandler((error, request, reply) => {
  reply.status(500).send({ error: error.message });
});

fastify.listen({ port: 3000 });
```

## 🎓 面试标准答题

### 问题 1："Express、Koa、Fastify 的性能差异是什么？为什么 Fastify 最快？"

**标准答案：**
> **性能排序：Fastify > Koa > Express**
> 
> **性能数据（基准测试）：**
> - Express: 15,234 req/s（基准）
> - Koa: 18,567 req/s（快 22%）
> - Fastify: 45,789 req/s（快 200%）
> 
> **Fastify 快的根本原因：**
> 
> **1. Schema 驱动的序列化优化**
> ```javascript
> // Fastify 使用 fast-json-stringify（预编译）
> const schema = {
>   response: {
>     200: {
>       type: 'object',
>       properties: {
>         id: { type: 'integer' },
>         name: { type: 'string' }
>       }
>     }
>   }
> };
> // 性能提升：2-5 倍
> ```
> 
> **2. 高性能路由（find-my-way）**
> - 使用 Radix Tree（基数树）
> - 路由匹配：O(log n) vs Express 的 O(n)
> 
> **3. 请求/响应对象复用**
> - 减少对象创建
> - 降低 GC 压力
> 
> **4. 异步优化**
> - 全面支持 async/await
> - 自动处理 Promise rejection

**加分项：**
> 在实际项目中，我们从 Express 迁移到 Fastify，API 响应时间从平均 50ms 降到 18ms，QPS 从 5000 提升到 15000，服务器成本降低 60%。

---

### 问题 2："Express 和 Koa 中间件机制的区别？"

**标准答案：**
> **核心区别：回调模式 vs 洋葱模型**
> 
> **Express（回调模式）：**
> ```javascript
> app.use((req, res, next) => {
>   console.log('Before');
>   next();  // ❌ 不等待异步完成
>   console.log('After');  // 立即执行
> });
> ```
> 
> **Koa（洋葱模型）：**
> ```javascript
> app.use(async (ctx, next) => {
>   console.log('Before');
>   await next();  // ✅ 等待所有后续中间件完成
>   console.log('After');  // 最后执行
> });
> ```
> 
> **执行流程对比：**
> 
> **Express：**
> ```
> M1-before → M2-before → M3-before → Response
> M1-after  → M2-after  → M3-after
> （After 不等待 Response）
> ```
> 
> **Koa（洋葱模型）：**
> ```
> M1-before → M2-before → M3-before → Response
>                                   → M3-after
>                         → M2-after
>           → M1-after
> （层层回溯）
> ```
> 
> **洋葱模型优势：**
> 1. ✅ 支持异步流程控制
> 2. ✅ 错误处理简洁（try/catch）
> 3. ✅ 执行顺序可预测
> 4. ✅ 适合日志、性能监控场景
