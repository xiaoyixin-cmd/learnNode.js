## 📚 核心知识点

### 1. Session 在分布式环境下的问题

**单机环境：**
```javascript
// 服务器 A
const session = require('express-session');
app.use(session({
  secret: 'secret-key',
  resave: false,
  saveUninitialized: true
}));

// Session 存储在内存中，只有服务器 A 能访问
```

**分布式环境（问题）：**
```
用户请求 → 负载均衡器 →
           ├─ 服务器 A（Session 1）
           ├─ 服务器 B（Session 2）
           └─ 服务器 C（Session 3）

问题：
1. 用户在服务器 A 登录，下次请求到服务器 B，Session 丢失
2. 每台服务器维护独立的 Session，无法共享
```

---

### 2. Session 存储方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **内存（默认）** | 快速 | 无法共享、重启丢失 | 单机开发 |
| **Cookie** | 无需服务端存储 | 大小限制 4KB、安全性低 | 简单数据 |
| **数据库（MySQL）** | 持久化 | 性能差 | 低并发 |
| **Redis** | 快速、可共享 | 需额外维护 | 高并发（推荐） |
| **Memcached** | 快速 | 无持久化 | 临时数据 |
| **Redis Cluster** | 高可用、可扩展 | 配置复杂 | 大规模分布式 |

## 💻 Redis 存储 Session

### 1. 基础实现（express-session + connect-redis）

```javascript
const express = require('express');
const session = require('express-session');
const RedisStore = require('connect-redis').default;
const { createClient } = require('redis');

const app = express();

// 创建 Redis 客户端
const redisClient = createClient({
  host: 'localhost',
  port: 6379,
  legacyMode: true
});

redisClient.connect().catch(console.error);

// 配置 Session（使用 Redis 存储）
app.use(session({
  store: new RedisStore({ client: redisClient }),
  secret: 'your-secret-key',
  resave: false,
  saveUninitialized: false,
  cookie: {
    secure: false,  // HTTPS 环境设为 true
    httpOnly: true,
    maxAge: 24 * 60 * 60 * 1000  // 1 天
  }
}));

// 登录
app.post('/login', (req, res) => {
  const { username, password } = req.body;
  
  // 验证用户...
  
  // 设置 Session
  req.session.userId = 123;
  req.session.username = username;
  
  res.json({ message: 'Login successful' });
});

// 获取用户信息
app.get('/profile', (req, res) => {
  if (!req.session.userId) {
    return res.status(401).json({ error: 'Unauthorized' });
  }
  
  res.json({
    userId: req.session.userId,
    username: req.session.username
  });
});

// 登出
app.post('/logout', (req, res) => {
  req.session.destroy(err => {
    if (err) {
      return res.status(500).json({ error: 'Logout failed' });
    }
    res.json({ message: 'Logout successful' });
  });
});

app.listen(3000);
```

### 2. Koa 实现（koa-session + koa-redis）

```javascript
const Koa = require('koa');
const session = require('koa-session');
const Redis = require('ioredis');

const app = new Koa();

// 创建 Redis 客户端
const redis = new Redis({
  host: 'localhost',
  port: 6379
});

// Session 配置
app.keys = ['your-secret-key'];

const SESSION_CONFIG = {
  key: 'koa.sess',
  maxAge: 24 * 60 * 60 * 1000,  // 1 天
  httpOnly: true,
  
  // 使用 Redis 存储
  store: {
    async get(key) {
      const data = await redis.get(`SESSION:${key}`);
      return data ? JSON.parse(data) : null;
    },
    
    async set(key, sess, maxAge) {
      await redis.set(
        `SESSION:${key}`,
        JSON.stringify(sess),
        'PX',
        maxAge
      );
    },
    
    async destroy(key) {
      await redis.del(`SESSION:${key}`);
    }
  }
};

app.use(session(SESSION_CONFIG, app));

// 登录
app.use(async ctx => {
  if (ctx.path === '/login' && ctx.method === 'POST') {
    const { username, password } = ctx.request.body;
    
    // 验证用户...
    
    // 设置 Session
    ctx.session.userId = 123;
    ctx.session.username = username;
    
    ctx.body = { message: 'Login successful' };
  }
});

// 获取用户信息
app.use(async ctx => {
  if (ctx.path === '/profile') {
    if (!ctx.session.userId) {
      ctx.status = 401;
      ctx.body = { error: 'Unauthorized' };
      return;
    }
    
    ctx.body = {
      userId: ctx.session.userId,
      username: ctx.session.username
    };
  }
});

app.listen(3000);
```

## 💻 Redis Cluster 实现

### 1. Redis Cluster 架构

```
客户端 →
  ├─ Master 1 (Slot 0-5460)    → Slave 1
  ├─ Master 2 (Slot 5461-10922) → Slave 2
  └─ Master 3 (Slot 10923-16383) → Slave 3

特性：
- 16384 个哈希槽（Hash Slot）
- 数据分片存储
- 主从复制
- 自动故障转移
```

---

### 2. 连接 Redis Cluster

```javascript
const Redis = require('ioredis');

// 创建 Redis Cluster 客户端
const redis = new Redis.Cluster([
  { host: '127.0.0.1', port: 7000 },
  { host: '127.0.0.1', port: 7001 },
  { host: '127.0.0.1', port: 7002 }
], {
  redisOptions: {
    password: 'your-password'
  },
  enableReadyCheck: true,
  maxRetriesPerRequest: 3
});

// Session Store（支持 Redis Cluster）
const store = {
  async get(key) {
    const data = await redis.get(`SESSION:${key}`);
    return data ? JSON.parse(data) : null;
  },
  
  async set(key, sess, maxAge) {
    await redis.set(
      `SESSION:${key}`,
      JSON.stringify(sess),
      'PX',
      maxAge
    );
  },
  
  async destroy(key) {
    await redis.del(`SESSION:${key}`);
  }
};

// 使用
app.use(session({
  store,
  secret: 'your-secret-key',
  resave: false,
  saveUninitialized: false,
  cookie: { maxAge: 24 * 60 * 60 * 1000 }
}));
```

### 3. 哈希槽（Hash Slot）计算

```javascript
// Redis Cluster 如何确定 Key 存储在哪个节点？

// 1. 计算 CRC16 校验和
const crc16 = require('crc').crc16;

function getSlot(key) {
  const slot = crc16(key) % 16384;
  return slot;
}

// 示例
console.log(getSlot('SESSION:abc123'));  // 5432 → Master 1
console.log(getSlot('SESSION:xyz789'));  // 12345 → Master 3

// 2. 哈希标签（Hash Tag）- 强制相同槽
const key1 = 'SESSION:{user:123}:data';
const key2 = 'SESSION:{user:123}:meta';

// {} 内的内容用于计算槽，确保存储在同一节点
console.log(getSlot(key1));  // 相同
console.log(getSlot(key2));  // 相同
```

**架构：**
```

        ↓              ↓              ↓
   服务器 A        服务器 B        服务器 C
        └─────────────┼─────────────┘
                       ↓
                 Redis Cluster
              (Session 共享存储)
```

**Nginx 配置：**
```nginx

  server 127.0.0.1:3001;
  server 127.0.0.1:3002;

server {
  listen 80;
  
  location / {
    proxy_pass http://backend;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
  }
}
```

**Node.js 服务（3 个实例）：**

### 案例 2：Session 持久化策略

```javascript
const Redis = require('ioredis');

const redis = new Redis.Cluster([...]);

// 高级 Session Store（支持持久化）
class SessionStore {
  constructor(redis) {
    this.redis = redis;
  }
  
  // 获取 Session
  async get(sid) {
    const key = `SESSION:${sid}`;
    const data = await this.redis.get(key);
    
    if (data) {
      const session = JSON.parse(data);
      
      // 记录访问时间
      session.lastAccess = Date.now();
      await this.set(sid, session, session.cookie.maxAge);
      
      return session;
    }
    
    return null;
  }
  
  // 设置 Session
  async set(sid, session, maxAge) {
    const key = `SESSION:${sid}`;
    
    // 1. 存储到 Redis（临时）
    await this.redis.set(
      key,
      JSON.stringify(session),
      'PX',
      maxAge
    );
    
    // 2. 持久化到数据库（可选）
    if (session.userId) {
      await db.sessions.upsert({
        id: sid,
        userId: session.userId,
        data: JSON.stringify(session),
        expiresAt: new Date(Date.now() + maxAge)
      });
    }
  }
  
  // 删除 Session
  async destroy(sid) {
    const key = `SESSION:${sid}`;
    
    // 1. 从 Redis 删除
    await this.redis.del(key);
    
    // 2. 从数据库删除
    await db.sessions.delete({ id: sid });
  }
  
  // 清理过期 Session（定时任务）
  async cleanup() {
    await db.sessions.delete({
      expiresAt: { $lt: new Date() }
    });
  }
}

// 使用
const store = new SessionStore(redis);

app.use(session({
  store,
  secret: 'your-secret-key',
  resave: false,
  saveUninitialized: false
}));

// 定时清理过期 Session（每小时）
setInterval(() => {
  store.cleanup().catch(console.error);
}, 60 * 60 * 1000);
```

## 🎓 面试标准答题

### 问题 1："分布式系统中 Session 如何共享？"

**标准答案：**
> **问题：** 多台服务器各自维护 Session，无法共享
> 
> **解决方案：**
> 
> **方案 1：Sticky Session（IP 绑定）**
> - 负载均衡器将同一 IP 的请求路由到同一台服务器
> - 缺点：负载不均衡、服务器宕机 Session 丢失
> 
> **方案 2：Session 复制**
> - 服务器间同步 Session 数据
> - 缺点：网络开销大、延迟高
> 
> **方案 3：集中式存储（推荐）**
> - 使用 Redis/Memcached 存储 Session
> - 优点：共享、高性能、可扩展
> 
> **实现（Express + Redis）：**
> ```javascript
> const session = require('express-session');
> const RedisStore = require('connect-redis').default;
> 
> app.use(session({
>   store: new RedisStore({ client: redisClient }),
>   secret: 'secret-key',
>   cookie: { maxAge: 24 * 60 * 60 * 1000 }
> }));
> ```
> 
> **流程：**
> ```
> 1. 用户登录 → 生成 Session ID → 存储到 Redis
> 2. 返回 Cookie（Session ID）
> 3. 后续请求携带 Cookie → 从 Redis 读取 Session
> 4. 任意服务器都能访问 Redis 中的 Session
> ```

**加分项：**
> 在实际项目中，我们使用 Redis Cluster 存储 Session，配合 Nginx 负载均衡，实现了：
> - 10 台服务器共享 Session
> - QPS 10万+
> - 99.99% 可用性

### 问题 2："为什么要用 Redis 存储 Session？对比其他方案"

**标准答案：**
> **Redis 的优势：**
> 
> **1. 高性能**
> - 内存存储，读写速度快（10万+ QPS）
> - 对比 MySQL：快 100 倍
> 
> **2. 数据结构丰富**
> - String：存储序列化的 Session
> - Hash：存储 Session 字段
> - 支持过期时间（自动清理）
> 
> **3. 分布式支持**
> - Redis Cluster：数据分片 + 高可用
> - 主从复制：数据备份
> 
> **4. 持久化**
> - RDB + AOF：数据不丢失
> 
> **对比其他方案：**
> 
> | 方案 | 性能 | 共享 | 持久化 | 适用场景 |
> |------|------|------|--------|---------|
> | 内存 | ⭐⭐⭐⭐⭐ | ❌ | ❌ | 单机开发 |
> | Cookie | ⭐⭐⭐⭐ | ✅ | ❌ | 简单数据 |
> | MySQL | ⭐⭐ | ✅ | ✅ | 低并发 |
> | Redis | ⭐⭐⭐⭐⭐ | ✅ | ✅ | 高并发（推荐） |
> | Memcached | ⭐⭐⭐⭐⭐ | ✅ | ❌ | 临时数据 |
