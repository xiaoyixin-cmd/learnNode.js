## 📚 核心知识点

### 1. 为什么需要连接池？

**问题：每次请求都创建新连接**
```javascript
// ❌ 错误：每次请求都创建连接
app.get('/users', async (req, res) => {
  const connection = await mysql.createConnection({
    host: 'localhost',
    user: 'root',
    password: 'password',
    database: 'mydb'
  });
  
  const [rows] = await connection.query('SELECT * FROM users');
  await connection.end();
  
  res.json(rows);
});

// 问题：
// 1. TCP 三次握手开销大（每次 ~10ms）
// 2. 数据库认证开销（每次 ~5ms）
// 3. 高并发时，数据库连接数爆炸
// 4. 性能差（QPS < 100）
```

**解决：连接池**
```javascript
// ✅ 正确：使用连接池
const pool = mysql.createPool({
  host: 'localhost',
  user: 'root',
  password: 'password',
  database: 'mydb',
  connectionLimit: 10  // 最多 10 个连接
});

app.get('/users', async (req, res) => {
  const [rows] = await pool.query('SELECT * FROM users');
  // 自动释放连接回池
  res.json(rows);
});

// 优势：
// 1. 复用连接，无需重复创建
// 2. 性能提升 10-100 倍（QPS > 1000）
// 3. 控制连接数，保护数据库
```

---

### 2. 连接池工作原理

```
连接池（Pool）
┌─────────────────────────────────┐
│ 空闲连接（Idle）                 │
│ [Conn1] [Conn2] [Conn3]          │
├─────────────────────────────────┤
│ 活跃连接（Active）               │
│ [Conn4] [Conn5]                  │
└─────────────────────────────────┘

流程：
1. 请求 → getConnection()
2. 连接池检查空闲连接
   - 有空闲 → 返回连接
   - 无空闲 → 等待或创建新连接
3. 使用连接执行查询
4. release() → 连接归还池
```

**状态机：**
```
创建 → 空闲 → 活跃 → 空闲 → ... → 销毁
        ↑_____________↓
        （复用循环）
```

## 💻 连接池耗尽的原因

### 原因 1：连接未释放（泄漏）

```javascript
// ❌ 错误：连接未释放
app.get('/users', async (req, res) => {
  const connection = await pool.getConnection();
  
  const [rows] = await connection.query('SELECT * FROM users');
  res.json(rows);
  
  // 忘记 connection.release()
});

// 问题：
// 1. 每次请求占用 1 个连接
// 2. 连接池耗尽（10 次请求后）
// 3. 后续请求超时（等待连接）
```

**正确做法：**
```javascript
// ✅ 方式 1：手动释放
app.get('/users', async (req, res) => {
  const connection = await pool.getConnection();
  
  try {
    const [rows] = await connection.query('SELECT * FROM users');
    res.json(rows);
  } finally {
    connection.release();  // 确保释放
  }
});

// ✅ 方式 2：使用 pool.query()（自动释放）
app.get('/users', async (req, res) => {
  const [rows] = await pool.query('SELECT * FROM users');
  res.json(rows);  // 自动释放
});
```

---

### 原因 2：慢查询阻塞

```javascript
// ❌ 问题：慢查询占用连接过久
app.get('/report', async (req, res) => {
  const connection = await pool.getConnection();
  
  // 慢查询（10 秒）
  const [rows] = await connection.query(`
    SELECT * FROM orders
    WHERE created_at > DATE_SUB(NOW(), INTERVAL 1 YEAR)
    ORDER BY total DESC
  `);
  
  connection.release();
  res.json(rows);
});

// 问题：
// 1. 每次请求占用连接 10 秒
// 2. 10 个请求 → 连接池耗尽
// 3. 后续请求超时
```

**解决方案：**
```javascript
// ✅ 方式 1：优化查询（添加索引）
CREATE INDEX idx_created_at ON orders(created_at);

// ✅ 方式 2：增加连接池大小
const pool = mysql.createPool({
  connectionLimit: 50  // 增加到 50
});

// ✅ 方式 3：使用读写分离
const readPool = mysql.createPool({ host: 'slave' });
const writePool = mysql.createPool({ host: 'master' });
```

---

### 原因 3：长事务未提交

```javascript
// ❌ 问题：长事务占用连接
app.post('/transfer', async (req, res) => {
  const connection = await pool.getConnection();
  
  await connection.beginTransaction();
  
  // 复杂业务逻辑（30 秒）
  await sleep(30000);
  
  await connection.commit();
  connection.release();
});

// 问题：
// 1. 事务期间，连接被占用
// 2. 长时间未提交/回滚
// 3. 连接池耗尽
```

**解决方案：**
```javascript
// ✅ 缩短事务时间
app.post('/transfer', async (req, res) => {
  const connection = await pool.getConnection();
  
  try {
    await connection.beginTransaction();
    
    // 只在事务中执行必要的数据库操作
    await connection.query('UPDATE accounts SET balance = balance - 100 WHERE id = 1');
    await connection.query('UPDATE accounts SET balance = balance + 100 WHERE id = 2');
    
    await connection.commit();
  } catch (error) {
    await connection.rollback();
    throw error;
  } finally {
    connection.release();
  }
});
```

//   等待请求: 50    ← 50 个请求在等待

### 2. 检测连接泄漏

```javascript
// 方式 1：手动追踪连接
const activeConnections = new Map();

pool.on('acquire', (connection) => {
  const stack = new Error().stack;
  activeConnections.set(connection.threadId, {
    acquiredAt: new Date(),
    stack
  });
});

pool.on('release', (connection) => {
  activeConnections.delete(connection.threadId);
});

// 定期检查长时间未释放的连接
setInterval(() => {
  const now = Date.now();
  
  for (const [threadId, info] of activeConnections.entries()) {
    const duration = now - info.acquiredAt.getTime();
    
    if (duration > 10000) {  // 超过 10 秒
      console.error('连接泄漏：', {
        threadId,
        duration: `${duration}ms`,
        stack: info.stack
      });
    }
  }
}, 5000);
```

## 💻 连接池配置优化

### 1. 关键参数

```javascript
const pool = mysql.createPool({
  // 基础配置
  host: 'localhost',
  user: 'root',
  password: 'password',
  database: 'mydb',
  
  // 连接池大小
  connectionLimit: 10,           // 最大连接数
  
  // 等待策略
  waitForConnections: true,      // 连接池满时是否等待
  queueLimit: 0,                 // 等待队列长度（0 = 无限）
  
  // 超时配置
  acquireTimeout: 10000,         // 获取连接超时（10 秒）
  timeout: 60000,                // 查询超时（60 秒）
  
  // 空闲连接
  idleTimeout: 60000,            // 空闲连接超时（60 秒）
  maxIdle: 10,                   // 最大空闲连接数
  
  // 连接检测
  enableKeepAlive: true,         // 启用 keep-alive
  keepAliveInitialDelay: 0       // keep-alive 初始延迟
});
```

### 2. 连接池大小计算

**公式：**
```
连接池大小 = ((核心数 * 2) + 磁盘数) * 并发系数

示例：
- 4 核 CPU
- 1 块磁盘
- 并发系数 = 1（中等负载）

连接池大小 = ((4 * 2) + 1) * 1 = 9

建议：10-20 个连接
```

**调优建议：**
```javascript
// 场景 1：高并发读（API 服务）
connectionLimit: 50

// 场景 2：事务密集（后台任务）
connectionLimit: 20

// 场景 3：低并发（管理后台）
connectionLimit: 10
```

## 💻 实战案例

### 案例：修复连接泄漏

```javascript
const express = require('express');
const mysql = require('mysql2/promise');

const app = express();

const pool = mysql.createPool({
  host: 'localhost',
  user: 'root',
  password: 'password',
  database: 'mydb',
  connectionLimit: 10
});

// ❌ 问题代码（连接泄漏）
app.get('/users-bad', async (req, res) => {
  const connection = await pool.getConnection();
  
  if (req.query.test) {
    // 提前返回，忘记释放连接
    return res.json({ test: true });
  }
  
  const [rows] = await connection.query('SELECT * FROM users');
  connection.release();
  res.json(rows);
});

// ✅ 修复后（确保释放）
app.get('/users-good', async (req, res) => {
  const connection = await pool.getConnection();
  
  try {
    if (req.query.test) {
      return res.json({ test: true });
    }
    
    const [rows] = await connection.query('SELECT * FROM users');
    res.json(rows);
  } finally {
    connection.release();  // 确保释放
  }
});

// 🔥 最佳实践（使用 pool.query）
app.get('/users-best', async (req, res) => {
  if (req.query.test) {
    return res.json({ test: true });
  }
  
  const [rows] = await pool.query('SELECT * FROM users');
  res.json(rows);  // 自动释放
});

app.listen(3000);
```

## 🎓 面试标准答题

### 问题："连接池耗尽的常见原因有哪些？如何排查？"

**标准答案：**
> **常见原因：**
> 
> **1. 连接泄漏（最常见）**
> - 获取连接后未释放
> - 异常情况下未释放（忘记 finally）
> 
> **2. 慢查询阻塞**
> - 查询耗时过长（未优化、无索引）
> - 占用连接时间过久
> 
> **3. 长事务未提交**
> - 事务中执行复杂业务逻辑
> - 长时间占用连接
> 
> **4. 连接池配置不合理**
> - `connectionLimit` 太小
> - `acquireTimeout` 太短
> 
> **排查方法：**
> 
> **1. 监控连接池状态**
> ```javascript
> console.log({
>   总连接数: pool._allConnections.length,
>   空闲连接: pool._freeConnections.length,
>   活跃连接: pool._allConnections.length - pool._freeConnections.length
> });
> ```
> 
> **2. 检测连接泄漏**
> - 监听 `acquire` 和 `release` 事件
> - 追踪长时间未释放的连接
> 
> **3. 查看数据库连接**
> ```sql
> SHOW PROCESSLIST;  -- 查看活跃连接
> SHOW STATUS LIKE 'Threads_connected';  -- 连接数
> ```
> 
> **4. 分析慢查询**
> ```sql
> SHOW VARIABLES LIKE 'slow_query_log';
> SHOW VARIABLES LIKE 'long_query_time';
> ```
