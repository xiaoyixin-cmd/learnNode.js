## 📚 核心知识点

### 1. Redis 五种数据类型与底层结构

| 数据类型 | 底层实现 | 使用场景 |
|---------|---------|---------|
| **String** | SDS（Simple Dynamic String） | 缓存、计数器、分布式锁 |
| **Hash** | 字典（Dict）/ 压缩列表（ZipList） | 对象存储、用户信息 |
| **List** | 快速链表（QuickList） | 消息队列、时间线 |
| **Set** | 整数集合（IntSet）/ 字典（Dict） | 标签、去重 |
| **ZSet** | 跳表（Skip List）+ 字典（Dict） | 排行榜、延时队列 |

## 💻 底层数据结构详解

### 1. SDS（Simple Dynamic String）

**C 字符串的问题：**
```c
// C 字符串
char str[] = "hello";
// 问题：
// 1. 获取长度需要遍历（O(n)）
// 2. 缓冲区溢出（strcat 不检查空间）
// 3. 二进制不安全（遇到 \0 截断）
```

**SDS 结构：**
```c
struct sdshdr {
    int len;        // 已使用长度
    int free;       // 未使用长度
    char buf[];     // 字符数组
};

// 示例
sdshdr {
    len = 5,
    free = 0,
    buf = "hello\0"
}
```

**SDS 优势：**
1. **O(1) 获取长度**
   ```c
   strlen(str);      // C 字符串：O(n)
   sdslen(sds);      // SDS：O(1)（直接读 len）
   ```

2. **防止缓冲区溢出**
   ```c
   // SDS 自动扩容
   sds = sdscatlen(sds, " world", 6);
   // 内部检查 free 空间，不够则自动扩容
   ```

3. **二进制安全**
   ```c
   // SDS 支持存储二进制数据（不依赖 \0）
   sds = sdsnewlen("\x00\x01\x02", 3);
   ```

4. **空间预分配**
   ```c
   // 扩容策略：
   // - len < 1MB：free = len（翻倍）
   // - len >= 1MB：free = 1MB
   ```

### 2. 跳表（Skip List）

**为什么需要跳表？**
```
有序链表查找：O(n)
       1 → 3 → 7 → 11 → 15 → 19

跳表（多层索引）：O(log n)
L3:            11
L2:      7           15
L1:   3     9     13     17
L0: 1   5   9  11  13  15  17  19
```

**跳表结构：**
```c
typedef struct zskiplistNode {
    sds ele;                    // 成员对象
    double score;               // 分值
    struct zskiplistNode *backward;  // 后退指针
    struct zskiplistLevel {
        struct zskiplistNode *forward;  // 前进指针
        unsigned int span;               // 跨度
    } level[];
} zskiplistNode;

typedef struct zskiplist {
    struct zskiplistNode *header, *tail;
    unsigned long length;  // 节点数量
    int level;            // 最大层数
} zskiplist;
```

**插入流程：**
```c
// 1. 随机层数（期望层数：1/p^k，p=0.25）
int randomLevel() {
    int level = 1;
    while (random() < 0.25 && level < MAX_LEVEL) {
        level++;
    }
    return level;
}

// 2. 插入节点
void insert(score, value) {
    int level = randomLevel();
    node = createNode(level, score, value);
    
    // 从高层到低层更新指针
    for (int i = level - 1; i >= 0; i--) {
        // 找到插入位置，更新 forward 指针
    }
}
```

**查询流程：**
```
查找 score = 13：
1. 从 L3 开始：11 → NULL（跳到 L2）
2. L2：11 → 15（13 < 15，跳到 L1）
3. L1：11 → 13（找到）

时间复杂度：O(log n)
```

### 3. 字典（Hash Table）

**字典结构：**
```c
typedef struct dict {
    dictType *type;
    dictht ht[2];      // 两个哈希表（用于渐进式 rehash）
    long rehashidx;    // rehash 进度（-1 表示未进行）
} dict;

typedef struct dictht {
    dictEntry **table;  // 哈希表数组
    unsigned long size; // 数组大小
    unsigned long used; // 已有节点数量
} dictht;

typedef struct dictEntry {
    void *key;
    void *val;
    struct dictEntry *next;  // 链表法解决冲突
} dictEntry;
```

**哈希冲突解决：链地址法**
```
Hash Table:
  [0] → NULL
  [1] → Entry1 → Entry2 → NULL  // 冲突链
  [2] → Entry3 → NULL
  [3] → NULL
```

**渐进式 Rehash：**
```c
// 问题：数据量大时，一次性 rehash 阻塞时间长

// 解决：渐进式 rehash
1. 分配新的 ht[1]（大小为 ht[0] 的 2 倍）
2. 将 rehashidx 设为 0
3. 每次操作时，顺带迁移 ht[0].table[rehashidx] 的节点到 ht[1]
4. rehashidx++
5. 全部迁移完成后，释放 ht[0]，ht[1] 变为 ht[0]

// 好处：
// - 分散到多次操作中
// - 每次操作增加少量开销（几微秒）
// - 避免长时间阻塞
```

### 4. 整数集合（IntSet）

**结构：**
```c
typedef struct intset {
    uint32_t encoding;  // 编码方式（int16/int32/int64）
    uint32_t length;    // 元素数量
    int8_t contents[];  // 整数数组（有序）
} intset;
```

**升级机制：**
```c
// 初始：int16 编码
intset = [1, 2, 3]  // 每个元素 2 字节

// 插入 65535（超出 int16 范围）
// 升级为 int32 编码
intset = [1, 2, 3, 65535]  // 每个元素 4 字节

// 优势：节省内存
// 缺点：不支持降级
```

### 5. 压缩列表（ZipList）

**结构：**
```
<zlbytes> <zltail> <zllen> <entry1> <entry2> ... <zlend>

zlbytes: 总字节数
zltail:  尾节点偏移量
zllen:   节点数量
entry:   节点数据
zlend:   结束标志（0xFF）
```

**节点结构：**
```
<prevlen> <encoding> <data>

prevlen:  前一个节点长度（用于反向遍历）
encoding: 数据类型和长度
data:     实际数据
```

**优势：**
- 紧凑存储，节省内存
- 连续内存，缓存友好

**缺点：**
- 插入/删除可能引发连锁更新（O(n²)）
- 不适合大数据量

```javascript
const Redis = require('ioredis');

// 添加用户分数（底层：跳表 + 字典）
```

### 案例 3：手写跳表（JavaScript 实现）

```javascript
class SkipListNode {
  constructor(score, value, level) {
    this.score = score;
    this.value = value;
    this.forward = new Array(level).fill(null);
  }
}

class SkipList {
  constructor(maxLevel = 16, p = 0.25) {
    this.maxLevel = maxLevel;
    this.p = p;
    this.level = 1;
    this.header = new SkipListNode(null, null, maxLevel);
  }
  
  randomLevel() {
    let level = 1;
    while (Math.random() < this.p && level < this.maxLevel) {
      level++;
    }
    return level;
  }
  
  insert(score, value) {
    const update = new Array(this.maxLevel).fill(null);
    let current = this.header;
    
    // 从高层到低层查找插入位置
    for (let i = this.level - 1; i >= 0; i--) {
      while (current.forward[i] && current.forward[i].score < score) {
        current = current.forward[i];
      }
      update[i] = current;
    }
    
    // 随机层数
    const level = this.randomLevel();
    
    // 创建新节点
    const node = new SkipListNode(score, value, level);
    
    // 更新指针
    for (let i = 0; i < level; i++) {
      node.forward[i] = update[i]?.forward[i] || null;
      if (update[i]) {
        update[i].forward[i] = node;
      }
    }
    
    if (level > this.level) {
      this.level = level;
    }
  }
  
  search(score) {
    let current = this.header;
    
    for (let i = this.level - 1; i >= 0; i--) {
      while (current.forward[i] && current.forward[i].score < score) {
        current = current.forward[i];
      }
    }
    
    current = current.forward[0];
    return current && current.score === score ? current.value : null;
  }
}

// 使用
const skipList = new SkipList();
skipList.insert(10, 'Alice');
skipList.insert(20, 'Bob');
skipList.insert(15, 'Charlie');

console.log(skipList.search(15));  // "Charlie"
```

## 🎓 面试标准答题

### 问题 1："Redis 五种数据类型分别用什么底层数据结构实现？"

**标准答案：**
> **Redis 五种数据类型的底层实现：**
> 
> | 类型 | 底层结构 | 编码转换 |
> |------|---------|---------|
> | **String** | SDS | 无 |
> | **Hash** | 压缩列表 / 字典 | 元素多时转字典 |
> | **List** | 快速链表 | 无（已优化） |
> | **Set** | 整数集合 / 字典 | 非整数时转字典 |
> | **ZSet** | 跳表 + 字典 | 元素多时用跳表 |
> 
> **编码转换规则（Hash 为例）：**
> ```bash
> # 压缩列表（ziplist）条件：
> # 1. 元素数量 <= 512
> # 2. 所有值长度 <= 64 字节
> 
> # 超过阈值 → 转为字典（hashtable）
> ```
> 
> **为什么这样设计？**
> 1. **内存优化**：小数据用压缩列表（节省内存）
> 2. **性能优化**：大数据用字典（查询快）
> 3. **自动转换**：无需手动选择

**加分项：**
> 可以使用 `OBJECT ENCODING key` 查看底层编码：
> ```bash
> 127.0.0.1:6379> OBJECT ENCODING mylist
> "quicklist"
> ```

### 问题 2："跳表（Skip List）的实现原理是什么？时间复杂度？"

**标准答案：**
> **跳表原理：有序链表 + 多层索引**
> 
> **1. 结构**
> ```
> L3:            11
> L2:      7           15
> L1:   3     9     13     17
> L0: 1   5   9  11  13  15  17  19
> ```
> 
> **2. 查询流程（查找 13）**
> - 从 L3 开始：11 → NULL（下降到 L2）
> - L2：11 → 15（13 < 15，下降到 L1）
> - L1：11 → 13（找到）
> 
> **3. 插入流程**
> - 随机层数：`level = 1 + 随机(0.25 概率)`
> - 从高层到低层更新指针
> 
> **4. 时间复杂度**
> - 查询：O(log n)
> - 插入：O(log n)
> - 删除：O(log n)
> 
> **5. 为什么用跳表而不用红黑树？**
> - 实现简单（红黑树复杂）
> - 范围查询友好（顺序遍历）
> - 并发性能好（局部修改）
