## 📚 核心知识点

### 1. MongoDB 索引类型

| 索引类型 | 语法 | 适用场景 |
|---------|------|---------|
| **单字段索引** | `{ field: 1 }` | 单个字段查询 |
| **复合索引** | `{ field1: 1, field2: -1 }` | 多字段查询、排序 |
| **多键索引** | `{ array: 1 }` | 数组字段查询 |
| **文本索引** | `{ field: "text" }` | 全文搜索 |
| **地理空间索引** | `{ location: "2dsphere" }` | 地理位置查询 |
| **哈希索引** | `{ field: "hashed" }` | 分片键 |
| **通配符索引** | `{ "$**": 1 }` | 动态字段查询 |

## 💻 单字段索引

### 1. 创建与使用

```javascript
const { MongoClient } = require('mongodb');

const client = new MongoClient('mongodb://localhost:27017');
await client.connect();
const db = client.db('test');
const users = db.collection('users');

// 创建单字段索引
await users.createIndex({ email: 1 });  // 1: 升序，-1: 降序

// 查询（自动使用索引）
const user = await users.findOne({ email: 'alice@example.com' });

// 查看索引
await users.indexes();
// [
//   { v: 2, key: { _id: 1 }, name: '_id_' },
//   { v: 2, key: { email: 1 }, name: 'email_1' }
// ]
```

---

### 2. 性能对比

```javascript
// 无索引（全表扫描）
await users.find({ email: 'alice@example.com' }).explain('executionStats');
// {
//   executionStats: {
//     totalDocsExamined: 1000000,  // 扫描 100 万条
//     executionTimeMillis: 500     // 500ms
//   }
// }

// 有索引（索引扫描）
await users.createIndex({ email: 1 });
await users.find({ email: 'alice@example.com' }).explain('executionStats');
// {
//   executionStats: {
//     totalDocsExamined: 1,        // 只扫描 1 条
//     executionTimeMillis: 2       // 2ms（快 250 倍）
//   }
// }
```

## 💻 复合索引

### 1. 最左前缀原则

**索引创建：**
```javascript
// 创建复合索引（age, gender, city）
await users.createIndex({ age: 1, gender: 1, city: 1 });
```

**查询匹配规则：**
```javascript
// ✅ 使用索引（匹配最左前缀）
await users.find({ age: 25 });                          // age
await users.find({ age: 25, gender: 'M' });             // age + gender
await users.find({ age: 25, gender: 'M', city: 'NY' }); // age + gender + city

// ✅ 部分使用索引
await users.find({ age: 25, city: 'NY' });  // 只用 age（跳过 gender）

// ❌ 不使用索引（不匹配最左前缀）
await users.find({ gender: 'M' });                      // 缺少 age
await users.find({ city: 'NY' });                       // 缺少 age
await users.find({ gender: 'M', city: 'NY' });          // 缺少 age
```

**原理：B树索引结构**
```
B树索引（age, gender, city）：
                      (25, 'M', 'NY')
                     /               \
          (20, 'F', 'LA')         (30, 'M', 'SF')
         /          \              /          \
  (18,'F','LA') (23,'M','NY') (28,'M','NY') (35,'F','LA')

查询流程：
1. 先按 age 查找
2. 再按 gender 过滤
3. 最后按 city 过滤

跳过 age → 无法定位起始位置 → 全表扫描
```

---

### 2. 排序优化

```javascript
// 创建复合索引
await users.createIndex({ age: 1, score: -1 });

// ✅ 使用索引排序（与索引顺序一致）
await users.find({ age: 25 }).sort({ score: -1 });

// ✅ 反向使用索引（索引可以反向遍历）
await users.find({ age: 25 }).sort({ age: -1, score: 1 });

// ❌ 不使用索引排序（顺序不一致）
await users.find({ age: 25 }).sort({ score: 1 });  // 索引是 -1，查询是 1
await users.find({ age: 25 }).sort({ name: 1 });   // name 不在索引中
```

## 💻 覆盖查询（Covered Query）

### 1. 什么是覆盖查询？

**定义：** 查询的所有字段都在索引中，无需访问文档

```javascript
// 创建复合索引
await users.createIndex({ email: 1, age: 1, name: 1 });

// ❌ 非覆盖查询（需要读取文档）
await users.find(
  { email: 'alice@example.com' },
  { projection: { email: 1, age: 1, name: 1, address: 1 } }  // address 不在索引中
);

// ✅ 覆盖查询（只读索引，不读文档）
await users.find(
  { email: 'alice@example.com' },
  { projection: { _id: 0, email: 1, age: 1, name: 1 } }  // 所有字段都在索引中
);

// 验证
await users.find(
  { email: 'alice@example.com' },
  { projection: { _id: 0, email: 1, age: 1, name: 1 } }
).explain('executionStats');

// {
//   executionStats: {
//     totalDocsExamined: 0,  // 0 条文档（覆盖查询）
//     totalKeysExamined: 1   // 只扫描索引
//   }
// }
```

**性能提升：**
- 无需读取文档（节省 I/O）
- 只读取索引（内存友好）
- 速度提升 10-100 倍

---

### 2. 注意事项

```javascript
// ❌ _id 字段会破坏覆盖查询
await users.find(
  { email: 'alice@example.com' },
  { projection: { email: 1, age: 1 } }  // 默认返回 _id
);
// totalDocsExamined: 1（需要读取文档获取 _id）

// ✅ 排除 _id
await users.find(
  { email: 'alice@example.com' },
  { projection: { _id: 0, email: 1, age: 1 } }
);
// totalDocsExamined: 0（覆盖查询）
```

## 💻 explain() 性能分析

### 1. explain() 输出解读

```javascript
await users.find({ age: 25 }).explain('executionStats');

// 输出：
{
  queryPlanner: {
    winningPlan: {
      stage: 'FETCH',           // 需要读取文档
      inputStage: {
        stage: 'IXSCAN',        // 索引扫描
        indexName: 'age_1',     // 使用的索引
        direction: 'forward'
      }
    }
  },
  executionStats: {
    executionSuccess: true,
    nReturned: 100,             // 返回 100 条
    executionTimeMillis: 5,     // 执行时间 5ms
    totalKeysExamined: 100,     // 扫描 100 个索引键
    totalDocsExamined: 100,     // 扫描 100 条文档
    executionStages: {
      stage: 'FETCH',
      nReturned: 100,
      executionTimeMillisEstimate: 0,
      works: 101,
      advanced: 100,
      docsExamined: 100
    }
  }
}
```

---

### 2. 关键指标

| 指标 | 含义 | 优化目标 |
|------|------|---------|
| **stage** | 执行阶段 | IXSCAN（索引扫描）> COLLSCAN（全表扫描） |
| **nReturned** | 返回文档数 | - |
| **totalKeysExamined** | 扫描索引数 | 越少越好 |
| **totalDocsExamined** | 扫描文档数 | 越少越好（0 = 覆盖查询） |
| **executionTimeMillis** | 执行时间 | 越少越好 |

**优化目标：**
```javascript
// 理想情况
totalKeysExamined == nReturned == totalDocsExamined

// 覆盖查询
totalDocsExamined == 0
```

## 💻 实战案例

### 案例 1：电商商品查询优化

```javascript
const products = db.collection('products');

// 场景：查询某分类下价格在 100-500 的商品，按销量排序
await products.find({
  category: 'Electronics',
  price: { $gte: 100, $lte: 500 }
}).sort({ sales: -1 }).limit(20);

// 优化前：无索引（全表扫描）
// executionTimeMillis: 500ms

// 优化 1：创建复合索引
await products.createIndex({ category: 1, price: 1, sales: -1 });

// 优化后
// executionTimeMillis: 5ms（快 100 倍）
```

---

### 案例 2：用户搜索优化（覆盖查询）

```javascript
const users = db.collection('users');

// 场景：搜索用户，返回 id、name、email
await users.find(
  { email: /^alice/ },
  { projection: { _id: 0, name: 1, email: 1 } }
);

// 优化：创建覆盖索引
await products.createIndex({ email: 1, name: 1 });

// 结果：totalDocsExamined = 0（覆盖查询）
```

### 案例 3：复合索引优化（最左前缀）

```javascript
const orders = db.collection('orders');

// 场景：查询某用户在某时间段的订单
await orders.find({
  userId: 123,
  createdAt: { $gte: new Date('2025-01-01'), $lte: new Date('2025-12-31') }
}).sort({ createdAt: -1 });

// ❌ 错误索引（顺序不当）
await orders.createIndex({ createdAt: -1, userId: 1 });
// 查询 userId 时不使用索引

// ✅ 正确索引（userId 在前）
await orders.createIndex({ userId: 1, createdAt: -1 });
// 查询 userId → 再过滤 createdAt → 排序（与索引一致）
```

## 🎓 面试标准答题

### 问题 1："复合索引的最左前缀原则是什么？"

**标准答案：**
> **最左前缀原则：** 复合索引的查询必须从最左边的字段开始，才能使用索引。
> 
> **示例：**
> ```javascript
> // 索引：{ age: 1, gender: 1, city: 1 }
> 
> // ✅ 使用索引
> find({ age: 25 })                          // age
> find({ age: 25, gender: 'M' })             // age + gender
> find({ age: 25, gender: 'M', city: 'NY' }) // age + gender + city
> 
> // ❌ 不使用索引
> find({ gender: 'M' })                      // 缺少 age
> find({ city: 'NY' })                       // 缺少 age
> find({ gender: 'M', city: 'NY' })          // 缺少 age
> ```
> 
> **原理：B树索引**
> - B树按 `(age, gender, city)` 顺序排列
> - 必须先定位 `age`，才能查找 `gender` 和 `city`
> - 跳过 `age` → 无法定位起始位置 → 全表扫描
> 
> **优化建议：**
> 1. **高频字段在前**：查询频率高的字段放在最左边
> 2. **区分度高的在前**：选择性高的字段（如 userId）在前
> 3. **多索引策略**：必要时创建多个索引覆盖不同查询

**加分项：**
> MongoDB 的复合索引可以**反向使用**（从右往左），但性能会下降。建议创建多个索引覆盖不同的查询模式。

### 问题 2："什么是覆盖查询（Covered Query）？如何优化？"

**标准答案：**
> **覆盖查询：** 查询的所有字段（包括查询条件和返回字段）都在索引中，MongoDB 无需读取文档，只读索引即可返回结果。
> 
> **示例：**
> ```javascript
> // 创建索引
> createIndex({ email: 1, name: 1, age: 1 })
> 
> // ✅ 覆盖查询
> find(
>   { email: 'alice@example.com' },
>   { projection: { _id: 0, email: 1, name: 1, age: 1 } }
> )
> // totalDocsExamined: 0（只读索引）
> 
> // ❌ 非覆盖查询
> find(
>   { email: 'alice@example.com' },
>   { projection: { email: 1, name: 1, age: 1, address: 1 } }
> )
> // totalDocsExamined: 1（需要读文档获取 address）
> ```
> 
> **优化方法：**
> 1. **排除 `_id`**：`{ _id: 0 }` （默认返回 `_id` 会破坏覆盖查询）
> 2. **索引包含所有字段**：查询条件 + 返回字段都在索引中
> 3. **避免查询不在索引中的字段**
> 
> **性能提升：**
> - 无需读取文档（节省磁盘 I/O）
> - 只读索引（内存友好）
> - 速度提升 10-100 倍
