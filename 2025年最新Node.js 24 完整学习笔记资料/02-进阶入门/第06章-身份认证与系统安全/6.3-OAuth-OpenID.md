# 6.3 OAuth 2.1 与 OpenID Connect

> 授权码模式流程图解，使用openid-client 5.x接入Google/GitHub

---

## 📖 知识点概述

### 核心概念

**OAuth 2.1**是授权协议，用于授权第三方应用访问用户资源，而不需要分享密码。**OpenID Connect (OIDC)**是在OAuth 2.0基础上构建的身份认证层，提供用户身份信息。

### OAuth 2.1 vs OAuth 2.0

**OAuth 2.1是2023年发布的OAuth 2.0的精简版，移除了不安全的流程：**

| 特性 | OAuth 2.0 | OAuth 2.1 |
|------|-----------|-----------|
| **授权码模式** | 支持（推荐） | 支持（唯一推荐） |
| **隐式授权** | 支持 | ❌ 移除（不安全） |
| **密码模式** | 支持 | ❌ 移除（不安全） |
| **客户端凭证** | 支持 | 支持（服务间） |
| **PKCE** | 可选 | ✅ 强制要求 |
| **刷新Token** | 支持 | 支持（需轮换） |

### OAuth 2.1 vs OpenID Connect

```
┌────────────────────────────────────────┐
│          OpenID Connect (OIDC)         │  ← 身份认证层
│  (ID Token, UserInfo, 标准化Claims)     │
├────────────────────────────────────────┤
│            OAuth 2.1                   │  ← 授权层
│  (Access Token, 授权码流程)             │
└────────────────────────────────────────┘
```

| 维度 | OAuth 2.1 | OpenID Connect |
|------|-----------|----------------|
| **目的** | 授权（Authorization） | 认证（Authentication） |
| **回答的问题** | "用户允许访问什么？" | "用户是谁？" |
| **核心产物** | Access Token | ID Token（JWT） |
| **用户信息** | 非标准化 | 标准化（sub、name、email） |
| **典型场景** | 第三方访问API | 单点登录（SSO） |

### 四种角色

```
┌─────────────────────────────────────────────────────────────┐
│                                                               │
│  1. Resource Owner（资源所有者 - 用户）                        │
│     └─ 拥有GitHub账号的开发者                                 │
│                                                               │
│  2. Client（客户端 - 你的应用）                               │
│     └─ 你开发的Web应用，想访问用户的GitHub数据                 │
│                                                               │
│  3. Authorization Server（授权服务器 - GitHub）                │
│     └─ https://github.com/login/oauth/authorize              │
│                                                               │
│  4. Resource Server（资源服务器 - GitHub API）                 │
│     └─ https://api.github.com/user                           │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

### 授权码流程（Authorization Code Flow + PKCE）

```
┌─────────┐                                   ┌─────────────┐           ┌──────────┐
│  用户    │                                   │ 你的应用     │           │  GitHub  │
└─────────┘                                   └─────────────┘           └──────────┘
     │                                              │                         │
     │  1. 点击"使用GitHub登录"按钮                  │                         │
     │ ───────────────────────────────────────────>│                         │
     │                                              │                         │
     │                         2. 生成code_verifier和code_challenge           │
     │                            (PKCE安全措施)                               │
     │                                              │                         │
     │  3. 重定向到GitHub授权页面                    │                         │
     │     https://github.com/login/oauth/authorize?                          │
     │       client_id=xxx&                         │                         │
     │       redirect_uri=http://localhost:3000/callback&                     │
     │       scope=read:user user:email&            │                         │
     │       code_challenge=xxx                     │                         │
     │ <────────────────────────────────────────────│                         │
     │ ───────────────────────────────────────────────────────────────────────>│
     │                                              │                         │
     │  4. GitHub显示授权页面：                     │                         │
     │     "MyApp想访问你的：                        │                         │
     │      - 基本信息                              │                         │
     │      - 邮箱地址"                             │                         │
     │     [授权] [取消]                            │                         │
     │                                              │                         │
     │  5. 用户点击[授权]                            │                         │
     │ ───────────────────────────────────────────────────────────────────────>│
     │                                              │                         │
     │  6. GitHub重定向回应用（带授权码）             │                         │
     │     http://localhost:3000/callback?code=abc123                         │
     │ <───────────────────────────────────────────────────────────────────────│
     │ ───────────────────────────────────────────>│                         │
     │                                              │                         │
     │              7. 应用用授权码+code_verifier换取Access Token              │
     │                                              │ ───────────────────────>│
     │                                              │                         │
     │                           8. 返回Access Token                          │
     │                                              │ <───────────────────────│
     │                                              │                         │
     │                    9. 使用Access Token获取用户信息                      │
     │                                              │ ───────────────────────>│
     │                                              │                         │
     │                          10. 返回用户数据                              │
     │                                              │ <───────────────────────│
     │                                              │                         │
     │  11. 登录成功，创建本地Session                │                         │
     │ <────────────────────────────────────────────│                         │
```

### 解决的问题

- **安全授权**：不需要分享密码
- **细粒度权限**：通过scope控制访问范围
- **单点登录**：一次登录访问多个应用
- **第三方集成**：快速集成Google/GitHub登录

### 典型应用场景

- **社交登录**："使用Google账号登录"
- **API授权**：允许第三方访问你的Twitter数据
- **企业SSO**：统一身份认证中心

## 🚀 使用指南

### 理解：为什么需要OAuth？

**传统方式的问题：**
```javascript
// ❌ 用户需要把GitHub密码告诉你的应用
const username = 'john';
const password = 'github_password_123';

// 你的应用用用户的密码访问GitHub
const response = await fetch('https://api.github.com/user', {
  headers: {
    'Authorization': 'Basic ' + btoa(`${username}:${password}`)
  }
});

// 问题：
// 1. 用户不信任你的应用
// 2. 你的应用可以访问用户所有数据
// 3. 密码泄露风险高
// 4. 无法撤销（除非改密码）
```

**OAuth方式：**
```javascript
// ✅ 用户只授权有限权限
// 1. 跳转到GitHub授权页面
// 2. 用户确认授权（只读个人信息）
// 3. GitHub返回Access Token
// 4. 你的应用用Token访问GitHub API（仅限授权的scope）

const accessToken = 'gho_abc123...';
const response = await fetch('https://api.github.com/user', {
  headers: {
    'Authorization': `Bearer ${accessToken}`
  }
});

// 优点：
// 1. 用户不需要分享密码
// 2. 细粒度权限控制（只读 vs 读写）
// 3. 可随时撤销
// 4. Token有过期时间
```

#### 第二步：基础OAuth实现（不使用库）

```javascript
// oauth-github-basic.js
import express from 'express';
import crypto from 'crypto';
import session from 'express-session';

const app = express();
app.use(express.json());
app.use(express.urlencoded({ extended: true }));

// 💡 Session配置
app.use(session({
  secret: process.env.SESSION_SECRET,
  resave: false,
  saveUninitialized: false,
  cookie: {
    secure: process.env.NODE_ENV === 'production', // 生产环境需HTTPS
    httpOnly: true,
    maxAge: 24 * 60 * 60 * 1000 // 24小时
  }
}));

// 💡 OAuth配置
const GITHUB_CONFIG = {
  clientId: process.env.GITHUB_CLIENT_ID,
  clientSecret: process.env.GITHUB_CLIENT_SECRET,
  callbackURL: process.env.GITHUB_CALLBACK_URL,
  authorizeURL: 'https://github.com/login/oauth/authorize',
  tokenURL: 'https://github.com/login/oauth/access_token',
  userURL: 'https://api.github.com/user'
};

// 💡 PKCE工具函数
function generateCodeVerifier() {
  return crypto.randomBytes(32).toString('base64url');
}

function generateCodeChallenge(verifier) {
  return crypto
    .createHash('sha256')
    .update(verifier)
    .digest('base64url');
}

// 💡 步骤1：跳转到GitHub授权页面
app.get('/auth/github', (req, res) => {
  // 生成PKCE参数
  const codeVerifier = generateCodeVerifier();
  const codeChallenge = generateCodeChallenge(codeVerifier);
  
  // 生成state（防CSRF攻击）
  const state = crypto.randomBytes(16).toString('hex');
  
  // 💡 保存到Session（回调时需要验证）
  req.session.oauth = {
    state,
    codeVerifier
  };
  
  // 构建授权URL
  const authURL = new URL(GITHUB_CONFIG.authorizeURL);
  authURL.searchParams.set('client_id', GITHUB_CONFIG.clientId);
  authURL.searchParams.set('redirect_uri', GITHUB_CONFIG.callbackURL);
  authURL.searchParams.set('scope', 'read:user user:email');
  authURL.searchParams.set('state', state);
  authURL.searchParams.set('code_challenge', codeChallenge);
  authURL.searchParams.set('code_challenge_method', 'S256');
  
  // 重定向到GitHub
  res.redirect(authURL.toString());
});

// 💡 步骤2：处理回调（授权码换Access Token）
app.get('/auth/github/callback', async (req, res) => {
  try {
    const { code, state } = req.query;
    
    // 1. 验证state（防CSRF）
    if (!state || state !== req.session.oauth?.state) {
      return res.status(403).send('State验证失败');
    }
    
    // 2. 验证code
    if (!code) {
      return res.status(400).send('缺少授权码');
    }
    
    const codeVerifier = req.session.oauth.codeVerifier;
    
    // 3. 💡 用授权码换取Access Token
    const tokenResponse = await fetch(GITHUB_CONFIG.tokenURL, {
      method: 'POST',
      headers: {
        'Accept': 'application/json',
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        client_id: GITHUB_CONFIG.clientId,
        client_secret: GITHUB_CONFIG.clientSecret,
        code: code,
        redirect_uri: GITHUB_CONFIG.callbackURL,
        code_verifier: codeVerifier // PKCE验证
      })
    });
    
    if (!tokenResponse.ok) {
      throw new Error('Token交换失败');
    }
    
    const tokenData = await tokenResponse.json();
    
    if (tokenData.error) {
      throw new Error(tokenData.error_description || tokenData.error);
    }
    
    const accessToken = tokenData.access_token;
    
    // 4. 💡 使用Access Token获取用户信息
    const userResponse = await fetch(GITHUB_CONFIG.userURL, {
      headers: {
        'Authorization': `Bearer ${accessToken}`,
        'Accept': 'application/json'
      }
    });
    
    if (!userResponse.ok) {
      throw new Error('获取用户信息失败');
    }
    
    const githubUser = await userResponse.json();
    
    // 5. 💡 保存用户信息到Session
    req.session.user = {
      id: githubUser.id,
      username: githubUser.login,
      name: githubUser.name,
      email: githubUser.email,
      avatar: githubUser.avatar_url,
      provider: 'github',
      accessToken // 保存Token以便后续API调用
    };
    
    // 清除OAuth临时数据
    delete req.session.oauth;
    
    // 6. 重定向到首页
    res.redirect('/profile');
  } catch (error) {
    console.error('GitHub OAuth回调失败:', error);
    res.status(500).send('登录失败: ' + error.message);
  }
});

// 💡 用户资料页面
app.get('/profile', (req, res) => {
  if (!req.session.user) {
    return res.redirect('/');
  }
  
  res.send(`
    <!DOCTYPE html>
    <html>
    <head>
      <title>用户资料</title>
      <style>
        body { font-family: Arial; max-width: 600px; margin: 50px auto; }
        .profile { border: 1px solid #ddd; padding: 20px; border-radius: 8px; }
        .avatar { width: 100px; height: 100px; border-radius: 50%; }
        button { margin-top: 20px; padding: 10px 20px; }
      </style>
    </head>
    <body>
      <div class="profile">
        <h1>个人资料</h1>
        <img class="avatar" src="${req.session.user.avatar}" alt="Avatar">
        <p><strong>用户名：</strong>${req.session.user.username}</p>
        <p><strong>姓名：</strong>${req.session.user.name || '未设置'}</p>
        <p><strong>邮箱：</strong>${req.session.user.email || '未公开'}</p>
        <p><strong>登录方式：</strong>${req.session.user.provider}</p>
        <form method="POST" action="/logout">
          <button type="submit">登出</button>
        </form>
      </div>
    </body>
    </html>
  `);
});

// 💡 登出
app.post('/logout', (req, res) => {
  req.session.destroy(err => {
    if (err) {
      console.error('登出失败:', err);
    }
    res.redirect('/');
  });
});

// 首页
app.get('/', (req, res) => {
  if (req.session.user) {
    return res.redirect('/profile');
  }
  
  res.send(`
    <!DOCTYPE html>
    <html>
    <head>
      <title>OAuth登录示例</title>
      <style>
        body { font-family: Arial; text-align: center; margin-top: 100px; }
        button { padding: 15px 30px; font-size: 16px; cursor: pointer; }
      </style>
    </head>
    <body>
      <h1>OAuth 2.1 登录示例</h1>
      <a href="/auth/github">
        <button>使用GitHub登录</button>
      </a>
    </body>
    </html>
  `);
});

app.listen(3000, () => {
  console.log('应用运行在 http://localhost:3000');
});
```

#### 第三步：使用openid-client库（推荐）

```javascript
// oauth-openid-client.js
import express from 'express';
import session from 'express-session';
import { Issuer, generators } from 'openid-client';

const app = express();
app.use(express.json());

app.use(session({
  secret: process.env.SESSION_SECRET,
  resave: false,
  saveUninitialized: false,
  cookie: {
    secure: process.env.NODE_ENV === 'production',
    httpOnly: true,
    maxAge: 24 * 60 * 60 * 1000
  }
}));

// 💡 初始化OAuth客户端
let githubClient;
let googleClient;

async function initializeClients() {
  // 1. 💡 发现GitHub的OAuth配置
  const githubIssuer = await Issuer.discover('https://token.actions.githubusercontent.com');
  
  // 由于GitHub不完全支持OIDC Discovery，手动配置
  const githubIssuerManual = new Issuer({
    issuer: 'https://github.com',
    authorization_endpoint: 'https://github.com/login/oauth/authorize',
    token_endpoint: 'https://github.com/login/oauth/access_token',
    userinfo_endpoint: 'https://api.github.com/user'
  });
  
  githubClient = new githubIssuerManual.Client({
    client_id: process.env.GITHUB_CLIENT_ID,
    client_secret: process.env.GITHUB_CLIENT_SECRET,
    redirect_uris: [process.env.GITHUB_CALLBACK_URL],
    response_types: ['code']
  });
  
  // 2. 💡 发现Google的OpenID Connect配置
  const googleIssuer = await Issuer.discover('https://accounts.google.com');
  
  googleClient = new googleIssuer.Client({
    client_id: process.env.GOOGLE_CLIENT_ID,
    client_secret: process.env.GOOGLE_CLIENT_SECRET,
    redirect_uris: [process.env.GOOGLE_CALLBACK_URL],
    response_types: ['code']
  });
  
  console.log('✅ OAuth客户端初始化完成');
}

initializeClients().catch(console.error);

// 💡 GitHub登录
app.get('/auth/github', (req, res) => {
  const codeVerifier = generators.codeVerifier();
  const codeChallenge = generators.codeChallenge(codeVerifier);
  const state = generators.state();
  
  // 保存到Session
  req.session.oauth = {
    state,
    codeVerifier,
    provider: 'github'
  };
  
  // 💡 生成授权URL
  const authorizationUrl = githubClient.authorizationUrl({
    scope: 'read:user user:email',
    state,
    code_challenge: codeChallenge,
    code_challenge_method: 'S256'
  });
  
  res.redirect(authorizationUrl);
});

// 💡 GitHub回调
app.get('/auth/github/callback', async (req, res) => {
  try {
    const params = githubClient.callbackParams(req);
    
    // 验证state
    if (params.state !== req.session.oauth?.state) {
      throw new Error('State不匹配');
    }
    
    const codeVerifier = req.session.oauth.codeVerifier;
    
    // 💡 交换Token
    const tokenSet = await githubClient.callback(
      GITHUB_CONFIG.callbackURL,
      params,
      { code_verifier: codeVerifier, state: params.state }
    );
    
    const accessToken = tokenSet.access_token;
    
    // 💡 获取用户信息
    const userResponse = await fetch('https://api.github.com/user', {
      headers: {
        'Authorization': `Bearer ${accessToken}`,
        'Accept': 'application/json'
      }
    });
    
    const githubUser = await userResponse.json();
    
    req.session.user = {
      id: githubUser.id,
      username: githubUser.login,
      name: githubUser.name,
      email: githubUser.email,
      avatar: githubUser.avatar_url,
      provider: 'github',
      accessToken
    };
    
    delete req.session.oauth;
    
    res.redirect('/profile');
  } catch (error) {
    console.error('GitHub登录失败:', error);
    res.status(500).send('登录失败');
  }
});

// 💡 Google登录（OpenID Connect）
app.get('/auth/google', (req, res) => {
  const codeVerifier = generators.codeVerifier();
  const codeChallenge = generators.codeChallenge(codeVerifier);
  const state = generators.state();
  const nonce = generators.nonce();
  
  req.session.oauth = {
    state,
    codeVerifier,
    nonce,
    provider: 'google'
  };
  
  // 💡 OpenID Connect特定的scope
  const authorizationUrl = googleClient.authorizationUrl({
    scope: 'openid email profile',
    state,
    nonce,
    code_challenge: codeChallenge,
    code_challenge_method: 'S256'
  });
  
  res.redirect(authorizationUrl);
});

// 💡 Google回调
app.get('/auth/google/callback', async (req, res) => {
  try {
    const params = googleClient.callbackParams(req);
    
    const codeVerifier = req.session.oauth.codeVerifier;
    const nonce = req.session.oauth.nonce;
    
    // 💡 交换Token并验证ID Token
    const tokenSet = await googleClient.callback(
      process.env.GOOGLE_CALLBACK_URL,
      params,
      {
        code_verifier: codeVerifier,
        state: params.state,
        nonce // OIDC特有，防重放攻击
      }
    );
    
    // 💡 解析ID Token（JWT）
    const claims = tokenSet.claims();
    
    // 💡 OpenID Connect标准化Claims
    req.session.user = {
      id: claims.sub,         // Subject（用户唯一ID）
      email: claims.email,    // 邮箱
      name: claims.name,      // 姓名
      avatar: claims.picture, // 头像
      provider: 'google',
      accessToken: tokenSet.access_token,
      idToken: tokenSet.id_token
    };
    
    delete req.session.oauth;
    
    res.redirect('/profile');
  } catch (error) {
    console.error('Google登录失败:', error);
    res.status(500).send('登录失败');
  }
});

app.listen(3000);
```

```javascript
app.use(session({

  saveUninitialized: false

// 💡 Provider配置注册表
const PROVIDERS = {

    icon: '🐙',
    config: {
      issuer: 'https://github.com',
      authorization_endpoint: 'https://github.com/login/oauth/authorize',

      client_secret: process.env.GOOGLE_CLIENT_SECRET,
      redirect_uris: ['http://localhost:3000/auth/google/callback']
    },
    scope: 'openid email profile',
    normalizeUser: (claims) => ({
      id: claims.sub,
      email: claims.email,
      name: claims.name,
      avatar: claims.picture
    })
  },
  
  microsoft: {
    name: 'Microsoft',
    icon: '🪟',
    discoveryUrl: 'https://login.microsoftonline.com/common/v2.0',
    clientConfig: {
      client_id: process.env.MICROSOFT_CLIENT_ID,
      client_secret: process.env.MICROSOFT_CLIENT_SECRET,
      redirect_uris: ['http://localhost:3000/auth/microsoft/callback']
    },
    scope: 'openid email profile',
    normalizeUser: (claims) => ({
      id: claims.sub,
      email: claims.email,
      name: claims.name,
      avatar: null
    })
  }
};

// 💡 初始化所有Provider的客户端
const clients = {};

async function initializeProviders() {
  for (const [key, provider] of Object.entries(PROVIDERS)) {
    try {
      let issuer;
      
      if (provider.discoveryUrl) {
        // 使用OIDC Discovery
        issuer = await Issuer.discover(provider.discoveryUrl);
      } else {
        // 手动配置
        issuer = new Issuer(provider.config);
      }
      
      clients[key] = new issuer.Client(provider.clientConfig);
      
      console.log(`✅ ${provider.name} 客户端初始化成功`);
    } catch (error) {
      console.error(`❌ ${provider.name} 初始化失败:`, error.message);
    }
  }
}

initializeProviders().catch(console.error);

// 💡 通用OAuth流程
function createOAuthRoutes(providerKey) {
  const provider = PROVIDERS[providerKey];
  
  // 登录入口
  app.get(`/auth/${providerKey}`, (req, res) => {
    const client = clients[providerKey];
    
    if (!client) {
      return res.status(500).send(`${provider.name} 未配置`);
    }
    
    const codeVerifier = generators.codeVerifier();
    const codeChallenge = generators.codeChallenge(codeVerifier);
    const state = generators.state();
    const nonce = generators.nonce();
    
    req.session.oauth = {
      provider: providerKey,
      state,
      codeVerifier,
      nonce
    };
    
    const authUrl = client.authorizationUrl({
      scope: provider.scope,
      state,
      nonce,
      code_challenge: codeChallenge,
      code_challenge_method: 'S256'
    });
    
    res.redirect(authUrl);
  });
  
  // 回调处理
  app.get(`/auth/${providerKey}/callback`, async (req, res) => {
    try {
      const client = clients[providerKey];
      const params = client.callbackParams(req);
      
      if (params.state !== req.session.oauth?.state) {
        throw new Error('State验证失败');
      }
      
      const codeVerifier = req.session.oauth.codeVerifier;
      const nonce = req.session.oauth.nonce;
      
      const tokenSet = await client.callback(
        provider.clientConfig.redirect_uris[0],
        params,
        {
          code_verifier: codeVerifier,
          state: params.state,
          nonce
        }
      );
      
      let userProfile;
      
      // 💡 根据Provider类型获取用户信息
      if (providerKey === 'github') {
        const response = await fetch(provider.config.userinfo_endpoint, {
          headers: {
            'Authorization': `Bearer ${tokenSet.access_token}`,
            'Accept': 'application/json'
          }
        });
        userProfile = await response.json();
      } else {
        // OpenID Connect使用claims
        userProfile = tokenSet.claims();
      }
      
      // 💡 标准化用户信息
      const normalizedUser = provider.normalizeUser(userProfile);
      
      req.session.user = {
        ...normalizedUser,
        provider: providerKey,
        accessToken: tokenSet.access_token,
        idToken: tokenSet.id_token
      };
      
      delete req.session.oauth;
      
      res.redirect('/profile');
    } catch (error) {
      console.error(`${provider.name} 登录失败:`, error);
      res.status(500).send('登录失败');
    }
  });
}

// 💡 创建所有Provider的路由
Object.keys(PROVIDERS).forEach(createOAuthRoutes);

// 首页
app.get('/', (req, res) => {
  if (req.session.user) {
    return res.redirect('/profile');
  }
  
  const buttons = Object.entries(PROVIDERS)
    .map(([key, provider]) => `
      <a href="/auth/${key}">
        <button>${provider.icon} 使用${provider.name}登录</button>
      </a>
    `)
    .join('\n');
  
  res.send(`
    <!DOCTYPE html>
    <html>
    <head>
      <title>多Provider OAuth登录</title>
      <style>
        body { font-family: Arial; text-align: center; margin-top: 100px; }
        button {
          display: block;
          margin: 15px auto;
          padding: 15px 30px;
          font-size: 16px;
          cursor: pointer;
          min-width: 300px;
        }
      </style>
    </head>
    <body>
      <h1>选择登录方式</h1>
      ${buttons}
    </body>
    </html>
  `);
});

app.listen(3000);
```

### 高频错误1：redirect_uri不匹配

**现象：**
```
redirect_uri_mismatch
The redirect URI provided does not match a registered redirect URI.
```

**原因：**
OAuth回调URL与注册时填写的不一致

**解决方案：**

```javascript

// 注册时：http://localhost:3000/auth/github/callback
// 代码中：http://localhost:3000/auth/github/callback
const callbackURL = process.env.GITHUB_CALLBACK_URL;
authURL.searchParams.set('redirect_uri', callbackURL);

// ✅ 解决方案2：检查协议、域名、端口、路径
// 以下都是不同的URL：
// http://localhost:3000/callback
// https://localhost:3000/callback ← HTTPS
// http://127.0.0.1:3000/callback ← IP地址
// http://localhost:3001/callback ← 端口不同
// http://localhost:3000/callback/ ← 末尾斜杠

// ✅ 解决方案3：多环境配置
const CALLBACK_URLS = {
  development: 'http://localhost:3000/auth/github/callback',
  staging: 'https://staging.example.com/auth/github/callback',
  production: 'https://example.com/auth/github/callback'
};

const callbackURL = CALLBACK_URLS[process.env.NODE_ENV] || CALLBACK_URLS.development;

// ✅ 解决方案4：动态获取（不推荐，不安全）

  const protocol = req.protocol; // http or https
  const host = req.get('host');  // localhost:3000

}

// 使用
```

### 高频错误2：State验证失败（CSRF攻击）

**现象：**
```
State不匹配，可能遭受CSRF攻击
```

**原因：**
回调时的state参数与发起时不一致

**解决方案：**

```javascript
// ❌ 问题代码
app.get('/auth/github', (req, res) => {
  const state = 'fixed-state'; // 固定值，容易被预测
  
  const authURL = new URL('https://github.com/login/oauth/authorize');
  authURL.searchParams.set('state', state);
  
  res.redirect(authURL.toString());
});

app.get('/auth/github/callback', (req, res) => {
  const { state } = req.query;
  
  // 无法验证state是否由本服务器生成
  if (state !== 'fixed-state') {
    return res.status(403).send('State验证失败');
  }
});

// ✅ 正确实现：使用Session存储随机state
import crypto from 'crypto';

app.get('/auth/github', (req, res) => {
  // 生成随机state
  const state = crypto.randomBytes(16).toString('hex');
  
  // 💡 保存到Session
  req.session.oauthState = state;
  
  const authURL = new URL('https://github.com/login/oauth/authorize');
  authURL.searchParams.set('state', state);
  
  res.redirect(authURL.toString());
});

app.get('/auth/github/callback', (req, res) => {
  const { state } = req.query;
  
  // 💡 从Session读取并验证
  const expectedState = req.session.oauthState;
  
  if (!state || !expectedState || state !== expectedState) {
    return res.status(403).send('State验证失败，可能遭受CSRF攻击');
  }
  
  // 💡 验证通过后删除（一次性使用）
  delete req.session.oauthState;
  
  // 继续处理...
});

// ✅ 进阶：添加过期时间
app.get('/auth/github', (req, res) => {
  const state = crypto.randomBytes(16).toString('hex');
  
  req.session.oauthState = {
    value: state,
    createdAt: Date.now()
  };
  
  // 授权URL...
});

app.get('/auth/github/callback', (req, res) => {
  const { state } = req.query;
  const sessionState = req.session.oauthState;
  
  // 验证state存在
  if (!sessionState) {
    return res.status(403).send('State不存在');
  }
  
  // 验证state匹配
  if (state !== sessionState.value) {
    return res.status(403).send('State不匹配');
  }
  
  // 💡 验证未过期（5分钟内）
  const age = Date.now() - sessionState.createdAt;
  if (age > 5 * 60 * 1000) {
    delete req.session.oauthState;
    return res.status(403).send('State已过期，请重新授权');
  }
  
  delete req.session.oauthState;
  
  // 继续处理...
});
```

## 📋 总结

### 核心要点

1. **OAuth 2.1**：授权协议（访问资源），**OIDC**：身份认证（用户是谁）
2. **授权码流程**：最安全的OAuth流程
3. **PKCE**：防止授权码拦截攻击
4. **State参数**：防CSRF攻击

### 实现关键

1. **注册应用**：获取Client ID和Client Secret
2. **配置回调URL**：必须与注册时完全一致
3. **验证State**：每次授权生成随机state并验证
4. **使用openid-client**：简化实现，支持OIDC Discovery

### 应用价值

- **安全授权**：不需要分享密码
- **快速集成**：2小时实现"使用Google登录"
- **用户体验**：一键登录，降低注册门槛
- **标准化**：OIDC标准化用户信息

### 面试重点

**Q1：OAuth 2.1和2.0的区别？**
答：2.1移除了不安全的隐式授权和密码模式，强制要求PKCE，Refresh Token必须轮换。

**Q2：为什么需要PKCE？**
答：防止授权码拦截攻击。攻击者即使截获授权码，也无法换取Token（因为缺少code_verifier）。

**Q3：State参数的作用？**
答：防CSRF攻击。验证回调请求是由本服务器发起的授权流程触发的，而非攻击者伪造的。

**Q4：OAuth和OIDC的区别？**
答：OAuth是授权协议，OIDC是认证协议。OIDC在OAuth基础上增加ID Token（JWT），标准化用户信息（sub、name、email）。

### 注意事项

⚠️ **redirect_uri必须完全匹配**：协议、域名、端口、路径都要一致
⚠️ **Token不能存储在localStorage**：易受XSS攻击
⚠️ **Client Secret不能暴露给客户端**：仅服务器端使用
⚠️ **验证State防CSRF**：每次授权生成新的随机state
