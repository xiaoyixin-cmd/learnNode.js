# 4.4 ORM/ODM 对比与选型

> 深度对比TypeORM、Prisma、Sequelize，助你做出正确选择

---

## 📖 知识点概述

### 核心概念

**ORM（Object-Relational Mapping）**：对象关系映射，用对象的方式操作关系型数据库。

**ODM（Object-Document Mapping）**：对象文档映射，用对象的方式操作文档数据库（如MongoDB）。

### 为什么需要ORM/ODM？

原生SQL的问题：
- **重复代码**：每次都写SQL查询
- **SQL注入风险**：手动拼接SQL易出错
- **缺少类型安全**：TypeScript支持不好
- **迁移困难**：数据库变更需手动同步

ORM/ODM的优势：
- **对象化操作**：`user.save()`而非SQL
- **类型安全**：TypeScript完美支持
- **自动迁移**：数据库结构自动同步
- **跨数据库**：切换数据库无需改代码

## 🔍 主流方案对比

### TypeORM 0.4

**特点**：
- **装饰器语法**：@Entity、@Column等
- **双模式**：Active Record + Data Mapper
- **迁移工具**：自动生成迁移文件
- **关系处理**：@OneToMany、@ManyToOne等

**适用场景**：
- TypeScript项目
- 需要Active Record模式
- 复杂关系模型

**示例**：

```typescript
// entities/User.ts
import { Entity, PrimaryGeneratedColumn, Column, OneToMany } from 'typeorm';
import { Post } from './Post';

@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ unique: true })
  username: string;

  @Column()
  email: string;

  @Column({ select: false })  // 查询时不返回
  password: string;

  @Column({ nullable: true })
  age: number;

  @Column('simple-array', { default: [] })
  tags: string[];

  @Column({ default: true })
  isActive: boolean;

  @OneToMany(() => Post, post => post.author)
  posts: Post[];

  @Column({ type: 'timestamp', default: () => 'CURRENT_TIMESTAMP' })
  createdAt: Date;
}
```

```typescript
// app.ts
import { DataSource } from 'typeorm';
import { User } from './entities/User';

// 💡 创建数据源
const AppDataSource = new DataSource({
  type: 'postgres',
  host: 'localhost',
  port: 5432,
  username: 'postgres',
  password: '',
  database: 'myapp',
  entities: [User],
  synchronize: true, // 开发环境自动同步（生产禁用！）
  logging: true
});

await AppDataSource.initialize();

// 💡 Active Record模式
const user = new User();
user.username = 'john';
user.email = 'john@example.com';
user.password = 'hashed_password';
await user.save();

// 💡 Data Mapper模式
const userRepository = AppDataSource.getRepository(User);

// 创建
const newUser = userRepository.create({
  username: 'jane',
  email: 'jane@example.com',
  password: 'hashed_password'
});
await userRepository.save(newUser);

// 查询
const users = await userRepository.find();
const activeUsers = await userRepository.find({ where: { isActive: true } });
const user = await userRepository.findOne({ where: { username: 'john' } });

// 关联查询
const userWithPosts = await userRepository.findOne({
  where: { id: 1 },
  relations: ['posts']
});

// 复杂查询
const results = await userRepository
  .createQueryBuilder('user')
  .leftJoinAndSelect('user.posts', 'post')
  .where('user.isActive = :isActive', { isActive: true })
  .andWhere('post.title LIKE :title', { title: '%Node.js%' })
  .orderBy('user.createdAt', 'DESC')
  .take(10)
  .getMany();

// 更新
await userRepository.update({ id: 1 }, { age: 26 });

// 删除
await userRepository.delete({ id: 1 });

// 事务
await AppDataSource.transaction(async (manager) => {
  const user = await manager.findOne(User, { where: { id: 1 } });
  user.age = 27;
  await manager.save(user);
  
  // 如果出错，自动回滚
});
```

**迁移**：

```bash
# 生成迁移文件
npx typeorm migration:generate ./src/migrations/CreateUser -d ./src/data-source.ts

# 运行迁移
npx typeorm migration:run -d ./src/data-source.ts

# 回滚迁移
npx typeorm migration:revert -d ./src/data-source.ts
```

### Prisma 5

**特点**：
- **Schema-first**：单一schema.prisma文件
- **类型安全**：自动生成TypeScript类型
- **Prisma Studio**：可视化数据库管理
- **迁移工具**：简洁的迁移命令
- **查询优化**：自动优化N+1问题

**适用场景**：
- 新项目
- 追求开发体验
- TypeScript项目
- 需要可视化工具

**示例**：

```prisma
// schema.prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id        Int      @id @default(autoincrement())
  username  String   @unique
  email     String   @unique
  password  String
  age       Int?
  tags      String[]
  isActive  Boolean  @default(true)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  
  posts     Post[]
}

model Post {
  id        Int      @id @default(autoincrement())
  title     String
  content   String
  authorId  Int
  author    User     @relation(fields: [authorId], references: [id])
  createdAt DateTime @default(now())
}
```

```bash
# 生成Prisma Client
npx prisma generate

# 创建迁移
npx prisma migrate dev --name init

# 应用迁移
npx prisma migrate deploy

# 打开Prisma Studio
npx prisma studio
```

```typescript
// app.ts
import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient();

// 💡 创建
const user = await prisma.user.create({
  data: {
    username: 'john',
    email: 'john@example.com',
    password: 'hashed_password',
    tags: ['developer', 'nodejs']
  }
});

// 💡 查询
const users = await prisma.user.findMany();

const activeUsers = await prisma.user.findMany({
  where: { isActive: true }
});

const user = await prisma.user.findUnique({
  where: { username: 'john' }
});

// 💡 关联查询（自动解决N+1）
const userWithPosts = await prisma.user.findUnique({
  where: { id: 1 },
  include: { posts: true }
});

// 💡 复杂查询
const results = await prisma.user.findMany({
  where: {
    isActive: true,
    age: { gte: 18, lte: 65 },
    tags: { has: 'nodejs' }
  },
  orderBy: { createdAt: 'desc' },
  take: 10,
  skip: 0
});

// 💡 更新
const updated = await prisma.user.update({
  where: { id: 1 },
  data: { age: 26 }
});

// 💡 删除
await prisma.user.delete({
  where: { id: 1 }
});

// 💡 事务
const result = await prisma.$transaction([
  prisma.user.update({
    where: { id: 1 },
    data: { age: 27 }
  }),
  prisma.post.create({
    data: {
      title: 'New Post',
      content: 'Content',
      authorId: 1
    }
  })
]);

// 💡 原生SQL（需要时）
const users = await prisma.$queryRaw`
  SELECT * FROM User WHERE age > ${18}
`;

// 关闭连接
await prisma.$disconnect();
```

### Sequelize 7

**特点**：
- **老牌ORM**：社区成熟
- **多数据库**：MySQL、PostgreSQL、SQLite、MSSQL
- **传统ORM**：类似Java Hibernate
- **钩子系统**：beforeCreate、afterUpdate等

**2025状态**：
- ⚠️ **TypeScript支持较弱**
- ⚠️ **API较老**
- ⚠️ **新项目不推荐**

**适用场景**：
- 维护老项目
- 需要支持多种数据库

**示例**：

```javascript
// models/User.js
import { DataTypes } from 'sequelize';
import { sequelize } from '../db.js';

export const User = sequelize.define('User', {
  id: {
    type: DataTypes.INTEGER,
    primaryKey: true,
    autoIncrement: true
  },
  username: {
    type: DataTypes.STRING,
    unique: true,
    allowNull: false
  },
  email: {
    type: DataTypes.STRING,
    unique: true,
    allowNull: false,
    validate: {
      isEmail: true
    }
  },
  password: {
    type: DataTypes.STRING,
    allowNull: false
  },
  age: {
    type: DataTypes.INTEGER,
    validate: {
      min: 0,
      max: 150
    }
  },
  isActive: {
    type: DataTypes.BOOLEAN,
    defaultValue: true
  }
}, {
  timestamps: true,
  hooks: {
    beforeCreate: async (user) => {
      // 加密密码
      console.log('创建用户前');
    },
    afterCreate: async (user) => {
      console.log('创建用户后');
    }
  }
});

// 定义关联
User.hasMany(Post, { foreignKey: 'authorId', as: 'posts' });
```

```javascript
// app.js
import { sequelize } from './db.js';
import { User } from './models/User.js';

// 同步数据库
await sequelize.sync({ alter: true });

// 创建
const user = await User.create({
  username: 'john',
  email: 'john@example.com',
  password: 'hashed_password'
});

// 查询
const users = await User.findAll();
const activeUsers = await User.findAll({ where: { isActive: true } });
const user = await User.findOne({ where: { username: 'john' } });

// 关联查询
const userWithPosts = await User.findOne({
  where: { id: 1 },
  include: ['posts']
});

// 更新
await User.update({ age: 26 }, { where: { id: 1 } });

// 删除
await User.destroy({ where: { id: 1 } });

// 事务
await sequelize.transaction(async (t) => {
  const user = await User.findByPk(1, { transaction: t });
  user.age = 27;
  await user.save({ transaction: t });
});
```

## 🎯 选型建议

### 选择Prisma如果：
- ✅ 新项目
- ✅ 追求开发体验
- ✅ TypeScript项目
- ✅ 需要自动优化查询
- ✅ 需要可视化工具

### 选择TypeORM如果：
- ✅ 需要Active Record模式
- ✅ 需要支持MongoDB
- ✅ 复杂关系模型
- ✅ 装饰器语法爱好者

### 选择Sequelize如果：
- ✅ 维护老项目
- ✅ 需要支持多种数据库
- ⚠️ 新项目不推荐

### 不使用ORM如果：
- ✅ 极致性能要求
- ✅ 复杂SQL查询
- ✅ 团队SQL能力强

## 💡 实战建议

### 1. 小型项目
```
推荐：Prisma
原因：快速开发、类型安全、易维护
```

### 2. 大型企业项目
```
推荐：TypeORM或Prisma
原因：完善的迁移、TypeScript支持、团队协作
```

### 3. 高性能项目
```
推荐：原生SQL + 连接池
原因：最大化性能，完全控制查询
```

### 4. 微服务
```
推荐：Prisma
原因：轻量、独立schema、易部署
```

## 📋 总结

### 核心要点

1. **ORM/ODM**：对象化操作数据库
2. **Prisma**：2025年最推荐，开发体验最佳
3. **TypeORM**：功能强大，支持MongoDB
4. **Sequelize**：老牌ORM，新项目不推荐

### 选型关键

1. **项目类型**：新项目选Prisma，老项目看情况
2. **团队技能**：SQL强选原生，否则选ORM
3. **性能要求**：极致性能选原生，一般选ORM
4. **数据库**：MongoDB选TypeORM或Mongoose

### 应用价值

- **开发效率**：减少重复代码
- **类型安全**：TypeScript完美支持
- **易维护**：迁移工具、自动同步
- **跨数据库**：切换数据库简单

### 面试重点

1. **ORM的优缺点？**
   - 优点：开发效率、类型安全、易维护
   - 缺点：性能略低、复杂查询受限

2. **Prisma vs TypeORM？**
   - Prisma：更现代、更简洁、类型安全最佳
   - TypeORM：功能更多、支持MongoDB

3. **何时不用ORM？**
   - 极致性能、复杂SQL、数据分析

4. **如何优化ORM性能？**
   - 使用索引、避免N+1、批量操作、原生SQL

### 注意事项

⚠️ **生产环境禁用synchronize**：
```typescript
// ❌ 生产环境
synchronize: true // 会删除表！

// ✅ 生产环境
synchronize: false
```

⚠️ **注意N+1查询问题**：
```typescript
// ❌ N+1问题
const users = await prisma.user.findMany();
for (const user of users) {
  user.posts = await prisma.post.findMany({ where: { authorId: user.id } });
}

// ✅ 使用include
const users = await prisma.user.findMany({ include: { posts: true } });
```
