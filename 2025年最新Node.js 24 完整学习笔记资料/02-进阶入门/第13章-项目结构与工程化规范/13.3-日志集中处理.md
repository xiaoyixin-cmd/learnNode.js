# 13.3 日志集中处理

> Pino高性能日志、结构化日志、日志级别、日志轮转、日志聚合

## 📖 知识点概述

### 核心概念

**日志系统**是应用可观测性的基础，记录应用运行状态、错误信息、用户行为等。好的日志系统应该具备高性能、结构化、易于查询的特点。

### 为什么需要日志系统？

```
❌ 使用console.log：

console.log('User logged in:', userId);
console.log('Error:', error.message);

问题：
- 性能差（同步I/O阻塞）
- 格式不统一
- 无法按级别过滤
- 难以查询和分析
- 生产环境无法禁用

✅ 使用Pino日志库：

logger.info({ userId }, 'User logged in');
logger.error({ err, userId }, 'Login failed');

优点：
- 高性能（异步I/O）
- 结构化（JSON格式）
- 级别控制（error/warn/info/debug）
- 易于查询（ELK、云日志服务）
- 生产环境可配置
```

### 日志级别

```
┌────────────────────────────────────┐
│         日志级别（从高到低）        │
├────────────────────────────────────┤
│                                     │
│  fatal (60)  致命错误，应用崩溃     │
│  error (50)  错误，需要立即处理     │
│  warn  (40)  警告，需要关注         │
│  info  (30)  信息，正常运行状态     │
│  debug (20)  调试信息               │
│  trace (10)  追踪信息，最详细       │
│                                     │
│  推荐配置：                         │
│  - 开发环境：debug                  │
│  - 测试环境：info                   │
│  - 生产环境：warn                   │
│                                     │
└────────────────────────────────────┘
```

```typescript
// utils/logger.ts
import pino from 'pino';
import path from 'path';

const isDevelopment = process.env.NODE_ENV === 'development';
const logLevel = process.env.LOG_LEVEL || (isDevelopment ? 'debug' : 'info');

// 💡 生产环境配置
const productionConfig = {
  level: logLevel,
  
  formatters: {
    level: (label: string) => ({ level: label }),
    bindings: (bindings: pino.Bindings) => ({
      pid: bindings.pid,
      hostname: bindings.hostname,
      node_version: process.version
    })
  },
  
  timestamp: pino.stdTimeFunctions.isoTime,
  
  // 💡 序列化错误对象
  serializers: {
    err: pino.stdSerializers.err,
    req: pino.stdSerializers.req,
    res: pino.stdSerializers.res
  },
  
  // 💡 重定向到文件
  transport: {
    targets: [
      {
        target: 'pino/file',
        level: 'info',
        options: {
          destination: path.join(process.cwd(), 'logs', 'app.log')
        }
      },
      {
        target: 'pino/file',
        level: 'error',
        options: {
          destination: path.join(process.cwd(), 'logs', 'error.log')
        }
      }
    ]
  }
};

// 💡 开发环境配置
const developmentConfig = {
  level: 'debug',
  transport: {
    target: 'pino-pretty',
    options: {
      colorize: true,
      translateTime: 'SYS:standard',
      ignore: 'pid,hostname',
      messageFormat: '{levelLabel} - {msg}'
    }
  }
};

export const logger = pino(
  isDevelopment ? developmentConfig : productionConfig
);

// 💡 创建子logger
export function createLogger(context: string) {
  return logger.child({ context });
}

// 使用
const userLogger = createLogger('UserService');
userLogger.info({ userId: '123' }, 'User created');
// 输出：{ "context": "UserService", "userId": "123", "msg": "User created" }
```

#### Express集成

```typescript
// middleware/logger.middleware.ts
import { Request, Response, NextFunction } from 'express';
import { logger } from '../utils/logger';
import { randomUUID } from 'crypto';

export function requestLoggerMiddleware(
  req: Request,
  res: Response,
  next: NextFunction
) {
  // 💡 生成请求ID
  const requestId = randomUUID();
  req.id = requestId;
  
  // 💡 创建请求级别的logger
  req.log = logger.child({
    requestId,
    method: req.method,
    url: req.url,
    userAgent: req.get('user-agent')
  });
  
  // 💡 记录请求开始
  req.log.info('Request started');
  
  // 💡 记录响应
  const startTime = Date.now();
  
  res.on('finish', () => {
    const duration = Date.now() - startTime;
    
    req.log.info({
      statusCode: res.statusCode,
      duration,
      contentLength: res.get('content-length')
    }, 'Request completed');
  });
  
  next();
}

// app.ts
import express from 'express';
import { requestLoggerMiddleware } from './middleware/logger.middleware';

const app = express();

app.use(requestLoggerMiddleware);

app.get('/users', (req, res) => {
  // 使用请求级别的logger
  req.log.info('Fetching users');
  res.json({ users: [] });
});

// 扩展Express类型
declare global {
  namespace Express {
    interface Request {
      id: string;
      log: pino.Logger;
    }
  }
}
```

### 实践5：结构化日志

```typescript
// services/user.service.ts
import { logger } from '../utils/logger';

export class UserService {
  private log = logger.child({ service: 'UserService' });
  
  async createUser(data: CreateUserDto) {
    // 💡 结构化日志
    this.log.info({ 
      email: data.email,
      role: data.role 
    }, 'Creating user');
    
    try {
      const user = await this.userRepository.create(data);
      
      // 成功日志
      this.log.info({ 
        userId: user.id,
        email: user.email 
      }, 'User created successfully');
      
      return user;
    } catch (error) {
      // 💡 错误日志（包含错误栈）
      this.log.error({ 
        err: error,
        email: data.email 
      }, 'Failed to create user');
      
      throw error;
    }
  }
  
  async updateUser(id: string, data: UpdateUserDto) {
    const startTime = Date.now();
    
    this.log.debug({ userId: id, updates: data }, 'Updating user');
    
    try {
      const user = await this.userRepository.update(id, data);
      
      const duration = Date.now() - startTime;
      
      // 💡 性能日志
      this.log.info({ 
        userId: id,
        duration,
        updatedFields: Object.keys(data)
      }, 'User updated');
      
      return user;
    } catch (error) {
      this.log.error({ err: error, userId: id }, 'Failed to update user');
      throw error;
    }
  }
}
```

## 🔧 错误处理

### 高频错误1：日志过多影响性能

**问题：**
```typescript
// ❌ 循环中打印大量日志
for (const user of users) {
  logger.info({ user }, 'Processing user');
}
```

**解决方案：**
```typescript
// ✅ 批量日志
logger.info({ count: users.length }, 'Processing users');

// 处理完成后记录
logger.info({ 
  total: users.length,
  success: successCount,
  failed: failedCount
}, 'Users processed');

// ✅ 使用debug级别（生产环境不输出）
for (const user of users) {
  logger.debug({ userId: user.id }, 'Processing user');
}
```

### 高频错误2：敏感信息泄露

**问题：**
```typescript
// ❌ 记录密码
logger.info({ user }, 'User created');
// 输出：{ "user": { "email": "...", "password": "secret123" } }
```

**解决方案：**
```typescript
// ✅ 过滤敏感字段
const sanitizeUser = (user: User) => {
  const { password, ...safe } = user;
  return safe;
};

logger.info({ user: sanitizeUser(user) }, 'User created');

// ✅ 使用Pino的redact功能
const logger = pino({
  redact: {
    paths: ['password', 'token', 'secret', '*.password', 'req.headers.authorization'],
    censor: '[REDACTED]'
  }
});

logger.info({ 
  user: { email: 'user@example.com', password: 'secret123' }
});
// 输出：{ "user": { "email": "...", "password": "[REDACTED]" } }
```

### 高频错误3：日志文件过大

**问题：**
```
logs/app.log: 10GB
```

**解决方案：**
```typescript
// ✅ 使用日志轮转
import pino from 'pino';
import { createStream } from 'rotating-file-stream';

const stream = createStream('app.log', {
  interval: '1d',        // 每天轮转

  compress: 'gzip',      // 压缩旧文件
  path: './logs'
});

export const logger = pino(stream);

// 或使用pino-roll

  pino.transport({
    target: 'pino-roll',

      file: './logs/app',
```

## 📋 总结

### 核心要点

1. **Pino日志库**：高性能、异步I/O
2. **结构化日志**：JSON格式，易于查询
3. **日志级别**：error/warn/info/debug
4. **日志轮转**：按日期/大小轮转

### 实现关键

1. **pino**: 创建logger实例
2. **pino-pretty**: 开发环境格式化
3. **pino-roll**: 日志轮转
4. **redact**: 过滤敏感信息

### 应用价值

- **可观测性**：了解应用运行状态
- **问题定位**：快速找到错误原因
- **性能分析**：记录接口响应时间
- **审计追踪**：记录用户操作

### 面试重点

**Q1：为什么不用console.log？**
答：性能差（同步I/O）、格式不统一、无法按级别过滤、难以查询。Pino使用异步I/O，性能高5-10倍。

**Q2：什么是结构化日志？**
答：JSON格式的日志，包含结构化字段（userId、method、duration等），易于查询和分析。

**Q3：如何保护敏感信息？**
答：使用redact过滤敏感字段（password、token），或在记录前手动过滤。

**Q4：如何管理日志文件？**
答：使用日志轮转（按日期/大小），压缩旧日志，定期删除过期日志。

### 注意事项

⚠️ **避免过度日志**：影响性能，占用磁盘
⚠️ **保护敏感信息**：过滤密码、token
⚠️ **日志轮转**：防止单文件过大
⚠️ **生产环境级别**：warn或error，避免debug
