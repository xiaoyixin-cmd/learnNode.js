# 13.7 Monorepo最佳实践

> Monorepo架构、Turborepo vs Nx、workspace配置、共享代码包、增量构建

---

## 📖 知识点概述

### 核心概念

**Monorepo（单一仓库）**是将多个相关项目（应用、库）放在同一个Git仓库中管理，共享代码、工具链和依赖，提高开发效率。

### Monorepo vs Polyrepo

```
Polyrepo（多仓库）：
├── api-repo/
│   ├── src/
│   └── package.json
├── web-repo/
│   ├── src/
│   └── package.json
└── shared-repo/
    ├── src/
    └── package.json

问题：
- 代码共享困难
- 依赖管理复杂
- CI/CD分散
- 版本管理困难

Monorepo（单仓库）：
monorepo/
├── apps/
│   ├── api/
│   │   ├── src/
│   │   └── package.json
│   └── web/
│       ├── src/
│       └── package.json
├── packages/
│   ├── shared/
│   │   ├── src/
│   │   └── package.json
│   └── ui/
│       ├── src/
│       └── package.json
├── package.json
└── turbo.json

优点：
- 代码共享简单
- 统一依赖管理
- 统一CI/CD
- 统一版本控制
```

### 实践3：Turborepo配置

```json
// turbo.json
{
  "$schema": "https://turbo.build/schema.json",
  "globalDependencies": [".env", "tsconfig.json"],
  
  "pipeline": {
    // 💡 构建任务
    "build": {
      "dependsOn": ["^build"],
      "outputs": ["dist/**", ".next/**", "build/**"],
      "env": ["NODE_ENV"]
    },
    
    // 💡 开发任务
    "dev": {
      "cache": false,
      "persistent": true
    },
    
    // 💡 测试任务
    "test": {
      "dependsOn": ["build"],
      "outputs": ["coverage/**"],
      "inputs": ["src/**", "test/**"]
    },
    
    // 💡 Lint任务
    "lint": {
      "outputs": []
    },
    
    // 💡 清理任务
    "clean": {
      "cache": false
    }
  }
}

// 说明：
// - dependsOn: ["^build"] 表示先构建依赖包
// - outputs: 指定输出目录（用于缓存）
// - cache: false 表示不缓存（如dev、clean）
// - persistent: true 表示持续运行（如dev server）
```

### 实践4：共享代码包

```typescript
// packages/shared/src/index.ts
export function formatDate(date: Date): string {
  return date.toISOString().split('T')[0];
}

export function generateId(): string {
  return Math.random().toString(36).substring(2, 15);
}

export class Logger {
  info(message: string, meta?: any) {
    console.log(`[INFO] ${message}`, meta);
  }
  
  error(message: string, meta?: any) {
    console.error(`[ERROR] ${message}`, meta);
  }
}

// packages/types/src/index.ts
export interface User {
  id: string;
  email: string;
  name: string;
  role: 'user' | 'admin';
  createdAt: Date;
}

export interface ApiResponse<T> {
  success: boolean;
  data?: T;
  error?: {
    code: string;
    message: string;
  };
}

// apps/api/src/server.ts
import express from 'express';
import { Logger, formatDate } from '@my-monorepo/shared';
import type { User, ApiResponse } from '@my-monorepo/types';

const logger = new Logger();
const app = express();

app.get('/users', (req, res) => {
  const users: User[] = [
    {
      id: '1',
      email: 'user@example.com',
      name: 'User',
      role: 'user',
      createdAt: new Date()
    }
  ];
  
  logger.info('Fetching users');
  
  res.json<ApiResponse<User[]>>({
    success: true,
    data: users
  });
});

app.listen(3000, () => {
  logger.info(`Server started at ${formatDate(new Date())}`);
});
```

## 🔧 错误处理

### 高频错误1：依赖循环

**问题：**
```
Error: Circular dependency detected:
  @my-monorepo/api -> @my-monorepo/shared -> @my-monorepo/api
```

**解决方案：**
```
// ❌ 错误：循环依赖
packages/shared/
  └─ 依赖 @my-monorepo/api

apps/api/
  └─ 依赖 @my-monorepo/shared

// ✅ 正确：单向依赖
apps/api/
  └─ 依赖 @my-monorepo/shared

packages/shared/
  └─ 不依赖任何应用

// 💡 原则：
// - packages 不依赖 apps
// - apps 可以依赖 packages
// - packages 之间避免循环依赖
```

### 高频错误2：workspace依赖未找到

**问题：**
```
Error: Cannot find module '@my-monorepo/shared'
```

**解决方案：**
```bash
# 1. 确认workspace配置正确
# pnpm-workspace.yaml
packages:
  - 'apps/*'
  - 'packages/*'

# 2. 确认package.json中依赖格式正确
# apps/api/package.json
{
  "dependencies": {
    "@my-monorepo/shared": "workspace:*"  # workspace协议
  }
}

# 3. 重新安装依赖
pnpm install

# 4. 构建依赖包
pnpm --filter @my-monorepo/shared build

# 5. 使用Turborepo构建所有依赖
pnpm turbo build
```

### 高频错误3：缓存问题

**问题：**
```
# 修改了代码但没有生效
```

**解决方案：**
```bash
# 💡 清除Turborepo缓存
npx turbo run build --force

# 💡 清除所有缓存和node_modules
rm -rf node_modules apps/*/node_modules packages/*/node_modules
rm -rf .turbo apps/*/.turbo packages/*/.turbo
pnpm install

# 💡 清除并重新构建
pnpm clean
pnpm build
```

## 📋 总结

### 核心要点

1. **Monorepo架构**：多项目单仓库管理
2. **Turborepo**：简单高效的构建工具
3. **workspace**：pnpm/npm/yarn workspace
4. **共享代码**：packages共享代码和配置

### 实现关键

1. **turbo.json**: 配置任务依赖和缓存
2. **workspace**: 配置workspace范围
3. **workspace:***: 依赖本地包
4. **增量构建**: 只构建变化的项目

### 应用价值

- **代码共享**：packages复用代码
- **统一工具链**：统一ESLint、TypeScript配置
- **增量构建**：只构建变化的项目，提速90%
- **统一版本管理**：一个仓库管理所有版本

### 面试重点

**Q1：什么是Monorepo？优势是什么？**
答：单一仓库管理多个项目。优势：代码共享简单、统一依赖管理、统一CI/CD、原子提交。

**Q2：Turborepo如何提速构建？**
答：1) 增量构建（只构建变化的项目）2) 并行构建 3) 任务缓存 4) 远程缓存。

**Q3：如何共享代码？**
答：创建packages，配置为workspace包，其他项目通过`workspace:*`依赖。

**Q4：Turborepo vs Nx怎么选？**
答：简单项目用Turborepo（配置少、易上手）；大型项目用Nx（功能强大、生成器丰富）。

### 注意事项

⚠️ **避免循环依赖**：packages不依赖apps
⚠️ **构建顺序**：dependsOn: ["^build"]
⚠️ **版本管理**：使用changesets管理版本
⚠️ **缓存配置**：正确配置outputs
