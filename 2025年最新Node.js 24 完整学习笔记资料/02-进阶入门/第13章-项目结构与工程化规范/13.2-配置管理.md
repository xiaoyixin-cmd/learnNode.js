# 13.2 配置管理

> dotenv环境变量、配置文件、多环境配置、配置验证、敏感信息管理

---

## 📖 知识点概述

### 核心概念

**配置管理**是将应用的可变参数（数据库连接、API密钥等）与代码分离，使同一份代码可以在不同环境（开发、测试、生产）运行，遵循12-Factor App原则。

### 为什么需要配置管理？

```
❌ 硬编码配置：

// config.ts
export const config = {
  database: {
    host: 'localhost',
    port: 5432,
    username: 'admin',
    password: 'secret123'  // ⚠️ 敏感信息暴露
  },
  jwt: {
    secret: 'my-jwt-secret'  // ⚠️ 提交到Git
  }
};

问题：
- 配置固定，无法适应不同环境
- 敏感信息泄露风险
- 修改配置需要重新构建
- 不符合12-Factor App原则

✅ 环境变量配置：

// .env
DATABASE_URL=postgresql://user:pass@localhost:5432/db
JWT_SECRET=super-secret-key-at-least-32-chars

// config.ts
export const config = {
  database: {
    url: process.env.DATABASE_URL
  },
  jwt: {
    secret: process.env.JWT_SECRET
  }
};

优点：
- 同一代码适用所有环境
- 敏感信息不进入代码库
- 配置灵活可变
- 符合云原生标准
```

### 实践2：配置验证（Joi）

```typescript
// config/index.ts
import dotenv from 'dotenv';
import Joi from 'joi';
import path from 'path';

dotenv.config();

// 💡 定义配置验证模式
const envSchema = Joi.object({
  NODE_ENV: Joi.string()
    .valid('development', 'staging', 'production', 'test')
    .default('development'),
  
  PORT: Joi.number().default(3000),
  
  DATABASE_URL: Joi.string().required()
    .description('MongoDB connection string'),
  
  REDIS_URL: Joi.string().required()
    .description('Redis connection string'),
  
  JWT_SECRET: Joi.string().min(32).required()
    .description('JWT secret key (at least 32 characters)'),
  
  JWT_EXPIRES_IN: Joi.string().default('24h'),
  
  SMTP_HOST: Joi.string().required(),
  SMTP_PORT: Joi.number().default(587),
  SMTP_USER: Joi.string().email().required(),
  SMTP_PASSWORD: Joi.string().required(),
  
  LOG_LEVEL: Joi.string()
    .valid('error', 'warn', 'info', 'debug')
    .default('info'),
  
  RATE_LIMIT_WINDOW_MS: Joi.number().default(15 * 60 * 1000),
  RATE_LIMIT_MAX_REQUESTS: Joi.number().default(100)
}).unknown(); // 允许其他环境变量

// 💡 验证环境变量
const { error, value: env } = envSchema.validate(process.env, {
  abortEarly: false // 显示所有错误
});

if (error) {
  const errors = error.details.map(detail => detail.message).join('\n');
  throw new Error(`Configuration validation failed:\n${errors}`);
}

// 💡 导出类型安全的配置
export const config = {
  env: env.NODE_ENV as 'development' | 'staging' | 'production' | 'test',
  port: env.PORT,
  
  database: {
    url: env.DATABASE_URL
  },
  
  redis: {
    url: env.REDIS_URL
  },
  
  jwt: {
    secret: env.JWT_SECRET,
    expiresIn: env.JWT_EXPIRES_IN
  },
  
  smtp: {
    host: env.SMTP_HOST,
    port: env.SMTP_PORT,
    auth: {
      user: env.SMTP_USER,
      pass: env.SMTP_PASSWORD
    }
  },
  
  logging: {
    level: env.LOG_LEVEL
  },
  
  rateLimit: {
    windowMs: env.RATE_LIMIT_WINDOW_MS,
    max: env.RATE_LIMIT_MAX_REQUESTS
  }
} as const;

// 💡 导出类型
export type Config = typeof config;
```

#### 使用HashiCorp Vault

```typescript
// config/vault.ts
import vault from 'node-vault';

const client = vault({
  endpoint: process.env.VAULT_ADDR || 'http://localhost:8200',
  token: process.env.VAULT_TOKEN
});

export async function getSecrets() {
  try {
    // 从Vault读取secrets
    const response = await client.read('secret/data/myapp');
    
    return {
      database: {
        url: response.data.data.DATABASE_URL
      },
      jwt: {
        secret: response.data.data.JWT_SECRET
      }
    };
  } catch (error) {
    console.error('Failed to read secrets from Vault:', error);
    throw error;
  }
}

// server.ts
import { getSecrets } from './config/vault';

async function bootstrap() {
  // 从Vault获取配置
  const secrets = await getSecrets();
  
  // 合并到配置
  Object.assign(config.database, secrets.database);
  Object.assign(config.jwt, secrets.jwt);
  
  // 启动应用
  app.listen(config.port);
}

bootstrap();
```

## 🔧 错误处理

### 高频错误1：环境变量缺失

**现象：**
```
TypeError: Cannot read property 'DATABASE_URL' of undefined
```

**解决方案：**
```typescript
// ❌ 错误：直接使用未验证的环境变量
const dbUrl = process.env.DATABASE_URL;

// ✅ 正确：使用Joi验证
import Joi from 'joi';

const schema = Joi.object({
  DATABASE_URL: Joi.string().required()
}).unknown();

const { error, value } = schema.validate(process.env);

if (error) {
  console.error('Missing environment variable: DATABASE_URL');
  process.exit(1);
}

// ✅ 或使用dotenv-safe
// npm install dotenv-safe
import 'dotenv-safe/config';

// .env.example（必需的环境变量）
// DATABASE_URL=
// JWT_SECRET=

// 如果.env缺少变量，启动时会报错
```

### 高频错误2：类型不正确

**现象：**
```typescript
const port = process.env.PORT; // string
app.listen(port); // 期望number
```

**解决方案：**
```typescript
// ✅ 正确：类型转换和验证
const port = parseInt(process.env.PORT || '3000', 10);

if (isNaN(port)) {
  throw new Error('PORT must be a number');
}

// ✅ 或使用Zod验证
import { z } from 'zod';

const envSchema = z.object({
  PORT: z.string().transform(val => parseInt(val, 10)).pipe(z.number().min(1).max(65535)),
  DATABASE_URL: z.string().url(),
  JWT_SECRET: z.string().min(32)
});

const env = envSchema.parse(process.env);
// env.PORT 是 number 类型
```

### 高频错误3：敏感信息泄露

**问题：**
```bash
# .env文件被提交到Git
git log --all --full-history -- .env
```

**防护方案：**
```bash
# 1. 立即添加到.gitignore
echo ".env" >> .gitignore
git add .gitignore
git commit -m "Add .env to .gitignore"

# 2. 从Git历史中移除
git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch .env' \
  --prune-empty --tag-name-filter cat -- --all

# 3. 强制推送（⚠️ 危险操作）
git push origin --force --all

# 4. 更新所有secrets
# 修改DATABASE_URL、JWT_SECRET等

# 5. 使用pre-commit hook防止未来泄露
# .husky/pre-commit
#!/bin/sh
if git diff --cached --name-only | grep -q "^\.env$"; then
  echo "Error: Attempting to commit .env file"
  exit 1
fi
```

## 📚 应用场景

### 场景：完整的配置管理系统

```typescript
// config/schema.ts
import Joi from 'joi';

export const configSchema = Joi.object({
  // 应用配置
  app: Joi.object({
    name: Joi.string().default('my-app'),
    version: Joi.string().default('1.0.0'),
    env: Joi.string().valid('development', 'staging', 'production', 'test').required(),
    port: Joi.number().min(1).max(65535).default(3000)
  }),

  // 数据库配置
  database: Joi.object({
    url: Joi.string().required(),
    poolSize: Joi.number().default(10),
    ssl: Joi.boolean().default(false)
  }),

  // Redis配置
  redis: Joi.object({
    url: Joi.string().required(),
    ttl: Joi.number().default(3600),
    maxRetriesPerRequest: Joi.number().default(3)
  }),

  // JWT配置
  jwt: Joi.object({
    secret: Joi.string().min(32).required(),
    expiresIn: Joi.string().default('24h'),
    refreshExpiresIn: Joi.string().default('7d')
  }),

  // SMTP配置
  smtp: Joi.object({
    host: Joi.string().required(),
    port: Joi.number().default(587),
    secure: Joi.boolean().default(false),
    user: Joi.string().email().required(),
    password: Joi.string().required()
  }),

  // AWS配置
  aws: Joi.object({
    region: Joi.string().default('us-east-1'),
    accessKeyId: Joi.string().when('$env', {
      is: 'production',
      then: Joi.required(),
      otherwise: Joi.optional()
    }),
    secretAccessKey: Joi.string().when('$env', {
      is: 'production',
      then: Joi.required(),
      otherwise: Joi.optional()
    }),
    s3: Joi.object({
      bucket: Joi.string().required()
    })
  }),

  // 日志配置
  logging: Joi.object({
    level: Joi.string().valid('error', 'warn', 'info', 'debug').default('info'),
    prettyPrint: Joi.boolean().default(false)
  }),

  // 限流配置
  rateLimit: Joi.object({
    windowMs: Joi.number().default(15 * 60 * 1000),
    max: Joi.number().default(100)
  }),

  // CORS配置
  cors: Joi.object({
    origin: Joi.alternatives().try(
      Joi.string(),
      Joi.array().items(Joi.string())
    ).default('*'),
    credentials: Joi.boolean().default(true)
  })
}).unknown();

// config/index.ts
import dotenv from 'dotenv';
import { configSchema } from './schema';

dotenv.config();

const rawConfig = {
  app: {
    name: process.env.APP_NAME,
    version: process.env.APP_VERSION,
    env: process.env.NODE_ENV,
    port: process.env.PORT
  },
  database: {
    url: process.env.DATABASE_URL,
    poolSize: process.env.DB_POOL_SIZE,
    ssl: process.env.DB_SSL === 'true'
  },
  redis: {
    url: process.env.REDIS_URL,
    ttl: process.env.REDIS_TTL,
    maxRetriesPerRequest: process.env.REDIS_MAX_RETRIES
  },
  jwt: {
    secret: process.env.JWT_SECRET,
    expiresIn: process.env.JWT_EXPIRES_IN,
    refreshExpiresIn: process.env.JWT_REFRESH_EXPIRES_IN
  },
  smtp: {
    host: process.env.SMTP_HOST,
    port: process.env.SMTP_PORT,
    secure: process.env.SMTP_SECURE === 'true',
    user: process.env.SMTP_USER,
    password: process.env.SMTP_PASSWORD
  },
  aws: {
    region: process.env.AWS_REGION,
    accessKeyId: process.env.AWS_ACCESS_KEY_ID,
    secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY,
    s3: {
      bucket: process.env.AWS_S3_BUCKET
    }
  },
  logging: {
    level: process.env.LOG_LEVEL,
    prettyPrint: process.env.LOG_PRETTY_PRINT === 'true'
  },
  rateLimit: {
    windowMs: process.env.RATE_LIMIT_WINDOW_MS,
    max: process.env.RATE_LIMIT_MAX
  },
  cors: {
    origin: process.env.CORS_ORIGIN?.split(','),
    credentials: process.env.CORS_CREDENTIALS === 'true'
  }
};

const { error, value: config } = configSchema.validate(rawConfig, {
  context: { env: process.env.NODE_ENV },
  abortEarly: false,
  stripUnknown: true
});

if (error) {
  const errors = error.details.map(d => `- ${d.message}`).join('\n');
  console.error(`Configuration validation failed:\n${errors}`);
  process.exit(1);
}

export default config;

// 💡 使用
import config from './config';

console.log(`Starting ${config.app.name} v${config.app.version}`);
console.log(`Environment: ${config.app.env}`);
console.log(`Port: ${config.app.port}`);

// 类型安全
const port: number = config.app.port; // ✅ 类型正确
```

## 📋 总结

### 核心要点

1. **环境变量优先**：使用.env文件
2. **配置验证**：Joi/Zod验证必需配置
3. **多环境支持**：.env.development/.env.production
4. **敏感信息**：不提交到Git，使用Vault

### 实现关键

1. **dotenv**: 加载.env文件
2. **Joi/Zod**: 验证配置
3. **.env.example**: 提供配置模板
4. **.gitignore**: 排除敏感文件

### 应用价值

- **环境一致性**：同一代码适用所有环境
- **安全性**：敏感信息不进入代码库
- **灵活性**：配置可变，无需重新构建
- **规范性**：符合12-Factor App原则

### 面试重点

**Q1：为什么使用环境变量而非配置文件？**
答：符合12-Factor App，配置与代码分离，同一代码适用所有环境，敏感信息不进入代码库。

**Q2：如何管理多环境配置？**
答：1) 多个.env文件（.env.development/.env.production） 2) 配置文件+环境变量覆盖 3) 根据NODE_ENV加载不同配置。

**Q3：如何保证配置正确性？**
答：使用Joi/Zod验证，定义必需字段、类型、默认值，启动时验证，错误立即退出。

**Q4：如何保护敏感信息？**
答：1) .env不提交Git 2) 使用dotenv-vault加密 3) 生产环境使用HashiCorp Vault 4) CI/CD平台的Secrets功能。

### 注意事项

⚠️ **.env不要提交**：添加到.gitignore
⚠️ **验证配置**：启动时检查必需变量
⚠️ **类型转换**：字符串转数字/布尔
⚠️ **敏感信息**：使用专门的Secret管理工具
