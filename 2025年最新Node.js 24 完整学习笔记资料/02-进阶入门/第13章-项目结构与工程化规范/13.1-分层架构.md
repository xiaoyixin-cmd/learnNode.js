# 13.1 分层架构

> MVC架构、三层架构设计、依赖注入、目录结构最佳实践

---

## 📖 知识点概述

### 核心概念

**分层架构（Layered Architecture）**是将应用按照职责划分为不同的层次，每层只负责特定的功能，层与层之间通过明确的接口通信。这种架构提高了代码的可维护性、可测试性和可扩展性。

### 为什么需要分层架构？

```
❌ 不使用分层架构：所有逻辑混在一起

// routes/user.js
app.post('/users', async (req, res) => {
  // 验证
  if (!req.body.email) {
    return res.status(400).json({ error: 'Email required' });
  }
  
  // 业务逻辑
  const hashedPassword = await bcrypt.hash(req.body.password, 10);
  
  // 数据库操作
  const user = await db.collection('users').insertOne({
    email: req.body.email,
    password: hashedPassword,
    createdAt: new Date()
  });
  
  // 发送邮件
  await sendEmail(user.email, 'Welcome');
  
  // 返回响应
  res.json({ id: user.insertedId });
});

问题：
- 路由处理、验证、业务逻辑、数据库操作全混在一起
- 难以测试（无法mock数据库）
- 难以复用（其他接口也要创建用户怎么办？）
- 难以维护（一个函数做了太多事情）

✅ 使用分层架构：职责清晰

// controllers/user.controller.ts
export class UserController {
  async createUser(req, res) {
    const dto = validate(CreateUserDto, req.body);
    const user = await this.userService.create(dto);
    res.json({ data: user });
  }
}

// services/user.service.ts
export class UserService {
  async create(dto: CreateUserDto) {
    const hashedPassword = await bcrypt.hash(dto.password, 10);
    const user = await this.userRepository.create({
      ...dto,
      password: hashedPassword
    });
    await this.emailService.sendWelcomeEmail(user.email);
    return user;
  }
}

// repositories/user.repository.ts
export class UserRepository {
  async create(data: CreateUserData) {
    return this.db.collection('users').insertOne(data);
  }
}

优点：
- 职责清晰，每层只做自己的事
- 易于测试（可以mock依赖）
- 易于复用（Service可以被多个Controller使用）
- 易于维护（修改数据库只需改Repository）
```

### 常见分层架构模式

```
┌────────────────────────────────────────────────┐
│             分层架构对比                        │
├────────────────────────────────────────────────┤
│                                                 │
│  MVC（Model-View-Controller）                   │
│  ├─ View: 前端展示（在Node.js API中不常用）    │
│  ├─ Controller: 处理HTTP请求                    │
│  └─ Model: 数据模型和业务逻辑                   │
│                                                 │
│  三层架构（最常用）                             │
│  ├─ Controller: 处理HTTP请求、参数验证          │
│  ├─ Service: 业务逻辑处理                       │
│  └─ Repository/DAO: 数据访问                    │
│                                                 │
│  四层架构（大型项目）                           │
│  ├─ Controller: HTTP层                          │
│  ├─ Service: 业务逻辑层                         │
│  ├─ Domain: 领域模型层                          │
│  └─ Infrastructure: 基础设施层（数据库、缓存）  │
│                                                 │
│  洋葱架构（Onion Architecture）                 │
│  ├─ 核心: Domain Models                         │
│  ├─ 应用层: Use Cases                           │
│  ├─ 基础设施层: 数据库、外部服务                │
│  └─ 表示层: API、Web UI                         │
│                                                 │
└────────────────────────────────────────────────┘
```

#### Controller层实现

```typescript
// controllers/user.controller.ts
import { Request, Response, NextFunction } from 'express';
import { UserService } from '../services/user.service';
import { CreateUserDto, UpdateUserDto } from '../dtos/user.dto';
import { ApiResponse } from '../utils/response';

export class UserController {
  constructor(private userService: UserService) {}

  /**
   * 💡 Controller职责：
   * 1. 接收HTTP请求
   * 2. 参数验证（通过DTO）
   * 3. 调用Service处理业务逻辑
   * 4. 格式化响应
   */
  
  // 创建用户
  async create(req: Request, res: Response, next: NextFunction) {
    try {
      const dto: CreateUserDto = req.body;
      const user = await this.userService.createUser(dto);
      
      res.status(201).json(
        ApiResponse.success(user, 'User created successfully')
      );
    } catch (error) {
      next(error);
    }
  }

  // 获取用户列表
  async findAll(req: Request, res: Response, next: NextFunction) {
    try {
      const { page = 1, limit = 10 } = req.query;
      
      const result = await this.userService.findAll({
        page: Number(page),
        limit: Number(limit)
      });
      
      res.json(ApiResponse.success(result));
    } catch (error) {
      next(error);
    }
  }

  // 获取单个用户
  async findOne(req: Request, res: Response, next: NextFunction) {
    try {
      const { id } = req.params;
      const user = await this.userService.findById(id);
      
      res.json(ApiResponse.success(user));
    } catch (error) {
      next(error);
    }
  }

  // 更新用户
  async update(req: Request, res: Response, next: NextFunction) {
    try {
      const { id } = req.params;
      const dto: UpdateUserDto = req.body;
      
      const user = await this.userService.updateUser(id, dto);
      
      res.json(ApiResponse.success(user, 'User updated successfully'));
    } catch (error) {
      next(error);
    }
  }

  // 删除用户
  async delete(req: Request, res: Response, next: NextFunction) {
    try {
      const { id } = req.params;
      await this.userService.deleteUser(id);
      
      res.json(ApiResponse.success(null, 'User deleted successfully'));
    } catch (error) {
      next(error);
    }
  }
}
```

#### Service层实现

```typescript
// services/user.service.ts
import { UserRepository } from '../repositories/user.repository';
import { CreateUserDto, UpdateUserDto } from '../dtos/user.dto';
import { User } from '../models/user.model';
import { NotFoundError, ConflictError } from '../utils/errors';
import bcrypt from 'bcrypt';
import { logger } from '../utils/logger';

export class UserService {
  constructor(
    private userRepository: UserRepository,
    private emailService: EmailService
  ) {}

  /**
   * 💡 Service职责：
   * 1. 实现业务逻辑
   * 2. 调用Repository进行数据操作
   * 3. 调用其他Service（如邮件服务）
   * 4. 事务管理
   */

  // 创建用户
  async createUser(dto: CreateUserDto): Promise<User> {
    // 检查邮箱是否已存在
    const existingUser = await this.userRepository.findByEmail(dto.email);
    if (existingUser) {
      throw new ConflictError('Email already exists');
    }

    // 密码加密
    const hashedPassword = await bcrypt.hash(dto.password, 10);

    // 创建用户
    const user = await this.userRepository.create({
      ...dto,
      password: hashedPassword,
      createdAt: new Date(),
      updatedAt: new Date()
    });

    // 发送欢迎邮件（异步，不阻塞）
    this.emailService.sendWelcomeEmail(user.email)
      .catch(err => logger.error('Failed to send welcome email', err));

    logger.info(`User created: ${user.id}`);

    // 返回时移除敏感信息
    return this.sanitizeUser(user);
  }

  // 查询所有用户（分页）
  async findAll(options: {
    page: number;
    limit: number;
  }): Promise<{ users: User[]; total: number }> {
    const skip = (options.page - 1) * options.limit;
    
    const [users, total] = await Promise.all([
      this.userRepository.findMany({ skip, limit: options.limit }),
      this.userRepository.count()
    ]);

    return {
      users: users.map(user => this.sanitizeUser(user)),
      total
    };
  }

  // 根据ID查询用户
  async findById(id: string): Promise<User> {
    const user = await this.userRepository.findById(id);
    
    if (!user) {
      throw new NotFoundError('User not found');
    }

    return this.sanitizeUser(user);
  }

  // 更新用户
  async updateUser(id: string, dto: UpdateUserDto): Promise<User> {
    const user = await this.userRepository.findById(id);
    
    if (!user) {
      throw new NotFoundError('User not found');
    }

    // 如果更新邮箱，检查是否冲突
    if (dto.email && dto.email !== user.email) {
      const existingUser = await this.userRepository.findByEmail(dto.email);
      if (existingUser) {
        throw new ConflictError('Email already exists');
      }
    }

    // 更新密码需要重新加密
    const updateData: Partial<User> = { ...dto };
    if (dto.password) {
      updateData.password = await bcrypt.hash(dto.password, 10);
    }

    const updatedUser = await this.userRepository.update(id, {
      ...updateData,
      updatedAt: new Date()
    });

    logger.info(`User updated: ${id}`);

    return this.sanitizeUser(updatedUser);
  }

  // 删除用户
  async deleteUser(id: string): Promise<void> {
    const user = await this.userRepository.findById(id);
    
    if (!user) {
      throw new NotFoundError('User not found');
    }

    await this.userRepository.delete(id);

    logger.info(`User deleted: ${id}`);
  }

  // 💡 工具方法：移除敏感信息
  private sanitizeUser(user: User): User {
    const { password, ...sanitized } = user;
    return sanitized as User;
  }
}
```

#### Repository层实现

```typescript
// repositories/user.repository.ts
import { Collection, Db, ObjectId } from 'mongodb';
import { User } from '../models/user.model';

export class UserRepository {
  private collection: Collection<User>;

  constructor(private db: Db) {
    this.collection = db.collection<User>('users');
  }

  /**
   * 💡 Repository职责：
   * 1. 数据库CRUD操作
   * 2. 查询构建
   * 3. 数据映射（数据库模型 <-> 领域模型）
   */

  // 创建用户
  async create(data: Omit<User, 'id'>): Promise<User> {
    const result = await this.collection.insertOne(data as any);
    
    return {
      id: result.insertedId.toString(),
      ...data
    };
  }

  // 根据ID查询
  async findById(id: string): Promise<User | null> {
    const result = await this.collection.findOne({
      _id: new ObjectId(id)
    });

    if (!result) return null;

    return this.mapToUser(result);
  }

  // 根据邮箱查询
  async findByEmail(email: string): Promise<User | null> {
    const result = await this.collection.findOne({ email });

    if (!result) return null;

    return this.mapToUser(result);
  }

  // 查询多个（分页）
  async findMany(options: {
    skip: number;
    limit: number;
    filter?: Partial<User>;
  }): Promise<User[]> {
    const results = await this.collection
      .find(options.filter || {})
      .skip(options.skip)
      .limit(options.limit)
      .toArray();

    return results.map(result => this.mapToUser(result));
  }

  // 统计数量
  async count(filter?: Partial<User>): Promise<number> {
    return this.collection.countDocuments(filter || {});
  }

  // 更新
  async update(id: string, data: Partial<User>): Promise<User> {
    const result = await this.collection.findOneAndUpdate(
      { _id: new ObjectId(id) },
      { $set: data },
      { returnDocument: 'after' }
    );

    if (!result) {
      throw new Error('User not found');
    }

    return this.mapToUser(result);
  }

  // 删除
  async delete(id: string): Promise<void> {
    await this.collection.deleteOne({ _id: new ObjectId(id) });
  }

  // 💡 数据映射：MongoDB文档 -> 领域模型
  private mapToUser(doc: any): User {
    return {
      id: doc._id.toString(),
      email: doc.email,
      password: doc.password,
      name: doc.name,
      role: doc.role,
      createdAt: doc.createdAt,
      updatedAt: doc.updatedAt
    };
  }
}
```

```typescript
// di/container.ts
import { Db } from 'mongodb';
import { UserController } from '../controllers/user.controller';
import { UserService } from '../services/user.service';
import { UserRepository } from '../repositories/user.repository';
import { EmailService } from '../services/email.service';

/**
 * 💡 依赖注入容器
 * 统一管理依赖关系
 */
export class Container {
  private db: Db;

  constructor(db: Db) {
    this.db = db;
  }

  // 创建UserController
  createUserController(): UserController {
    const userRepository = this.createUserRepository();
    const emailService = this.createEmailService();
    const userService = new UserService(userRepository, emailService);
    
    return new UserController(userService);
  }

  // 创建UserRepository
  private createUserRepository(): UserRepository {
    return new UserRepository(this.db);
  }

  // 创建EmailService
  private createEmailService(): EmailService {
    return new EmailService();
  }
}

// 使用
import { MongoClient } from 'mongodb';

const client = await MongoClient.connect(process.env.MONGODB_URI!);
const db = client.db();

const container = new Container(db);
const userController = container.createUserController();
```

## 🔧 错误处理

### 高频错误1：循环依赖

**现象：**
```typescript
// user.service.ts
import { ProductService } from './product.service';

export class UserService {
  constructor(private productService: ProductService) {}
}

// product.service.ts
import { UserService } from './user.service';

  constructor(private userService: UserService) {}

// Error: Circular dependency detected
```

```typescript
// 💡 方案1：提取公共Service
// common.service.ts
export class CommonService {
  // 共享逻辑
}

// user.service.ts
export class UserService {
  constructor(private commonService: CommonService) {}
}

// product.service.ts
export class ProductService {
  constructor(private commonService: CommonService) {}
}

// 💡 方案2：使用事件驱动
import { EventEmitter } from 'events';

export class UserService {
```

```typescript
// models/order.model.ts
export interface Order {
  id: string;
  userId: string;
  items: OrderItem[];
  totalAmount: number;
  status: 'pending' | 'paid' | 'shipped' | 'completed' | 'cancelled';
  createdAt: Date;
  updatedAt: Date;
}

// controllers/order.controller.ts
@Service()
export class OrderController {
  constructor(private orderService: OrderService) {}

  async create(req, res, next) {
    try {
      const userId = req.user.id;
      const dto: CreateOrderDto = req.body;
      
      const order = await this.orderService.createOrder(userId, dto);

    } catch (error) {

    }
  }
}

// services/order.service.ts
@Service()
export class OrderService {
  constructor(
    private orderRepository: OrderRepository,
    private productService: ProductService,
    private userService: UserService,
    private paymentService: PaymentService,
    private emailService: EmailService
  ) {}

  async createOrder(userId: string, dto: CreateOrderDto): Promise<Order> {
    // 1. 验证用户
    const user = await this.userService.findById(userId);
    
    // 2. 验证商品和库存
    const products = await this.productService.findByIds(
      dto.items.map(item => item.productId)
    );
    
    for (const item of dto.items) {
      const product = products.find(p => p.id === item.productId);
      if (!product) {
        throw new NotFoundError(`Product ${item.productId} not found`);
      }
      if (product.stock < item.quantity) {
        throw new BadRequestError(`Insufficient stock for ${product.name}`);
      }
    }
    
    // 3. 计算总金额
    const totalAmount = dto.items.reduce((sum, item) => {
      const product = products.find(p => p.id === item.productId)!;
      return sum + product.price * item.quantity;
    }, 0);
    
    // 4. 创建订单
    const order = await this.orderRepository.create({
      userId,
      items: dto.items,
      totalAmount,
      status: 'pending',
      createdAt: new Date(),
      updatedAt: new Date()
    });
    
    // 5. 扣减库存（事务）
    await this.productService.decreaseStock(
      dto.items.map(item => ({
        productId: item.productId,
        quantity: item.quantity
      }))
    );
    
    // 6. 发送订单确认邮件（异步）
    this.emailService.sendOrderConfirmation(user.email, order)
      .catch(err => logger.error('Failed to send order email', err));
    
    logger.info(`Order created: ${order.id}`);
    
    return order;
  }
}
```

## 📋 总结

### 核心要点

1. **分层架构**：Controller/Service/Repository
2. **职责分离**：每层只做自己的事
3. **依赖注入**：解耦依赖关系
4. **易于测试**：可以mock依赖

### 实现关键

1. **Controller**: 处理HTTP请求、参数验证、格式化响应
2. **Service**: 业务逻辑、事务管理、调用其他Service
3. **Repository**: 数据库操作、查询构建、数据映射
4. **DI容器**: 统一管理依赖关系

### 应用价值

- **可维护性**：代码结构清晰，易于理解和修改
- **可测试性**：各层独立测试，可以mock依赖
- **可扩展性**：新增功能不影响现有代码
- **可复用性**：Service可以被多个Controller使用

### 面试重点

**Q1：什么是分层架构？为什么要使用？**
答：将应用按职责划分为不同层次，每层只负责特定功能。优点：职责清晰、易于测试、易于维护、易于扩展。

**Q2：Controller和Service的区别？**
答：Controller处理HTTP请求、参数验证、格式化响应；Service实现业务逻辑、事务管理、调用其他Service。

**Q3：为什么需要Repository层？**
答：封装数据访问逻辑，Service不直接操作数据库。优点：数据库逻辑集中、易于测试、易于切换数据库。

**Q4：什么是依赖注入？**
答：通过构造函数或setter注入依赖，而非在类内部创建依赖。优点：解耦、易于测试、易于替换实现。

### 注意事项

⚠️ **避免循环依赖**：提取公共Service或使用事件驱动
⚠️ **职责单一**：每个类只做一件事
⚠️ **依赖倒置**：依赖抽象而非具体实现
⚠️ **测试驱动**：设计时考虑可测试性
