# 13.6 代码质量

> ESLint 9 Flat Config、Prettier 3、TypeScript严格模式、代码覆盖率、SonarQube

---

## 📖 知识点概述

### 核心概念

**代码质量工具**通过静态分析、格式化、类型检查等手段，保证代码的一致性、可读性和可维护性。

### 为什么需要代码质量工具？

```
❌ 没有代码质量工具：

// 团队成员A
function getUserData(id){
  const user=db.find(id)
  return user
}

// 团队成员B
function getProductData (productId) {
    const product = db.find( productId );
    return product;
}

问题：
- 代码风格不一致
- 潜在bug难以发现
- 代码审查耗时
- 团队协作困难

✅ 使用代码质量工具：

// ESLint + Prettier统一格式
function getUserData(id: string): User | null {
  const user = db.find(id);
  return user;
}

function getProductData(productId: string): Product | null {
  const product = db.find(productId);
  return product;
}

优点：
- 风格统一
- 自动发现问题
- 提高代码质量
- 减少code review负担
```

## 🚀 使用指南

### 实践1：ESLint 9配置（Flat Config）

```bash
# 安装ESLint 9
npm install --save-dev eslint@9

# 安装TypeScript插件
npm install --save-dev @typescript-eslint/parser @typescript-eslint/eslint-plugin

# 安装React插件（如果需要）
npm install --save-dev eslint-plugin-react eslint-plugin-react-hooks
```

```javascript
// eslint.config.js（ESLint 9 Flat Config）
import js from '@eslint/js';
import tseslint from 'typescript-eslint';
import react from 'eslint-plugin-react';
import reactHooks from 'eslint-plugin-react-hooks';

export default [
  // 💡 忽略文件
  {
    ignores: [
      'dist/**',
      'build/**',
      'node_modules/**',
      '*.config.js'
    ]
  },
  
  // 💡 JavaScript推荐规则
  js.configs.recommended,
  
  // 💡 TypeScript配置
  {
    files: ['**/*.ts', '**/*.tsx'],
    plugins: {
      '@typescript-eslint': tseslint.plugin
    },
    languageOptions: {
      parser: tseslint.parser,
      parserOptions: {
        project: './tsconfig.json'
      }
    },
    rules: {
      // TypeScript规则
      '@typescript-eslint/no-unused-vars': ['error', {
        argsIgnorePattern: '^_',
        varsIgnorePattern: '^_'
      }],
      '@typescript-eslint/no-explicit-any': 'warn',
      '@typescript-eslint/explicit-function-return-type': ['warn', {
        allowExpressions: true
      }],
      '@typescript-eslint/no-non-null-assertion': 'warn',
      
      // JavaScript规则
      'no-console': ['warn', { allow: ['warn', 'error'] }],
      'no-debugger': 'error',
      'prefer-const': 'error',
      'no-var': 'error',
      'eqeqeq': ['error', 'always']
    }
  },
  
  // 💡 React配置
  {
    files: ['**/*.tsx'],
    plugins: {
      react,
      'react-hooks': reactHooks
    },
    settings: {
      react: {
        version: 'detect'
      }
    },
    rules: {
      ...react.configs.recommended.rules,
      ...reactHooks.configs.recommended.rules,
      'react/react-in-jsx-scope': 'off', // React 17+不需要
      'react/prop-types': 'off' // 使用TypeScript类型
    }
  }
];

// package.json
{
  "scripts": {
    "lint": "eslint .",
    "lint:fix": "eslint . --fix"
  }
}
```

### 实践3：TypeScript严格模式

```json
// tsconfig.json
{
  "compilerOptions": {
    // 💡 严格模式（启用所有严格检查）
    "strict": true,
    
    // 或单独配置
    "noImplicitAny": true,              // 禁止隐式any
    "strictNullChecks": true,           // 严格空值检查
    "strictFunctionTypes": true,        // 严格函数类型
    "strictBindCallApply": true,        // 严格bind/call/apply
    "strictPropertyInitialization": true, // 严格属性初始化
    "noImplicitThis": true,             // 禁止隐式this
    "alwaysStrict": true,               // 始终使用严格模式
    
    // 💡 额外检查
    "noUnusedLocals": true,             // 禁止未使用的局部变量
    "noUnusedParameters": true,         // 禁止未使用的参数
    "noImplicitReturns": true,          // 禁止隐式返回
    "noFallthroughCasesInSwitch": true, // 禁止switch穿透
    "noUncheckedIndexedAccess": true,   // 索引访问检查
    "noImplicitOverride": true,         // 覆盖方法必须使用override
    
    // 基础配置
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "esModuleInterop": true,
    "skipLibCheck": true,
    "resolveJsonModule": true
  },
  
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
```

## 🔧 错误处理

### 高频错误1：ESLint配置冲突

**问题：**
```
Parsing error: Cannot read file 'tsconfig.json'
```

**解决方案：**
```javascript
// eslint.config.js
export default [
  {
    files: ['**/*.ts'],
    languageOptions: {
      parser: tseslint.parser,
      parserOptions: {
        // 💡 确保路径正确
        project: './tsconfig.json',
        tsconfigRootDir: import.meta.dirname
      }
    }
  }
];

// 或关闭需要类型信息的规则
export default [
  {
    files: ['**/*.ts'],
    rules: {
      '@typescript-eslint/no-floating-promises': 'off'
    }
  }
];
```

### 高频错误2：Prettier和ESLint冲突

**问题：**
```
Delete `␍` prettier/prettier
```

**解决方案：**
```json
// .prettierrc
{
  // 💡 统一换行符
  "endOfLine": "lf"
}

// .editorconfig
root = true

[*]
end_of_line = lf
insert_final_newline = true
charset = utf-8

// Git配置
// .gitattributes
* text=auto eol=lf
```

### 高频错误3：TypeScript类型错误过多

**问题：**
```
Type 'string | undefined' is not assignable to type 'string'
```

**解决方案：**
```typescript
// ❌ 错误
function getUserName(user: User): string {
  return user.name; // user.name可能是undefined
}

// ✅ 正确：使用可选链和空值合并
function getUserName(user: User): string {
  return user.name ?? 'Anonymous';
}

// ✅ 正确：类型守卫
function getUserName(user: User): string {
  if (!user.name) {
    throw new Error('User name is required');
  }
  return user.name;
}

// ✅ 正确：使用非空断言（确定不为空时）
function getUserName(user: User): string {
  return user.name!; // 确定name存在
}
```

## 📚 应用场景

### 场景：CI/CD集成

```yaml
# .github/workflows/quality-check.yml
name: Quality Check

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  quality:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '20'
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      # 💡 TypeScript类型检查
      - name: Type check
        run: npm run type-check
      
      # 💡 ESLint检查
      - name: Lint
        run: npm run lint
      
      # 💡 Prettier检查
      - name: Format check
        run: npm run format:check
      
      # 💡 运行测试
      - name: Test
        run: npm run test:coverage
      
      # 💡 上传覆盖率报告
      - name: Upload coverage
        uses: codecov/codecov-action@v3
        with:
          files: ./coverage/lcov.info
      
      # 💡 检查覆盖率阈值
      - name: Check coverage threshold
        run: |
          npm run test:coverage
          if [ $? -ne 0 ]; then
            echo "Coverage threshold not met"
            exit 1
          fi
```

## 📋 总结

### 核心要点

1. **ESLint 9**：Flat Config新配置方式
2. **Prettier 3**：代码格式化
3. **TypeScript严格模式**：类型安全
4. **代码覆盖率**：c8测试覆盖率

### 实现关键

1. **eslint.config.js**: ESLint 9配置
2. **.prettierrc**: Prettier格式配置
3. **tsconfig.json**: TypeScript严格模式
4. **c8**: 代码覆盖率工具

### 应用价值

- **一致性**：统一代码风格
- **质量保证**：自动发现潜在问题
- **可维护性**：提高代码可读性
- **团队协作**：减少code review负担

### 面试重点

**Q1：ESLint和Prettier的区别？**
答：ESLint检查代码质量（逻辑错误、最佳实践），Prettier格式化代码（缩进、换行）。两者互补，一起使用。

**Q2：TypeScript严格模式的作用？**
答：启用所有严格类型检查（noImplicitAny、strictNullChecks等），提高类型安全，减少运行时错误。

**Q3：如何提高代码覆盖率？**
答：1) 编写单元测试 2) 使用c8统计覆盖率 3) 设置覆盖率阈值 4) CI/CD中检查覆盖率。

**Q4：ESLint 9 Flat Config的优势？**
答：1) 配置更简洁 2) 支持ESM 3) 更好的TypeScript支持 4) 性能提升。

### 注意事项

⚠️ **不要禁用过多规则**：保持代码质量
⚠️ **团队统一配置**：使用相同的配置文件
⚠️ **CI/CD集成**：自动化检查
⚠️ **覆盖率不是越高越好**：关注有价值的测试
