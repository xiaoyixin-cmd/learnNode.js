## 📖 知识点概述

### 核心概念

**数据库测试**是确保应用与数据库交互正确的关键环节。核心挑战是如何在保证测试速度的同时实现数据隔离和真实环境模拟。

### 数据库测试策略

```
┌──────────────────────────────────────────────────────────┐
│              数据库测试策略对比                            │
├──────────────────────────────────────────────────────────┤
│  1. Mock数据库                                            │
│     ├─ 速度：极快 ⚡⚡⚡                                    │
│     ├─ 真实性：低 ❌                                      │
│     ├─ 隔离性：完美 ✅                                    │
│     └─ 适用：单元测试                                     │
│                                                            │
│  2. 内存数据库（SQLite、MongoDB Memory）                   │
│     ├─ 速度：很快 ⚡⚡                                     │
│     ├─ 真实性：中 ⚠️                                      │
│     ├─ 隔离性：好 ✅                                      │
│     └─ 适用：集成测试（快速反馈）                          │
│                                                            │
│  3. 测试数据库（专用实例）                                 │
│     ├─ 速度：中 ⚡                                        │
│     ├─ 真实性：高 ✅                                      │
│     ├─ 隔离性：需手动清理 ⚠️                              │
│     └─ 适用：集成测试（完整功能）                          │
│                                                            │
│  4. Docker容器（docker-compose）⭐推荐                    │
│     ├─ 速度：中 ⚡                                        │
│     ├─ 真实性：完美 ✅                                    │
│     ├─ 隔离性：完美 ✅                                    │
│     └─ 适用：CI/CD + 本地开发                             │
└──────────────────────────────────────────────────────────┘
```

### 数据隔离方案

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **每次清空数据库** | beforeEach删除所有数据 | 完全隔离 | 慢 |
| **事务回滚** | 每个测试在事务中运行 | 快速 | 不支持NoSQL |
| **测试数据库** | 专用测试DB | 真实 | 需手动清理 |
| **内存数据库** | 使用内存DB | 极快 | 功能受限 |

```javascript
// db.js
import { Sequelize } from 'sequelize';

export function createDatabase(isTest = false) {
  if (isTest) {
    // 💡 测试环境：使用内存SQLite
    return new Sequelize('sqlite::memory:', {
      logging: false
    });
  } else {
    // 生产环境：PostgreSQL/MySQL
    return new Sequelize(process.env.DATABASE_URL);
  }
}

// models/User.js
export function defineUserModel(sequelize) {
  return sequelize.define('User', {
    id: {
      type: DataTypes.INTEGER,
      primaryKey: true,
      autoIncrement: true
    },
    username: {
      type: DataTypes.STRING,
      allowNull: false,
      unique: true
    },
    email: {
      type: DataTypes.STRING,
      allowNull: false,
      validate: {
        isEmail: true
      }
    },
    age: {
      type: DataTypes.INTEGER,
      validate: {
        min: 0,
        max: 150
      }
    }
  });
}

// user.test.js
import { createDatabase } from './db.js';
import { defineUserModel } from './models/User.js';

describe('User Model', () => {
  let sequelize;
  let User;

    sequelize = createDatabase(true);
    User = defineUserModel(sequelize);
    await sequelize.sync();
  });
  
  // 💡 每个测试后：清空数据
  afterEach(async () => {
    await User.destroy({ where: {}, truncate: true });
  });
  
  // 💡 所有测试后：关闭连接
  afterAll(async () => {
    await sequelize.close();
  });
  
  it('创建用户', async () => {
    const user = await User.create({
      username: 'john',

    });
```

```javascript
import { MongoMemoryServer } from 'mongodb-memory-server';
import mongoose from 'mongoose';

describe('MongoDB测试', () => {
  let mongoServer;
  
  // 💡 启动内存MongoDB
  beforeAll(async () => {
    mongoServer = await MongoMemoryServer.create();
    const uri = mongoServer.getUri();
    
    await mongoose.connect(uri);
  });
  
  // 💡 清理数据库
  afterEach(async () => {
    const collections = await mongoose.connection.db.collections();

      await collection.deleteMany({});
    }
  });
  
  // 💡 关闭连接和服务器
  afterAll(async () => {
    await mongoose.disconnect();
    await mongoServer.stop();
  });
  
  // 定义Schema
  const UserSchema = new mongoose.Schema({
    username: { type: String, required: true, unique: true },
    email: { type: String, required: true },
    age: { type: Number, min: 0 }
  });
  
  const User = mongoose.model('User', UserSchema);
  
  it('创建用户', async () => {
    const user = await User.create({
      username: 'john',
      email: 'john@example.com',
      age: 25
    });
    
    expect(user._id).toBeDefined();
    expect(user.username).toBe('john');
  });
  
  it('查询用户', async () => {
    await User.create({
      username: 'alice',
      email: 'alice@example.com',
      age: 30
    });
    
    const user = await User.findOne({ username: 'alice' });
    
    expect(user).not.toBeNull();
    expect(user.age).toBe(30);
  });
  
  it('更新用户', async () => {
    const user = await User.create({
      username: 'bob',
      email: 'bob@example.com',

    });

      { _id: user._id },
```

### 策略4：事务回滚测试

```javascript
// transaction.test.js
import { Sequelize } from 'sequelize';

describe('事务测试', () => {
  let sequelize;
  let User;
  let Account;
  
  beforeAll(async () => {
    sequelize = new Sequelize('sqlite::memory:', { logging: false });
    
    // 定义模型
    User = sequelize.define('User', {
      username: DataTypes.STRING
    });
    
    Account = sequelize.define('Account', {
      balance: DataTypes.DECIMAL(10, 2)
    });
    
    User.hasOne(Account);
    Account.belongsTo(User);
    
    await sequelize.sync();
  });
  
  afterAll(async () => {
    await sequelize.close();
  });
  
  // 💡 每个测试在事务中运行
  let transaction;
  
  beforeEach(async () => {
    transaction = await sequelize.transaction();
  });
  
  afterEach(async () => {
    // 💡 回滚事务（自动清理数据）
    await transaction.rollback();
  });
  
  it('创建用户和账户', async () => {
    const user = await User.create(
      { username: 'john' },
      { transaction }
    );
    
    await Account.create(
      { balance: 1000, UserId: user.id },
      { transaction }
    );
    
    const account = await Account.findOne({
      where: { UserId: user.id },
      transaction
    });
    
    expect(account.balance).toBe('1000.00');
  });
  
  it('转账操作', async () => {
    // 创建两个账户
    const user1 = await User.create(
      { username: 'alice' },
      { transaction }
    );
    const account1 = await Account.create(
      { balance: 1000, UserId: user1.id },
      { transaction }
    );
    
    const user2 = await User.create(
      { username: 'bob' },
      { transaction }
    );
    const account2 = await Account.create(
      { balance: 500, UserId: user2.id },
      { transaction }
    );
    
    // 💡 转账
    const transferAmount = 200;
    
    await account1.update(
      { balance: account1.balance - transferAmount },
      { transaction }
    );
    
    await account2.update(
      { balance: account2.balance + transferAmount },
      { transaction }
    );
    
    // 验证余额
    await account1.reload({ transaction });
    await account2.reload({ transaction });
    
    expect(parseFloat(account1.balance)).toBe(800);
    expect(parseFloat(account2.balance)).toBe(700);
  });
  
  it('余额不足时回滚', async () => {
    const user = await User.create(
      { username: 'charlie' },
      { transaction }
    );
    const account = await Account.create(
      { balance: 100, UserId: user.id },
      { transaction }
    );
    
    // 💡 尝试提取超额
    const withdrawAmount = 200;
    
    await expect(async () => {
      if (account.balance < withdrawAmount) {
        throw new Error('Insufficient funds');
      }
      
      await account.update(
        { balance: account.balance - withdrawAmount },
        { transaction }
      );
    }).rejects.toThrow('Insufficient funds');
    
    // 余额应保持不变
    await account.reload({ transaction });
    expect(parseFloat(account.balance)).toBe(100);
  });
});
```

### 策略5：测试数据工厂

```javascript
// factories/user.factory.js
import { faker } from '@faker-js/faker';

export class UserFactory {
  // 💡 生成单个用户数据
  static build(overrides = {}) {
    return {
      username: faker.internet.userName(),
      email: faker.internet.email(),
      age: faker.number.int({ min: 18, max: 80 }),
      firstName: faker.person.firstName(),
      lastName: faker.person.lastName(),
      ...overrides
    };
  }
  
  // 💡 创建用户到数据库
  static async create(User, overrides = {}) {
    const userData = this.build(overrides);
    return await User.create(userData);
  }
  
  // 💡 批量创建
  static async createMany(User, count, overrides = {}) {
    const users = Array.from({ length: count }, () =>
      this.build(overrides)
    );
    
    return await User.bulkCreate(users);
  }
  
  // 💡 创建特定状态的用户
  static async createActive(User) {
    return await this.create(User, {
      status: 'active',
      emailVerified: true
    });
  }
  
  static async createInactive(User) {
    return await this.create(User, {
      status: 'inactive',
      emailVerified: false
    });
  }
}

// user.test.js
import { UserFactory } from './factories/user.factory.js';

describe('使用工厂创建测试数据', () => {
  it('创建单个用户', async () => {
    const user = await UserFactory.create(User);
    
    expect(user.username).toBeDefined();
    expect(user.email).toMatch(/@/);
  });
  
  it('创建特定数据的用户', async () => {
    const user = await UserFactory.create(User, {
      username: 'specific-user',
      age: 25
    });
    
    expect(user.username).toBe('specific-user');
    expect(user.age).toBe(25);
  });
  
  it('批量创建用户', async () => {
    const users = await UserFactory.createMany(User, 10);
    
    expect(users).toHaveLength(10);
  });
  
  it('创建活跃用户', async () => {
    const user = await UserFactory.createActive(User);
    
    expect(user.status).toBe('active');
    expect(user.emailVerified).toBe(true);
  });
});
```

## 🔧 错误处理

### 高频错误1：数据未清理（测试相互影响）

**现象：**
```javascript
it('test 1', async () => {
  await User.create({ username: 'john' });
  const count = await User.count();
  expect(count).toBe(1);  // ✅ 通过
});

it('test 2', async () => {
  const count = await User.count();
  expect(count).toBe(0);  // ❌ 失败：count = 1
});
```

**解决方案：**
```javascript
// ✅ 方案1：每个测试后清空
afterEach(async () => {
  await User.destroy({ where: {}, truncate: true });
});

// ✅ 方案2：使用事务回滚
beforeEach(async () => {
  transaction = await sequelize.transaction();
});

afterEach(async () => {
  await transaction.rollback();
});

// ✅ 方案3：每个测试前重建数据库
beforeEach(async () => {
  await sequelize.sync({ force: true });
});
```

### 高频错误2：数据库连接未关闭

**现象：**
```
Jest did not exit one second after the test run has completed.
```

**原因：**
数据库连接未关闭，导致进程挂起

**解决方案：**
```javascript
// ✅ 必须关闭连接
afterAll(async () => {
  await sequelize.close();        // Sequelize
  await mongoose.disconnect();    // Mongoose
  await mongoServer.stop();       // MongoDB Memory Server
});

// 💡 或使用--forceExit（不推荐）
// jest --forceExit
```

### 高频错误3：异步测试超时

**现象：**
```
Timeout - Async callback was not invoked within the 5000 ms timeout
```

**原因：**
数据库操作耗时较长

**解决方案：**
```javascript
// ✅ 增加超时时间
jest.setTimeout(10000);  // 10秒

// 或针对单个测试
it('slow test', async () => {
  // ...
}, 10000);

// 💡 优化数据库操作
beforeAll(async () => {
  // 只创建一次
  await sequelize.sync();
});

// 💡 使用内存数据库
const sequelize = new Sequelize('sqlite::memory:');
```

## 📋 总结

### 核心要点

1. **策略选择**：内存DB快速 vs Docker真实
2. **数据隔离**：事务回滚 vs 每次清空
3. **测试数据**：工厂模式 vs Faker
4. **集成测试**：API + DB完整验证

### 实现关键

1. **beforeAll/afterAll**：管理数据库生命周期
2. **beforeEach/afterEach**：数据隔离
3. **工厂模式**：快速生成测试数据
4. **Docker Compose**：真实环境测试

### 应用价值

- **发现问题**：真实数据库约束测试
- **重构信心**：安全修改数据层
- **文档作用**：测试即数据模型文档
- **CI/CD**：自动化数据库测试

### 面试重点

**Q1：数据库测试为什么不用Mock？**
答：Mock无法测试真实的SQL语法、数据库约束、事务、索引性能等。集成测试需要真实或接近真实的数据库。

**Q2：如何保证测试数据隔离？**
答：1) 事务回滚（快） 2) 每次清空数据（慢但彻底） 3) 每个测试独立数据库（完全隔离） 4) Docker容器（CI/CD推荐）。

**Q3：生产数据库和测试数据库的差异如何处理？**
答：1) 使用相同数据库类型 2) Docker确保版本一致 3) 使用Migration同步schema 4) 集成测试覆盖关键查询。

### 注意事项

⚠️ **连接关闭**：afterAll中必须关闭连接
⚠️ **数据清理**：afterEach清空数据
⚠️ **超时配置**：数据库操作增加超时时间
⚠️ **并发测试**：避免多测试同时写同一数据
