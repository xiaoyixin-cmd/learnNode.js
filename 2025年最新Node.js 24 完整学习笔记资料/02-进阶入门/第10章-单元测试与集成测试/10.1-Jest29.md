## 📖 知识点概述

### 核心概念

**Jest**是Facebook开源的JavaScript测试框架，具有零配置、快照测试、并行测试、强大的Mock功能等特点，是Node.js和React生态中最流行的测试工具。

### 测试文件结构

```javascript
// 典型的Jest测试文件
describe('测试套件名称', () => {
  // 在所有测试前运行一次
  beforeAll(() => {
    // 初始化数据库连接
  });
  
  // 在每个测试前运行
  beforeEach(() => {
    // 重置数据
  });
  
  // 测试用例
  it('测试用例描述', () => {
    // 断言
    expect(actual).toBe(expected);
  });
  
  // 另一个测试用例
  test('另一个测试', () => {
    // test 和 it 是别名
  });
  
  // 在每个测试后运行
  afterEach(() => {
    // 清理数据
  });
  
  // 在所有测试后运行一次
  afterAll(() => {
    // 关闭数据库连接
  });
});
```

## 🚀 使用指南

### 理解：为什么需要测试？

#### 无测试的问题

```javascript
// ❌ 没有测试：修改代码时心惊胆战
function calculateDiscount(price, discountPercent) {
  return price - (price * discountPercent / 100);
}

// 修改代码
function calculateDiscount(price, discountPercent) {
  // 新增：折扣不能超过50%
  const maxDiscount = 50;
  const finalDiscount = Math.min(discountPercent, maxDiscount);
  return price - (price * finalDiscount / 100);
}

// 问题：
// 1. 不知道修改是否破坏了原有功能
// 2. 需要手动测试所有场景
// 3. 容易遗漏边缘情况
```

#### 有测试的好处

```javascript
// ✅ 有测试：放心重构
describe('calculateDiscount', () => {
  it('正确计算折扣', () => {
    expect(calculateDiscount(100, 10)).toBe(90);
    expect(calculateDiscount(100, 20)).toBe(80);
  });
  
  it('折扣不超过50%', () => {
    expect(calculateDiscount(100, 70)).toBe(50); // 最多打5折
  });
  
  it('处理边缘情况', () => {
    expect(calculateDiscount(100, 0)).toBe(100);
    expect(calculateDiscount(0, 10)).toBe(0);
  });
});

// 运行测试：npm test
// 所有测试通过 → 修改没有破坏功能 ✅
```

    expect(value).toBeLessThan(5);
    expect(value).toBeLessThanOrEqual(4);

export async function fetchUser(id) {
  const response = await fetch(`https://api.example.com/users/${id}`);

export function fetchUserCallback(id, callback) {
  fetch(`https://api.example.com/users/${id}`)
    .then(res => res.json())
    .then(data => callback(null, data))
    .catch(err => callback(err));
}

// api.test.js
import { fetchUser, fetchUserCallback } from './api.js';

describe('异步测试', () => {
  // 💡 方式1：返回Promise
  it('使用Promise', () => {
    return fetchUser(1).then(user => {
      expect(user.name).toBe('John');
    });
  });
  
  // 💡 方式2：async/await（推荐）
  it('使用async/await', async () => {
    const user = await fetchUser(1);
    expect(user.name).toBe('John');
  });
  
  // 💡 方式3：回调函数（done）
  it('使用回调', (done) => {
    fetchUserCallback(1, (err, user) => {
      expect(err).toBeNull();
      expect(user.name).toBe('John');
      done(); // 必须调用done
    });
  });
  
  // 💡 测试异步错误
  it('处理异步错误', async () => {
    await expect(fetchUser(999)).rejects.toThrow('User not found');
  });
  
  // 💡 测试Promise resolve
  it('测试resolve', async () => {
    await expect(Promise.resolve('success')).resolves.toBe('success');

// scheduler.js
export function scheduleTask(callback, delay) {
  setTimeout(() => {
    callback();
  }, delay);
}

export function repeatTask(callback, interval) {
  return setInterval(() => {
    callback();
  }, interval);
}

// scheduler.test.js
import { scheduleTask, repeatTask } from './scheduler.js';

describe('定时器测试', () => {
  beforeEach(() => {
    // 💡 启用假定时器
    jest.useFakeTimers();
  });
  
  afterEach(() => {
    // 💡 恢复真实定时器
    jest.useRealTimers();
  });
  
  it('延迟执行', () => {
    const callback = jest.fn();
    
    scheduleTask(callback, 1000);
    
    // 立即检查：还没执行
    expect(callback).not.toHaveBeenCalled();
    
    // 💡 快进1秒
    jest.advanceTimersByTime(1000);
    
    // 现在应该执行了
    expect(callback).toHaveBeenCalledTimes(1);
  });
  
  it('重复执行', () => {
    const callback = jest.fn();
    
    const intervalId = repeatTask(callback, 1000);
    
    // 💡 快进3秒
    jest.advanceTimersByTime(3000);
    
    // 应该执行了3次
    expect(callback).toHaveBeenCalledTimes(3);
    
    clearInterval(intervalId);
  });
  
  it('运行所有定时器', () => {
    const callback = jest.fn();
    
    scheduleTask(callback, 1000);
    scheduleTask(callback, 2000);
    scheduleTask(callback, 3000);
    
    // 💡 运行所有定时器
    jest.runAllTimers();
    
    expect(callback).toHaveBeenCalledTimes(3);
  });
  
  it('只运行待处理的定时器', () => {
    const callback = jest.fn();
    
    scheduleTask(() => {
      callback();
      // 再设置一个定时器
      scheduleTask(callback, 1000);
    }, 1000);
    
    // 💡 只运行当前待处理的定时器
    jest.runOnlyPendingTimers();
    
    // 第一个定时器执行了，但第二个没有
    expect(callback).toHaveBeenCalledTimes(1);
  });

## 🔧 错误处理

### 高频错误1：异步测试超时

**现象：**
```
Timeout - Async callback was not invoked within the 5000 ms timeout
```

**原因：**
异步测试没有正确返回Promise或调用done

**解决方案：**

```javascript
// ❌ 错误：没有返回Promise
it('错误的异步测试', () => {
  fetchUser(1).then(user => {
    expect(user.name).toBe('John');
  });
  // 测试立即结束，断言还没执行
});

// ✅ 正确：返回Promise
it('正确的异步测试', () => {
  return fetchUser(1).then(user => {
    expect(user.name).toBe('John');
  });
});

// ✅ 更好：使用async/await
it('使用async/await', async () => {
  const user = await fetchUser(1);
  expect(user.name).toBe('John');
});

// ✅ 回调方式：调用done
it('使用done', (done) => {
  fetchUserCallback(1, (err, user) => {
    expect(user.name).toBe('John');
    done(); // ❗ 必须调用
  });
});

// ✅ 增加超时时间（慢速测试）
it('慢速测试', async () => {
  const result = await slowOperation();
  expect(result).toBe('done');
}, 10000); // 10秒超时
```

### 高频错误2：Mock没有重置

**现象：**
测试互相影响，单独运行通过，一起运行失败

**原因：**
Mock状态在测试间共享

**解决方案：**

```javascript
// ❌ 问题：Mock状态累积
describe('测试套件', () => {
  const mockFn = jest.fn();
  
  it('测试1', () => {
    mockFn();
    expect(mockFn).toHaveBeenCalledTimes(1);
  });
  
  it('测试2', () => {
    mockFn();
    // ❌ 失败：期望1次，实际2次（包含测试1的调用）
    expect(mockFn).toHaveBeenCalledTimes(1);
  });
});

// ✅ 正确：每个测试前重置
describe('测试套件', () => {
  const mockFn = jest.fn();
  
  beforeEach(() => {
    jest.clearAllMocks(); // 清除调用记录
  });
  
  it('测试1', () => {
    mockFn();
    expect(mockFn).toHaveBeenCalledTimes(1);
  });
  
  it('测试2', () => {
    mockFn();
    expect(mockFn).toHaveBeenCalledTimes(1); // ✅ 通过
  });
});

// 💡 清除Mock的三种方法
beforeEach(() => {
  jest.clearAllMocks();   // 清除调用记录和返回值
  jest.resetAllMocks();   // 清除 + 重置实现
  jest.restoreAllMocks(); // 恢复原始实现（用于spyOn）
});
```

## 📋 总结

### 核心要点

1. **测试结构**：describe + it + beforeEach + afterEach
2. **断言**：toBe（原始类型）、toEqual（对象/数组）
3. **Mock**：jest.fn()、jest.mock()、jest.spyOn()
4. **异步**：async/await、返回Promise、done回调

### 实现关键

1. **命名清晰**：测试描述要明确测试内容
2. **独立性**：每个测试独立，不依赖其他测试
3. **重置Mock**：beforeEach中清除Mock状态
4. **覆盖率**：目标80%+，关键路径100%

### 应用价值

- **重构信心**：修改代码时有测试保护
- **文档作用**：测试即文档，展示使用方式
- **提前发现Bug**：在开发阶段发现问题
- **团队协作**：测试作为契约

### 面试重点

**Q1：toBe和toEqual的区别？**
答：toBe用于原始类型和引用比较（===），toEqual用于对象/数组的深度比较（值相等）。

**Q2：如何测试异步函数？**
答：三种方式：1) 返回Promise 2) async/await（推荐） 3) 使用done回调。

**Q3：Mock和Spy的区别？**
答：Mock完全替换函数，Spy监视真实函数的调用并可选择性mock返回值。

**Q4：如何提高测试覆盖率？**
答：1) 测试边缘情况 2) 测试错误处理 3) 测试所有分支 4) 使用覆盖率报告找盲点。

### 注意事项

⚠️ **异步测试**：必须返回Promise或调用done
⚠️ **Mock重置**：每个测试前清除Mock状态
⚠️ **快照审查**：不要盲目更新快照
⚠️ **测试隔离**：测试间不应互相影响
