## 📖 知识点概述

### 性能对比

| 维度 | Jest 29 | Vitest 2 | 提升 |
|------|---------|----------|------|
| **冷启动** | 8-12秒 | 0.5-1秒 | **10倍+** |
| **热重载** | 3-5秒 | 0.1-0.3秒 | **15倍+** |
| **Watch模式** | 全量重跑 | 智能增量 | **智能** |
| **内存占用** | 200-400MB | 50-100MB | **4倍** |
| **ESM支持** | 实验性 | 原生 | **原生** |

### 为什么更快？

```
Jest架构：
源码 → Babel转译 → Jest → 虚拟文件系统 → Node.js
      ⏱️ 慢              ⏱️ 慢

Vitest架构：
源码 → Vite（esbuild） → Vitest → 真实import → Node.js
      ⚡ 快（100倍）        ⚡ 原生ESM
```

## 🚀 使用指南

### 理解：为什么选择Vitest？

#### Jest的痛点

```javascript
// ❌ Jest需要转译ESM
// package.json
{
  "type": "module", // ESM项目
  "scripts": {
    "test": "node --experimental-vm-modules node_modules/jest/bin/jest.js"
  }
}

// 问题：
// 1. 需要experimental flag
// 2. 速度慢（Babel转译）
// 3. 配置复杂（transform、moduleNameMapper）
// 4. 某些ESM特性不支持
```

#### Vitest的优势

```javascript
// ✅ Vitest原生支持ESM
// package.json
{
  "type": "module",
  "scripts": {
    "test": "vitest"
  }
}

// 优点：
// 1. 零配置（继承Vite配置）
// 2. 原生ESM（无需转译）
// 3. 速度快（10倍+）
// 4. 完整ESM特性支持
```

### 准备：安装Vitest

```bash
# 安装Vitest
npm install -D vitest

# 安装UI模式（可选）
npm install -D @vitest/ui

# package.json添加脚本
{
  "scripts": {
    "test": "vitest",
    "test:ui": "vitest --ui",
    "test:run": "vitest run",
    "test:coverage": "vitest --coverage"
  }
}
```

#### 第四步：Spy功能

```javascript
// logger.js
export const logger = {
  info: (message) => console.log(`[INFO] ${message}`),
  error: (message) => console.error(`[ERROR] ${message}`)
};

// service.js
import { logger } from './logger.js';

export function processData(data) {
  logger.info('Processing data...');
  
  try {
    const result = data.map(x => x * 2);
    logger.info('Processing complete');
    return result;
  } catch (error) {
    logger.error('Processing failed');
    throw error;
  }
}

// service.test.js
import { describe, it, expect, vi } from 'vitest';
import { logger } from './logger.js';
import { processData } from './service.js';

describe('processData', () => {
  it('logs processing steps', () => {
    // 💡 Spy监视
    const infoSpy = vi.spyOn(logger, 'info');
    
    processData([1, 2, 3]);
    
    expect(infoSpy).toHaveBeenCalledTimes(2);
    expect(infoSpy).toHaveBeenNthCalledWith(1, 'Processing data...');
    expect(infoSpy).toHaveBeenNthCalledWith(2, 'Processing complete');
    
    // 恢复
    infoSpy.mockRestore();
  });
  
  it('logs errors', () => {
    const errorSpy = vi.spyOn(logger, 'error');
    
    expect(() => processData(null)).toThrow();
    expect(errorSpy).toHaveBeenCalledWith('Processing failed');
    
    errorSpy.mockRestore();
  });
});
```

// scheduler.js
export function delayedGreeting(name, delay) {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve(`Hello, ${name}!`);
    }, delay);
  });
}

// scheduler.test.js
import { describe, it, expect, vi } from 'vitest';
import { delayedGreeting } from './scheduler.js';

describe('delayedGreeting', () => {
  it('greets after delay', async () => {
    // 💡 启用假定时器
    vi.useFakeTimers();
    
    const promise = delayedGreeting('John', 1000);
    
    // 快进1秒
    await vi.advanceTimersByTimeAsync(1000);
    
    const result = await promise;
    expect(result).toBe('Hello, John!');
    
    // 恢复真实定时器
    vi.useRealTimers();
  });
  
  it('使用runAllTimersAsync', async () => {
    vi.useFakeTimers();
    
    const promise = delayedGreeting('Alice', 5000);
    
    // 💡 运行所有定时器
    await vi.runAllTimersAsync();
    
    const result = await promise;
    expect(result).toBe('Hello, Alice!');
    
    vi.useRealTimers();
  });
});

## 🔧 错误处理

### 高频错误1：ESM导入问题

**现象：**
```
Cannot use import statement outside a module
```

**原因：**
package.json未设置`"type": "module"`

**解决方案：**

```json
// ✅ package.json
{
  "type": "module",
  "scripts": {
    "test": "vitest"
  }
}

// 或者使用.mjs扩展名
// sum.test.mjs
```

### 高频错误2：全局API未定义

**现象：**
```
ReferenceError: describe is not defined
```

**原因：**
未启用globals或未导入

**解决方案：**

```javascript
// ✅ 方案1：启用globals
// vitest.config.js
export default {
  test: {
    globals: true
  }
};

// 无需import
describe('test', () => {
  it('works', () => {
    expect(1).toBe(1);
  });
});

// ✅ 方案2：手动导入
import { describe, it, expect } from 'vitest';

describe('test', () => {
  it('works', () => {
    expect(1).toBe(1);
  });
});
```

### 高频错误3：覆盖率工具未安装

**现象：**
```
Coverage provider 'v8' is not found
```

**原因：**
未安装覆盖率工具

**解决方案：**

```bash
# 安装v8覆盖率工具（推荐）
npm install -D @vitest/coverage-v8

# 或使用istanbul
npm install -D @vitest/coverage-istanbul

# vitest.config.js
export default {
  test: {
    coverage: {
      provider: 'v8' // 或 'istanbul'
    }
  }
};

# 运行覆盖率测试
npm test -- --coverage
```

## 📋 总结

### 核心要点

1. **性能**：比Jest快10倍+，HMR支持
2. **ESM**：原生支持，无需转译
3. **Vite集成**：共享配置，统一体验
4. **Jest兼容**：API兼容，无缝迁移

### 实现关键

1. **配置简单**：继承Vite配置，零配置
2. **UI模式**：可视化测试，提升体验
3. **并发测试**：加快测试速度
4. **类型测试**：TypeScript完美支持

### 应用价值

- **开发体验**：HMR + Watch模式，实时反馈
- **团队效率**：测试速度快，等待时间少
- **现代化**：ESM原生支持，面向未来
- **可视化**：UI模式，直观查看测试结果

### 面试重点

**Q1：Vitest为什么比Jest快？**
答：1) Vite基于esbuild，转译速度快100倍 2) 原生ESM，无需转译 3) HMR支持，增量更新 4) 智能监听，只运行相关测试。

**Q2：Vitest与Jest的API兼容性如何？**
答：高度兼容，describe/it/expect/mock等API完全相同，可无缝迁移。区别在于Vitest原生支持ESM和TypeScript。

**Q3：什么时候选择Vitest而不是Jest？**
答：1) 使用Vite构建工具 2) ESM项目 3) 追求极致速度 4) TypeScript项目。Jest适合传统项目或React生态。

**Q4：Vitest的UI模式有什么用？**
答：可视化测试结果、实时运行状态、覆盖率热图、模块依赖图，提升测试体验。

### 注意事项

⚠️ **package.json**：必须设置`"type": "module"`
⚠️ **globals**：需要配置或手动导入API
⚠️ **覆盖率工具**：需要安装@vitest/coverage-v8
⚠️ **浏览器模式**：需要安装浏览器驱动
