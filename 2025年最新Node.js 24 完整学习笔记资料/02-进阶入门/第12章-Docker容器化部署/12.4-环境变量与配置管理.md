# 12.4 环境变量与配置管理

> 环境变量传递、.env文件、敏感信息管理、多环境配置、12-Factor App原则

---

## 📖 知识点概述

### 核心概念

**环境变量**是容器化应用配置的核心方式，符合12-Factor App原则。通过环境变量，可以在不修改代码的情况下，为不同环境（开发、测试、生产）提供不同配置。

### 为什么使用环境变量？

```
❌ 传统方式：硬编码配置

// config.js
module.exports = {
  database: {
    host: 'localhost',     ← 硬编码
    port: 5432,
    user: 'admin',
    password: 'secret123'  ← 敏感信息泄露风险
  },
  jwt: {
    secret: 'my-secret-key'
  }
};

问题：
- 配置固定，无法适应不同环境
- 敏感信息提交到代码仓库
- 修改配置需要重新构建镜像
- 不符合安全最佳实践

✅ 环境变量方式：

// config.js
module.exports = {
  database: {
    host: process.env.DB_HOST,
    port: process.env.DB_PORT,
    user: process.env.DB_USER,
    password: process.env.DB_PASSWORD
  },
  jwt: {
    secret: process.env.JWT_SECRET
  }
};

优点：
- 同一镜像适用于所有环境
- 敏感信息不进入代码仓库
- 配置与代码分离
- 符合12-Factor App原则
```

### 配置管理层级

```
┌────────────────────────────────────────────┐
│        配置管理优先级（从高到低）          │
├────────────────────────────────────────────┤
│                                             │
│  1. docker run -e                           │
│     docker run -e NODE_ENV=production       │
│     优先级：最高                            │
│                                             │
│  2. docker-compose.yml environment          │
│     environment:                            │
│       NODE_ENV: production                  │
│     优先级：高                              │
│                                             │
│  3. docker-compose.yml env_file             │
│     env_file:                               │
│       - .env                                │
│     优先级：中                              │
│                                             │
│  4. Dockerfile ENV                          │
│     ENV NODE_ENV=production                 │
│     优先级：低                              │
│                                             │
│  5. 应用默认值                              │
│     const port = process.env.PORT || 3000   │
│     优先级：最低                            │
│                                             │
└────────────────────────────────────────────┘
```

## 🚀 使用指南

### 实践1：基础环境变量

#### Dockerfile中定义

```dockerfile
# Dockerfile
FROM node:20-alpine

WORKDIR /app

# 💡 定义构建时变量（ARG）
ARG NODE_VERSION=20
ARG BUILD_DATE

# 💡 定义运行时变量（ENV）
ENV NODE_ENV=production
ENV PORT=3000
ENV LOG_LEVEL=info

COPY package*.json ./
RUN npm ci --only=production

COPY . .

EXPOSE $PORT

CMD ["node", "server.js"]

# 💡 构建时传递ARG
# docker build --build-arg BUILD_DATE=$(date -u +'%Y-%m-%dT%H:%M:%SZ') -t my-app .

# 💡 运行时覆盖ENV
# docker run -e NODE_ENV=development -e PORT=4000 my-app
```

#### docker run传递环境变量

```bash
# 💡 方式1：-e 单个变量
docker run -d \
  -e NODE_ENV=production \
  -e PORT=3000 \
  -e DATABASE_URL=postgres://user:pass@localhost:5432/db \
  my-app

# 💡 方式2：--env-file 从文件加载
# .env文件内容：
# NODE_ENV=production
# PORT=3000
# DATABASE_URL=postgres://user:pass@localhost:5432/db

docker run -d --env-file .env my-app

# 💡 方式3：从宿主机传递
docker run -d \
  -e DATABASE_URL=$DATABASE_URL \
  -e JWT_SECRET=$JWT_SECRET \
  my-app

# 💡 查看容器环境变量
docker exec my-app env

# 💡 查看特定变量
docker exec my-app printenv NODE_ENV
```

### 实践2：docker-compose中的环境变量

#### 基础配置

```yaml
# docker-compose.yml
version: '3.8'

services:
  api:
    build: .
    ports:
      - "3000:3000"
    
    # 💡 方式1：直接定义环境变量
    environment:
      NODE_ENV: production
      PORT: 3000
      LOG_LEVEL: info
    
    # 💡 方式2：YAML列表形式
    environment:
      - NODE_ENV=production
      - PORT=3000
      - LOG_LEVEL=info
    
    # 💡 方式3：从宿主机传递
    environment:
      DATABASE_URL: ${DATABASE_URL}
      JWT_SECRET: ${JWT_SECRET}
    
    # 💡 方式4：env_file加载
    env_file:
      - .env
      - .env.local

  mongodb:
    image: mongo:7-jammy
    environment:
      MONGO_INITDB_ROOT_USERNAME: ${MONGO_USERNAME:-admin}
      MONGO_INITDB_ROOT_PASSWORD: ${MONGO_PASSWORD}
    # ${VAR:-default} 语法：VAR不存在时使用default

# 💡 .env文件（docker-compose自动加载）
# NODE_ENV=production
# DATABASE_URL=mongodb://localhost:27017/mydb
# JWT_SECRET=your-secret-key
```

#### 多环境配置

```yaml
# docker-compose.yml（基础配置）
```

### 实践3：敏感信息管理

#### 方式1：Docker Secrets（Swarm模式）

```yaml
# docker-compose.yml（需要Docker Swarm）
version: '3.8'

services:
  api:
    build: .
    secrets:
      - db_password
      - jwt_secret
    environment:
      # 💡 读取secret文件内容
      DATABASE_PASSWORD_FILE: /run/secrets/db_password
      JWT_SECRET_FILE: /run/secrets/jwt_secret

secrets:
  db_password:
    file: ./secrets/db_password.txt
  jwt_secret:
    file: ./secrets/jwt_secret.txt

# 应用代码读取secret
// config.js
import fs from 'fs';

function getSecret(envKey) {
  const secretFile = process.env[`${envKey}_FILE`];
  if (secretFile && fs.existsSync(secretFile)) {
    return fs.readFileSync(secretFile, 'utf8').trim();
  }
  return process.env[envKey];
}

module.exports = {
  database: {
    password: getSecret('DATABASE_PASSWORD')
  },
  jwt: {
    secret: getSecret('JWT_SECRET')
  }
};
```

### 实践4：配置文件挂载

#### 挂载配置文件

```yaml
# docker-compose.yml
version: '3.8'

services:
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      # 💡 挂载配置文件（只读）
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/conf.d:/etc/nginx/conf.d:ro

  api:
    build: .
    volumes:
      # 💡 挂载应用配置
      - ./config/production.json:/app/config/production.json:ro

  postgres:
    image: postgres:16-alpine
    volumes:
      # 💡 挂载初始化脚本
      - ./init-scripts:/docker-entrypoint-initdb.d:ro
      # 💡 挂载自定义配置
      - ./postgres/postgresql.conf:/etc/postgresql/postgresql.conf:ro
    command: postgres -c config_file=/etc/postgresql/postgresql.conf

# nginx/nginx.conf
events {
  worker_connections 1024;
}

http {
  upstream api_backend {
    server api:3000;
  }
  
  server {
    listen 80;
    
    location /api {
      proxy_pass http://api_backend;
      proxy_set_header Host $host;
      proxy_set_header X-Real-IP $remote_addr;
    }
  }
}
```

#### 动态配置更新

```yaml
# docker-compose.yml
version: '3.8'

services:
  api:
    build: .
    volumes:
      # 💡 挂载配置目录（支持热重载）
      - ./config:/app/config:ro
    environment:
      CONFIG_PATH: /app/config

# 应用代码实现配置热重载
// config-loader.js
import fs from 'fs';
import path from 'path';

class ConfigLoader {
  constructor(configPath) {
    this.configPath = configPath;
    this.config = this.loadConfig();
    this.watchConfig();
  }

  loadConfig() {
    const configFile = path.join(
      this.configPath,
      `${process.env.NODE_ENV}.json`
    );
    
    if (fs.existsSync(configFile)) {
      console.log('Loading config from:', configFile);
      const content = fs.readFileSync(configFile, 'utf8');
      return JSON.parse(content);
    }
    
    return {};
  }

  watchConfig() {
    const configFile = path.join(
      this.configPath,
      `${process.env.NODE_ENV}.json`
    );
    
    // 💡 监听配置文件变化
    fs.watch(configFile, (eventType) => {
      if (eventType === 'change') {
        console.log('Config file changed, reloading...');
        this.config = this.loadConfig();
      }
    });
  }

  get(key) {
    return this.config[key];
  }
}

export default new ConfigLoader(process.env.CONFIG_PATH || './config');
```

## 🔧 错误处理

### 高频错误1：环境变量未定义

**现象：**
```javascript
// 应用启动失败
Error: DATABASE_URL is not defined
```

**解决方案：**
```javascript
// ❌ 直接使用（不安全）
const dbUrl = process.env.DATABASE_URL;

// ✅ 使用joi验证（推荐）
import Joi from 'joi';

const envSchema = Joi.object({
  NODE_ENV: Joi.string()
    .valid('development', 'staging', 'production')
    .default('production'),
  PORT: Joi.number().default(3000),
  DATABASE_URL: Joi.string().required(),
  JWT_SECRET: Joi.string().min(32).required(),
  REDIS_URL: Joi.string().required()
}).unknown();

const { error, value: env } = envSchema.validate(process.env);

if (error) {
  throw new Error(`Config validation error: ${error.message}`);
}

module.exports = env;

// ✅ 使用dotenv-safe（简单场景）
// .env.example
// NODE_ENV=
// DATABASE_URL=
// JWT_SECRET=

// 应用启动时自动检查
import 'dotenv-safe/config';
// 如果.env缺少变量，会抛出错误
```

### 高频错误2：敏感信息泄露

**现象：**
```bash
# 敏感信息提交到Git
git log -p | grep -i password
# 发现：DATABASE_PASSWORD=secret123
```

**防护方案：**
```bash
# 1. 立即修改.gitignore
echo ".env" >> .gitignore
echo ".env.local" >> .gitignore
echo ".env.*.local" >> .gitignore

# 2. 从Git历史中移除敏感文件
git filter-branch --force --index-filter \
  "git rm --cached --ignore-unmatch .env" \
  --prune-empty --tag-name-filter cat -- --all

# 3. 使用git-secrets防止未来泄露
brew install git-secrets  # macOS
git secrets --install
git secrets --register-aws

# 4. 添加pre-commit hook
# .git/hooks/pre-commit
#!/bin/sh
if git diff --cached --name-only | grep -q "\.env$"; then
  echo "Error: Attempting to commit .env file"
  exit 1
fi

# 5. 使用GitHub Secret Scanning（自动）
# GitHub会自动扫描提交中的敏感信息
```

## 📋 总结

### 核心要点

1. **环境变量优先**：配置与代码分离
2. **.env文件**：不提交到Git，使用.env.example模板
3. **敏感信息**：使用Secrets、Vault等工具
4. **配置验证**：启动时验证必需配置

### 实现关键

1. **Dockerfile ENV**: 默认值
2. **docker-compose environment**: 服务配置
3. **.env文件**: 本地开发配置
4. **Secrets**: 敏感信息管理

### 应用价值

- **安全性**：敏感信息不进入代码仓库
- **灵活性**：同一镜像适用所有环境
- **可维护性**：配置集中管理
- **符合标准**：遵循12-Factor App原则

### 面试重点

**Q1：为什么使用环境变量而非配置文件？**
答：符合12-Factor App原则，配置与代码分离，同一镜像适用所有环境，敏感信息不进入代码仓库。

**Q2：如何管理敏感信息？**
答：1) Docker Secrets 2) 外部Vault工具 3) CI/CD平台的Secret功能 4) .env文件不提交Git。

**Q3：环境变量的优先级是什么？**
答：docker run -e > docker-compose environment > env_file > Dockerfile ENV > 应用默认值。

### 注意事项

⚠️ **.env不要提交**：使用.gitignore排除
⚠️ **验证配置**：启动时检查必需变量
⚠️ **使用模板**：提供.env.example给团队
⚠️ **最小权限**：只暴露必需的环境变量
