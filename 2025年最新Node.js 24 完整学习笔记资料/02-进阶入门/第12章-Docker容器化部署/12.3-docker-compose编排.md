# 12.3 docker-compose编排

> docker-compose.yml语法、多服务编排、网络配置、数据卷管理、服务依赖、健康检查

## 📖 知识点概述

### 核心概念

**docker-compose**是Docker官方的容器编排工具，通过YAML文件定义和运行多容器应用。一个命令即可启动、停止、管理整个应用栈。

### 为什么需要docker-compose？

```
❌ 不使用docker-compose：手动管理多个容器

# 启动MongoDB
docker run -d --name mongodb \
  -p 27017:27017 \
  -v mongo-data:/data/db \
  -e MONGO_INITDB_ROOT_USERNAME=admin \
  -e MONGO_INITDB_ROOT_PASSWORD=secret \
  mongo:7-jammy

# 启动Redis
docker run -d --name redis \
  -p 6379:6379 \
  -v redis-data:/data \
  redis:7-alpine

# 创建网络
docker network create app-network
docker network connect app-network mongodb
docker network connect app-network redis

# 启动API（依赖MongoDB和Redis）
docker run -d --name api \
  --network app-network \
  -p 3000:3000 \
  -e MONGODB_URI=mongodb://admin:secret@mongodb:27017/mydb \
  -e REDIS_URL=redis://redis:6379 \
  my-api:latest

问题：
- 命令繁琐，容易出错
- 难以管理多个容器
- 无法保证启动顺序
- 难以在团队中共享配置

✅ 使用docker-compose：一个文件搞定

# docker-compose.yml
version: '3.8'
services:
  mongodb:
    image: mongo:7-jammy
    volumes:
      - mongo-data:/data/db
    environment:
      MONGO_INITDB_ROOT_USERNAME: admin
      MONGO_INITDB_ROOT_PASSWORD: secret

  redis:
    image: redis:7-alpine
    volumes:
      - redis-data:/data

  api:
    build: .
    ports:
      - "3000:3000"
    depends_on:
      - mongodb
      - redis
    environment:
      MONGODB_URI: mongodb://admin:secret@mongodb:27017/mydb
      REDIS_URL: redis://redis:6379

volumes:
  mongo-data:
  redis-data:

# 💡 一键启动所有服务
docker-compose up -d

优点：
- 声明式配置，清晰易懂
- 一个命令管理所有容器
- 自动创建网络
- 服务依赖管理
- 易于版本控制和共享
```

### 准备：安装docker-compose

```bash
# 💡 Docker Desktop自带docker-compose

# 验证安装
docker-compose --version
# 输出：Docker Compose version v2.23.3

# 💡 Linux手动安装（如果需要）
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 💡 v2版本命令（推荐）

# 输出：Docker Compose version v2.23.3

# ⚠️ 兼容性说明：
# v1: docker-compose (独立命令)
# v2: docker compose (docker子命令)
```

### 理解1：基础配置

#### 最简单的docker-compose.yml

```yaml
# docker-compose.yml
version: '3.8'

services:
  # 定义一个Node.js服务
  app:
    image: node:20-alpine
    command: node app.js
    working_dir: /app
    volumes:
      - ./:/app
    ports:
      - "3000:3000"
    environment:
      NODE_ENV: development

# 💡 启动
# docker compose up

# 💡 后台启动
# docker compose up -d

# 💡 停止
# docker compose down
```

#### 服务配置详解

```yaml
version: '3.8'

services:
  api:
    # 💡 方式1：使用现有镜像
    image: node:20-alpine
    
    # 💡 方式2：构建镜像
    build:
      context: .              # 构建上下文
      dockerfile: Dockerfile  # Dockerfile路径
      args:                   # 构建参数
        NODE_VERSION: 20
    
    # 💡 方式3：指定镜像名称
    image: my-api:latest      # 与build同时使用，指定构建后的镜像名
    
    # 💡 容器名称
    container_name: my-api-container
    
    # 💡 端口映射
    ports:
      - "3000:3000"           # 宿主机:容器
      - "3001:3001"
    
    # 💡 环境变量
    environment:
      NODE_ENV: production
      PORT: 3000
      DATABASE_URL: postgres://user:pass@postgres:5432/db
    
    # 💡 或使用env_file
    env_file:
      - .env
      - .env.production
    
    # 💡 数据卷
    volumes:
      - ./src:/app/src        # Bind mount
      - node_modules:/app/node_modules  # Named volume
      - /app/logs             # Anonymous volume
    
    # 💡 工作目录
    working_dir: /app
    
    # 💡 启动命令
    command: node server.js
    
    # 💡 服务依赖
    depends_on:
      - postgres
      - redis
    
    # 💡 重启策略
    restart: unless-stopped   # no, always, on-failure, unless-stopped
    
    # 💡 网络
    networks:
      - app-network
    
    # 💡 健康检查
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s

networks:
  app-network:
    driver: bridge

volumes:
  node_modules:
```

### 实践1：Node.js + MongoDB + Redis

#### 完整的全栈应用

```yaml
# docker-compose.yml
version: '3.8'

services:
  # 💡 Node.js API服务
  api:
    build:
      context: ./api
      dockerfile: Dockerfile
    image: my-api:latest
    container_name: api-server
    ports:
      - "3000:3000"
    environment:
      NODE_ENV: production
      PORT: 3000
      MONGODB_URI: mongodb://admin:secret@mongodb:27017/mydb?authSource=admin
      REDIS_URL: redis://redis:6379
    volumes:
      - ./api/logs:/app/logs
    depends_on:
      mongodb:
        condition: service_healthy
      redis:
        condition: service_started
    restart: unless-stopped
    networks:
      - app-network
    healthcheck:
      test: ["CMD", "node", "healthcheck.js"]
      interval: 30s
      timeout: 5s
      retries: 3

  # 💡 MongoDB数据库
  mongodb:
    image: mongo:7-jammy
    container_name: mongodb-server
    ports:
      - "27017:27017"
    environment:
      MONGO_INITDB_ROOT_USERNAME: admin
      MONGO_INITDB_ROOT_PASSWORD: secret
      MONGO_INITDB_DATABASE: mydb
    volumes:
      - mongo-data:/data/db
      - ./init-scripts:/docker-entrypoint-initdb.d
    restart: unless-stopped
    networks:
      - app-network
    healthcheck:
      test: ["CMD", "mongosh", "--eval", "db.adminCommand('ping')"]
      interval: 10s
      timeout: 5s
      retries: 5

  # 💡 Redis缓存
  redis:
    image: redis:7-alpine
    container_name: redis-server
    ports:
      - "6379:6379"
    command: redis-server --appendonly yes
    volumes:
      - redis-data:/data
    restart: unless-stopped
    networks:
      - app-network
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 3s
      retries: 3

  # 💡 Nginx反向代理
  nginx:
    image: nginx:alpine
    container_name: nginx-proxy
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/ssl:/etc/nginx/ssl:ro
    depends_on:
      - api
    restart: unless-stopped
    networks:
      - app-network

# 💡 网络配置
networks:
  app-network:
    driver: bridge

# 💡 数据卷配置
volumes:
  mongo-data:
    driver: local
  redis-data:
    driver: local

# 💡 常用命令
# 启动所有服务：docker compose up -d
# 查看日志：docker compose logs -f
# 停止所有服务：docker compose down
# 重启单个服务：docker compose restart api
# 查看服务状态：docker compose ps
```

#### 初始化脚本

```javascript
// init-scripts/init-db.js
// MongoDB初始化脚本（容器首次启动时执行）

db = db.getSiblingDB('mydb');

// 创建用户
db.createUser({
  user: 'app_user',
  pwd: 'app_password',
  roles: [
    {
      role: 'readWrite',
      db: 'mydb'
    }
  ]
});

// 创建集合和索引
db.createCollection('users');
db.users.createIndex({ email: 1 }, { unique: true });

db.createCollection('products');
db.products.createIndex({ name: 'text' });

print('Database initialized successfully!');
```

```yaml
# docker-compose.yml（基础配置）
version: '3.8'

services:
  api:

  mongo-data:
  redis-data:

version: '3.8'

services:
  api:
    build:
      target: development
    volumes:
      - ./src:/app/src        # 代码热重载
      - /app/node_modules
    environment:
      LOG_LEVEL: debug
    command: npm run dev      # 使用nodemon

  mongodb:
    ports:
      - "27017:27017"         # 暴露端口用于调试

  redis:
    ports:
      - "6379:6379"

  # 💡 开发工具：Mongo Express
  mongo-express:
    image: mongo-express:latest
    ports:
      - "8081:8081"
    environment:
      ME_CONFIG_MONGODB_ADMINUSERNAME: admin
      ME_CONFIG_MONGODB_ADMINPASSWORD: secret
      ME_CONFIG_MONGODB_URL: mongodb://admin:secret@mongodb:27017/

# docker-compose.prod.yml（生产环境）
version: '3.8'

services:
  api:
    build:
      target: production
    deploy:
      replicas: 3
      resources:
        limits:
          cpus: '1'
          memory: 512M
        reservations:
          cpus: '0.5'
          memory: 256M
    restart: always
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

  mongodb:
    # 生产环境不暴露端口
    environment:
      MONGO_INITDB_ROOT_USERNAME: ${MONGO_USERNAME}
      MONGO_INITDB_ROOT_PASSWORD: ${MONGO_PASSWORD}
    deploy:
      resources:
        limits:
          memory: 2G

  redis:
    command: redis-server --requirepass ${REDIS_PASSWORD}

# 💡 使用方法
# 开发环境（自动加载override）：
# docker compose up -d

# 生产环境：
# docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d
```

## 🔧 错误处理

### 高频错误1：服务无法启动

**现象：**
```bash
docker compose up
# ERROR: Service 'api' failed to build
```

**排查步骤：**
```bash
# 1. 查看详细日志
docker compose up --build

# 2. 单独构建服务
docker compose build api

# 3. 检查Dockerfile
# 确保Dockerfile存在且语法正确

# 4. 清理并重建
docker compose down -v
docker compose build --no-cache
docker compose up -d

# 5. 常见原因：
# - Dockerfile路径错误
# - 构建上下文不正确
# - 依赖安装失败
# - 网络问题
```

### 高频错误2：服务间无法通信

**现象：**
```
api-1  | Error: connect ECONNREFUSED mongodb:27017
```

**解决方案：**
```yaml
# ❌ 错误：服务不在同一网络
version: '3.8'
services:
  api:
    image: my-api
    # 没有指定网络
  
  mongodb:
    image: mongo:7-jammy
    # 没有指定网络

# ✅ 正确：指定同一网络
version: '3.8'
services:
  api:
    image: my-api
    networks:
      - app-network
  
  mongodb:
    image: mongo:7-jammy
    networks:
      - app-network

networks:
  app-network:

# 💡 或使用默认网络（推荐）
version: '3.8'
services:
  api:
    image: my-api
  mongodb:
    image: mongo:7-jammy
# compose自动创建默认网络

# 💡 连接测试
docker compose exec api ping mongodb
docker compose exec api curl mongodb:27017
```

### 高频错误3：端口冲突

**现象：**
```
Error: bind: address already in use
```

**解决方案：**
```bash
# 1. 检查端口占用
lsof -i :3000
netstat -ano | findstr :3000  # Windows

# 2. 停止占用端口的容器
docker compose down

# 3. 修改端口映射
# docker-compose.yml
services:
  api:
    ports:
      - "3001:3000"  # 改为3001

# 4. 或停止其他占用端口的服务
docker ps | grep 3000
docker stop <container_id>
```

## 📚 应用场景

### 场景：完整的微服务架构

```yaml
# docker-compose.yml
version: '3.8'

services:
  # 💡 前端应用（Next.js）
  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    image: my-frontend:latest
    container_name: frontend
    ports:
      - "3000:3000"
    environment:
      NEXT_PUBLIC_API_URL: http://localhost/api
    depends_on:
      - api
    restart: unless-stopped
    networks:
      - app-network

  # 💡 后端API（Node.js + Express）
  api:
    build:
      context: ./api
      dockerfile: Dockerfile
    image: my-api:latest
    container_name: api
    environment:
      NODE_ENV: production
      PORT: 3000
      DATABASE_URL: postgresql://postgres:secret@postgres:5432/mydb
      REDIS_URL: redis://redis:6379
      JWT_SECRET: ${JWT_SECRET}
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
    restart: unless-stopped
    networks:
      - app-network
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/health"]
      interval: 30s
      timeout: 10s
      retries: 3

  # 💡 PostgreSQL数据库
  postgres:
    image: postgres:16-alpine
    container_name: postgres
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: secret
      POSTGRES_DB: mydb
    volumes:
      - postgres-data:/var/lib/postgresql/data
      - ./init-scripts:/docker-entrypoint-initdb.d
    restart: unless-stopped
    networks:
      - app-network
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5

  # 💡 Redis缓存
  redis:
    image: redis:7-alpine
    container_name: redis
    command: redis-server --requirepass ${REDIS_PASSWORD}
    volumes:
      - redis-data:/data
    restart: unless-stopped
    networks:
      - app-network
    healthcheck:
      test: ["CMD", "redis-cli", "--raw", "incr", "ping"]
      interval: 10s
      timeout: 3s
      retries: 3

  # 💡 消息队列（RabbitMQ）
  rabbitmq:
    image: rabbitmq:3-management-alpine
    container_name: rabbitmq
    ports:
      - "5672:5672"
      - "15672:15672"
    environment:
      RABBITMQ_DEFAULT_USER: admin
      RABBITMQ_DEFAULT_PASS: ${RABBITMQ_PASSWORD}
    volumes:
      - rabbitmq-data:/var/lib/rabbitmq
    restart: unless-stopped
    networks:
      - app-network

  # 💡 Nginx反向代理
  nginx:
    image: nginx:alpine
    container_name: nginx
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/ssl:/etc/nginx/ssl:ro
    depends_on:
      - frontend
      - api
    restart: unless-stopped
    networks:
      - app-network

  # 💡 监控（Prometheus）
  prometheus:
    image: prom/prometheus:latest
    container_name: prometheus
    ports:
      - "9090:9090"
    volumes:
      - ./prometheus/prometheus.yml:/etc/prometheus/prometheus.yml:ro
      - prometheus-data:/prometheus
    command:
      - '--config.file=/etc/prometheus/prometheus.yml'
      - '--storage.tsdb.path=/prometheus'
    restart: unless-stopped
    networks:
      - app-network

  # 💡 监控可视化（Grafana）
  grafana:
    image: grafana/grafana:latest
    container_name: grafana
    ports:
      - "3001:3000"
    environment:
      GF_SECURITY_ADMIN_PASSWORD: ${GRAFANA_PASSWORD}
    volumes:
      - grafana-data:/var/lib/grafana
    depends_on:
      - prometheus
    restart: unless-stopped
    networks:
      - app-network

networks:
  app-network:
    driver: bridge
```

## 📋 总结

### 核心要点

1. **声明式配置**：YAML文件定义所有服务
2. **服务编排**：一键启动/停止多个容器
3. **网络管理**：自动创建网络，容器名互通
4. **依赖管理**：depends_on + healthcheck

### 实现关键

1. **version**: 指定Compose文件版本
2. **services**: 定义所有服务
3. **networks**: 配置网络
4. **volumes**: 配置数据卷
5. **depends_on**: 服务依赖关系

### 应用价值

- **简化部署**：一个命令启动整个应用栈
- **环境一致**：开发、测试、生产环境统一
- **易于维护**：配置集中管理
- **快速迭代**：快速启停服务

### 面试重点

**Q1：docker-compose和docker的区别？**
答：docker管理单个容器，docker-compose管理多个容器。compose通过YAML文件定义服务，一个命令管理整个应用栈。

**Q2：如何实现服务间通信？**
答：服务连接到同一网络后，可通过服务名互相访问。compose自动创建默认网络，服务名即DNS名。

**Q3：depends_on能保证服务启动顺序吗？**
答：只能保证启动顺序，不能保证服务就绪。需要配合healthcheck使用`condition: service_healthy`。

**Q4：如何区分开发和生产环境？**
答：使用多个compose文件：docker-compose.yml（基础）+ docker-compose.override.yml（开发）+ docker-compose.prod.yml（生产）。

### 注意事项

⚠️ **网络隔离**：不同compose项目的服务默认隔离
⚠️ **数据持久化**：使用named volume而非anonymous volume
⚠️ **健康检查**：关键服务必须配置healthcheck
⚠️ **环境变量**：敏感信息不要提交到版本控制
