## 📖 知识点概述

### 核心概念

**镜像发布**是将构建好的Docker镜像推送到Registry（镜像仓库），供团队成员或生产环境拉取使用。常见的Registry包括Docker Hub、阿里云ACR、Harbor私有仓库等。

### 镜像仓库类型

```
┌─────────────────────────────────────────────┐
│          镜像仓库对比                        │
├─────────────────────────────────────────────┤
│                                              │
│  Docker Hub（官方）                          │
│  ├─ 优点：全球可用、免费公开仓库             │
│  ├─ 缺点：国内访问慢、私有仓库收费           │
│  └─ 适用：开源项目、个人学习                 │
│                                              │
│  阿里云ACR（推荐）                           │
│  ├─ 优点：国内访问快、免费个人版             │
│  ├─ 缺点：需要阿里云账号                     │
│  └─ 适用：国内生产环境                       │
│                                              │
│  腾讯云TCR                                   │
│  ├─ 优点：国内访问快、企业版功能丰富         │
│  ├─ 缺点：个人版功能有限                     │
│  └─ 适用：腾讯云用户                         │
│                                              │
│  Harbor（私有）                              │
│  ├─ 优点：完全自主可控、企业级功能           │
│  ├─ 缺点：需要自行维护                       │
│  └─ 适用：大型企业、内网环境                 │
│                                              │
└─────────────────────────────────────────────┘
```

### 镜像标签策略

```
推荐的标签策略：

1. 语义化版本号
   my-app:1.0.0
   my-app:1.0.1
   my-app:2.0.0

2. Git提交哈希
   my-app:abc1234
   my-app:main-abc1234

3. 环境标签
   my-app:production
   my-app:staging
   my-app:dev

4. 日期标签
   my-app:2025-01-01
   my-app:20250101-1430

5. latest标签（指向最新稳定版）
   my-app:latest

⚠️ 注意：生产环境不要使用latest标签！
```

### 实践1：Docker Hub发布

#### 注册与登录

```bash
# 💡 1. 注册Docker Hub账号
# 访问：https://hub.docker.com/signup

# 💡 2. 登录Docker Hub
docker login

# 输入用户名和密码
# Username: your-username
# Password: 
# Login Succeeded

# 💡 3. 查看登录信息
cat ~/.docker/config.json

# 💡 4. 登出
docker logout
```

#### 构建与推送镜像

```bash
# 💡 1. 构建镜像（带用户名）
docker build -t your-username/my-app:1.0.0 .

# 💡 2. 给现有镜像打标签
docker tag my-app:latest your-username/my-app:1.0.0
docker tag my-app:latest your-username/my-app:latest

# 💡 3. 推送镜像到Docker Hub
docker push your-username/my-app:1.0.0
docker push your-username/my-app:latest

# 输出：
# The push refers to repository [docker.io/your-username/my-app]
# abc123def456: Pushed
# def456ghi789: Layer already exists
# 1.0.0: digest: sha256:xxx size: 1234

# 💡 4. 推送所有标签
docker push your-username/my-app --all-tags

# 💡 5. 拉取镜像
docker pull your-username/my-app:1.0.0

# 💡 6. 删除远程镜像（需要在Web界面操作）
# 访问：https://hub.docker.com/r/your-username/my-app/tags
```

#### 创建自动化构建

```bash
# Docker Hub Automated Builds（已弃用）
# 推荐使用GitHub Actions替代

# .github/workflows/docker-publish.yml
name: Docker Publish

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout
        uses: actions/checkout@v3
      
      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v2
      
      - name: Login to Docker Hub
        uses: docker/login-action@v2
        with:
          username: ${{ secrets.DOCKERHUB_USERNAME }}
          password: ${{ secrets.DOCKERHUB_TOKEN }}
      
      - name: Extract metadata
        id: meta
        uses: docker/metadata-action@v4
        with:
          images: your-username/my-app
          tags: |
            type=semver,pattern={{version}}
            type=semver,pattern={{major}}.{{minor}}
            type=semver,pattern={{major}}
            type=sha
      
      - name: Build and push
        uses: docker/build-push-action@v4
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=registry,ref=your-username/my-app:latest
          cache-to: type=inline

# 💡 使用方法
# 1. 创建Docker Hub Access Token
# 2. 在GitHub仓库设置Secrets：
#    DOCKERHUB_USERNAME
#    DOCKERHUB_TOKEN
# 3. 推送tag触发构建
# git tag v1.0.0
# git push origin v1.0.0
```

#### CI/CD集成阿里云ACR

```yaml
# .github/workflows/aliyun-acr.yml
name: Build and Push to Aliyun ACR

on:
  push:
    branches:
      - main
    tags:
      - 'v*'

env:
  REGION: cn-hangzhou
  NAMESPACE: my-namespace
  IMAGE_NAME: my-app

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout
        uses: actions/checkout@v3
      
      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v2
      
      - name: Login to Aliyun ACR
        uses: docker/login-action@v2
        with:
          registry: registry.${{ env.REGION }}.aliyuncs.com
          username: ${{ secrets.ALIYUN_USERNAME }}
          password: ${{ secrets.ALIYUN_PASSWORD }}
      
      - name: Extract version
        id: version
        run: |
          if [[ $GITHUB_REF == refs/tags/* ]]; then
            VERSION=${GITHUB_REF#refs/tags/v}
          else
            VERSION=dev-${GITHUB_SHA::8}
          fi
          echo "VERSION=$VERSION" >> $GITHUB_OUTPUT
      
      - name: Build and push
        uses: docker/build-push-action@v4
        with:
          context: .
          push: true
          tags: |
            registry.${{ env.REGION }}.aliyuncs.com/${{ env.NAMESPACE }}/${{ env.IMAGE_NAME }}:${{ steps.version.outputs.VERSION }}
            registry.${{ env.REGION }}.aliyuncs.com/${{ env.NAMESPACE }}/${{ env.IMAGE_NAME }}:latest
          cache-from: type=gha
          cache-to: type=gha,mode=max

# 💡 配置GitHub Secrets
# ALIYUN_USERNAME: 阿里云用户名
# ALIYUN_PASSWORD: Registry固定密码
```

### 实践4：镜像安全扫描

#### 使用Trivy扫描

```bash
# 💡 1. 安装Trivy
brew install trivy  # macOS
# 或
wget https://github.com/aquasecurity/trivy/releases/download/v0.48.0/trivy_0.48.0_Linux-64bit.tar.gz

# 💡 2. 扫描本地镜像
trivy image my-app:latest

# 输出：
# my-app:latest (alpine 3.19.0)
# =================================
# Total: 15 (CRITICAL: 2, HIGH: 5, MEDIUM: 8)
# 
# ┌────────────┬───────────────┬──────────┬──────────────────┐
# │  Library   │ Vulnerability │ Severity │ Installed Ver    │
# ├────────────┼───────────────┼──────────┼──────────────────┤
# │ openssl    │ CVE-2023-1234 │ CRITICAL │ 3.0.8-r0         │
# │ curl       │ CVE-2023-5678 │ HIGH     │ 8.4.0-r0         │
# └────────────┴───────────────┴──────────┴──────────────────┘

# 💡 3. 只显示CRITICAL和HIGH
trivy image --severity CRITICAL,HIGH my-app:latest

# 💡 4. 输出为JSON
trivy image -f json -o results.json my-app:latest

# 💡 5. 扫描远程镜像
trivy image registry.cn-hangzhou.aliyuncs.com/my-namespace/my-app:1.0.0

# 💡 6. 忽略未修复的漏洞
trivy image --ignore-unfixed my-app:latest

# 💡 7. 设置退出代码（CI/CD）
trivy image --exit-code 1 --severity CRITICAL my-app:latest
# 如果有CRITICAL漏洞，返回非0退出码
```

#### CI/CD集成安全扫描

```yaml
# .github/workflows/security-scan.yml
name: Security Scan

on:
  push:
    branches:
      - main
  pull_request:

jobs:
  scan:
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout
        uses: actions/checkout@v3
      
      - name: Build image
        run: docker build -t my-app:${{ github.sha }} .
      
      - name: Run Trivy vulnerability scanner
        uses: aquasecurity/trivy-action@master
        with:
          image-ref: 'my-app:${{ github.sha }}'
          format: 'sarif'
          output: 'trivy-results.sarif'
          severity: 'CRITICAL,HIGH'
      
      - name: Upload Trivy results to GitHub Security
        uses: github/codeql-action/upload-sarif@v2
        with:
          sarif_file: 'trivy-results.sarif'
      
      - name: Fail if CRITICAL vulnerabilities
        uses: aquasecurity/trivy-action@master
        with:
          image-ref: 'my-app:${{ github.sha }}'
          exit-code: '1'
          severity: 'CRITICAL'
          ignore-unfixed: true
```

### 实践5：镜像标签管理

#### 语义化版本标签

```bash
# 💡 构建并打多个标签
docker build \
  -t my-app:1.2.3 \
  -t my-app:1.2 \
  -t my-app:1 \
  -t my-app:latest \
  .

# 💡 推送所有标签
docker push my-app:1.2.3
docker push my-app:1.2
docker push my-app:1
docker push my-app:latest

# 💡 或使用脚本自动化
#!/bin/bash
VERSION=$1
MAJOR=$(echo $VERSION | cut -d. -f1)
MINOR=$(echo $VERSION | cut -d. -f1-2)

docker build \
  -t my-app:${VERSION} \
  -t my-app:${MINOR} \
  -t my-app:${MAJOR} \
  -t my-app:latest \
  .

docker push my-app --all-tags

# 使用：
# ./build.sh 1.2.3
```

#### Git提交哈希标签

```bash
# 💡 使用Git提交哈希作为标签
GIT_COMMIT=$(git rev-parse --short HEAD)
GIT_BRANCH=$(git rev-parse --abbrev-ref HEAD)

docker build \
  -t my-app:${GIT_COMMIT} \
  -t my-app:${GIT_BRANCH}-${GIT_COMMIT} \
  -t my-app:${GIT_BRANCH}-latest \
  .

# 示例：
# my-app:abc1234
# my-app:main-abc1234
# my-app:main-latest
```

## 🔧 错误处理

### 高频错误1：推送权限不足

**现象：**
```
denied: requested access to the resource is denied
```

**解决方案：**
```bash
# 1. 确认已登录
docker login

# 2. 确认镜像名称格式正确
# ❌ 错误
docker push my-app:latest

# ✅ 正确（Docker Hub）
docker push your-username/my-app:latest

# ✅ 正确（阿里云）
docker push registry.cn-hangzhou.aliyuncs.com/namespace/my-app:latest

# 3. 检查仓库是否存在
# Docker Hub: https://hub.docker.com/r/your-username/my-app
# 阿里云: https://cr.console.aliyun.com/

# 4. 检查Access Token权限
# Docker Hub Settings -> Security -> Access Tokens
# 权限需要：Read, Write, Delete
```

### 高频错误2：镜像推送超时

**现象：**
```
net/http: request canceled while waiting for connection
```

**解决方案：**
```bash
# 1. 使用国内镜像加速
# ~/.docker/daemon.json
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com",
    "https://registry.docker-cn.com"
  ]
}

# 重启Docker
sudo systemctl restart docker

# 2. 推送到阿里云ACR（国内）
docker tag my-app:latest registry.cn-hangzhou.aliyuncs.com/namespace/my-app:latest
docker push registry.cn-hangzhou.aliyuncs.com/namespace/my-app:latest

# 3. 增加推送超时时间
export DOCKER_CLIENT_TIMEOUT=300
export COMPOSE_HTTP_TIMEOUT=300
```

### 高频错误3：镜像层已存在但推送失败

**现象：**
```
layer already exists but failed to push
```

**解决方案：**
```bash
# 1. 清理本地缓存
docker system prune -a

# 2. 重新构建（不使用缓存）
docker build --no-cache -t my-app:latest .

# 3. 删除远程镜像（如果存在）
# 在Registry Web界面删除

# 4. 重新推送
docker push my-app:latest
```

## 📚 应用场景

### 场景：完整的CI/CD流程

```yaml
# .github/workflows/complete-cicd.yml
name: Complete CI/CD Pipeline

on:
  push:
    branches:
      - main
      - develop
    tags:
      - 'v*'
  pull_request:
    branches:
      - main

env:
  REGISTRY: registry.cn-hangzhou.aliyuncs.com
  NAMESPACE: my-namespace
  IMAGE_NAME: my-app

jobs:
  # 💡 阶段1：代码检查
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '20'
      - run: npm ci
      - run: npm run lint

  # 💡 阶段2：单元测试
  test:
    runs-on: ubuntu-latest
    needs: lint
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '20'
      - run: npm ci
      - run: npm test
      - name: Upload coverage
        uses: codecov/codecov-action@v3

  # 💡 阶段3：构建镜像
  build:
    runs-on: ubuntu-latest
    needs: test
    steps:
      - uses: actions/checkout@v3
      
      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v2
      
      - name: Login to ACR
        uses: docker/login-action@v2
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ secrets.ACR_USERNAME }}
          password: ${{ secrets.ACR_PASSWORD }}
      
      - name: Extract metadata
        id: meta
        run: |
          if [[ $GITHUB_REF == refs/tags/* ]]; then
            VERSION=${GITHUB_REF#refs/tags/v}
            TAGS="${{ env.REGISTRY }}/${{ env.NAMESPACE }}/${{ env.IMAGE_NAME }}:${VERSION}"
            TAGS="${TAGS},${{ env.REGISTRY }}/${{ env.NAMESPACE }}/${{ env.IMAGE_NAME }}:latest"
          elif [[ $GITHUB_REF == refs/heads/main ]]; then
            TAGS="${{ env.REGISTRY }}/${{ env.NAMESPACE }}/${{ env.IMAGE_NAME }}:main-${GITHUB_SHA::8}"
          else
            TAGS="${{ env.REGISTRY }}/${{ env.NAMESPACE }}/${{ env.IMAGE_NAME }}:dev-${GITHUB_SHA::8}"
          fi
          echo "TAGS=$TAGS" >> $GITHUB_OUTPUT
      
      - name: Build and push
        uses: docker/build-push-action@v4
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.TAGS }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

  # 💡 阶段4：安全扫描
  scan:
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Run Trivy scanner
        uses: aquasecurity/trivy-action@master
        with:
          image-ref: '${{ env.REGISTRY }}/${{ env.NAMESPACE }}/${{ env.IMAGE_NAME }}:main-${{ github.sha }}'
          format: 'table'
          exit-code: '1'
          severity: 'CRITICAL,HIGH'
          ignore-unfixed: true

  # 💡 阶段5：部署到测试环境
  deploy-staging:
    runs-on: ubuntu-latest
    needs: scan
    if: github.ref == 'refs/heads/develop'
    steps:
      - name: Deploy to staging
        run: |
          # 使用SSH或kubectl部署
          echo "Deploying to staging..."

  # 💡 阶段6：部署到生产环境
  deploy-production:
    runs-on: ubuntu-latest
    needs: scan
    if: startsWith(github.ref, 'refs/tags/v')
    environment:
      name: production
      url: https://myapp.com
    steps:
      - name: Deploy to production
        run: |
          # 使用SSH或kubectl部署
          echo "Deploying to production..."
```

## 📋 总结

### 核心要点

1. **Docker Hub**：全球最大的公共Registry
2. **阿里云ACR**：国内推荐，访问速度快
3. **Harbor**：企业级私有Registry
4. **安全扫描**：Trivy扫描漏洞

### 实现关键

1. **docker login**: 登录Registry
2. **docker tag**: 标记镜像
3. **docker push**: 推送镜像
4. **trivy scan**: 安全扫描

### 应用价值

- **统一管理**：集中存储和分发镜像
- **版本控制**：镜像标签管理
- **安全保障**：漏洞扫描和修复
- **CI/CD集成**：自动化构建和发布

### 面试重点

**Q1：镜像推送到Registry的流程？**
答：1) docker login登录 2) docker tag标记镜像 3) docker push推送 4) docker pull拉取。

**Q2：如何管理镜像版本？**
答：使用语义化版本号（1.0.0）、Git提交哈希（abc1234）、环境标签（production）。生产环境避免使用latest。

**Q3：如何保证镜像安全？**
答：1) 定期扫描漏洞（Trivy） 2) 使用官方基础镜像 3) 最小权限原则 4) 镜像签名验证。

### 注意事项

⚠️ **不要使用latest**：生产环境使用明确版本号
⚠️ **定期扫描**：集成Trivy到CI/CD
⚠️ **Access Token**：使用Token而非密码
⚠️ **私有镜像**：敏感项目使用私有仓库
