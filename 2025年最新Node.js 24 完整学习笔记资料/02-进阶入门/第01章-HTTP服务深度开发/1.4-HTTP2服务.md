# 1.4 HTTP/2 服务搭建

> 配置和使用HTTP/2协议，提升Web应用性能

---

## 📖 知识点概述

### 核心概念

**HTTP/2**是HTTP协议的第二个主要版本（2015年发布），相比HTTP/1.1带来了显著的性能提升。Node.js 8.4+开始原生支持HTTP/2。

### 解决的问题

HTTP/1.1的性能瓶颈：
- **队头阻塞**：一个请求阻塞后续请求
- **连接数限制**：浏览器对同域名限制6-8个连接
- **头部冗余**：每次请求都发送完整headers
- **无优先级**：无法指定资源加载优先级

HTTP/2的解决方案：
- **多路复用**：单连接并发处理多个请求
- **头部压缩**：HPACK算法压缩headers
- **服务器推送**：主动推送资源到客户端
- **请求优先级**：客户端可指定资源优先级

### 主要特点

1. **二进制分帧**：数据以二进制帧传输（而非文本）
2. **多路复用**：单TCP连接处理所有请求/响应
3. **头部压缩**：使用HPACK算法，减少50-90%的头部大小
4. **服务器推送**：服务器可主动推送CSS/JS等资源
5. **流优先级**：客户端可设置流的优先级和依赖关系

### 典型应用场景

- **高性能Web应用**：减少延迟，提升加载速度
- **移动端应用**：节省带宽，提升弱网环境体验
- **API服务**：降低请求开销，提高吞吐量
- **实时应用**：利用服务器推送实现准实时通信

### 实践：HTTP/2服务器

#### 1. 基础HTTP/2服务器

```javascript
// http2-server.js
import http2 from 'node:http2';
import fs from 'node:fs';

// 读取TLS证书
const options = {
  key: fs.readFileSync('./server.key'),
  cert: fs.readFileSync('./server.crt')
};

// 创建HTTP/2服务器
const server = http2.createSecureServer(options);

server.on('stream', (stream, headers) => {
  // stream：HTTP/2流对象
  // headers：请求头（小写）
  
  console.log('收到请求:', headers[':path']);
  
  // 设置响应头
  stream.respond({
    ':status': 200,
    'content-type': 'text/html; charset=utf-8'
  });
  
  // 发送响应体
  stream.end('<h1>Hello HTTP/2!</h1>');
});

server.listen(3000, () => {
  console.log('HTTP/2服务器运行在 https://localhost:3000');
  console.log('注意：需要HTTPS，浏览器可能提示证书不安全（开发环境正常）');
});
```

#### 2. HTTP/2服务器推送

```javascript
// http2-push.js
import http2 from 'node:http2';
import fs from 'node:fs';
import path from 'node:path';
import { fileURLToPath } from 'node:url';

const __filename = fileURLToPath(import.meta.url);
const __dirname = path.dirname(__filename);

const options = {
  key: fs.readFileSync('./server.key'),
  cert: fs.readFileSync('./server.crt')
};

const server = http2.createSecureServer(options);

server.on('stream', (stream, headers) => {
  const requestPath = headers[':path'];
  
  if (requestPath === '/') {
    // 💡 技巧：服务器推送CSS和JS文件
    // 在客户端请求之前就主动推送
    
    // 推送CSS
    stream.pushStream({ ':path': '/style.css' }, (err, pushStream) => {
      if (err) {
        console.error('推送CSS失败:', err);
        return;
      }
      
      pushStream.respond({
        ':status': 200,
        'content-type': 'text/css'
      });
      
      pushStream.end('body { color: blue; font-size: 20px; }');
    });
    
    // 推送JS
    stream.pushStream({ ':path': '/app.js' }, (err, pushStream) => {
      if (err) {
        console.error('推送JS失败:', err);
        return;
      }
      
      pushStream.respond({
        ':status': 200,
        'content-type': 'application/javascript'
      });
      
      pushStream.end('console.log("JS from server push!");');
    });
    
    // 返回HTML
    stream.respond({
      ':status': 200,
      'content-type': 'text/html; charset=utf-8'
    });
    
    stream.end(`
      <!DOCTYPE html>
      <html>
        <head>
          <title>HTTP/2 Server Push</title>
          <link rel="stylesheet" href="/style.css">
        </head>
        <body>
          <h1>HTTP/2 服务器推送演示</h1>
          <p>打开开发者工具的Network面板，查看推送的资源（Initiator显示Push）</p>
          <script src="/app.js"></script>
        </body>
      </html>
    `);
  } 
  else if (requestPath === '/style.css') {
    // 💡 如果客户端请求了CSS（推送失败或被拒绝），正常返回
    stream.respond({
      ':status': 200,
      'content-type': 'text/css'
    });
    stream.end('body { color: blue; font-size: 20px; }');
  }
  else if (requestPath === '/app.js') {
    stream.respond({
      ':status': 200,
      'content-type': 'application/javascript'
    });
    stream.end('console.log("JS from normal request!");');
  }
  else {
    stream.respond({ ':status': 404 });
    stream.end('Not Found');
  }
});

server.listen(3000, () => {
  console.log('HTTP/2推送服务器运行在 https://localhost:3000');
  console.log('打开浏览器开发者工具，查看Network面板的Initiator列');
});
```

#### 3. 完整示例：API服务器

```javascript
// http2-api.js
import http2 from 'node:http2';
import fs from 'node:fs';
import { URL } from 'node:url';

const options = {
  key: fs.readFileSync('./server.key'),
  cert: fs.readFileSync('./server.crt')
};

// 模拟数据库
const users = new Map([
  [1, { id: 1, name: 'Alice', email: 'alice@example.com' }],
  [2, { id: 2, name: 'Bob', email: 'bob@example.com' }]
]);

// 工具函数：发送JSON响应
function sendJSON(stream, status, data) {
  stream.respond({
    ':status': status,
    'content-type': 'application/json'
  });
  stream.end(JSON.stringify(data));
}

// 工具函数：读取请求体
async function readBody(stream) {
  return new Promise((resolve, reject) => {
    let body = '';
    stream.on('data', chunk => { body += chunk; });
    stream.on('end', () => {
      try {
        resolve(JSON.parse(body));
      } catch (error) {
        resolve(body);
      }
    });
    stream.on('error', reject);
  });
}

const server = http2.createSecureServer(options);

server.on('stream', async (stream, headers) => {
  try {
    const method = headers[':method'];
    const path = headers[':path'];
    const url = new URL(path, 'https://localhost:3000');
    const pathname = url.pathname;
    
    // ===== 路由处理 =====
    
    // GET /api/users - 获取所有用户
    if (pathname === '/api/users' && method === 'GET') {
      const userList = Array.from(users.values());
      sendJSON(stream, 200, { data: userList });
      return;
    }
    
    // GET /api/users/:id - 获取单个用户
    const userMatch = pathname.match(/^\/api\/users\/(\d+)$/);
    if (userMatch && method === 'GET') {
      const id = parseInt(userMatch[1]);
      const user = users.get(id);
      
      if (user) {
        sendJSON(stream, 200, { data: user });
      } else {
        sendJSON(stream, 404, { error: 'User not found' });
      }
      return;
    }
    
    // POST /api/users - 创建用户
    if (pathname === '/api/users' && method === 'POST') {
      const body = await readBody(stream);
      
      if (!body.name || !body.email) {
        sendJSON(stream, 400, { error: 'Name and email are required' });
        return;
      }
      
      const newUser = {
        id: users.size + 1,
        name: body.name,
        email: body.email
      };
      
      users.set(newUser.id, newUser);
      
      stream.respond({
        ':status': 201,
        'content-type': 'application/json',
        'location': `/api/users/${newUser.id}`
      });
      stream.end(JSON.stringify({ data: newUser }));
      return;
    }
    
    // 404
    sendJSON(stream, 404, { error: 'Not Found' });
    
  } catch (error) {
    console.error('请求处理错误:', error);
    sendJSON(stream, 500, { error: 'Internal Server Error' });
  }
});

server.listen(3000, () => {
  console.log('HTTP/2 API服务器运行在 https://localhost:3000');
  console.log('测试命令:');
  console.log('  curl --http2 -k https://localhost:3000/api/users');
  console.log('  curl --http2 -k https://localhost:3000/api/users/1');
  console.log('  curl --http2 -k -X POST -H "Content-Type: application/json" -d \'{"name":"Charlie","email":"charlie@example.com"}\' https://localhost:3000/api/users');
});
```

## 🎯 应用场景

### 场景1：高性能Web应用

#### 问题背景
电商网站首页需要加载：
- 1个HTML
- 3个CSS文件
- 5个JS文件
- 20+个图片

HTTP/1.1下，浏览器限制6个并发连接，需要多轮请求，首屏时间长达3秒。

#### 技术选型理由
使用HTTP/2的原因：
- **多路复用**：所有资源通过单连接并发加载
- **服务器推送**：HTML返回时主动推送CSS/JS
- **头部压缩**：减少cookie等重复头部开销

#### 实现代码

```javascript
// ecommerce-server.js
import http2 from 'node:http2';
import fs from 'node:fs';
import path from 'node:path';

const options = {
  key: fs.readFileSync('./server.key'),
  cert: fs.readFileSync('./server.crt')
};

const PUBLIC_DIR = './public';

const server = http2.createSecureServer(options);

server.on('stream', (stream, headers) => {
  const requestPath = headers[':path'];
  
  if (requestPath === '/') {
    // 💡 服务器推送关键资源
    const criticalResources = [
      { path: '/css/main.css', type: 'text/css' },
      { path: '/css/header.css', type: 'text/css' },
      { path: '/js/vendor.js', type: 'application/javascript' },
      { path: '/js/app.js', type: 'application/javascript' }
    ];
    
    criticalResources.forEach(resource => {
      stream.pushStream({ ':path': resource.path }, (err, pushStream) => {
        if (err) return;
        
        const filePath = path.join(PUBLIC_DIR, resource.path);
        const content = fs.readFileSync(filePath);
        
        pushStream.respond({
          ':status': 200,
          'content-type': resource.type
        });
        
        pushStream.end(content);
      });
    });
    
    // 返回HTML
    stream.respond({
      ':status': 200,
      'content-type': 'text/html; charset=utf-8'
    });
    
    const html = fs.readFileSync(path.join(PUBLIC_DIR, 'index.html'));
    stream.end(html);
  } else {
    // 处理其他静态资源
    const filePath = path.join(PUBLIC_DIR, requestPath);
    
    try {
      const content = fs.readFileSync(filePath);
      stream.respond({ ':status': 200 });
      stream.end(content);
    } catch (error) {
      stream.respond({ ':status': 404 });
      stream.end('Not Found');
    }
  }
});

server.listen(3000);
```

#### 可量化效果
- **首屏时间**：从3000ms降到800ms（降低73%）
- **请求数**：从29个降到1个连接
- **带宽占用**：头部压缩节省30%流量
- **用户体验**：LCP指标从3.5s降到1.2s

| **请求优先级** | ❌ | ✅ | 更智能 |
| **队头阻塞** | ✅（HTTP层） | ❌（HTTP层） | 但TCP层仍存在 |

## 📋 总结

### 核心要点

1. **HTTP/2核心**：二进制分帧+多路复用+头部压缩
2. **必需HTTPS**：浏览器只支持h2（HTTP/2 over TLS）
3. **服务器推送**：主动推送关键资源，减少请求

### 实现关键

1. **使用createSecureServer**：`http2.createSecureServer(options)`
2. **监听stream事件**：每个请求触发一个stream
3. **响应头格式**：使用`:status`、`:method`等伪头部
4. **推送资源**：`stream.pushStream()`
5. **证书配置**：开发用mkcert，生产用Let's Encrypt

### 应用价值

- **性能提升**：首屏时间降低50-70%
- **用户体验**：移动端弱网环境明显改善
- **资源节省**：头部压缩节省流量
- **SEO优化**：Google将HTTP/2作为排名因素

### 面试重点

**常见问题**：
1. **HTTP/2相比HTTP/1.1有哪些改进？**
   - 多路复用、头部压缩、服务器推送、二进制分帧

2. **HTTP/2如何解决队头阻塞？**
   - HTTP层解决了（多路复用），但TCP层仍存在（HTTP/3用QUIC解决）

3. **服务器推送的原理？**
   - 服务器在返回HTML时，主动推送CSS/JS等关键资源

4. **为什么浏览器只支持h2（HTTPS）？**
   - 安全性考虑，且ALPN协商需要TLS

### 注意事项

⚠️ **HTTP/2需要HTTPS**：
```javascript
// ❌ 错误：createServer不支持HTTP/2

// ✅ 正确：createSecureServer

```

⚠️ **服务器推送不是万能的**：
- 推送的资源如果已缓存，浏览器会拒绝
- 过度推送会浪费带宽
