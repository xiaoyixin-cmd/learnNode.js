## 📖 知识点概述

### 核心概念

**http模块**是Node.js内置的核心模块，用于创建HTTP服务器和客户端。它是所有Node.js Web框架（Express、Koa、Fastify）的底层基础。

### 解决的问题

- **快速搭建Web服务**：无需额外依赖即可创建HTTP服务器
- **处理HTTP请求**：接收客户端请求，返回响应
- **自定义服务逻辑**：完全控制请求处理流程
- **理解框架原理**：学习框架底层如何工作

### 主要特点

1. **内置模块**：无需安装，直接import使用
2. **事件驱动**：基于EventEmitter，监听request事件
3. **流式处理**：请求和响应都是Stream对象
4. **高性能**：直接操作底层API，性能极佳
5. **灵活可控**：完全自定义请求处理逻辑

### 典型应用场景

- **学习HTTP协议**：深入理解HTTP工作原理
- **构建API服务**：创建RESTful API接口
- **静态文件服务器**：提供静态资源访问
- **反向代理**：转发请求到其他服务
- **性能要求极高的场景**：框架开销不可接受时

## 🚀 使用指南

### 理解：http模块做什么？

http模块提供了两个核心功能：
1. **创建服务器**（Server）：监听端口，接收HTTP请求
2. **创建客户端**（Client）：发送HTTP请求到其他服务器

本节重点学习如何创建服务器。

```javascript
// server.js
import http from 'node:http';

// 创建服务器
// 💡 技巧：使用箭头函数简化代码
const server = http.createServer((req, res) => {
  // req：IncomingMessage对象，包含请求信息
  // res：ServerResponse对象，用于发送响应
  
  // 设置响应头
  // ⚠️ 注意：必须在写入响应体之前设置
  res.writeHead(200, { 'Content-Type': 'text/plain; charset=utf-8' });
  
  // 写入响应体
  res.end('你好，Node.js！');
});

// 监听端口
// 💡 技巧：使用3000端口进行本地开发（不需要root权限）
server.listen(3000, () => {
  console.log('服务器运行在 http://localhost:3000');
});
```

#### 2. 使用async/await处理异步逻辑（2025推荐写法）

```javascript
// server-async.js
import http from 'node:http';

// 模拟异步数据库查询
async function fetchDataFromDB() {
  // 模拟延迟
  await new Promise(resolve => setTimeout(resolve, 100));
  return { message: '来自数据库的数据', timestamp: Date.now() };
}

// 💡 技巧：将请求处理器提取为async函数
async function handleRequest(req, res) {
  try {
    // 异步获取数据
    const data = await fetchDataFromDB();
    
    // 设置响应头
    res.writeHead(200, { 'Content-Type': 'application/json' });
    
    // 返回JSON数据
    res.end(JSON.stringify(data));
  } catch (error) {
    // ⚠️ 注意：必须捕获异步错误，否则会导致服务器崩溃
    console.error('处理请求失败:', error);
    res.writeHead(500, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({ error: 'Internal Server Error' }));
  }
}

const server = http.createServer((req, res) => {
  // 调用async处理器
  // ⚠️ 注意：这里不能直接await，需要用.catch()捕获错误
  handleRequest(req, res).catch(error => {
    console.error('未捕获的错误:', error);
    // 如果响应还没发送，发送500错误
    if (!res.headersSent) {
      res.writeHead(500);
      res.end('Internal Server Error');
    }
  });
});

server.listen(3000, () => {
  console.log('异步服务器运行在 http://localhost:3000');
});
```

#### 3. 完整示例：简单的API服务器

```javascript
// api-server.js
import http from 'node:http';
import { URL } from 'node:url';

// 模拟数据库
const users = [
  { id: 1, name: 'Alice', age: 25 },
  { id: 2, name: 'Bob', age: 30 }
];

async function handleRequest(req, res) {
  try {
    // 解析URL
    // 💡 技巧：使用URL类解析完整URL
    const url = new URL(req.url, `http://${req.headers.host}`);
    const pathname = url.pathname;
    const method = req.method;
    
    // 设置CORS头（允许跨域）
    // 💡 技巧：在实际项目中应该限制origin
    res.setHeader('Access-Control-Allow-Origin', '*');
    res.setHeader('Content-Type', 'application/json');
    
    // 路由处理
    if (pathname === '/api/users' && method === 'GET') {
      // 获取所有用户
      res.writeHead(200);
      res.end(JSON.stringify({ data: users }));
    } 
    else if (pathname.startsWith('/api/users/') && method === 'GET') {
      // 获取单个用户
      const id = parseInt(pathname.split('/')[3]);
      const user = users.find(u => u.id === id);
      
      if (user) {
        res.writeHead(200);
        res.end(JSON.stringify({ data: user }));
      } else {
        // ⚠️ 注意：资源不存在时返回404
        res.writeHead(404);
        res.end(JSON.stringify({ error: 'User not found' }));
      }
    }
    else if (pathname === '/api/users' && method === 'POST') {
      // 创建用户
      // 读取请求体
      let body = '';
      
      // ⚠️ 注意：req是Stream，需要监听data和end事件
      req.on('data', chunk => {
        body += chunk.toString();
      });
      
      req.on('end', () => {
        try {
          const newUser = JSON.parse(body);
          newUser.id = users.length + 1;
          users.push(newUser);
          
          // 💡 技巧：创建资源成功返回201状态码
          res.writeHead(201);
          res.end(JSON.stringify({ data: newUser }));
        } catch (error) {
          // JSON解析失败
          res.writeHead(400);
          res.end(JSON.stringify({ error: 'Invalid JSON' }));
        }
      });
      
      return; // ⚠️ 注意：异步处理，提前返回
    }
    else {
      // 路由不存在
      res.writeHead(404);
      res.end(JSON.stringify({ error: 'Not Found' }));
    }
  } catch (error) {
    console.error('请求处理错误:', error);
    res.writeHead(500);
    res.end(JSON.stringify({ error: 'Internal Server Error' }));
  }
}

const server = http.createServer((req, res) => {
  handleRequest(req, res).catch(error => {
    console.error('未捕获的错误:', error);
  });
});

const PORT = process.env.PORT || 3000;
server.listen(PORT, () => {
  console.log(`API服务器运行在 http://localhost:${PORT}`);
  console.log('测试命令:');
  console.log(`  curl http://localhost:${PORT}/api/users`);
  console.log(`  curl http://localhost:${PORT}/api/users/1`);
  console.log(`  curl -X POST -H "Content-Type: application/json" -d '{"name":"Charlie","age":35}' http://localhost:${PORT}/api/users`);
});
```

### 错误处理：3个高频问题

#### 问题1：端口被占用

**错误信息**：
```
Error: listen EADDRINUSE: address already in use :::3000
```

**原因**：端口3000已被其他进程占用

**解决方案**：
```bash
# macOS/Linux：查找并结束占用端口的进程
lsof -ti:3000 | xargs kill -9

# Windows：查找并结束进程
netstat -ano | findstr :3000
taskkill /PID <进程ID> /F

# 或者：使用其他端口
const PORT = 3001; // 改为其他端口
```

#### 问题2：响应头已发送后再次设置

**错误信息**：
```
Error: Cannot set headers after they are sent to the client
```

**原因**：调用`res.end()`后又尝试设置响应头或再次发送响应

**错误代码**：
```javascript
// ❌ 错误示例
res.writeHead(200);
res.end('Hello');
res.writeHead(404); // 错误！响应已结束
```

**解决方案**：
```javascript
// ✅ 正确做法：确保只发送一次响应
if (!res.headersSent) {
  res.writeHead(200);
  res.end('Hello');
}

// ✅ 或使用return提前结束
function handleRequest(req, res) {
  if (someCondition) {
    res.writeHead(404);
    res.end('Not Found');
    return; // 提前返回，避免后续代码执行
  }
  
  res.writeHead(200);
  res.end('OK');
}
```

#### 问题3：异步错误未捕获导致服务器崩溃

**错误信息**：
```
(node:12345) UnhandledPromiseRejectionWarning: Error: ...
```

**原因**：async函数中的错误未被捕获

**错误代码**：
```javascript
// ❌ 错误示例
const server = http.createServer(async (req, res) => {
  // 如果fetchData抛出错误，服务器会崩溃
  const data = await fetchData();
  res.end(data);
});
```

**解决方案**：
```javascript
// ✅ 方案1：使用try-catch
const server = http.createServer(async (req, res) => {
  try {
    const data = await fetchData();
    res.end(data);
  } catch (error) {
    console.error('错误:', error);
    res.writeHead(500);
    res.end('Internal Server Error');
  }
});

// ✅ 方案2：使用.catch()
const server = http.createServer((req, res) => {
  handleRequest(req, res).catch(error => {
    console.error('错误:', error);
    if (!res.headersSent) {
      res.writeHead(500);
      res.end('Internal Server Error');
    }
  });
});

// ✅ 方案3：全局错误处理（兜底）
process.on('unhandledRejection', (reason, promise) => {
  console.error('未处理的Promise拒绝:', reason);
  // 在生产环境中，应该记录日志并优雅退出
});
```

## 🎯 应用场景

### 场景1：构建高性能API网关

#### 问题背景
公司有多个微服务（用户服务、订单服务、商品服务），需要一个统一的API网关来：
- 统一鉴权
- 请求转发
- 限流
- 日志记录

使用Express等框架会有额外开销，在高并发场景下（QPS 10万+）性能不够。

#### 技术选型理由
使用原生http模块的原因：
- **极致性能**：框架中间件链有开销，原生http直接处理请求
- **可控性强**：完全自定义请求处理逻辑
- **内存占用小**：无框架依赖，内存占用更低

对比其他方案：
- Express：中间件链开销，QPS约5万
- Fastify：性能较好，QPS约8万
- 原生http：QPS可达12万+

#### 实现代码

```javascript
// gateway.js
import http from 'node:http';
import { URL } from 'node:url';

// 服务映射表
const services = {
  '/api/users': 'http://localhost:4001',
  '/api/orders': 'http://localhost:4002',
  '/api/products': 'http://localhost:4003'
};

// 简单的限流器（令牌桶算法）
class RateLimiter {
  constructor(maxTokens, refillRate) {
    this.maxTokens = maxTokens; // 最大令牌数
    this.tokens = maxTokens;    // 当前令牌数
    this.refillRate = refillRate; // 每秒补充的令牌数
    
    // 定时补充令牌
    setInterval(() => {
      this.tokens = Math.min(this.maxTokens, this.tokens + this.refillRate);
    }, 1000);
  }
  
  // 尝试获取令牌
  tryAcquire() {
    if (this.tokens > 0) {
      this.tokens--;
      return true;
    }
    return false;
  }
}

// 创建限流器：每秒最多100个请求
const limiter = new RateLimiter(100, 100);

// 转发请求到目标服务
async function proxyRequest(targetUrl, req, res) {
  return new Promise((resolve, reject) => {
    const targetURL = new URL(targetUrl);
    
    // 创建到目标服务的请求
    const proxyReq = http.request({
      hostname: targetURL.hostname,
      port: targetURL.port,
      path: targetURL.pathname + req.url.substring(req.url.indexOf('/', 1)),
      method: req.method,
      headers: {
        ...req.headers,
        'X-Forwarded-For': req.socket.remoteAddress,
        'X-Forwarded-Proto': 'http'
      }
    }, (proxyRes) => {
      // 转发响应头
      res.writeHead(proxyRes.statusCode, proxyRes.headers);
      
      // 转发响应体
      proxyRes.pipe(res);
      
      proxyRes.on('end', resolve);
    });
    
    // 转发请求体
    req.pipe(proxyReq);
    
    proxyReq.on('error', (error) => {
      console.error('代理请求失败:', error);
      res.writeHead(502);
      res.end('Bad Gateway');
      reject(error);
    });
  });
}

// 请求处理器
async function handleRequest(req, res) {
  const startTime = Date.now();
  
  try {
    // 1. 限流检查
    if (!limiter.tryAcquire()) {
      res.writeHead(429, { 'Content-Type': 'application/json' });
      res.end(JSON.stringify({ error: 'Too Many Requests' }));
      return;
    }
    
    // 2. 鉴权检查（简化版）
    const token = req.headers.authorization;
    if (!token) {
      res.writeHead(401, { 'Content-Type': 'application/json' });
      res.end(JSON.stringify({ error: 'Unauthorized' }));
      return;
    }
    
    // 3. 路由匹配
    const pathname = new URL(req.url, `http://${req.headers.host}`).pathname;
    let targetService = null;
    
    for (const [prefix, service] of Object.entries(services)) {
      if (pathname.startsWith(prefix)) {
        targetService = service;
        break;
      }
    }
    
    if (!targetService) {
      res.writeHead(404, { 'Content-Type': 'application/json' });
      res.end(JSON.stringify({ error: 'Service Not Found' }));
      return;
    }
    
    // 4. 转发请求
    await proxyRequest(targetService, req, res);
    
  } catch (error) {
    console.error('网关错误:', error);
    if (!res.headersSent) {
      res.writeHead(500);
      res.end('Internal Server Error');
    }
  } finally {
    // 记录请求日志
    const duration = Date.now() - startTime;
    console.log(`[${new Date().toISOString()}] ${req.method} ${req.url} - ${duration}ms`);
  }
}

const server = http.createServer((req, res) => {
  handleRequest(req, res).catch(error => {
    console.error('未捕获的错误:', error);
  });
});

server.listen(3000, () => {
  console.log('API网关运行在 http://localhost:3000');
});
```

### 场景2：静态文件服务器

#### 问题背景
需要一个轻量级的静态文件服务器，用于开发环境提供前端资源（HTML、CSS、JS、图片）。不想安装Nginx或使用http-server等工具，希望自己掌控细节（如缓存策略、文件类型识别）。

#### 技术选型理由
使用原生http模块的原因：
- **学习价值**：理解静态服务器的工作原理
- **可定制性**：完全自定义缓存、压缩、CORS等策略
- **轻量级**：单文件实现，无额外依赖

#### 实现代码

```javascript
// static-server.js
import http from 'node:http';
import fs from 'node:fs/promises';
import path from 'node:path';
import { fileURLToPath } from 'node:url';
import { createReadStream } from 'node:fs';

const __filename = fileURLToPath(import.meta.url);
const __dirname = path.dirname(__filename);

// 静态文件根目录
const PUBLIC_DIR = path.join(__dirname, 'public');

// MIME类型映射
const MIME_TYPES = {
  '.html': 'text/html; charset=utf-8',
  '.css': 'text/css; charset=utf-8',
  '.js': 'application/javascript; charset=utf-8',
  '.json': 'application/json',
  '.png': 'image/png',
  '.jpg': 'image/jpeg',
  '.jpeg': 'image/jpeg',
  '.gif': 'image/gif',
  '.svg': 'image/svg+xml',
  '.ico': 'image/x-icon',
  '.pdf': 'application/pdf',
  '.txt': 'text/plain; charset=utf-8'
};

// 获取MIME类型
function getMimeType(filePath) {
  const ext = path.extname(filePath).toLowerCase();
  return MIME_TYPES[ext] || 'application/octet-stream';
}

// 处理静态文件请求
async function handleStaticFile(req, res) {
  try {
    // 解析请求路径
    let pathname = new URL(req.url, `http://${req.headers.host}`).pathname;
    
    // 默认文件
    if (pathname === '/') {
      pathname = '/index.html';
    }
    
    // 拼接文件路径
    const filePath = path.join(PUBLIC_DIR, pathname);
    
    // ⚠️ 安全检查：防止路径遍历攻击（../)
    if (!filePath.startsWith(PUBLIC_DIR)) {
      res.writeHead(403);
      res.end('Forbidden');
      return;
    }
    
    // 检查文件是否存在
    try {
      const stats = await fs.stat(filePath);
      
      if (stats.isDirectory()) {
        // 如果是目录，尝试返回index.html
        const indexPath = path.join(filePath, 'index.html');
        await fs.access(indexPath);
        return handleStaticFile(
          { ...req, url: pathname + '/index.html' },
          res
        );
      }
      
      // 设置响应头
      const mimeType = getMimeType(filePath);
      const lastModified = stats.mtime.toUTCString();
      
      // 💡 技巧：设置缓存头（1小时）
      res.writeHead(200, {
        'Content-Type': mimeType,
        'Content-Length': stats.size,
        'Last-Modified': lastModified,
        'Cache-Control': 'public, max-age=3600',
        'ETag': `"${stats.size}-${stats.mtime.getTime()}"`
      });
      
      // 💡 技巧：使用流式传输，节省内存
      const stream = createReadStream(filePath);
      stream.pipe(res);
      
    } catch (error) {
      // 文件不存在
      res.writeHead(404, { 'Content-Type': 'text/html; charset=utf-8' });
      res.end('<h1>404 Not Found</h1><p>文件不存在</p>');
    }
    
  } catch (error) {
    console.error('处理静态文件错误:', error);
    res.writeHead(500);
    res.end('Internal Server Error');
  }
}

const server = http.createServer((req, res) => {
  console.log(`[${new Date().toISOString()}] ${req.method} ${req.url}`);
  handleStaticFile(req, res);
});

const PORT = 8080;
server.listen(PORT, () => {
  console.log(`静态文件服务器运行在 http://localhost:${PORT}`);
  console.log(`静态文件目录: ${PUBLIC_DIR}`);
});
```

| **路由功能** | 无（需手写） | 内置 | 需插件 | 内置 |
| **类型安全** | 需手动定义 | 社区类型 | 社区类型 | 原生TS |
| **适用场景** | 学习、极致性能 | 传统Web应用 | 现代化应用 | 高性能API |

**选择建议**：
- 学习HTTP原理 → 原生http
- 快速开发 → Express
- 追求现代化 → Koa
- 追求性能 → Fastify 或 原生http

### 性能优化案例

#### 优化前：同步读取文件

```javascript
// ❌ 性能问题：同步读取文件会阻塞事件循环
import { readFileSync } from 'node:fs';

const server = http.createServer((req, res) => {
  // 同步读取文件（阻塞）
  const data = readFileSync('./large-file.json', 'utf8');
  res.writeHead(200, { 'Content-Type': 'application/json' });
  res.end(data);
});

// 问题分析：
// 1. 阻塞事件循环，其他请求无法处理
// 2. 大文件时内存占用高（整个文件加载到内存）
// 3. QPS很低（单线程被阻塞）
```

#### 优化后：流式传输

```javascript
// ✅ 性能优化：使用流式传输
import { createReadStream } from 'node:fs';

const server = http.createServer((req, res) => {
  // 流式读取文件（非阻塞）
  res.writeHead(200, { 'Content-Type': 'application/json' });
  
  const stream = createReadStream('./large-file.json');
  
  // 💡 技巧：使用pipe()自动处理背压
  stream.pipe(res);
  
  // 错误处理
  stream.on('error', (error) => {
    console.error('文件读取错误:', error);
    if (!res.headersSent) {
      res.writeHead(500);
      res.end('Internal Server Error');
    }
  });
});

// 优化原理：
// 1. 非阻塞：事件循环继续处理其他请求
// 2. 低内存：数据分块传输，不需要全部加载到内存
// 3. 背压处理：pipe()自动处理写入速度慢于读取速度的情况
```

#### 性能指标对比

| 指标 | 优化前（同步） | 优化后（流式） | 提升 |
|------|---------------|---------------|------|
| **QPS** | 100 | 5,000 | **50倍** |
| **内存占用** | 500MB | 50MB | **降低90%** |
| **响应时间（P99）** | 500ms | 50ms | **降低90%** |
| **并发处理能力** | 10 | 1000+ | **100倍** |

## 📋 总结

### 核心要点

1. **http模块是基础**：所有Node.js Web框架的底层都是http模块
2. **事件驱动模型**：请求和响应都是事件和流
3. **性能极致**：直接使用原生API可获得最佳性能

### 实现关键

1. **使用ESM语法**：`import http from 'node:http'`
2. **async/await处理异步**：避免回调地狱
3. **错误处理**：必须捕获所有异步错误
4. **流式处理**：使用Stream处理大文件
5. **安全防护**：验证路径、限流、鉴权

### 应用价值

- **学习阶段**：理解HTTP协议和Web服务器原理
- **性能场景**：API网关、高并发服务
- **定制需求**：完全自定义请求处理逻辑
- **面试加分**：展示对底层原理的理解

### 面试重点

**常见问题**：
1. **http.createServer()的回调函数参数是什么？**
   - `req`（IncomingMessage）和`res`（ServerResponse）
   
2. **如何处理POST请求的body？**
   - 监听`req.on('data')`和`req.on('end')`事件
   
3. **原生http模块和Express的主要区别？**
   - http模块是底层API，Express提供路由、中间件等高级功能
   
4. **如何避免阻塞事件循环？**
   - 使用异步API（fs.promises、Stream），避免同步操作

### 注意事项

⚠️ **错误处理是必须的**：
- async函数必须用try-catch或.catch()捕获错误
- 未捕获的Promise rejection会导致服务器崩溃（Node.js 15+）

⚠️ **响应只能发送一次**：
- 调用`res.end()`后不能再设置响应头或发送数据
- 使用`res.headersSent`检查响应是否已发送
