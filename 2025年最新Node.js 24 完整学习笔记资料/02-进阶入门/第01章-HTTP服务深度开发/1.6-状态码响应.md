# 1.6 HTTP状态码与语义化响应

> 正确使用HTTP状态码，构建符合RESTful规范的API

---

## 📖 知识点概述

### 核心概念

**HTTP状态码**是服务器告诉客户端请求处理结果的三位数字代码。正确使用状态码是构建规范API的基础，也是前后端协作的契约。

### 解决的问题

- **明确请求结果**：成功/失败/重定向/客户端错误/服务器错误
- **错误定位**：快速判断问题出在哪一方
- **缓存策略**：浏览器根据状态码决定缓存行为
- **SEO优化**：搜索引擎根据状态码索引内容

### 主要特点

1. **五大类别**：1xx信息、2xx成功、3xx重定向、4xx客户端错误、5xx服务器错误
2. **语义明确**：每个状态码都有明确的含义
3. **标准化**：遵循HTTP规范，全球通用
4. **可扩展**：自定义业务错误码

### 典型应用场景

- **RESTful API**：CRUD操作返回正确的状态码
- **错误处理**：区分客户端错误和服务器错误
- **缓存控制**：利用304 Not Modified减少传输
- **重定向**：301永久重定向、302临时重定向

## 🚀 使用指南

### 理解：HTTP状态码分类

```
1xx 信息性（Informational）
├─ 100 Continue：继续请求
└─ 101 Switching Protocols：协议切换

2xx 成功（Success）
├─ 200 OK：请求成功
├─ 201 Created：资源已创建
├─ 204 No Content：成功但无返回内容
└─ 206 Partial Content：部分内容

3xx 重定向（Redirection）
├─ 301 Moved Permanently：永久重定向
├─ 302 Found：临时重定向
├─ 304 Not Modified：内容未修改（使用缓存）
└─ 307 Temporary Redirect：临时重定向（保持方法）

4xx 客户端错误（Client Error）
├─ 400 Bad Request：请求格式错误
├─ 401 Unauthorized：未认证
├─ 403 Forbidden：无权限
├─ 404 Not Found：资源不存在
├─ 405 Method Not Allowed：方法不允许
├─ 409 Conflict：资源冲突
├─ 422 Unprocessable Entity：参数验证失败
└─ 429 Too Many Requests：请求过多

5xx 服务器错误（Server Error）
├─ 500 Internal Server Error：服务器内部错误
├─ 502 Bad Gateway：网关错误
├─ 503 Service Unavailable：服务不可用
└─ 504 Gateway Timeout：网关超时
```

### 准备：常用状态码速查

```bash
# 成功类（2xx）
200: 通用成功
201: POST创建资源成功
204: DELETE成功（无返回内容）

# 客户端错误类（4xx）
400: 请求参数错误
401: 需要登录
403: 已登录但无权限
404: 资源不存在
422: 参数验证失败

# 服务器错误类（5xx）
500: 服务器内部错误
503: 服务暂时不可用
```

### 实践：RESTful API状态码实战

#### 1. 完整的CRUD操作

```javascript
// restful-status-codes.js
import http from 'node:http';
import { URL } from 'node:url';

// 模拟数据库
const users = new Map([
  [1, { id: 1, name: 'Alice', email: 'alice@example.com' }],
  [2, { id: 2, name: 'Bob', email: 'bob@example.com' }]
]);

let nextId = 3;

// 工具函数
function sendJSON(res, statusCode, data) {
  res.writeHead(statusCode, { 'Content-Type': 'application/json' });
  res.end(JSON.stringify(data));
}

async function parseBody(req) {
  return new Promise((resolve, reject) => {
    let body = '';
    req.on('data', chunk => { body += chunk; });
    req.on('end', () => {
      try {
        resolve(JSON.parse(body));
      } catch (error) {
        reject(error);
      }
    });
  });
}

const server = http.createServer(async (req, res) => {
  try {
    const url = new URL(req.url, `http://${req.headers.host}`);
    const pathname = url.pathname;
    const method = req.method;
    
    // ===== GET /api/users - 200 OK =====
    if (pathname === '/api/users' && method === 'GET') {
      const userList = Array.from(users.values());
      
      // 💡 200：成功返回数据
      sendJSON(res, 200, {
        data: userList,
        total: userList.length
      });
      return;
    }
    
    // ===== GET /api/users/:id - 200 OK / 404 Not Found =====
    const userMatch = pathname.match(/^\/api\/users\/(\d+)$/);
    if (userMatch && method === 'GET') {
      const id = parseInt(userMatch[1]);
      const user = users.get(id);
      
      if (user) {
        // 💡 200：资源存在
        sendJSON(res, 200, { data: user });
      } else {
        // 💡 404：资源不存在
        sendJSON(res, 404, {
          error: 'Not Found',
          message: `User with id ${id} does not exist`
        });
      }
      return;
    }
    
    // ===== POST /api/users - 201 Created / 400 Bad Request / 422 Unprocessable Entity =====
    if (pathname === '/api/users' && method === 'POST') {
      let body;
      
      try {
        body = await parseBody(req);
      } catch (error) {
        // 💡 400：请求体格式错误（JSON解析失败）
        sendJSON(res, 400, {
          error: 'Bad Request',
          message: 'Invalid JSON'
        });
        return;
      }
      
      // 参数验证
      const errors = [];
      if (!body.name || body.name.trim() === '') {
        errors.push({ field: 'name', message: 'Name is required' });
      }
      if (!body.email || !/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(body.email)) {
        errors.push({ field: 'email', message: 'Valid email is required' });
      }
      
      if (errors.length > 0) {
        // 💡 422：参数验证失败
        sendJSON(res, 422, {
          error: 'Unprocessable Entity',
          message: 'Validation failed',
          errors
        });
        return;
      }
      
      // 检查邮箱是否已存在
      const emailExists = Array.from(users.values()).some(u => u.email === body.email);
      if (emailExists) {
        // 💡 409：资源冲突
        sendJSON(res, 409, {
          error: 'Conflict',
          message: 'Email already exists'
        });
        return;
      }
      
      // 创建用户
      const newUser = {
        id: nextId++,
        name: body.name,
        email: body.email
      };
      users.set(newUser.id, newUser);
      
      // 💡 201：资源已创建
      res.writeHead(201, {
        'Content-Type': 'application/json',
        'Location': `/api/users/${newUser.id}` // 指向新资源的URL
      });
      res.end(JSON.stringify({ data: newUser }));
      return;
    }
    
    // ===== PUT /api/users/:id - 200 OK / 404 Not Found =====
    if (userMatch && method === 'PUT') {
      const id = parseInt(userMatch[1]);
      const user = users.get(id);
      
      if (!user) {
        sendJSON(res, 404, {
          error: 'Not Found',
          message: `User with id ${id} does not exist`
        });
        return;
      }
      
      const body = await parseBody(req);
      
      // 更新用户
      const updatedUser = {
        ...user,
        ...body,
        id // ID不允许修改
      };
      users.set(id, updatedUser);
      
      // 💡 200：更新成功
      sendJSON(res, 200, { data: updatedUser });
      return;
    }
    
    // ===== DELETE /api/users/:id - 204 No Content / 404 Not Found =====
    if (userMatch && method === 'DELETE') {
      const id = parseInt(userMatch[1]);
      
      if (users.has(id)) {
        users.delete(id);
        
        // 💡 204：删除成功，无返回内容
        res.writeHead(204);
        res.end();
      } else {
        sendJSON(res, 404, {
          error: 'Not Found',
          message: `User with id ${id} does not exist`
        });
      }
      return;
    }
    
    // ===== 405 Method Not Allowed =====
    if (pathname.startsWith('/api/users')) {
      // 路径匹配但方法不允许
      res.writeHead(405, {
        'Content-Type': 'application/json',
        'Allow': 'GET, POST, PUT, DELETE' // 告诉客户端允许的方法
      });
      res.end(JSON.stringify({
        error: 'Method Not Allowed',
        message: `Method ${method} is not allowed for ${pathname}`
      }));
      return;
    }
    
    // ===== 404 Not Found =====
    sendJSON(res, 404, {
      error: 'Not Found',
      message: `Endpoint ${pathname} does not exist`
    });
    
  } catch (error) {
    console.error('服务器错误:', error);
    
    // ===== 500 Internal Server Error =====
    if (!res.headersSent) {
      sendJSON(res, 500, {
        error: 'Internal Server Error',
        message: 'An unexpected error occurred'
      });
    }
  }
});

server.listen(3000, () => {
  console.log('RESTful API运行在 http://localhost:3000');
  console.log('\n测试命令:');
  console.log('  200: curl http://localhost:3000/api/users');
  console.log('  201: curl -X POST -H "Content-Type: application/json" -d \'{"name":"Charlie","email":"charlie@example.com"}\' http://localhost:3000/api/users');
  console.log('  204: curl -X DELETE http://localhost:3000/api/users/1');
  console.log('  400: curl -X POST -H "Content-Type: application/json" -d \'invalid json\' http://localhost:3000/api/users');
  console.log('  404: curl http://localhost:3000/api/users/999');
  console.log('  405: curl -X PATCH http://localhost:3000/api/users/1');
  console.log('  422: curl -X POST -H "Content-Type: application/json" -d \'{"name":""}\' http://localhost:3000/api/users');
});
```

#### 2. 认证和权限控制

```javascript
// auth-status-codes.js
import http from 'node:http';

const users = new Map([
  ['user1', { username: 'user1', password: 'pass1', role: 'user' }],
  ['admin1', { username: 'admin1', password: 'admin_pass', role: 'admin' }]
]);

// 简化的token验证（实际应用应使用JWT）
const sessions = new Map();

function generateToken() {
  return Math.random().toString(36).substring(2);
}

const server = http.createServer(async (req, res) => {
  const url = new URL(req.url, `http://${req.headers.host}`);
  const pathname = url.pathname;
  
  // ===== POST /api/login - 200 OK / 401 Unauthorized =====
  if (pathname === '/api/login' && req.method === 'POST') {
    let body = '';
    req.on('data', chunk => { body += chunk; });
    req.on('end', () => {
      const { username, password } = JSON.parse(body);
      const user = users.get(username);
      
      if (!user || user.password !== password) {
        // 💡 401：认证失败（用户名或密码错误）
        res.writeHead(401, { 'Content-Type': 'application/json' });
        res.end(JSON.stringify({
          error: 'Unauthorized',
          message: 'Invalid username or password'
        }));
        return;
      }
      
      // 生成token
      const token = generateToken();
      sessions.set(token, { username, role: user.role });
      
      // 💡 200：登录成功
      res.writeHead(200, { 'Content-Type': 'application/json' });
      res.end(JSON.stringify({
        token,
        user: { username: user.username, role: user.role }
      }));
    });
    return;
  }
  
  // ===== GET /api/profile - 200 OK / 401 Unauthorized =====
  if (pathname === '/api/profile' && req.method === 'GET') {
    const token = req.headers.authorization?.replace('Bearer ', '');
    
    if (!token || !sessions.has(token)) {
      // 💡 401：未认证（token缺失或无效）
      res.writeHead(401, {
        'Content-Type': 'application/json',
        'WWW-Authenticate': 'Bearer realm="api"' // 告诉客户端需要Bearer认证
      });
      res.end(JSON.stringify({
        error: 'Unauthorized',
        message: 'Authentication required'
      }));
      return;
    }
    
    const session = sessions.get(token);
    
    // 💡 200：成功返回用户信息
    res.writeHead(200, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({
      data: session
    }));
    return;
  }
  
  // ===== DELETE /api/users/:username - 200 OK / 403 Forbidden =====
  if (pathname.startsWith('/api/admin/') && req.method === 'DELETE') {
    const token = req.headers.authorization?.replace('Bearer ', '');
    
    if (!token || !sessions.has(token)) {
      // 401：未认证
      res.writeHead(401, { 'Content-Type': 'application/json' });
      res.end(JSON.stringify({
        error: 'Unauthorized',
        message: 'Authentication required'
      }));
      return;
    }
    
    const session = sessions.get(token);
    
    if (session.role !== 'admin') {
      // 💡 403：已认证但无权限
      res.writeHead(403, { 'Content-Type': 'application/json' });
      res.end(JSON.stringify({
        error: 'Forbidden',
        message: 'Admin role required'
      }));
      return;
    }
    
    // 执行删除操作...
    res.writeHead(200, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({ message: 'User deleted' }));
    return;
  }
  
  res.writeHead(404, { 'Content-Type': 'application/json' });
  res.end(JSON.stringify({ error: 'Not Found' }));
});

server.listen(3000, () => {
  console.log('认证API运行在 http://localhost:3000');
  console.log('\n测试流程:');
  console.log('1. 登录获取token:');
  console.log('   curl -X POST -H "Content-Type: application/json" -d \'{"username":"user1","password":"pass1"}\' http://localhost:3000/api/login');
  console.log('2. 使用token访问profile（将<TOKEN>替换为实际token）:');
  console.log('   curl -H "Authorization: Bearer <TOKEN>" http://localhost:3000/api/profile');
  console.log('3. 普通用户访问admin接口（403）:');
  console.log('   curl -H "Authorization: Bearer <TOKEN>" -X DELETE http://localhost:3000/api/admin/users/someone');
});
```

### 错误处理：3个高频问题

#### 问题1：混淆4xx和5xx错误

**错误做法**：
```javascript
// ❌ 错误：参数错误返回500
if (!req.body.name) {
  res.writeHead(500);
  res.end('Name is required');
}
```

**正确做法**：
```javascript
// ✅ 正确：参数错误返回4xx
if (!req.body.name) {
  res.writeHead(400); // 或422
  res.end(JSON.stringify({
    error: 'Bad Request',
    message: 'Name is required'
  }));
}

// 规则：
// 4xx = 客户端的问题（参数错误、权限不足等）
// 5xx = 服务器的问题（代码错误、数据库连接失败等）
```

#### 问题2：资源创建返回200而非201

**错误做法**：
```javascript
// ❌ 错误：创建资源返回200
app.post('/api/users', (req, res) => {
  const user = createUser(req.body);
  res.writeHead(200); // 应该是201
  res.end(JSON.stringify({ data: user }));
});
```

**正确做法**：
```javascript
// ✅ 正确：创建资源返回201并包含Location头
app.post('/api/users', (req, res) => {
  const user = createUser(req.body);
  res.writeHead(201, {
    'Content-Type': 'application/json',
    'Location': `/api/users/${user.id}` // 指向新资源
  });
  res.end(JSON.stringify({ data: user }));
});
```

#### 问题3：删除不存在的资源返回204

**错误做法**：
```javascript
// ❌ 错误：资源不存在也返回204
app.delete('/api/users/:id', (req, res) => {
  // 没有检查资源是否存在
  res.writeHead(204);
  res.end();
});
```

**正确做法**：
```javascript
// ✅ 正确：先检查资源是否存在
app.delete('/api/users/:id', (req, res) => {
  const user = users.get(req.params.id);
  
  if (!user) {
    res.writeHead(404);
    res.end(JSON.stringify({
      error: 'Not Found',
      message: 'User does not exist'
    }));
    return;
  }
  
  users.delete(req.params.id);
  res.writeHead(204);
  res.end();
});
```

## 🎯 应用场景

### 场景1：RESTful API规范设计

#### 问题背景
前后端团队需要一套统一的API规范，明确：
- 什么情况返回什么状态码
- 错误信息的格式
- 成功响应的格式

#### 技术选型理由
制定明确的状态码使用规范能够：
- 减少沟通成本
- 前端可以根据状态码自动处理错误
- 易于API文档生成

#### 规范示例

```javascript
// api-standard.js
// API响应规范

class APIResponse {
  // 成功响应
  static success(res, data, statusCode = 200) {
    res.writeHead(statusCode, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({
      success: true,
      data
    }));
  }
  
  // 创建成功
  static created(res, data, location) {
    res.writeHead(201, {
      'Content-Type': 'application/json',
      'Location': location
    });
    res.end(JSON.stringify({
      success: true,
      data
    }));
  }
  
  // 删除成功
  static noContent(res) {
    res.writeHead(204);
    res.end();
  }
  
  // 客户端错误
  static clientError(res, message, statusCode = 400, details = null) {
    res.writeHead(statusCode, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({
      success: false,
      error: {
        type: 'client_error',
        message,
        details
      }
    }));
  }
  
  // 未认证
  static unauthorized(res, message = 'Authentication required') {
    res.writeHead(401, {
      'Content-Type': 'application/json',
      'WWW-Authenticate': 'Bearer realm="api"'
    });
    res.end(JSON.stringify({
      success: false,
      error: {
        type: 'unauthorized',
        message
      }
    }));
  }
  
  // 无权限
  static forbidden(res, message = 'Permission denied') {
    res.writeHead(403, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({
      success: false,
      error: {
        type: 'forbidden',
        message
      }
    }));
  }
  
  // 资源不存在
  static notFound(res, resource = 'Resource') {
    res.writeHead(404, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({
      success: false,
      error: {
        type: 'not_found',
        message: `${resource} not found`
      }
    }));
  }
  
  // 参数验证失败
  static validationError(res, errors) {
    res.writeHead(422, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({
      success: false,
      error: {
        type: 'validation_error',
        message: 'Validation failed',
        errors
      }
    }));
  }
  
  // 服务器错误
  static serverError(res, message = 'Internal server error') {
    res.writeHead(500, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({
      success: false,
      error: {
        type: 'server_error',
        message
      }
    }));
  }
}

// 使用示例
const server = http.createServer((req, res) => {
  // 成功
  APIResponse.success(res, { users: [] });
  
  // 创建成功
  APIResponse.created(res, { id: 1 }, '/api/users/1');
  
  // 参数错误
  APIResponse.validationError(res, [
    { field: 'email', message: 'Invalid email' }
  ]);
  
  // 资源不存在
  APIResponse.notFound(res, 'User');
  
  // 服务器错误
  APIResponse.serverError(res);
});
```

#### 可量化效果
- **开发效率**：前端错误处理统一，减少50%沟通时间
- **Bug率**：状态码使用规范，减少30%错误
- **可维护性**：新人快速理解API规范

### 场景2：缓存优化（304 Not Modified）

#### 问题背景
静态资源（图片、CSS、JS）每次请求都返回完整内容，浪费带宽。

#### 实现代码

```javascript
// cache-server.js
import http from 'node:http';
import fs from 'node:fs';
import crypto from 'node:crypto';

function getFileETag(filePath) {
  const content = fs.readFileSync(filePath);
  return crypto.createHash('md5').update(content).digest('hex');
}

const server = http.createServer((req, res) => {
  const filePath = './static/image.png';
  
  // 计算文件的ETag
  const etag = getFileETag(filePath);
  
  // 检查客户端的If-None-Match头
  const clientETag = req.headers['if-none-match'];
  
  if (clientETag === etag) {
    // 💡 304：文件未修改，使用缓存
    res.writeHead(304);
    res.end();
    console.log('304 Not Modified - 使用缓存');
  } else {
    // 文件已修改或首次请求，返回完整内容
    const content = fs.readFileSync(filePath);
    
    res.writeHead(200, {
      'Content-Type': 'image/png',
      'ETag': etag,
      'Cache-Control': 'max-age=3600' // 1小时缓存
    });
    res.end(content);
    console.log('200 OK - 返回完整内容');
  }
});

server.listen(3000);
```

#### 可量化效果
- **带宽节省**：304响应仅几百字节，节省99%带宽
- **加载速度**：从300ms降到10ms
- **服务器负载**：减少80%（无需读取文件）

## 🔍 深度解析

### 对比分析：常见状态码选择

| 场景 | 错误做法 | 正确做法 | 原因 |
|------|---------|---------|------|
| 创建资源 | 200 | 201 | 201明确表示资源已创建 |
| 删除成功 | 200 | 204 | 204表示无返回内容 |
| 参数错误 | 500 | 400/422 | 参数错误是客户端问题 |
| 认证失败 | 403 | 401 | 401表示需要认证，403表示已认证但无权限 |
| 资源冲突 | 400 | 409 | 409明确表示资源冲突（如邮箱已存在）|

### 性能优化案例

#### 优化前：每次返回完整内容

```javascript
// ❌ 性能问题：不使用缓存
app.get('/api/config', (req, res) => {
  const config = fs.readFileSync('./config.json');
  res.writeHead(200);
  res.end(config);
});

// 问题：
// 1. 每次都读取文件
// 2. 每次都传输完整内容
// 3. 带宽浪费
```

#### 优化后：使用ETag缓存

```javascript
// ✅ 性能优化：使用ETag和304
let configCache = null;
let configETag = null;

app.get('/api/config', (req, res) => {
  // 首次或文件变化时重新加载
  const stats = fs.statSync('./config.json');
  const newETag = `"${stats.mtime.getTime()}"`;
  
  if (configETag !== newETag) {
    configCache = fs.readFileSync('./config.json');
    configETag = newETag;
  }
  
  // 检查客户端缓存
  if (req.headers['if-none-match'] === configETag) {
    res.writeHead(304);
    res.end();
    return;
  }
  
  res.writeHead(200, {
    'Content-Type': 'application/json',
    'ETag': configETag,
    'Cache-Control': 'max-age=60'
  });
  res.end(configCache);
});
```

## 📋 总结

### 核心要点

1. **状态码五大类**：1xx信息、2xx成功、3xx重定向、4xx客户端错误、5xx服务器错误
2. **语义明确**：每个状态码都有明确含义，不可随意使用
3. **RESTful规范**：遵循REST规范使用状态码

### 实现关键

1. **200 OK**：通用成功
2. **201 Created**：资源创建成功 + Location头
3. **204 No Content**：删除成功
4. **400 Bad Request**：请求格式错误
5. **401 Unauthorized**：需要认证
6. **403 Forbidden**：已认证但无权限
7. **404 Not Found**：资源不存在
8. **422 Unprocessable Entity**：参数验证失败
9. **500 Internal Server Error**：服务器错误

### 应用价值

- **规范API**：前后端协作契约
- **错误定位**：快速判断问题方
- **性能优化**：利用304减少传输
- **用户体验**：明确的错误提示

### 面试重点

**常见问题**：
1. **401和403的区别？**
   - 401：未认证（需要登录）
   - 403：已认证但无权限

2. **200和201的区别？**
   - 200：通用成功
   - 201：资源创建成功，应包含Location头

3. **什么时候返回204？**
   - DELETE成功且无返回内容时

4. **400和422的区别？**
   - 400：请求格式错误（如JSON解析失败）
   - 422：请求格式正确但参数验证失败

### 注意事项

⚠️ **4xx vs 5xx的区分原则**：
```
4xx（客户端错误）：
- 客户端的问题
- 重试相同请求不会成功
- 需要客户端修改请求

5xx（服务器错误）：
- 服务器的问题
- 重试可能成功
- 服务器需要修复
```

⚠️ **201必须包含Location头**：
```javascript
res.writeHead(201, {
  'Location': '/api/users/123'
});
```
