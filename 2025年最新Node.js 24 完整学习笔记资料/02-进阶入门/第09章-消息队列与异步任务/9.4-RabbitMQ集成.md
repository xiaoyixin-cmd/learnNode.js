## 📖 知识点概述

### 核心概念

**RabbitMQ**是使用Erlang编写的开源消息代理（Message Broker），实现了AMQP（Advanced Message Queuing Protocol）协议。它提供了比Redis更强大的消息路由、可靠性保证和集群支持。

### RabbitMQ架构

```
┌────────────────────────────────────────────────────────────┐
│                    RabbitMQ 架构                            │
├────────────────────────────────────────────────────────────┤
│                                                              │
│  Producer（生产者）                                          │
│      │                                                       │
│      │ 1. 发送消息                                          │
│      ▼                                                       │
│  Exchange（交换机）← 路由规则                                │
│      │                                                       │
│      │ 2. 根据 Routing Key 路由                             │
│      ▼                                                       │
│  Queue（队列）← 存储消息                                     │
│      │                                                       │
│      │ 3. 推送消息                                          │
│      ▼                                                       │
│  Consumer（消费者）                                          │
│      │                                                       │
│      │ 4. ACK/NACK 确认                                     │
│      ▼                                                       │
│  RabbitMQ Server                                            │
│                                                              │
└────────────────────────────────────────────────────────────┘
```

### 核心组件

| 组件 | 作用 | 类比 |
|------|------|------|
| **Producer** | 生产者，发送消息 | 寄件人 |
| **Exchange** | 交换机，路由消息 | 邮局分拣中心 |
| **Queue** | 队列，存储消息 | 收件箱 |
| **Consumer** | 消费者，接收消息 | 收件人 |
| **Binding** | 绑定，Exchange与Queue的关系 | 投递规则 |
| **Routing Key** | 路由键，消息的标签 | 邮编/地址 |
| **Virtual Host** | 虚拟主机，多租户隔离 | 独立邮局 |

### 四种Exchange类型

```
1. Direct（直连）
   Producer → [Exchange] → Queue(exact match)
   用途：点对点，精确匹配
   
2. Fanout（扇出）
   Producer → [Exchange] → Queue1
                        → Queue2
                        → Queue3
   用途：广播，发布订阅
   
3. Topic（主题）
   Producer → [Exchange] → Queue1 (logs.error)
                        → Queue2 (logs.*)
                        → Queue3 (*.error)
   用途：模式匹配，灵活路由
   
4. Headers（头部）
   Producer → [Exchange] → Queue (x-match: all/any)
   用途：基于消息头属性匹配（较少使用）
```

### RabbitMQ vs Redis (Bull)

| 维度 | RabbitMQ | Redis (Bull) |
|------|----------|-------------|
| **协议** | AMQP | Redis Protocol |
| **可靠性** | 极高（持久化、集群） | 中（依赖Redis持久化） |
| **路由能力** | 强大（4种Exchange） | 弱（单一队列） |
| **性能** | 5,000-10,000 msg/s | 10,000+ msg/s |
| **消息大小** | 128MB+ | 512MB（受Redis限制） |
| **学习曲线** | 中 | 低 |
| **部署复杂度** | 中 | 低 |
| **适用场景** | 微服务、复杂路由 | 后台任务、简单队列 |

## 🚀 使用指南

### 理解：为什么需要RabbitMQ？

#### Redis队列的局限

```javascript
// ❌ Redis队列：无法实现复杂路由
const errorQueue = new Queue('errors');
const warningQueue = new Queue('warnings');
const infoQueue = new Queue('info');

// 生产者需要判断日志级别，手动选择队列
if (log.level === 'error') {
  await errorQueue.add(log);
} else if (log.level === 'warning') {
  await warningQueue.add(log);
} else {
  await infoQueue.add(log);
}

// 问题：
// 1. 生产者需要知道所有队列
// 2. 无法动态添加新队列
// 3. 无法实现"所有日志"+"仅错误日志"的双重订阅
```

#### RabbitMQ的优势

```javascript
// ✅ RabbitMQ：使用Topic Exchange
// 生产者只需发送到Exchange，无需关心队列
await channel.publish('logs', `logs.${level}`, Buffer.from(JSON.stringify(log)));

// 消费者1：订阅所有日志
await channel.bindQueue('all-logs-queue', 'logs', 'logs.*');

// 消费者2：只订阅错误日志
await channel.bindQueue('error-logs-queue', 'logs', 'logs.error');

// 消费者3：订阅错误和警告
await channel.bindQueue('important-logs-queue', 'logs', 'logs.error');
await channel.bindQueue('important-logs-queue', 'logs', 'logs.warning');

// 优点：
// 1. 生产者与消费者解耦
// 2. 灵活的路由规则
// 3. 动态添加消费者无需修改生产者
```

#### 第一步：基础连接与队列

```javascript
// rabbitmq-basic.js
import amqp from 'amqplib';

// 💡 连接RabbitMQ
async function connectRabbitMQ() {
  try {
    // 连接到RabbitMQ服务器
    const connection = await amqp.connect({
      protocol: 'amqp',
      hostname: process.env.RABBITMQ_HOST || 'localhost',
      port: process.env.RABBITMQ_PORT || 5672,
      username: process.env.RABBITMQ_USER || 'guest',
      password: process.env.RABBITMQ_PASS || 'guest',
      vhost: '/',
      // 💡 心跳检测（秒）
      heartbeat: 60,
      // 💡 连接超时（毫秒）
      connectionTimeout: 30000
    });
    
    console.log('✅ RabbitMQ连接成功');
    
    // 💡 监听连接事件
    connection.on('error', (err) => {
      console.error('❌ RabbitMQ连接错误:', err);
    });
    
    connection.on('close', () => {
      console.log('🔌 RabbitMQ连接关闭');
    });
    
    // 💡 创建通道（Channel）
    const channel = await connection.createChannel();
    
    console.log('✅ 通道创建成功');
    
    return { connection, channel };
  } catch (error) {
    console.error('❌ RabbitMQ连接失败:', error);
    throw error;
  }
}

// 💡 简单队列模式（Work Queue）
async function workQueueExample() {
  const { connection, channel } = await connectRabbitMQ();
  
  const queueName = 'task_queue';
  
  // 💡 声明队列
  await channel.assertQueue(queueName, {
    durable: true // 持久化队列（服务器重启后仍存在）
  });
  
  // 生产者：发送任务
  for (let i = 1; i <= 10; i++) {
    const message = `Task ${i}`;
    
    channel.sendToQueue(queueName, Buffer.from(message), {
      persistent: true // 持久化消息（服务器重启后仍存在）
    });
    
    console.log(`📤 发送任务: ${message}`);
  }
  
  // 消费者：处理任务
  await channel.consume(queueName, async (msg) => {
    if (msg) {
      const task = msg.content.toString();
      console.log(`📥 接收任务: ${task}`);
      
      // 模拟处理
      await new Promise(resolve => setTimeout(resolve, 1000));
      
      console.log(`✅ 完成任务: ${task}`);
      
      // 💡 手动确认（ACK）
      channel.ack(msg);
    }
  }, {
    noAck: false // 手动确认模式
  });
  
  // 💡 预取数量（每次只处理1个任务）
  await channel.prefetch(1);
}

// 运行
await workQueueExample();
```

#### 第二步：Direct Exchange（直连交换机）

```javascript
// direct-exchange.js
import amqp from 'amqplib';

async function directExchangeExample() {
  const { connection, channel } = await connectRabbitMQ();
  
  const exchangeName = 'direct_logs';
  
  // 💡 声明Direct Exchange
  await channel.assertExchange(exchangeName, 'direct', {
    durable: true
  });
  
  // 💡 生产者：发送不同级别的日志
  async function sendLog(level, message) {
    const log = {
      level,
      message,
      timestamp: Date.now()
    };
    
    // 发送到Exchange，routing key = level
    channel.publish(
      exchangeName,
      level, // routing key
      Buffer.from(JSON.stringify(log))
    );
    
    console.log(`📤 发送日志 [${level}]: ${message}`);
  }
  
  // 💡 消费者1：只接收error日志
  async function consumeErrorLogs() {
    const queueName = 'error_logs';
    
    // 声明队列
    await channel.assertQueue(queueName, { durable: true });
    
    // 💡 绑定队列到Exchange（routing key = 'error'）
    await channel.bindQueue(queueName, exchangeName, 'error');
    
    // 消费消息
    channel.consume(queueName, (msg) => {
      if (msg) {
        const log = JSON.parse(msg.content.toString());
        console.log(`❌ 错误日志: ${log.message}`);
        channel.ack(msg);
      }
    });
    
    console.log('👂 监听错误日志...');
  }
  
  // 💡 消费者2：接收所有日志
  async function consumeAllLogs() {
    const queueName = 'all_logs';
    
    await channel.assertQueue(queueName, { durable: true });
    
    // 绑定多个routing key
    await channel.bindQueue(queueName, exchangeName, 'error');
    await channel.bindQueue(queueName, exchangeName, 'warning');
    await channel.bindQueue(queueName, exchangeName, 'info');
    
    channel.consume(queueName, (msg) => {
      if (msg) {
        const log = JSON.parse(msg.content.toString());
        console.log(`📝 日志 [${log.level}]: ${log.message}`);
        channel.ack(msg);
      }
    });
    
    console.log('👂 监听所有日志...');
  }
  
  // 启动消费者
  await consumeErrorLogs();
  await consumeAllLogs();
  
  // 发送日志
  await sendLog('info', '用户登录成功');
  await sendLog('warning', '磁盘使用率80%');
  await sendLog('error', '数据库连接失败');
  
  // 输出：
  // 消费者1只收到：error
  // 消费者2收到：info, warning, error
}

await directExchangeExample();
```

import amqp from 'amqplib';

async function topicExchangeExample() {
  const { connection, channel } = await connectRabbitMQ();
  
  const exchangeName = 'topic_logs';
  
  // 💡 声明Topic Exchange
  await channel.assertExchange(exchangeName, 'topic', {
    durable: true
  });
  
  // 💡 生产者：发送带层级的日志
  async function sendLog(routingKey, message) {
    const log = {
      routingKey,
      message,
      timestamp: Date.now()
    };
    
    channel.publish(
      exchangeName,
      routingKey, // 如：'user.login.success'
      Buffer.from(JSON.stringify(log))
    );
    
    console.log(`📤 发送日志 [${routingKey}]: ${message}`);
  }
  
  // 💡 消费者1：订阅所有user相关日志
  async function consumeUserLogs() {
    const queueName = 'user_logs';
    
    await channel.assertQueue(queueName, { durable: true });
    
    // 💡 通配符：* 匹配一个单词，# 匹配零个或多个单词
    await channel.bindQueue(queueName, exchangeName, 'user.*');
    
    channel.consume(queueName, (msg) => {
      if (msg) {
        const log = JSON.parse(msg.content.toString());
        console.log(`👤 用户日志 [${log.routingKey}]: ${log.message}`);
        channel.ack(msg);
      }
    });
    
    console.log('👂 监听 user.* ...');
  }
  
  // 💡 消费者2：订阅所有error级别日志
  async function consumeAllErrors() {
    const queueName = 'error_logs';
    
    await channel.assertQueue(queueName, { durable: true });
    
    // 💡 匹配所有以.error结尾的routing key
    await channel.bindQueue(queueName, exchangeName, '*.error');
    
    channel.consume(queueName, (msg) => {
      if (msg) {
        const log = JSON.parse(msg.content.toString());
        console.log(`❌ 错误 [${log.routingKey}]: ${log.message}`);
        channel.ack(msg);
      }
    });
    
    console.log('👂 监听 *.error ...');
  }
  
  // 💡 消费者3：订阅system模块的所有日志
  async function consumeSystemLogs() {
    const queueName = 'system_logs';
    
    await channel.assertQueue(queueName, { durable: true });
    
    // 💡 匹配system开头的所有层级
    await channel.bindQueue(queueName, exchangeName, 'system.#');
    
    channel.consume(queueName, (msg) => {
      if (msg) {
        const log = JSON.parse(msg.content.toString());
        console.log(`🖥️ 系统日志 [${log.routingKey}]: ${log.message}`);
        channel.ack(msg);
      }
    });
    
    console.log('👂 监听 system.# ...');
  }
  
  // 启动消费者
  await consumeUserLogs();
  await consumeAllErrors();
  await consumeSystemLogs();
  
  // 发送日志
  await sendLog('user.login', '用户登录');          // 消费者1
  await sendLog('user.error', '用户认证失败');       // 消费者1, 2
  await sendLog('system.error', '磁盘空间不足');     // 消费者2, 3
  await sendLog('system.cpu.high', 'CPU使用率90%'); // 消费者3
  
  // Topic Exchange通配符规则：
  // * 匹配一个单词：user.* 匹配 user.login, user.logout
  // # 匹配零个或多个单词：system.# 匹配 system, system.cpu, system.cpu.high
}

await topicExchangeExample();

#### 第五步：消息确认与可靠性

```javascript
// message-reliability.js
import amqp from 'amqplib';

async function reliableMessaging() {
  const { connection, channel } = await connectRabbitMQ();
  
  const exchangeName = 'reliable_exchange';
  const queueName = 'reliable_queue';
  
  // 💡 启用发布确认模式
  await channel.confirmSelect();
  
  // 声明持久化Exchange和Queue
  await channel.assertExchange(exchangeName, 'direct', {
    durable: true // 持久化
  });
  
  await channel.assertQueue(queueName, {
    durable: true, // 持久化队列
    arguments: {
      'x-message-ttl': 60000, // 消息TTL（60秒）
      'x-max-length': 10000,  // 最大消息数
      'x-dead-letter-exchange': 'dlx', // 死信交换机
      'x-dead-letter-routing-key': 'dead'
    }
  });
  
  await channel.bindQueue(queueName, exchangeName, 'task');
  
  // 💡 生产者：可靠发送
  async function sendReliableMessage(message) {
    return new Promise((resolve, reject) => {
      channel.publish(
        exchangeName,
        'task',
        Buffer.from(JSON.stringify(message)),
        {
          persistent: true, // 持久化消息
          contentType: 'application/json',
          timestamp: Date.now(),
          messageId: crypto.randomUUID()
        },
        (err, ok) => {
          // 💡 发布确认回调
          if (err) {
            console.error('❌ 消息发送失败:', err);
            reject(err);
          } else {
            console.log('✅ 消息已确认');
            resolve(ok);
          }
        }
      );
    });
  }
  
  // 💡 消费者：可靠消费
  async function consumeReliably() {
    // 设置预取数量
    await channel.prefetch(10);
    
    channel.consume(queueName, async (msg) => {
      if (!msg) return;
      
      try {
        const data = JSON.parse(msg.content.toString());
        console.log(`📥 处理消息: ${data.message}`);
        
        // 模拟处理
        await processMessage(data);
        
        // 💡 成功：ACK确认
        channel.ack(msg);
        console.log('✅ 消息处理成功');
      } catch (error) {
        console.error('❌ 消息处理失败:', error);
        
        // 💡 失败：NACK并重新入队
        const requeue = msg.fields.redelivered === false; // 第一次失败才重新入队
        
        channel.nack(msg, false, requeue);
        
        if (!requeue) {
          console.log('💀 消息进入死信队列');
        }
      }
    }, {
      noAck: false // 手动确认
    });
  }
  
  // 💡 死信队列处理
  async function setupDLX() {
    await channel.assertExchange('dlx', 'direct', { durable: true });
    await channel.assertQueue('dead_letter_queue', { durable: true });
    await channel.bindQueue('dead_letter_queue', 'dlx', 'dead');
    
    channel.consume('dead_letter_queue', (msg) => {
      if (msg) {
        console.log('💀 死信消息:', msg.content.toString());
        
        // 记录到数据库或告警
        // ...
        
        channel.ack(msg);
      }
    });
  }
  
  await setupDLX();
  await consumeReliably();
  
  // 发送消息
  await sendReliableMessage({ message: 'Task 1', userId: 123 });
}

await reliableMessaging();
```

## 🔧 错误处理

### 高频错误1：连接拒绝

**现象：**
```
Error: connect ECONNREFUSED 127.0.0.1:5672
```

**原因：**
RabbitMQ服务未启动

**解决方案：**

```javascript
// ✅ 检查RabbitMQ服务状态
// Docker
docker ps | grep rabbitmq

// 本地服务
// Mac/Linux: systemctl status rabbitmq-server
// Windows: 服务管理器查看

// ✅ 健康检查
async function checkRabbitMQ() {
  try {
    const connection = await amqp.connect('amqp://localhost');
    console.log('✅ RabbitMQ连接成功');
    await connection.close();
    return true;
  } catch (error) {
    console.error('❌ RabbitMQ连接失败:', error.message);
    return false;
  }
}

// 启动前检查
if (!(await checkRabbitMQ())) {
  console.error('请先启动RabbitMQ服务');
  process.exit(1);
}

// ✅ 自动重连
async function connectWithRetry(maxRetries = 10) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      const connection = await amqp.connect('amqp://localhost');
      console.log('✅ RabbitMQ连接成功');
      return connection;
    } catch (error) {
      console.log(`⏳ 连接失败，${3000}ms后重试... (${i + 1}/${maxRetries})`);
      await new Promise(resolve => setTimeout(resolve, 3000));
    }
  }
  
  throw new Error('RabbitMQ连接失败，已达最大重试次数');
}
```

### 高频错误2：消息丢失

**现象：**
消息发送成功，但服务器重启后消息丢失

**原因：**
未开启消息持久化

**解决方案：**

```javascript
// ❌ 非持久化（重启丢失）
await channel.assertExchange('temp_exchange', 'direct');
await channel.assertQueue('temp_queue');
channel.sendToQueue('temp_queue', Buffer.from('message'));

// ✅ 完整持久化方案
// 1. 持久化Exchange
await channel.assertExchange('persistent_exchange', 'direct', {
  durable: true // ✅ Exchange持久化
});

// 2. 持久化Queue
await channel.assertQueue('persistent_queue', {
  durable: true // ✅ Queue持久化
});

// 3. 持久化消息
channel.publish(
  'persistent_exchange',
  'routing_key',
  Buffer.from('message'),
  {
    persistent: true // ✅ 消息持久化
  }
);

// 4. 启用发布确认
await channel.confirmSelect();

channel.publish(
  'persistent_exchange',
  'routing_key',
  Buffer.from('message'),
  { persistent: true },
  (err) => {
    if (err) {
      console.error('❌ 消息未持久化');
    } else {
      console.log('✅ 消息已持久化到磁盘');
    }
  }
);
```

### 高频错误3：消息堆积

**现象：**
队列消息数持续增长，消费速度慢于生产速度

**原因：**
1. 消费者处理太慢
2. 消费者数量不足
3. 预取数量设置不当

**解决方案：**

```javascript
// ✅ 方案1：增加消费者并发
await channel.prefetch(10); // 每个消费者同时处理10条

// 启动多个消费者进程
for (let i = 0; i < 5; i++) {
  channel.consume(queueName, async (msg) => {
    // 处理消息
  });
}

// ✅ 方案2：优化处理逻辑
channel.consume(queueName, async (msg) => {
  const startTime = Date.now();
  
  // 异步并行处理
  await Promise.all([
    task1(msg),
    task2(msg),
    task3(msg)
  ]);
  
  const duration = Date.now() - startTime;
  console.log(`⏱️ 处理耗时: ${duration}ms`);
  
  channel.ack(msg);
});

// ✅ 方案3：消息TTL（防止无限堆积）
await channel.assertQueue(queueName, {
  durable: true,
  arguments: {
    'x-message-ttl': 3600000, // 1小时后过期
    'x-max-length': 100000     // 最多10万条消息
  }
});

// ✅ 方案4：监控队列深度
async function monitorQueue() {
  setInterval(async () => {
    const queue = await channel.checkQueue(queueName);
    console.log(`📊 队列消息数: ${queue.messageCount}`);
    
    if (queue.messageCount > 10000) {
      console.warn('⚠️ 队列堆积告警！');
      // 发送告警、自动扩容等
    }
  }, 60000); // 每分钟检查
}
```

## 📋 总结

### 核心要点

1. **Exchange类型**：Direct（精确）、Fanout（广播）、Topic（模式）、Headers（属性）
2. **消息持久化**：Exchange + Queue + Message三者都要持久化
3. **消息确认**：ACK/NACK机制，防止消息丢失
4. **预取数量**：prefetch控制消费者并发

### 实现关键

1. **连接管理**：使用连接池，监听close/error事件
2. **发布确认**：confirmSelect()启用，回调确认成功
3. **死信队列**：处理失败消息，防止丢失
4. **监控告警**：队列深度、消费速率

### 应用价值

- **复杂路由**：Topic Exchange灵活匹配
- **解耦服务**：生产者消费者独立
- **高可靠**：持久化+确认机制
- **水平扩展**：增加消费者节点

### 面试重点

**Q1：RabbitMQ的四种Exchange有什么区别？**
答：Direct精确匹配routing key；Fanout广播所有队列；Topic支持通配符（*匹配一个词，#匹配多个词）；Headers基于消息属性匹配。

**Q2：如何保证RabbitMQ消息不丢失？**
答：1) Exchange/Queue/Message三者持久化 2) 启用发布确认 3) 消费者手动ACK 4) 使用死信队列。

**Q3：RabbitMQ vs Redis队列如何选择？**
答：简单后台任务用Redis（Bull），复杂路由、微服务用RabbitMQ。Redis性能更高但路由能力弱，RabbitMQ可靠性更高但部署复杂。

**Q4：prefetch的作用是什么？**
答：限制消费者同时处理的消息数量，防止消费者过载，实现负载均衡。

### 注意事项

⚠️ **持久化三要素**：Exchange、Queue、Message都要设置
⚠️ **手动ACK**：消费成功才ACK，否则消息重新入队
⚠️ **预取数量**：根据处理耗时设置，避免过大或过小
⚠️ **连接复用**：创建连接昂贵，应复用连接创建多个Channel
