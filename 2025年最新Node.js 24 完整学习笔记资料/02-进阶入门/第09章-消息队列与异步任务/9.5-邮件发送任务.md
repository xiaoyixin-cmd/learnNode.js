# 9.5 邮件发送任务（nodemailer）

> Nodemailer配置、邮件模板引擎、批量发送与速率控制、发送状态追踪、附件处理

## 📖 知识点概述

### 核心概念

**Nodemailer**是Node.js中最流行的邮件发送库，支持SMTP、SendGrid、AWS SES等多种邮件服务。结合消息队列，可以实现可靠的、大规模的邮件发送系统。

### 邮件发送架构

```
┌──────────────────────────────────────────────────────────┐
│              邮件发送系统架构                              │
├──────────────────────────────────────────────────────────┤
│                                                            │
│  1. 触发器（用户注册、订单确认、营销活动）                  │
│      │                                                     │
│      ▼                                                     │
│  2. Bull队列（邮件任务入队）                               │
│      ├─ 优先级：VIP > 普通                                │
│      ├─ 延迟：定时发送                                    │
│      └─ 重试：失败自动重试                                │
│      │                                                     │
│      ▼                                                     │
│  3. Worker（并发处理）                                     │
│      ├─ 模板渲染（Handlebars/EJS）                        │
│      ├─ 速率限制（每秒10封）                              │
│      └─ Nodemailer发送                                    │
│      │                                                     │
│      ▼                                                     │
│  4. SMTP服务器 / SendGrid / AWS SES                       │
│      │                                                     │
│      ▼                                                     │
│  5. 状态追踪（成功/失败/退信/打开/点击）                    │
│                                                            │
└──────────────────────────────────────────────────────────┘
```

### Nodemailer传输方式

| 传输方式 | 优点 | 缺点 | 适用场景 |
|---------|------|------|---------|
| **SMTP** | 灵活、通用 | 需要自己管理 | 小规模、开发环境 |
| **SendGrid** | 高送达率、API简单 | 付费 | 商业应用 |
| **AWS SES** | 成本低、可靠 | 配置复杂 | 大规模发送 |
| **Gmail** | 免费、简单 | 限制500封/天 | 个人项目 |
| **Mailgun** | 功能强大 | 付费 | 中等规模 |

### 邮件类型

```
1. 事务邮件（Transactional）
   - 触发式：用户操作触发
   - 示例：注册确认、密码重置、订单确认
   - 特点：实时性高、个性化强
   
2. 营销邮件（Marketing）
   - 批量式：定时发送
   - 示例：促销活动、新闻通讯
   - 特点：数量大、需要退订链接
   
3. 通知邮件（Notification）
   - 系统式：自动触发
   - 示例：系统告警、每日摘要
   - 特点：定时发送、批量处理
```

## 🚀 使用指南

### 理解：为什么需要邮件队列？

#### 同步发送的问题

```javascript
// ❌ 同步发送：阻塞用户请求
app.post('/register', async (req, res) => {
  const user = await createUser(req.body);
  
  // 发送欢迎邮件（2-5秒）⚠️ 用户需要等待
  await sendWelcomeEmail(user.email, user.name);
  
  res.json({ success: true, user });
});

// 问题：
// 1. 用户注册体验差（等待邮件发送）
// 2. 邮件发送失败会导致注册失败
// 3. 高峰期大量请求会导致超时
```

#### 使用邮件队列

```javascript
// ✅ 异步发送：立即返回，后台处理
import Queue from 'bull';

const emailQueue = new Queue('email', 'redis://localhost:6379');

app.post('/register', async (req, res) => {
  const user = await createUser(req.body);
  
  // 💡 添加到邮件队列（1ms）
  await emailQueue.add('welcome', {
    to: user.email,
    name: user.name,
    userId: user.id
  }, {
    attempts: 3,
    backoff: {
      type: 'exponential',
      delay: 2000
    }
  });
  
  res.json({ success: true, user });
});

// Worker在后台发送
emailQueue.process('welcome', async (job) => {
  await sendWelcomeEmail(job.data.to, job.data.name);
});
```

```javascript
  constructor() {
    this.templates = new Map();

  async loadTemplate(name, filePath) {
    const html = await fs.readFile(filePath, 'utf-8');
    const compiled = handlebars.compile(html);
    this.templates.set(name, compiled);

    // 渲染HTML
    let html = template(data);
    
    // 💡 使用juice内联CSS（提高兼容性）

    return html;
  }
}
```

#### 第三步：邮件队列系统

```javascript
// email-queue-system.js
import Queue from 'bull';
import nodemailer from 'nodemailer';

const emailQueue = new Queue('email', {
  redis: { host: 'localhost', port: 6379 },
  defaultJobOptions: {
    attempts: 3,
    backoff: {
      type: 'exponential',
      delay: 2000
    },
    removeOnComplete: 100,
    removeOnFail: 200
  }
});

const transporter = nodemailer.createTransporter({
  host: process.env.SMTP_HOST,
  port: 587,
  auth: {
    user: process.env.SMTP_USER,
    pass: process.env.SMTP_PASS
  },
  pool: true,
  maxConnections: 5
});

// 💡 邮件类型定义
const EMAIL_TYPES = {
  WELCOME: 'welcome',
  RESET_PASSWORD: 'reset-password',
  ORDER_CONFIRMATION: 'order-confirmation',
  MARKETING: 'marketing',
  NOTIFICATION: 'notification'
};

// 💡 添加邮件任务
async function queueEmail(type, recipient, data, options = {}) {
  const job = await emailQueue.add(type, {
    to: recipient,
    ...data
  }, {
    priority: options.priority || 5,
    delay: options.delay || 0,
    jobId: options.jobId,
    ...options
  });
  
  console.log(`📤 邮件任务已入队: ${job.id} (${type})`);
  return job;
}

// 💡 Worker - 处理欢迎邮件
emailQueue.process(EMAIL_TYPES.WELCOME, async (job) => {
  const { to, name, verifyToken } = job.data;
  
  console.log(`📧 发送欢迎邮件: ${to}`);
  
  const html = templateManager.render('welcome', {
    name,
    verifyUrl: `https://example.com/verify?token=${verifyToken}`
  });
  
  const info = await transporter.sendMail({
    from: '"My App" <noreply@example.com>',
    to,
    subject: '欢迎注册',
    html
  });
  
  // 💡 记录发送状态
  await db.emails.create({
    jobId: job.id,
    type: EMAIL_TYPES.WELCOME,
    to,
    messageId: info.messageId,
    status: 'sent',
    sentAt: Date.now()
  });
  
  return { success: true, messageId: info.messageId };
});

// 💡 Worker - 处理密码重置邮件（高优先级）
emailQueue.process(EMAIL_TYPES.RESET_PASSWORD, async (job) => {
  const { to, resetToken } = job.data;
  
  const html = templateManager.render('reset-password', {
    resetUrl: `https://example.com/reset?token=${resetToken}`,
    expiresIn: '1小时'
  });
  
  const info = await transporter.sendMail({
    from: '"My App" <noreply@example.com>',
    to,
    subject: '重置密码',
    html
  });
  
  return { success: true, messageId: info.messageId };
});

// 💡 Worker - 处理营销邮件（低优先级、速率限制）
const marketingLimiter = {
  max: 10,      // 每秒最多10封
  duration: 1000
};

emailQueue.process(
  EMAIL_TYPES.MARKETING,
  { concurrency: 5, limiter: marketingLimiter },
  async (job) => {
    const { to, campaign, unsubscribeToken } = job.data;
    
    const html = templateManager.render('marketing', {
      ...campaign,
      unsubscribeUrl: `https://example.com/unsubscribe?token=${unsubscribeToken}`
    });
    
    const info = await transporter.sendMail({
      from: '"My App Marketing" <marketing@example.com>',
      to,
      subject: campaign.subject,
      html,
      // 💡 营销邮件必须有退订链接
      headers: {
        'List-Unsubscribe': `<https://example.com/unsubscribe?token=${unsubscribeToken}>`
      }
    });
    
    return { success: true, messageId: info.messageId };
  }
);

// 💡 监听事件
emailQueue.on('completed', (job, result) => {
  console.log(`✅ 邮件发送成功: ${job.id}`);
});

emailQueue.on('failed', (job, err) => {
  console.error(`❌ 邮件发送失败: ${job.id}`, err.message);
  
  // 记录失败
  db.emails.update(job.id, {
    status: 'failed',
    error: err.message,
    failedAt: Date.now()
  });
});

// 💡 使用示例
// 1. 注册欢迎邮件（高优先级）
await queueEmail(EMAIL_TYPES.WELCOME, 'user@example.com', {
  name: 'John',
  verifyToken: 'abc123'
}, {
  priority: 1
});

// 2. 密码重置邮件（最高优先级）
await queueEmail(EMAIL_TYPES.RESET_PASSWORD, 'user@example.com', {
  resetToken: 'xyz789'
}, {
  priority: 1
});

// 3. 营销邮件（低优先级、批量发送）
const users = await db.users.find({ subscribed: true });

for (const user of users) {
  await queueEmail(EMAIL_TYPES.MARKETING, user.email, {
    campaign: {
      subject: '限时优惠活动',
      title: '全场5折',
      content: '...'
    },
    unsubscribeToken: user.unsubscribeToken
  }, {
    priority: 10, // 低优先级
    jobId: `marketing-${user.id}` // 防重复
  });
}
```

#### 第五步：批量发送与追踪

```javascript
// batch-email-system.js
import Queue from 'bull';
import nodemailer from 'nodemailer';

const batchQueue = new Queue('batch-email', {
  redis: { host: 'localhost', port: 6379 }
});

// 💡 批量邮件任务
async function sendBatchEmails(campaign) {
  const { subject, template, recipients } = campaign;
  
  console.log(`📬 开始批量发送: ${recipients.length}封邮件`);
  
  // 💡 分批添加到队列（每批1000封）
  const batchSize = 1000;
  
  for (let i = 0; i < recipients.length; i += batchSize) {
    const batch = recipients.slice(i, i + batchSize);
    
    await batchQueue.add('send-batch', {
      campaignId: campaign.id,
      subject,
      template,
      recipients: batch,
      batchNumber: Math.floor(i / batchSize) + 1
    }, {
      priority: 10, // 低优先级
      removeOnComplete: true
    });
  }
  
  console.log(`✅ 已添加 ${Math.ceil(recipients.length / batchSize)} 个批次`);
}

// 💡 Worker - 处理批次
batchQueue.process('send-batch', 10, async (job) => {
  const { campaignId, subject, template, recipients, batchNumber } = job.data;
  
  console.log(`📧 处理批次 ${batchNumber}: ${recipients.length}封`);
  
  const results = {
    success: 0,
    failed: 0,
    errors: []
  };
  
  // 💡 并发发送（控制并发数）
  const concurrency = 5;
  
  for (let i = 0; i < recipients.length; i += concurrency) {
    const chunk = recipients.slice(i, i + concurrency);
    
    const promises = chunk.map(async (recipient) => {
      try {
        const html = templateManager.render(template, {
          ...recipient,
          unsubscribeUrl: `https://example.com/unsubscribe?email=${recipient.email}`
        });
        
        const info = await transporter.sendMail({
          from: '"My App" <noreply@example.com>',
          to: recipient.email,
          subject,
          html
        });
        
        // 记录发送
        await db.emailLogs.create({
          campaignId,
          recipientId: recipient.id,
          messageId: info.messageId,
          status: 'sent',
          sentAt: Date.now()
        });
        
        results.success++;
      } catch (error) {
        console.error(`❌ 发送失败: ${recipient.email}`, error.message);
        
        results.failed++;
        results.errors.push({
          email: recipient.email,
          error: error.message
        });
        
        // 记录失败
        await db.emailLogs.create({
          campaignId,
          recipientId: recipient.id,
          status: 'failed',
          error: error.message,
          failedAt: Date.now()
        });
      }
    });
    
    await Promise.all(promises);
    
    // 💡 更新进度
    job.progress((i + chunk.length) / recipients.length * 100);
    
    // 💡 速率限制（休息100ms）
    await new Promise(resolve => setTimeout(resolve, 100));
  }
  
  console.log(`✅ 批次完成: 成功${results.success}, 失败${results.failed}`);
  
  return results;
});

// 💡 追踪邮件状态
class EmailTracker {
  // 打开追踪
  addOpenTracking(html, trackingId) {
    // 💡 嵌入1x1透明像素图片
    const trackingPixel = `<img src="https://example.com/track/open/${trackingId}" width="1" height="1" style="display:none;">`;
    
    return html + trackingPixel;
  }
  
  // 点击追踪
  addClickTracking(html, trackingId) {
    // 💡 替换所有链接
    return html.replace(
      /<a href="([^"]+)"/g,
      (match, url) => {
        const trackedUrl = `https://example.com/track/click/${trackingId}?url=${encodeURIComponent(url)}`;
        return `<a href="${trackedUrl}"`;
      }
    );
  }
}

const tracker = new EmailTracker();

import express from 'express';

  // 记录打开
  await db.emailLogs.update({ trackingId }, {
    opened: true,
    openedAt: Date.now()
  });
```

### 高频错误3：速率限制

**现象：**
```
Error: 454 4.7.1 <example.com>: Relay access denied
```

**原因：**
超过SMTP服务器速率限制（如Gmail限制500封/天）

**解决方案：**

```javascript
// ✅ 方案1：实现速率限制
import Queue from 'bull';

const emailQueue = new Queue('email', {
  redis: { host: 'localhost', port: 6379 },
  limiter: {
    max: 10,      // 每10秒最多10封
    duration: 10000
  }
});

// ✅ 方案2：监控发送量
class EmailRateMonitor {
  constructor(maxPerHour = 500) {
    this.maxPerHour = maxPerHour;
    this.sent = [];
  }
  
  canSend() {
    const now = Date.now();
    const oneHourAgo = now - 3600000;
    
    // 清理1小时前的记录
    this.sent = this.sent.filter(time => time > oneHourAgo);
    
    return this.sent.length < this.maxPerHour;
  }
  
  record() {
    this.sent.push(Date.now());
  }
}

const monitor = new EmailRateMonitor(500);

emailQueue.process(async (job) => {
  if (!monitor.canSend()) {
    // 延迟1小时
    throw new Error('达到速率限制，1小时后重试');
  }
  
  await transporter.sendMail(job.data);
  monitor.record();
});

// ✅ 方案3：使用专业服务（无速率限制）
// SendGrid: 100封/天免费，付费无限制
// AWS SES: 62,000封/月免费
```

## 📋 总结

### 核心要点

1. **Nodemailer传输**：SMTP、SendGrid、AWS SES
2. **邮件队列**：Bull实现异步发送、重试、速率限制
3. **模板引擎**：Handlebars渲染动态内容
4. **批量发送**：分批、并发控制、进度追踪

### 实现关键

1. **连接池**：复用SMTP连接，提升性能
2. **速率限制**：防止超过服务器限制
3. **追踪系统**：打开、点击追踪
4. **错误处理**：自动重试、死信队列

### 应用价值

- **用户体验**：异步发送，响应快99%
- **可靠性**：自动重试，成功率99.5%+
- **可扩展**：水平扩展Worker节点
- **可监控**：发送状态、打开率、点击率

### 面试重点

**Q1：如何保证邮件发送可靠性？**
答：1) 使用邮件队列（Bull）2) 自动重试机制 3) 死信队列 4) 发送状态追踪。

**Q2：如何防止邮件进入垃圾箱？**
答：1) 配置SPF/DKIM/DMARC 2) 使用专业邮件服务 3) 避免垃圾词汇 4) 提供退订链接。

**Q3：如何实现大规模邮件发送？**
答：1) 分批发送 2) 速率限制 3) 并发控制 4) 使用专业服务（SendGrid/SES）。

**Q4：邮件模板为什么要内联CSS？**
答：很多邮件客户端不支持`<style>`标签，内联CSS兼容性最好（使用juice库）。

### 注意事项

⚠️ **Gmail限制**：500封/天，需要应用专用密码
⚠️ **退订链接**：营销邮件必须提供退订功能
⚠️ **附件大小**：限制10MB以内，大文件用链接
⚠️ **速率控制**：避免被标记为垃圾邮件发送者
