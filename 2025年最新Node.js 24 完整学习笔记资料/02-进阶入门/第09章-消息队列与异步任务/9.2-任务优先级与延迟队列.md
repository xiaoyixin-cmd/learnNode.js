# 9.2 任务优先级与延迟队列

> 任务优先级设置、延迟任务、定时任务、Repeat可重复任务、任务去重与幂等性

---

## 📖 知识点概述

### 核心概念

**任务优先级**和**延迟队列**是消息队列的高级特性，用于实现复杂的任务调度场景。优先级队列允许重要任务优先处理，延迟队列允许任务在未来某个时间点执行。

### 任务调度类型

```
┌──────────────────────────────────────────────────────────┐
│               Bull 任务调度类型                            │
├──────────────────────────────────────────────────────────┤
│  1. 立即任务（Immediate）                                  │
│     └─ 立即添加到队列，按FIFO或优先级处理                   │
│                                                            │
│  2. 延迟任务（Delayed）                                    │
│     └─ 指定延迟时间，到期后自动转为waiting                  │
│                                                            │
│  3. 定时任务（Scheduled）                                  │
│     └─ 指定未来某个时间点执行                               │
│                                                            │
│  4. 重复任务（Repeatable）                                 │
│     ├─ Cron表达式（每天、每周、每月）                       │
│     └─ 固定间隔（every）                                   │
│                                                            │
│  5. 优先级任务（Prioritized）                              │
│     └─ 高优先级任务插队处理                                 │
└──────────────────────────────────────────────────────────┘
```

### 优先级实现原理

```
Redis Sorted Set（按优先级排序）

Score（分数） │ Member（任务ID）
─────────────┼──────────────────
    1        │ job:101  ← 最高优先级（先处理）
    1        │ job:102
    3        │ job:103
    5        │ job:104  ← 中等优先级
    10       │ job:105  ← 最低优先级（后处理）
```

**注意**：Bull的优先级是1-10，数字**越小**优先级**越高**（与常规认知相反）。

### 延迟队列实现原理

```
Redis Sorted Set（按执行时间排序）

Score（时间戳）    │ Member（任务ID）
──────────────────┼──────────────────
1704067200000     │ job:201  ← 最早执行
1704070800000     │ job:202
1704074400000     │ job:203
1704078000000     │ job:204  ← 最晚执行

Bull定期扫描（每秒），将到期任务移到waiting队列
```

### 应用场景

| 场景 | 类型 | 示例 |
|------|------|------|
| **VIP用户优先** | 优先级 | VIP订单优先发货 |
| **超时取消** | 延迟 | 订单15分钟未支付自动取消 |
| **定时发送** | 定时 | 生日祝福邮件 |
| **周期任务** | 重复 | 每日数据统计、每周报表 |
| **限流** | 延迟+优先级 | API速率限制 |

## 🚀 使用指南

### 理解：优先级队列的必要性

#### 无优先级的问题

```javascript
// ❌ FIFO队列：所有任务平等对待
await queue.add('task', { type: 'normal', userId: 1 });
await queue.add('task', { type: 'vip', userId: 2 });      // VIP用户
await queue.add('task', { type: 'critical', orderId: 3 }); // 紧急订单

// 处理顺序：1 → 2 → 3（按添加顺序）
// 问题：VIP用户和紧急订单需要等待
```

#### 使用优先级

```javascript
// ✅ 优先级队列：重要任务优先
await queue.add('task', { type: 'normal', userId: 1 }, { priority: 5 });
await queue.add('task', { type: 'vip', userId: 2 }, { priority: 1 });      // 最高优先级
await queue.add('task', { type: 'critical', orderId: 3 }, { priority: 2 }); // 高优先级

// 处理顺序：2(VIP) → 3(紧急) → 1(普通)
```

### 实践：完整实现

#### 第一步：优先级队列

```javascript
// priority-queue.js
import Queue from 'bull';

const taskQueue = new Queue('tasks', {
  redis: {
    host: 'localhost',
    port: 6379
  }
});

// 💡 定义优先级常量
const PRIORITY = {
  CRITICAL: 1,  // 最高优先级（系统级任务）
  HIGH: 3,      // 高优先级（VIP用户）
  NORMAL: 5,    // 普通优先级（默认）
  LOW: 7,       // 低优先级（后台任务）
  IDLE: 10      // 最低优先级（闲时任务）
};

// 💡 添加不同优先级的任务
async function addTask(data, priority = PRIORITY.NORMAL) {
  const job = await taskQueue.add('process', data, {
    priority,
    removeOnComplete: 100,
    attempts: 3
  });
  
  console.log(`➕ 任务已添加: ${job.id}, 优先级: ${priority}`);
  return job;
}

// 💡 Worker按优先级处理
taskQueue.process('process', 5, async (job) => {
  console.log(`🔨 处理任务: ${job.id}, 优先级: ${job.opts.priority}`);
  
  await new Promise(resolve => setTimeout(resolve, 2000));
  
  return { success: true, processedAt: Date.now() };
});

// 使用示例
await addTask({ type: 'normal-order', orderId: 1 }, PRIORITY.NORMAL);
await addTask({ type: 'vip-order', orderId: 2, vip: true }, PRIORITY.HIGH);
await addTask({ type: 'system-backup', task: 'backup' }, PRIORITY.CRITICAL);
await addTask({ type: 'analytics', task: 'daily-report' }, PRIORITY.LOW);

// 处理顺序：backup(1) → vip-order(3) → normal-order(5) → analytics(7)

// 💡 动态调整优先级
async function promotePriority(jobId, newPriority) {
  const job = await taskQueue.getJob(jobId);
  
  if (!job) {
    throw new Error('任务不存在');
  }
  
  // Bull不支持直接修改优先级，需要重新添加
  const newJob = await taskQueue.add(job.name, job.data, {
    ...job.opts,
    priority: newPriority
  });
  
  // 删除旧任务
  await job.remove();
  
  console.log(`⬆️ 任务优先级已提升: ${jobId} → ${newJob.id}`);
  return newJob;
}
```

#### 第二步：延迟队列

```javascript
// delayed-queue.js
import Queue from 'bull';

const delayQueue = new Queue('delayed-tasks', {
  redis: { host: 'localhost', port: 6379 }
});

// 💡 延迟任务（相对时间）
async function addDelayedTask(data, delayMs) {
  const job = await delayQueue.add('delayed', data, {
    delay: delayMs, // 延迟毫秒数
    removeOnComplete: true
  });
  
  console.log(`⏰ 延迟任务已添加: ${job.id}, 延迟: ${delayMs}ms`);
  console.log(`   将在 ${new Date(Date.now() + delayMs).toLocaleString()} 执行`);
  
  return job;
}

// 💡 定时任务（绝对时间）
async function addScheduledTask(data, executeAt) {
  const now = Date.now();
  const executeTime = new Date(executeAt).getTime();
  const delay = executeTime - now;
  
  if (delay < 0) {
    throw new Error('执行时间必须是未来时间');
  }
  
  const job = await delayQueue.add('scheduled', data, {
    delay,
    removeOnComplete: true
  });
  
  console.log(`📅 定时任务已添加: ${job.id}`);
  console.log(`   将在 ${new Date(executeTime).toLocaleString()} 执行`);
  
  return job;
}

// Worker处理
delayQueue.process('delayed', async (job) => {
  console.log(`⏰ 延迟任务触发: ${job.id}`);
  console.log(`   延迟了 ${Date.now() - job.timestamp}ms`);
  
  await processTask(job.data);
  
  return { success: true };
});

delayQueue.process('scheduled', async (job) => {
  console.log(`📅 定时任务触发: ${job.id}`);
  
  await processTask(job.data);
  
  return { success: true };
});

// 使用示例
// 1. 5分钟后执行
await addDelayedTask(
  { action: 'send-reminder', userId: 123 },
  5 * 60 * 1000
);

// 2. 明天上午10点执行
const tomorrow10am = new Date();
tomorrow10am.setDate(tomorrow10am.getDate() + 1);
tomorrow10am.setHours(10, 0, 0, 0);

await addScheduledTask(
  { action: 'daily-report', date: tomorrow10am.toISOString() },
  tomorrow10am
);

// 3. 订单超时取消（15分钟）
async function createOrder(orderData) {
  const order = await db.orders.create(orderData);
  
  // 添加延迟取消任务
  await addDelayedTask(
    {
      action: 'cancel-unpaid-order',
      orderId: order.id
    },
    15 * 60 * 1000
  );
  
  return order;
}

// 💡 取消延迟任务
async function cancelDelayedTask(jobId) {
  const job = await delayQueue.getJob(jobId);
  
  if (!job) {
    return { success: false, message: '任务不存在' };
  }
  
  const state = await job.getState();
  
  if (state === 'delayed') {
    await job.remove();
    console.log(`✅ 延迟任务已取消: ${jobId}`);
    return { success: true };
  }
  
  return { success: false, message: `任务已经是${state}状态，无法取消` };
}
```

// repeatable-jobs.js
import Queue from 'bull';

const cronQueue = new Queue('cron-tasks', {
  redis: { host: 'localhost', port: 6379 }
});

// 💡 Cron表达式任务
async function addCronJob(name, data, cronExpression, options = {}) {
  const job = await cronQueue.add(name, data, {
    repeat: {
      cron: cronExpression,
      tz: options.timezone || 'Asia/Shanghai' // 时区
    },
    ...options
  });
  
  console.log(`🔁 重复任务已添加: ${name}`);
  console.log(`   Cron: ${cronExpression}`);
  
  return job;
}

// 💡 固定间隔任务
async function addIntervalJob(name, data, intervalMs, options = {}) {
  const job = await cronQueue.add(name, data, {
    repeat: {
      every: intervalMs, // 每隔X毫秒
      limit: options.limit // 可选：限制执行次数
    },
    ...options
  });
  
  console.log(`🔁 间隔任务已添加: ${name}`);
  console.log(`   间隔: ${intervalMs}ms`);
  
  return job;
}

// Worker处理
cronQueue.process('daily-report', async (job) => {
  console.log('📊 生成每日报表...');
  
  const report = await generateDailyReport();
  await sendReportEmail(report);
  
  return { success: true, reportDate: new Date().toISOString() };
});

cronQueue.process('heartbeat', async (job) => {
  console.log('💓 心跳检测...');
  
  const health = await checkSystemHealth();
  
  if (!health.ok) {
    await sendAlert('系统异常', health);
  }
  
  return health;
});

// 💡 Cron表达式示例
// 每分钟
await addCronJob('test-minute', {}, '* * * * *');

// 每天凌晨0点
await addCronJob('daily-report', {}, '0 0 * * *');

// 每周一上午9点
await addCronJob('weekly-summary', {}, '0 9 * * 1');

// 每月1号上午10点
await addCronJob('monthly-billing', {}, '0 10 1 * *');

// 工作日上午9点
await addCronJob('weekday-reminder', {}, '0 9 * * 1-5');

// 每30分钟
await addCronJob('half-hour-task', {}, '*/30 * * * *');

// 💡 固定间隔示例
// 每10秒（心跳检测）
await addIntervalJob('heartbeat', {}, 10 * 1000);

// 每5分钟（缓存清理）
await addIntervalJob('cache-cleanup', {}, 5 * 60 * 1000);

// 💡 查询所有重复任务
async function getRepeatableJobs() {
  const jobs = await cronQueue.getRepeatableJobs();
  
  return jobs.map(job => ({
    name: job.name,
    id: job.id,
    key: job.key,
    cron: job.cron,
    next: job.next, // 下次执行时间（时间戳）
    nextDate: new Date(job.next).toLocaleString()
  }));
}

// 💡 删除重复任务
async function removeRepeatableJob(name, repeatOpts) {
  // repeatOpts必须与创建时完全相同
  await cronQueue.removeRepeatable(name, repeatOpts);
  
  console.log(`✅ 重复任务已删除: ${name}`);
}

// 示例
await removeRepeatableJob('daily-report', {
  cron: '0 0 * * *',
  tz: 'Asia/Shanghai'
});

// 💡 删除所有重复任务
async function removeAllRepeatableJobs() {
  const jobs = await cronQueue.getRepeatableJobs();
  
  for (const job of jobs) {
    await cronQueue.removeRepeatableByKey(job.key);
  }
  
  console.log(`✅ 已删除 ${jobs.length} 个重复任务`);
}

#### 第四步：任务去重与幂等性

```javascript
// job-deduplication.js
import Queue from 'bull';
import crypto from 'crypto';

const dedupeQueue = new Queue('dedupe', {
  redis: { host: 'localhost', port: 6379 }
});

// 💡 方案1：使用jobId去重
async function addUniqueJob(name, data) {
  // 根据数据生成唯一ID
  const jobId = generateJobId(name, data);
  
  try {
    const job = await dedupeQueue.add(name, data, {
      jobId, // 相同jobId的任务只会添加一次
      removeOnComplete: true
    });
    
    console.log(`✅ 任务已添加: ${job.id}`);
    return { success: true, job };
  } catch (error) {
    if (error.message.includes('Job already exists')) {
      console.log(`⚠️ 任务已存在，跳过: ${jobId}`);
      return { success: false, reason: 'duplicate' };
    }
    throw error;
  }
}

function generateJobId(name, data) {
  // 使用内容哈希作为ID
  const content = JSON.stringify({ name, ...data });
  return crypto.createHash('md5').update(content).digest('hex');
}

// 使用示例
await addUniqueJob('send-email', {
  to: 'user@example.com',
  subject: '欢迎注册'
});

// 重复添加（会被忽略）
await addUniqueJob('send-email', {
  to: 'user@example.com',
  subject: '欢迎注册'
});

// 💡 方案2：幂等性处理（Worker层面）
const processedJobs = new Set();

dedupeQueue.process('send-email', async (job) => {
  const jobHash = generateJobId(job.name, job.data);
  
  // 检查是否已处理
  if (processedJobs.has(jobHash)) {
    console.log(`⚠️ 任务已处理过，跳过: ${job.id}`);
    return { success: true, skipped: true };
  }
  
  // 处理任务
  await sendEmail(job.data);
  
  // 标记为已处理
  processedJobs.add(jobHash);
  
  // 定期清理（避免内存泄漏）
  setTimeout(() => processedJobs.delete(jobHash), 24 * 3600 * 1000);
  
  return { success: true, sent: true };
});

// 💡 方案3：使用Redis去重（推荐）
import Redis from 'ioredis';

const redis = new Redis();

async function addIdempotentJob(name, data) {
  const jobKey = generateJobId(name, data);
  const lockKey = `job:lock:${jobKey}`;
  
  // 尝试获取锁（24小时过期）
  const locked = await redis.set(lockKey, '1', 'EX', 86400, 'NX');
  
  if (!locked) {
    console.log(`⚠️ 任务正在处理或已完成: ${jobKey}`);
    return { success: false, reason: 'duplicate' };
  }
  
  // 添加任务
  const job = await dedupeQueue.add(name, data, {
    jobId: jobKey,
    removeOnComplete: true
  });
  
  console.log(`✅ 任务已添加: ${job.id}`);
  return { success: true, job };
}

// Worker处理完成后删除锁
dedupeQueue.on('completed', async (job) => {
  // 可选：保留锁一段时间防止重复
  // await redis.del(`job:lock:${job.id}`);
});
```

## 🔧 错误处理

### 高频错误1：延迟任务未执行

**现象：**
延迟任务添加成功，但到期后没有执行

**原因：**
1. Worker未启动
2. Redis时间不同步
3. 延迟时间计算错误

**解决方案：**

```javascript
// ✅ 检查1：确保Worker正在运行
queue.process('delayed', async (job) => {
  console.log(`⏰ 延迟任务执行: ${job.id}`);
});

// ✅ 检查2：验证延迟时间
const job = await queue.add('test', { data: 'test' }, {
  delay: 10000 // 10秒
});

console.log('任务添加时间:', new Date(job.timestamp).toLocaleString());
console.log('预计执行时间:', new Date(job.timestamp + 10000).toLocaleString());

// ✅ 检查3：Redis时间同步
import Redis from 'ioredis';

const redis = new Redis();
const [seconds] = await redis.time();
const redisTime = parseInt(seconds) * 1000;
const localTime = Date.now();
const diff = Math.abs(redisTime - localTime);

if (diff > 5000) { // 超过5秒
  console.warn(`⚠️ Redis时间偏差: ${diff}ms`);
  console.warn(`   Redis时间: ${new Date(redisTime).toLocaleString()}`);
  console.warn(`   本地时间: ${new Date(localTime).toLocaleString()}`);
}

// ✅ 检查4：查询延迟任务状态
async function checkDelayedJob(jobId) {
  const job = await queue.getJob(jobId);
  
  if (!job) {
    return { error: '任务不存在' };
  }
  
  const state = await job.getState();
  const delay = job.opts.delay;
  const addedAt = job.timestamp;
  const executeAt = addedAt + delay;
  const now = Date.now();
  
  return {
    id: job.id,
    state,
    addedAt: new Date(addedAt).toLocaleString(),
    executeAt: new Date(executeAt).toLocaleString(),
    remaining: Math.max(0, executeAt - now),
    remainingSeconds: Math.max(0, Math.round((executeAt - now) / 1000))
  };
}
```

### 高频错误2：Cron任务重复执行

**现象：**
Cron任务每次执行时创建多个实例，导致重复

**原因：**
1. 多次调用`queue.add()`添加相同的Cron任务
2. repeat选项配置不一致

**解决方案：**

```javascript
// ❌ 错误：每次启动都添加任务
app.listen(3000, async () => {
  // 每次重启都会添加新的Cron任务，导致重复
  await queue.add('daily-report', {}, {
    repeat: { cron: '0 0 * * *' }
  });
});

// ✅ 正确：先删除再添加
async function ensureCronJob(name, data, cronExpression) {
  // 1. 删除已存在的任务
  const existingJobs = await queue.getRepeatableJobs();
  
  for (const job of existingJobs) {
    if (job.name === name) {
      await queue.removeRepeatableByKey(job.key);
      console.log(`🗑️ 删除旧任务: ${name}`);
    }
  }
  
  // 2. 添加新任务
  await queue.add(name, data, {
    repeat: {
      cron: cronExpression,
      tz: 'Asia/Shanghai'
    }
  });
  
  console.log(`✅ Cron任务已添加: ${name}`);
}

app.listen(3000, async () => {
  await ensureCronJob('daily-report', {}, '0 0 * * *');
});

// ✅ 或使用jobId（推荐）
await queue.add('daily-report', {}, {
  jobId: 'cron:daily-report', // 固定ID，防止重复
  repeat: {
    cron: '0 0 * * *',
    tz: 'Asia/Shanghai'
  }
});
```

### 高频错误3：优先级不生效

**现象：**
设置了优先级，但任务仍按FIFO顺序执行

**原因：**
Redis没有启用优先级排序（需要配置）

**解决方案：**

```javascript
// ❌ 问题：未设置lifo选项
const queue = new Queue('tasks', {
  redis: { host: 'localhost', port: 6379 }
});

// ✅ 正确：启用lifo和优先级
const queue = new Queue('tasks', {
  redis: { host: 'localhost', port: 6379 },
  defaultJobOptions: {
    lifo: false // 确保是false（FIFO + 优先级）
  }
});

// ✅ 验证优先级
async function testPriority() {
  // 添加不同优先级的任务
  await queue.add('test', { id: 1 }, { priority: 5 });
  await queue.add('test', { id: 2 }, { priority: 1 }); // 应该优先
  await queue.add('test', { id: 3 }, { priority: 10 });
  
  // 查询waiting任务顺序
  const waiting = await queue.getWaiting();
  console.log('任务顺序:', waiting.map(j => ({
    id: j.data.id,
    priority: j.opts.priority
  })));
  
  // 期望输出：[{id:2, priority:1}, {id:1, priority:5}, {id:3, priority:10}]
}

// ✅ 注意：已经在active状态的任务不会重新排序
// 只有waiting状态的任务会按优先级排序
```

## 📋 总结

### 核心要点

1. **优先级**：1-10，数字越小优先级越高
2. **延迟任务**：使用`delay`选项，单位毫秒
3. **Cron任务**：使用`repeat.cron`，支持标准Cron表达式
4. **去重**：使用`jobId`或Redis锁

### 实现关键

1. **优先级启用**：`lifo: false`（默认）
2. **Cron时区**：指定`tz`避免时区问题
3. **任务清理**：Cron任务需手动删除
4. **幂等性**：Worker层面检查重复

### 应用价值

- **VIP优先**：提升高价值用户体验
- **定时任务**：替代Linux Cron
- **延迟处理**：超时取消、定时提醒
- **去重保证**：避免重复处理

### 面试重点

**Q1：Bull的优先级是如何实现的？**
答：使用Redis Sorted Set，score为优先级（1-10），数字越小优先级越高。

**Q2：如何确保任务只执行一次？**
答：使用`jobId`去重，相同ID的任务只会添加一次；Worker层面使用Redis锁或Set去重。

**Q3：Cron任务如何防止重复添加？**
答：使用固定`jobId`，或启动前删除已存在的重复任务。

**Q4：延迟任务和Cron任务的区别？**
答：延迟任务是一次性的，到期后执行一次；Cron任务是周期性的，按表达式重复执行。

### 注意事项

⚠️ **优先级方向**：Bull是1最高，10最低（与常规相反）
⚠️ **Cron时区**：必须指定`tz`，否则使用服务器时区
⚠️ **任务清理**：Cron任务不会自动删除，需手动管理
⚠️ **Redis时间**：延迟任务依赖Redis时间，确保同步
