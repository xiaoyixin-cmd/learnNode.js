# 9.3 任务失败重试与死信队列

> 自动重试机制（指数退避）、重试次数与间隔配置、死信队列处理、任务清理与归档

---

## 📖 知识点概述

### 核心概念

**任务失败重试**和**死信队列（Dead Letter Queue, DLQ）**是保证消息队列可靠性的关键机制。当任务执行失败时，系统会自动重试；多次重试仍失败的任务会被移入死信队列，等待人工介入处理。

### 任务失败流程

```
┌─────────────────────────────────────────────────────────┐
│              任务失败处理流程                             │
├─────────────────────────────────────────────────────────┤
│  1. 任务执行失败                                          │
│     ├─ 网络错误                                          │
│     ├─ 外部API超时                                       │
│     ├─ 数据库连接失败                                     │
│     └─ 业务逻辑异常                                      │
│                                                           │
│  2. 检查重试次数                                          │
│     ├─ 未达到最大次数 → 进入重试                          │
│     └─ 已达到最大次数 → 进入死信队列                      │
│                                                           │
│  3. 重试策略                                              │
│     ├─ 固定间隔（Fixed）                                  │
│     ├─ 指数退避（Exponential）← 推荐                      │
│     └─ 自定义策略                                         │
│                                                           │
│  4. 死信队列处理                                          │
│     ├─ 人工审查                                          │
│     ├─ 手动重试                                          │
│     ├─ 数据修复                                          │
│     └─ 归档或删除                                        │
└─────────────────────────────────────────────────────────┘
```

### 重试策略对比

```
固定间隔（Fixed）
尝试1 ──2s──> 尝试2 ──2s──> 尝试3 ──2s──> 尝试4
问题：短时间内大量重试，可能加重系统压力

指数退避（Exponential）✅ 推荐
尝试1 ──2s──> 尝试2 ──4s──> 尝试3 ──8s──> 尝试4 ──16s──> 尝试5
优点：逐渐增加间隔，给系统恢复时间
```

### 重试策略配置

| 策略 | 间隔计算 | 适用场景 |
|------|---------|---------|
| **固定间隔** | delay | 网络抖动、临时故障 |
| **指数退避** | delay * 2^(attempts-1) | 外部API、数据库（推荐） |
| **线性增长** | delay * attempts | 中等恢复时间 |
| **自定义函数** | 自定义逻辑 | 复杂场景 |

### 死信队列（DLQ）

**死信队列**用于存储多次重试仍失败的任务，防止丢失，便于排查和恢复。

**典型处理流程：**
1. **收集死信**：自动移入DLQ
2. **告警通知**：发送邮件/短信/Slack
3. **人工审查**：查看错误原因
4. **修复数据**：修复错误数据或代码
5. **手动重试**：修复后重新执行
6. **归档清理**：定期清理或归档

## 🚀 使用指南

### 理解：为什么需要重试机制？

#### 无重试的问题

```javascript
// ❌ 无重试：一次失败就放弃
queue.process(async (job) => {
  try {
    await sendEmail(job.data.to, job.data.subject);
    return { success: true };
  } catch (error) {
    // 网络抖动、SMTP服务器临时不可用
    console.error('发送失败:', error);
    throw error; // 任务直接标记为失败
  }
});

// 问题：
// 1. 临时故障（网络抖动）导致任务永久失败
// 2. 用户无法收到重要邮件
// 3. 需要人工介入重新发送
```

#### 使用重试机制

```javascript
// ✅ 自动重试：给系统恢复的机会
queue.process(async (job) => {
  // Bull自动处理重试，无需手动try-catch
  await sendEmail(job.data.to, job.data.subject);
  return { success: true };
});

// 配置
await queue.add('send-email', data, {
  attempts: 5,           // 最多重试5次
  backoff: {
    type: 'exponential', // 指数退避
    delay: 2000          // 初始延迟2秒
  }
});

// 重试时间线：
// 尝试1：立即（失败）
// 尝试2：2秒后（失败）
// 尝试3：4秒后（失败）
// 尝试4：8秒后（失败）
// 尝试5：16秒后（成功）✅
```

#### 第一步：基础重试配置

```javascript
// retry-basic.js
import Queue from 'bull';

const retryQueue = new Queue('retry-tasks', {
  redis: { host: 'localhost', port: 6379 },
  // 💡 默认重试配置
  defaultJobOptions: {
    attempts: 3,              // 最多重试3次
    backoff: {
      type: 'exponential',    // 指数退避
      delay: 1000             // 初始1秒
    },
    removeOnComplete: 100,
    removeOnFail: false       // ⚠️ 失败任务不删除，用于排查
  }
});

// 💡 不同类型任务的重试配置
async function addEmailTask(data) {
  return await retryQueue.add('send-email', data, {
    attempts: 5,              // 邮件重试5次
    backoff: {
      type: 'exponential',
      delay: 2000             // 2秒起
    }
  });
}

async function addAPITask(data) {
  return await retryQueue.add('call-api', data, {
    attempts: 10,             // API重试10次
    backoff: {
      type: 'exponential',
      delay: 1000
    },
    timeout: 30000            // 30秒超时
  });
}

async function addCriticalTask(data) {
  return await retryQueue.add('critical', data, {
    attempts: 20,             // 关键任务重试20次
    backoff: {
      type: 'exponential',
      delay: 5000,
      // 💡 最大延迟限制
      options: {
        max: 60000            // 最大60秒
      }
    }
  });
}

// 💡 Worker处理
retryQueue.process('send-email', async (job) => {
  const { attemptsMade } = job;
  
  console.log(`📧 发送邮件 (尝试 ${attemptsMade + 1}/${job.opts.attempts})`);
  
  try {
    await sendEmail(job.data);
    console.log('✅ 邮件发送成功');
    return { success: true };
  } catch (error) {
    console.error(`❌ 发送失败 (尝试 ${attemptsMade + 1}):`, error.message);
    
    // 💡 记录失败原因
    job.log(`失败原因: ${error.message}`);
    
    // 抛出错误触发重试
    throw error;
  }
});

// 💡 监听重试事件
retryQueue.on('failed', (job, err) => {
  console.log(`❌ 任务失败: ${job.id}, 尝试次数: ${job.attemptsMade}`);
  
  if (job.attemptsMade < job.opts.attempts) {
    const nextDelay = calculateBackoff(job.opts.backoff, job.attemptsMade);
    console.log(`⏰ 将在 ${nextDelay}ms 后重试`);
  } else {
    console.log(`💀 任务已达到最大重试次数，进入死信队列`);
  }
});

// 计算退避延迟
function calculateBackoff(backoff, attemptsMade) {
  if (backoff.type === 'exponential') {
    return backoff.delay * Math.pow(2, attemptsMade - 1);
  } else if (backoff.type === 'fixed') {
    return backoff.delay;
  }
  return 0;
}
```

#### 第二步：自定义重试策略

```javascript
// retry-custom.js
import Queue from 'bull';

const customQueue = new Queue('custom-retry', {
  redis: { host: 'localhost', port: 6379 }
});

// 💡 自定义重试函数
function customBackoff(attemptsMade, err) {
  // 根据错误类型决定重试间隔
  if (err.code === 'ETIMEDOUT') {
    // 超时错误：快速重试
    return 1000 * attemptsMade; // 1s, 2s, 3s...
  } else if (err.code === 'ECONNREFUSED') {
    // 连接拒绝：慢速重试
    return 10000 * Math.pow(2, attemptsMade - 1); // 10s, 20s, 40s...
  } else if (err.statusCode === 429) {
    // 速率限制：等待更久
    return 60000 * attemptsMade; // 1min, 2min, 3min...
  } else {
    // 默认：指数退避
    return 2000 * Math.pow(2, attemptsMade - 1);
  }
}

// 💡 添加任务
await customQueue.add('api-call', { url: 'https://api.example.com' }, {
  attempts: 10,
  backoff: customBackoff // 自定义函数
});

// 💡 条件性重试
customQueue.process('api-call', async (job) => {
  try {
    const response = await fetch(job.data.url);
    
    if (!response.ok) {
      const error = new Error(`HTTP ${response.status}`);
      error.statusCode = response.status;
      
      // 💡 根据状态码决定是否重试
      if (response.status === 400) {
        // 客户端错误（参数错误）：不重试
        error.retriable = false;
        throw error;
      } else if (response.status === 429) {
        // 速率限制：重试
        error.retriable = true;
        throw error;
      } else if (response.status >= 500) {
        // 服务器错误：重试
        error.retriable = true;
        throw error;
      }
    }
    
    return await response.json();
  } catch (error) {
    // 💡 标记为不可重试
    if (error.retriable === false) {
      // 直接标记为失败，不再重试
      job.attemptsMade = job.opts.attempts;
    }
    
    throw error;
  }
});

// 💡 带熔断器的重试
class CircuitBreaker {
  constructor(threshold = 5, timeout = 60000) {
    this.threshold = threshold; // 失败阈值
    this.timeout = timeout;     // 熔断时长
    this.failures = 0;
    this.state = 'CLOSED';      // CLOSED/OPEN/HALF_OPEN
    this.nextAttempt = Date.now();
  }
  
  async call(fn) {
    if (this.state === 'OPEN') {
      if (Date.now() < this.nextAttempt) {
        throw new Error('熔断器开启，拒绝调用');
      }
      // 尝试恢复
      this.state = 'HALF_OPEN';
    }
    
    try {
      const result = await fn();
      this.onSuccess();
      return result;
    } catch (error) {
      this.onFailure();
      throw error;
    }
  }
  
  onSuccess() {
    this.failures = 0;
    this.state = 'CLOSED';
  }
  
  onFailure() {
    this.failures++;
    
    if (this.failures >= this.threshold) {
      this.state = 'OPEN';
      this.nextAttempt = Date.now() + this.timeout;
      console.log(`🔴 熔断器开启，${this.timeout}ms 后重试`);
    }
  }
}

const breaker = new CircuitBreaker(5, 60000);

customQueue.process('with-breaker', async (job) => {
  return await breaker.call(async () => {
    return await callExternalAPI(job.data);
  });
});
```

#### 第三步：死信队列实现

```javascript
// dead-letter-queue.js
import Queue from 'bull';
import nodemailer from 'nodemailer';

const mainQueue = new Queue('main', {
  redis: { host: 'localhost', port: 6379 }
});

const dlQueue = new Queue('dead-letter', {
  redis: { host: 'localhost', port: 6379 }
});

// 💡 监听失败任务，移入死信队列
mainQueue.on('failed', async (job, err) => {
  // 检查是否达到最大重试次数
  if (job.attemptsMade >= job.opts.attempts) {
    console.log(`💀 任务进入死信队列: ${job.id}`);
    
    // 💡 移入死信队列
    await dlQueue.add('dead-letter', {
      originalQueue: 'main',
      originalJobId: job.id,
      originalJobName: job.name,
      data: job.data,
      failedReason: err.message,
      stackTrace: err.stack,
      attemptsMade: job.attemptsMade,
      failedAt: Date.now(),
      logs: await job.getLogs()
    }, {
      removeOnComplete: false, // 不自动删除
      removeOnFail: false
    });
    
    // 💡 发送告警
    await sendAlert({
      subject: `任务失败告警: ${job.name}`,
      jobId: job.id,
      error: err.message,
      attempts: job.attemptsMade
    });
  }
});

// 💡 死信队列处理器（仅记录，不自动处理）
dlQueue.process('dead-letter', async (job) => {
  console.log('📝 记录死信任务:', job.data.originalJobId);
  
  // 记录到数据库
  await db.deadLetters.create({
    jobId: job.data.originalJobId,
    queue: job.data.originalQueue,
    data: job.data.data,
    error: job.data.failedReason,
    createdAt: job.data.failedAt
  });
  
  return { success: true, logged: true };
});

// 💡 查询死信任务
async function getDeadLetters(limit = 50) {
  const jobs = await dlQueue.getCompleted(0, limit);
  
  return jobs.map(job => ({
    id: job.id,
    originalJobId: job.data.originalJobId,
    queue: job.data.originalQueue,
    data: job.data.data,
    error: job.data.failedReason,
    failedAt: new Date(job.data.failedAt).toLocaleString(),
    attempts: job.data.attemptsMade
  }));
}

// 💡 手动重试死信任务
async function retryDeadLetter(deadLetterId) {
  const deadJob = await dlQueue.getJob(deadLetterId);
  
  if (!deadJob) {
    throw new Error('死信任务不存在');
  }
  
  const { originalQueue, data } = deadJob.data;
  
  // 💡 重新添加到原队列
  const newJob = await mainQueue.add(deadJob.data.originalJobName, data, {
    attempts: 3,
    backoff: {
      type: 'exponential',
      delay: 2000
    }
  });
  
  // 标记死信任务为已重试
  await deadJob.update({
    ...deadJob.data,
    retriedAt: Date.now(),
    newJobId: newJob.id
  });
  
  console.log(`✅ 死信任务已重试: ${deadLetterId} → ${newJob.id}`);
  
  return newJob;
}

// 💡 批量重试
async function retryAllDeadLetters() {
  const deadJobs = await dlQueue.getCompleted();
  const results = [];
  
  for (const deadJob of deadJobs) {
    try {
      const newJob = await retryDeadLetter(deadJob.id);
      results.push({ success: true, deadJobId: deadJob.id, newJobId: newJob.id });
    } catch (error) {
      results.push({ success: false, deadJobId: deadJob.id, error: error.message });
    }
  }
  
  return results;
}

// 💡 清理死信队列
async function cleanDeadLetters(olderThan = 30 * 24 * 3600 * 1000) {
  const jobs = await dlQueue.getCompleted();
  let cleaned = 0;
  
  for (const job of jobs) {
    const age = Date.now() - job.data.failedAt;
    
    if (age > olderThan) {
      // 归档到数据库
      await db.deadLettersArchive.create({
        jobId: job.data.originalJobId,
        data: job.data.data,
        error: job.data.failedReason,
        archivedAt: Date.now()
      });
      
      // 删除
      await job.remove();
      cleaned++;
    }
  }
  
  console.log(`🧹 已清理 ${cleaned} 个死信任务`);
  return { cleaned };
}

// 💡 发送告警
async function sendAlert(alert) {
  const transporter = nodemailer.createTransport({
    host: process.env.SMTP_HOST,
    port: 587,
    auth: {
      user: process.env.SMTP_USER,
      pass: process.env.SMTP_PASS
    }
  });
  
  await transporter.sendMail({
    from: 'alerts@example.com',
    to: 'admin@example.com',
    subject: alert.subject,
    html: `
      <h2>任务失败告警</h2>
      <p><strong>任务ID:</strong> ${alert.jobId}</p>
      <p><strong>错误信息:</strong> ${alert.error}</p>
      <p><strong>重试次数:</strong> ${alert.attempts}</p>
      <p><strong>时间:</strong> ${new Date().toLocaleString()}</p>
    `
  });
}
```

#### 第四步：任务监控与告警

```javascript
// monitoring.js
import Queue from 'bull';
import { createClient } from 'redis';

const monitorQueue = new Queue('monitor', {
  redis: { host: 'localhost', port: 6379 }
});

const redis = createClient();
await redis.connect();

// 💡 实时监控指标
class QueueMonitor {
  constructor(queue) {
    this.queue = queue;
    this.metrics = {
      completed: 0,
      failed: 0,
      totalProcessingTime: 0,
      failureRate: 0
    };
    
    this.setupListeners();
  }
  
  setupListeners() {
    // 监听完成
    this.queue.on('completed', (job, result) => {
      this.metrics.completed++;
      
      const processingTime = job.finishedOn - job.processedOn;
      this.metrics.totalProcessingTime += processingTime;
      
      this.updateMetrics();
    });
    
    // 监听失败
    this.queue.on('failed', (job, err) => {
      this.metrics.failed++;
      this.updateMetrics();
      
      // 💡 失败率过高告警
      if (this.metrics.failureRate > 0.5) { // 50%
        this.sendAlert('高失败率告警', {
          failureRate: this.metrics.failureRate,
          failed: this.metrics.failed,
          completed: this.metrics.completed
        });
      }
    });
    
    // 监听卡死
    this.queue.on('stalled', (job) => {
      console.warn(`⚠️ 任务卡死: ${job.id}`);
      this.sendAlert('任务卡死告警', { jobId: job.id });
    });
  }
  
  updateMetrics() {
    const total = this.metrics.completed + this.metrics.failed;
    this.metrics.failureRate = total > 0 ? this.metrics.failed / total : 0;
    this.metrics.avgProcessingTime = this.metrics.completed > 0 
      ? this.metrics.totalProcessingTime / this.metrics.completed 
      : 0;
  }
  
  async getMetrics() {
    const [waiting, active, completed, failed, delayed] = await Promise.all([
      this.queue.getWaitingCount(),
      this.queue.getActiveCount(),
      this.queue.getCompletedCount(),
      this.queue.getFailedCount(),
      this.queue.getDelayedCount()
    ]);
    
    return {
      waiting,
      active,
      completed,
      failed,
      delayed,
      failureRate: this.metrics.failureRate,
      avgProcessingTime: Math.round(this.metrics.avgProcessingTime)
    };
  }
  
  async sendAlert(title, data) {
    console.log(`🚨 告警: ${title}`, data);
    
    // 发送到Slack/钉钉/企业微信
    // await sendToSlack(title, data);
  }
  
  // 💡 定期报告
  startPeriodicReport(intervalMs = 60000) {
    setInterval(async () => {
      const metrics = await this.getMetrics();
      console.log('📊 队列监控报告:', metrics);
      
      // 保存到时序数据库（如InfluxDB）
      await redis.hSet('queue:metrics', {
        timestamp: Date.now(),
        ...metrics
      });
    }, intervalMs);
  }
}

// 使用
const monitor = new QueueMonitor(monitorQueue);
monitor.startPeriodicReport(60000); // 每分钟报告

// 💡 API接口
import express from 'express';
const app = express();

app.get('/api/queue/metrics', async (req, res) => {

  res.json(metrics);
});

app.get('/api/queue/failed', async (req, res) => {

  res.json({

  });
});

app.post('/api/queue/retry/:jobId', async (req, res) => {

  if (!job) {
    return res.status(404).json({ error: '任务不存在' });
  }

});

app.listen(3000);
```

## 🔧 错误处理

### 高频错误1：重试次数配置过多

**现象：**
任务失败后持续重试，占用大量资源

**原因：**
`attempts`设置过大（如100次），导致无意义的重试

**解决方案：**

```javascript
// ❌ 问题：重试次数过多
await queue.add('task', data, {
  attempts: 100, // 100次重试
  backoff: {
    type: 'exponential',
    delay: 1000
  }
});

// 问题：
// 1. 指数退避会导致极长的等待时间（2^99秒）
// 2. 占用Redis内存
// 3. 无意义的重试（错误不会自动修复）

// ✅ 正确：合理的重试次数
const RETRY_CONFIG = {
  transient: {
    // 临时性错误（网络抖动）：快速重试
    attempts: 5,
    backoff: {
      type: 'exponential',
      delay: 1000
    }
  },
  external: {
    // 外部服务（API、数据库）：中等重试
    attempts: 10,
    backoff: {
      type: 'exponential',
      delay: 2000
    }
  },
  critical: {
    // 关键任务：多次重试
    attempts: 20,
    backoff: {
      type: 'exponential',
      delay: 5000,
      options: {
        max: 300000 // 最大5分钟
      }
    }
  }
};

// ✅ 添加超时
await queue.add('api-call', data, {
  ...RETRY_CONFIG.external,
  timeout: 30000 // 单次尝试30秒超时
});

// ✅ 提前放弃不可重试的错误
queue.process(async (job) => {
  try {
    await processTask(job.data);
  } catch (error) {
    if (error.code === 'INVALID_INPUT') {
      // 参数错误，重试无意义
      job.attemptsMade = job.opts.attempts; // 直接进入DLQ
      throw error;
    }
    
    throw error; // 可重试的错误
  }
});
```

### 高频错误3：重试导致数据重复

**现象：**
任务重试时重复创建数据（如重复扣款、重复发邮件）

**原因：**
任务未实现幂等性

**解决方案：**

```javascript
// ✅ 方案1：幂等键（Idempotency Key）
import crypto from 'crypto';

async function processPayment(job) {
  const { orderId, amount } = job.data;
  
  // 💡 生成幂等键
  const idempotencyKey = crypto
    .createHash('md5')
    .update(`payment:${orderId}:${amount}`)
    .digest('hex');
  
  // 检查是否已处理
  const processed = await redis.get(`idempotency:${idempotencyKey}`);
  
  if (processed) {
    console.log('⚠️ 支付已处理，跳过');
    return JSON.parse(processed);
  }
  
  // 处理支付
  const result = await chargeCustomer(orderId, amount);
  
  // 💡 记录已处理（24小时过期）
  await redis.set(
    `idempotency:${idempotencyKey}`,
    JSON.stringify(result),
    'EX',
    86400
  );
  
  return result;
}

// ✅ 方案2：数据库唯一约束
// 在数据库层面防止重复
CREATE TABLE payments (
  order_id VARCHAR(255) UNIQUE, -- 唯一约束
  amount DECIMAL(10,2),
  status VARCHAR(50),
  created_at TIMESTAMP
);

async function processOrder(job) {
  const { orderId, items } = job.data;
  
  try {
    // 插入订单（如果重复会抛出错误）
    await db.orders.create({
      id: orderId,
      items,
      status: 'processing'
    });
  } catch (error) {
    if (error.code === 'ER_DUP_ENTRY') {
      console.log('⚠️ 订单已存在，跳过');
      return { success: true, duplicate: true };
    }
    throw error;
  }
  
  // 继续处理...
}

// ✅ 方案3：状态机
async function processTask(job) {
  const { taskId } = job.data;
  
  // 获取当前状态
  const task = await db.tasks.findOne({ id: taskId });
  
  if (task.status === 'completed') {
    console.log('⚠️ 任务已完成，跳过');
    return { success: true, skipped: true };
  }
  
  // 更新为processing（防止并发）
  const updated = await db.tasks.updateOne(
    { id: taskId, status: 'pending' }, // 只有pending才能更新
    { status: 'processing' }
  );
  
  if (updated.modifiedCount === 0) {
    console.log('⚠️ 任务正在被处理，跳过');
    return { success: true, skipped: true };
  }
  
  try {
    // 处理任务
    await doWork(job.data);
    
    // 标记为完成
    await db.tasks.updateOne(
      { id: taskId },
      { status: 'completed' }
    );
    
    return { success: true };
  } catch (error) {
    // 失败回滚为pending
    await db.tasks.updateOne(
      { id: taskId },
      { status: 'pending' }
    );
    
    throw error;
  }
}
```

## 📋 总结

### 核心要点

1. **重试策略**：指数退避（推荐）、固定间隔、自定义
2. **重试次数**：根据任务类型设置（临时5次、外部10次、关键20次）
3. **死信队列**：存储最终失败任务，防止丢失
4. **幂等性**：防止重试导致数据重复

### 实现关键

1. **合理配置**：`attempts`不宜过大，添加`timeout`
2. **条件重试**：根据错误类型决定是否重试
3. **监控告警**：实时监控失败率、死信队列
4. **定期清理**：归档或删除过期死信任务

### 应用价值

- **可靠性**：自动重试提升成功率90%+
- **容错性**：临时故障自动恢复
- **可追溯**：死信队列保留所有失败记录
- **运维友好**：批量重试、手动干预

### 面试重点

**Q1：什么是指数退避？为什么推荐？**
答：指数退避是重试间隔呈指数增长（2s, 4s, 8s...），逐渐增加间隔给系统恢复时间，避免短时间内大量重试加重压力。

**Q2：如何防止重试导致数据重复？**
答：实现幂等性，使用幂等键（Redis）、数据库唯一约束、状态机。

**Q3：死信队列的作用是什么？**
答：存储多次重试仍失败的任务，防止丢失，便于人工排查和恢复。

**Q4：如何设置合理的重试次数？**
答：根据任务类型：临时性错误5次，外部服务10次，关键任务20次。同时设置timeout和最大退避时间。

### 注意事项

⚠️ **重试次数**：不要设置过大（如100次）
⚠️ **幂等性**：重试可能导致数据重复
⚠️ **死信清理**：定期归档或删除
⚠️ **监控告警**：失败率过高及时通知
