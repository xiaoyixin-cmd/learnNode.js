# 8.1 RESTful API 设计规范

> 掌握REST架构风格和API设计最佳实践

---

## 📖 知识点概述

### 核心概念

**REST（Representational State Transfer）**是由Roy Fielding在2000年博士论文中提出的一种软件架构风格，用于设计网络应用的API。REST将网络上的一切内容抽象为"资源"，通过HTTP方法对资源进行操作。

### 解决的问题

- **API混乱**：没有统一规范，每个接口设计不一致
- **语义不清**：URL和方法名无法表达操作意图
- **难以维护**：接口众多，缺乏组织
- **版本管理**：新旧版本如何共存

### REST核心原则

1. **资源（Resource）**：一切皆资源，用URI标识
2. **表现层（Representation）**：资源的表现形式（JSON、XML）
3. **状态转移（State Transfer）**：通过HTTP方法操作资源状态
4. **无状态（Stateless）**：每个请求独立，服务器不保存客户端状态
5. **统一接口（Uniform Interface）**：使用标准HTTP方法

### 典型应用场景

- **Web API**：前后端分离的Web应用
- **移动端API**：iOS/Android App后端
- **开放平台**：提供给第三方的API
- **微服务**：服务间通信

### 理解：RESTful API的本质

#### REST vs 传统RPC对比

```javascript
// ❌ 传统RPC风格（非RESTful）
POST /api/getUser
POST /api/createUser
POST /api/updateUser
POST /api/deleteUser
POST /api/getUserList

// ✅ RESTful风格
GET    /api/users/:id    // 获取单个用户
POST   /api/users        // 创建用户
PUT    /api/users/:id    // 更新用户（完整）
PATCH  /api/users/:id    // 更新用户（部分）
DELETE /api/users/:id    // 删除用户
GET    /api/users        // 获取用户列表
```

**核心区别：**
- REST使用资源名词（users），RPC使用动词（getUser）
- REST通过HTTP方法表达操作，RPC都用POST
- REST语义清晰，RPC需要看文档

### 实践：RESTful API设计规范

#### 第一步：资源命名规范

```javascript
// ✅ 正确的资源命名
GET    /api/users                  // 用户集合
GET    /api/users/123              // 单个用户
GET    /api/users/123/posts        // 用户的文章
GET    /api/users/123/posts/456    // 用户的某篇文章
POST   /api/users                  // 创建用户
PUT    /api/users/123              // 完整更新用户
PATCH  /api/users/123              // 部分更新用户
DELETE /api/users/123              // 删除用户

// ❌ 错误的命名
GET    /api/getUsers               // 不要用动词
POST   /api/user                   // 使用复数
GET    /api/Users                  // 小写
GET    /api/user-list              // 使用users而非user-list
GET    /api/users/deleteUser/123  // 不要在URL中加动词
```

**命名原则：**
1. **使用名词复数**：`/users`而非`/user`
2. **小写字母**：`/api/users`而非`/api/Users`
3. **用连字符分隔**：`/user-profiles`而非`/userProfiles`
4. **层级清晰**：`/users/123/posts`表示用户123的文章
5. **避免动词**：用HTTP方法表达动作

#### 第二步：HTTP方法语义

```javascript
// users-api.js
import express from 'express';
import { body, param, query, validationResult } from 'express-validator';

const router = express.Router();

// 💡 GET - 获取资源（幂等、安全）
router.get('/users', [
  query('page').optional().isInt({ min: 1 }),
  query('limit').optional().isInt({ min: 1, max: 100 }),
  query('sort').optional().isIn(['createdAt', '-createdAt', 'name', '-name']),
  query('filter').optional()
], async (req, res) => {
  try {
    const errors = validationResult(req);
    if (!errors.isEmpty()) {
      return res.status(400).json({ errors: errors.array() });
    }
    
    const { page = 1, limit = 10, sort = '-createdAt', filter } = req.query;
    
    // 构建查询
    const query = {};
    if (filter) {
      query.name = { $regex: filter, $options: 'i' };
    }
    
    const total = await User.countDocuments(query);
    const users = await User.find(query)
      .sort(sort)
      .limit(limit * 1)
      .skip((page - 1) * limit)
      .select('-password'); // 不返回密码
    
    res.json({
      data: users,
      pagination: {
        page: parseInt(page),
        limit: parseInt(limit),
        total,
        totalPages: Math.ceil(total / limit),
        hasNext: page * limit < total,
        hasPrev: page > 1
      }
    });
  } catch (error) {
    console.error('获取用户列表失败:', error);
    res.status(500).json({ error: '服务器错误' });
  }
});

// 💡 GET - 获取单个资源
router.get('/users/:id', [
  param('id').isMongoId()
], async (req, res) => {
  try {
    const errors = validationResult(req);
    if (!errors.isEmpty()) {
      return res.status(400).json({ errors: errors.array() });
    }
    
    const user = await User.findById(req.params.id).select('-password');
    
    if (!user) {
      return res.status(404).json({
        error: '用户不存在',
        code: 'USER_NOT_FOUND'
      });
    }
    
    res.json({ data: user });
  } catch (error) {
    console.error('获取用户失败:', error);
    res.status(500).json({ error: '服务器错误' });
  }
});

// 💡 POST - 创建资源（非幂等）
router.post('/users', [
  body('username').trim().isLength({ min: 3, max: 20 }),
  body('email').isEmail().normalizeEmail(),
  body('password').isLength({ min: 6 }),
  body('age').optional().isInt({ min: 1, max: 150 })
], async (req, res) => {
  try {
    const errors = validationResult(req);
    if (!errors.isEmpty()) {
      return res.status(400).json({ errors: errors.array() });
    }
    
    const { username, email, password, age } = req.body;
    
    // 检查邮箱是否已存在
    const existingUser = await User.findOne({ email });
    if (existingUser) {
      return res.status(409).json({
        error: '邮箱已被注册',
        code: 'EMAIL_EXISTS'
      });
    }
    
    // 加密密码
    const hashedPassword = await bcrypt.hash(password, 10);
    
    // 创建用户
    const user = await User.create({
      username,
      email,
      password: hashedPassword,
      age
    });
    
    // 201 Created
    res.status(201)
      .location(`/api/users/${user._id}`) // Location头指向新资源
      .json({
        data: {
          id: user._id,
          username: user.username,
          email: user.email,
          age: user.age,
          createdAt: user.createdAt
        }
      });
  } catch (error) {
    console.error('创建用户失败:', error);
    res.status(500).json({ error: '服务器错误' });
  }
});

// 💡 PUT - 完整更新资源（幂等）
router.put('/users/:id', [
  param('id').isMongoId(),
  body('username').trim().isLength({ min: 3, max: 20 }),
  body('email').isEmail().normalizeEmail(),
  body('age').optional().isInt({ min: 1, max: 150 })
], async (req, res) => {
  try {
    const errors = validationResult(req);
    if (!errors.isEmpty()) {
      return res.status(400).json({ errors: errors.array() });
    }
    
    const { username, email, age } = req.body;
    
    // PUT应该替换整个资源
    const user = await User.findByIdAndUpdate(
      req.params.id,
      { username, email, age }, // 只更新这些字段
      {
        new: true,
        overwrite: true, // 完全替换
        runValidators: true
      }
    ).select('-password');
    
    if (!user) {
      return res.status(404).json({
        error: '用户不存在',
        code: 'USER_NOT_FOUND'
      });
    }
    
    res.json({ data: user });
  } catch (error) {
    console.error('更新用户失败:', error);
    res.status(500).json({ error: '服务器错误' });
  }
});

// 💡 PATCH - 部分更新资源（幂等）
router.patch('/users/:id', [
  param('id').isMongoId(),
  body('username').optional().trim().isLength({ min: 3, max: 20 }),
  body('email').optional().isEmail().normalizeEmail(),
  body('age').optional().isInt({ min: 1, max: 150 })
], async (req, res) => {
  try {
    const errors = validationResult(req);
    if (!errors.isEmpty()) {
      return res.status(400).json({ errors: errors.array() });
    }
    
    // PATCH只更新提供的字段
    const updates = {};
    if (req.body.username) updates.username = req.body.username;
    if (req.body.email) updates.email = req.body.email;
    if (req.body.age !== undefined) updates.age = req.body.age;
    
    const user = await User.findByIdAndUpdate(
      req.params.id,
      { $set: updates },
      {
        new: true,
        runValidators: true
      }
    ).select('-password');
    
    if (!user) {
      return res.status(404).json({
        error: '用户不存在',
        code: 'USER_NOT_FOUND'
      });
    }
    
    res.json({ data: user });
  } catch (error) {
    console.error('更新用户失败:', error);
    res.status(500).json({ error: '服务器错误' });
  }
});

// 💡 DELETE - 删除资源（幂等）
router.delete('/users/:id', [
  param('id').isMongoId()
], async (req, res) => {
  try {
    const errors = validationResult(req);
    if (!errors.isEmpty()) {
      return res.status(400).json({ errors: errors.array() });
    }
    
    const user = await User.findByIdAndDelete(req.params.id);
    
    if (!user) {
      return res.status(404).json({
        error: '用户不存在',
        code: 'USER_NOT_FOUND'
      });
    }
    
    // 204 No Content（无响应体）
    res.status(204).send();
  } catch (error) {
    console.error('删除用户失败:', error);
    res.status(500).json({ error: '服务器错误' });
  }
});

export default router;
```

**HTTP方法特性：**

| 方法 | 用途 | 幂等 | 安全 | 响应码 |
|------|------|------|------|--------|
| GET | 获取资源 | ✅ | ✅ | 200 |
| POST | 创建资源 | ❌ | ❌ | 201 |
| PUT | 完整更新 | ✅ | ❌ | 200 |
| PATCH | 部分更新 | ✅ | ❌ | 200 |
| DELETE | 删除资源 | ✅ | ❌ | 204 |

- **幂等**：多次请求结果相同
- **安全**：不修改服务器状态

```javascript
    "details": [
      {
        "field": "email",
        "message": "邮箱格式不正确",
        "value": "invalid-email"
      },
      {
        "field": "password",
        "message": "密码至少6个字符"

  "meta": {
    "timestamp": "2025-01-15T10:30:00Z",
    "requestId": "abc-123-def"
  }
}

// 💡 实现统一响应中间件
function responseFormatter(req, res, next) {
  // 成功响应
  res.success = (data, statusCode = 200, meta = {}) => {
    res.status(statusCode).json({
      success: true,
      data,
      meta: {
        timestamp: new Date().toISOString(),
        ...meta
      }
    });
  };
  
  // 分页响应
  res.paginated = (data, pagination, statusCode = 200) => {
    res.status(statusCode).json({
      success: true,
      data,
      pagination,
      meta: {
        timestamp: new Date().toISOString()
      }
    });
  };
  
  // 错误响应
  res.error = (message, statusCode = 500, code = 'ERROR', details = null) => {
    res.status(statusCode).json({
      success: false,
      error: {
        code,
        message,
        ...(details && { details })
      },
      meta: {
        timestamp: new Date().toISOString(),
        requestId: req.id // 需要request-id中间件
      }
    });
  };
  
  next();
}

// 使用
app.use(responseFormatter);

router.get('/users/:id', async (req, res) => {
  const user = await User.findById(req.params.id);
  
  if (!user) {
    return res.error('用户不存在', 404, 'USER_NOT_FOUND');
  }
  
  res.success(user);
});
```

## 🔧 错误处理

### 高频错误1：URL设计不规范

**错误示例：**
```javascript
// ❌ 使用动词
POST /api/createUser
GET  /api/getUsers
POST /api/deleteUser/123

// ❌ 单数名词
GET /api/user/123

// ❌ 混用大小写
GET /api/Users
GET /api/userProfiles
```

**正确示例：**
```javascript
// ✅ 使用名词复数 + HTTP方法
POST   /api/users
GET    /api/users
DELETE /api/users/123

// ✅ 全小写，连字符分隔
GET /api/user-profiles
```

### 高频错误2：状态码使用不当

**错误示例：**
```javascript
// ❌ 所有请求都返回200
router.post('/users', async (req, res) => {
  try {
    const user = await User.create(req.body);
    res.status(200).json({ success: true, data: user }); // 应该是201
  } catch (error) {
    res.status(200).json({ success: false, error: error.message }); // 应该是4xx或5xx
  }
});

// ❌ DELETE返回200和数据
router.delete('/users/:id', async (req, res) => {
  await User.findByIdAndDelete(req.params.id);
  res.status(200).json({ message: '删除成功' }); // 应该是204 No Content
});
```

**正确示例：**
```javascript
// ✅ 使用正确的状态码
router.post('/users', async (req, res) => {
  try {
    const user = await User.create(req.body);
    res.status(201).json({ data: user }); // 201 Created
  } catch (error) {
    if (error.code === 11000) {
      res.status(409).json({ error: '资源冲突' }); // 409 Conflict
    } else {
      res.status(500).json({ error: '服务器错误' }); // 500 Server Error
    }
  }
});

// ✅ DELETE返回204
router.delete('/users/:id', async (req, res) => {
  await User.findByIdAndDelete(req.params.id);
  res.status(204).send(); // 204 No Content
});
```

## 🎯 应用场景

### 场景1：电商平台RESTful API

**问题背景：**
电商平台有商品、订单、用户、购物车等多个资源，需要设计一套清晰、易维护的API体系。

**完整实现：**

```javascript
// ecommerce-api.js
import express from 'express';

const app = express();
app.use(express.json());

// 💡 商品API
const productsRouter = express.Router();

// 获取商品列表（支持分页、排序、过滤）
productsRouter.get('/', async (req, res) => {
  const {
    page = 1,
    limit = 20,
    sort = '-createdAt',
    category,
    minPrice,
    maxPrice,
    q // 搜索关键词
  } = req.query;
  
  const filter = {};
  if (category) filter.category = category;
  if (minPrice || maxPrice) {
    filter.price = {};
    if (minPrice) filter.price.$gte = parseFloat(minPrice);
    if (maxPrice) filter.price.$lte = parseFloat(maxPrice);
  }
  if (q) {
    filter.$text = { $search: q };
  }
  
  const [products, total] = await Promise.all([
    Product.find(filter)
      .sort(sort)
      .limit(limit * 1)
      .skip((page - 1) * limit),
    Product.countDocuments(filter)
  ]);
  
  res.json({
    data: products,
    pagination: {
      page: parseInt(page),
      limit: parseInt(limit),
      total,
      totalPages: Math.ceil(total / limit)
    }
  });
});

// 获取单个商品
productsRouter.get('/:id', async (req, res) => {
  const product = await Product.findById(req.params.id);
  
  if (!product) {
    return res.status(404).json({
      error: '商品不存在',
      code: 'PRODUCT_NOT_FOUND'
    });
  }
  
  res.json({ data: product });
});

// 创建商品（需要管理员权限）
productsRouter.post('/', requireAdmin, async (req, res) => {
  const product = await Product.create(req.body);
  res.status(201).json({ data: product });
});

// 更新商品
productsRouter.patch('/:id', requireAdmin, async (req, res) => {
  const product = await Product.findByIdAndUpdate(
    req.params.id,
    { $set: req.body },
    { new: true, runValidators: true }
  );
  
  if (!product) {
    return res.status(404).json({ error: '商品不存在' });
  }
  
  res.json({ data: product });
});

// 删除商品
productsRouter.delete('/:id', requireAdmin, async (req, res) => {
  const product = await Product.findByIdAndDelete(req.params.id);
  
  if (!product) {
    return res.status(404).json({ error: '商品不存在' });
  }
  
  res.status(204).send();
});

// 💡 订单API
const ordersRouter = express.Router();

// 获取用户的订单列表
ordersRouter.get('/', requireAuth, async (req, res) => {
  const { status, page = 1, limit = 10 } = req.query;
  
  const filter = { userId: req.user.id };
  if (status) filter.status = status;
  
  const orders = await Order.find(filter)
    .sort('-createdAt')
    .limit(limit * 1)
    .skip((page - 1) * limit)
    .populate('items.productId');
  
  res.json({ data: orders });
});

// 创建订单
ordersRouter.post('/', requireAuth, async (req, res) => {
  const { items, shippingAddress } = req.body;
  
  // 计算总价
  let totalPrice = 0;
  for (const item of items) {
    const product = await Product.findById(item.productId);
    if (!product) {
      return res.status(400).json({ error: `商品${item.productId}不存在` });
    }
    totalPrice += product.price * item.quantity;
  }
  
  // 创建订单
  const order = await Order.create({
    userId: req.user.id,
    items,
    totalPrice,
    shippingAddress,
    status: 'pending'
  });
  
  res.status(201).json({ data: order });
});

// 💡 购物车API
const cartRouter = express.Router();

// 获取购物车
cartRouter.get('/', requireAuth, async (req, res) => {
  const cart = await Cart.findOne({ userId: req.user.id })
    .populate('items.productId');
  
  res.json({ data: cart || { items: [] } });
});

// 添加商品到购物车
cartRouter.post('/items', requireAuth, async (req, res) => {
  const { productId, quantity = 1 } = req.body;
  
  let cart = await Cart.findOne({ userId: req.user.id });
  
  if (!cart) {
    cart = await Cart.create({
      userId: req.user.id,
      items: []
    });
  }
  
  // 检查商品是否已在购物车
  const existingItem = cart.items.find(
    item => item.productId.toString() === productId
  );
  
  if (existingItem) {
    existingItem.quantity += quantity;
  } else {
    cart.items.push({ productId, quantity });
  }
  
  await cart.save();
  await cart.populate('items.productId');
  
  res.json({ data: cart });
});

// 删除购物车商品
cartRouter.delete('/items/:productId', requireAuth, async (req, res) => {
  const cart = await Cart.findOne({ userId: req.user.id });
  
  if (!cart) {
    return res.status(404).json({ error: '购物车不存在' });
  }
  
  cart.items = cart.items.filter(
    item => item.productId.toString() !== req.params.productId
  );
  
  await cart.save();
  
  res.status(204).send();
});

// 挂载路由
app.use('/api/products', productsRouter);
app.use('/api/orders', ordersRouter);
app.use('/api/cart', cartRouter);

app.listen(3000);
```

| | GET | `/api/products/:id` | 获取单个商品 |

| **订单** | GET | `/api/orders` | 获取订单列表 |
| | POST | `/api/orders` | 创建订单 |

| | PATCH | `/api/orders/:id` | 更新订单状态 |
| **购物车** | GET | `/api/cart` | 获取购物车 |
| | POST | `/api/cart/items` | 添加商品 |
| | PATCH | `/api/cart/items/:id` | 更新数量 |
| | DELETE | `/api/cart/items/:id` | 删除商品 |

## 📋 总结

### 核心要点

1. **资源导向**：使用名词复数，不用动词
2. **HTTP方法**：GET（查）、POST（增）、PUT/PATCH（改）、DELETE（删）
3. **状态码**：合理使用HTTP状态码表达结果
4. **统一格式**：成功和错误响应格式一致

### 实现关键

1. **URL命名**：小写、复数、连字符、层级清晰
2. **幂等性**：GET、PUT、DELETE是幂等的
3. **版本控制**：URL版本（`/api/v1`）最直观
4. **查询参数**：分页、排序、过滤、搜索

### 应用价值

- **易理解**：清晰的语义，自解释的API
- **易维护**：统一规范，降低维护成本
- **易扩展**：资源化设计，方便添加新功能
- **易测试**：标准HTTP协议，工具支持好

### 面试重点

**Q1：PUT和PATCH有什么区别？**
答：PUT是完整更新（替换整个资源），PATCH是部分更新（只更新提供的字段）。PUT是幂等的，多次调用结果相同。

**Q2：什么是幂等性？哪些HTTP方法是幂等的？**
答：幂等性指多次执行相同操作，结果相同。GET、PUT、DELETE是幂等的，POST不是（多次POST会创建多个资源）。

**Q3：RESTful API如何做版本控制？**
答：三种方式：(1)URL版本`/api/v1/users`（推荐）(2)Header版本`API-Version: v1` (3)Accept Header`application/vnd.api.v1+json`

**Q4：如何设计分页接口？**
答：使用`page`和`limit`参数，返回数据+分页信息（总数、总页数、当前页、是否有下一页）

### 注意事项

⚠️ **不要在URL中使用动词**：用HTTP方法表达动作
⚠️ **不要所有请求都返回200**：合理使用4xx和5xx
⚠️ **不要忽略幂等性**：PUT和DELETE应该设计为幂等
⚠️ **不要返回敏感信息**：如密码字段
