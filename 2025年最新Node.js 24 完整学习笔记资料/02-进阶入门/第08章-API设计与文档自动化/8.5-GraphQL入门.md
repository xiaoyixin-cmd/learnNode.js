## 📖 知识点概述

### 核心概念

**GraphQL**是Facebook于2015年开源的查询语言和运行时，用于API开发。与RESTful不同，GraphQL允许客户端精确指定需要的数据，避免过度获取或获取不足。

### 解决的问题

- **过度获取**：REST返回所有字段，浪费带宽
- **获取不足**：需要多次请求才能获取完整数据
- **版本管理**：REST需要维护多个版本
- **文档维护**：GraphQL自带类型系统和文档

### GraphQL vs REST

| 维度 | REST | GraphQL |
|------|------|---------|
| **端点** | 多个端点 | 单一端点 |
| **数据获取** | 固定结构 | 按需获取 |
| **过度获取** | 常见 | 不存在 |
| **请求数** | 多次请求 | 一次请求 |
| **版本控制** | URL版本 | 无需版本 |
| **学习曲线** | 低 | 中等 |

### 典型应用场景

- **移动应用**：减少请求次数，节省流量
- **聚合API**：整合多个数据源
- **前端驱动**：前端决定需要哪些数据
- **实时应用**：Subscription订阅实时数据

#### 第一步：Schema定义

```javascript
// schema.js
import { gql } from 'graphql-tag';

// 💡 定义类型系统
export const typeDefs = gql`
  # 用户类型
  type User {
    id: ID!
    username: String!
    email: String!
    age: Int
    role: Role!
    posts: [Post!]!
    createdAt: String!
  }
  
  # 角色枚举
  enum Role {
    ADMIN
    USER
    GUEST
  }
  
  # 文章类型
  type Post {
    id: ID!
    title: String!
    content: String!
    author: User!
    published: Boolean!
    createdAt: String!
  }
  
  # 输入类型（创建用户）
  input CreateUserInput {
    username: String!
    email: String!
    password: String!
    age: Int
  }
  
  # 输入类型（更新用户）
  input UpdateUserInput {
    username: String
    email: String
    age: Int
  }
  
  # 查询
  type Query {
    # 获取所有用户
    users(limit: Int, offset: Int): [User!]!
    
    # 获取单个用户
    user(id: ID!): User
    
    # 搜索用户
    searchUsers(query: String!): [User!]!
    
    # 获取所有文章
    posts: [Post!]!
    
    # 获取单个文章
    post(id: ID!): Post
  }
  
  # 变更（修改数据）
  type Mutation {
    # 创建用户
    createUser(input: CreateUserInput!): User!
    
    # 更新用户
    updateUser(id: ID!, input: UpdateUserInput!): User!
    
    # 删除用户
    deleteUser(id: ID!): Boolean!
    
    # 创建文章
    createPost(title: String!, content: String!): Post!
  }
  
  # 订阅（实时数据）
  type Subscription {
    # 用户创建事件
    userCreated: User!
    
    # 文章发布事件
    postPublished: Post!
  }
`;
```

#### 第二步：Resolvers实现

```javascript
// resolvers.js

// 模拟数据库
let users = [
  {
    id: '1',
    username: 'john',
    email: 'john@example.com',
    age: 25,
    role: 'USER',
    createdAt: new Date().toISOString()
  },
  {
    id: '2',
    username: 'jane',
    email: 'jane@example.com',
    age: 30,
    role: 'ADMIN',
    createdAt: new Date().toISOString()
  }
];

let posts = [
  {
    id: '1',
    title: 'First Post',
    content: 'Hello GraphQL!',
    authorId: '1',
    published: true,
    createdAt: new Date().toISOString()
  }
];

// 💡 Resolvers（解析器）
export const resolvers = {
  // 查询解析器
  Query: {
    // 获取所有用户
    users: (parent, args, context, info) => {
      const { limit = 10, offset = 0 } = args;
      return users.slice(offset, offset + limit);
    },
    
    // 获取单个用户
    user: (parent, args) => {
      return users.find(user => user.id === args.id);
    },
    
    // 搜索用户
    searchUsers: (parent, args) => {
      const query = args.query.toLowerCase();
      return users.filter(user =>
        user.username.toLowerCase().includes(query) ||
        user.email.toLowerCase().includes(query)
      );
    },
    
    // 获取所有文章
    posts: () => posts,
    
    // 获取单个文章
    post: (parent, args) => {
      return posts.find(post => post.id === args.id);
    }
  },
  
  // 变更解析器
  Mutation: {
    // 创建用户
    createUser: (parent, args) => {
      const newUser = {
        id: String(users.length + 1),
        ...args.input,
        role: 'USER',
        createdAt: new Date().toISOString()
      };
      
      // 检查邮箱是否已存在
      const existingUser = users.find(u => u.email === args.input.email);
      if (existingUser) {
        throw new Error('邮箱已存在');
      }
      
      users.push(newUser);
      
      // 触发订阅（如果有）
      // pubsub.publish('USER_CREATED', { userCreated: newUser });
      
      return newUser;
    },
    
    // 更新用户
    updateUser: (parent, args) => {
      const userIndex = users.findIndex(u => u.id === args.id);
      
      if (userIndex === -1) {
        throw new Error('用户不存在');
      }
      
      users[userIndex] = {
        ...users[userIndex],
        ...args.input
      };
      
      return users[userIndex];
    },
    
    // 删除用户
    deleteUser: (parent, args) => {
      const userIndex = users.findIndex(u => u.id === args.id);
      
      if (userIndex === -1) {
        return false;
      }
      
      users.splice(userIndex, 1);
      return true;
    },
    
    // 创建文章
    createPost: (parent, args, context) => {
      // 从context获取当前用户
      if (!context.user) {
        throw new Error('需要登录');
      }
      
      const newPost = {
        id: String(posts.length + 1),
        title: args.title,
        content: args.content,
        authorId: context.user.id,
        published: false,
        createdAt: new Date().toISOString()
      };
      
      posts.push(newPost);
      return newPost;
    }
  },
  
  // 💡 字段解析器（解析关联数据）
  User: {
    // 获取用户的文章
    posts: (parent) => {
      return posts.filter(post => post.authorId === parent.id);
    }
  },
  
  Post: {
    // 获取文章作者
    author: (parent) => {
      return users.find(user => user.id === parent.authorId);
    }
  }
};
```

#### 第三步：启动Apollo Server

```javascript
// server.js
import { ApolloServer } from '@apollo/server';
import { startStandaloneServer } from '@apollo/server/standalone';
import { typeDefs } from './schema.js';
import { resolvers } from './resolvers.js';

// 💡 创建Apollo Server
const server = new ApolloServer({
  typeDefs,
  resolvers,
  // 格式化错误
  formatError: (formattedError, error) => {
    // 隐藏内部错误详情
    if (formattedError.message.startsWith('Database Error: ')) {
      return { message: '内部服务器错误' };
    }
    return formattedError;
  }
});

// 💡 启动服务器
const { url } = await startStandaloneServer(server, {
  listen: { port: 4000 },
  // Context函数（每个请求调用）
  context: async ({ req }) => {
    // 从请求头获取token
    const token = req.headers.authorization || '';
    
    // 验证token并返回用户信息
    // const user = await getUserFromToken(token);
    
    return {
      // user,
      // 其他context数据
    };
  }
});

console.log(`🚀 GraphQL Server ready at ${url}`);
console.log(`📝 GraphQL Playground: ${url}`);
```

#### 第四步：Express集成

```javascript
// server-express.js
import express from 'express';
import { ApolloServer } from '@apollo/server';
import { expressMiddleware } from '@apollo/server/express4';
import cors from 'cors';
import bodyParser from 'body-parser';
import { typeDefs } from './schema.js';
import { resolvers } from './resolvers.js';

const app = express();

// 💡 创建Apollo Server
const server = new ApolloServer({
  typeDefs,
  resolvers
});

await server.start();

// 中间件
app.use(cors());
app.use(bodyParser.json());

// 💡 GraphQL端点
app.use('/graphql', expressMiddleware(server, {
  context: async ({ req }) => {
    const token = req.headers.authorization || '';
    // 验证token
    return { token };
  }
}));

// REST端点（可以共存）
app.get('/api/health', (req, res) => {
  res.json({ status: 'ok' });
});

app.listen(4000, () => {
  console.log('🚀 Server ready at http://localhost:4000/graphql');
});
```

## 💡 GraphQL查询示例

### 查询（Query）

```graphql
# 💡 获取所有用户（只要需要的字段）
query {
  users {
    id
    username
    email
  }
}

# 💡 获取单个用户及其文章
query {
  user(id: "1") {
    id
    username
    email
    posts {
      id
      title
      published
    }
  }
}

# 💡 使用变量
query GetUser($userId: ID!) {
  user(id: $userId) {
    username
    email
  }
}

# Variables:
# {
#   "userId": "1"
# }

# 💡 使用别名
query {
  user1: user(id: "1") {
    username
  }
  user2: user(id: "2") {
    username
  }
}

# 💡 使用Fragment（复用字段）
fragment UserFields on User {
  id
  username
  email
  createdAt
}

query {
  user1: user(id: "1") {
    ...UserFields
  }
  user2: user(id: "2") {
    ...UserFields
  }
}
```

## 🔧 高级功能

### 分页（Pagination）

```javascript
// schema.js
type Query {
  users(
    limit: Int = 10
    offset: Int = 0
  ): UserConnection!
}

type UserConnection {
  edges: [UserEdge!]!
  pageInfo: PageInfo!
  totalCount: Int!
}

type UserEdge {
  node: User!
  cursor: String!
}

type PageInfo {
  hasNextPage: Boolean!
  hasPreviousPage: Boolean!
  startCursor: String
  endCursor: String
}

// resolvers.js
Query: {
  users: (parent, args) => {
    const { limit, offset } = args;
    const allUsers = users;
    const paginatedUsers = allUsers.slice(offset, offset + limit);
    
    return {
      edges: paginatedUsers.map((user, index) => ({
        node: user,
        cursor: Buffer.from(`${offset + index}`).toString('base64')
      })),
      pageInfo: {
        hasNextPage: offset + limit < allUsers.length,
        hasPreviousPage: offset > 0,
        startCursor: Buffer.from(`${offset}`).toString('base64'),
        endCursor: Buffer.from(`${offset + limit - 1}`).toString('base64')
      },
      totalCount: allUsers.length
    };
  }
}
```

// dataloader.js
import DataLoader from 'dataloader';

// 批量加载用户
const batchUsers = async (userIds) => {
  const users = await User.find({ _id: { $in: userIds } });
  // 保持顺序
  return userIds.map(id => users.find(user => user.id === id));
};

// 创建DataLoader
export const userLoader = new DataLoader(batchUsers);

// 在resolver中使用
Post: {
  author: (parent, args, context) => {
    return context.userLoader.load(parent.authorId);
  }
}

## 📋 总结

### 核心要点

1. **GraphQL**：查询语言，客户端按需获取数据
2. **Schema**：类型系统，定义数据结构
3. **Resolver**：解析器，实现数据获取逻辑
4. **Apollo Server**：最流行的GraphQL服务器

### GraphQL优势

- **按需获取**：避免过度获取和获取不足
- **单一端点**：只需一个/graphql端点
- **强类型**：类型系统提供自动验证和文档
- **实时数据**：Subscription支持实时推送

### 适用场景

- **移动应用**：减少请求次数和数据传输
- **复杂查询**：需要聚合多个数据源
- **快速迭代**：前端可以独立调整数据需求
- **实时应用**：需要WebSocket推送

### 面试重点

**Q1：GraphQL和REST的区别？**
答：GraphQL单一端点、按需获取数据、强类型系统；REST多端点、固定结构、容易过度获取

**Q2：什么是N+1问题？如何解决？**
答：查询列表时，对每个item再查询关联数据，导致N+1次查询。使用DataLoader批量加载解决

**Q3：GraphQL的缺点？**
答：学习曲线较陡、缓存复杂（不能用HTTP缓存）、查询复杂度难控制（需要限制深度）
