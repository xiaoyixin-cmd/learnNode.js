# 11.4 Worker Threads（多线程编程）

> Worker Threads原理、主线程与工作线程通信、SharedArrayBuffer、CPU密集任务处理、线程池模式

---

## 📖 知识点概述

### 核心概念

**Worker Threads**是Node.js 10.5+引入的多线程API，允许在独立线程中执行JavaScript代码。不同于Cluster的多进程，Worker Threads共享内存，开销更小，特别适合CPU密集型任务。

### Worker Threads vs Cluster

```
┌──────────────────────────────────────────────────────────┐
│          Worker Threads vs Cluster 对比                   │
├──────────────────────────────────────────────────────────┤
│                                                            │
│  Worker Threads（多线程）                                  │
│  ├─ 共享内存（SharedArrayBuffer）                         │
│  ├─ 开销小（共享V8实例）                                   │
│  ├─ 适合：CPU密集任务                                     │
│  └─ 示例：图像处理、加密、复杂计算                         │
│                                                            │
│  Cluster（多进程）                                         │
│  ├─ 完全隔离（独立内存）                                   │
│  ├─ 开销大（独立V8实例）                                   │
│  ├─ 适合：Web服务、负载均衡                               │
│  └─ 示例：HTTP服务器、API服务                             │
│                                                            │
└──────────────────────────────────────────────────────────┘
```

### Worker Threads架构

| 维度 | 主线程 | 工作线程 |
|------|--------|----------|
| **职责** | 调度任务 | 执行任务 |
| **数量** | 1个 | 多个 |
| **内存** | 主内存 | 独立内存+共享内存 |
| **通信** | postMessage | postMessage/SharedArrayBuffer |
| **Event Loop** | 独立 | 独立 |

### 为什么需要Worker Threads？

```javascript
// ❌ 没有Worker Threads：CPU密集任务阻塞主线程
import express from 'express';
import crypto from 'crypto';

const app = express();

app.get('/hash', (req, res) => {
  const hash = crypto.pbkdf2Sync('password', 'salt', 100000, 64, 'sha512');
  res.send(hash.toString('hex'));
  // 问题：这个请求需要1秒，期间所有其他请求都被阻塞
});

app.get('/ping', (req, res) => {
  res.send('pong');
  // 问题：/hash 执行期间，/ping 也无法响应
});

// ✅ 使用Worker Threads：CPU任务在工作线程执行
import { Worker } from 'worker_threads';

app.get('/hash', (req, res) => {
  const worker = new Worker('./hash-worker.js');
  
  worker.on('message', (hash) => {
    res.send(hash);
  });
  
  worker.postMessage({ password: 'password', salt: 'salt' });
  // 优点：主线程不被阻塞，/ping 仍能快速响应
});
```

## 🚀 使用指南

### 理解：基础概念

#### 最简单的Worker Threads示例

```javascript
// main.js（主线程）
import { Worker } from 'worker_threads';

console.log('主线程开始');

// 💡 创建工作线程
const worker = new Worker('./worker.js');

// 💡 接收工作线程消息
worker.on('message', (msg) => {
  console.log('收到工作线程消息:', msg);
});

// 💡 监听工作线程错误
worker.on('error', (err) => {
  console.error('工作线程错误:', err);
});

// 💡 监听工作线程退出
worker.on('exit', (code) => {
  console.log(`工作线程退出，代码: ${code}`);
});

// 💡 向工作线程发送消息
worker.postMessage('Hello from main thread');

// worker.js（工作线程）
import { parentPort } from 'worker_threads';

console.log('工作线程开始');

// 💡 接收主线程消息
parentPort.on('message', (msg) => {
  console.log('收到主线程消息:', msg);
  
  // 💡 向主线程发送消息
  parentPort.postMessage('Hello from worker thread');
});

// 运行：node main.js
// 输出：
// 主线程开始
// 工作线程开始
// 收到主线程消息: Hello from main thread
// 收到工作线程消息: Hello from worker thread
// 工作线程退出，代码: 0
```

    });

function compressImage(inputPath, outputPath, quality) {
  return new Promise((resolve, reject) => {
    const worker = new Worker('./image-worker.js', {
      workerData: { inputPath, outputPath, quality }
    });
    
    worker.on('message', resolve);
    worker.on('error', reject);
    worker.on('exit', (code) => {
      if (code !== 0) {
        reject(new Error(`Worker stopped with exit code ${code}`));
      }
    });
  });
}

app.listen(3000);

import { parentPort, workerData } from 'worker_threads';
import sharp from 'sharp';
import fs from 'fs/promises';

const { inputPath, outputPath, quality } = workerData;

async function processImage() {
  try {
    // 💡 CPU密集：图像处理
    await sharp(inputPath)
      .jpeg({ quality })
      .toFile(outputPath);
    
    const stats = await fs.stat(outputPath);
    
    parentPort.postMessage({
      size: stats.size,
      path: outputPath
    });
  } catch (error) {
    parentPort.postMessage({ error: error.message });
  }
}

processImage();

import express from 'express';

const app = express();
app.use(express.json());

      operation: 'decrypt',
      encrypted,
      password
    });
    console.timeEnd('decrypt');
    
    res.json({ decrypted });
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});

function runWorker(workerScript, workerData) {
  return new Promise((resolve, reject) => {
    const worker = new Worker(workerScript, { workerData });
    
    worker.on('message', resolve);
    worker.on('error', reject);
  });
}

app.listen(3000);

// crypto-task.js（工作线程）
import { parentPort, workerData } from 'worker_threads';
import crypto from 'crypto';

const { operation, text, encrypted, password } = workerData;

const algorithm = 'aes-256-cbc';
const key = crypto.scryptSync(password, 'salt', 32);
const iv = Buffer.alloc(16, 0);

if (operation === 'encrypt') {
  // 💡 CPU密集：加密
  const cipher = crypto.createCipheriv(algorithm, key, iv);
  let result = cipher.update(text, 'utf8', 'hex');
  result += cipher.final('hex');
  
  parentPort.postMessage(result);
} else if (operation === 'decrypt') {
  // 💡 CPU密集：解密
  const decipher = crypto.createDecipheriv(algorithm, key, iv);
  let result = decipher.update(encrypted, 'hex', 'utf8');
  result += decipher.final('utf8');
  
  parentPort.postMessage(result);
}

### 实践：线程池模式

#### 简单线程池实现

```javascript
// thread-pool.js
import { Worker } from 'worker_threads';
import os from 'os';
import path from 'path';

export class WorkerPool {
  constructor(workerScript, poolSize = os.cpus().length) {
    this.workerScript = path.resolve(workerScript);
    this.poolSize = poolSize;
    this.workers = [];
    this.taskQueue = [];
    
    // 💡 创建固定数量的工作线程
    for (let i = 0; i < poolSize; i++) {
      this.addWorker();
    }
  }
  
  addWorker() {
    const worker = {
      instance: new Worker(this.workerScript),
      busy: false,
      currentTask: null
    };
    
    worker.instance.on('message', (result) => {
      if (worker.currentTask) {
        worker.currentTask.resolve(result);
        worker.busy = false;
        worker.currentTask = null;
        
        // 处理队列中的下一个任务
        this.processQueue();
      }
    });
    
    worker.instance.on('error', (err) => {
      if (worker.currentTask) {
        worker.currentTask.reject(err);
        worker.busy = false;
        worker.currentTask = null;
      }
      
      // 重新创建工作线程
      this.workers = this.workers.filter(w => w !== worker);
      this.addWorker();
    });
    
    this.workers.push(worker);
  }
  
  // 💡 执行任务
  exec(data) {
    return new Promise((resolve, reject) => {
      const task = { data, resolve, reject };
      
      // 找到空闲的工作线程
      const availableWorker = this.workers.find(w => !w.busy);
      
      if (availableWorker) {
        this.runTask(availableWorker, task);
      } else {
        // 加入队列
        this.taskQueue.push(task);
      }
    });
  }
  
  runTask(worker, task) {
    worker.busy = true;
    worker.currentTask = task;
    worker.instance.postMessage(task.data);
  }
  
  processQueue() {
    if (this.taskQueue.length === 0) return;
    
    const availableWorker = this.workers.find(w => !w.busy);
    if (availableWorker) {
      const task = this.taskQueue.shift();
      this.runTask(availableWorker, task);
    }
  }
  
  // 💡 关闭线程池
  async destroy() {
    await Promise.all(
      this.workers.map(w => w.instance.terminate())
    );
    this.workers = [];
    this.taskQueue = [];
  }
}

// 使用示例
// app.js
import { WorkerPool } from './thread-pool.js';
import express from 'express';

const app = express();

// 💡 创建线程池（4个工作线程）
const pool = new WorkerPool('./compute-worker.js', 4);

app.get('/compute', async (req, res) => {
  try {
    const { n } = req.query;
    
    console.time('compute');
    const result = await pool.exec({ n: parseInt(n) });
    console.timeEnd('compute');
    
    res.json({ result });
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});

app.listen(3000, () => {
  console.log('Server running on port 3000');
  console.log(`Thread pool size: ${pool.poolSize}`);
});

// 优雅关闭
process.on('SIGTERM', async () => {
  console.log('Shutting down...');
  await pool.destroy();
  process.exit(0);
});

// compute-worker.js
import { parentPort } from 'worker_threads';

parentPort.on('message', ({ n }) => {
  // 计算斐波那契数列（CPU密集）
  function fib(n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
  }
  
  const result = fib(n);
  parentPort.postMessage(result);

```

## 🔧 错误处理

### 高频错误1：工作线程未正确退出

**现象：**
```
Warning: Worker thread did not exit gracefully
```

**解决方案：**
```javascript
// ❌ 错误：工作线程有定时器未清理
// worker.js
import { parentPort } from 'worker_threads';

setInterval(() => {
  console.log('tick');
}, 1000);

parentPort.on('message', (msg) => {
  parentPort.postMessage('done');
  // ❌ 定时器未清理，线程无法退出
});

// ✅ 正确：清理定时器
// worker.js
import { parentPort } from 'worker_threads';

let timer = null;

parentPort.on('message', (msg) => {
  if (msg === 'start') {
    timer = setInterval(() => {
      parentPort.postMessage('tick');
    }, 1000);
  } else if (msg === 'stop') {
    if (timer) {
      clearInterval(timer);
      timer = null;
    }
    parentPort.postMessage('stopped');
    process.exit(0);  // 显式退出
  }
});
```

## 📋 总结

### 核心要点

1. **多线程**：共享内存、低开销
2. **CPU密集任务**：图像处理、加密、复杂计算
3. **线程池**：复用Worker，控制并发
4. **SharedArrayBuffer**：高性能内存共享

### 实现关键

1. **创建Worker**：`new Worker(script, { workerData })`
2. **通信**：`postMessage` / `on('message')`
3. **线程池**：避免频繁创建/销毁Worker
4. **原子操作**：`Atomics` 实现线程安全

### 应用价值

- **性能提升**：CPU密集任务不阻塞主线程
- **并发处理**：多个任务并行执行
- **资源高效**：比多进程开销小
- **响应及时**：主线程保持响应

### 面试重点

**Q1：Worker Threads和Cluster的区别？**
答：Worker Threads是多线程，共享内存，适合CPU密集任务；Cluster是多进程，完全隔离，适合Web服务负载均衡。

**Q2：如何避免Worker线程频繁创建？**
答：使用线程池模式，预创建固定数量的Worker，复用Worker处理多个任务。

**Q3：Worker Threads如何传递数据？**
答：1) postMessage（序列化传递）2) workerData（初始化传递）3) SharedArrayBuffer（共享内存）。

### 注意事项

⚠️ **线程数量**：不要超过CPU核心数
⚠️ **内存泄漏**：使用线程池避免频繁创建
⚠️ **优雅退出**：清理定时器、关闭连接
⚠️ **数据序列化**：postMessage只能传递可序列化对象
