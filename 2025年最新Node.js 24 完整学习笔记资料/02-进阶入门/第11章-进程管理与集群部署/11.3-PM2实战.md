## 📖 知识点概述

### 核心概念

**PM2（Process Manager 2）**是Node.js生产环境最流行的进程管理器，提供进程守护、负载均衡、日志管理、监控、零停机重载等功能，是生产部署的标准工具。

### PM2核心特性

```
┌──────────────────────────────────────────────────────────┐

├──────────────────────────────────────────────────────────┤
│                                                            │
```

### Cluster Mode vs Fork Mode

| 特性 | Cluster Mode | Fork Mode |
|------|--------------|-----------|
| **进程数** | 多个（CPU核心数） | 单个 |
| **负载均衡** | ✅ 自动 | ❌ 无 |
| **适用场景** | Web服务、API | 定时任务、单例服务 |
| **端口共享** | ✅ 是 | ❌ 否 |
| **内存占用** | 高（多进程） | 低（单进程） |

### 为什么需要PM2？

```bash
# ❌ 不使用PM2：手动管理进程
node app.js &  # 后台运行
# 问题：
# - 崩溃后不会自动重启
# - 需要手动查找PID才能停止
# - 无法查看日志
# - 无法监控资源使用情况
# - 服务器重启后不会自动启动

# ✅ 使用PM2：自动化管理
pm2 start app.js --name my-app -i max
# 优点：
# - 自动重启崩溃的进程
# - 简单的启动/停止/重启命令
# - 自动日志管理
# - 实时监控
# - 开机自启动
# - 零停机重载
```

### 实践1：基础命令

#### 启动应用

```bash
# 💡 最简单的启动
pm2 start app.js

# 💡 指定应用名称
pm2 start app.js --name "my-api"

# 💡 Cluster模式（多进程）
pm2 start app.js -i 4              # 启动4个进程
pm2 start app.js -i max            # 启动CPU核心数个进程
pm2 start app.js -i -1             # 启动CPU核心数-1个进程

# 💡 指定环境变量
pm2 start app.js --env production
pm2 start app.js --env staging

# 💡 传递参数给应用
pm2 start app.js -- arg1 arg2

# 💡 监听文件变化自动重启（开发模式）
pm2 start app.js --watch

# 💡 指定日志文件
pm2 start app.js \
  --name my-app \
  --log ./logs/app.log \
  --error ./logs/error.log \
  --output ./logs/output.log
```

#### 管理应用

```bash
# 💡 查看所有应用
pm2 list
pm2 ls

# 输出示例：
# ┌─────┬────────┬─────────┬─────────┬─────────┬──────────┐
# │ id  │ name   │ mode    │ ↺      │ status  │ cpu      │
# ├─────┼────────┼─────────┼─────────┼─────────┼──────────┤
# │ 0   │ my-app │ cluster │ 15      │ online  │ 0%       │
# │ 1   │ api    │ fork    │ 0       │ online  │ 0%       │
# └─────┴────────┴─────────┴─────────┴─────────┴──────────┘

# 💡 停止应用
pm2 stop my-app           # 按名称停止
pm2 stop 0                # 按ID停止
pm2 stop all              # 停止所有

# 💡 重启应用
pm2 restart my-app        # 普通重启
pm2 restart all           # 重启所有

# 💡 优雅重载（零停机）
pm2 reload my-app         # 适用于cluster模式
pm2 gracefulReload my-app # 等待所有连接关闭

# 💡 删除应用
pm2 delete my-app         # 按名称删除
pm2 delete 0              # 按ID删除
pm2 delete all            # 删除所有

# 💡 重置重启次数
pm2 reset my-app

# 💡 查看应用详细信息
pm2 show my-app
pm2 describe my-app
```

#### 日志管理

```bash
# 💡 查看日志
pm2 logs                  # 所有应用日志
pm2 logs my-app           # 特定应用日志
pm2 logs --lines 100      # 显示最近100行
pm2 logs --err            # 只显示错误日志
pm2 logs --out            # 只显示标准输出

# 💡 清空日志
pm2 flush                 # 清空所有日志
pm2 flush my-app          # 清空特定应用日志

# 💡 日志轮转（需要pm2-logrotate）
pm2 install pm2-logrotate

# 配置日志轮转
pm2 set pm2-logrotate:max_size 10M         # 单个日志文件最大10MB
pm2 set pm2-logrotate:retain 30            # 保留30个日志文件
pm2 set pm2-logrotate:compress true        # 压缩旧日志
pm2 set pm2-logrotate:rotateInterval '0 0 * * *'  # 每天凌晨轮转
```

#### 监控

```bash

# 显示：

# 💡 查看资源使用

# 💡 生成启动脚本（开机自启）
```

### 实践3：优雅重载（Zero-Downtime）

#### 实现优雅关闭

```javascript
// app.js
import express from 'express';

const app = express();

// API路由
app.get('/', (req, res) => {
  res.json({ message: 'Hello from PM2', pid: process.pid });
});

app.get('/long-task', async (req, res) => {
  // 模拟长时间任务
  await new Promise(resolve => setTimeout(resolve, 10000));
  res.json({ message: 'Task completed', pid: process.pid });
});

const server = app.listen(3000, () => {
  console.log(`Server running on port 3000, PID: ${process.pid}`);
  
  // 💡 通知PM2进程已就绪
  if (process.send) {
    process.send('ready');
  }
});

// 💡 优雅关闭处理
process.on('SIGINT', gracefulShutdown);
process.on('SIGTERM', gracefulShutdown);

// PM2的优雅重载信号
process.on('message', (msg) => {
  if (msg === 'shutdown') {
    gracefulShutdown();
  }
});

function gracefulShutdown() {
  console.log(`[${process.pid}] 收到关闭信号，开始优雅关闭...`);
  
  // 停止接受新连接
  server.close(() => {
    console.log(`[${process.pid}] HTTP服务器已关闭`);
    
    // 关闭数据库连接等
    closeDatabase()
      .then(() => {
        console.log(`[${process.pid}] 数据库连接已关闭`);
        process.exit(0);
      })
      .catch((err) => {
        console.error(`[${process.pid}] 关闭失败:`, err);
        process.exit(1);
      });
  });
  
  // 超时强制关闭
  setTimeout(() => {
    console.error(`[${process.pid}] 超时，强制关闭`);
    process.exit(1);
  }, 10000);
}

async function closeDatabase() {
  // 关闭数据库连接、Redis连接等
  return new Promise(resolve => setTimeout(resolve, 1000));
}

// ecosystem.config.js
module.exports = {
  apps: [{
    name: 'my-app',
    script: './app.js',
    instances: 4,
    exec_mode: 'cluster',
    
    // 💡 优雅重载配置
    kill_timeout: 5000,        // 等待5秒后强制关闭
    listen_timeout: 3000,      // 等待3秒后认为进程已就绪
    shutdown_with_message: true, // 使用message事件通知关闭
  }]
};

// 💡 零停机重载
// pm2 reload my-app
// 
// 流程：
// 1. PM2启动新进程
// 2. 新进程就绪后，PM2向旧进程发送shutdown消息
// 3. 旧进程停止接受新请求，等待现有请求完成
// 4. 旧进程关闭，新进程接管
// 5. 重复上述步骤，直到所有进程都被替换
```

```javascript
// monitoring.js
import pm2 from 'pm2';

pm2.connect((err) => {
  if (err) {
    console.error(err);
    process.exit(2);
  }
  
  // 💡 获取所有进程信息
  pm2.list((err, processes) => {
    if (err) {
      console.error(err);
      return;
    }
    
    processes.forEach((proc) => {
      const { name, pid, pm2_env } = proc;
      const { status, restart_time, unstable_restarts } = pm2_env;
      
      console.log(`
应用名称: ${name}
进程ID: ${pid}
状态: ${status}
重启次数: ${restart_time}
不稳定重启: ${unstable_restarts}
内存使用: ${(proc.monit.memory / 1024 / 1024).toFixed(2)} MB
CPU使用: ${proc.monit.cpu}%
      `);
      
      // 💡 告警逻辑
      if (unstable_restarts > 5) {
        console.error(`⚠️  警告：${name} 频繁重启！`);
        // 发送告警（邮件/Slack/钉钉）
        sendAlert(`${name} 频繁重启，请检查`);
      }
      
      if (proc.monit.memory > 500 * 1024 * 1024) {
        console.error(`⚠️  警告：${name} 内存使用超过500MB！`);
        sendAlert(`${name} 内存使用过高`);
      }
    });
    
    pm2.disconnect();
  });
});

function sendAlert(message) {
  // 实现告警发送逻辑
  console.log(`📧 发送告警: ${message}`);
}
```

## 🔧 错误处理

### 高频错误1：应用无限重启

**现象：**
```bash
pm2 list
# status: errored
# restarts: 15 (不断增长)
```

**原因：**
应用启动后立即崩溃，PM2不断尝试重启

**解决方案：**
```javascript
// ecosystem.config.js
module.exports = {
  apps: [{
    name: 'my-app',
    script: './app.js',
    
    // 💡 限制重启
    min_uptime: '10s',           // 必须运行至少10秒才算成功启动
    max_restarts: 10,            // 最多重启10次
    restart_delay: 4000,         // 每次重启延迟4秒
    
    // 💡 查看错误日志
    error_file: './logs/error.log',
  }]
};

// 排查步骤：
// 1. 查看错误日志
pm2 logs my-app --err --lines 50

// 2. 手动运行应用，查看错误
node app.js

// 3. 常见问题：
// - 端口被占用
// - 环境变量缺失
// - 数据库连接失败
// - 语法错误
```

```bash

```

**解决方案：**
```javascript
// ecosystem.config.js

    // 💡 堆快照分析
    // node --inspect app.js
    // chrome://inspect
  }]
};

// 💡 检查内存使用
pm2 monit
pm2 show my-app | grep memory
```

## 📋 总结

### 核心要点

1. **进程管理**：启动/停止/重启/删除
2. **集群模式**：充分利用多核CPU
3. **日志管理**：自动收集、轮转
4. **零停机重载**：gracefulReload更新代码

### 实现关键

1. **ecosystem配置**：统一管理所有配置
2. **开机自启**：pm2 startup + pm2 save
3. **优雅关闭**：监听SIGTERM/SIGINT信号
4. **监控告警**：pm2 monit + PM2 Plus

### 应用价值

- **生产级部署**：稳定可靠的进程管理
- **自动化运维**：自动重启、日志管理
- **高可用**：集群模式、自动故障恢复
- **零停机部署**：更新代码不中断服务

### 面试重点

**Q1：PM2的cluster mode和fork mode有什么区别？**
答：cluster mode创建多个进程，支持负载均衡，适合Web服务；fork mode单进程，适合定时任务或单例服务。

**Q2：如何实现PM2的零停机重载？**
答：使用`pm2 reload`命令。PM2逐个重启进程：先启动新进程，等待就绪后再关闭旧进程，确保始终有进程处理请求。

**Q3：PM2如何处理进程崩溃？**
答：监听进程退出，自动重启。可配置`min_uptime`、`max_restarts`、`restart_delay`避免无限重启循环。

**Q4：如何在PM2中共享状态？**
答：不要依赖进程内存。使用Redis、数据库或文件系统共享状态。Session可以用Redis存储。

### 注意事项

⚠️ **生产环境必须使用ecosystem配置文件**
⚠️ **配置开机自启动**：pm2 startup + pm2 save
⚠️ **定期清理日志**：使用pm2-logrotate
⚠️ **监控内存使用**：设置max_memory_restart
