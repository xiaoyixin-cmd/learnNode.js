## 📖 知识点概述

### 核心概念

**Node.js采用单线程事件循环模型**，这是其最显著的特点之一。虽然是单线程，但通过非阻塞I/O和事件驱动机制，Node.js能够高效处理大量并发连接。然而，单线程也带来了限制，无法充分利用多核CPU。

### Node.js单线程架构

```

├──────────────────────────────────────────────────────────┤
```

### 单线程 vs 多线程对比

| 维度 | 单线程（Node.js） | 多线程（Java/C#） |
|------|------------------|------------------|
| **并发模型** | 事件驱动、非阻塞I/O | 线程池、阻塞I/O |
| **内存占用** | 低（每个连接不需要线程） | 高（每个线程1-2MB） |
| **上下文切换** | 无（单线程） | 频繁（多线程） |
| **复杂度** | 低（无锁、无竞态） | 高（需要锁、同步） |
| **CPU密集任务** | ❌ 阻塞主线程 | ✅ 多核并行 |
| **I/O密集任务** | ✅ 高效处理 | ⚠️ 线程阻塞 |
| **适用场景** | Web服务、API、实时应用 | 计算密集、并行处理 |

### 为什么Node.js选择单线程？

```javascript
// 1. 简化编程模型
// ✅ 单线程：无需考虑锁和竞态条件
let counter = 0;
function increment() {
  counter++; // 安全，不需要加锁
}

// ❌ 多线程（伪代码）：需要考虑线程安全
let counter = 0;
let lock = new Lock();
function increment() {
  lock.acquire();
  counter++; // 需要加锁
  lock.release();
}

// 2. 高效处理I/O密集任务
// Node.js 可以同时处理 10,000+ 并发连接
// 而多线程模型受限于线程数量（通常几百个）
```

### 理解：事件循环机制

#### Event Loop原理

```javascript
// Event Loop 简化模型
┌───────────────────────────┐
┌─>│        timers（定时器）    │
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
│  │     pending callbacks      │
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
│  │       idle, prepare        │
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
│  │          poll（轮询）       │
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
│  │          check             │
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
└──┤      close callbacks       │
   └───────────────────────────┘

// 💡 Event Loop 执行顺序示例
console.log('1: 同步代码');

setTimeout(() => {
  console.log('2: setTimeout (timers 阶段)');
}, 0);

setImmediate(() => {
  console.log('3: setImmediate (check 阶段)');
});

Promise.resolve().then(() => {
  console.log('4: Promise (微任务)');
});

process.nextTick(() => {
  console.log('5: nextTick (微任务优先级最高)');
});

console.log('6: 同步代码');

// 输出顺序：1 → 6 → 5 → 4 → 2 → 3
```

#### I/O密集 vs CPU密集任务

```javascript
// 💡 I/O密集任务（Node.js擅长）
import fs from 'fs/promises';
import http from 'http';

// ✅ 非阻塞I/O：高效处理
async function handleIOIntensive() {
  const [file1, file2, apiData] = await Promise.all([
    fs.readFile('file1.txt', 'utf-8'),           // 文件I/O
    fs.readFile('file2.txt', 'utf-8'),           // 文件I/O
    fetch('https://api.example.com/data')        // 网络I/O
  ]);
  
  return { file1, file2, apiData };
}

// 💡 CPU密集任务（Node.js弱项）
function calculatePrimes(max) {
  const primes = [];
  
  // ❌ 阻塞主线程：其他请求无法处理
  for (let i = 2; i <= max; i++) {
    let isPrime = true;
    for (let j = 2; j <= Math.sqrt(i); j++) {
      if (i % j === 0) {
        isPrime = false;
        break;
      }
    }
    if (isPrime) primes.push(i);
  }
  
  return primes;
}

// 演示阻塞问题

const app = express();

app.get('/primes', (req, res) => {
  console.time('calculate');
  const primes = calculatePrimes(1000000); // ❌ 阻塞主线程
  console.timeEnd('calculate'); // 可能需要几秒
  res.json({ count: primes.length });
});

app.get('/health', (req, res) => {
  res.json({ status: 'ok' });
});

// 问题：/primes 计算时，/health 请求会被阻塞！
```

### 准备：多核CPU利用策略

#### 策略对比

```javascript
// 💡 策略1：多进程（Cluster）
// 优点：充分利用多核CPU，进程隔离
// 缺点：内存占用高，进程间通信复杂
// 适用：Web服务、API服务

import cluster from 'cluster';
import os from 'os';

if (cluster.isMaster) {
  const numCPUs = os.cpus().length;
  
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork(); // 创建工作进程
  }
} else {
  // 工作进程运行应用
  import('./app.js');
}

// 💡 策略2：Worker Threads（工作线程）
// 优点：共享内存，低开销
// 缺点：共享V8实例，可能互相影响
// 适用：CPU密集任务（图像处理、加密等）

import { Worker } from 'worker_threads';

function runCpuIntensiveTask(data) {
  return new Promise((resolve, reject) => {
    const worker = new Worker('./worker.js', {
      workerData: data
    });
    
    worker.on('message', resolve);
    worker.on('error', reject);
  });
}

// 💡 策略3：子进程（Child Process）
// 优点：完全隔离，可运行其他语言
// 缺点：开销大，通信慢
// 适用：调用外部程序、Python脚本等

import { spawn } from 'child_process';

function runPythonScript(scriptPath, args) {
  return new Promise((resolve, reject) => {
    const python = spawn('python3', [scriptPath, ...args]);
    
    let output = '';
    python.stdout.on('data', (data) => {
      output += data.toString();
    });
    
    python.on('close', (code) => {
      if (code === 0) {
        resolve(output);
      } else {
        reject(new Error(`Python script failed with code ${code}`));
      }
    });
  });
}

// 💡 策略4：任务队列（Bull/RabbitMQ）
// 优点：解耦、可扩展、支持分布式
// 缺点：引入中间件，增加复杂度
// 适用：后台任务、异步处理

import Bull from 'bull';

const queue = new Bull('cpu-intensive-tasks', {
  redis: { host: 'localhost', port: 6379 }
});

// 添加任务
queue.add({ type: 'compute', data: [1, 2, 3] });

// 处理任务（可在多个进程/机器上运行）
queue.process(async (job) => {
  const result = await heavyComputation(job.data);
  return result;
});
```

## 🔧 错误处理

### 高频错误1：CPU密集任务阻塞主线程

**现象：**
```javascript
// ❌ 问题代码
app.get('/report', async (req, res) => {
  const users = await db.users.findAll();
  
  // CPU密集：生成PDF报告
  const pdf = generatePDFReport(users); // 阻塞10秒
  
  res.send(pdf);
});

// 问题：生成PDF期间，所有其他请求都被阻塞
```

**解决方案：**
```javascript
// ✅ 方案1：使用Worker Threads
import { Worker } from 'worker_threads';

app.get('/report', async (req, res) => {
  const users = await db.users.findAll();
  
  // 在工作线程中生成PDF
  const worker = new Worker('./pdf-worker.js', {
    workerData: { users }
  });
  
  worker.on('message', (pdf) => {
    res.send(pdf);
  });
  
  worker.on('error', (err) => {
    res.status(500).json({ error: err.message });
  });
});

// pdf-worker.js
import { parentPort, workerData } from 'worker_threads';

const pdf = generatePDFReport(workerData.users);
parentPort.postMessage(pdf);

// ✅ 方案2：使用任务队列（异步处理）
import Bull from 'bull';

const reportQueue = new Bull('reports');

app.post('/report', async (req, res) => {
  const users = await db.users.findAll();
  
  // 添加到队列，立即返回
  const job = await reportQueue.add({ users });
  
  res.json({
    message: 'Report generation started',
    jobId: job.id,
    statusUrl: `/report/status/${job.id}`
  });
});

// 后台处理
reportQueue.process(async (job) => {
  const pdf = generatePDFReport(job.data.users);
  
  // 保存到存储或发送邮件
  await storage.save(`report-${job.id}.pdf`, pdf);
  
  return { url: storage.getUrl(`report-${job.id}.pdf`) };
});
```

### 高频错误2：误用CPU核心数创建过多进程

**现象：**
```javascript
// ❌ 错误：为每个请求创建工作线程
app.get('/compute', (req, res) => {
  const worker = new Worker('./compute-worker.js');
  
  worker.on('message', (result) => {
    res.json({ result });
  });
});

// 问题：高并发时创建大量线程，资源耗尽
```

**解决方案：**
```javascript
// ✅ 方案1：线程池模式
import { Worker } from 'worker_threads';
import os from 'os';

class WorkerPool {
  constructor(workerScript, poolSize = os.cpus().length) {
    this.workers = [];
    this.queue = [];
    
    // 💡 创建固定数量的工作线程
    for (let i = 0; i < poolSize; i++) {
      const worker = new Worker(workerScript);
      
      worker.on('message', (result) => {
        const task = this.workers[i].currentTask;
        if (task) {
          task.resolve(result);
          this.workers[i].currentTask = null;
          this.processQueue();
        }
      });
      
      this.workers.push({ worker, currentTask: null });
    }
  }
  
  exec(data) {
    return new Promise((resolve, reject) => {
      // 找空闲线程
      const availableWorker = this.workers.find(w => !w.currentTask);
      
      if (availableWorker) {
        availableWorker.currentTask = { resolve, reject };
        availableWorker.worker.postMessage(data);
      } else {
        // 加入队列
        this.queue.push({ data, resolve, reject });
      }
    });
  }
  
  processQueue() {
    if (this.queue.length === 0) return;
    
    const availableWorker = this.workers.find(w => !w.currentTask);
    if (availableWorker) {
      const task = this.queue.shift();
      availableWorker.currentTask = task;
      availableWorker.worker.postMessage(task.data);
    }
  }
}

// 使用线程池
const pool = new WorkerPool('./compute-worker.js', 4);

app.get('/compute', async (req, res) => {
  const result = await pool.exec({ input: req.query.data });
  res.json({ result });
});

// ✅ 方案2：使用现成的线程池库
import Piscina from 'piscina';

const pool = new Piscina({
  filename: './compute-worker.js',
  minThreads: 2,
  maxThreads: os.cpus().length
});

app.get('/compute', async (req, res) => {
  const result = await pool.run({ input: req.query.data });
  res.json({ result });
});
```

```javascript
// ✅ 方案1：使用Redis共享状态
import Redis from 'ioredis';

const redis = new Redis();

app.get('/api', async (req, res) => {
  const count = await redis.incr('request_count');
  res.json({ count });
});

// ✅ 方案2：使用主进程协调
import cluster from 'cluster';

if (cluster.isMaster) {
  let requestCount = 0;
  
  // 监听工作进程消息
  cluster.on('message', (worker, message) => {
    if (message.type === 'increment') {
      requestCount++;
      worker.send({ type: 'count', value: requestCount });
    }
  });
  
  // 创建工作进程
  for (let i = 0; i < os.cpus().length; i++) {
    cluster.fork();
  }
} else {
  app.get('/api', async (req, res) => {
    // 向主进程请求计数
    process.send({ type: 'increment' });
    
    process.once('message', (msg) => {
      if (msg.type === 'count') {
        res.json({ count: msg.value });
      }
    });
  });
  
  app.listen(3000);
}

// ✅ 方案3：使用数据库
app.get('/api', async (req, res) => {
  const result = await db.query(
    'UPDATE counters SET count = count + 1 WHERE id = 1 RETURNING count'
  );
```

```javascript
import express from 'express';
import { Pool } from 'pg';

const app = express();
const db = new Pool({ connectionString: process.env.DATABASE_URL });

// ✅ I/O密集：Node.js单线程高效处理
app.get('/api/products', async (req, res) => {
  const { page = 1, limit = 20 } = req.query;
  
  // 并行查询
  const [products, total] = await Promise.all([
    db.query('SELECT * FROM products LIMIT $1 OFFSET $2', [

  ]);
```

### 场景2：图像处理服务（CPU密集）

```javascript
// 场景：图片压缩、水印、格式转换
// 策略：Worker Threads + 线程池

import express from 'express';
import multer from 'multer';
import Piscina from 'piscina';
import path from 'path';

const app = express();
const upload = multer({ dest: 'uploads/' });

// 💡 创建线程池（4个工作线程）
const imagePool = new Piscina({
  filename: path.resolve('./image-worker.js'),
  maxThreads: 4
});

app.post('/api/image/compress', upload.single('image'), async (req, res) => {
  try {
    const { quality = 80 } = req.body;
    
    console.time('compress');
    
    // ✅ 在工作线程中处理
    const result = await imagePool.run({
      inputPath: req.file.path,
      outputPath: `compressed/${req.file.filename}.jpg`,
      quality: parseInt(quality)
    });
    
    console.timeEnd('compress');
    
    res.json({
      message: 'Image compressed successfully',
      originalSize: req.file.size,
      compressedSize: result.size,
      savings: `${((1 - result.size / req.file.size) * 100).toFixed(2)}%`,
      url: `/compressed/${req.file.filename}.jpg`
    });
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});

app.post('/api/image/watermark', upload.single('image'), async (req, res) => {
  try {
    const { text, position = 'bottom-right' } = req.body;
    
    const result = await imagePool.run({
      type: 'watermark',
      inputPath: req.file.path,
      outputPath: `watermarked/${req.file.filename}.jpg`,
      text,
      position
    });
    
    res.json({
      message: 'Watermark added successfully',
      url: `/watermarked/${req.file.filename}.jpg`
```

## 📋 总结

### 核心要点

1. **单线程模型**：事件驱动、非阻塞I/O、高并发
2. **I/O密集擅长**：数据库查询、API调用、文件读写
3. **CPU密集弱项**：复杂计算、图像处理、加密解密
4. **多核利用策略**：Cluster、Worker Threads、子进程、任务队列

### 实现关键

1. **识别任务类型**：I/O密集 vs CPU密集
2. **选择合适策略**：
   - I/O密集 → 单进程异步I/O
   - CPU密集 → Worker Threads / 任务队列
   - Web服务 → Cluster多进程
3. **避免阻塞主线程**：CPU密集任务移到工作线程
4. **线程池模式**：控制线程数量，避免资源耗尽

### 应用价值

- **高并发处理**：单进程10万+并发连接
- **资源高效**：内存占用低，上下文切换少
- **编程简单**：无锁、无竞态，易于理解
- **充分利用硬件**：通过Cluster/Worker Threads利用多核

### 面试重点

**Q1：为什么Node.js是单线程？单线程有什么优势？**
答：简化编程模型，避免锁和竞态条件；内存占用低，无上下文切换开销；通过非阻塞I/O和事件驱动实现高并发。

**Q2：Node.js单线程如何处理并发？**
答：通过事件循环（Event Loop）和非阻塞I/O。I/O操作交给Libuv线程池或操作系统处理，完成后通过回调通知主线程，主线程只负责调度。

**Q3：什么是I/O密集和CPU密集任务？Node.js适合哪种？**
答：I/O密集（数据库、文件、网络），CPU密集（计算、加密、图像）。Node.js适合I/O密集任务，CPU密集任务会阻塞主线程。

**Q4：如何让Node.js利用多核CPU？**
答：1) Cluster模块（多进程）2) Worker Threads（多线程）3) 子进程 4) 任务队列（分布式）。

**Q5：Cluster和Worker Threads的区别？**
答：Cluster是多进程，完全隔离，适合Web服务；Worker Threads是多线程，共享内存，适合CPU密集任务。

### 注意事项

⚠️ **CPU密集任务**：必须使用Worker Threads或任务队列
⚠️ **线程池大小**：不要超过CPU核心数
⚠️ **进程间状态**：使用Redis/数据库共享状态
⚠️ **内存限制**：默认1.4GB，可通过--max-old-space-size调整
