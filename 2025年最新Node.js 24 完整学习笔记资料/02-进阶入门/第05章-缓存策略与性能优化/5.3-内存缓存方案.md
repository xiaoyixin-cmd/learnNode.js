# 5.3 内存缓存方案

> 掌握node-cache、lru-cache的使用和LRU算法原理

---

## 📖 知识点概述

### 核心概念

**内存缓存**：将数据存储在应用服务器的内存中，访问速度最快（ns级），但容量有限且不跨进程共享。

### 内存缓存 vs Redis

| 特性 | 内存缓存 | Redis |
|------|---------|-------|
| **速度** | 极快（ns级） | 快（ms级） |
| **容量** | 小（MB-GB） | 大（GB-TB） |
| **共享** | ❌ 单进程 | ✅ 集群共享 |
| **持久化** | ❌ 无 | ✅ 有 |
| **网络开销** | ❌ 无 | ✅ 有 |
| **适用场景** | 热点数据、计算结果 | 跨进程数据 |

### 何时使用内存缓存？

- ✅ **不需要共享**：单机应用
- ✅ **数据量小**：几百MB以内
- ✅ **极高频访问**：每秒数万次
- ✅ **可以丢失**：重启后可重建

---

## 🚀 主流方案对比

### node-cache

**特点**：
- 简单易用，开箱即用
- 支持TTL和定期检查
- 支持统计信息
- 无LRU算法（只能设置maxKeys）

**适用场景**：简单缓存需求

### lru-cache

**特点**：
- LRU（最近最少使用）算法
- 自动淘汰旧数据
- 高性能，内存可控
- 支持maxSize和maxAge

**适用场景**：内存有限、需要LRU淘汰

## 💡 实践：node-cache

### 安装和基础使用

```bash
npm install node-cache
```

```javascript
// node-cache-basic.js
import NodeCache from 'node-cache';

// 💡 创建缓存实例
const cache = new NodeCache({
  stdTTL: 600, // 默认TTL 10分钟
  checkperiod: 120, // 每2分钟检查过期key
  useClones: false, // 不克隆对象（提升性能）
  deleteOnExpire: true, // 过期时自动删除
  maxKeys: 1000 // 最大key数量（超过则set失败）
});

// ===== 基础操作 =====

// SET
cache.set('key', 'value');
cache.set('user:123', { name: 'John', age: 25 }, 60); // 60秒TTL

// GET
const value = cache.get('key');
console.log(value); // 'value'

const user = cache.get('user:123');
console.log(user); // { name: 'John', age: 25 }

// MGET
const values = cache.mget(['key1', 'key2', 'key3']);
console.log(values); // { key1: 'value1', key2: 'value2', key3: undefined }

// MSET
cache.mset([
  { key: 'key1', val: 'value1', ttl: 60 },
  { key: 'key2', val: 'value2', ttl: 120 }
]);

// DEL
cache.del('key');
cache.del(['key1', 'key2']); // 批量删除

// HAS
const exists = cache.has('key');
console.log(exists); // false

// TTL
const ttl = cache.getTtl('user:123');
console.log(`还剩 ${(ttl - Date.now()) / 1000} 秒`);

// 修改TTL
cache.ttl('user:123', 120); // 改为120秒

// 获取所有keys
const keys = cache.keys();
console.log('所有keys:', keys);

// 清空缓存
cache.flushAll();

// ===== 统计信息 =====

const stats = cache.getStats();
console.log('缓存统计:', stats);
// {
//   keys: 10,     // key数量
//   hits: 100,    // 命中次数
//   misses: 20,   // 未命中次数
//   ksize: 50,    // key占用字节
//   vsize: 1024   // value占用字节
// }

// ===== 事件监听 =====

cache.on('set', (key, value) => {
  console.log(`设置缓存: ${key}`);
});

cache.on('del', (key, value) => {
  console.log(`删除缓存: ${key}`);
});

cache.on('expired', (key, value) => {
  console.log(`缓存过期: ${key}`);
});

cache.on('flush', () => {
  console.log('缓存已清空');
});
```

### 实战：用户信息缓存

```javascript
// user-cache.js
import NodeCache from 'node-cache';
import pool from './db.js';

const userCache = new NodeCache({
  stdTTL: 600, // 10分钟
  checkperiod: 120,
  useClones: false
});

class UserService {
  async getUser(userId) {
    const key = `user:${userId}`;
    
    // 1. 查询缓存
    let user = userCache.get(key);
    if (user) {
      console.log('缓存命中');
      return user;
    }
    
    console.log('缓存未命中，查询数据库');
    
    // 2. 查询数据库
    const [rows] = await pool.query(
      'SELECT id, username, email FROM users WHERE id = ?',
      [userId]
    );
    
    user = rows[0];
    if (!user) return null;
    
    // 3. 写入缓存
    userCache.set(key, user);
    
    return user;
  }
  
  async updateUser(userId, updates) {
    // 1. 更新数据库
    await pool.query('UPDATE users SET ? WHERE id = ?', [updates, userId]);
    
    // 2. 删除缓存
    const key = `user:${userId}`;
    userCache.del(key);
  }
  
  // 获取缓存命中率
  getCacheHitRate() {
    const stats = userCache.getStats();
    const total = stats.hits + stats.misses;
    const hitRate = total > 0 ? (stats.hits / total * 100).toFixed(2) : 0;
    
    return {
      hits: stats.hits,
      misses: stats.misses,
      hitRate: `${hitRate}%`,
      keys: stats.keys
    };
  }
}

export default new UserService();
```

## 💡 实践：lru-cache

### 安装和基础使用

```bash
npm install lru-cache
```

```javascript
// lru-cache-basic.js
import { LRUCache } from 'lru-cache';

// 💡 创建LRU缓存
const cache = new LRUCache({
  max: 100, // 最多100个key
  maxSize: 5000, // 最多5000字节（可选）
  sizeCalculation: (value) => {
    // 计算value大小
    return JSON.stringify(value).length;
  },
  ttl: 1000 * 60 * 10, // 默认TTL 10分钟
  allowStale: false, // 不允许返回过期数据
  updateAgeOnGet: false, // get时不更新age
  updateAgeOnHas: false // has时不更新age
});

// ===== 基础操作 =====

// SET
cache.set('key', 'value');
cache.set('user:123', { name: 'John' }, { ttl: 60000 }); // 60秒

// GET
const value = cache.get('key');
console.log(value); // 'value'

// PEEK（查看但不更新LRU顺序）
const peeked = cache.peek('key');

// HAS
const exists = cache.has('key');

// DELETE
cache.delete('key');

// CLEAR
cache.clear();

// SIZE
console.log('缓存大小:', cache.size); // key数量
console.log('缓存字节:', cache.calculatedSize); // 总字节数

// ===== LRU特性演示 =====

const lru = new LRUCache({ max: 3 });

lru.set('a', 1);
lru.set('b', 2);
lru.set('c', 3);
console.log([...lru.keys()]); // ['a', 'b', 'c']

// 访问a，a变为最新
lru.get('a');
console.log([...lru.keys()]); // ['b', 'c', 'a']

// 添加d，最旧的b被淘汰
lru.set('d', 4);
console.log([...lru.keys()]); // ['c', 'a', 'd']

// ===== 遍历 =====

for (const [key, value] of cache) {
  console.log(key, value);
}

// 或
cache.forEach((value, key) => {
  console.log(key, value);
});
```

### LRU算法原理

```
LRU（Least Recently Used）最近最少使用算法

数据结构：哈希表 + 双向链表

[head] ⇄ [最新访问] ⇄ [最近访问] ⇄ ... ⇄ [最久未访问] ⇄ [tail]

操作：
1. GET：
   - 在哈希表中查找（O(1)）
   - 将节点移到链表头部（O(1)）
   
2. SET：
   - 如果key存在：更新value，移到头部
   - 如果key不存在：
     - 添加到头部
     - 如果超过max，删除尾部节点
     
3. 淘汰：
   - 删除链表尾部节点（最久未访问）
```

### 手写简易LRU缓存

```javascript
// simple-lru.js

class SimpleLRU {
  constructor(maxSize) {
    this.maxSize = maxSize;
    this.cache = new Map(); // 💡 Map保证插入顺序
  }
  
  get(key) {
    if (!this.cache.has(key)) {
      return undefined;
    }
    
    // 💡 获取value并移到最后（最新）
    const value = this.cache.get(key);
    this.cache.delete(key);
    this.cache.set(key, value);
    
    return value;
  }
  
  set(key, value) {
    // 如果key存在，先删除
    if (this.cache.has(key)) {
      this.cache.delete(key);
    }
    
    // 添加到最后
    this.cache.set(key, value);
    
    // 💡 超过maxSize，删除第一个（最久未使用）
    if (this.cache.size > this.maxSize) {
      const firstKey = this.cache.keys().next().value;
      this.cache.delete(firstKey);
    }
  }
  
  has(key) {
    return this.cache.has(key);
  }
  
  delete(key) {
    return this.cache.delete(key);
  }
  
  get size() {
    return this.cache.size;
  }
}

// 测试
const lru = new SimpleLRU(3);

lru.set('a', 1);
lru.set('b', 2);
lru.set('c', 3);
console.log('size:', lru.size); // 3

lru.get('a'); // a成为最新
lru.set('d', 4); // b被淘汰

console.log('has b:', lru.has('b')); // false
console.log('has a:', lru.has('a')); // true
```

### 实战：计算结果缓存

```javascript
// computation-cache.js
import { LRUCache } from 'lru-cache';

// 💡 缓存昂贵的计算结果
const cache = new LRUCache({
  max: 500,
  ttl: 1000 * 60 * 30 // 30分钟
});

// 斐波那契数列（递归版本，未优化）
function fibonacciSlow(n) {
  if (n <= 1) return n;
  return fibonacciSlow(n - 1) + fibonacciSlow(n - 2);
}

// 带缓存的版本
function fibonacci(n) {
  const key = `fib:${n}`;
  
  // 查询缓存
  let result = cache.get(key);
  if (result !== undefined) {
    console.log(`缓存命中: fib(${n})`);
    return result;
  }
  
  console.log(`计算: fib(${n})`);
  
  // 计算
  if (n <= 1) {
    result = n;
  } else {
    result = fibonacci(n - 1) + fibonacci(n - 2);
  }
  
  // 写入缓存
  cache.set(key, result);
  
  return result;
}

// 性能对比
console.time('未缓存');
console.log('fib(35)=', fibonacciSlow(35));
console.timeEnd('未缓存'); // ~500ms

console.time('有缓存');
console.log('fib(35)=', fibonacci(35));
console.timeEnd('有缓存'); // ~1ms（首次计算）

console.time('有缓存-二次');
console.log('fib(35)=', fibonacci(35));
console.timeEnd('有缓存-二次'); // ~0.01ms（缓存命中）
```

## 🎯 选型建议

### 选择node-cache如果：

- ✅ 简单场景，不需要LRU
- ✅ 需要统计信息
- ✅ 需要事件监听

### 选择lru-cache如果：

- ✅ 内存有限，需要自动淘汰
- ✅ 大量数据，需要LRU算法
- ✅ 高性能要求

### 选择Redis如果：

- ✅ 需要跨进程共享
- ✅ 需要持久化
- ✅ 分布式系统

## 📋 总结

### 核心要点

1. **内存缓存**：速度最快，但不共享
2. **node-cache**：简单易用，适合基础场景
3. **lru-cache**：LRU算法，适合内存受限
4. **LRU算法**：淘汰最久未使用的数据

### 实现关键

1. **node-cache**：`new NodeCache({ stdTTL, maxKeys })`
2. **lru-cache**：`new LRUCache({ max, ttl })`
3. **Cache-Aside**：先查缓存，未命中查数据库
4. **更新时删除**：避免不一致

### 应用价值

- **极高性能**：ns级响应
- **降低延迟**：无网络开销
- **节省资源**：减少数据库查询

### 面试重点

1. **LRU算法原理？**
   - 哈希表+双向链表，O(1)时间复杂度

2. **内存缓存 vs Redis？**
   - 内存缓存更快但不共享，Redis可共享

3. **如何实现LRU缓存？**
   - Map保证插入顺序，get时移到最后

4. **内存缓存的限制？**
   - 容量有限、不持久化、不跨进程

### 注意事项

⚠️ **控制内存使用**：
```javascript
const cache = new LRUCache({
  max: 1000,
  maxSize: 50 * 1024 * 1024 // 50MB
});
```

⚠️ **避免缓存过大对象**：
```javascript
// ❌ 缓存整个数据集（可能很大）
cache.set('allUsers', users);

// ✅ 只缓存需要的字段
cache.set(`user:${id}`, { id, username, email });
```
