# 10.4 领域驱动设计（DDD）

> 使用领域驱动设计构建复杂业务系统

---

## 📚 知识导读

### 什么是 DDD？

**定义：**  
领域驱动设计（Domain-Driven Design，DDD）是一种软件开发方法论，强调将业务领域知识融入软件设计中。

**核心理念：**
- **领域优先**：深入理解业务领域
- **统一语言**：开发人员和领域专家使用相同术语
- **模型驱动**：用模型表达业务逻辑

**两大分支：**
- **战略设计**：限界上下文、子域、上下文映射
- **战术设计**：实体、值对象、聚合、领域服务

## 🎯 战略设计

### 1. 限界上下文（Bounded Context）

**定义：**  
限界上下文是一个明确的边界，在此边界内，领域模型的含义是一致的。

**示例：电商系统**
```
┌─────────────────────┐  ┌─────────────────────┐
│   订单上下文         │  │   库存上下文         │
│                     │  │                     │
│  - Order            │  │  - Product          │
│  - OrderItem        │  │  - Stock            │
│  - Customer         │  │  - Warehouse        │
└─────────────────────┘  └─────────────────────┘
         │                        │
         └───────┬────────────────┘
                 │
        ┌────────▼────────┐
        │   支付上下文     │
        │                 │
        │  - Payment      │
        │  - Transaction  │
        └─────────────────┘
```

### 3. 上下文映射（Context Mapping）

**模式：**
- **共享内核（Shared Kernel）**：共享部分模型
- **客户-供应商（Customer-Supplier）**：下游依赖上游
- **防腐层（Anti-Corruption Layer）**：隔离外部系统

**示例：订单 ← 库存**
```typescript
// 防腐层：将库存上下文转换为订单上下文
// contexts/order/infrastructure/inventory.acl.ts
import { InventoryContext } from '../../inventory/domain/product.entity';

export class InventoryAntiCorruptionLayer {
  async checkProductAvailability(productId: string, quantity: number): Promise<boolean> {
    // 调用库存上下文
    const product = await InventoryContext.getProduct(productId);
    
    // 转换为订单上下文的概念
    return product.stock >= quantity;
  }
  
  async reserveProduct(productId: string, quantity: number): Promise<void> {
    // 调用库存 API 并转换
    await InventoryContext.reserveStock(productId, quantity);
  }
}
```

## 🎯 战术设计

### 1. 实体（Entity）

**特征：**
- 有唯一标识（ID）
- 有生命周期
- 可变

```typescript
// domain/entities/order.entity.ts
import { v4 as uuidv4 } from 'uuid';

export class Order {
  constructor(
    public readonly id: string,
    public customerId: string,
    public status: OrderStatus,
    public items: OrderItem[],
    public createdAt: Date,
    public updatedAt: Date
  ) {}
  
  static create(customerId: string, items: OrderItem[]): Order {
    return new Order(
      uuidv4(),
      customerId,
      OrderStatus.Pending,
      items,
      new Date(),
      new Date()
    );
  }
  
  // 业务方法
  confirm(): void {
    if (this.status !== OrderStatus.Pending) {
      throw new Error('Only pending orders can be confirmed');
    }
    
    this.status = OrderStatus.Confirmed;
    this.updatedAt = new Date();
  }
  
  cancel(): void {
    if (this.status === OrderStatus.Shipped) {
      throw new Error('Cannot cancel shipped orders');
    }
    
    this.status = OrderStatus.Cancelled;
    this.updatedAt = new Date();
  }
  
  getTotalPrice(): number {
    return this.items.reduce((sum, item) => sum + item.getSubtotal(), 0);
  }
}

export enum OrderStatus {
  Pending = 'pending',
  Confirmed = 'confirmed',
  Shipped = 'shipped',
  Delivered = 'delivered',
  Cancelled = 'cancelled',
}
```

### 2. 值对象（Value Object）

**特征：**
- 无唯一标识
- 不可变
- 通过值比较相等

```typescript
// domain/value-objects/money.vo.ts
export class Money {
  constructor(
    public readonly amount: number,
    public readonly currency: string
  ) {
    if (amount < 0) {
      throw new Error('Amount cannot be negative');
    }
  }
  
  add(other: Money): Money {
    this.assertSameCurrency(other);
    return new Money(this.amount + other.amount, this.currency);
  }
  
  subtract(other: Money): Money {
    this.assertSameCurrency(other);
    return new Money(this.amount - other.amount, this.currency);
  }
  
  multiply(factor: number): Money {
    return new Money(this.amount * factor, this.currency);
  }
  
  equals(other: Money): boolean {
    return this.amount === other.amount && this.currency === other.currency;
  }
  
  private assertSameCurrency(other: Money): void {
    if (this.currency !== other.currency) {
      throw new Error('Currency mismatch');
    }
  }
}

// 使用
const price = new Money(100, 'USD');
const tax = new Money(10, 'USD');
const total = price.add(tax);  // Money(110, 'USD')
```

### 3. 聚合根（Aggregate Root）

**定义：**  
聚合是一组相关对象的集合，聚合根是聚合的入口，保证聚合内的一致性。

```typescript
// domain/aggregates/order.aggregate.ts
export class OrderAggregate {
  private items: OrderItem[] = [];
  
  constructor(
    public readonly id: string,
    public readonly customerId: string,
    public status: OrderStatus,
    private shippingAddress: Address
  ) {}
  
  // 只能通过聚合根修改
  addItem(product: Product, quantity: number): void {
    const existingItem = this.items.find(item => item.productId === product.id);
    
    if (existingItem) {
      existingItem.increaseQuantity(quantity);
    } else {
      this.items.push(new OrderItem(product.id, product.price, quantity));
    }
  }
  
  removeItem(productId: string): void {
    this.items = this.items.filter(item => item.productId !== productId);
  }
  
  changeShippingAddress(newAddress: Address): void {
    if (this.status !== OrderStatus.Pending) {
      throw new Error('Cannot change address of confirmed order');
    }
    
    this.shippingAddress = newAddress;
  }
  
  // 业务规则
  confirm(): void {
    if (this.items.length === 0) {
      throw new Error('Cannot confirm empty order');
    }
    
    if (!this.shippingAddress) {
      throw new Error('Shipping address is required');
    }
    
    this.status = OrderStatus.Confirmed;
  }
  
  // 获取只读副本（防止外部修改）
  getItems(): readonly OrderItem[] {
    return [...this.items];
  }
}

export class OrderItem {
  constructor(
    public readonly productId: string,
    public readonly unitPrice: Money,
    private quantity: number
  ) {}
  
  increaseQuantity(amount: number): void {
    this.quantity += amount;
  }
  
  getSubtotal(): Money {
    return this.unitPrice.multiply(this.quantity);
  }
  
  getQuantity(): number {
    return this.quantity;
  }
}
```

### 4. 领域服务（Domain Service）

**使用场景：**  
业务逻辑不属于任何单个实体或值对象时。

```typescript
// domain/services/order-pricing.service.ts
export class OrderPricingService {
  constructor(
    private readonly taxCalculator: TaxCalculator,
    private readonly shippingCalculator: ShippingCalculator
  ) {}
  
  calculateTotalPrice(order: OrderAggregate): Money {
    // 商品总价
    const itemsTotal = order.getItems().reduce(
      (sum, item) => sum.add(item.getSubtotal()),
      new Money(0, 'USD')
    );
    
    // 税费
    const tax = this.taxCalculator.calculate(itemsTotal, order.getShippingAddress());
    
    // 运费
    const shipping = this.shippingCalculator.calculate(
      order.getItems(),
      order.getShippingAddress()
    );
    
    return itemsTotal.add(tax).add(shipping);
  }
}

// 使用
const pricingService = new OrderPricingService(taxCalculator, shippingCalculator);
const total = pricingService.calculateTotalPrice(order);
```

### 6. 仓储（Repository）

**定义：**  
仓储负责聚合的持久化和重建。

```typescript
// domain/repositories/order.repository.ts
export interface OrderRepository {
  findById(orderId: string): Promise<OrderAggregate | null>;
  save(order: OrderAggregate): Promise<void>;
  delete(orderId: string): Promise<void>;
}

// infrastructure/repositories/mongo-order.repository.ts
import { OrderRepository } from '../../domain/repositories/order.repository';

export class MongoOrderRepository implements OrderRepository {
  constructor(private readonly db: Db) {}
  
  async findById(orderId: string): Promise<OrderAggregate | null> {
    const doc = await this.db.collection('orders').findOne({ _id: orderId });
    
    if (!doc) return null;
    
    // 从数据库模型重建领域模型
    return new OrderAggregate(
      doc._id,
      doc.customerId,
      doc.status,
      new Address(doc.address.street, doc.address.city, doc.address.zip)
    );
  }
  
  async save(order: OrderAggregate): Promise<void> {
    // 将领域模型转换为数据库模型
    await this.db.collection('orders').updateOne(
      { _id: order.id },
      {
        $set: {
          customerId: order.customerId,
          status: order.status,
          items: order.getItems().map(item => ({
            productId: item.productId,
            unitPrice: item.unitPrice.amount,
            quantity: item.getQuantity(),
          })),
        },
      },
      { upsert: true }
    );
  }
  
  async delete(orderId: string): Promise<void> {
    await this.db.collection('orders').deleteOne({ _id: orderId });
  }
}
```

## 🎯 完整示例：订单系统

### 1. 应用服务（Application Service）

```typescript
// application/order.service.ts
export class OrderApplicationService {
  constructor(
    private readonly orderRepository: OrderRepository,
    private readonly inventoryService: InventoryAntiCorruptionLayer,
    private readonly pricingService: OrderPricingService,
    private readonly eventBus: EventBus
  ) {}
  
  async placeOrder(command: PlaceOrderCommand): Promise<string> {
    // 1. 检查库存
    for (const item of command.items) {
      const available = await this.inventoryService.checkProductAvailability(
        item.productId,
        item.quantity
      );
      
      if (!available) {
        throw new Error(`Product ${item.productId} is out of stock`);
      }
    }
    
    // 2. 创建订单聚合
    const order = new OrderAggregate(
      uuidv4(),
      command.customerId,
      OrderStatus.Pending,
      new Address(command.shippingAddress)
    );
    
    for (const item of command.items) {
      order.addItem(item.product, item.quantity);
    }
    
    // 3. 计算价格
    const totalPrice = this.pricingService.calculateTotalPrice(order);
    
    // 4. 保存订单
    await this.orderRepository.save(order);
    
    // 5. 预留库存
    for (const item of command.items) {
      await this.inventoryService.reserveProduct(item.productId, item.quantity);
    }
    
    // 6. 发布事件
    await this.eventBus.publish(
      new OrderPlacedEvent(order.id, order.customerId, order.getItems())
    );
    
    return order.id;
  }
  
  async confirmOrder(orderId: string): Promise<void> {
    // 1. 加载聚合
    const order = await this.orderRepository.findById(orderId);
    if (!order) {
      throw new Error('Order not found');
    }
    
    // 2. 执行业务逻辑
    order.confirm();
    
    // 3. 保存
    await this.orderRepository.save(order);
    
    // 4. 发布事件
    const totalPrice = this.pricingService.calculateTotalPrice(order);
    await this.eventBus.publish(new OrderConfirmedEvent(order.id, totalPrice));
  }
}
```

## 🎯 规格模式（Specification Pattern）

### 1. 定义规格

```typescript
// domain/specifications/specification.ts
export abstract class Specification<T> {
  abstract isSatisfiedBy(candidate: T): boolean;
  
  and(other: Specification<T>): Specification<T> {
    return new AndSpecification(this, other);
  }
  
  or(other: Specification<T>): Specification<T> {
    return new OrSpecification(this, other);
  }
  
  not(): Specification<T> {
    return new NotSpecification(this);
  }
}

class AndSpecification<T> extends Specification<T> {
  constructor(
    private left: Specification<T>,
    private right: Specification<T>
  ) {
    super();
  }
  
  isSatisfiedBy(candidate: T): boolean {
    return this.left.isSatisfiedBy(candidate) && this.right.isSatisfiedBy(candidate);
  }
}

class OrSpecification<T> extends Specification<T> {
  constructor(
    private left: Specification<T>,
    private right: Specification<T>
  ) {
    super();
  }
  
  isSatisfiedBy(candidate: T): boolean {
    return this.left.isSatisfiedBy(candidate) || this.right.isSatisfiedBy(candidate);
  }
}

class NotSpecification<T> extends Specification<T> {
  constructor(private spec: Specification<T>) {
    super();
  }
  
  isSatisfiedBy(candidate: T): boolean {
    return !this.spec.isSatisfiedBy(candidate);
  }
}
```

### 2. 业务规格

```typescript
// domain/specifications/order.specifications.ts
export class PendingOrderSpecification extends Specification<OrderAggregate> {
  isSatisfiedBy(order: OrderAggregate): boolean {
    return order.status === OrderStatus.Pending;
  }
}

export class HasItemsSpecification extends Specification<OrderAggregate> {
  isSatisfiedBy(order: OrderAggregate): boolean {
    return order.getItems().length > 0;
  }
}

export class HasShippingAddressSpecification extends Specification<OrderAggregate> {
  isSatisfiedBy(order: OrderAggregate): boolean {
    return order.shippingAddress !== null;
  }
}

// 组合规格
export class CanBeConfirmedSpecification extends Specification<OrderAggregate> {
  constructor() {
    super();
    this.spec = new PendingOrderSpecification()
      .and(new HasItemsSpecification())
      .and(new HasShippingAddressSpecification());
  }
  
  private spec: Specification<OrderAggregate>;
  
  isSatisfiedBy(order: OrderAggregate): boolean {
    return this.spec.isSatisfiedBy(order);
  }
}

// 使用
const canConfirmSpec = new CanBeConfirmedSpecification();

if (canConfirmSpec.isSatisfiedBy(order)) {
  order.confirm();
} else {
  throw new Error('Order cannot be confirmed');
}
```

## 📚 FAQ

### 1. DDD vs 传统三层架构？

| 特性 | 三层架构 | DDD |
|------|----------|-----|
| **关注点** | 技术分层 | 业务领域 |
| **模型** | 贫血模型 | 充血模型 |
| **复杂度** | 低 | 高 |
| **适用** | 简单 CRUD | 复杂业务 |

### 2. 何时使用 DDD？

**适用：**
- 业务逻辑复杂
- 需要长期维护
- 领域专家参与

**不适用：**
- 简单 CRUD
- 快速原型
- 技术驱动项目

### 3. 如何开始 DDD？

**步骤：**
1. **事件风暴**：识别领域事件
2. **划分上下文**：定义限界上下文
3. **建模**：识别实体、值对象、聚合
4. **实现**：编写代码
