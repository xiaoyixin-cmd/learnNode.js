# 10.1 微内核架构

> 设计可扩展的插件化系统，实现核心与扩展分离

---

## 📚 知识导读

### 什么是微内核架构？

**定义：**  
微内核架构（Microkernel Architecture）是一种将系统分为核心系统和插件模块的架构模式，核心提供最小功能集，扩展功能通过插件实现。

**核心组成：**
- **核心系统（Core System）**：最小化功能集
- **插件（Plugins）**：独立的扩展模块
- **插件管理器**：加载、卸载、生命周期管理
- **事件总线**：核心与插件通信

**典型应用：**
- VS Code、Eclipse（IDE）
- Webpack、Rollup（构建工具）
- WordPress、Strapi（CMS）
- Chrome、Firefox（浏览器）

### 1. 核心架构

```typescript
// core/plugin-manager.ts
import { EventEmitter } from 'events';

interface Plugin {
  name: string;
  version: string;
  activate?(context: PluginContext): Promise<void> | void;
  deactivate?(): Promise<void> | void;
}

interface PluginContext {
  registerCommand(name: string, handler: Function): void;
  registerService(name: string, service: any): void;
  getService(name: string): any;
}

class PluginManager extends EventEmitter {
  private plugins = new Map<string, Plugin>();
  private services = new Map<string, any>();
  private commands = new Map<string, Function>();
  
  // 加载插件
  async loadPlugin(pluginPath: string): Promise<void> {
    try {
      // 动态导入插件
      const module = await import(pluginPath);
      const plugin: Plugin = module.default || module;
      
      if (this.plugins.has(plugin.name)) {
        throw new Error(`Plugin ${plugin.name} already loaded`);
      }
      
      // 创建插件上下文
      const context: PluginContext = {
        registerCommand: (name, handler) => {
          this.commands.set(name, handler);
        },
        registerService: (name, service) => {
          this.services.set(name, service);
        },
        getService: (name) => {
          return this.services.get(name);
        },
      };
      
      // 激活插件
      await plugin.activate?.(context);
      
      this.plugins.set(plugin.name, plugin);
      this.emit('plugin:loaded', plugin.name);
      
      console.log(`✅ Plugin loaded: ${plugin.name}@${plugin.version}`);
    } catch (error) {
      console.error(`❌ Failed to load plugin:`, error);
      throw error;
    }
  }
  
  // 卸载插件
  async unloadPlugin(name: string): Promise<void> {
    const plugin = this.plugins.get(name);
    if (!plugin) {
      throw new Error(`Plugin ${name} not found`);
    }
    
    await plugin.deactivate?.();
    this.plugins.delete(name);
    this.emit('plugin:unloaded', name);
    
    console.log(`🗑️ Plugin unloaded: ${name}`);
  }
  
  // 执行命令
  async executeCommand(name: string, ...args: any[]): Promise<any> {
    const handler = this.commands.get(name);
    if (!handler) {
      throw new Error(`Command ${name} not found`);
    }
    
    return await handler(...args);
  }
  
  // 获取服务
  getService<T = any>(name: string): T {
    const service = this.services.get(name);
    if (!service) {
      throw new Error(`Service ${name} not found`);
    }
    return service;
  }
  
  // 获取所有插件
  getAllPlugins(): Plugin[] {
    return Array.from(this.plugins.values());
  }
}

export const pluginManager = new PluginManager();
```

## 🎯 热加载与热更新

### 1. 文件监听

```typescript
import chokidar from 'chokidar';
import path from 'path';

class HotReloadManager {
  private watcher: chokidar.FSWatcher | null = null;
  
  watch(pluginsDir: string) {
    this.watcher = chokidar.watch(path.join(pluginsDir, '**/*.{ts,js}'), {
      persistent: true,
      ignoreInitial: true,
    });
    
    this.watcher.on('change', async (filePath) => {
      console.log(`🔄 File changed: ${filePath}`);
      
      // 清除模块缓存
      delete require.cache[require.resolve(filePath)];
      
      // 重新加载插件
      const pluginName = this.getPluginName(filePath);
      
      try {
        await pluginManager.unloadPlugin(pluginName);
        await pluginManager.loadPlugin(filePath);
        console.log(`✅ Plugin reloaded: ${pluginName}`);
      } catch (error) {
        console.error(`❌ Failed to reload plugin:`, error);
      }
    });
    
    console.log(`👀 Watching plugins in ${pluginsDir}`);
  }
  
  stop() {
    this.watcher?.close();
  }
  
  private getPluginName(filePath: string): string {
    // 从路径提取插件名
    const parts = filePath.split(path.sep);
    return parts[parts.indexOf('plugins') + 1];
  }
}

export const hotReloadManager = new HotReloadManager();
```

```typescript
// utils/module-loader.ts
import path from 'path';

function clearModuleCache(modulePath: string) {
  const resolved = require.resolve(modulePath);
  
  // 删除主模块缓存
  delete require.cache[resolved];
  
  // 删除依赖模块缓存
  const module = require.cache[resolved];
  if (module?.children) {
    module.children.forEach((child) => {
      delete require.cache[child.id];
    });
  }
}

async function loadModule(modulePath: string) {
  // 清除缓存
  clearModuleCache(modulePath);
  
  // 重新加载
  return await import(modulePath + '?t=' + Date.now());  // 防止浏览器缓存
}

export { clearModuleCache, loadModule };
```

### 2. 依赖解析

```typescript
class PluginManager {
  // ... 其他代码
  
  async loadPluginWithDependencies(pluginPath: string): Promise<void> {
    const module = await import(pluginPath);
    const plugin = module.default || module;
    
    // 检查依赖
    if (plugin.dependencies) {
      for (const dep of plugin.dependencies) {
        if (!this.plugins.has(dep)) {
          throw new Error(
            `Plugin ${plugin.name} requires ${dep}, but it's not loaded`
          );
        }
      }
    }
    
    await this.loadPlugin(pluginPath);
  }
  
  // 拓扑排序加载
  async loadPluginsInOrder(pluginPaths: string[]): Promise<void> {
    const graph = new Map<string, string[]>();
    const plugins = new Map<string, any>();
    
    // 构建依赖图
    for (const path of pluginPaths) {
      const module = await import(path);
      const plugin = module.default || module;
      plugins.set(plugin.name, { path, plugin });
      graph.set(plugin.name, plugin.dependencies || []);
    }
    
    // 拓扑排序
    const sorted = this.topologicalSort(graph);
    
    // 按顺序加载
    for (const name of sorted) {
      const { path } = plugins.get(name)!;
      await this.loadPlugin(path);
    }
  }
  
  private topologicalSort(graph: Map<string, string[]>): string[] {
    const visited = new Set<string>();
    const result: string[] = [];
    
    const visit = (node: string) => {
      if (visited.has(node)) return;
      visited.add(node);
      
      const deps = graph.get(node) || [];
      for (const dep of deps) {
        visit(dep);
      }
      
      result.push(node);
    };
    
    for (const node of graph.keys()) {
      visit(node);
    }
    
    return result;
  }
}
```

```typescript
// core/event-bus.ts
import { EventEmitter } from 'events';

class EventBus extends EventEmitter {
  // 发布事件
  publish(event: string, data?: any) {
    this.emit(event, data);
  }
  
  // 订阅事件
  subscribe(event: string, handler: (data?: any) => void) {
    this.on(event, handler);
    
    // 返回取消订阅函数
    return () => this.off(event, handler);
  }
  
  // 订阅一次
  subscribeOnce(event: string, handler: (data?: any) => void) {
    this.once(event, handler);
  }
}

export const eventBus = new EventBus();
```

```typescript
// core/vscode-like-manager.ts
class ExtensionManager {
  private extensions = new Map();
  
  async activateExtension(extensionPath: string) {
    const module = await import(extensionPath);
    
    const context = {
      subscriptions: [],
      extensionPath,
      globalState: new GlobalState(),
      workspaceState: new WorkspaceState(),
    };
    
    // 调用激活函数
    await module.activate(context);
    
    this.extensions.set(extensionPath, { module, context });
  }
  
  async deactivateExtension(extensionPath: string) {
    const ext = this.extensions.get(extensionPath);
    if (!ext) return;
    
    // 清理订阅
    for (const subscription of ext.context.subscriptions) {
      subscription.dispose();
    }
    
    // 调用停用函数
    await ext.module.deactivate?.();
    
    this.extensions.delete(extensionPath);
  }
}

class GlobalState implements Memento {
  private storage = new Map();
  
  get<T>(key: string): T | undefined {
    return this.storage.get(key);
  }
  
  async update(key: string, value: any): Promise<void> {
    this.storage.set(key, value);
  }
}
```

```typescript
import { Worker } from 'worker_threads';

class WorkerPlugin {
  async execute(scriptPath: string, data: any): Promise<any> {
    return new Promise((resolve, reject) => {
      const worker = new Worker(scriptPath, {
        workerData: data,
      });
      
      worker.on('message', resolve);
      worker.on('error', reject);
      worker.on('exit', (code) => {
        if (code !== 0) {
          reject(new Error(`Worker stopped with exit code ${code}`));
        }
      });
    });
  }
```

## 📚 FAQ

### 1. 微内核 vs 微服务？

| 特性 | 微内核 | 微服务 |
|------|--------|--------|
| **粒度** | 进程内模块 | 独立进程 |
| **通信** | 函数调用/事件 | HTTP/RPC |
| **部署** | 单体 | 分布式 |
| **适用** | 单应用扩展 | 多服务系统 |

### 2. 何时使用微内核架构？

**适用场景：**
- 功能高度可扩展（IDE、CMS）
- 需要第三方插件
- 核心功能稳定，扩展频繁

**不适用：**
- 简单应用（过度设计）
- 需要强隔离（用微服务）

### 3. 如何管理插件版本？

**方案：**
1. 语义化版本（SemVer）
2. 插件市场/注册表
3. 自动更新机制
