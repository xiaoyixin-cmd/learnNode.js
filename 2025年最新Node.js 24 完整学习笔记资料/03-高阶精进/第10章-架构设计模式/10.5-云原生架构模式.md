## 📚 知识导读

### 什么是云原生？

**定义：**  
云原生（Cloud Native）是一种构建和运行应用的方法，充分利用云计算的优势（弹性、分布式、自动化）。

**核心特征：**
- **容器化**：Docker、Kubernetes
- **微服务**：松耦合、独立部署
- **DevOps**：CI/CD、自动化
- **声明式 API**：YAML 配置
- **弹性伸缩**：自动扩缩容

**CNCF 定义：**  
云原生技术有利于各组织在公有云、私有云和混合云等新型动态环境中，构建和运行可弹性扩展的应用。

## 🎯 12-Factor 应用

### 1. 代码库（Codebase）

**原则：**  
一份基准代码，多份部署。

```bash
# ✅ 正确：单一代码库，多环境配置
my-app/
├── src/
├── config/
│   ├── development.yaml
│   ├── staging.yaml
│   └── production.yaml
└── package.json

# 部署到不同环境
npm run deploy:dev
npm run deploy:staging
npm run deploy:prod
```

### 2. 依赖（Dependencies）

**原则：**  
显式声明依赖，不依赖系统工具。

```json
// package.json
{
  "name": "my-app",
  "version": "1.0.0",
  "dependencies": {
    "express": "^4.18.0",
    "mongodb": "^6.0.0"
  },
  "devDependencies": {
    "typescript": "^5.0.0"
  },
  "engines": {
    "node": ">=20.0.0"
  }
}
```

```dockerfile
# Dockerfile（锁定依赖版本）
FROM node:20-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci  # 使用 package-lock.json 精确版本

COPY . .
CMD ["npm", "start"]
```

### 3. 配置（Config）

**原则：**  
在环境中存储配置，不硬编码。

```typescript
// ❌ 错误：硬编码
const dbUrl = 'mongodb://localhost:27017/mydb';

// ✅ 正确：环境变量
const dbUrl = process.env.DATABASE_URL || 'mongodb://localhost:27017/mydb';

// config/database.ts
import { config } from 'dotenv';

config();

export const databaseConfig = {
  url: process.env.DATABASE_URL!,
  poolSize: parseInt(process.env.DB_POOL_SIZE || '10'),
  ssl: process.env.DB_SSL === 'true',
};
```

```yaml
# Kubernetes ConfigMap
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  DATABASE_URL: "mongodb://mongo:27017/mydb"
  REDIS_URL: "redis://redis:6379"
  LOG_LEVEL: "info"
```

### 4. 后端服务（Backing Services）

**原则：**  
把后端服务当作附加资源。

```typescript
// infrastructure/database.factory.ts
export class DatabaseFactory {
  static create(): Database {
    const dbUrl = process.env.DATABASE_URL!;
    
    if (dbUrl.startsWith('mongodb://')) {
      return new MongoDatabase(dbUrl);
    } else if (dbUrl.startsWith('postgresql://')) {
      return new PostgresDatabase(dbUrl);
    }
    
    throw new Error('Unsupported database');
  }
}

// 切换数据库只需更改环境变量
// DATABASE_URL=mongodb://... → DATABASE_URL=postgresql://...
```

### 5. 构建、发布、运行（Build, Release, Run）

**原则：**  
严格分离构建、发布和运行阶段。

```yaml
# .github/workflows/deploy.yml
name: Deploy

on:
  push:
    branches: [main]

jobs:
  # 1. 构建阶段
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - run: npm ci
      - run: npm run build
      - run: docker build -t myapp:${{ github.sha }} .
      - run: docker push myapp:${{ github.sha }}
  
  # 2. 发布阶段
  release:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - run: |
          kubectl set image deployment/myapp \
            myapp=myapp:${{ github.sha }}
  
  # 3. 运行阶段（Kubernetes 自动）
```

### 6. 进程（Processes）

**原则：**  
应用以无状态进程运行。

```typescript
// ❌ 错误：存储在内存
const sessions = new Map();  // 进程重启丢失

app.post('/login', (req, res) => {
  const sessionId = generateId();
  sessions.set(sessionId, req.body.user);  // 存储在内存
  res.cookie('session', sessionId);
});

// ✅ 正确：存储在外部服务
import { createClient } from 'redis';

const redis = createClient({ url: process.env.REDIS_URL });

app.post('/login', async (req, res) => {
  const sessionId = generateId();
  await redis.set(`session:${sessionId}`, JSON.stringify(req.body.user), {
    EX: 3600,  // 1小时过期
  });
  res.cookie('session', sessionId);
});
```

### 9. 易处理（Disposability）

**原则：**  
快速启动和优雅终止。

```typescript
// server.ts
const server = app.listen(PORT);

// 优雅关闭
process.on('SIGTERM', async () => {
  console.log('SIGTERM received, shutting down gracefully');
  
  // 1. 停止接受新请求
  server.close(() => {
    console.log('HTTP server closed');
  });
  
  // 2. 完成正在处理的请求
  await new Promise((resolve) => {
    setTimeout(resolve, 5000);  // 最多等待5秒
  });
  
  // 3. 关闭数据库连接
  await mongoose.connection.close();
  
  // 4. 退出
  process.exit(0);
});
```

## 🎯 云原生模式

### 1. Retry 模式

```typescript
// utils/retry.ts
export async function withRetry<T>(
  fn: () => Promise<T>,
  options = { maxRetries: 3, delay: 1000 }
): Promise<T> {
  let lastError: Error;
  
  for (let i = 0; i < options.maxRetries; i++) {
    try {
      return await fn();
    } catch (error: any) {
      lastError = error;
      console.warn(`Retry ${i + 1}/${options.maxRetries} failed:`, error.message);
      
      if (i < options.maxRetries - 1) {
        await new Promise((resolve) => setTimeout(resolve, options.delay * Math.pow(2, i)));
      }
    }
  }
  
  throw lastError!;
}

// 使用
const data = await withRetry(
  () => fetch('https://api.example.com/data').then(r => r.json()),
  { maxRetries: 3, delay: 1000 }
);
```

### 2. Circuit Breaker 模式

```typescript
// utils/circuit-breaker.ts
enum CircuitState {
  CLOSED,
  OPEN,
  HALF_OPEN,
}

export class CircuitBreaker {
  private state = CircuitState.CLOSED;
  private failureCount = 0;
  private nextAttempt = Date.now();
  
  constructor(
    private threshold = 5,
    private timeout = 60000
  ) {}
  
  async execute<T>(fn: () => Promise<T>): Promise<T> {
    if (this.state === CircuitState.OPEN) {
      if (Date.now() < this.nextAttempt) {
        throw new Error('Circuit breaker is OPEN');
      }
      this.state = CircuitState.HALF_OPEN;
    }
    
    try {
      const result = await fn();
      this.onSuccess();
      return result;
    } catch (error) {
      this.onFailure();
      throw error;
    }
  }
  
  private onSuccess(): void {
    this.failureCount = 0;
    this.state = CircuitState.CLOSED;
  }
  
  private onFailure(): void {
    this.failureCount++;
    
    if (this.failureCount >= this.threshold) {
      this.state = CircuitState.OPEN;
      this.nextAttempt = Date.now() + this.timeout;
      console.warn('Circuit breaker OPEN');
    }
  }
}

// 使用
const breaker = new CircuitBreaker(5, 60000);

app.get('/api/data', async (req, res) => {
  try {
    const data = await breaker.execute(() => 
      fetch('https://external-api.com/data').then(r => r.json())
    );
    res.json(data);
  } catch (error: any) {
    res.status(503).json({ error: 'Service unavailable' });
  }
});
```

### 3. Bulkhead 模式（隔离舱模式）

```typescript
// utils/bulkhead.ts
import PQueue from 'p-queue';

export class Bulkhead {
  private queues = new Map<string, PQueue>();
  
  getQueue(name: string, concurrency: number = 10): PQueue {
    if (!this.queues.has(name)) {
      this.queues.set(name, new PQueue({ concurrency }));
    }
    return this.queues.get(name)!;
  }
}

const bulkhead = new Bulkhead();

// 数据库操作限制并发10
app.post('/users', async (req, res) => {
  await bulkhead.getQueue('database', 10).add(async () => {
    const user = await User.create(req.body);
    res.json(user);
  });
});

// 外部 API 限制并发5
app.get('/external-data', async (req, res) => {
  await bulkhead.getQueue('external-api', 5).add(async () => {
    const data = await fetch('https://api.example.com').then(r => r.json());
    res.json(data);
  });
});
```

## 📚 FAQ

### 1. 12-Factor vs 传统部署？

| 特性 | 传统部署 | 12-Factor |
|------|----------|-----------|
| **配置** | 硬编码 | 环境变量 |
| **状态** | 有状态 | 无状态 |
| **扩展** | 垂直 | 水平 |
| **日志** | 文件 | Stdout |

### 2. Service Mesh 的优势？

**优势：**
- 流量管理（路由、负载均衡）
- 可观测性（Metrics、Tracing、Logging）
- 安全（mTLS、访问控制）

### 3. 何时使用 Sidecar？

**适用：**
- 日志收集
- 配置同步
- 服务代理
- 监控代理
