# 10.3 CQRS 与事件溯源

> 命令查询职责分离 + 事件溯源，构建高性能、可追溯的系统

---

## 📚 知识导读

### 什么是 CQRS？

**定义：**  
CQRS（Command Query Responsibility Segregation）是一种将读操作（Query）和写操作（Command）分离的架构模式。

**核心思想：**
- **Command（命令）**：修改状态，不返回数据
- **Query（查询）**：返回数据，不修改状态
- **分离模型**：读模型和写模型独立

**优势：**
- ✅ 读写优化独立（读用 Redis，写用 PostgreSQL）
- ✅ 扩展性好（读写分库）
- ✅ 复杂查询简化（查询模型预计算）

---

### 什么是事件溯源？

**定义：**  
Event Sourcing（事件溯源）是一种将所有状态变更记录为事件序列的模式，通过事件回放重建系统状态。

**核心概念：**
- **事件（Event）**：不可变的状态变更记录
- **事件流（Event Stream）**：按时间顺序的事件序列
- **重放（Replay）**：通过事件流重建状态

**优势：**
- ✅ 完整历史记录（审计、溯源）
- ✅ 时间旅行（回到任意时刻）
- ✅ 易于调试（事件重放）

### 2. 聚合根（Event Sourced）

```typescript
// domain/user.aggregate.ts
import { v4 as uuidv4 } from 'uuid';

export class User {
  public id: string;
  public name: string;
  public email: string;
  
  private changes: DomainEvent[] = [];  // 未提交的事件
  private version: number = 0;
  
  // 从事件流重建
  static fromEvents(events: DomainEvent[]): User {
    const user = new User();
    
    for (const event of events) {
      user.apply(event, false);  // 不记录到 changes
      user.version++;
    }
    
    return user;
  }
  
  // 创建新用户
  static create(name: string, email: string): User {
    const user = new User();
    user.apply(new UserCreatedEvent(uuidv4(), name, email));
    return user;
  }
  
  // 修改名字
  changeName(newName: string): void {
    if (this.name === newName) return;
    this.apply(new UserNameChangedEvent(this.id, newName));
  }
  
  // 修改邮箱
  changeEmail(newEmail: string): void {
    if (this.email === newEmail) return;
    this.apply(new UserEmailChangedEvent(this.id, newEmail));
  }
  
  // 应用事件
  private apply(event: DomainEvent, isNew: boolean = true): void {
    // 更新状态
    if (event instanceof UserCreatedEvent) {
      this.id = event.aggregateId;
      this.name = event.name;
      this.email = event.email;
    } else if (event instanceof UserNameChangedEvent) {
      this.name = event.newName;
    } else if (event instanceof UserEmailChangedEvent) {
      this.email = event.newEmail;
    }
    
    // 记录未提交的事件
    if (isNew) {
      this.changes.push(event);
    }
  }
  
  // 获取未提交的事件
  getUncommittedChanges(): DomainEvent[] {
    return this.changes;
  }
  
  // 标记已提交
  markChangesAsCommitted(): void {
    this.changes = [];
  }
  
  getVersion(): number {
    return this.version;
  }
}
```

### 3. 事件存储

```typescript
// infrastructure/event-store.ts
import { Pool } from 'pg';

interface StoredEvent {
  id: number;
  aggregateId: string;
  eventType: string;
  eventData: any;
  occurredOn: Date;
  version: number;
}

export class EventStore {
  constructor(private readonly db: Pool) {}
  
  async save(aggregateId: string, events: DomainEvent[], expectedVersion: number): Promise<void> {
    const client = await this.db.connect();
    
    try {
      await client.query('BEGIN');
      
      // 检查版本（乐观锁）
      const { rows } = await client.query(
        'SELECT MAX(version) as current_version FROM events WHERE aggregate_id = $1',
        [aggregateId]
      );
      
      const currentVersion = rows[0]?.current_version || 0;
      
      if (currentVersion !== expectedVersion) {
        throw new Error('Concurrency conflict');
      }
      
      // 保存事件
      for (const [index, event] of events.entries()) {
        await client.query(
          `INSERT INTO events (aggregate_id, event_type, event_data, occurred_on, version)
           VALUES ($1, $2, $3, $4, $5)`,
          [
            aggregateId,
            event.constructor.name,
            JSON.stringify(event),
            event.occurredOn,
            expectedVersion + index + 1,
          ]
        );
      }
      
      await client.query('COMMIT');
    } catch (error) {
      await client.query('ROLLBACK');
      throw error;
    } finally {
      client.release();
    }
  }
  
  async getEvents(aggregateId: string): Promise<DomainEvent[]> {
    const { rows } = await this.db.query<StoredEvent>(
      'SELECT * FROM events WHERE aggregate_id = $1 ORDER BY version ASC',
      [aggregateId]
    );
    
    return rows.map((row) => this.deserializeEvent(row));
  }
  
  async getAllEvents(fromVersion: number = 0): Promise<DomainEvent[]> {
    const { rows } = await this.db.query<StoredEvent>(
      'SELECT * FROM events WHERE id > $1 ORDER BY id ASC',
      [fromVersion]
    );
    
    return rows.map((row) => this.deserializeEvent(row));
  }
  
  private deserializeEvent(stored: StoredEvent): DomainEvent {
    const data = JSON.parse(stored.eventData);
    
    switch (stored.eventType) {
      case 'UserCreatedEvent':
        return new UserCreatedEvent(stored.aggregateId, data.name, data.email);
      case 'UserNameChangedEvent':
        return new UserNameChangedEvent(stored.aggregateId, data.newName);
      case 'UserEmailChangedEvent':
        return new UserEmailChangedEvent(stored.aggregateId, data.newEmail);
      default:
        throw new Error(`Unknown event type: ${stored.eventType}`);
    }
  }
}
```

### 4. 仓储（Event Sourced）

```typescript
// repositories/user.event-sourced.repository.ts
export class EventSourcedUserRepository {
  constructor(private readonly eventStore: EventStore) {}
  
  async findById(userId: string): Promise<User | null> {
    const events = await this.eventStore.getEvents(userId);
    
    if (events.length === 0) {
      return null;
    }
    
    return User.fromEvents(events);
  }
  
  async save(user: User): Promise<void> {
    const uncommittedEvents = user.getUncommittedChanges();
    
    if (uncommittedEvents.length === 0) {
      return;
    }
    
    await this.eventStore.save(user.id, uncommittedEvents, user.getVersion());
    
    user.markChangesAsCommitted();
  }
}
```

```typescript
import { EventEmitter } from 'events';

export class EventBus extends EventEmitter {
  async publish(event: DomainEvent): Promise<void> {
    this.emit(event.constructor.name, event);
  }
  
  subscribe(eventType: string, handler: (event: DomainEvent) => Promise<void>): void {
    this.on(eventType, handler);

}
```

```typescript
// app.ts
import express from 'express';

const app = express();
app.use(express.json());

const eventStore = new EventStore(pool);
const userRepository = new EventSourcedUserRepository(eventStore);
const eventBus = new EventBus();

// 命令：创建用户
app.post('/users', async (req, res) => {
  try {
    const { name, email } = req.body;
    
    // 1. 创建聚合根
    const user = User.create(name, email);
    
    // 2. 保存到事件存储
    await userRepository.save(user);
    
    // 3. 发布事件（更新读模型）
    for (const event of user.getUncommittedChanges()) {
      await eventBus.publish(event);
    }
    
    res.status(201).json({ id: user.id });
  } catch (error: any) {
    res.status(400).json({ error: error.message });
  }
});

// 查询：获取用户（从读模型）
app.get('/users/:id', async (req, res) => {
  const { id } = req.params;
  
  const data = await redis.hGetAll(`user:${id}`);
  
  if (!data || Object.keys(data).length === 0) {
    return res.status(404).json({ error: 'User not found' });
  }
  
  res.json(data);
});

// 命令：修改用户
app.put('/users/:id', async (req, res) => {
  try {
    const { id } = req.params;
    const { name, email } = req.body;
    
    // 1. 从事件流重建聚合根
    const user = await userRepository.findById(id);
    if (!user) {
      return res.status(404).json({ error: 'User not found' });
    }
    
    // 2. 执行业务逻辑
    if (name) user.changeName(name);
    if (email) user.changeEmail(email);
    
    // 3. 保存事件
    await userRepository.save(user);
    
    // 4. 发布事件
    for (const event of user.getUncommittedChanges()) {
      await eventBus.publish(event);
    }
    
    res.json({ success: true });
  } catch (error: any) {
    res.status(400).json({ error: error.message });
  }
});

app.listen(3000);
```

## 🎯 快照优化

### 1. 快照存储

```typescript
// infrastructure/snapshot-store.ts
interface Snapshot {
  aggregateId: string;
  version: number;
  state: any;
  createdAt: Date;
}

export class SnapshotStore {
  constructor(private readonly db: Pool) {}
  
  async save(aggregateId: string, version: number, state: any): Promise<void> {
    await this.db.query(
      `INSERT INTO snapshots (aggregate_id, version, state, created_at)
       VALUES ($1, $2, $3, $4)
       ON CONFLICT (aggregate_id) DO UPDATE
       SET version = $2, state = $3, created_at = $4`,
      [aggregateId, version, JSON.stringify(state), new Date()]
    );
  }
  
  async get(aggregateId: string): Promise<Snapshot | null> {
    const { rows } = await this.db.query(
      'SELECT * FROM snapshots WHERE aggregate_id = $1',
      [aggregateId]
    );
    
    if (rows.length === 0) return null;
    
    return {
      aggregateId: rows[0].aggregate_id,
      version: rows[0].version,
      state: JSON.parse(rows[0].state),
      createdAt: rows[0].created_at,
    };
  }
}
```

### 2. 带快照的仓储

```typescript
export class SnapshotUserRepository {
  constructor(
    private readonly eventStore: EventStore,
    private readonly snapshotStore: SnapshotStore
  ) {}
  
  async findById(userId: string): Promise<User | null> {
    // 1. 尝试加载快照
    const snapshot = await this.snapshotStore.get(userId);
    
    let user: User;
    let fromVersion: number;
    
    if (snapshot) {
      // 从快照恢复
      user = User.fromState(snapshot.state);
      fromVersion = snapshot.version;
    } else {
      user = new User();
      fromVersion = 0;
    }
    
    // 2. 加载快照之后的事件
    const events = await this.eventStore.getEventsAfterVersion(userId, fromVersion);
    
    for (const event of events) {
      user.apply(event, false);
    }
    
    return user;
  }
  
  async save(user: User): Promise<void> {
    const uncommittedEvents = user.getUncommittedChanges();
    
    if (uncommittedEvents.length === 0) return;
    
    // 保存事件
    await this.eventStore.save(user.id, uncommittedEvents, user.getVersion());
    
    // 每10个事件创建一次快照
    const newVersion = user.getVersion() + uncommittedEvents.length;
    if (newVersion % 10 === 0) {
      await this.snapshotStore.save(user.id, newVersion, user.toState());
    }
    
    user.markChangesAsCommitted();
  }
}
```

```typescript
import { EventStoreDBClient, jsonEvent } from '@eventstore/db-client';

const client = EventStoreDBClient.connectionString('esdb://localhost:2113?tls=false');

// 写入事件
async function appendEvent(streamName: string, event: DomainEvent) {
  const eventData = jsonEvent({
    type: event.constructor.name,
    data: {
      ...event,
      occurredOn: event.occurredOn.toISOString(),
    },
  });
  
  await client.appendToStream(streamName, eventData);
}

// 读取事件
async function readEvents(streamName: string): Promise<DomainEvent[]> {
  const events = client.readStream(streamName);
  
  const result: DomainEvent[] = [];
  
  for await (const resolvedEvent of events) {
    const eventData = resolvedEvent.event!;
    
    // 反序列化
    const event = deserializeEvent(eventData.type, eventData.data);
    result.push(event);
  }
  
  return result;
}

// 订阅事件流
async function subscribeToStream(streamName: string) {
  const subscription = client.subscribeToStream(streamName);
  
  for await (const resolvedEvent of subscription) {
    const event = deserializeEvent(
      resolvedEvent.event!.type,
      resolvedEvent.event!.data
    );
    
    // 处理事件
    await eventBus.publish(event);
  }
}
```

## 📚 FAQ

### 1. CQRS + 事件溯源 vs 传统 CRUD？

| 特性 | 传统 CRUD | CQRS + Event Sourcing |
|------|-----------|----------------------|
| **复杂度** | 低 | 高 |
| **性能** | 中 | 高（读写分离） |
| **历史记录** | 无 | 完整 |
| **调试** | 难 | 易（事件重放） |
| **适用场景** | 简单应用 | 复杂业务、审计需求 |

### 2. 何时使用事件溯源？

**适用：**
- 需要完整审计日志
- 时间旅行（回到任意状态）
- 复杂业务规则

**不适用：**
- 简单 CRUD
- 事件量巨大（存储成本）

### 3. 读模型延迟怎么办？

**方案：**
1. **最终一致性**：接受延迟
2. **同步写入**：写入时同步更新读模型
3. **版本号**：返回版本号，前端轮询
