# 3.4 API 网关：使用 Fastify 构建统一入口

> API Gateway：微服务架构的流量守门员

## 📚 核心知识

### 1. 为什么需要 API 网关？

**没有 API 网关的痛点：**

```
客户端（移动端/Web）
  ├─ 直接调用 10+ 个微服务
  ├─ 每个服务都需要实现认证鉴权
  ├─ 跨域配置重复
  ├─ 限流策略分散
  └─ 监控日志分散

问题：
❌ 客户端需要维护多个服务地址
❌ 安全策略重复实现
❌ 无法统一管理流量
❌ 难以实现灰度发布
```

**API 网关架构：**

```
┌──────────────────────────────────────────────────────────┐
│                       API Gateway                        │
│  ┌────────────────────────────────────────────────────┐ │
│  │  核心功能                                           │ │
│  │  ✅ 路由转发    ✅ 认证鉴权    ✅ 限流熔断          │ │
│  │  ✅ 请求聚合    ✅ 协议转换    ✅ 日志监控          │ │
│  │  ✅ 缓存       ✅ 灰度发布    ✅ API 版本管理       │ │
│  └────────────────────────────────────────────────────┘ │
└────────────┬─────────────────────────────┬───────────────┘
             │                             │
    ┌────────┴────────┐           ┌────────┴────────┐
    │  User Service   │           │  Order Service  │
    │  (内部服务)      │           │  (内部服务)      │
    └─────────────────┘           └─────────────────┘
```

**核心价值：**

1. **统一入口**：客户端只需对接一个地址
2. **安全屏障**：集中认证鉴权，内部服务无需关心
3. **流量管理**：限流、熔断、降级
4. **协议转换**：HTTP → gRPC、REST → GraphQL
5. **请求聚合**：减少客户端请求次数
6. **可观测性**：统一日志、追踪、监控

### 2. API 网关核心功能

```javascript
// API 网关处理流程
客户端请求
  ↓
[1. 认证鉴权] ← JWT 验证、API Key
  ↓
[2. 限流控制] ← 基于 IP/用户的速率限制
  ↓
[3. 路由匹配] ← /api/users/* → user-service
  ↓
[4. 服务发现] ← Consul/etcd 查询健康实例
  ↓
[5. 负载均衡] ← 轮询/随机/最少连接
  ↓
[6. 请求转发] ← HTTP Proxy / gRPC 调用
  ↓
[7. 响应聚合] ← 合并多个服务响应（可选）
  ↓
[8. 响应缓存] ← Redis 缓存热点数据
  ↓
[9. 日志追踪] ← 记录请求日志、追踪 ID
  ↓
返回客户端
```

### 实战2：集成服务发现（Consul）

```javascript
// src/gateway-with-consul.js
import Fastify from 'fastify';
import proxy from '@fastify/http-proxy';
import Consul from 'consul';

const fastify = Fastify({
  logger: true,
});

const consul = new Consul({
  host: '127.0.0.1',
  port: 8500,
  promisify: true,
});

// 服务实例缓存
const serviceInstances = new Map();

// 服务发现
async function discoverService(serviceName) {
  try {
    const result = await consul.health.service({
      service: serviceName,
      passing: true,
    });
    
    if (!result || result.length === 0) {
      throw new Error(`No healthy instances found for ${serviceName}`);
    }
    
    const instances = result.map(item => ({
      address: item.Service.Address,
      port: item.Service.Port,
    }));
    
    serviceInstances.set(serviceName, instances);
    return instances;
  } catch (error) {
    fastify.log.error(`Service discovery failed for ${serviceName}:`, error);
    throw error;
  }
}

// 定期更新服务列表
async function updateServiceInstances() {
  const services = ['user-service', 'order-service'];
  
  for (const service of services) {
    await discoverService(service);
  }
  
  fastify.log.info('Service instances updated');
}

// 初始化并启动定时更新
updateServiceInstances();
setInterval(updateServiceInstances, 30000);

// 客户端负载均衡
function getServiceInstance(serviceName) {
  const instances = serviceInstances.get(serviceName);
  
  if (!instances || instances.length === 0) {
    throw new Error(`No available instances for ${serviceName}`);
  }
  
  // 简单轮询
  const index = Math.floor(Math.random() * instances.length);
  const instance = instances[index];
  
  return `http://${instance.address}:${instance.port}`;
}

// 动态代理中间件
fastify.register(async (instance) => {
  instance.addHook('onRequest', async (request, reply) => {
    // 解析服务名
    const match = request.url.match(/^\/api\/([^\/]+)/);
    
    if (!match) {
      reply.code(404).send({ error: 'Service not found' });
      return;
    }
    
    const serviceName = `${match[1]}-service`;  // users → user-service
    
    try {
      // 获取服务实例
      const upstream = getServiceInstance(serviceName);
      
      // 动态设置代理目标
      request.upstream = upstream;
    } catch (error) {
      fastify.log.error(error);
      reply.code(503).send({ error: 'Service unavailable' });
    }
  });
  
  // 用户服务
  instance.register(proxy, {
    upstream: (request) => request.upstream,
    prefix: '/api/users',
    rewritePrefix: '/users',
  });
  
  // 订单服务
  instance.register(proxy, {
    upstream: (request) => request.upstream,
    prefix: '/api/orders',
    rewritePrefix: '/orders',
  });
});

fastify.listen({ port: 8080, host: '0.0.0.0' }, (err) => {
  if (err) {
    fastify.log.error(err);
    process.exit(1);
  }
  console.log('🚀 API Gateway with Consul running on http://localhost:8080');
});
```

### 实战3：JWT 认证鉴权

```javascript
// src/gateway-with-auth.js
import Fastify from 'fastify';
import proxy from '@fastify/http-proxy';
import jwt from '@fastify/jwt';

const fastify = Fastify({ logger: true });

// 注册 JWT 插件
fastify.register(jwt, {
  secret: 'your-secret-key-change-in-production',  // ⚠️ 生产环境使用环境变量
});

// JWT 验证装饰器
fastify.decorate('authenticate', async function(request, reply) {
  try {
    await request.jwtVerify();
  } catch (err) {
    reply.code(401).send({ error: 'Unauthorized' });
  }
});

// 登录接口（生成 Token）
fastify.post('/auth/login', async (request, reply) => {
  const { username, password } = request.body;
  
  // ⚠️ 实际应用中应该查询数据库并验证密码
  if (username === 'admin' && password === 'password') {
    const token = fastify.jwt.sign({
      userId: '123',
      username: 'admin',
      role: 'admin',
    }, {
      expiresIn: '1h',  // 1 小时过期
    });
    
    return { token };
  }
  
  reply.code(401).send({ error: 'Invalid credentials' });
});

// 公开路由（无需认证）
fastify.get('/api/public/products', async (request, reply) => {
  return { message: 'Public API' };
});

// 受保护路由（需要认证）
fastify.register(async (instance) => {
  // 所有路由都需要认证
  instance.addHook('onRequest', instance.authenticate);
  
  // 用户服务
  instance.register(proxy, {
    upstream: 'http://localhost:3000',
    prefix: '/api/users',
    rewritePrefix: '/users',
    preHandler: (request, reply, done) => {
      // 将用户信息传递给下游服务
      request.headers['x-user-id'] = request.user.userId;
      request.headers['x-username'] = request.user.username;
      done();
    },
  });
  
  // 订单服务
  instance.register(proxy, {
    upstream: 'http://localhost:4000',
    prefix: '/api/orders',
    rewritePrefix: '/orders',
    preHandler: (request, reply, done) => {
      request.headers['x-user-id'] = request.user.userId;
      done();
    },
  });
});

fastify.listen({ port: 8080, host: '0.0.0.0' });
```

### 实战4：限流控制

```javascript
// src/gateway-with-rate-limit.js
import Fastify from 'fastify';
import proxy from '@fastify/http-proxy';
import rateLimit from '@fastify/rate-limit';

const fastify = Fastify({ logger: true });

// 全局限流：每个 IP 每分钟最多 100 次请求
fastify.register(rateLimit, {
  max: 100,
  timeWindow: '1 minute',
  errorResponseBuilder: (request, context) => {
    return {
      statusCode: 429,
      error: 'Too Many Requests',
      message: `Rate limit exceeded, retry in ${context.after}`,
    };
  },
});

// 特定路由限流（更严格）
fastify.register(async (instance) => {
  instance.register(rateLimit, {
    max: 10,
    timeWindow: '1 minute',
    keyGenerator: (request) => {
      // 基于用户 ID 限流（需先认证）
      return request.user?.userId || request.ip;
    },
  });
  
  // 敏感操作（如创建订单）
  instance.register(proxy, {
    upstream: 'http://localhost:4000',
    prefix: '/api/orders',
    rewritePrefix: '/orders',
    method: 'POST',  // 只限制 POST 请求
  });
});

// 查询接口不限流
fastify.register(proxy, {
  upstream: 'http://localhost:3000',
  prefix: '/api/users',
  rewritePrefix: '/users',
  method: 'GET',
});

fastify.listen({ port: 8080, host: '0.0.0.0' });
```

### 实战5：请求聚合（API Composition）

```javascript
// src/gateway-with-aggregation.js
import Fastify from 'fastify';
import axios from 'axios';

const fastify = Fastify({ logger: true });

// 聚合接口：获取用户完整信息（用户 + 订单 + 地址）
fastify.get('/api/user-profile/:userId', async (request, reply) => {
  const { userId } = request.params;
  
  try {
    // 并发调用多个服务
    const [userRes, ordersRes, addressRes] = await Promise.all([
      axios.get(`http://localhost:3000/users/${userId}`),
      axios.get(`http://localhost:4000/orders?userId=${userId}`),
      axios.get(`http://localhost:5000/addresses?userId=${userId}`),
    ]);
    
    // 聚合数据
    return {
      user: userRes.data,
      orders: ordersRes.data,
      addresses: addressRes.data,
    };
  } catch (error) {
    fastify.log.error(error);
    reply.code(503).send({ error: 'Service unavailable' });
  }
});

// 对比：不聚合时，客户端需要发起 3 次请求
// GET /api/users/123
// GET /api/orders?userId=123
// GET /api/addresses?userId=123

// 聚合后：只需 1 次请求
// GET /api/user-profile/123

fastify.listen({ port: 8080, host: '0.0.0.0' });
```

**性能对比：**
```
不聚合（3 次请求）：
- 总延迟: 50ms + 60ms + 40ms = 150ms（串行）
- 或: max(50, 60, 40) = 60ms（并行，但客户端复杂）

聚合（1 次请求）：
- 网关并发调用: max(50, 60, 40) = 60ms
- 客户端只需等待 1 次网络往返
```

### 实战6：灰度发布（金丝雀部署）

```javascript
// src/gateway-with-canary.js
import Fastify from 'fastify';
import proxy from '@fastify/http-proxy';

const fastify = Fastify({ logger: true });

// 灰度配置
const canaryConfig = {
  'user-service': {
    stable: 'http://localhost:3000',  // 稳定版本
    canary: 'http://localhost:3001',  // 金丝雀版本
    canaryPercentage: 10,             // 10% 流量走金丝雀
  },
};

// 灰度路由中间件
fastify.addHook('onRequest', async (request, reply) => {
  const match = request.url.match(/^\/api\/([^\/]+)/);
  
  if (!match) return;
  
  const serviceName = `${match[1]}-service`;
  const config = canaryConfig[serviceName];
  
  if (!config) return;
  
  // 根据随机数决定路由到哪个版本
  const random = Math.random() * 100;
  
  if (random < config.canaryPercentage) {
    request.upstream = config.canary;
    request.headers['x-version'] = 'canary';
    fastify.log.info(`🐤 Routing to canary: ${serviceName}`);
  } else {
    request.upstream = config.stable;
    request.headers['x-version'] = 'stable';
  }
});

// 用户服务（支持灰度）
fastify.register(proxy, {
  upstream: (request) => request.upstream || 'http://localhost:3000',
  prefix: '/api/users',
  rewritePrefix: '/users',
});

fastify.listen({ port: 8080, host: '0.0.0.0' });
```

### 实战7：响应缓存

```javascript
// src/gateway-with-cache.js
import Fastify from 'fastify';
import proxy from '@fastify/http-proxy';
import Redis from 'ioredis';

const fastify = Fastify({ logger: true });

const redis = new Redis({
  host: '127.0.0.1',
  port: 6379,
});

// 缓存中间件
async function cacheMiddleware(request, reply) {
  // 只缓存 GET 请求
  if (request.method !== 'GET') return;
  
  const cacheKey = `cache:${request.url}`;
  
  // 尝试从缓存获取
  const cached = await redis.get(cacheKey);
  
  if (cached) {
    fastify.log.info(`✅ Cache hit: ${cacheKey}`);
    reply
      .header('X-Cache', 'HIT')
      .send(JSON.parse(cached));
    return reply;
  }
  
  // 缓存未命中，继续处理请求
  request.cacheKey = cacheKey;
}

// 用户服务（带缓存）
fastify.register(async (instance) => {
  instance.addHook('onRequest', cacheMiddleware);
  
  instance.register(proxy, {
    upstream: 'http://localhost:3000',
    prefix: '/api/users',
    rewritePrefix: '/users',
    preHandler: async (request, reply) => {
      // 保存原始 reply.send 方法
      const originalSend = reply.send.bind(reply);
      
      // 重写 send 方法，拦截响应并缓存
      reply.send = function(payload) {
        if (request.cacheKey && reply.statusCode === 200) {
          redis.setex(request.cacheKey, 60, JSON.stringify(payload));  // 缓存 60 秒
          reply.header('X-Cache', 'MISS');
          fastify.log.info(`📦 Cached: ${request.cacheKey}`);
        }
        
        return originalSend(payload);
      };
    },
  });
});

fastify.listen({ port: 8080, host: '0.0.0.0' });
```

**测试：**
```bash
# 第一次请求（缓存未命中）
curl -i http://localhost:8080/api/users/123
# X-Cache: MISS

# 第二次请求（缓存命中）
curl -i http://localhost:8080/api/users/123
# X-Cache: HIT

# 60 秒后缓存过期，再次请求会重新缓存
```

## ❌ 高频错误

### 错误1：未处理下游服务超时

**错误代码：**
```javascript
// ❌ 错误：未设置超时
fastify.register(proxy, {
  upstream: 'http://localhost:3000',
  prefix: '/api/users',
});

// 后果：下游服务慢响应会阻塞网关
```

**正确做法：**
```javascript
// ✅ 设置超时和重试
fastify.register(proxy, {
  upstream: 'http://localhost:3000',
  prefix: '/api/users',
  httpMethods: ['GET', 'POST', 'PUT', 'DELETE'],
  replyOptions: {
    rewriteRequestHeaders: (originalReq, headers) => ({
      ...headers,
      'x-request-id': originalReq.id,  // 追踪 ID
    }),
  },
  http: {
    timeout: 3000,  // 3 秒超时
    retries: 2,     // 失败重试 2 次
  },
});

// 超时降级
fastify.setErrorHandler((error, request, reply) => {
  if (error.code === 'ETIMEDOUT') {
    reply.code(504).send({
      error: 'Gateway Timeout',
      message: 'Upstream service timeout',
    });
  } else {
    reply.send(error);
  }
});
```

### 错误2：未实现熔断器导致雪崩

**错误代码：**
```javascript
// ❌ 错误：下游服务故障时持续重试
fastify.get('/api/users/:id', async (request, reply) => {
  const response = await axios.get(`http://localhost:3000/users/${request.params.id}`);
  return response.data;
});

// 后果：下游服务宕机时，网关也被拖垮
```

**正确做法：使用熔断器**
```javascript
// ✅ 使用 Opossum 熔断器
import CircuitBreaker from 'opossum';

const userServiceBreaker = new CircuitBreaker(async (userId) => {
  const response = await axios.get(`http://localhost:3000/users/${userId}`, {
    timeout: 3000,
  });
  return response.data;
}, {
  timeout: 3000,                // 3 秒超时
  errorThresholdPercentage: 50, // 50% 错误率时熔断
  resetTimeout: 30000,          // 30 秒后尝试恢复
  volumeThreshold: 10,          // 至少 10 次请求后才计算错误率
});

// 熔断时的降级逻辑
userServiceBreaker.fallback(() => ({
  id: 'unknown',
  name: 'Service Unavailable',
  email: 'N/A',
}));

// 监听熔断器事件
userServiceBreaker.on('open', () => {
  fastify.log.warn('❌ Circuit breaker opened for user-service');
});

userServiceBreaker.on('halfOpen', () => {
  fastify.log.info('⚠️ Circuit breaker half-open, testing user-service');
});

userServiceBreaker.on('close', () => {
  fastify.log.info('✅ Circuit breaker closed, user-service recovered');
});

// 使用熔断器
fastify.get('/api/users/:id', async (request, reply) => {
  try {
    const user = await userServiceBreaker.fire(request.params.id);
    return user;
  } catch (error) {
    reply.code(503).send({ error: 'Service unavailable' });
  }
});
```

## 🤔 FAQ

### Q1：API 网关会成为性能瓶颈吗？

**答案：不会，如果正确设计。**

**性能优化建议：**

1. **使用 Fastify**（比 Express 快 2-3 倍）
2. **启用 HTTP/2**
   ```javascript
   import fs from 'fs';
   const fastify = Fastify({
     http2: true,
     https: {
       key: fs.readFileSync('key.pem'),
       cert: fs.readFileSync('cert.pem'),
     },
   });
   ```
3. **响应缓存**（Redis）
4. **压缩响应**
   ```javascript
   import compress from '@fastify/compress';
   fastify.register(compress, { global: true });
   ```
5. **水平扩展**（多实例 + 负载均衡）
   ```bash
   # Nginx 负载均衡
   upstream api_gateway {
     server 192.168.1.10:8080;
     server 192.168.1.11:8080;
     server 192.168.1.12:8080;
   }
   ```

**性能测试：**
```bash
# 单实例 Fastify API 网关
autocannon -c 100 -d 30 http://localhost:8080/api/users/123

# 典型结果：
# Requests/sec: 25000+

```

### Q2：如何实现 API 版本管理？

**方案1：URL 路径版本**
```javascript
// v1 版本
fastify.register(proxy, {
  upstream: 'http://localhost:3000',
  prefix: '/api/v1/users',
});

// v2 版本
fastify.register(proxy, {
  upstream: 'http://localhost:3001',
  prefix: '/api/v2/users',
});

// 客户端调用：
// GET /api/v1/users/123 → 旧版本
// GET /api/v2/users/123 → 新版本
```

**方案2：请求头版本**
```javascript
fastify.addHook('onRequest', async (request, reply) => {
  const version = request.headers['api-version'] || '1';
  
  if (version === '2') {
    request.upstream = 'http://localhost:3001';  // v2
  } else {
    request.upstream = 'http://localhost:3000';  // v1
  }
});

// 客户端调用：
// GET /api/users/123
// Header: api-version: 2
```

### Q3：网关如何处理跨域（CORS）？

```javascript
import cors from '@fastify/cors';

fastify.register(cors, {
  origin: (origin, cb) => {
    // 允许的域名列表
    const allowedOrigins = [
      'https://example.com',
      'https://app.example.com',
    ];
    
    if (!origin || allowedOrigins.includes(origin)) {
      cb(null, true);
    } else {
      cb(new Error('Not allowed by CORS'));
    }
  },
  credentials: true,  // 允许携带 Cookie
  methods: ['GET', 'POST', 'PUT', 'DELETE'],
  allowedHeaders: ['Content-Type', 'Authorization'],
});
```
