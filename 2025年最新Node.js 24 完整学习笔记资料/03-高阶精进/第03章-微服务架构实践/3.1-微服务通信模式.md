# 3.1 微服务通信模式：REST vs gRPC vs GraphQL vs 消息队列

> 选择合适的通信模式，是微服务架构成功的第一步

---

## 📚 核心知识

### 1. 微服务通信的本质

在微服务架构中，**服务间通信**是核心挑战之一。与单体应用的函数调用不同，微服务通信需要考虑：

**关键差异：**
```javascript
// 单体应用：函数调用（同步、本地、确定性）
const user = getUserById(123);
const orders = getOrdersByUserId(user.id);

// 微服务：网络调用（可能失败、延迟、不确定性）
const user = await userService.get(123);       // 可能超时
const orders = await orderService.list(user.id); // 可能失败
```

**核心考虑因素：**

| 维度 | 单体应用 | 微服务 |
|------|---------|--------|
| **调用方式** | 函数调用 | 网络请求 |
| **失败概率** | 极低（内存错误） | 较高（网络、服务宕机） |
| **延迟** | 纳秒级 | 毫秒级（放大1000倍） |
| **数据一致性** | ACID 事务 | 最终一致性 |
| **序列化** | 无需序列化 | JSON/Protobuf/Avro |
| **协议** | 无 | HTTP/gRPC/AMQP |

### 2. 四种通信模式对比

#### 2.1 REST（Representational State Transfer）

**核心特点：**
- 基于 HTTP/1.1 协议
- 使用标准 HTTP 方法（GET、POST、PUT、DELETE）
- 数据格式通常为 JSON
- 无状态（Stateless）

**优势：**
```
✅ 生态成熟，工具链完善
✅ 易于调试（浏览器、Postman）
✅ 跨平台兼容性好
✅ 学习曲线平缓
```

**劣势：**
```
❌ 性能较低（HTTP/1.1 文本协议）
❌ 数据冗余（JSON 冗长）
❌ 无类型安全（需手动校验）
❌ N+1 查询问题
```

#### 2.2 gRPC（Google Remote Procedure Call）

**核心特点：**
- 基于 HTTP/2 协议
- 使用 Protocol Buffers 序列化
- 支持四种通信模式（Unary、Server Streaming、Client Streaming、Bidirectional Streaming）
- 强类型约束

**优势：**
```
✅ 高性能（二进制协议，HTTP/2 多路复用）
✅ 类型安全（.proto 文件定义）
✅ 支持流式传输
✅ 多语言代码生成
```

**劣势：**
```
❌ 生态不如 REST 成熟
❌ 调试困难（二进制数据）
❌ 浏览器支持弱（需 gRPC-Web）
❌ 学习成本较高
```

#### 2.3 GraphQL

**核心特点：**
- 查询语言 + 类型系统
- 客户端按需查询字段
- 单一端点（通常是 POST /graphql）
- 实时订阅（Subscriptions）

**优势：**
```
✅ 按需获取数据（避免 Over-fetching/Under-fetching）
✅ 强类型 Schema
✅ 聚合多个服务数据
✅ 开发体验好（GraphiQL、Apollo Client）
```

**劣势：**
```
❌ 查询复杂度难以控制
❌ 缓存策略复杂
❌ 性能优化困难（N+1 问题）
❌ 学习曲线陡峭
```

#### 2.4 消息队列（Message Queue）

**核心特点：**
- 异步通信
- 发布/订阅模式
- 消息持久化
- 解耦生产者和消费者

**常见实现：**
- RabbitMQ（AMQP 协议）
- Kafka（高吞吐量）
- Redis Pub/Sub（轻量级）

**优势：**
```
✅ 异步解耦，提升系统吞吐量
✅ 削峰填谷，应对流量高峰
✅ 支持广播（一对多）
✅ 消息持久化，保证可靠性
```

**劣势：**
```
❌ 复杂度增加（需维护 MQ 集群）
❌ 消息顺序性保证困难
❌ 调试困难（异步链路）
❌ 数据一致性问题
```

### 场景：订单服务调用库存服务

我们将实现同一业务场景的四种通信方式对比。

#### 1. REST 实现

**库存服务（Inventory Service）**

```javascript
// inventory-service/server.js
import express from 'express';

const app = express();
app.use(express.json());

// 模拟库存数据
const inventory = new Map([
  ['SKU001', { productId: 'SKU001', stock: 100, reserved: 0 }],
  ['SKU002', { productId: 'SKU002', stock: 50, reserved: 0 }],
]);

// 查询库存
app.get('/api/inventory/:productId', (req, res) => {
  const item = inventory.get(req.params.productId);
  if (!item) {
    return res.status(404).json({ error: 'Product not found' });
  }
  res.json(item);
});

// 锁定库存
app.post('/api/inventory/:productId/reserve', (req, res) => {
  const { quantity } = req.body;
  const item = inventory.get(req.params.productId);
  
  if (!item) {
    return res.status(404).json({ error: 'Product not found' });
  }
  
  const available = item.stock - item.reserved;
  if (available < quantity) {
    return res.status(400).json({ 
      error: 'Insufficient stock',
      available 
    });
  }
  
  item.reserved += quantity;
  res.json({ 
    success: true, 
    reserved: item.reserved,
    available: item.stock - item.reserved 
  });
});

app.listen(3001, () => {
  console.log('📦 Inventory Service running on http://localhost:3001');
});
```

**订单服务（Order Service）**

```javascript
// order-service/server.js
import express from 'express';
import axios from 'axios';

const app = express();
app.use(express.json());

const INVENTORY_SERVICE = 'http://localhost:3001';

// 创建订单
app.post('/api/orders', async (req, res) => {
  const { productId, quantity } = req.body;
  
  try {
    // 1. 调用库存服务检查库存
    const inventoryRes = await axios.get(
      `${INVENTORY_SERVICE}/api/inventory/${productId}`
    );
    
    const available = inventoryRes.data.stock - inventoryRes.data.reserved;
    if (available < quantity) {
      return res.status(400).json({ 
        error: 'Insufficient stock',
        available 
      });
    }
    
    // 2. 锁定库存
    await axios.post(
      `${INVENTORY_SERVICE}/api/inventory/${productId}/reserve`,
      { quantity }
    );
    
    // 3. 创建订单（简化）
    const order = {
      id: `ORD${Date.now()}`,
      productId,
      quantity,
      status: 'pending',
      createdAt: new Date(),
    };
    
    res.json(order);
  } catch (error) {
    console.error('Order creation failed:', error.message);
    res.status(500).json({ error: 'Service unavailable' });
  }
});

app.listen(3000, () => {
  console.log('🛒 Order Service running on http://localhost:3000');
});
```

#### 2. gRPC 实现

**定义 Protocol Buffers**

```protobuf
// protos/inventory.proto
syntax = "proto3";

package inventory;

service InventoryService {
  rpc GetStock (GetStockRequest) returns (StockInfo);
  rpc ReserveStock (ReserveRequest) returns (ReserveResponse);
}

message GetStockRequest {
  string product_id = 1;
}

message StockInfo {
  string product_id = 1;
  int32 stock = 2;
  int32 reserved = 3;
}

message ReserveRequest {
  string product_id = 1;
  int32 quantity = 2;
}

message ReserveResponse {
  bool success = 1;
  int32 available = 2;
  string error = 3;
}
```

**库存服务（gRPC Server）**

```javascript
// inventory-grpc-service/server.js
import grpc from '@grpc/grpc-js';
import protoLoader from '@grpc/proto-loader';
import path from 'path';

const PROTO_PATH = path.join(process.cwd(), 'protos/inventory.proto');

// 加载 proto 文件
const packageDefinition = protoLoader.loadSync(PROTO_PATH, {
  keepCase: true,
  longs: String,
  enums: String,
  defaults: true,
  oneofs: true,
});
const inventoryProto = grpc.loadPackageDefinition(packageDefinition).inventory;

// 模拟库存数据
const inventory = new Map([
  ['SKU001', { product_id: 'SKU001', stock: 100, reserved: 0 }],
  ['SKU002', { product_id: 'SKU002', stock: 50, reserved: 0 }],
]);

// 实现服务方法
function getStock(call, callback) {
  const productId = call.request.product_id;
  const item = inventory.get(productId);
  
  if (!item) {
    return callback({
      code: grpc.status.NOT_FOUND,
      message: 'Product not found',
    });
  }
  
  callback(null, item);
}

function reserveStock(call, callback) {
  const { product_id, quantity } = call.request;
  const item = inventory.get(product_id);
  
  if (!item) {
    return callback({
      code: grpc.status.NOT_FOUND,
      message: 'Product not found',
    });
  }
  
  const available = item.stock - item.reserved;
  if (available < quantity) {
    return callback(null, {
      success: false,
      available,
      error: 'Insufficient stock',
    });
  }
  
  item.reserved += quantity;
  callback(null, {
    success: true,
    available: item.stock - item.reserved,
    error: '',
  });
}

// 启动服务器
const server = new grpc.Server();
server.addService(inventoryProto.InventoryService.service, {
  GetStock: getStock,
  ReserveStock: reserveStock,
});

server.bindAsync(
  '0.0.0.0:50051',
  grpc.ServerCredentials.createInsecure(),
  (err, port) => {
    if (err) {
      console.error('Failed to bind server:', err);
      return;
    }
    console.log(`📦 Inventory gRPC Service running on port ${port}`);
    server.start();
  }
);
```

**订单服务（gRPC Client）**

```javascript
// order-grpc-service/server.js
import express from 'express';
import grpc from '@grpc/grpc-js';
import protoLoader from '@grpc/proto-loader';
import path from 'path';

const app = express();
app.use(express.json());

const PROTO_PATH = path.join(process.cwd(), 'protos/inventory.proto');

// 加载 proto 文件
const packageDefinition = protoLoader.loadSync(PROTO_PATH, {
  keepCase: true,
  longs: String,
  enums: String,
  defaults: true,
  oneofs: true,
});
const inventoryProto = grpc.loadPackageDefinition(packageDefinition).inventory;

// 创建 gRPC 客户端
const client = new inventoryProto.InventoryService(
  'localhost:50051',
  grpc.credentials.createInsecure()
);

// 封装 gRPC 调用为 Promise
function getStock(productId) {
  return new Promise((resolve, reject) => {
    client.GetStock({ product_id: productId }, (err, response) => {
      if (err) reject(err);
      else resolve(response);
    });
  });
}

function reserveStock(productId, quantity) {
  return new Promise((resolve, reject) => {
    client.ReserveStock(
      { product_id: productId, quantity },
      (err, response) => {
        if (err) reject(err);
        else resolve(response);
      }
    );
  });
}

// 创建订单接口
app.post('/api/orders', async (req, res) => {
  const { productId, quantity } = req.body;
  
  try {
    // 1. 调用 gRPC 服务检查库存
    const stockInfo = await getStock(productId);
    const available = stockInfo.stock - stockInfo.reserved;
    
    if (available < quantity) {
      return res.status(400).json({ 
        error: 'Insufficient stock',
        available 
      });
    }
    
    // 2. 锁定库存
    const reserveResult = await reserveStock(productId, quantity);
    
    if (!reserveResult.success) {
      return res.status(400).json({ error: reserveResult.error });
    }
    
    // 3. 创建订单
    const order = {
      id: `ORD${Date.now()}`,
      productId,
      quantity,
      status: 'pending',
      createdAt: new Date(),
    };
    
    res.json(order);
  } catch (error) {
    console.error('Order creation failed:', error);
    res.status(500).json({ error: 'Service unavailable' });
  }
});

app.listen(3002, () => {
  console.log('🛒 Order Service (gRPC) running on http://localhost:3002');
});
```

**库存服务（GraphQL Server）**

```javascript
// inventory-graphql-service/server.js
import { ApolloServer } from 'apollo-server';

// 模拟库存数据
const inventory = new Map([
  ['SKU001', { productId: 'SKU001', stock: 100, reserved: 0 }],
  ['SKU002', { productId: 'SKU002', stock: 50, reserved: 0 }],
]);

// 定义 Schema
const typeDefs = `
  type Stock {
    productId: String!
    stock: Int!
    reserved: Int!
    available: Int!
  }

  type ReserveResult {
    success: Boolean!
    available: Int
    error: String
  }

  type Query {
    stock(productId: String!): Stock
    stocks: [Stock!]!
  }

  type Mutation {
    reserveStock(productId: String!, quantity: Int!): ReserveResult!
  }
`;

// 定义 Resolvers
const resolvers = {
  Query: {
    stock: (_, { productId }) => {
      const item = inventory.get(productId);
      if (!item) {
        throw new Error('Product not found');
      }
      return {
        ...item,
        available: item.stock - item.reserved,
      };
    },
    stocks: () => {
      return Array.from(inventory.values()).map(item => ({
        ...item,
        available: item.stock - item.reserved,
      }));
    },
  },
  Mutation: {
    reserveStock: (_, { productId, quantity }) => {
      const item = inventory.get(productId);
      
      if (!item) {
        return {
          success: false,
          available: 0,
          error: 'Product not found',
        };
      }
      
      const available = item.stock - item.reserved;
      if (available < quantity) {
        return {
          success: false,
          available,
          error: 'Insufficient stock',
        };
      }
      
      item.reserved += quantity;
      return {
        success: true,
        available: item.stock - item.reserved,
        error: null,
      };
    },
  },
};

// 启动服务器
const server = new ApolloServer({ typeDefs, resolvers });

server.listen({ port: 4000 }).then(({ url }) => {
  console.log(`📦 Inventory GraphQL Service running at ${url}`);
});
```

```javascript
// order-mq-service/server.js
import express from 'express';
import amqp from 'amqplib';

const app = express();
app.use(express.json());

let channel;

// 连接 RabbitMQ
async function connectRabbitMQ() {
  const connection = await amqp.connect('amqp://localhost');
  channel = await connection.createChannel();
  
  // 声明队列
  await channel.assertQueue('order.created', { durable: true });
  
  console.log('✅ Connected to RabbitMQ');
}

connectRabbitMQ();

// 创建订单（异步）
app.post('/api/orders', async (req, res) => {
  const { productId, quantity } = req.body;
  
  // 立即返回订单（状态为 pending）
  const order = {
    id: `ORD${Date.now()}`,
    productId,
    quantity,
    status: 'pending',
    createdAt: new Date(),
  };
  
  // 发送消息到队列
  channel.sendToQueue(
    'order.created',
    Buffer.from(JSON.stringify(order)),
    { persistent: true }  // 消息持久化
  );
  
  console.log(`📤 Order ${order.id} sent to queue`);
  
  res.status(202).json({  // 202 Accepted
    ...order,
    message: 'Order is being processed',
  });
});

app.listen(3003, () => {
  console.log('🛒 Order Service (MQ) running on http://localhost:3003');
});
```

**库存服务（消费者）**

```javascript
// inventory-mq-service/consumer.js
import amqp from 'amqplib';
import axios from 'axios';

const inventory = new Map([
  ['SKU001', { productId: 'SKU001', stock: 100, reserved: 0 }],
  ['SKU002', { productId: 'SKU002', stock: 50, reserved: 0 }],
]);

async function startConsumer() {
  const connection = await amqp.connect('amqp://localhost');
  const channel = await connection.createChannel();
  
  await channel.assertQueue('order.created', { durable: true });
  
  // 设置预取数量（一次只处理1条消息）
  channel.prefetch(1);
  
  console.log('📥 Waiting for orders...');
  
  channel.consume('order.created', async (msg) => {
    if (!msg) return;
    
    const order = JSON.parse(msg.content.toString());
    console.log(`📦 Processing order ${order.id}`);
    
    const item = inventory.get(order.productId);
    
    if (!item) {
      console.error(`❌ Product ${order.productId} not found`);
      channel.ack(msg);  // 确认消息（避免重复消费）
      return;
    }
    
    const available = item.stock - item.reserved;
    
    if (available < order.quantity) {
      console.error(`❌ Insufficient stock for order ${order.id}`);
      // 发送到死信队列（实际应用中）
      channel.nack(msg, false, false);
      return;
    }
    
    // 锁定库存
    item.reserved += order.quantity;
    console.log(`✅ Reserved ${order.quantity} units of ${order.productId}`);
    
    // 确认消息
    channel.ack(msg);
    
    // 通知订单服务（可选）
    // await notifyOrderService(order.id, 'confirmed');
  }, { noAck: false });
}

startConsumer();
```

## 🎯 应用场景对比

### 场景1：电商订单系统

**需求分析：**
- 用户下单（同步反馈）
- 库存扣减（强一致性）
- 支付处理（异步）
- 物流通知（异步）

**推荐方案：**
```
用户 → 订单服务 (REST)
         ↓
      库存服务 (gRPC) ← 高性能同步调用
         ↓
      消息队列 (RabbitMQ)
         ↓
      ├─ 支付服务
      ├─ 物流服务
      └─ 通知服务
```

**代码示例：**
```javascript
// 订单服务：混合使用 REST + gRPC + MQ
app.post('/api/orders', async (req, res) => {
  const { userId, items } = req.body;
  
  try {
    // 1. 同步调用库存服务（gRPC，快速响应）
    const stockCheck = await inventoryGrpcClient.checkAndReserve(items);
    
    if (!stockCheck.success) {
      return res.status(400).json({ error: 'Insufficient stock' });
    }
    
    // 2. 创建订单
    const order = await createOrder({ userId, items });
    
    // 3. 发送异步消息（支付、物流）
    await publishMessage('order.created', order);
    
    // 4. 立即返回给用户
    res.json({
      orderId: order.id,
      status: 'pending_payment',
      paymentUrl: `/payment/${order.id}`,
    });
  } catch (error) {
    console.error('Order creation failed:', error);
    res.status(500).json({ error: 'Service unavailable' });
  }
```

## ❌ 高频错误

### 错误1：REST API 的 N+1 查询问题

**错误代码：**
```javascript
// ❌ 错误：为每个订单单独查询用户信息
app.get('/api/orders', async (req, res) => {
  const orders = await Order.find();
  
  // N+1 查询！每个订单都调用一次用户服务
  const ordersWithUser = await Promise.all(
    orders.map(async (order) => {
      const user = await axios.get(
        `http://user-service/api/users/${order.userId}`
      );
      return { ...order, user: user.data };
    })
  );
  
  res.json(ordersWithUser);
});

// 100 个订单 = 1 次订单查询 + 100 次用户查询 = 101 次请求！
```

**原因分析：**
- 循环中发起网络请求
- 微服务环境下，每次请求延迟放大（5ms → 50ms）

**正确做法：**
```javascript
// ✅ 方案1：批量查询用户
app.get('/api/orders', async (req, res) => {
  const orders = await Order.find();
  
  // 提取所有用户 ID
  const userIds = [...new Set(orders.map(o => o.userId))];
  
  // 批量查询用户（一次请求）
  const users = await axios.post(
    'http://user-service/api/users/batch',
    { ids: userIds }
  );
  
  const userMap = new Map(users.data.map(u => [u.id, u]));
  
  const ordersWithUser = orders.map(order => ({
    ...order,
    user: userMap.get(order.userId),
  }));
  
  res.json(ordersWithUser);
});

// 100 个订单 = 1 次订单查询 + 1 次用户批量查询 = 2 次请求！
```

```javascript
// ✅ 方案2：使用 GraphQL（自动解决 N+1）
const resolvers = {
  Order: {
    user: async (order, _, { dataloaders }) => {
      // DataLoader 自动批量查询和缓存
      return dataloaders.userLoader.load(order.userId);
    },
  },
};

const userLoader = new DataLoader(async (userIds) => {
  const users = await UserService.findByIds(userIds);
  return userIds.map(id => users.find(u => u.id === id));
});
```

### 错误2：gRPC 超时未设置导致服务雪崩

**错误代码：**
```javascript
// ❌ 错误：未设置超时时间
const client = new inventoryProto.InventoryService(
  'localhost:50051',
  grpc.credentials.createInsecure()
);

// 如果服务端卡住，客户端会一直等待！
client.GetStock({ product_id: 'SKU001' }, (err, response) => {
  // 可能永远不会执行到这里
});
```

**后果：**
```
客户端线程耗尽 → 无法处理新请求 → 服务不可用
```

**正确做法：**
```javascript
// ✅ 设置超时时间（deadline）
import grpc from '@grpc/grpc-js';

const deadline = new Date();
deadline.setSeconds(deadline.getSeconds() + 5);  // 5秒超时

client.GetStock(
  { product_id: 'SKU001' },
  { deadline: deadline.getTime() },
  (err, response) => {
    if (err && err.code === grpc.status.DEADLINE_EXCEEDED) {
      console.error('Request timeout');
      // 降级处理：返回缓存数据或默认值
      return fallbackResponse;
    }
    // 正常处理
  }
);
```

```javascript
// ✅ 使用拦截器统一设置超时
function timeoutInterceptor(options, nextCall) {
  const deadline = new Date();
  deadline.setSeconds(deadline.getSeconds() + 3);
  
  options.deadline = deadline.getTime();
  
  return new grpc.InterceptingCall(nextCall(options));
}

const client = new inventoryProto.InventoryService(

  grpc.credentials.createInsecure(),
```

### 错误3：消息队列消息丢失

**错误代码：**
```javascript
// ❌ 错误：未确认消息就处理完成
channel.consume('order.created', async (msg) => {
  const order = JSON.parse(msg.content.toString());
  
  // 立即确认（消息会被删除）
  channel.ack(msg);
  
  // 如果后续处理失败，消息已丢失！
  await processOrder(order);  // 可能抛出异常
});
```

**后果：**
- 处理失败时消息已被删除
- 数据不一致

**正确做法：**
```javascript
// ✅ 处理成功后再确认
channel.consume('order.created', async (msg) => {
  const order = JSON.parse(msg.content.toString());
  
  try {
    // 先处理
    await processOrder(order);
    
    // 成功后确认
    channel.ack(msg);
    
    console.log(`✅ Order ${order.id} processed`);
  } catch (error) {
    console.error(`❌ Failed to process order ${order.id}:`, error);
    
    // 拒绝消息并重新入队（最多重试3次）
    const retryCount = msg.properties.headers['x-retry-count'] || 0;
    
    if (retryCount < 3) {
      channel.nack(msg, false, true);  // requeue = true
    } else {
      // 重试3次后发送到死信队列
      channel.nack(msg, false, false);  // requeue = false
      console.error(`⚠️ Order ${order.id} sent to DLQ`);
    }
  }
});
```

## 🤔 FAQ

### Q1：什么时候应该用 REST，什么时候用 gRPC？

**使用 REST：**
- 外部 API（第三方集成）
- 前端直接调用
- 需要浏览器兼容性
- 团队对 REST 熟悉

**使用 gRPC：**
- 内部服务间通信
- 高性能要求（如实时交易）
- 需要流式传输（如视频流）
- 多语言异构系统

**混合方案（推荐）：**
```
外部客户端 ──REST──> API 网关 ──gRPC──> 后端服务
```

### Q2：GraphQL 真的能替代 REST 吗？

**不能完全替代，两者各有优势：**

**GraphQL 适用场景：**
- 移动端应用（流量敏感，按需查询）
- 数据聚合场景（Dashboard、报表）
- 前端主导的项目（BFF 模式）

**REST 更适合：**
- 简单 CRUD 操作
- 文件上传/下载
- 缓存要求高（HTTP 缓存）

**实际项目中常见混合使用：**
```javascript
// REST：处理文件上传
POST /api/upload

// GraphQL：复杂查询
POST /graphql
{
  query {
    user(id: "123") {
      orders { products { reviews } }
    }
  }
}
```

### Q3：消息队列何时使用，如何选择？

**使用场景：**
- ✅ 异步处理（发送邮件、生成报表）
- ✅ 削峰填谷（秒杀、大促）
- ✅ 解耦系统（订单系统 → 多个下游系统）
- ✅ 最终一致性（分布式事务）

**选择标准：**

| MQ | 适用场景 | 优势 | 劣势 |
|----|---------|------|------|
| **RabbitMQ** | 企业应用、复杂路由 | 功能丰富、可靠性高 | 吞吐量较低 |
| **Kafka** | 日志收集、大数据 | 高吞吐量、持久化 | 延迟较高 |
| **Redis Pub/Sub** | 实时通知、简单场景 | 超低延迟、简单 | 不持久化 |

## 📝 本节总结

### 核心要点

1. **四种通信模式各有优势**：
   - REST：生态成熟，易于调试
   - gRPC：高性能，类型安全
   - GraphQL：按需查询，聚合数据
   - 消息队列：异步解耦，削峰填谷

2. **选择标准**：
   - 同步 + 高性能 → gRPC
   - 外部 API → REST
   - 复杂数据聚合 → GraphQL
   - 异步处理 → 消息队列

3. **实际项目中混合使用**：
   ```
   API 网关（REST）──> 后端服务（gRPC）──> 消息队列（MQ）
   ```

4. **注意事项**：
   - REST：避免 N+1 查询
   - gRPC：设置超时和重试
   - GraphQL：控制查询复杂度
   - MQ：保证消息可靠性
