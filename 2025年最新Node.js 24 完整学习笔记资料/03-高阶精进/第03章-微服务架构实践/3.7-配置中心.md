## 📚 核心知识

### 1. 为什么需要配置中心？

**传统配置管理的痛点：**

```javascript
// ❌ 硬编码配置
const DB_HOST = '192.168.1.100';
const DB_PORT = 5432;
const MAX_CONNECTIONS = 20;

// 问题：
❌ 修改配置需要重新部署
❌ 多环境配置管理混乱（dev/test/prod）
❌ 敏感信息泄露风险（数据库密码）
❌ 无法实时调整参数（如限流阈值）
❌ 配置变更无审计日志
```

**配置中心架构：**

```
┌──────────────────────────────────────────────────────┐
```

### 2. Apollo vs Nacos 对比

| 特性 | Apollo | Nacos |

```javascript
// config/apollo.js
import ApolloClient from 'ctrip-apollo';

const apollo = new ApolloClient({
  configServerUrl: 'http://localhost:8080',  // Config Service 地址
  appId: 'order-service',
  clusterName: 'default',
  namespaceName: ['application'],  // 命名空间
  releaseKey: '',  // 可选
  ip: '',  // 可选，客户端 IP
  dataCenter: '',  // 可选，数据中心
  env: 'DEV',  // 环境：DEV/FAT/UAT/PRO
});

// 启动客户端（拉取配置并监听变化）
await apollo.fetchConfig();

// 导出配置对象
export const config = {
  // 数据库配置
  db: {
    host: apollo.get('db.host'),
    port: parseInt(apollo.get('db.port')),
    database: apollo.get('db.name'),
    user: apollo.get('db.user'),
    password: apollo.get('db.password'),
    pool: {
      min: parseInt(apollo.get('db.pool.min')),
      max: parseInt(apollo.get('db.pool.max')),
    },
  },
  
  // API 配置
  api: {
    rateLimit: parseInt(apollo.get('api.rate_limit')),
    timeout: parseInt(apollo.get('api.timeout')),
  },
  
  // 功能开关
  features: {
    newCheckout: apollo.get('feature.newCheckout') === 'true',
  },
};

// 监听配置变化
apollo.onChange((data) => {
  console.log('🔄 Config updated:', data);
  
  // 动态更新配置
  config.api.rateLimit = parseInt(apollo.get('api.rate_limit'));
  config.api.timeout = parseInt(apollo.get('api.timeout'));
  config.features.newCheckout = apollo.get('feature.newCheckout') === 'true';
  
  // 触发配置重新加载（如重新连接数据库）
  reloadConfig();
});

export default apollo;
```

**使用配置：**

```javascript

// 功能开关

app.listen(3000, () => {
```

```javascript
// config/nacos.js
import NacosNamingClient from 'nacos';

const nacosConfig = new NacosNamingClient({
  serverAddr: '127.0.0.1:8848',  // Nacos 服务器地址
  namespace: 'dev-env',  // 命名空间 ID
});

// 配置对象
let config = {};

// 获取配置
async function fetchConfig() {
  try {
    const content = await nacosConfig.getConfig({
      dataId: 'order-service.yaml',
      group: 'DEFAULT_GROUP',
    });
    
    // 解析 YAML
    const yaml = await import('yaml');
    config = yaml.parse(content);
    
    console.log('✅ Config loaded from Nacos:', config);
  } catch (error) {
    console.error('❌ Failed to load config:', error);
    process.exit(1);
  }
}

// 监听配置变化
nacosConfig.subscribe({
  dataId: 'order-service.yaml',
  group: 'DEFAULT_GROUP',
}, (content) => {
  console.log('🔄 Config updated from Nacos');
  
  const yaml = require('yaml');
  config = yaml.parse(content);
  
  console.log('New config:', config);
  
  // 触发配置重新加载
  reloadConfig();
});

// 初始化配置
await fetchConfig();

export { config };
```

```javascript
// config/apollo.js
import ApolloClient from 'ctrip-apollo';
import jasypt from 'jasypt';

const apollo = new ApolloClient({
  // ...
});

await apollo.fetchConfig();

// 创建解密器
const encryptor = jasypt.encryptor({
  password: process.env.JASYPT_PASSWORD,  // 从环境变量读取主密钥
  algorithm: 'PBEWITHHMACSHA512ANDAES_256',
});

// 解密配置
function decryptConfig(value) {
  const match = value.match(/^ENC\((.*)\)$/);
  if (match) {
    return encryptor.decrypt(match[1]);
  }
  return value;
}

export const config = {
  db: {
    host: apollo.get('db.host'),
    user: apollo.get('db.user'),
    password: decryptConfig(apollo.get('db.password')),  // 自动解密
  },
};
```

```javascript
import { config } from './config/apollo.js';
import rateLimit from 'express-rate-limit';

// 动态限流器
let limiter = rateLimit({
  windowMs: 60000,
  max: config.api.rateLimit,
});

// 监听配置变化，重新创建限流器
apollo.onChange((data) => {
  if ('api.rate_limit' in data) {
    console.log(`🔄 Rate limit updated: ${data['api.rate_limit']}`);
    limiter = rateLimit({
      windowMs: 60000,
      max: parseInt(data['api.rate_limit']),

  }
});

// 使用限流器
app.use((req, res, next) => {
  limiter(req, res, next);
});
```

### 错误2：敏感信息泄露

**错误代码：**
```javascript
// ❌ 错误：日志打印完整配置
console.log('Config:', config);
// 输出：{ db: { password: 'secret123' } }  ← 密码泄露
```

**正确做法：**
```javascript
// ✅ 过滤敏感字段
function maskSensitiveFields(obj) {
  const masked = { ...obj };
  const sensitiveKeys = ['password', 'secret', 'token', 'apiKey'];
  
  for (const key of Object.keys(masked)) {
    if (sensitiveKeys.some(k => key.toLowerCase().includes(k))) {
      masked[key] = '****';
    } else if (typeof masked[key] === 'object') {
      masked[key] = maskSensitiveFields(masked[key]);
    }
  }
  
  return masked;
}

console.log('Config:', maskSensitiveFields(config));
// 输出：{ db: { password: '****' } }  ← 已脱敏
```

## 🤔 FAQ

### Q1：配置中心宕机会影响应用吗？

**答案：不会，应用有本地缓存。**

**Apollo 缓存机制：**
```
1. 应用启动时从配置中心拉取配置
2. 配置缓存到本地文件（config-cache/order-service.yaml）
3. 配置中心宕机时，应用读取本地缓存
4. 配置中心恢复后，自动同步最新配置
```

**Nacos 缓存位置：**
```
~/.nacos/config/
  └─ dev-env/
      └─ DEFAULT_GROUP/
          └─ order-service.yaml
```

### Q2：如何实现配置变更审计？

**Apollo 内置审计日志：**

```
Web UI → 配置管理 → 历史版本

可查看：
- 谁修改了配置
- 修改了哪些字段
- 修改前后的值对比
- 修改时间
- 发布时间
```

**Nacos 审计：**

```bash
# 查询配置历史
curl -X GET "http://localhost:8848/nacos/v1/cs/history?dataId=order-service.yaml&group=DEFAULT_GROUP&tenant=dev-env"

# 返回：
# [
#   {
#     "id": 123,
#     "dataId": "order-service.yaml",
#     "content": "...",
#     "srcUser": "admin",
#     "opType": "U",  # U=Update, D=Delete
#     "createdTime": "2025-01-15 08:30:00"
#   }
# ]
```

### Q3：如何防止误操作？

**最佳实践：**

1. **权限管理**：
   - 开发人员：只读权限
   - 运维人员：发布权限
   - 管理员：删除权限

2. **发布审批流程**：
   - 提交变更 → 主管审批 → 发布

3. **配置锁定**：
   - 生产环境配置需解锁才能修改

4. **紧急回滚**：
   - Apollo：点击历史版本 → 回滚
   - Nacos：配置列表 → 历史版本 → 回滚
