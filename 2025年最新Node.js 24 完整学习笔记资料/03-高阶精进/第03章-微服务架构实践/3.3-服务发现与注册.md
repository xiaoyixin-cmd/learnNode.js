# 3.3 服务发现与注册：Consul / etcd 集成

> 动态服务治理的核心：让微服务自动感知彼此

---

## 📚 核心知识

### 1. 为什么需要服务注册与发现？

**传统方式的痛点：**

```javascript
// ❌ 硬编码服务地址
const INVENTORY_SERVICE = 'http://192.168.1.100:3001';
const USER_SERVICE = 'http://192.168.1.101:3002';

// 问题：
// 1. 服务IP变更需要修改代码并重新部署
// 2. 无法动态扩容（新增节点需要手动配置）
// 3. 服务宕机无法自动摘除
// 4. 无法实现负载均衡
```

**服务注册与发现架构：**

```
┌────────────────────────────────────────────────────────────┐
│                    服务注册中心                             │
│                   (Consul / etcd)                          │
│  ┌──────────────────────────────────────────────────────┐ │
│  │  服务列表（Service Registry）                         │ │
│  │  ┌─────────────────────────────────────────────────┐ │ │
│  │  │ user-service:                                   │ │ │
│  │  │   - 192.168.1.10:3000 (healthy)                 │ │ │
│  │  │   - 192.168.1.11:3000 (healthy)                 │ │ │
│  │  │ order-service:                                  │ │ │
│  │  │   - 192.168.1.20:4000 (healthy)                 │ │ │
│  │  │   - 192.168.1.21:4000 (unhealthy) ← 自动摘除   │ │ │
│  │  └─────────────────────────────────────────────────┘ │ │
│  └──────────────────────────────────────────────────────┘ │
│              ↑ 注册/心跳                ↓ 查询             │
└──────────────┼─────────────────────────┼──────────────────┘
               │                         │
    ┌──────────┴──────────┐   ┌─────────┴─────────┐
    │  User Service 实例   │   │  Order Service   │
    │   自动注册           │   │   服务发现       │
    │   健康检查           │   │   负载均衡       │
    └─────────────────────┘   └──────────────────┘
```

**核心能力：**

1. **服务注册**：服务启动时自动注册到注册中心
2. **健康检查**：定期检测服务健康状态，自动摘除异常节点
3. **服务发现**：客户端动态查询可用服务节点
4. **负载均衡**：在多个服务实例间分发请求
5. **配置管理**：统一管理服务配置（KV 存储）

### 2. Consul vs etcd 对比

| 特性 | Consul | etcd |
|------|--------|------|
| **开发语言** | Go | Go |
| **数据模型** | KV + Service Catalog | 纯 KV（层级结构） |
| **健康检查** | ✅ 内置（HTTP/TCP/Script） | ❌ 需自己实现 |
| **服务发现** | ✅ 原生支持（DNS/HTTP API） | ⚠️ 需基于 KV 自己实现 |
| **UI 界面** | ✅ 内置 Web UI | ❌ 需第三方工具 |
| **Watch 机制** | ✅ Blocking Query | ✅ Watch API（gRPC Stream） |
| **CAP 理论** | CP（强一致性） | CP（强一致性） |
| **适用场景** | 微服务治理、服务网格 | Kubernetes、分布式锁 |
| **生态** | Consul Connect（Service Mesh） | Kubernetes 依赖 |

**选择建议：**

- **首选 Consul**：如果主要用于微服务注册发现（功能更完善）
- **选择 etcd**：如果已使用 Kubernetes（内置 etcd）或需要分布式锁

```javascript
// src/services/user-service.js
import express from 'express';
import Consul from 'consul';
import os from 'os';

const app = express();
app.use(express.json());

const PORT = process.env.PORT || 3000;
const SERVICE_NAME = 'user-service';
const SERVICE_ID = `${SERVICE_NAME}-${os.hostname()}-${PORT}`;

// 创建 Consul 客户端
const consul = new Consul({
  host: '127.0.0.1',
  port: 8500,
  promisify: true,  // 使用 Promise API
});

// 模拟用户数据
const users = new Map([
  ['1', { id: '1', name: 'Alice', email: 'alice@example.com' }],
  ['2', { id: '2', name: 'Bob', email: 'bob@example.com' }],
]);

// API 路由
app.get('/users/:id', (req, res) => {
  const user = users.get(req.params.id);
  if (!user) {
    return res.status(404).json({ error: 'User not found' });
  }
  res.json(user);
});

app.get('/users', (req, res) => {
  res.json(Array.from(users.values()));
});

// 健康检查端点（必需）
app.get('/health', (req, res) => {
  res.json({ status: 'UP', timestamp: new Date().toISOString() });
});

// 注册服务到 Consul
async function registerService() {
  try {
    await consul.agent.service.register({
      id: SERVICE_ID,
      name: SERVICE_NAME,
      address: getLocalIP(),  // 服务实际 IP
      port: PORT,
      tags: ['v1', 'http'],
      meta: {
        version: '1.0.0',
        protocol: 'http',
      },
      check: {
        http: `http://${getLocalIP()}:${PORT}/health`,  // 健康检查 URL
        interval: '10s',  // 每 10 秒检查一次
        timeout: '5s',    // 5 秒超时
        deregistercriticalserviceafter: '1m',  // 失败 1 分钟后自动注销
      },
    });
    
    console.log(`✅ Service registered: ${SERVICE_ID}`);
    console.log(`📍 Address: http://${getLocalIP()}:${PORT}`);
  } catch (error) {
    console.error('❌ Failed to register service:', error);
    process.exit(1);
  }
}

// 注销服务
async function deregisterService() {
  try {
    await consul.agent.service.deregister(SERVICE_ID);
    console.log(`✅ Service deregistered: ${SERVICE_ID}`);
  } catch (error) {
    console.error('❌ Failed to deregister service:', error);
  }
}

// 获取本机 IP
function getLocalIP() {
  const interfaces = os.networkInterfaces();
  for (const name of Object.keys(interfaces)) {
    for (const iface of interfaces[name]) {
      if (iface.family === 'IPv4' && !iface.internal) {
        return iface.address;
      }
    }
  }
  return '127.0.0.1';
}

// 启动服务器
const server = app.listen(PORT, async () => {
  console.log(`🚀 User Service running on port ${PORT}`);
  await registerService();
});

// 优雅退出
process.on('SIGINT', async () => {
  console.log('\n⏹️ Shutting down...');
  await deregisterService();
  server.close(() => {
    console.log('✅ Server closed');
    process.exit(0);
  });
});

process.on('SIGTERM', async () => {
  await deregisterService();
  server.close(() => process.exit(0));
});
```

#### 步骤5：实现服务发现（订单服务）

```javascript
// src/services/order-service.js
import express from 'express';
import Consul from 'consul';
import axios from 'axios';

const app = express();
app.use(express.json());

const PORT = 4000;

// 创建 Consul 客户端
const consul = new Consul({
  host: '127.0.0.1',
  port: 8500,
  promisify: true,
});

// 服务发现：获取健康的服务实例
async function discoverService(serviceName) {
  try {
    const result = await consul.health.service({
      service: serviceName,
      passing: true,  // 只返回健康的实例
    });
    
    if (!result || result.length === 0) {
      throw new Error(`No healthy instances found for ${serviceName}`);
    }
    
    // 返回所有健康实例
    return result.map(item => ({
      id: item.Service.ID,
      address: item.Service.Address,
      port: item.Service.Port,
      tags: item.Service.Tags,
    }));
  } catch (error) {
    console.error(`❌ Service discovery failed for ${serviceName}:`, error.message);
    throw error;
  }
}

// 客户端负载均衡（轮询）
class LoadBalancer {
  constructor() {
    this.instances = [];
    this.currentIndex = 0;
  }
  
  setInstances(instances) {
    this.instances = instances;
    this.currentIndex = 0;
  }
  
  getNextInstance() {
    if (this.instances.length === 0) {
      throw new Error('No available instances');
    }
    
    const instance = this.instances[this.currentIndex];
    this.currentIndex = (this.currentIndex + 1) % this.instances.length;
    return instance;
  }
}

const userServiceLB = new LoadBalancer();

// 定期更新服务列表（每 30 秒）
async function updateServiceInstances() {
  try {
    const instances = await discoverService('user-service');
    userServiceLB.setInstances(instances);
    console.log(`✅ Updated user-service instances: ${instances.length} available`);
  } catch (error) {
    console.error('❌ Failed to update service instances:', error.message);
  }
}

// 初始化并启动定时更新
updateServiceInstances();
setInterval(updateServiceInstances, 30000);

// 创建订单接口
app.post('/orders', async (req, res) => {
  const { userId, productId, quantity } = req.body;
  
  try {
    // 1. 通过服务发现调用用户服务
    const instance = userServiceLB.getNextInstance();
    const userServiceUrl = `http://${instance.address}:${instance.port}`;
    
    console.log(`📞 Calling user-service: ${userServiceUrl}`);
    
    const userResponse = await axios.get(`${userServiceUrl}/users/${userId}`, {
      timeout: 3000,
    });
    
    const user = userResponse.data;
    
    // 2. 创建订单
    const order = {
      id: `ORD${Date.now()}`,
      userId,
      user: user.name,
      productId,
      quantity,
      status: 'pending',
      createdAt: new Date(),
    };
    
    res.json(order);
  } catch (error) {
    console.error('❌ Order creation failed:', error.message);
    
    if (error.response) {
      res.status(error.response.status).json({ error: error.response.data });
    } else {
      res.status(503).json({ error: 'Service unavailable' });
    }
  }
});

app.listen(PORT, () => {
  console.log(`🚀 Order Service running on port ${PORT}`);
});
```

### 实战2：Consul KV 存储配置管理

```javascript
// src/config/consul-config.js
import Consul from 'consul';

const consul = new Consul({
  host: '127.0.0.1',
  port: 8500,
  promisify: true,
});

// 设置配置
async function setConfig(key, value) {
  await consul.kv.set(key, JSON.stringify(value));
  console.log(`✅ Config set: ${key}`);
}

// 获取配置
async function getConfig(key) {
  const result = await consul.kv.get(key);
  if (!result) {
    return null;
  }
  return JSON.parse(result.Value);
}

// 监听配置变化
async function watchConfig(key, callback) {
  let index = 0;
  
  while (true) {
    try {
      const result = await consul.kv.get({
        key,
        index,  // 阻塞查询（Long Polling）
        wait: '5m',  // 最多等待 5 分钟
      });
      
      if (result && result.ModifyIndex > index) {
        index = result.ModifyIndex;
        const value = JSON.parse(result.Value);
        callback(value);
      }
    } catch (error) {
      console.error('❌ Watch error:', error);
      await new Promise(r => setTimeout(r, 5000));  // 等待 5 秒重试
    }
  }
}

// 使用示例
async function main() {
  // 1. 设置数据库配置
  await setConfig('app/database', {
    host: 'localhost',
    port: 5432,
    database: 'myapp',
    pool: {
      min: 2,
      max: 10,
    },
  });
  
  // 2. 获取配置
  const dbConfig = await getConfig('app/database');
  console.log('Database config:', dbConfig);
  
  // 3. 监听配置变化
  watchConfig('app/database', (newConfig) => {
    console.log('🔄 Config updated:', newConfig);
    // 重新连接数据库等操作
  });
}

main();
```

```javascript
import { Etcd3 } from 'etcd3';
import os from 'os';

const app = express();
app.use(express.json());

const PORT = process.env.PORT || 3000;
const SERVICE_NAME = 'user-service';
const SERVICE_KEY = `/services/${SERVICE_NAME}/${os.hostname()}-${PORT}`;

// 创建 etcd 客户端
const etcd = new Etcd3({
  hosts: 'http://127.0.0.1:2379',
});

// 服务信息
const serviceInfo = {
  name: SERVICE_NAME,
  address: getLocalIP(),
  port: PORT,
  protocol: 'http',
  version: '1.0.0',
  timestamp: Date.now(),
};

// API 路由（同上）
app.get('/users/:id', (req, res) => {
  // ...
});

app.get('/health', (req, res) => {
  res.json({ status: 'UP' });
});

// 注册服务到 etcd（使用租约 Lease）
async function registerService() {
  try {
    // 创建租约（TTL 10 秒）
    const lease = etcd.lease(10);
    
    // 注册服务（绑定租约）
    await etcd.put(SERVICE_KEY)
      .value(JSON.stringify(serviceInfo))
      .lease(lease)
      .exec();
    
    console.log(`✅ Service registered: ${SERVICE_KEY}`);
    
    // 保持租约（每 3 秒续约）
    const keepAlive = await lease.keepalive();
    
    keepAlive.on('lost', async () => {
      console.error('❌ Lease lost, re-registering...');
      await registerService();
    });
  } catch (error) {
    console.error('❌ Failed to register service:', error);
    process.exit(1);
  }
}

// 注销服务

    console.error('❌ Failed to deregister service:', error);
  }
}
```

#### 步骤4：实现服务发现（Watch 机制）

```javascript
// src/services/order-service-etcd.js
import express from 'express';
import { Etcd3 } from 'etcd3';
import axios from 'axios';

const app = express();
app.use(express.json());

const PORT = 4000;
const SERVICE_NAME = 'user-service';
const SERVICE_PREFIX = `/services/${SERVICE_NAME}/`;

const etcd = new Etcd3({
  hosts: 'http://127.0.0.1:2379',
});

const instances = new Map();
let currentIndex = 0;

// 服务发现：获取所有服务实例
async function discoverServices() {
  try {
    const result = await etcd.getAll()
      .prefix(SERVICE_PREFIX)
      .strings();
    
    instances.clear();
    
    for (const [key, value] of Object.entries(result)) {
      const service = JSON.parse(value);
      instances.set(key, service);
    }
    
    console.log(`✅ Discovered ${instances.size} instances of ${SERVICE_NAME}`);
  } catch (error) {
    console.error('❌ Service discovery failed:', error);
  }
}

// 监听服务变化
async function watchServices() {
  const watcher = await etcd
    .watch()
    .prefix(SERVICE_PREFIX)
    .create();
  
  watcher
    .on('put', (res) => {
      const service = JSON.parse(res.value.toString());
      instances.set(res.key.toString(), service);
      console.log(`✅ Service added: ${res.key}`);
    })
    .on('delete', (res) => {
      instances.delete(res.key.toString());
      console.log(`❌ Service removed: ${res.key}`);
    });
  
  console.log(`👀 Watching services: ${SERVICE_PREFIX}`);
}

// 负载均衡（轮询）
function getNextInstance() {
  if (instances.size === 0) {
    throw new Error('No available instances');
  }
  
  const instanceArray = Array.from(instances.values());
  const instance = instanceArray[currentIndex % instanceArray.length];
  currentIndex++;
  
  return instance;
}

// 初始化
(async () => {
  await discoverServices();
  await watchServices();
})();

// 创建订单接口
app.post('/orders', async (req, res) => {
  const { userId, productId, quantity } = req.body;
  
  try {
    const instance = getNextInstance();
    const userServiceUrl = `http://${instance.address}:${instance.port}`;
    
    console.log(`📞 Calling user-service: ${userServiceUrl}`);
    
    const userResponse = await axios.get(`${userServiceUrl}/users/${userId}`, {
      timeout: 3000,
    });
    
    const order = {
      id: `ORD${Date.now()}`,
      userId,
      user: userResponse.data.name,
      productId,
      quantity,
      status: 'pending',
      createdAt: new Date(),
    };
    
    res.json(order);
  } catch (error) {
    console.error('❌ Order creation failed:', error.message);
    res.status(503).json({ error: 'Service unavailable' });
  }
});

app.listen(PORT, () => {
  console.log(`🚀 Order Service running on port ${PORT}`);
});
```

### 场景2：灰度发布（金丝雀部署）

**需求：**
- 发布新版本服务（v2.0），但不确定稳定性
- 先让 10% 流量走新版本，观察效果
- 如果正常，逐步切换到 100%

**解决方案：使用 Consul Tags**

```javascript
// 服务注册时添加版本标签
await consul.agent.service.register({
  id: SERVICE_ID,
  name: SERVICE_NAME,
  tags: ['v2', 'canary'],  // 标记为金丝雀版本
  // ...
});

// 客户端按权重分配流量
async function getServiceWithCanary(serviceName) {
  const allInstances = await discoverService(serviceName);
  
  // 10% 流量走 canary 版本
  const random = Math.random();
  const canaryInstances = allInstances.filter(i => i.tags.includes('canary'));
  
  if (random < 0.1 && canaryInstances.length > 0) {
    return canaryInstances[Math.floor(Math.random() * canaryInstances.length)];
  }
  
  // 90% 流量走稳定版本
  const stableInstances = allInstances.filter(i => !i.tags.includes('canary'));
  return stableInstances[Math.floor(Math.random() * stableInstances.length)];
}
```

### 错误1：未实现健康检查导致调用失败节点

**错误代码：**
```javascript
// ❌ 错误：未注册健康检查

  name: SERVICE_NAME,
  address: '192.168.1.10',
  port: 3000,
  // 缺少 check 字段

// 后果：服务宕机后仍在注册表中，客户端会调用失败
```

**正确做法：**
```javascript
// ✅ 必须实现健康检查
await consul.agent.service.register({
  id: SERVICE_ID,
  name: SERVICE_NAME,
  address: '192.168.1.10',
  port: 3000,
  check: {
    http: 'http://192.168.1.10:3000/health',
    interval: '10s',  // 每 10 秒检查
    timeout: '5s',    // 5 秒超时
    deregistercriticalserviceafter: '1m',  // 失败 1 分钟后自动注销
  },
});

// 必须实现 /health 端点
app.get('/health', (req, res) => {
  // 检查依赖服务（数据库、Redis 等）
  const isHealthy = checkDependencies();
  
  if (isHealthy) {
    res.json({ status: 'UP' });
  } else {
    res.status(503).json({ status: 'DOWN' });
  }
});
```

### 错误2：未优雅退出导致僵尸节点

**错误代码：**
```javascript
// ❌ 错误：进程退出时未注销服务
process.on('SIGINT', () => {
  console.log('Bye!');
  process.exit(0);  // 直接退出
});

// 后果：Consul 中仍有该节点记录，等待 TTL 超时才会摘除
```

**正确做法：**
```javascript
// ✅ 优雅退出：先注销服务再退出
process.on('SIGINT', async () => {
  console.log('\n⏹️ Shutting down...');
  
  // 1. 注销服务
  await consul.agent.service.deregister(SERVICE_ID);
  
  // 2. 关闭服务器
  server.close(() => {
    console.log('✅ Server closed');
    
    // 3. 关闭数据库连接等
    // await db.close();
    
    process.exit(0);
  });
  
  // 4. 强制退出（超时 30 秒）
  setTimeout(() => {
    console.error('❌ Forced shutdown');
    process.exit(1);
  }, 30000);
});

// 同时处理 SIGTERM（Docker/K8s 使用）
process.on('SIGTERM', async () => {
  await consul.agent.service.deregister(SERVICE_ID);
  server.close(() => process.exit(0));
});
```

```javascript
async function watchServiceChanges(serviceName) {
  let index = 0;
  
  while (true) {

      const result = await consul.health.service({
```

## 🤔 FAQ

### Q1：Consul 和 Nacos 如何选择？

**Consul：**
- ✅ 成熟稳定，生态完善
- ✅ 内置 Service Mesh（Consul Connect）
- ✅ 适合国际化项目

**Nacos：**
- ✅ 阿里开源，中文文档友好
- ✅ 配置中心功能更强（支持命名空间、配置历史）
- ✅ 适合 Spring Cloud 生态

**选择建议：**
- 新项目 + Node.js → Consul
- 已有 Spring Cloud → Nacos
- Kubernetes → etcd（内置）

### Q2：如何实现多数据中心服务发现？

**Consul 原生支持多数据中心：**

```javascript
// 查询远程数据中心的服务
const instances = await consul.health.service({
  service: 'user-service',
  dc: 'us-west',  // 指定数据中心
  passing: true,
});

// 跨数据中心调用（需要 Consul Connect）
const client = new userProto.UserService(
  'user-service.service.us-west.consul:50051',
  grpc.credentials.createInsecure()
);
```

### Q3：如何防止服务雪崩？

**核心策略：**

1. **健康检查**（自动摘除故障节点）
2. **超时控制**（防止慢调用阻塞）
3. **熔断器**（Circuit Breaker）
4. **限流**（Rate Limiting）

**代码示例：**
```javascript
import CircuitBreaker from 'opossum';

// 使用熔断器包装服务调用
const breaker = new CircuitBreaker(async (userId) => {
  const instance = getNextInstance();
  const url = `http://${instance.address}:${instance.port}/users/${userId}`;
  
  const response = await axios.get(url, { timeout: 3000 });
  return response.data;
}, {
  timeout: 3000,      // 3 秒超时
  errorThresholdPercentage: 50,  // 50% 错误率时熔断
  resetTimeout: 30000,  // 30 秒后尝试恢复
});

// 熔断时的降级逻辑
breaker.fallback(() => ({
  id: 'unknown',
  name: 'Guest',
  email: 'guest@example.com',
}));

// 调用
const user = await breaker.fire(userId);
```
