# 8.4 数据库连接池深度优化

> 精准计算连接池参数，解决连接耗尽与连接泄漏问题

---

## 📚 知识导读

### 为什么需要连接池？

**没有连接池：**
```javascript
// ❌ 每次请求都创建新连接
app.get('/api/users', async (req, res) => {
  const connection = await mysql.createConnection({
    host: 'localhost',
    user: 'root',
    password: 'password',
    database: 'myapp',
  });

  const [users] = await connection.execute('SELECT * FROM users');
  await connection.end();  // 关闭连接

  res.json(users);
});

// 问题：
// - 创建连接耗时（TCP 三次握手 + MySQL 认证）：50-200ms
// - 高并发时大量创建/销毁连接：CPU 飙升
// - 数据库连接数耗尽：max_connections 超限
```

**有连接池：**
```javascript
// ✅ 复用连接
const pool = mysql.createPool({ ... });

app.get('/api/users', async (req, res) => {
  const [users] = await pool.execute('SELECT * FROM users');
  // 连接自动归还池
  res.json(users);
});

// 优势：
// - 复用连接：0-10ms
// - 控制连接数：不超过 max
// - 减少数据库压力
```

## 🎯 连接池工作原理

### 1. 连接生命周期

```
┌─────────────────────────────────────────────────────┐
│  连接池（Connection Pool）                           │
│                                                     │
│  空闲连接（Idle）：[conn1] [conn2] [conn3]          │
│                       ↓        ↓        ↓           │
│  活跃连接（Active）：[conn4] [conn5]                │
│                                                     │
│  参数：                                             │
│  - min: 3（最小连接数）                              │
│  - max: 10（最大连接数）                             │
│  - idleTimeout: 30s（空闲超时）                      │
│  - acquireTimeout: 10s（获取超时）                   │
└─────────────────────────────────────────────────────┘

流程：
1. 应用请求连接 → 从空闲池获取
2. 无空闲连接 → 创建新连接（总数 < max）
3. 达到 max → 等待（acquireTimeout）
4. 使用完毕 → 归还到空闲池
5. 空闲超时（idleTimeout）→ 销毁连接
```

## 🎯 关键参数调优

### 1. 最大连接数（connectionLimit / max）

**如何计算？**

```
公式：
connectionLimit = (核心数 × 2) + 有效磁盘数

示例：
- CPU 核心数：8
- 有效磁盘数：1（单个 SSD）
connectionLimit = 8 × 2 + 1 = 17

推荐值：
- 开发环境：10-20
- 生产环境：20-50（根据实际测试）
- 过大：数据库压力增加，上下文切换频繁
- 过小：请求排队，响应变慢
```

**压测确定最优值：**
```bash
# 使用 Artillery 压测
artillery quick -d 60 -r 100 http://localhost:3000/api/users

# 观察：
# - 响应时间（P95/P99）
# - 数据库 CPU/内存
# - 连接池队列长度

# 逐步调整 connectionLimit：10 → 20 → 30 → 40
# 找到最优值（响应时间最低 + 数据库负载可接受）
```

## 🎯 mysql2 连接池优化

### 1. 最佳实践配置

```javascript
import mysql from 'mysql2/promise';

const pool = mysql.createPool({
  // 连接配置
  host: process.env.DB_HOST,
  port: parseInt(process.env.DB_PORT, 10) || 3306,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  database: process.env.DB_NAME,

  // 连接池配置
  waitForConnections: true,       // 无可用连接时等待（不抛错）
  connectionLimit: 20,            // 最大连接数
  queueLimit: 0,                  // 等待队列无限制
  enableKeepAlive: true,          // 启用 TCP Keep-Alive
  keepAliveInitialDelay: 0,       // 立即发送 Keep-Alive

  // 连接验证
  connectTimeout: 10000,          // 连接超时 10 秒
  acquireTimeout: 10000,          // 获取连接超时 10 秒
  timeout: 60000,                 // 查询超时 60 秒

  // 字符集
  charset: 'utf8mb4',

  // 时区
  timezone: '+08:00',

  // 错误处理
  debug: false,
  multipleStatements: false,      // 禁止多语句（安全）
});

// 错误监听
pool.on('connection', (connection) => {
  console.log('New connection established:', connection.threadId);
});

pool.on('acquire', (connection) => {
  console.log('Connection acquired:', connection.threadId);
});

pool.on('release', (connection) => {
  console.log('Connection released:', connection.threadId);
});

pool.on('enqueue', () => {
  console.log('Waiting for available connection slot');
});

export default pool;
```

### 2. 正确使用连接

```javascript
// ✅ 方法1：使用 pool.execute（推荐）
const [users] = await pool.execute('SELECT * FROM users WHERE age > ?', [18]);
// 自动获取连接 → 执行查询 → 释放连接

// ✅ 方法2：使用 pool.query
const [users] = await pool.query('SELECT * FROM users WHERE age > ?', [18]);

// ✅ 方法3：手动获取连接（事务场景）
const connection = await pool.getConnection();
try {
  await connection.beginTransaction();
  await connection.execute('UPDATE accounts SET balance = balance - 100 WHERE id = ?', [1]);
  await connection.execute('UPDATE accounts SET balance = balance + 100 WHERE id = ?', [2]);
  await connection.commit();
} catch (error) {
  await connection.rollback();
  throw error;
} finally {
  connection.release();  // ← 必须释放！
}

// ❌ 错误：忘记释放连接
const connection = await pool.getConnection();
await connection.execute('SELECT * FROM users');
// 没有 connection.release() → 连接泄漏 ❌
```

## 🎯 连接泄漏检测与修复

### 1. 检测连接泄漏

```javascript
// 连接泄漏检测器
class ConnectionLeakDetector {
  constructor(pool) {
    this.pool = pool;
    this.leakThreshold = 10000;  // 10 秒
    this.activeConnections = new Map();
  }

  trackConnection(connection, stackTrace) {
    this.activeConnections.set(connection.threadId, {
      acquiredAt: Date.now(),
      stackTrace,
    });
  }

  releaseConnection(connection) {
    this.activeConnections.delete(connection.threadId);
  }

  checkLeaks() {
    const now = Date.now();

    for (const [threadId, info] of this.activeConnections.entries()) {
      const duration = now - info.acquiredAt;

      if (duration > this.leakThreshold) {
        console.error(`Connection leak detected! Thread ${threadId} held for ${duration}ms`);
        console.error('Stack trace:', info.stackTrace);
      }
    }
  }
}

// 使用
const detector = new ConnectionLeakDetector(pool);

// 包装 getConnection
const originalGetConnection = pool.getConnection.bind(pool);
pool.getConnection = async function() {
  const connection = await originalGetConnection();
  const stackTrace = new Error().stack;

  detector.trackConnection(connection, stackTrace);

  // 包装 release
  const originalRelease = connection.release.bind(connection);
  connection.release = function() {
    detector.releaseConnection(connection);
    return originalRelease();
  };

  return connection;
};

// 定期检查
setInterval(() => detector.checkLeaks(), 5000);
```

### 2. 修复连接泄漏

```javascript
// ❌ 错误示例：连接泄漏
async function badExample() {
  const connection = await pool.getConnection();
  const [users] = await connection.execute('SELECT * FROM users');

  if (users.length === 0) {
    return [];  // ← 提前返回，忘记释放连接 ❌
  }

  connection.release();
  return users;
}

// ✅ 修复1：使用 try-finally
async function goodExample1() {
  const connection = await pool.getConnection();
  try {
    const [users] = await connection.execute('SELECT * FROM users');
    return users.length === 0 ? [] : users;
  } finally {
    connection.release();  // ← 确保释放 ✅
  }
}

// ✅ 修复2：使用 pool.execute（推荐）
async function goodExample2() {
  const [users] = await pool.execute('SELECT * FROM users');
  return users.length === 0 ? [] : users;  // 自动释放 ✅
}

// ✅ 修复3：包装工具函数
async function withConnection(callback) {
  const connection = await pool.getConnection();
  try {
    return await callback(connection);
  } finally {
    connection.release();
  }
}

// 使用
const users = await withConnection(async (conn) => {
  const [users] = await conn.execute('SELECT * FROM users');
  return users;
});
```

## 📚 FAQ

### 1. 连接池大小如何确定？

**经验公式：**
```
最优连接池大小 = (核心数 × 2) + 有效磁盘数
```

### 2. 连接池耗尽如何处理？

**方案：**
1. 增加 `connectionLimit`
2. 优化慢查询（减少连接占用时间）
3. 使用队列限流（`queueLimit`）
4. 返回 503 提示用户重试

### 3. 如何监控连接池健康？

```javascript
// 导出监控指标（Prometheus 格式）
import promClient from 'prom-client';

const poolSizeGauge = new promClient.Gauge({
  name: 'db_pool_size',
  help: 'Database connection pool size',
  labelNames: ['state'],
});

setInterval(() => {
  poolSizeGauge.set({ state: 'total' }, pool._allConnections.length);
  poolSizeGauge.set({ state: 'free' }, pool._freeConnections.length);
  poolSizeGauge.set({ state: 'active' }, pool._allConnections.length - pool._freeConnections.length);
```
