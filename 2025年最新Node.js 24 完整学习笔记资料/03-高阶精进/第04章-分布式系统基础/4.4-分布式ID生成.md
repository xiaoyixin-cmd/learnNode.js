# 4.4 分布式 ID 生成

> Snowflake / UUID / NanoID / ULID 的原理与实现

---

## 📚 核心知识

### 1. 为什么需要分布式ID？

**数据库自增ID的局限性：**

```
单机数据库：
┌──────────────┐
│  MySQL       │
│  AUTO_INCREMENT  → 1, 2, 3, 4, ...
└──────────────┘

分布式数据库：
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│  Shard 1     │  │  Shard 2     │  │  Shard 3     │
│  1, 2, 3...  │  │  1, 2, 3...  │  │  1, 2, 3...  │
└──────────────┘  └──────────────┘  └──────────────┘
       ↓                 ↓                 ↓
    ❌ ID 冲突！每个分片都从 1 开始

问题：
❌ ID 冲突
❌ 无法按时间排序
❌ 无法水平扩展
```

**分布式ID的需求：**

1. **全局唯一性**：任何时刻、任何节点生成的ID不重复
2. **趋势递增**：大致按时间顺序递增（有利于数据库索引）
3. **高性能**：每秒生成数万个ID
4. **高可用**：单点故障不影响ID生成
5. **信息安全**：不暴露业务信息（如订单量）

## 💻 实战代码

### 实战1：Snowflake 算法（Twitter 雪花算法）

**原理：**

```
Snowflake ID 结构（64位）：

┌─────────────────────────────────────────────────────────┐
│ 1位  │  41位时间戳  │ 10位机器ID │ 12位序列号          │
│ 符号 │  (毫秒)     │  (节点)   │  (同一毫秒内)       │
├─────┼─────────────┼───────────┼──────────────────────┤
│  0  │ 1234567890  │   0001    │  0000 0000 0001      │
└─────┴─────────────┴───────────┴──────────────────────┘

特点：
✅ 41位时间戳：可用约69年（2^41 / 31536000000）
✅ 10位机器ID：支持1024个节点
✅ 12位序列号：同一毫秒内可生成4096个ID
✅ 总容量：每秒可生成 4096 × 1000 = 409万个ID
```

**代码实现：**

```javascript
// snowflake.js
class Snowflake {
  constructor(workerId, datacenterId) {
    // 起始时间戳（2020-01-01 00:00:00）
    this.epoch = 1577836800000n;
    
    // 各部分位数
    this.workerIdBits = 5n;
    this.datacenterIdBits = 5n;
    this.sequenceBits = 12n;
    
    // 最大值
    this.maxWorkerId = -1n ^ (-1n << this.workerIdBits);  // 31
    this.maxDatacenterId = -1n ^ (-1n << this.datacenterIdBits);  // 31
    this.sequenceMask = -1n ^ (-1n << this.sequenceBits);  // 4095
    
    // 位移
    this.workerIdShift = this.sequenceBits;
    this.datacenterIdShift = this.sequenceBits + this.workerIdBits;
    this.timestampLeftShift = this.sequenceBits + this.workerIdBits + this.datacenterIdBits;
    
    // 初始化
    this.workerId = BigInt(workerId);
    this.datacenterId = BigInt(datacenterId);
    this.sequence = 0n;
    this.lastTimestamp = -1n;
    
    // 参数校验
    if (this.workerId > this.maxWorkerId || this.workerId < 0n) {
      throw new Error(`Worker ID must be between 0 and ${this.maxWorkerId}`);
    }
    
    if (this.datacenterId > this.maxDatacenterId || this.datacenterId < 0n) {
      throw new Error(`Datacenter ID must be between 0 and ${this.maxDatacenterId}`);
    }
  }
  
  // 生成ID
  nextId() {
    let timestamp = BigInt(Date.now());
    
    // 时钟回拨检测
    if (timestamp < this.lastTimestamp) {
      throw new Error(`Clock moved backwards. Refusing to generate id for ${this.lastTimestamp - timestamp} milliseconds`);
    }
    
    // 同一毫秒内
    if (timestamp === this.lastTimestamp) {
      this.sequence = (this.sequence + 1n) & this.sequenceMask;
      
      // 序列号溢出（同一毫秒内生成超过4096个ID）
      if (this.sequence === 0n) {
        // 等待下一毫秒
        timestamp = this.tilNextMillis(this.lastTimestamp);
      }
    } else {
      // 不同毫秒，重置序列号
      this.sequence = 0n;
    }
    
    this.lastTimestamp = timestamp;
    
    // 组装ID
    const id = ((timestamp - this.epoch) << this.timestampLeftShift) |
               (this.datacenterId << this.datacenterIdShift) |
               (this.workerId << this.workerIdShift) |
               this.sequence;
    
    return id.toString();
  }
  
  // 等待下一毫秒
  tilNextMillis(lastTimestamp) {
    let timestamp = BigInt(Date.now());
    
    while (timestamp <= lastTimestamp) {
      timestamp = BigInt(Date.now());
    }
    
    return timestamp;
  }
  
  // 解析ID
  parse(id) {
    const bigId = BigInt(id);
    
    const timestamp = (bigId >> this.timestampLeftShift) + this.epoch;
    const datacenterId = (bigId >> this.datacenterIdShift) & ((1n << this.datacenterIdBits) - 1n);
    const workerId = (bigId >> this.workerIdShift) & ((1n << this.workerIdBits) - 1n);
    const sequence = bigId & this.sequenceMask;
    
    return {
      timestamp: Number(timestamp),
      datacenterId: Number(datacenterId),
      workerId: Number(workerId),
      sequence: Number(sequence),
      date: new Date(Number(timestamp)).toISOString(),
    };
  }
}

// 测试
const snowflake = new Snowflake(1, 1);

console.log('=== Snowflake ID Generator ===\n');

// 生成10个ID
for (let i = 0; i < 10; i++) {
  const id = snowflake.nextId();
  console.log(`ID: ${id}`);
  
  const parsed = snowflake.parse(id);
  console.log(`  - Time: ${parsed.date}`);
  console.log(`  - Datacenter: ${parsed.datacenterId}`);
  console.log(`  - Worker: ${parsed.workerId}`);
  console.log(`  - Sequence: ${parsed.sequence}\n`);
}

// 性能测试
console.log('=== Performance Test ===\n');

const startTime = Date.now();
const count = 100000;

for (let i = 0; i < count; i++) {
  snowflake.nextId();
}

const endTime = Date.now();
const duration = endTime - startTime;

console.log(`Generated ${count} IDs in ${duration}ms`);
console.log(`QPS: ${Math.floor(count / (duration / 1000))}`);
```

## 🎯 应用场景

### 场景1：订单号生成（Snowflake）

```javascript
class OrderIdGenerator {
  constructor(workerId, datacenterId) {
    this.snowflake = new Snowflake(workerId, datacenterId);
  }
  
  generate() {
    const id = this.snowflake.nextId();
    
    // 格式化为订单号（添加前缀）
    return `ORD${id}`;
  }
}

const generator = new OrderIdGenerator(1, 1);

// 生成订单号
const orderId = generator.generate();
console.log('Order ID:', orderId);
// Output: ORD1234567890123456789
```

## ❌ 高频错误

### 错误1：时钟回拨导致ID重复

**问题：**
```
服务器时间被NTP同步向前调整 → Snowflake生成重复ID
```

**解决方案：**
```javascript
// ✅ 检测时钟回拨
nextId() {
  let timestamp = BigInt(Date.now());
  
  if (timestamp < this.lastTimestamp) {
    const offset = this.lastTimestamp - timestamp;
    
    if (offset <= 5n) {
      // 回拨 < 5ms，等待
      while (timestamp <= this.lastTimestamp) {
        timestamp = BigInt(Date.now());
      }
    } else {
      // 回拨 > 5ms，拒绝生成
      throw new Error(`Clock moved backwards by ${offset}ms`);
    }
  }
  
  // ...
}
```

## 🤔 FAQ

### Q1：如何选择分布式ID方案？

| 方案 | 长度 | 有序性 | 性能 | 适用场景 |
|------|------|--------|------|---------|
| Snowflake | 19位数字 | ✅ 趋势递增 | 极高 | 订单号、用户ID |
| UUID v4 | 36字符 | ❌ 无序 | 高 | 文件名、会话ID |
| NanoID | 21字符 | ❌ 无序 | 极高 | 短链接、前端ID |
| ULID | 26字符 | ✅ 字典序 | 高 | 日志ID、事件ID |
