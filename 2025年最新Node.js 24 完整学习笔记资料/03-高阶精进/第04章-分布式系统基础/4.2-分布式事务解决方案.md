# 4.2 分布式事务解决方案

> 保证分布式系统的数据一致性：从 2PC 到 Saga

---

## 📚 核心知识

### 1. 什么是分布式事务？

**单机事务 vs 分布式事务：**

```javascript
// ❌ 单机事务（简单，ACID保证）
BEGIN TRANSACTION;
  UPDATE accounts SET balance = balance - 100 WHERE id = 'A';
  UPDATE accounts SET balance = balance + 100 WHERE id = 'B';
COMMIT;

// ✅ 分布式事务（复杂，跨多个数据库）
// 数据库1：扣减A账户（192.168.1.10）
// 数据库2：增加B账户（192.168.1.11）
// 问题：如何保证两个操作要么都成功，要么都失败？
```

**分布式事务的挑战：**

```
┌───────────────────────────────────────────────────────┐
│              分布式事务的核心问题                      │
├───────────────────────────────────────────────────────┤
│                                                       │
│  1. 网络不可靠                                         │
│     - 请求丢失                                         │
│     - 响应超时                                         │
│     - 网络分区                                         │
│                                                       │
│  2. 节点故障                                           │
│     - 参与者宕机                                       │
│     - 协调者宕机                                       │
│     - 部分节点故障                                     │
│                                                       │
│  3. 并发控制                                           │
│     - 多个事务同时执行                                 │
│     - 死锁问题                                         │
│     - 资源竞争                                         │
│                                                       │
└───────────────────────────────────────────────────────┘
```

### 2. 五种解决方案对比

| 方案 | 一致性 | 性能 | 复杂度 | 可用性 | 适用场景 |
|------|--------|------|--------|--------|---------|
| **2PC** | 强一致 | 低 | 中 | 低 | 强一致性要求（金融） |
| **3PC** | 强一致 | 低 | 高 | 中 | 2PC改进版（很少用） |
| **TCC** | 强一致 | 中 | 高 | 中 | 电商下单 |
| **Saga** | 最终一致 | 高 | 高 | 高 | 长流程事务 |
| **消息队列** | 最终一致 | 高 | 中 | 高 | 异步场景 |

### 实战1：2PC（两阶段提交）

**原理：**

```
┌─────────────────────────────────────────────────────┐
│              2PC 协议流程                            │
├─────────────────────────────────────────────────────┤
│                                                     │
│  协调者 (Coordinator)                               │
│     │                                               │
│     ├──── Phase 1: Prepare ───┐                    │
│     │                           │                   │
│     ▼                           ▼                   │
│  参与者A                      参与者B               │
│  (执行但不提交)               (执行但不提交)        │
│     │                           │                   │
│     ├──── YES ─────────────────┤                   │
│     │                           │                   │
│     │◄──── Phase 2: Commit ────┤                   │
│     │                           │                   │
│     ▼                           ▼                   │
│  提交事务                      提交事务             │
│                                                     │
│  任一参与者返回 NO → 所有参与者回滚                 │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**代码实现：**

```javascript
// 2pc-transaction.js
import { Client as PgClient } from 'pg';

class TwoPhaseCommitCoordinator {
  constructor() {
    this.participants = [];
  }
  
  // 添加参与者
  addParticipant(db) {
    this.participants.push(db);
  }
  
  // 执行分布式事务
  async execute(operations) {
    const txId = `tx_${Date.now()}`;
    
    try {
      // ===== 阶段1：准备（Prepare） =====
      console.log(`\n[2PC] Phase 1: Prepare (txId: ${txId})`);
      
      for (let i = 0; i < this.participants.length; i++) {
        const db = this.participants[i];
        const operation = operations[i];
        
        await db.connect();
        await db.query('BEGIN');
        
        // 执行操作（但不提交）
        await db.query(operation.sql, operation.params);
        
        // 准备事务
        await db.query(`PREPARE TRANSACTION '${txId}_${i}'`);
        
        console.log(`  ✅ Participant ${i} prepared`);
      }
      
      console.log('[2PC] All participants prepared successfully');
      
      // ===== 阶段2：提交（Commit） =====
      console.log(`\n[2PC] Phase 2: Commit`);
      
      for (let i = 0; i < this.participants.length; i++) {
        const db = this.participants[i];
        
        await db.query(`COMMIT PREPARED '${txId}_${i}'`);
        
        console.log(`  ✅ Participant ${i} committed`);
      }
      
      console.log('[2PC] Transaction committed successfully ✅\n');
      
      return { success: true };
    } catch (error) {
      // 任何阶段失败，回滚所有参与者
      console.error(`\n[2PC] Transaction failed: ${error.message}`);
      console.log('[2PC] Rolling back all participants...');
      
      for (let i = 0; i < this.participants.length; i++) {
        try {
          const db = this.participants[i];
          await db.query(`ROLLBACK PREPARED '${txId}_${i}'`);
          console.log(`  ✅ Participant ${i} rolled back`);
        } catch (e) {
          console.error(`  ❌ Failed to rollback participant ${i}:`, e.message);
        }
      }
      
      return { success: false, error: error.message };
    } finally {
      // 关闭连接
      for (const db of this.participants) {
        await db.end();
      }
    }
  }
}

// 测试：跨数据库转账
async function testTransfer() {
  const db1 = new PgClient({
    host: 'localhost',
    port: 5432,
    user: 'postgres',

    database: 'shard1',
  });
  
  const db2 = new PgClient({
```

**2PC 的缺点：**

```
❌ 同步阻塞：所有参与者等待协调者指令
❌ 单点故障：协调者宕机导致参与者一直阻塞
❌ 数据不一致：网络分区导致部分提交、部分回滚
❌ 性能差：需要多轮网络通信
```

### 实战2：TCC（Try-Confirm-Cancel）

**原理：**

```
TCC = Try + Confirm + Cancel

┌─────────────────────────────────────────────────────┐
│                TCC 三个阶段                          │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. Try 阶段（预留资源）                             │
│     - 检查资源是否足够                               │
│     - 预留资源（如冻结库存）                         │
│     - 不直接扣减                                     │
│                                                     │
│  2. Confirm 阶段（确认）                             │
│     - 真正执行业务逻辑                               │
│     - 扣减预留的资源                                 │
│     - 必须成功（幂等）                               │
│                                                     │
│  3. Cancel 阶段（取消）                              │
│     - 释放预留的资源                                 │
│     - 回滚 Try 阶段的操作                            │
│     - 必须成功（幂等）                               │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**代码实现：**

```javascript
// tcc-transaction.js
import Redis from 'ioredis';

class TCCCoordinator {
  constructor() {
    this.redis = new Redis();
    this.participants = [];
  }
  
  addParticipant(participant) {
    this.participants.push(participant);
  }
  
  async execute(txId) {
    try {
      // ===== 阶段1：Try（预留资源） =====
      console.log(`\n[TCC] Phase 1: Try (txId: ${txId})`);
      
      for (const participant of this.participants) {
        const result = await participant.try(txId);
        
        if (!result.success) {
          throw new Error(`Try failed: ${participant.name}`);
        }
        
        console.log(`  ✅ ${participant.name} try success`);
      }
      
      // 记录事务状态
      await this.redis.set(`tcc:${txId}:status`, 'TRIED');
      
      console.log('[TCC] All participants tried successfully');
      
      // ===== 阶段2：Confirm（确认） =====
      console.log(`\n[TCC] Phase 2: Confirm`);
      
      for (const participant of this.participants) {
        await participant.confirm(txId);
        console.log(`  ✅ ${participant.name} confirm success`);
      }
      
      await this.redis.set(`tcc:${txId}:status`, 'CONFIRMED');
      
      console.log('[TCC] Transaction confirmed successfully ✅\n');
      
      return { success: true };
    } catch (error) {
      // ===== 阶段3：Cancel（取消） =====
      console.error(`\n[TCC] Transaction failed: ${error.message}`);
      console.log('[TCC] Phase 3: Cancel');
      
      for (const participant of this.participants) {
        try {
          await participant.cancel(txId);
          console.log(`  ✅ ${participant.name} cancel success`);
        } catch (e) {
          console.error(`  ❌ ${participant.name} cancel failed:`, e.message);
        }
      }
      
      await this.redis.set(`tcc:${txId}:status`, 'CANCELLED');
      
      return { success: false, error: error.message };
    }
  }
}

// 库存服务（TCC 参与者）
class InventoryServiceTCC {
  constructor() {
    this.name = 'InventoryService';
    this.redis = new Redis();
  }
  
  // Try：冻结库存
  async try(txId) {
    const stock = await this.redis.get('inventory:SKU001:stock');
    const frozen = await this.redis.get('inventory:SKU001:frozen') || 0;
    
    const available = parseInt(stock) - parseInt(frozen);
    const quantity = 10;  // 需要扣减的数量
    
    if (available < quantity) {
      return { success: false, error: 'Insufficient stock' };
    }
    
    // 冻结库存
    await this.redis.incrby('inventory:SKU001:frozen', quantity);
    await this.redis.set(`tcc:${txId}:inventory:frozen`, quantity);
    
    console.log(`    Frozen ${quantity} units of SKU001`);
    
    return { success: true };
  }
  
  // Confirm：真正扣减库存
  async confirm(txId) {
    const frozen = await this.redis.get(`tcc:${txId}:inventory:frozen`);
    
    // 扣减库存
    await this.redis.decrby('inventory:SKU001:stock', parseInt(frozen));
    
    // 释放冻结数量
    await this.redis.decrby('inventory:SKU001:frozen', parseInt(frozen));
    
    // 清理事务数据
    await this.redis.del(`tcc:${txId}:inventory:frozen`);
    
    console.log(`    Deducted ${frozen} units from stock`);
  }
  
  // Cancel：释放冻结库存
  async cancel(txId) {
    const frozen = await this.redis.get(`tcc:${txId}:inventory:frozen`);
    
    if (frozen) {
      // 释放冻结数量
      await this.redis.decrby('inventory:SKU001:frozen', parseInt(frozen));
      
      // 清理事务数据
      await this.redis.del(`tcc:${txId}:inventory:frozen`);
      
      console.log(`    Released ${frozen} units of frozen stock`);
    }
  }
}

// 积分服务（TCC 参与者）
class PointsServiceTCC {
  constructor() {
    this.name = 'PointsService';
    this.redis = new Redis();
  }
  
  // Try：预留积分
  async try(txId) {
    const points = await this.redis.get('points:user123');
    const cost = 100;  // 需要扣减的积分
    
    if (parseInt(points) < cost) {
      return { success: false, error: 'Insufficient points' };
    }
    
    // 预留积分
    await this.redis.set(`tcc:${txId}:points:reserved`, cost);
    
    console.log(`    Reserved ${cost} points`);
    
    return { success: true };
  }
  
  // Confirm：真正扣减积分
  async confirm(txId) {
    const reserved = await this.redis.get(`tcc:${txId}:points:reserved`);
    
    // 扣减积分
    await this.redis.decrby('points:user123', parseInt(reserved));
    
    // 清理事务数据
    await this.redis.del(`tcc:${txId}:points:reserved`);
    
    console.log(`    Deducted ${reserved} points`);
  }
  
  // Cancel：释放预留积分
  async cancel(txId) {
    // 清理事务数据
    await this.redis.del(`tcc:${txId}:points:reserved`);
    
    console.log(`    Released reserved points`);
  }
}

// 测试
async function testTCC() {
  const redis = new Redis();
  
  // 初始化数据
  await redis.set('inventory:SKU001:stock', 100);
  await redis.set('inventory:SKU001:frozen', 0);
  await redis.set('points:user123', 500);
  
  const coordinator = new TCCCoordinator();
  coordinator.addParticipant(new InventoryServiceTCC());
  coordinator.addParticipant(new PointsServiceTCC());
  
  // 执行事务
  const txId = `TX${Date.now()}`;
  const result = await coordinator.execute(txId);
  
  console.log('Transaction result:', result);
  
  // 查看最终状态
  console.log('\n=== Final State ===');
  console.log('Stock:', await redis.get('inventory:SKU001:stock'));
  console.log('Frozen:', await redis.get('inventory:SKU001:frozen'));
  console.log('Points:', await redis.get('points:user123'));
  
  await redis.quit();
}

testTCC();
```

### 实战3：Saga 模式（长事务）

**原理：**

```
Saga = 一系列本地事务 + 补偿事务

正向流程：
T1 → T2 → T3 → T4 (全部成功)

失败回滚（补偿）：
T1 → T2 → T3 → T4失败 → C3 → C2 → C1 (反向补偿)
```

**代码实现（编排式 Saga）：**

```javascript
// saga-orchestrator.js
class SagaOrchestrator {
  constructor() {
    this.steps = [];
  }
  
  // 添加步骤
  addStep(name, action, compensation) {
    this.steps.push({ name, action, compensation });
  }
  
  // 执行 Saga
  async execute() {
    const executedSteps = [];
    
    try {
      console.log('\n[Saga] Starting transaction...\n');
      
      // 正向执行所有步骤
      for (const step of this.steps) {
        console.log(`[Saga] Executing: ${step.name}`);
        
        await step.action();
        executedSteps.push(step);
        
        console.log(`  ✅ ${step.name} success`);
      }
      
      console.log('\n[Saga] Transaction completed successfully ✅\n');
      
      return { success: true };
    } catch (error) {
      console.error(`\n[Saga] Transaction failed: ${error.message}`);
      console.log('[Saga] Starting compensation...\n');
      
      // 反向执行补偿
      for (let i = executedSteps.length - 1; i >= 0; i--) {
        const step = executedSteps[i];
        
        try {
          console.log(`[Saga] Compensating: ${step.name}`);
          
          await step.compensation();
          
          console.log(`  ✅ ${step.name} compensated`);
        } catch (e) {
          console.error(`  ❌ ${step.name} compensation failed:`, e.message);
        }
      }
      
      console.log('\n[Saga] Transaction rolled back ❌\n');
      
      return { success: false, error: error.message };
    }
  }
}

// 示例：电商下单流程
import Redis from 'ioredis';

async function testSaga() {
  const redis = new Redis();
  
  // 初始化数据
  await redis.set('order:status', 'pending');
  await redis.set('inventory:SKU001', 100);
  await redis.set('balance:user123', 1000);
  
  const saga = new SagaOrchestrator();
  
  // 步骤1：创建订单
  saga.addStep(
    'CreateOrder',
    async () => {
      const orderId = `ORD${Date.now()}`;
      await redis.set(`order:${orderId}`, JSON.stringify({
        id: orderId,
        userId: 'user123',
        productId: 'SKU001',

        status: 'created',
      }));
      await redis.set('order:currentId', orderId);
    },
    async () => {
      const orderId = await redis.get('order:currentId');
      await redis.del(`order:${orderId}`);
      console.log('    Deleted order');
    }
  );
  
  // 步骤2：扣减库存
  saga.addStep(
    'DeductInventory',
    async () => {
      const stock = await redis.get('inventory:SKU001');
      
      if (parseInt(stock) < 10) {
        throw new Error('Insufficient stock');
      }
      
      await redis.decrby('inventory:SKU001', 10);
    },
    async () => {
      await redis.incrby('inventory:SKU001', 10);

    }
  );
  
  // 步骤3：扣减余额
  saga.addStep(
    'DeductBalance',
    async () => {
      const balance = await redis.get('balance:user123');
      
      if (parseInt(balance) < 100) {
        throw new Error('Insufficient balance');
      }
      
      await redis.decrby('balance:user123', 100);
    },
    async () => {
      await redis.incrby('balance:user123', 100);

    }
  );
  
  // 步骤4：发送通知
  saga.addStep(
    'SendNotification',
    async () => {
      // 模拟发送邮件/短信
      console.log('    Sent notification');
    },
    async () => {
      // 补偿：发送取消通知
      console.log('    Sent cancellation notification');
    }
  );
  
  // 执行 Saga
  const result = await saga.execute();
  
  console.log('Saga result:', result);
  
  // 查看最终状态
  console.log('\n=== Final State ===');
  console.log('Inventory:', await redis.get('inventory:SKU001'));
  console.log('Balance:', await redis.get('balance:user123'));
  
  await redis.quit();
}
```

### 实战4：基于消息队列的最终一致性

**原理：本地消息表 + 定时任务**

```javascript
// mq-eventual-consistency.js
import { Client as PgClient } from 'pg';
import amqp from 'amqplib';

class LocalMessageTable {
  constructor() {
    this.db = new PgClient({
      host: 'localhost',
      port: 5432,
      user: 'postgres',
      password: 'secret123',
      database: 'distributed_demo',
    });
  }
  
  async init() {
    await this.db.connect();
    
    // 创建本地消息表
    await this.db.query(`
      CREATE TABLE IF NOT EXISTS local_messages (
        id SERIAL PRIMARY KEY,
        topic VARCHAR(255),
        content TEXT,
        status VARCHAR(50),
        retry_count INT DEFAULT 0,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
      )
    `);
  }
  
  // 执行业务 + 保存消息（同一事务）
  async executeWithMessage(businessSql, businessParams, topic, message) {
    try {
      await this.db.query('BEGIN');
      
      // 1. 执行业务逻辑
      await this.db.query(businessSql, businessParams);
      
      // 2. 保存消息到本地表（同一事务）
      await this.db.query(
        'INSERT INTO local_messages (topic, content, status) VALUES ($1, $2, $3)',
        [topic, JSON.stringify(message), 'PENDING']
      );
      
      await this.db.query('COMMIT');
      
      console.log('✅ Business logic and message saved');
      
      return { success: true };
    } catch (error) {
      await this.db.query('ROLLBACK');
      console.error('❌ Transaction failed:', error.message);
      return { success: false, error: error.message };
    }
  }
  
  // 定时任务：发送未发送的消息
  async sendPendingMessages() {
    const result = await this.db.query(
      'SELECT * FROM local_messages WHERE status = \'PENDING\' AND retry_count < 3 LIMIT 10'
    );
    
    for (const row of result.rows) {
      try {
        // 发送消息到 MQ
        await this.sendToMQ(row.topic, row.content);
        
        // 标记为已发送
        await this.db.query(
          'UPDATE local_messages SET status = $1, updated_at = NOW() WHERE id = $2',
          ['SENT', row.id]
        );
        
        console.log(`✅ Message ${row.id} sent to MQ`);
      } catch (error) {
        // 重试次数+1
        await this.db.query(
          'UPDATE local_messages SET retry_count = retry_count + 1, updated_at = NOW() WHERE id = $1',
          [row.id]
        );
        
        console.error(`❌ Failed to send message ${row.id}:`, error.message);
      }
    }
  }
  
  async sendToMQ(topic, content) {
    const connection = await amqp.connect('amqp://localhost');
    const channel = await connection.createChannel();
    
    await channel.assertQueue(topic, { durable: true });
    channel.sendToQueue(topic, Buffer.from(content), { persistent: true });
    
    await channel.close();
    await connection.close();
  }
}

// 测试
async function testLocalMessageTable() {
  const service = new LocalMessageTable();
  await service.init();
  
  // 创建订单 + 发送消息（同一事务）
  await service.executeWithMessage(
    'INSERT INTO orders (user_id, product_id, amount) VALUES ($1, $2, $3)',
    ['user123', 'SKU001', 100],
    'order.created',
    { orderId: 'ORD123', userId: 'user123', amount: 100 }
  );
  
  // 模拟定时任务（每隔5秒执行一次）
  setInterval(async () => {
    await service.sendPendingMessages();
  }, 5000);
}

testLocalMessageTable();
```

### 场景1：电商下单（TCC）

```

  - 冻结库存

  - 真正扣减库存
  - 真正扣减积分
```

```
  - 银行A：提交事务
  - 银行B：提交事务
```

## ❌ 高频错误

### 错误1：忘记实现幂等性

**错误：**
```javascript
// ❌ Confirm/Cancel 未实现幂等
async confirm(txId) {
  await this.redis.decrby('stock', 10);  // 重复调用会多次扣减！
}
```

**正确：**
```javascript
// ✅ 使用事务ID判断是否已执行
async confirm(txId) {
  const executed = await this.redis.get(`tcc:${txId}:confirmed`);
  
  if (executed) {
    console.log('Already confirmed, skipping');
    return;
  }
  
  await this.redis.decrby('stock', 10);
  await this.redis.set(`tcc:${txId}:confirmed`, '1');
}
```

## 🤔 FAQ

### Q1：如何选择分布式事务方案？

**选择决策树：**
```
是否需要强一致性？
├─ 是 → 是否跨数据库？
│         ├─ 是 → 2PC（性能差）或 TCC（性能好）
│         └─ 否 → 本地事务
│
└─ 否 → 是否可接受补偿逻辑？
          ├─ 是 → Saga（长流程）或消息队列（异步）
          └─ 否 → 重新考虑需求
```

### 核心要点

1. **2PC**：强一致性，性能差，有单点故障
2. **TCC**：强一致性，性能好，实现复杂
3. **Saga**：最终一致性，适合长流程
4. **消息队列**：最终一致性，异步解耦
