# 6.7 健康检查与优雅关闭

> 确保服务稳定性：探针配置与信号处理

---

## 📚 知识导读

### 为什么需要健康检查？

**没有健康检查的问题：**

```
场景：应用启动缓慢（需要10秒初始化）

1. Pod 启动
2. Kubernetes 立即将流量发送到 Pod
3. 应用尚未准备好 → 请求失败 ❌
4. 用户看到 502/503 错误

场景：应用内部异常（数据库连接断开）

1. 应用运行中
2. 数据库连接断开
3. Kubernetes 不知道应用异常
4. 继续发送流量 → 所有请求失败 ❌
```

**有健康检查的解决方案：**

```
┌────────────────────────────────────────────────────────┐
│           Kubernetes 三种探针                           │
├────────────────────────────────────────────────────────┤
│                                                        │
│  1. Startup Probe（启动探针）                          │
│     - 检查应用是否启动完成                             │
│     - 启动期间不执行其他探针                           │
│     - 失败 → 重启 Pod                                  │
│                                                        │
│  2. Liveness Probe（存活探针）                         │
│     - 检查应用是否健康运行                             │
│     - 失败 → 重启 Pod                                  │
│     - 用于检测死锁、无限循环等                         │
│                                                        │
│  3. Readiness Probe（就绪探针）                        │
│     - 检查应用是否准备好接收流量                       │
│     - 失败 → 从 Service 移除，不接收流量               │
│     - 成功 → 重新加入 Service，接收流量                │
│                                                        │
└────────────────────────────────────────────────────────┘
```

## 🎯 探针配置

### 1. HTTP 探针

**最常用的探针类型：**

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: my-app
spec:
  containers:
  - name: app
    image: my-app:1.0.0
    ports:
    - containerPort: 3000
    
    # 启动探针
    startupProbe:
      httpGet:
        path: /health/startup
        port: 3000
      initialDelaySeconds: 0
      periodSeconds: 5
      failureThreshold: 30  # 最多等待 150 秒（5×30）
    
    # 存活探针
    livenessProbe:
      httpGet:
        path: /health/live
        port: 3000
      initialDelaySeconds: 30
      periodSeconds: 10
      timeoutSeconds: 5
      successThreshold: 1
      failureThreshold: 3  # 失败 3 次后重启
    
    # 就绪探针
    readinessProbe:
      httpGet:
        path: /health/ready
        port: 3000
      initialDelaySeconds: 5
      periodSeconds: 5
      timeoutSeconds: 3
      successThreshold: 1
      failureThreshold: 2  # 失败 2 次后从 Service 移除
```

**Node.js 健康检查端点：**

```javascript
// health.js
import express from 'express';

const app = express();

let isReady = false;
let isAlive = true;

// 启动探针
app.get('/health/startup', (req, res) => {
  // 检查应用是否启动完成
  if (isReady) {
    res.status(200).json({ status: 'started' });
  } else {
    res.status(503).json({ status: 'starting' });
  }
});

// 存活探针
app.get('/health/live', (req, res) => {
  if (isAlive) {
    res.status(200).json({ status: 'alive' });
  } else {
    res.status(500).json({ status: 'dead' });
  }
});

// 就绪探针
app.get('/health/ready', async (req, res) => {
  try {
    // 检查数据库连接
    await db.ping();
    
    // 检查 Redis 连接
    await redis.ping();
    
    // 检查外部依赖
    // ...
    
    res.status(200).json({
      status: 'ready',
      checks: {
        database: 'ok',
        redis: 'ok',
      },
    });
  } catch (error) {
    res.status(503).json({
      status: 'not ready',
      error: error.message,
    });
  }
});

// 模拟启动过程
setTimeout(() => {
  isReady = true;
  console.log('Application is ready');
}, 10000);  // 10 秒后就绪

app.listen(3000);
```

      if [ -f /tmp/healthcheck ] && [ $(( $(date +%s) - $(stat -c %Y /tmp/healthcheck) )) -gt 300 ]; then

      fi

  periodSeconds: 10

## 💻 健康检查最佳实践

### 1. 分层健康检查

```javascript
// health-checker.js
import { createClient } from 'redis';
import mongoose from 'mongoose';

class HealthChecker {
  constructor() {
    this.checks = {
      database: null,
      redis: null,
      externalAPI: null,
    };
  }

  // 轻量级健康检查（Liveness）
  async checkLiveness() {
    // 只检查进程是否响应
    return {
      status: 'ok',
      uptime: process.uptime(),
      timestamp: new Date().toISOString(),
    };
  }

  // 完整健康检查（Readiness）
  async checkReadiness() {
    const results = {
      status: 'ok',
      checks: {},
      timestamp: new Date().toISOString(),
    };

    // 检查数据库
    try {
      await mongoose.connection.db.admin().ping();
      results.checks.database = 'ok';
    } catch (error) {
      results.status = 'degraded';
      results.checks.database = 'error';
      results.error = error.message;
    }

    // 检查 Redis
    try {
      const redis = createClient({ url: process.env.REDIS_URL });
      await redis.connect();
      await redis.ping();
      await redis.disconnect();
      results.checks.redis = 'ok';
    } catch (error) {
      results.status = 'degraded';
      results.checks.redis = 'error';
    }

    // 检查外部 API（可选）
    try {
      const response = await fetch('https://api.example.com/health', {
        timeout: 3000,
      });
      results.checks.externalAPI = response.ok ? 'ok' : 'error';
    } catch (error) {
      // 外部 API 失败不影响就绪状态
      results.checks.externalAPI = 'warning';
    }

    return results;
  }

  // 启动检查
  async checkStartup() {
    // 检查是否完成初始化
    if (!global.appInitialized) {
      throw new Error('Application not initialized');
    }

    return { status: 'started' };
  }
}

export default new HealthChecker();
```

**使用：**

```javascript
import express from 'express';
import healthChecker from './health-checker.js';

const app = express();

app.get('/health/startup', async (req, res) => {
  try {
    const result = await healthChecker.checkStartup();
    res.json(result);
  } catch (error) {
    res.status(503).json({ status: 'starting', error: error.message });
  }
});

app.get('/health/live', async (req, res) => {
  const result = await healthChecker.checkLiveness();
  res.json(result);
});

app.get('/health/ready', async (req, res) => {
  try {
    const result = await healthChecker.checkReadiness();
    
    if (result.status === 'ok') {
      res.json(result);
    } else {
      res.status(503).json(result);
    }
  } catch (error) {
    res.status(503).json({
      status: 'not ready',
      error: error.message,
    });
  }
});
```

### 2. 探针参数调优

**慢启动应用（如 Java）：**

```yaml
startupProbe:
  httpGet:
    path: /health/startup
    port: 8080
  failureThreshold: 30  # 30 次失败
  periodSeconds: 10     # 每 10 秒检查一次
  # 最多等待 300 秒（5 分钟）
```

**快速响应应用（如 Node.js）：**

```yaml
startupProbe:
  httpGet:
    path: /health/startup
    port: 3000
  failureThreshold: 10  # 10 次失败
  periodSeconds: 2      # 每 2 秒检查一次
  # 最多等待 20 秒
```

**高可用服务（快速检测故障）：**

```yaml
livenessProbe:
  httpGet:
    path: /health/live
    port: 3000
  periodSeconds: 5      # 频繁检查
  timeoutSeconds: 2     # 快速超时
  failureThreshold: 2   # 2 次失败立即重启
```

## 🎯 优雅关闭

### 为什么需要优雅关闭？

**问题场景：**

```
1. Kubernetes 发送 SIGTERM 信号
2. Node.js 进程立即退出
3. 正在处理的请求被中断 ❌
4. 用户看到错误

正确流程：
1. Kubernetes 发送 SIGTERM 信号
2. 应用停止接收新请求
3. 等待现有请求完成
4. 关闭数据库连接
5. 清理资源
6. 进程退出 ✅
```

---

### 1. Express 优雅关闭

```javascript
// server.js
import express from 'express';
import mongoose from 'mongoose';
import { createClient } from 'redis';

const app = express();
const PORT = 3000;

// 数据库连接
const db = await mongoose.connect(process.env.MONGODB_URI);
const redis = createClient({ url: process.env.REDIS_URL });
await redis.connect();

// 路由
app.get('/', (req, res) => {
  res.json({ message: 'Hello World' });
});

// 启动服务器
const server = app.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`);
});

// 优雅关闭函数
async function gracefulShutdown(signal) {
  console.log(`\n${signal} received, shutting down gracefully...`);

  // 1. 停止接收新请求
  server.close(async () => {
    console.log('HTTP server closed');

    try {
      // 2. 关闭数据库连接
      await mongoose.connection.close();
      console.log('MongoDB connection closed');

      // 3. 关闭 Redis 连接
      await redis.disconnect();
      console.log('Redis connection closed');

      // 4. 退出进程
      console.log('Graceful shutdown completed');
      process.exit(0);
    } catch (error) {
      console.error('Error during shutdown:', error);
      process.exit(1);
    }
  });

  // 5. 强制超时（30秒后强制退出）
  setTimeout(() => {
    console.error('Forced shutdown after timeout');
    process.exit(1);
  }, 30000);
}

// 监听信号
process.on('SIGTERM', () => gracefulShutdown('SIGTERM'));
process.on('SIGINT', () => gracefulShutdown('SIGINT'));

// 捕获未处理的异常
process.on('uncaughtException', (error) => {
  console.error('Uncaught Exception:', error);
  gracefulShutdown('UNCAUGHT_EXCEPTION');
});

process.on('unhandledRejection', (reason, promise) => {
  console.error('Unhandled Rejection at:', promise, 'reason:', reason);
  gracefulShutdown('UNHANDLED_REJECTION');
});
```

### 3. Kubernetes 优雅关闭配置

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  template:
    spec:
      # 优雅终止宽限期（默认 30 秒）
      terminationGracePeriodSeconds: 60
      
      containers:
      - name: app
        image: my-app:1.0.0
        
        # Prestart Hook（启动前执行）
        lifecycle:
          postStart:
            exec:
              command:
              - /bin/sh
              - -c
              - echo "Container started at $(date)" > /tmp/started
          
          # Prestop Hook（停止前执行）
          preStop:
            exec:
              command:
              - /bin/sh
              - -c
              - |
                echo "Received SIGTERM, waiting for graceful shutdown..."
                sleep 5
                # 给应用时间完成请求
```

**关闭流程：**

```
1. kubectl delete pod my-app-xxx
   ↓
2. Pod 状态变为 Terminating
   ↓
3. 从 Service Endpoints 移除（停止接收新流量）
   ↓
4. 执行 preStop Hook（可选）
   ↓
5. 发送 SIGTERM 信号给容器
   ↓
6. 等待 terminationGracePeriodSeconds（默认30秒）
   ↓
7. 如果进程未退出，发送 SIGKILL 强制杀死
```

### 4. 处理长时间任务

```javascript
// worker.js
import Bull from 'bull';

const queue = new Bull('tasks', process.env.REDIS_URL);

let isShuttingDown = false;
const activeJobs = new Set();

// 处理任务
queue.process(async (job) => {
  if (isShuttingDown) {
    throw new Error('Server is shutting down');
  }

  activeJobs.add(job.id);

  try {
    // 执行长时间任务
    await processTask(job.data);
    activeJobs.delete(job.id);
  } catch (error) {
    activeJobs.delete(job.id);
    throw error;
  }
});

// 优雅关闭
async function gracefulShutdown() {
  isShuttingDown = true;
  console.log('Shutting down worker...');

  // 1. 停止接收新任务
  await queue.pause();
  console.log('Queue paused');

  // 2. 等待当前任务完成（最多30秒）
  const timeout = 30000;
  const start = Date.now();

  while (activeJobs.size > 0 && Date.now() - start < timeout) {
    console.log(`Waiting for ${activeJobs.size} jobs to complete...`);
    await new Promise((resolve) => setTimeout(resolve, 1000));
  }

  if (activeJobs.size > 0) {
    console.warn(`${activeJobs.size} jobs did not complete in time`);
  }

  // 3. 关闭队列
  await queue.close();
  console.log('Queue closed');

  process.exit(0);
}

process.on('SIGTERM', gracefulShutdown);
process.on('SIGINT', gracefulShutdown);
```

## ⚠️ 三个高频错误

### 错误 1：Liveness 和 Readiness 使用同一个端点

**问题：** 依赖服务故障导致 Pod 不断重启。

```yaml
# ❌ 错误
livenessProbe:
  httpGet:
    path: /health  # 检查所有依赖
readinessProbe:
  httpGet:
    path: /health  # 同一个端点
```

**问题：** 如果数据库连接断开
- Readiness 失败 → Pod 从 Service 移除 ✅（正确）
- Liveness 也失败 → Pod 被重启 ❌（错误，重启无法解决外部问题）

**解决方案：** 使用不同的端点。

```yaml
# ✅ 正确
livenessProbe:
  httpGet:
    path: /health/live  # 只检查进程本身
readinessProbe:
  httpGet:
    path: /health/ready  # 检查所有依赖
```

---

### 错误 2：没有设置 terminationGracePeriodSeconds

**问题：** 默认 30 秒可能不够。

```yaml
# ❌ 错误：处理请求需要 60 秒，但只等待 30 秒
terminationGracePeriodSeconds: 30  # 默认值
```

**解决方案：** 根据实际需求设置。

```yaml
# ✅ 正确
terminationGracePeriodSeconds: 120  # 长时间任务需要更长时间
```

---

### 错误 3：忘记处理 SIGTERM

**问题：** Node.js 默认不处理 SIGTERM，导致强制杀死。

```javascript
// ❌ 错误：没有 SIGTERM 处理
const server = app.listen(3000);
// Kubernetes 发送 SIGTERM → 30秒后强制 SIGKILL
```

**解决方案：** 监听 SIGTERM。

```javascript
// ✅ 正确
const server = app.listen(3000);

process.on('SIGTERM', () => {
  server.close(() => {
    process.exit(0);
  });
});
```

### 2. preStop Hook vs SIGTERM 如何选择？

| 场景 | 推荐方式 |
|------|---------|
| 简单关闭逻辑 | SIGTERM（在应用中处理） |
| 需要外部命令 | preStop Hook |
| 需要通知其他服务 | preStop Hook |
| 需要等待外部资源 | preStop Hook |

# 查看日志确认
