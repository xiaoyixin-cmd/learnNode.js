## 📚 知识导读

### 什么是 Kubernetes？

**Kubernetes（K8s）** 是Google开源的容器编排平台，用于自动化部署、扩展和管理容器化应用程序。

**Docker vs Kubernetes：**

```
┌────────────────────────────────────────────────────────┐
│                Docker（单机）                           │
├────────────────────────────────────────────────────────┤
│                                                        │
│  docker run -d --name app1 -p 3000:3000 my-app        │
│  docker run -d --name app2 -p 3001:3000 my-app        │
│  docker run -d --name app3 -p 3002:3000 my-app        │
│                                                        │
│  ❌ 手动管理多个容器                                   │
│  ❌ 单点故障（服务器宕机 = 服务不可用）                │
│  ❌ 手动扩缩容                                         │
│  ❌ 无法跨服务器部署                                   │
│  ❌ 无统一的负载均衡                                   │
│                                                        │
└────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────┐
│           Kubernetes（集群）                            │
├────────────────────────────────────────────────────────┤
│                                                        │
│  kubectl apply -f deployment.yaml                     │
│                                                        │
│  ✅ 自动管理容器（声明式配置）                         │
│  ✅ 自动故障恢复（Pod 崩溃自动重启）                   │
│  ✅ 自动扩缩容（HPA 根据 CPU/内存自动调整）            │
│  ✅ 跨节点部署（多台服务器组成集群）                   │
│  ✅ 内置负载均衡（Service）                            │
│  ✅ 滚动更新、零停机部署                               │
│  ✅ 服务发现、配置管理                                 │
│                                                        │
└────────────────────────────────────────────────────────┘
```

### Kubernetes 核心概念

```
┌─────────────────────────────────────────────────────────┐
│              Kubernetes 架构                             │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Master 节点（控制平面）                                │
│  ┌──────────────────────────────────────────────────┐  │
│  │  - API Server（API 入口）                        │  │
│  │  - Scheduler（调度 Pod 到节点）                  │  │
│  │  - Controller Manager（管理控制器）              │  │
│  │  - etcd（存储集群状态）                          │  │
│  └──────────────────────────────────────────────────┘  │
│                        ↓                                │
│  Worker 节点（工作节点）                                │
│  ┌──────────────────────────────────────────────────┐  │
│  │  Node 1:                                         │  │
│  │  ├─ Pod 1 (容器1, 容器2)                         │  │
│  │  ├─ Pod 2 (容器3)                                │  │
│  │  └─ kubelet（节点代理）                          │  │
│  └──────────────────────────────────────────────────┘  │
│  ┌──────────────────────────────────────────────────┐  │
│  │  Node 2:                                         │  │
│  │  ├─ Pod 3 (容器4, 容器5)                         │  │
│  │  └─ kubelet                                      │  │
│  └──────────────────────────────────────────────────┘  │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**核心资源对象：**

| 资源 | 作用 | 类比 |
|------|------|------|
| **Pod** | 最小部署单元，包含1个或多个容器 | 一个虚拟机 |
| **Deployment** | 管理 Pod 的创建、更新、回滚 | 进程管理器（PM2） |
| **Service** | 为 Pod 提供稳定的网络访问 | 负载均衡器（Nginx） |
| **Ingress** | HTTP(S) 路由规则 | 反向代理（Nginx） |
| **ConfigMap** | 存储配置文件 | .env 文件 |
| **Secret** | 存储敏感信息（密码、密钥） | 加密的 .env 文件 |
| **HPA** | 自动扩缩容 | Auto Scaling |

## 🎯 Deployment：声明式部署

### 1. 基础 Deployment

```yaml
# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
  labels:
    app: my-app
spec:
  # 副本数量
  replicas: 3
  
  # 选择器（匹配 Pod 标签）
  selector:
    matchLabels:
      app: my-app
  
  # Pod 模板
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: app
        image: my-app:1.0.0
        ports:
        - containerPort: 3000
        env:
        - name: NODE_ENV
          value: "production"
        - name: PORT
          value: "3000"
        
        # 资源限制
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
```

**部署应用：**

```bash
# 应用 Deployment
kubectl apply -f deployment.yaml

# 查看 Deployment
kubectl get deployments
# NAME     READY   UP-TO-DATE   AVAILABLE   AGE
# my-app   3/3     3            3           1m

# 查看 Pod
kubectl get pods
# NAME                      READY   STATUS    RESTARTS   AGE
# my-app-5d5c8f9b4d-7x8qz   1/1     Running   0          1m
# my-app-5d5c8f9b4d-9k2l3   1/1     Running   0          1m
# my-app-5d5c8f9b4d-xr5w4   1/1     Running   0          1m

# 查看详细信息
kubectl describe deployment my-app

# 查看 Pod 日志
kubectl logs my-app-5d5c8f9b4d-7x8qz

# 进入 Pod 调试
kubectl exec -it my-app-5d5c8f9b4d-7x8qz -- sh
```

### 2. 滚动更新（Rolling Update）

**更新镜像版本：**

```bash
# 方法 1：直接修改 YAML 文件
# 修改 deployment.yaml 中的 image: my-app:1.1.0
kubectl apply -f deployment.yaml

# 方法 2：使用 kubectl set image
kubectl set image deployment/my-app app=my-app:1.1.0

# 查看滚动更新状态
kubectl rollout status deployment/my-app
# Waiting for deployment "my-app" rollout to finish: 1 out of 3 new replicas have been updated...
# deployment "my-app" successfully rolled out

# 查看滚动更新历史
kubectl rollout history deployment/my-app
# REVISION  CHANGE-CAUSE
# 1         <none>
# 2         kubectl set image deployment/my-app app=my-app:1.1.0
```

**滚动更新策略配置：**

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 10
  
  # 滚动更新策略
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 2        # 最多可以多创建 2 个 Pod
      maxUnavailable: 1  # 最多允许 1 个 Pod 不可用
  
  template:
    spec:
      containers:
      - name: app
        image: my-app:1.1.0
```

**更新过程：**

```
初始状态：10 个 Pod (v1.0.0)

步骤 1：创建 2 个新 Pod (v1.1.0)    - 总计 12 个 Pod
步骤 2：删除 1 个旧 Pod (v1.0.0)    - 总计 11 个 Pod
步骤 3：再创建 1 个新 Pod (v1.1.0)  - 总计 12 个 Pod
步骤 4：删除 1 个旧 Pod (v1.0.0)    - 总计 11 个 Pod
...
最终状态：10 个 Pod (v1.1.0)

优势：
✅ 零停机部署
✅ 逐步替换（降低风险）
✅ 随时可回滚
```

## 🎯 Service：服务发现与负载均衡

### 1. ClusterIP（集群内部访问）

**默认类型，只能在集群内部访问。**

```yaml
# service-clusterip.yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app-service
spec:
  type: ClusterIP  # 默认类型
  selector:
    app: my-app    # 选择标签为 app=my-app 的 Pod
  ports:
  - protocol: TCP
    port: 80       # Service 端口
    targetPort: 3000  # Pod 端口
```

**应用 Service：**

```bash
kubectl apply -f service-clusterip.yaml

# 查看 Service
kubectl get services
# NAME              TYPE        CLUSTER-IP      EXTERNAL-IP   PORT(S)   AGE
# my-app-service    ClusterIP   10.96.123.45    <none>        80/TCP    1m

# 在集群内部访问（临时创建一个 Pod 测试）
kubectl run curl --image=curlimages/curl -i --rm --restart=Never -- \
  curl http://my-app-service.default.svc.cluster.local
```

---

### 2. NodePort（外部访问）

**在每个节点上开放一个固定端口（30000-32767）。**

```yaml
# service-nodeport.yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app-nodeport
spec:
  type: NodePort
  selector:
    app: my-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 3000
    nodePort: 30080  # 可选，不指定则自动分配
```

**访问方式：**

```bash
# 获取节点 IP
kubectl get nodes -o wide
# NAME       STATUS   ROLES    AGE   VERSION   INTERNAL-IP
# minikube   Ready    master   1d    v1.29.0   192.168.49.2

# 访问服务
curl http://192.168.49.2:30080
```

### 2. 基础 Ingress 配置

```yaml
# ingress.yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: my-app-ingress
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
spec:
  ingressClassName: nginx
  rules:
  - host: api.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: my-app-service
            port:
              number: 80
```

### 4. HTTPS 配置（自动证书）

**使用 cert-manager 自动管理 Let's Encrypt 证书。**

**安装 cert-manager：**

```bash
kubectl apply -f https://github.com/cert-manager/cert-manager/releases/download/v1.14.0/cert-manager.yaml

# 验证安装
kubectl get pods -n cert-manager
```

**创建 ClusterIssuer：**

```yaml
# cluster-issuer.yaml
apiVersion: cert-manager.io/v1
kind: ClusterIssuer
metadata:
  name: letsencrypt-prod
spec:
  acme:
    server: https://acme-v02.api.letsencrypt.org/directory
    email: your-email@example.com
    privateKeySecretRef:
      name: letsencrypt-prod
    solvers:
    - http01:
        ingress:
          class: nginx
```

**配置 Ingress 使用 HTTPS：**

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: my-app-ingress
  annotations:
    cert-manager.io/cluster-issuer: "letsencrypt-prod"
    nginx.ingress.kubernetes.io/ssl-redirect: "true"
spec:
  ingressClassName: nginx
  tls:
  - hosts:
    - api.example.com
    secretName: api-example-com-tls  # 自动创建的证书 Secret
  rules:
  - host: api.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: my-app-service
            port:
              number: 80
```

**应用后，cert-manager 会自动：**
1. 向 Let's Encrypt 申请证书
2. 创建 Secret 存储证书
3. 自动续期（证书到期前 30 天）

## 🎯 ConfigMap 与 Secret

### 1. ConfigMap（配置文件）

**创建 ConfigMap：**

```yaml
# configmap.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: my-app-config
data:
  # 键值对
  DATABASE_HOST: "postgres.default.svc.cluster.local"
  DATABASE_PORT: "5432"
  LOG_LEVEL: "info"
  
  # 文件内容
  app.conf: |
    server {
      port: 3000
      timeout: 30s
    }
    
    database {
      maxConnections: 100
    }
```

**在 Pod 中使用 ConfigMap：**

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: app
        image: my-app:1.0.0
        
        # 方法 1：作为环境变量
        env:
        - name: DATABASE_HOST
          valueFrom:
            configMapKeyRef:
              name: my-app-config
              key: DATABASE_HOST
        - name: DATABASE_PORT
          valueFrom:
            configMapKeyRef:
              name: my-app-config
              key: DATABASE_PORT
        
        # 方法 2：挂载为文件
        volumeMounts:
        - name: config-volume
          mountPath: /etc/config
      
      volumes:
      - name: config-volume
        configMap:
          name: my-app-config
```

**访问配置文件：**

```javascript
// Node.js 代码
import fs from 'fs';

// 读取挂载的配置文件
const config = fs.readFileSync('/etc/config/app.conf', 'utf8');
console.log(config);

// 或使用环境变量
const dbHost = process.env.DATABASE_HOST;
const dbPort = process.env.DATABASE_PORT;
```

### 2. Secret（敏感信息）

```yaml
apiVersion: v1

  JWT_SECRET: and0LXNlY3JldC1rZXk=
```

## 🎯 HPA：水平自动扩缩容

### 1. 安装 Metrics Server

**Metrics Server 收集集群资源使用情况（CPU、内存）。**

```bash
kubectl apply -f https://github.com/kubernetes-sigs/metrics-server/releases/latest/download/components.yaml

# 如果是本地环境（Minikube），启用 insecure TLS
kubectl patch deployment metrics-server -n kube-system --type='json' \
  -p='[{"op": "add", "path": "/spec/template/spec/containers/0/args/-", "value": "--kubelet-insecure-tls"}]'

# 验证
kubectl top nodes
kubectl top pods
```

---

### 2. 配置 HPA

```yaml
# hpa.yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: my-app-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: my-app
  minReplicas: 2      # 最小副本数
  maxReplicas: 10     # 最大副本数
  metrics:
  # CPU 目标利用率 70%
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
  # 内存目标利用率 80%
  - type: Resource
    resource:
      name: memory
      target:
        type: Utilization
        averageUtilization: 80
  
  # 扩缩容行为
  behavior:
    scaleDown:
      stabilizationWindowSeconds: 300  # 缩容稳定窗口（5分钟）
      policies:
      - type: Percent
        value: 50        # 每次最多缩容 50%
        periodSeconds: 60
    scaleUp:
      stabilizationWindowSeconds: 0  # 扩容立即生效
      policies:
      - type: Percent
        value: 100       # 每次最多扩容 100%
        periodSeconds: 15
      - type: Pods
        value: 4         # 或每次最多增加 4 个 Pod
        periodSeconds: 15
      selectPolicy: Max  # 选择扩容最快的策略
```

**应用 HPA：**

```bash
kubectl apply -f hpa.yaml

# 查看 HPA 状态
kubectl get hpa
# NAME          REFERENCE           TARGETS         MINPODS   MAXPODS   REPLICAS   AGE
# my-app-hpa    Deployment/my-app   45%/70%         2         10        3          5m

# 查看详细信息
kubectl describe hpa my-app-hpa
```

### 1. Deployment

```yaml
# k8s/deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
  labels:
    app: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      # 安全上下文
      securityContext:
        runAsNonRoot: true
        runAsUser: 1001
        fsGroup: 1001
      
      containers:
      - name: app
        image: my-app:1.0.0
        ports:
        - containerPort: 3000
          name: http
        
        # 环境变量
        env:
        - name: NODE_ENV
          value: "production"
        - name: PORT
          value: "3000"
        - name: DATABASE_HOST
          valueFrom:
            configMapKeyRef:
              name: my-app-config
              key: DATABASE_HOST
        - name: DATABASE_PASSWORD
          valueFrom:
            secretKeyRef:
              name: my-app-secret
              key: DATABASE_PASSWORD
        
        # 资源限制
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
        
        # 健康检查
        livenessProbe:
          httpGet:
            path: /health
            port: 3000
          initialDelaySeconds: 30
          periodSeconds: 10
        
        readinessProbe:
          httpGet:
            path: /health/ready
            port: 3000
          initialDelaySeconds: 5
          periodSeconds: 5
        
        # 容器安全上下文
        securityContext:
          allowPrivilegeEscalation: false
          readOnlyRootFilesystem: true
          capabilities:
            drop: [ALL]
        
        # 挂载数据卷
        volumeMounts:
        - name: tmp
          mountPath: /tmp
        - name: logs
          mountPath: /app/logs
      
      volumes:
      - name: tmp
        emptyDir: {}
      - name: logs
        emptyDir: {}
```

## ⚠️ 三个高频错误

### 错误 1：忘记设置资源限制

**问题：** Pod 可能消耗所有节点资源，导致其他 Pod 无法调度。

```yaml
# ❌ 错误
containers:
- name: app
  image: my-app:1.0.0
  # 缺少 resources
```

**解决方案：** 始终设置 requests 和 limits。

```yaml
# ✅ 正确
containers:
- name: app
  image: my-app:1.0.0
  resources:
    requests:
      memory: "256Mi"
      cpu: "250m"
    limits:
      memory: "512Mi"
      cpu: "500m"
```

---

### 错误 2：Service Selector 不匹配

**问题：** Service 无法找到 Pod。

```yaml
# Deployment
metadata:
  labels:
    app: my-app  # ❌ 标签错误

# Service
spec:
  selector:
    app: myapp  # 不匹配！
```

**调试：**

```bash
kubectl describe service my-app-service
# Endpoints: <none>  # 没有找到匹配的 Pod
```

**解决方案：** 确保 Selector 与 Pod 标签完全匹配。

---

### 错误 3：镜像拉取失败

**问题：** 私有镜像仓库认证失败。

```bash
kubectl get pods
# NAME                      READY   STATUS         RESTARTS   AGE
# my-app-5d5c8f9b4d-7x8qz   0/1     ImagePullBackOff   0      1m
```

**解决方案：** 创建 Docker Registry Secret。

```bash
# 创建 Secret
kubectl create secret docker-registry regcred \
  --docker-server=registry.example.com \
  --docker-username=your-username \
  --docker-password=your-password \
  --docker-email=your-email@example.com

# 在 Deployment 中引用
spec:
  template:
    spec:
      imagePullSecrets:
      - name: regcred
```

## 📚 FAQ

### 1. 如何选择 Service 类型？

| 场景 | 推荐类型 |
|------|---------|
| 集群内部通信 | ClusterIP |
| 开发/测试环境 | NodePort |
| 生产环境 | Ingress + ClusterIP |
| 云环境负载均衡 | LoadBalancer |

### 2. 如何调试 Pod 启动失败？

```bash
# 查看 Pod 状态
kubectl get pods

# 查看 Pod 详细信息
kubectl describe pod my-app-xxxxx

# 查看 Pod 日志
kubectl logs my-app-xxxxx

# 查看之前崩溃的日志
kubectl logs my-app-xxxxx --previous

# 进入 Pod 调试
kubectl exec -it my-app-xxxxx -- sh
```

### 3. 如何实现零停机部署？

**关键配置：**
1. 配置 Readiness Probe（确保 Pod 就绪后才接收流量）
2. 配置合理的滚动更新策略（maxSurge、maxUnavailable）
3. 配置优雅关闭（SIGTERM 处理）
