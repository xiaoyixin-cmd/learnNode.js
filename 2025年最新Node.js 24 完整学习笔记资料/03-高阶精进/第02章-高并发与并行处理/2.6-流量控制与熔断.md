## 📖 一、理论知识

### 1.1 什么是熔断器（Circuit Breaker）？

**熔断器** 是一种保护机制，当依赖服务故障率超过阈值时，自动切断请求，避免级联故障。

```
类比：家用电路断路器

正常状态（Closed）:
电流正常 → 断路器闭合 → 电器工作

故障状态（Open）:
短路/过载 → 断路器跳闸 → 切断电流 → 保护电器

恢复状态（Half-Open）:
等待一段时间 → 尝试送电 → 成功则恢复 → 失败则继续断开
```

### 1.2 熔断器三种状态

```
┌───────────────────────────────────────────┐
│        Circuit Breaker 状态机            │
├───────────────────────────────────────────┤
│                                           │
│  ┌─────────────┐                          │
│  │   Closed    │ ← 正常状态，请求通过     │
│  │  （闭合）    │                          │
│  └──────┬──────┘                          │
│         │ 失败率 > 阈值                    │
│         ↓                                 │
│  ┌─────────────┐                          │
│  │    Open     │ ← 熔断状态，拒绝请求     │
│  │  （打开）    │                          │
│  └──────┬──────┘                          │
│         │ 等待超时                         │
│         ↓                                 │
│  ┌─────────────┐                          │
│  │  Half-Open  │ ← 探测状态，尝试恢复     │
│  │ （半开）     │                          │
│  └──────┬──────┘                          │
│         │ 成功 → Closed                    │
│         │ 失败 → Open                      │
└───────────────────────────────────────────┘
```

| 状态 | 行为 | 触发条件 |
|-----|------|---------|
| **Closed（闭合）** | 请求正常通过 | 初始状态或恢复后 |
| **Open（打开）** | 快速失败，返回降级响应 | 失败率 > 阈值（如 50%） |
| **Half-Open（半开）** | 允许少量请求探测 | Open 状态超时后 |

### 1.3 为什么需要熔断？

**级联故障场景**：

```
用户请求 → API网关 → 服务A → 服务B（故障）

无熔断：
1. 服务B 响应慢（超时 30 秒）
2. 服务A 等待服务B → 线程堆积
3. 服务A 资源耗尽 → 崩溃
4. API网关 等待服务A → 崩溃
5. 整个系统崩溃 ❌

有熔断：
1. 服务B 故障率 > 50%
2. 熔断器 Open → 快速失败（1ms）
3. 服务A 返回降级响应 → 继续运行 ✅
4. 系统保持可用
```

const CircuitBreaker = require('opossum');

const breaker = new CircuitBreaker(unreliableService, {

breaker.on('open', () => {
  console.log('⚠️ 熔断器打开');

breaker.on('halfOpen', () => {
  console.log('🔄 熔断器半开，尝试恢复');

breaker.on('close', () => {
  console.log('✅ 熔断器关闭，服务恢复');

  try {
    const result = await breaker.fire();

// db/circuitBreaker.js
const CircuitBreaker = require('opossum');
const mongoose = require('mongoose');

class DatabaseCircuitBreaker {
  constructor() {
    // 包装数据库查询
    const queryWrapper = async (model, method, ...args) => {
      return await model[method](...args);
    };
    
    this.breaker = new CircuitBreaker(queryWrapper, {
      timeout: 10000,
      errorThresholdPercentage: 60,
      resetTimeout: 60000,
      volumeThreshold: 10,  // 至少 10 个请求才触发熔断
      fallback: () => {
        throw new Error('Database circuit breaker open');
      }
    });
    
    this.setupMonitoring();
  }
  
  setupMonitoring() {
    this.breaker.on('open', () => {
      console.error('❌ 数据库熔断器打开');
      sendAlert({
        level: 'critical',
        message: 'Database circuit breaker opened',
        stats: this.breaker.stats
      });
    });
    
    this.breaker.on('halfOpen', () => {
      console.warn('🔄 数据库熔断器半开');
    });
    
    this.breaker.on('close', () => {
      console.log('✅ 数据库熔断器恢复');
    });
  }
  
  async query(model, method, ...args) {
    try {
      return await this.breaker.fire(model, method, ...args);
    } catch (error) {
      if (this.breaker.opened) {
        // 熔断器打开，返回缓存或默认值
        return this.getFromCache(model, method, args);
      }
      throw error;
    }
  }
  
  getFromCache(model, method, args) {
    // 从 Redis 缓存读取
    console.log('从缓存读取数据');
    return null;
  }
}

module.exports = new DatabaseCircuitBreaker();

### 2.4 微服务调用熔断

```javascript
// services/userService.js
const CircuitBreaker = require('opossum');
const axios = require('axios');
const Redis = require('ioredis');

const redis = new Redis();

class UserService {
  constructor() {
    this.breakers = {
      authService: this.createBreaker('authService', 'http://auth-service:3001'),
      orderService: this.createBreaker('orderService', 'http://order-service:3002'),
      paymentService: this.createBreaker('paymentService', 'http://payment-service:3003')
    };
  }
  
  createBreaker(name, baseURL) {
    const callService = async (endpoint, options = {}) => {
      const response = await axios({
        baseURL,
        url: endpoint,
        ...options,
        timeout: 5000
      });
      return response.data;
    };
    
    const breaker = new CircuitBreaker(callService, {
      timeout: 5000,
      errorThresholdPercentage: 50,
      resetTimeout: 30000,
      volumeThreshold: 5,
      fallback: async (endpoint, options) => {
        // 尝试从缓存获取
        const cacheKey = `${name}:${endpoint}:${JSON.stringify(options)}`;
        const cached = await redis.get(cacheKey);
        
        if (cached) {
          console.log(`[${name}] 返回缓存数据`);
          return JSON.parse(cached);
        }
        
        throw new Error(`${name} circuit breaker open and no cache available`);
      }
    });
    
    breaker.on('success', async (result, latency) => {
      // 成功响应缓存到 Redis
      const cacheKey = `${name}:${result.endpoint}`;
      await redis.setex(cacheKey, 300, JSON.stringify(result));
    });
    
    breaker.on('open', () => {
      console.error(`⚠️ ${name} 熔断器打开`);
    });
    
    return breaker;
  }
  
  async getUserProfile(userId) {
    try {
      // 调用 Auth Service（带熔断保护）
      const authData = await this.breakers.authService.fire(`/users/${userId}`);
      
      // 调用 Order Service
      const orders = await this.breakers.orderService.fire(`/users/${userId}/orders`);
      
      return {
        ...authData,
        orders
      };
    } catch (error) {
      throw new Error(`Failed to get user profile: ${error.message}`);
    }
  }
  
  getStats() {
    const stats = {};
    Object.entries(this.breakers).forEach(([name, breaker]) => {
      stats[name] = {
        state: breaker.opened ? 'OPEN' : breaker.halfOpen ? 'HALF_OPEN' : 'CLOSED',
        ...breaker.stats
      };
    });
    return stats;
  }
}

module.exports = new UserService();
```

// ✅ 正确：合理的阈值
const breaker = new CircuitBreaker(service, {

  rollingCountTimeout: 10000  // 10 秒滚动窗口

## 🎯 四、2个应用场景

### 场景1：电商支付系统

#### 需求背景

- 对接多个支付渠道（支付宝、微信、银联）
- 支付渠道偶尔故障
- 需要快速切换到备用渠道

#### 完整实现

```javascript
// payment/circuitBreaker.js
const CircuitBreaker = require('opossum');
const Redis = require('ioredis');

const redis = new Redis();

class PaymentCircuitBreaker {
  constructor() {
    this.channels = {
      alipay: this.createChannelBreaker('支付宝', this.callAlipay),
      wechat: this.createChannelBreaker('微信支付', this.callWechat),
      unionpay: this.createChannelBreaker('银联', this.callUnionpay)
    };
  }
  
  createChannelBreaker(name, paymentFn) {
    const breaker = new CircuitBreaker(paymentFn, {
      timeout: 10000,
      errorThresholdPercentage: 50,
      resetTimeout: 60000,
      volumeThreshold: 5
    });
    
    breaker.on('open', () => {
      console.error(`❌ ${name}支付通道熔断`);
      sendAlert({
        level: 'critical',
        message: `${name} payment channel circuit breaker opened`,
        channel: name
      });
    });
    
    return breaker;
  }
  
  async callAlipay(orderData) {
    // 调用支付宝 API
    const response = await axios.post('https://api.alipay.com/pay', orderData);
    return response.data;
  }
  
  async callWechat(orderData) {
    // 调用微信支付 API
    const response = await axios.post('https://api.wechat.com/pay', orderData);
    return response.data;
  }
  
  async callUnionpay(orderData) {
    // 调用银联 API
    const response = await axios.post('https://api.unionpay.com/pay', orderData);
    return response.data;
  }
  
  async pay(orderData, preferredChannel = 'alipay') {
    // 优先级列表
    const priority = [preferredChannel, ...Object.keys(this.channels).filter(c => c !== preferredChannel)];
    
    for (const channel of priority) {
      const breaker = this.channels[channel];
      
      // 跳过已熔断的通道
      if (breaker.opened) {
        console.log(`跳过已熔断通道: ${channel}`);
        continue;
      }
      
      try {
        console.log(`尝试使用通道: ${channel}`);
        const result = await breaker.fire(orderData);
        
        return {
          success: true,
          channel,
          ...result
        };
      } catch (error) {
        console.error(`${channel} 支付失败:`, error.message);
        // 继续尝试下一个通道
      }
    }
    
    // 所有通道都失败
    throw new Error('所有支付通道不可用');
  }
  
  getChannelStatus() {
    const status = {};
    Object.entries(this.channels).forEach(([name, breaker]) => {
      status[name] = {
        state: breaker.opened ? 'OPEN' : breaker.halfOpen ? 'HALF_OPEN' : 'CLOSED',
        stats: breaker.stats
      };
    });
    return status;
  }
}

module.exports = new PaymentCircuitBreaker();
```

### 场景2：API 网关

#### 需求背景

- API 网关需要调用多个后端服务
- 某个服务故障不应影响其他服务
- 需要限流 + 熔断 + 降级

#### 完整实现

```javascript
// gateway/service-breaker.js
const CircuitBreaker = require('opossum');
const RateLimiter = require('rate-limiter-flexible').RateLimiterMemory;
const axios = require('axios');

class ServiceBreakerGateway {
  constructor() {
    this.services = new Map();
    this.limiters = new Map();
  }
  
  registerService(name, config) {
    // 限流器
    const limiter = new RateLimiter({
      points: config.rateLimit || 100,  // 100 req/s
      duration: 1
    });
    
    // 服务调用函数
    const callService = async (endpoint, options = {}) => {
      // 限流检查
      try {
        await limiter.consume(name, 1);
      } catch (error) {
        throw new Error('Rate limit exceeded');
      }
      
      // 调用服务
      const response = await axios({
        baseURL: config.baseURL,
        url: endpoint,
        ...options,
        timeout: config.timeout || 5000
      });
      
      return response.data;
    };
    
    // 熔断器
    const breaker = new CircuitBreaker(callService, {
      timeout: config.timeout || 5000,
      errorThresholdPercentage: config.errorThreshold || 50,
      resetTimeout: config.resetTimeout || 30000,
      volumeThreshold: 10,
      fallback: config.fallback || (() => {
        throw new Error(`${name} service unavailable`);
      })
    });
    
    // 监控
    breaker.on('open', () => {
      console.error(`⚠️ ${name} 服务熔断`);
    });
    
    breaker.on('success', (result, latency) => {
      if (latency > 1000) {
        console.warn(`${name} 响应慢: ${latency}ms`);
      }
    });
    
    this.services.set(name, breaker);
    this.limiters.set(name, limiter);
    
    return breaker;
  }
  
  async callService(name, endpoint, options) {
    const breaker = this.services.get(name);
    
    if (!breaker) {
      throw new Error(`Service ${name} not registered`);
    }
    
    return await breaker.fire(endpoint, options);
  }
  
  getServiceStats() {
    const stats = {};
    this.services.forEach((breaker, name) => {
      stats[name] = {
        state: breaker.opened ? 'OPEN' : breaker.halfOpen ? 'HALF_OPEN' : 'CLOSED',
        ...breaker.stats
      };
    });
    return stats;
  }
}

module.exports = new ServiceBreakerGateway();
```

## 📝 五、总结

### 核心要点

✅ **熔断器** 三种状态：Closed（正常）、Open（熔断）、Half-Open（探测）  
✅ **阈值设置**：错误率 50%、最小请求数 10、重试时间 30秒  
✅ **降级策略**：返回缓存、默认值、提示信息  
✅ **监控告警**：监听熔断器事件，及时发送告警  
✅ **Opossum** 是 Node.js 最成熟的熔断器库  
✅ **级联故障**：通过熔断避免整个系统崩溃  
✅ **快速失败**：熔断后立即返回，不浪费资源

### 最佳实践

1. **合理阈值**：根据业务特点设置错误率和请求量
2. **降级方案**：必须实现 fallback 降级逻辑
3. **监控可视化**：集成 Prometheus + Grafana
4. **告警通知**：熔断时立即通知运维团队
5. **定期演练**：定期模拟故障，验证熔断效果

## ❓ 六、常见问题

### Q1: 熔断器和限流器有什么区别？

**A**: 目的和触发条件不同。

| 特性 | 熔断器 | 限流器 |
|-----|--------|--------|
| **目的** | 保护下游服务，避免级联故障 | 控制流量，保护自身 |
| **触发** | 错误率超过阈值 | 请求量超过限制 |
| **响应** | 快速失败 + 降级 | 拒绝请求或排队 |
| **使用场景** | 调用外部/下游服务 | 保护自己的 API |

### Q2: 如何设置合理的阈值？

**A**: 根据业务特点和历史数据。

```javascript
// 推荐配置（根据场景调整）

// 场景1：核心业务（严格）
errorThresholdPercentage: 30,  // 30% 错误率
volumeThreshold: 20,  // 至少 20 个请求
resetTimeout: 60000   // 60 秒

// 场景2：一般业务（平衡）
errorThresholdPercentage: 50,  // 50% 错误率
volumeThreshold: 10,  // 至少 10 个请求
resetTimeout: 30000   // 30 秒

// 场景3：非关键业务（宽松）
errorThresholdPercentage: 70,  // 70% 错误率
volumeThreshold: 5,   // 至少 5 个请求
resetTimeout: 20000   // 20 秒
```

### Q3: Half-Open 状态允许多少请求？

**A**: Opossum 默认允许1个请求。

```javascript
// 自定义 Half-Open 请求数
const breaker = new CircuitBreaker(service, {
  halfOpenRequests: 3  // Half-Open 时允许 3 个探测请求
});

// 逻辑：
// - 3 个请求都成功 → 熔断器关闭
// - 任意 1 个失败 → 熔断器重新打开
```
