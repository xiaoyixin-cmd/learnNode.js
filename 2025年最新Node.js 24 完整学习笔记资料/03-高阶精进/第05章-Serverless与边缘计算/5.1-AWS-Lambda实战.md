## 📚 知识导读

### 什么是 AWS Lambda？

AWS Lambda 是 Amazon Web Services 提供的无服务器计算服务，允许开发者在云端运行代码，无需管理服务器。Lambda 会自动运行代码以响应事件，并自动管理计算资源。

**核心概念：**

```
┌─────────────────────────────────────────────────────────┐
│                   Lambda 执行模型                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 事件触发                                            │
│     ┌────────────┐                                     │
│     │ API Gateway│───┐                                 │
│     └────────────┘   │                                 │
│     ┌────────────┐   │    ┌──────────────┐            │
│     │    S3      │───┼───→│ Lambda 函数   │            │
│     └────────────┘   │    └──────────────┘            │
│     ┌────────────┐   │           │                     │
│     │  DynamoDB  │───┘           │                     │
│     └────────────┘               ↓                     │
│                         ┌─────────────────┐            │
│  2. 容器初始化（冷启动）│  Execution      │            │
│     - 下载代码          │  Environment    │            │
│     - 启动运行时        │                 │            │
│     - 执行初始化代码    │  ┌───────────┐  │            │
│                         │  │   Handler │  │            │
│  3. 执行 Handler        │  └───────────┘  │            │
│     - 处理事件          │  ┌───────────┐  │            │
│     - 返回响应          │  │  Runtime  │  │            │
│                         │  │  (Node 20)│  │            │
│  4. 容器复用（热启动）  │  └───────────┘  │            │
│     - 跳过初始化        │                 │            │
│     - 直接执行 Handler  └─────────────────┘            │
│                                                         │
│  5. 容器回收（15分钟无请求后）                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 🎯 Lambda 架构与执行模型

### 1. Lambda 函数结构

**基本组成：**

```javascript
// index.mjs (ES Module)

/**
 * Lambda Handler 函数
 * @param {Object} event - 事件对象（包含请求数据）
 * @param {Object} context - 上下文对象（包含函数元信息）
 * @returns {Object} 响应对象
 */
export const handler = async (event, context) => {
  // 1. 打印日志（会自动发送到 CloudWatch Logs）
  console.log('Event:', JSON.stringify(event, null, 2));
  console.log('Context:', JSON.stringify(context, null, 2));

  try {
    // 2. 业务逻辑
    const result = await processEvent(event);

    // 3. 返回成功响应
    return {
      statusCode: 200,
      headers: {
        'Content-Type': 'application/json',
        'Access-Control-Allow-Origin': '*', // CORS
      },
      body: JSON.stringify({
        success: true,
        data: result,
        requestId: context.requestId, // AWS 请求 ID
      }),
    };
  } catch (error) {
    // 4. 错误处理
    console.error('Error:', error);

    return {
      statusCode: 500,
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        success: false,
        error: error.message,
        requestId: context.requestId,
      }),
    };
  }
};

async function processEvent(event) {
  // 模拟业务逻辑
  return { message: 'Hello from Lambda!' };
}
```

### 2. DynamoDB 客户端封装

```javascript
// src/lib/dynamodb.mjs
import { DynamoDBClient } from '@aws-sdk/client-dynamodb';
import {
  DynamoDBDocumentClient,
  PutCommand,
  GetCommand,
  ScanCommand,
  UpdateCommand,
  DeleteCommand,
} from '@aws-sdk/lib-dynamodb';

// 创建 DynamoDB 客户端（使用单例模式）
const client = new DynamoDBClient({
  region: process.env.AWS_REGION || 'us-east-1',
});

// 包装为 Document Client（自动转换数据类型）
const docClient = DynamoDBDocumentClient.from(client, {
  marshallOptions: {
    removeUndefinedValues: true, // 移除 undefined 值
    convertEmptyValues: false,
  },
});

const TABLE_NAME = process.env.USERS_TABLE;

/**
 * 创建用户
 */
export async function createUser(user) {
  const command = new PutCommand({
    TableName: TABLE_NAME,
    Item: {
      ...user,
      createdAt: new Date().toISOString(),
      updatedAt: new Date().toISOString(),
    },
    ConditionExpression: 'attribute_not_exists(id)', // 防止覆盖
  });

  await docClient.send(command);
  return user;
}

/**
 * 获取单个用户
 */
export async function getUser(id) {
  const command = new GetCommand({
    TableName: TABLE_NAME,
    Key: { id },
  });

  const { Item } = await docClient.send(command);
  return Item;
}

/**
 * 列出所有用户（带分页）
 */
export async function listUsers(limit = 20, lastEvaluatedKey = null) {
  const command = new ScanCommand({
    TableName: TABLE_NAME,
    Limit: limit,
    ...(lastEvaluatedKey && { ExclusiveStartKey: lastEvaluatedKey }),
  });

  const { Items, LastEvaluatedKey } = await docClient.send(command);
  return { users: Items, lastEvaluatedKey: LastEvaluatedKey };
}

/**
 * 更新用户
 */
export async function updateUser(id, updates) {
  const updateExpression = [];
  const expressionAttributeNames = {};
  const expressionAttributeValues = {};

  Object.keys(updates).forEach((key, index) => {
    updateExpression.push(`#field${index} = :value${index}`);
    expressionAttributeNames[`#field${index}`] = key;
    expressionAttributeValues[`:value${index}`] = updates[key];
  });

  updateExpression.push(`#updatedAt = :updatedAt`);
  expressionAttributeNames['#updatedAt'] = 'updatedAt';
  expressionAttributeValues[':updatedAt'] = new Date().toISOString();

  const command = new UpdateCommand({
    TableName: TABLE_NAME,
    Key: { id },
    UpdateExpression: `SET ${updateExpression.join(', ')}`,
    ExpressionAttributeNames: expressionAttributeNames,
    ExpressionAttributeValues: expressionAttributeValues,
    ReturnValues: 'ALL_NEW',
    ConditionExpression: 'attribute_exists(id)', // 确保记录存在
  });

  const { Attributes } = await docClient.send(command);
  return Attributes;
}

/**
 * 删除用户
 */
export async function deleteUser(id) {
  const command = new DeleteCommand({
    TableName: TABLE_NAME,
    Key: { id },
    ReturnValues: 'ALL_OLD',
  });

  const { Attributes } = await docClient.send(command);
  return Attributes;
}
```

### 4. 输入验证

```javascript
// src/lib/validator.mjs

/**
 * 验证用户创建数据
 */
export function validateCreateUser(data) {
  const errors = [];

  if (!data.name || typeof data.name !== 'string' || data.name.length < 2) {
    errors.push('Name must be at least 2 characters long');
  }

  if (!data.email || !isValidEmail(data.email)) {
    errors.push('Invalid email address');
  }

  if (data.age !== undefined && (typeof data.age !== 'number' || data.age < 0 || data.age > 150)) {
    errors.push('Age must be between 0 and 150');
  }

  return errors;
}

/**
 * 验证用户更新数据
 */
export function validateUpdateUser(data) {
  const errors = [];

  if (data.name !== undefined && (typeof data.name !== 'string' || data.name.length < 2)) {
    errors.push('Name must be at least 2 characters long');
  }

  if (data.email !== undefined && !isValidEmail(data.email)) {
    errors.push('Invalid email address');
  }

  if (data.age !== undefined && (typeof data.age !== 'number' || data.age < 0 || data.age > 150)) {
    errors.push('Age must be between 0 and 150');
  }

  return errors;
}

function isValidEmail(email) {
  return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);
}
```

### 5. Handler 实现

```javascript
// src/handlers/createUser.mjs
import { randomUUID } from 'node:crypto';
import { createUser } from '../lib/dynamodb.mjs';
import { validateCreateUser } from '../lib/validator.mjs';
import { success, error, validationError } from '../utils/response.mjs';

export const handler = async (event) => {
  try {
    // 1. 解析请求体
    const body = JSON.parse(event.body || '{}');
    console.log('Create user request:', body);

    // 2. 验证输入
    const errors = validateCreateUser(body);
    if (errors.length > 0) {
      return validationError(errors);
    }

    // 3. 创建用户
    const user = {
      id: randomUUID(),
      name: body.name,
      email: body.email,
      age: body.age,
      status: 'active',
    };

    const createdUser = await createUser(user);
    console.log('User created:', createdUser);

    // 4. 返回成功响应
    return success(createdUser, 201);
  } catch (err) {
    // 5. 错误处理
    if (err.name === 'ConditionalCheckFailedException') {
      return error('User with this ID already exists', 409);
    }

    console.error('Failed to create user:', err);
    return error('Failed to create user', 500);
  }
};
```

## 💻 实战案例 2：S3 图片处理函数

### 场景：自动生成缩略图

当用户上传图片到 S3 时，Lambda 自动生成缩略图并保存到另一个 S3 存储桶。

### 实现代码

```javascript
// src/handlers/imageProcessor.mjs
import { S3Client, GetObjectCommand, PutObjectCommand } from '@aws-sdk/client-s3';
import sharp from 'sharp';  // 图片处理库

const s3Client = new S3Client({ region: process.env.AWS_REGION });
const THUMBNAIL_BUCKET = process.env.THUMBNAIL_BUCKET;

export const handler = async (event) => {
  try {
    // 1. 解析 S3 事件
    const record = event.Records[0];
    const srcBucket = record.s3.bucket.name;
    const srcKey = decodeURIComponent(record.s3.object.key.replace(/\+/g, ' '));

    console.log(`Processing image: ${srcBucket}/${srcKey}`);

    // 2. 下载原始图片
    const getObjectCommand = new GetObjectCommand({
      Bucket: srcBucket,
      Key: srcKey,
    });
    const { Body, ContentType } = await s3Client.send(getObjectCommand);

    // 3. 将流转换为 Buffer
    const imageBuffer = await streamToBuffer(Body);

    // 4. 生成缩略图（300x300）
    const thumbnailBuffer = await sharp(imageBuffer)
      .resize(300, 300, {
        fit: 'cover',
        position: 'center',
      })
      .toFormat('jpeg', { quality: 80 })
      .toBuffer();

    // 5. 上传缩略图到目标桶
    const dstKey = `thumbnails/${srcKey.replace(/\.\w+$/, '.jpg')}`;
    const putObjectCommand = new PutObjectCommand({
      Bucket: THUMBNAIL_BUCKET,
      Key: dstKey,
      Body: thumbnailBuffer,
      ContentType: 'image/jpeg',
      Metadata: {
        'original-key': srcKey,
        'processed-at': new Date().toISOString(),
      },
    });

    await s3Client.send(putObjectCommand);

    console.log(`Thumbnail created: ${THUMBNAIL_BUCKET}/${dstKey}`);

    return {
      statusCode: 200,
      body: JSON.stringify({
        message: 'Thumbnail created successfully',
        thumbnail: `${THUMBNAIL_BUCKET}/${dstKey}`,
      }),
    };
  } catch (error) {
    console.error('Error processing image:', error);
    throw error;  // Lambda 会自动重试（最多 2 次）
  }
};

/**
 * 将流转换为 Buffer
 */
async function streamToBuffer(stream) {
  const chunks = [];
  for await (const chunk of stream) {
    chunks.push(chunk);
  }
  return Buffer.concat(chunks);
}
```

## 💻 实战案例 3：定时任务（Cron Job）

### 场景：每天凌晨生成报表

```javascript
// src/handlers/dailyReport.mjs
import { DynamoDBClient } from '@aws-sdk/client-dynamodb';
import { DynamoDBDocumentClient, ScanCommand } from '@aws-sdk/lib-dynamodb';
import { SESClient, SendEmailCommand } from '@aws-sdk/client-ses';

const dynamoClient = DynamoDBDocumentClient.from(new DynamoDBClient({}));
const sesClient = new SESClient({ region: process.env.AWS_REGION });

export const handler = async (event) => {
  try {
    console.log('Starting daily report generation...');

    // 1. 统计昨天的数据
    const yesterday = new Date();
    yesterday.setDate(yesterday.getDate() - 1);
    const yesterdayStr = yesterday.toISOString().split('T')[0];

    const { Items: orders } = await dynamoClient.send(
      new ScanCommand({
        TableName: process.env.ORDERS_TABLE,
        FilterExpression: 'begins_with(createdAt, :date)',
        ExpressionAttributeValues: {
          ':date': yesterdayStr,
        },
      })
    );

    // 2. 计算统计数据
    const stats = {
      totalOrders: orders.length,
      totalRevenue: orders.reduce((sum, order) => sum + order.amount, 0),
      averageOrderValue: 0,
    };
    stats.averageOrderValue = stats.totalOrders > 0 ? stats.totalRevenue / stats.totalOrders : 0;

    // 3. 生成 HTML 报表
    const htmlReport = `
      <html>
        <body>
          <h1>Daily Report - ${yesterdayStr}</h1>
          <table border="1">
            <tr><th>Metric</th><th>Value</th></tr>
            <tr><td>Total Orders</td><td>${stats.totalOrders}</td></tr>
            <tr><td>Total Revenue</td><td>$${stats.totalRevenue.toFixed(2)}</td></tr>
            <tr><td>Average Order Value</td><td>$${stats.averageOrderValue.toFixed(2)}</td></tr>
          </table>
        </body>
      </html>
    `;

    // 4. 发送邮件
    await sesClient.send(
      new SendEmailCommand({
        Source: process.env.SENDER_EMAIL,
        Destination: {
          ToAddresses: [process.env.RECIPIENT_EMAIL],
        },
        Message: {
          Subject: { Data: `Daily Report - ${yesterdayStr}` },
          Body: { Html: { Data: htmlReport } },
        },
      })
    );

    console.log('Daily report sent successfully');

    return {
      statusCode: 200,
      body: JSON.stringify({ message: 'Report generated and sent', stats }),
    };
  } catch (error) {
    console.error('Error generating report:', error);
    throw error;
  }
};
```

## ⚠️ 三个高频错误

### 错误 1：忘记处理异步操作导致函数提前返回

**错误代码：**

```javascript
export const handler = (event) => {
  // ❌ 缺少 async/await
  fetchData().then((data) => {
    console.log(data);
  });

  return { statusCode: 200, body: 'OK' };  // 立即返回，fetchData 未完成
};
```

**正确做法：**

```javascript
export const handler = async (event) => {
  // ✅ 使用 async/await
  const data = await fetchData();
  console.log(data);

  return { statusCode: 200, body: 'OK' };
};
```

---

### 错误 2：在每次调用时初始化客户端（性能浪费）

**错误代码：**

```javascript
export const handler = async (event) => {
  // ❌ 每次调用都创建新的客户端
  const dynamoClient = new DynamoDBClient({});
  const docClient = DynamoDBDocumentClient.from(dynamoClient);

  // 使用 docClient...
};
```

**正确做法（复用连接）：**

```javascript
// ✅ 在 Handler 外部初始化（容器复用时会复用）
const dynamoClient = new DynamoDBClient({});
const docClient = DynamoDBDocumentClient.from(dynamoClient);

export const handler = async (event) => {
  // 直接使用 docClient
};
```

---

### 错误 3：未设置 CORS 导致前端无法调用

**错误代码：**

```javascript
export const handler = async (event) => {
  return {
    statusCode: 200,
    body: JSON.stringify({ message: 'Hello' }),
    // ❌ 缺少 CORS headers
  };
};
```

**正确做法：**

```javascript
export const handler = async (event) => {
  return {
    statusCode: 200,
    headers: {
      'Content-Type': 'application/json',
      'Access-Control-Allow-Origin': '*',  // ✅ 或指定具体域名
      'Access-Control-Allow-Credentials': true,
      'Access-Control-Allow-Methods': 'GET,POST,PUT,DELETE,OPTIONS',
      'Access-Control-Allow-Headers': 'Content-Type,Authorization',
    },
    body: JSON.stringify({ message: 'Hello' }),
  };
};
```

## 📚 FAQ

### 1. Lambda 函数能运行多久？

- **默认超时：** 3 秒
- **最大超时：** 15 分钟
- **建议：** 根据实际需求设置，避免浪费

### 2. 如何调试 Lambda 函数？

```bash
# 方法 1：使用 serverless-offline（本地模拟）
npm install --save-dev serverless-offline
serverless offline

# 方法 2：查看 CloudWatch Logs
serverless logs --function myFunction --tail

# 方法 3：使用 AWS SAM CLI
sam local invoke myFunction --event event.json
```

### 3. 冷启动有多慢？

| 因素 | 影响 |
|------|------|
| 包大小 | 100MB vs 10MB：+500ms |
| 运行时 | Node.js vs Java：Java 慢 2-3 倍 |
| 内存 | 128MB vs 1024MB：小内存慢 3-5 倍 |

### 4. 如何避免冷启动？

- **使用预留并发（Provisioned Concurrency）**（有额外成本）
- **减小包体积**（使用 esbuild、移除未使用依赖）
- **使用 Lambda Layers 分离依赖**

### 5. Lambda 能访问 VPC 内的资源吗？

**可以！** 配置 VPC 后可访问 RDS、ElastiCache 等：

```yaml
# serverless.yml
provider:
  vpc:
    securityGroupIds:
      - sg-xxxxxx
    subnetIds:
      - subnet-xxxxxx
      - subnet-yyyyyy
```

**注意：** VPC 配置会增加冷启动时间（+1-3 秒）。

### 6. Lambda 函数之间如何通信？

- **同步调用：** 使用 AWS SDK 的 `InvokeCommand`
- **异步调用：** 使用 SQS、SNS、EventBridge
- **推荐：** 使用事件驱动架构，避免直接调用
