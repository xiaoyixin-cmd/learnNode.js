## 🎯 Lambda 定价模型

### 1. 请求次数费用

**定价：**

```
免费额度：
- 每月前 100 万次请求：免费

付费额度：
- $0.20 per 1M requests（每百万次请求）
```

**示例计算：**

```javascript
// 每月 500 万次请求
const requests = 5_000_000;
const freeRequests = 1_000_000;
const paidRequests = requests - freeRequests;

const requestCost = (paidRequests / 1_000_000) * 0.2;
console.log(`请求费用: $${requestCost}`);  // $0.80
```

---

### 2. 执行时间费用

**定价：**

```
免费额度：
- 每月前 400,000 GB-秒：免费

付费额度：
- $0.00001667 per GB-second
```

**GB-秒计算公式：**

```
GB-秒 = (内存 GB) × (执行时间 秒)

示例：
- 内存：512MB (0.5GB)
- 执行时间：200ms (0.2s)
- 单次请求 GB-秒：0.5 × 0.2 = 0.1 GB-秒
```

**示例计算：**

```javascript
// 配置
const requests = 5_000_000;
const memoryGB = 0.5;  // 512MB
const durationSec = 0.2;  // 200ms

// 计算总 GB-秒
const totalGBSeconds = requests * memoryGB * durationSec;
console.log(`总 GB-秒: ${totalGBSeconds}`);  // 500,000

// 减去免费额度
const freeGBSeconds = 400_000;
const paidGBSeconds = totalGBSeconds - freeGBSeconds;

// 计算费用
const durationCost = paidGBSeconds * 0.00001667;
console.log(`执行费用: $${durationCost.toFixed(2)}`);  // $1.67
```

### 3. 总成本计算

```javascript
function calculateLambdaCost(requests, memoryMB, durationMs) {
  // 1. 请求费用
  const freeRequests = 1_000_000;
  const paidRequests = Math.max(0, requests - freeRequests);
  const requestCost = (paidRequests / 1_000_000) * 0.2;

  // 2. 执行时间费用
  const memoryGB = memoryMB / 1024;
  const durationSec = durationMs / 1000;
  const totalGBSeconds = requests * memoryGB * durationSec;
  const freeGBSeconds = 400_000;
  const paidGBSeconds = Math.max(0, totalGBSeconds - freeGBSeconds);
  const durationCost = paidGBSeconds * 0.00001667;

  // 3. 总成本
  const totalCost = requestCost + durationCost;

  return {
    requests,
    requestCost: requestCost.toFixed(2),
    durationCost: durationCost.toFixed(2),
    totalCost: totalCost.toFixed(2),
  };
}

// 示例 1：低流量
console.log(calculateLambdaCost(100_000, 512, 200));
// { requests: 100000, requestCost: '0.00', durationCost: '0.00', totalCost: '0.00' }

// 示例 2：中流量
console.log(calculateLambdaCost(5_000_000, 512, 200));
// { requests: 5000000, requestCost: '0.80', durationCost: '1.67', totalCost: '2.47' }

// 示例 3：高流量
console.log(calculateLambdaCost(50_000_000, 512, 200));
// { requests: 50000000, requestCost: '9.80', durationCost: '41.67', totalCost: '51.47' }
```

```
内存 512MB：
- 执行时间：200ms
- GB-秒：0.5 × 0.2 = 0.1 GB-秒
- 成本：$0.00001667 × 0.1 = $0.0000017

内存 1024MB：
- 执行时间：100ms
- GB-秒：1 × 0.1 = 0.1 GB-秒
- 成本：$0.00001667 × 0.1 = $0.0000017
```

### 策略 2：减少执行时间

#### 2.1 复用连接

**❌ 错误做法（每次创建新连接）：**

```javascript
export const handler = async (event) => {
  // 每次调用都创建新的 DynamoDB 客户端（慢！）

  const docClient = DynamoDBDocumentClient.from(client);

  // 查询数据...
```

**✅ 正确做法（复用连接）：**

```javascript
// 在 handler 外部初始化（容器复用时会复用）
const client = new DynamoDBClient({});
const docClient = DynamoDBDocumentClient.from(client);

export const handler = async (event) => {
  // 直接使用 docClient（快！）
  // 查询数据...
};
```

**效果：**
- 冷启动：节省 50-100ms
- 热启动：节省 10-20ms
- 成本降低 10-20%

#### 2.2 使用缓存

**场景：** 频繁读取的配置数据

**优化前：**

```javascript
export const handler = async (event) => {
  // 每次调用都从 DynamoDB 读取配置（慢且费钱）
  const config = await docClient.send(new GetCommand({
    TableName: 'config',
    Key: { id: 'app-config' },
  }));

  // 使用 config...
};
```

**优化后（容器级缓存）：**

```javascript
let configCache = null;
let cacheExpiry = 0;

export const handler = async (event) => {
  const now = Date.now();

  // 如果缓存未过期，直接使用
  if (configCache && now < cacheExpiry) {
    console.log('Using cached config');
    return processWithConfig(configCache);
  }

  // 缓存过期，重新读取
  console.log('Fetching fresh config');
  const { Item } = await docClient.send(new GetCommand({
    TableName: 'config',
    Key: { id: 'app-config' },
  }));

  configCache = Item;
  cacheExpiry = now + 60000;  // 缓存 60 秒

  return processWithConfig(configCache);
};
```

**效果：**
- DynamoDB 读取次数减少 90%+
- 执行时间减少 50-100ms
- 成本降低 40-60%

#### 2.3 并行处理

**优化前（串行）：**

```javascript
export const handler = async (event) => {
```

## ⚠️ 三个高频错误

### 错误 1：过度使用预留并发

**问题：** 为所有函数配置预留并发。

```yaml
# ❌ 错误（成本爆炸）
functions:
  lowTrafficFunction:  # 每小时只调用 1 次
    handler: src/handler.handler
    provisionedConcurrency: 5  # 每月额外成本 $7.60
```

**正确做法：** 只为高频、对延迟敏感的函数配置预留并发。

---

### 错误 2：忽略 DynamoDB 成本

**问题：** DynamoDB 按需计费，大量读写会产生高额成本。

```javascript
// ❌ 每次请求都扫描整个表（昂贵！）
const { Items } = await docClient.send(new ScanCommand({
  TableName: 'users',
}));
```

**优化：** 使用查询（Query）代替扫描（Scan），添加缓存。

---

### 错误 3：未设置超时限制

**问题：** 默认超时 3 秒，最大 15 分钟。如果函数卡住，会持续计费。

```yaml
# ✅ 设置合理的超时
functions:
  myFunction:
    handler: src/handler.handler
    timeout: 10  # 10 秒（根据实际需求设置）
```

## 📚 FAQ

### 1. Serverless 什么时候不划算？

**不划算的场景：**
- 持续高流量（每秒数百次请求）
- 长时间运行的任务（如视频转码）
- 需要大量内存（> 3GB）

**解决方案：**
- 混合架构（高频 API 用 ECS，低频用 Lambda）
- 使用 Fargate 或 ECS（容器化）
- 使用 EC2 Spot 实例

### 3. 如何预测 Serverless 成本？

**使用 AWS Pricing Calculator：**
https://calculator.aws/#/

**手动计算公式：**

```javascript
function estimateMonthlyCost(
  requestsPerMonth,
  avgDurationMs,
  memoryMB
) {
  // 请求费用
  const requestCost = Math.max(0, (requestsPerMonth - 1_000_000) / 1_000_000 * 0.2);

  // 执行时间费用
  const gbSeconds = requestsPerMonth * (memoryMB / 1024) * (avgDurationMs / 1000);
  const durationCost = Math.max(0, (gbSeconds - 400_000) * 0.00001667);

  return {
    requestCost: requestCost.toFixed(2),
    durationCost: durationCost.toFixed(2),
    totalCost: (requestCost + durationCost).toFixed(2),
  };
}

// 预测示例
console.log(estimateMonthlyCost(10_000_000, 200, 512));
// { requestCost: '1.80', durationCost: '8.34', totalCost: '10.14' }
```
