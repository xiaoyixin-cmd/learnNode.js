# 5.3 Vercel Edge Functions

> 使用 Web 标准 API 构建极速边缘函数

---

## 📚 知识导读

### 什么是 Vercel Edge Functions？

**Vercel Edge Functions** 是运行在 Vercel Edge Network 上的无服务器函数，使用 **V8 Isolate** 技术实现极致的冷启动性能（< 10ms），支持 **Web 标准 API**（Fetch、Request、Response、Web Crypto 等）。

**核心特点：**

```
┌────────────────────────────────────────────────────────┐
│        Edge Functions vs 传统 Serverless Lambda        │
├────────────────────────────────────────────────────────┤
│                                                        │
│  传统 Lambda（AWS Lambda、Cloud Functions）：           │
│  - 运行时：Node.js 完整环境                             │
│  - 冷启动：100-500ms                                   │
│  - 内存：128MB-10GB                                    │
│  - 执行时间：最长 15 分钟                               │
│  - 地理位置：区域性（单个数据中心）                      │
│  - API：Node.js 原生 API + AWS SDK                    │
│                                                        │
│  Vercel Edge Functions：                              │
│  - 运行时：V8 Isolate（轻量级沙箱）                     │
│  - 冷启动：<10ms（几乎无感知）                          │
│  - 内存：最大 128MB                                    │
│  - 执行时间：最长 30 秒                                 │
│  - 地理位置：全球边缘节点（距离用户更近）                 │
│  - API：Web 标准 API（Fetch、Request、Response）       │
│                                                        │
└────────────────────────────────────────────────────────┘
```

**Edge Network 原理：**

```
用户请求 → 就近的边缘节点（全球 300+ 节点）→ Edge Function
              ↓
         低延迟响应（<50ms）

对比传统架构：
用户请求 → CDN → Origin Server（可能在另一个大陆）→ Lambda
              ↓
         高延迟响应（200-500ms）
```

### 2. Request 对象

**解析请求信息：**

```javascript
export default async function handler(request) {
  // 1. 获取请求方法
  const method = request.method;

  // 2. 获取 URL 信息
  const url = new URL(request.url);
  const pathname = url.pathname;
  const searchParams = url.searchParams;

  // 3. 获取请求头
  const userAgent = request.headers.get('user-agent');
  const authorization = request.headers.get('authorization');

  // 4. 解析请求体
  let body = null;
  if (method === 'POST') {
    body = await request.json();  // 或 request.text()、request.formData()
  }

  // 5. 获取 Cookie
  const cookies = request.headers.get('cookie');

  return new Response(
    JSON.stringify({
      method,
      pathname,
      query: Object.fromEntries(searchParams),
      userAgent,
      body,
    }),
    { status: 200, headers: { 'Content-Type': 'application/json' } }
  );
}

export const config = { runtime: 'edge' };
```

---

### 3. Response 对象

**返回响应：**

```javascript
export default async function handler(request) {
  // 1. JSON 响应
  return new Response(JSON.stringify({ message: 'Hello' }), {
    status: 200,
    headers: {
      'Content-Type': 'application/json',
      'Cache-Control': 'public, max-age=60',
    },
  });

  // 2. HTML 响应
  return new Response('<h1>Hello World</h1>', {
    status: 200,
    headers: { 'Content-Type': 'text/html' },
  });

  // 3. 重定向
  return Response.redirect('https://vercel.com', 302);

  // 4. 流式响应（Streaming）
  const stream = new ReadableStream({
    async start(controller) {
      for (let i = 0; i < 10; i++) {
        controller.enqueue(`data: ${i}\n\n`);
        await new Promise((resolve) => setTimeout(resolve, 100));
      }
      controller.close();
    },
  });

  return new Response(stream, {
    headers: { 'Content-Type': 'text/event-stream' },
  });
}

export const config = { runtime: 'edge' };
```

### 4. Web Crypto API

**加密和哈希：**

```javascript
export default async function handler(request) {
  const { password } = await request.json();

  // 1. 生成 SHA-256 哈希
  const encoder = new TextEncoder();
  const data = encoder.encode(password);
  const hashBuffer = await crypto.subtle.digest('SHA-256', data);
  const hashArray = Array.from(new Uint8Array(hashBuffer));
  const hashHex = hashArray.map((b) => b.toString(16).padStart(2, '0')).join('');

  // 2. 生成随机 UUID
  const uuid = crypto.randomUUID();

  // 3. HMAC 签名
  const key = await crypto.subtle.generateKey(
    { name: 'HMAC', hash: 'SHA-256' },
    false,
    ['sign']
  );
  const signature = await crypto.subtle.sign('HMAC', key, data);

  return new Response(
    JSON.stringify({
      sha256: hashHex,
      uuid,
      signature: btoa(String.fromCharCode(...new Uint8Array(signature))),
    }),
    { status: 200, headers: { 'Content-Type': 'application/json' } }
  );
}

export const config = { runtime: 'edge' };
```

## 💻 实战案例 1：地理位置重定向

### 场景：根据用户地理位置重定向到不同域名

**需求：**
- 中国用户 → `cn.example.com`
- 美国用户 → `us.example.com`
- 其他地区 → `global.example.com`

**实现代码：**

```javascript
// pages/api/geo-redirect.js
export default async function handler(request) {
  // 1. 从 Vercel 自动注入的 Geo Headers 获取地理位置
  const country = request.headers.get('x-vercel-ip-country') || 'UNKNOWN';
  const city = request.headers.get('x-vercel-ip-city');
  const latitude = request.headers.get('x-vercel-ip-latitude');
  const longitude = request.headers.get('x-vercel-ip-longitude');

  console.log('Geo Info:', { country, city, latitude, longitude });

  // 2. 根据国家代码重定向
  const redirectMap = {
    CN: 'https://cn.example.com',
    US: 'https://us.example.com',
    JP: 'https://jp.example.com',
  };

  const targetUrl = redirectMap[country] || 'https://global.example.com';

  // 3. 重定向
  return Response.redirect(targetUrl, 302);
}

export const config = {
  runtime: 'edge',
};
```

**Vercel 提供的 Geo Headers：**

| Header | 说明 | 示例 |
|--------|------|------|
| `x-vercel-ip-country` | 国家代码 | `US`, `CN`, `JP` |
| `x-vercel-ip-city` | 城市 | `San Francisco` |
| `x-vercel-ip-region` | 地区 | `California` |
| `x-vercel-ip-latitude` | 纬度 | `37.7749` |
| `x-vercel-ip-longitude` | 经度 | `-122.4194` |

## 💻 实战案例 2：A/B 测试中间件

### 场景：50% 用户看到新版页面，50% 看到旧版页面

**实现代码：**

```javascript
// middleware.js（在 Next.js 13+ 中，middleware 自动运行在 Edge）
import { NextResponse } from 'next/server';

export function middleware(request) {
  // 1. 检查是否已有 A/B 测试 Cookie
  const variant = request.cookies.get('ab-test-variant');

  if (variant) {
    // 已分配变体，直接使用
    if (variant === 'B') {
      return NextResponse.rewrite(new URL('/beta', request.url));
    }
    return NextResponse.next();
  }

  // 2. 随机分配变体（50/50）
  const newVariant = Math.random() < 0.5 ? 'A' : 'B';

  // 3. 设置 Cookie
  const response =
    newVariant === 'B'
      ? NextResponse.rewrite(new URL('/beta', request.url))
      : NextResponse.next();

  response.cookies.set('ab-test-variant', newVariant, {
    maxAge: 60 * 60 * 24 * 30, // 30 天
    httpOnly: true,
  });

  return response;
}

export const config = {
  matcher: '/',  // 只在首页生效
};
```

**效果：**
- 用户首次访问 `/` 时，随机分配到 A 或 B 变体
- A 变体：显示原始首页
- B 变体：重写到 `/beta` 页面（用户看到的仍是 `/` URL）
- 30 天内保持相同变体

## 💻 实战案例 3：API 速率限制（Rate Limiting）

### 场景：限制每个 IP 每分钟最多 10 次请求

**使用 Vercel KV（Edge-compatible）：**

```javascript
// pages/api/rate-limited.js
import { kv } from '@vercel/kv';

export default async function handler(request) {
  // 1. 获取客户端 IP
  const ip = request.headers.get('x-forwarded-for') || request.headers.get('x-real-ip');

  if (!ip) {
    return new Response(JSON.stringify({ error: 'Unable to determine IP' }), {
      status: 400,
      headers: { 'Content-Type': 'application/json' },
    });
  }

  // 2. 构建 Redis Key
  const key = `rate-limit:${ip}`;

  // 3. 获取当前请求次数
  const current = await kv.get(key);
  const limit = 10;
  const ttl = 60; // 60 秒

  if (current !== null && current >= limit) {
    // 超过限制
    return new Response(
      JSON.stringify({
        error: 'Too many requests',
        limit,
        retryAfter: await kv.ttl(key),
      }),
      {
        status: 429,
        headers: {
          'Content-Type': 'application/json',
          'Retry-After': String(await kv.ttl(key)),
        },
      }
    );
  }

  // 4. 增加计数
  if (current === null) {
    await kv.set(key, 1, { ex: ttl });
  } else {
    await kv.incr(key);
  }

  // 5. 正常响应
  return new Response(
    JSON.stringify({
      message: 'Request successful',
      remaining: limit - (current || 0) - 1,
    }),
    {
      status: 200,
      headers: {
        'Content-Type': 'application/json',
        'X-RateLimit-Limit': String(limit),
        'X-RateLimit-Remaining': String(limit - (current || 0) - 1),
      },
    }
  );
}

export const config = {
  runtime: 'edge',
};
```

**安装依赖：**

```bash

```

1. 进入项目设置 → Storage
2. 创建 KV 存储
3. 连接到项目（自动注入环境变量）

## 💻 实战案例 4：边缘缓存（Edge Config）

### 场景：在边缘节点缓存配置数据

**什么是 Edge Config？**

Edge Config 是 Vercel 提供的全球分布式键值存储，数据同步到所有边缘节点，读取延迟 < 10ms。

**适用场景：**
- 功能开关（Feature Flags）
- A/B 测试配置
- 重定向规则
- IP 黑名单

**实现代码：**

```javascript
// pages/api/feature-flag.js
import { get } from '@vercel/edge-config';

export default async function handler(request) {
  // 1. 从 Edge Config 读取功能开关
  const featureFlags = await get('features');

  // 2. 检查特定功能是否启用
  const newUIEnabled = featureFlags?.newUI === true;

  return new Response(
    JSON.stringify({
      features: featureFlags,
      newUIEnabled,
    }),
    {
      status: 200,
      headers: {
        'Content-Type': 'application/json',
        'Cache-Control': 'public, s-maxage=60',
      },
    }
  );
}

export const config = {
  runtime: 'edge',
};
```

**配置 Edge Config：**

1. 创建 Edge Config（Vercel Dashboard → Storage → Edge Config）
2. 添加键值对：
   ```json
   {
     "features": {
       "newUI": true,
       "darkMode": false,
       "betaFeature": true
     }
   }
   ```
3. 连接到项目（自动注入 `EDGE_CONFIG` 环境变量）

## 💻 实战案例 5：JWT 认证中间件

### 场景：验证 JWT Token

```javascript
// middleware.js
import { NextResponse } from 'next/server';
import { jwtVerify, importSPKI } from 'jose';

const PUBLIC_KEY = `-----BEGIN PUBLIC KEY-----
MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AMIIBCgKCAQEA...
-----END PUBLIC KEY-----`;

export async function middleware(request) {
  // 1. 获取 Authorization Header
  const authHeader = request.headers.get('authorization');

  if (!authHeader || !authHeader.startsWith('Bearer ')) {
    return new NextResponse(JSON.stringify({ error: 'Missing or invalid token' }), {
      status: 401,
      headers: { 'Content-Type': 'application/json' },
    });
  }

  const token = authHeader.substring(7);

  try {
    // 2. 验证 JWT
    const publicKey = await importSPKI(PUBLIC_KEY, 'RS256');
    const { payload } = await jwtVerify(token, publicKey);

    // 3. 将用户信息注入请求头（传递给 API 路由）
    const requestHeaders = new Headers(request.headers);
    requestHeaders.set('x-user-id', payload.sub);
    requestHeaders.set('x-user-email', payload.email);

    return NextResponse.next({
      request: {
        headers: requestHeaders,
      },
    });
  } catch (error) {
    console.error('JWT verification failed:', error);
    return new NextResponse(JSON.stringify({ error: 'Invalid token' }), {
      status: 401,
      headers: { 'Content-Type': 'application/json' },
    });
  }
}

export const config = {
  matcher: '/api/protected/:path*',  // 只保护 /api/protected/* 路由
};
```

// pages/api/stream-html.js
export default async function handler(request) {

    async start(controller) {

          </body>

    },
  });

    headers: { 'Content-Type': 'text/html; charset=utf-8' },
  });
}

  runtime: 'edge',

## ⚠️ 三个高频错误

### 错误 1：使用 Node.js 原生 API

**问题：** Edge Runtime 不支持 Node.js 原生模块（如 `fs`、`path`、`crypto`）。

```javascript
// ❌ 错误
import fs from 'fs';

export default async function handler(request) {
  const data = fs.readFileSync('/path/to/file');  // 运行时报错
  return new Response(data);
}

export const config = { runtime: 'edge' };
```

**正确做法：** 使用 Web 标准 API 或 Vercel 提供的边缘兼容库。

```javascript
// ✅ 正确
import { kv } from '@vercel/kv';

export default async function handler(request) {
  const data = await kv.get('my-key');
  return new Response(JSON.stringify(data), {
    headers: { 'Content-Type': 'application/json' },
  });
}

export const config = { runtime: 'edge' };
```

---

### 错误 2：忘记设置 `runtime: 'edge'`

**问题：** 默认情况下，API 路由运行在 Node.js 环境，不享受 Edge 的性能优势。

```javascript
// ❌ 错误（运行在 Node.js 环境）
export default async function handler(req, res) {
  res.json({ message: 'Hello' });
}
// 缺少 export const config = { runtime: 'edge' };
```

**正确做法：**

```javascript
// ✅ 正确（运行在 Edge 环境）
export default async function handler(request) {
  return new Response(JSON.stringify({ message: 'Hello' }), {
    headers: { 'Content-Type': 'application/json' },
  });
}

export const config = {
  runtime: 'edge',  // 必须显式声明
};
```

### 错误 3：超过 128MB 内存限制

**问题：** Edge Functions 最大内存 128MB，处理大文件时容易超限。

```javascript
// ❌ 错误（加载大文件到内存）
export default async function handler(request) {
  const response = await fetch('https://example.com/large-video.mp4');
  const buffer = await response.arrayBuffer();  // 可能超过 128MB
  return new Response(buffer);
}

export const config = { runtime: 'edge' };
```

**正确做法：** 使用流式处理。

```javascript
// ✅ 正确（流式传输）
export default async function handler(request) {
  const response = await fetch('https://example.com/large-video.mp4');
  return new Response(response.body, {
    headers: { 'Content-Type': 'video/mp4' },
  });
}

export const config = { runtime: 'edge' };
```

## 📚 FAQ

### 1. Edge Functions vs Lambda 该如何选择？

| 特性 | Edge Functions | Lambda |
|------|---------------|--------|
| 冷启动 | <10ms | 100-500ms |
| 地理分布 | 全球边缘节点 | 单一区域 |
| 执行时间 | 最长 30 秒 | 最长 15 分钟 |
| 内存 | 最大 128MB | 最大 10GB |
| API | Web 标准 | Node.js 全功能 |

**选择建议：**
- **Edge Functions：** API 路由、中间件、地理路由、实时个性化
- **Lambda：** 复杂计算、长时间任务、需要大内存

### 2. 如何调试 Edge Functions？

```bash
# 方法 1：本地运行（Next.js 13+）
npm run dev

# 方法 2：查看日志（Vercel Dashboard）
vercel logs --follow

# 方法 3：使用 console.log（自动输出到日志）
export default async function handler(request) {
  console.log('Request URL:', request.url);
  return new Response('OK');
}
```

### 3. Edge Functions 支持环境变量吗？

**支持！** 但需注意安全性。

```javascript
export default async function handler(request) {
  const apiKey = process.env.API_KEY;  // ✅ 可用

  // ⚠️ 警告：环境变量会被打包到边缘函数中
  // 敏感信息建议使用 Vercel KV 或 Edge Config
  return new Response(`API Key: ${apiKey}`);
}

export const config = { runtime: 'edge' };
```
