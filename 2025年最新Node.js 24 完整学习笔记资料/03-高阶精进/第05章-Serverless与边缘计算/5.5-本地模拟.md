# 5.5 本地模拟与调试

> 在本地环境高效开发和调试 Serverless 应用

---

## 📚 知识导读

### 为什么需要本地模拟？

**Serverless 开发的挑战：**

```
传统开发流程：
编写代码 → 本地运行 → 测试 → 提交代码
    ↑________|（即时反馈）

Serverless 开发（无本地模拟）：
编写代码 → 部署到云端 → 测试 → 修改 → 再次部署...
               ↑__________________|（反馈慢，成本高）

Serverless 开发（有本地模拟）：
编写代码 → 本地模拟 → 测试 → 提交 → 部署
    ↑________|（即时反馈，零成本）
```

**本地模拟的优势：**

1. **快速迭代**：无需等待部署
2. **离线开发**：不依赖网络
3. **降低成本**：不产生云端费用
4. **调试方便**：可使用断点调试
5. **团队协作**：统一开发环境

## 🎯 Lambda 本地模拟

### 方案 1：Serverless Offline（推荐）

**Serverless Offline** 是 Serverless Framework 的插件，模拟 Lambda + API Gateway 环境。

**安装：**

```bash
npm install --save-dev serverless-offline
```

**配置 `serverless.yml`：**

```yaml
service: my-lambda-api

frameworkVersion: '3'

provider:
  name: aws
  runtime: nodejs20.x
  region: us-east-1
  environment:
    USERS_TABLE: ${self:service}-users-${self:provider.stage}

functions:
  createUser:
    handler: src/handlers/createUser.handler
    events:
      - httpApi:
          path: /users
          method: POST

  getUser:
    handler: src/handlers/getUser.handler
    events:
      - httpApi:
          path: /users/{id}
          method: GET

plugins:
  - serverless-offline

custom:
  serverless-offline:
    httpPort: 3000  # API Gateway 端口
    websocketPort: 3001
    lambdaPort: 3002
```

**启动本地服务器：**

```bash
serverless offline

# 或简写
sls offline
```

**输出：**

```
Serverless: Starting Offline at stage dev (us-east-1)

Serverless: Routes for createUser:
POST /users

Serverless: Routes for getUser:
GET /users/{id}

Serverless: Offline [http for lambda] listening on http://localhost:3002
Serverless: Offline [HTTP] listening on http://localhost:3000
```

**测试 API：**

```bash
# 创建用户
curl -X POST http://localhost:3000/users \
  -H "Content-Type: application/json" \
  -d '{"name":"Alice","email":"alice@example.com"}'

# 获取用户
curl http://localhost:3000/users/123
```

### 方案 2：AWS SAM CLI

**AWS SAM（Serverless Application Model）** 是 AWS 官方工具，完全模拟 Lambda 运行时。

**安装：**

```bash
# macOS
brew install aws-sam-cli

# Windows
choco install aws-sam-cli

# Linux
pip install aws-sam-cli
```

**创建 `template.yaml`（SAM 模板）：**

```yaml
AWSTemplateFormatVersion: '2010-09-09'
Transform: AWS::Serverless-2016-10-31
Description: My Lambda API

Globals:
  Function:
    Timeout: 10
    MemorySize: 512
    Runtime: nodejs20.x
    Environment:
      Variables:
        USERS_TABLE: users-table

Resources:
  CreateUserFunction:
    Type: AWS::Serverless::Function
    Properties:
      CodeUri: ./
      Handler: src/handlers/createUser.handler
      Events:
        CreateUserAPI:
          Type: Api
          Properties:
            Path: /users
            Method: POST

  GetUserFunction:
    Type: AWS::Serverless::Function
    Properties:
      CodeUri: ./
      Handler: src/handlers/getUser.handler
      Events:
        GetUserAPI:
          Type: Api
          Properties:
            Path: /users/{id}
            Method: GET
```

**启动本地 API：**

```bash
sam local start-api --port 3000
```

**单独调用函数（不启动 API）：**

```bash
# 创建测试事件
cat > event.json <<EOF
{
  "body": "{\"name\":\"Alice\",\"email\":\"alice@example.com\"}",
  "httpMethod": "POST"
}
EOF

# 调用函数
sam local invoke CreateUserFunction --event event.json
```

**启动 Lambda 函数（监听端口）：**

```bash
sam local start-lambda --port 3001

# 然后使用 AWS SDK 调用
aws lambda invoke --function-name CreateUserFunction --endpoint-url http://localhost:3001 output.json
```

### 方案 3：LocalStack（模拟完整 AWS 环境）

**LocalStack** 模拟整个 AWS 生态（Lambda、DynamoDB、S3、SQS 等）。

**安装（Docker）：**

```bash
# 拉取镜像
docker pull localstack/localstack

# 启动 LocalStack
docker run --rm -it \
  -p 4566:4566 \
  -p 4571:4571 \
  -e SERVICES=lambda,dynamodb,s3,sqs \
  localstack/localstack
```

**配置 AWS CLI 使用 LocalStack：**

```bash
# 设置环境变量
export AWS_ACCESS_KEY_ID=test
export AWS_SECRET_ACCESS_KEY=test
export AWS_DEFAULT_REGION=us-east-1

# 创建 DynamoDB 表
aws dynamodb create-table \
  --table-name users \
  --attribute-definitions AttributeName=id,AttributeType=S \
  --key-schema AttributeName=id,KeyType=HASH \
  --provisioned-throughput ReadCapacityUnits=5,WriteCapacityUnits=5 \
  --endpoint-url http://localhost:4566

# 部署 Lambda 函数
aws lambda create-function \
  --function-name my-function \
  --runtime nodejs20.x \
  --role arn:aws:iam::000000000000:role/lambda-role \
  --handler index.handler \
  --zip-file fileb://function.zip \
  --endpoint-url http://localhost:4566

# 调用函数
aws lambda invoke \
  --function-name my-function \
  --payload '{"key":"value"}' \
  output.json \
  --endpoint-url http://localhost:4566
```

### 调试技巧

**使用 VS Code 断点调试：**

**1. 安装 `serverless-offline` 和 `serverless-plugin-typescript`（如果使用 TS）：**

```bash
npm install --save-dev serverless-offline serverless-plugin-typescript
```

**2. 配置 `.vscode/launch.json`：**

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Debug Serverless Offline",
      "runtimeExecutable": "npm",
      "runtimeArgs": ["run", "offline"],
      "port": 9229,
      "skipFiles": ["<node_internals>/**"],
      "console": "integratedTerminal"
    }
  ]
}
```

**3. 修改 `package.json`：**

```json
{
  "scripts": {
    "offline": "serverless offline --inspect"
  }
}
```

**4. 在代码中设置断点，按 F5 启动调试。**

**输出：**

```
⎔ Ready on http://localhost:8787
```

**特点：**

- 自动热重载
- 支持 WebSocket

---

### 方案 2：Miniflare（高级模拟）

**Miniflare** 是功能完整的 Workers 模拟器，支持本地 KV、Durable Objects、Cron Triggers。

**安装：**

```bash
npm install -g miniflare
```

**启动：**

```bash
miniflare src/index.js --port 8787 \
  --kv MY_KV \
  --durable-objects COUNTER=Counter
```

**配置文件（`.miniflare.toml`）：**

```toml
[site]
bucket = "./public"

[[kv_namespaces]]
binding = "MY_KV"

[[durable_objects.bindings]]
name = "COUNTER"
class_name = "Counter"
script_name = "my-worker"

[env.dev]
kv_persist = true
```

**特点：**
- 完整本地模拟（无需远程连接）
- 持久化 KV 和 Durable Objects 数据
- 支持 Cron Triggers、Queues
- 内置测试环境

### 场景：电商 API（Lambda + DynamoDB + S3）

**项目结构：**

```
ecommerce-api/
├── src/
│   ├── handlers/
│   │   ├── products.mjs
│   │   ├── orders.mjs
│   │   └── uploads.mjs
│   ├── lib/
│   │   └── dynamodb.mjs
│   └── utils/
│       └── response.mjs
├── tests/
│   └── integration.test.mjs
├── docker-compose.yml  # LocalStack + DynamoDB Local
├── serverless.yml
├── package.json
└── .env.local
```

---

### 1. Docker Compose 配置

```yaml
# docker-compose.yml
version: '3.8'

services:
  localstack:
    image: localstack/localstack
    ports:
      - '4566:4566'  # LocalStack 主端口
    environment:
      - SERVICES=s3,dynamodb,lambda
      - EDGE_PORT=4566
      - DEBUG=1
    volumes:
      - './localstack-data:/var/lib/localstack'
      - '/var/run/docker.sock:/var/run/docker.sock'

  dynamodb-admin:
    image: aaronshaf/dynamodb-admin
    ports:
      - '8001:8001'
    environment:
      - DYNAMO_ENDPOINT=http://localstack:4566
```

**启动服务：**

```bash
docker-compose up -d
```

---

### 2. 初始化本地资源

```bash
# init-local.sh
#!/bin/bash

# 等待 LocalStack 启动
echo "Waiting for LocalStack..."
sleep 5

# 创建 DynamoDB 表
aws dynamodb create-table \
  --table-name products \
  --attribute-definitions AttributeName=id,AttributeType=S \
  --key-schema AttributeName=id,KeyType=HASH \
  --provisioned-throughput ReadCapacityUnits=5,WriteCapacityUnits=5 \
  --endpoint-url http://localhost:4566

# 创建 S3 桶
aws s3 mb s3://product-images --endpoint-url http://localhost:4566

echo "✅ Local resources initialized!"
```

**运行：**

```bash
chmod +x init-local.sh
./init-local.sh
```

### 4. 修改代码支持本地环境

```javascript
// src/lib/dynamodb.mjs
import { DynamoDBClient } from '@aws-sdk/client-dynamodb';
import { DynamoDBDocumentClient } from '@aws-sdk/lib-dynamodb';

const isLocal = process.env.AWS_ENDPOINT !== undefined;

const client = new DynamoDBClient({
  region: process.env.AWS_REGION || 'us-east-1',
  ...(isLocal && {
    endpoint: process.env.AWS_ENDPOINT,
    credentials: {
      accessKeyId: 'test',
      secretAccessKey: 'test',
    },
  }),
});

export const docClient = DynamoDBDocumentClient.from(client);
```

### 6. 集成测试

```javascript
// tests/integration.test.mjs
import { describe, test, expect, beforeAll } from 'vitest';

const API_BASE = 'http://localhost:3000';

describe('Products API', () => {
  let productId;

  test('Create product', async () => {
    const response = await fetch(`${API_BASE}/products`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        name: 'Test Product',
        price: 99.99,
      }),
    });

    const data = await response.json();
    expect(response.status).toBe(201);
    expect(data.success).toBe(true);

    productId = data.data.id;
  });

  test('Get product', async () => {
    const response = await fetch(`${API_BASE}/products/${productId}`);
    const data = await response.json();

    expect(response.status).toBe(200);
    expect(data.data.name).toBe('Test Product');
  });

  test('List products', async () => {
    const response = await fetch(`${API_BASE}/products`);
    const data = await response.json();

    expect(response.status).toBe(200);
    expect(Array.isArray(data.data)).toBe(true);
    expect(data.data.length).toBeGreaterThan(0);
  });
});
```

```bash
npm test
```

## ⚠️ 三个高频错误

### 错误 1：环境变量未加载

**问题：** 本地运行时找不到环境变量。

```javascript
// ❌ 错误
const tableName = process.env.USERS_TABLE;  // undefined
```

**解决方案：**

```javascript
// 方法 1：使用 dotenv
import 'dotenv/config';

// 方法 2：在 serverless.yml 中配置
provider:
  environment:
    USERS_TABLE: ${env:USERS_TABLE, 'users-dev'}

// 方法 3：使用 Serverless Offline 的 env 配置
custom:
  serverless-offline:
    env:
      USERS_TABLE: users-dev
```

---

### 错误 2：LocalStack 端口冲突

**问题：** 4566 端口被占用。

```bash
# 查看端口占用
lsof -i :4566

# 修改 docker-compose.yml
ports:
  - '14566:4566'  # 使用其他端口

# 修改代码
const client = new DynamoDBClient({
  endpoint: 'http://localhost:14566',

```

### 错误 3：忘记初始化本地资源

**问题：** 启动 Serverless Offline 后，DynamoDB 表不存在。

## 📚 FAQ

### 1. 本地模拟能完全替代云端测试吗？

**不能！** 本地模拟主要用于开发阶段，部署前仍需在真实环境测试：

- **本地模拟的局限性**：
  - 无法模拟网络延迟
  - 无法模拟冷启动
  - 部分服务特性未完全实现

- **推荐流程**：
  ```
  本地开发 → 本地测试 → 部署到测试环境 → 云端测试 → 部署到生产环境
  ```

### 2. 如何模拟 Lambda 冷启动？

```bash
# Serverless Offline 不支持冷启动模拟
# 需要部署到真实环境测试

# 或使用 SAM CLI 的 --warm-containers 参数
sam local start-api --warm-containers LAZY  # 首次调用启动容器
```

### 3. 如何加速本地开发？

```bash
# 1. 使用 --watch 模式（自动重启）
serverless offline --watch

# 2. 使用 esbuild 加速打包
npm install --save-dev esbuild

# serverless.yml
plugins:
  - serverless-esbuild
  - serverless-offline

# 3. 禁用不必要的插件
# serverless.yml
plugins:
  - serverless-offline  # 只保留必需插件
```
