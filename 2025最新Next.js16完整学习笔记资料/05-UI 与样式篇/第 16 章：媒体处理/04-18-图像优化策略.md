# 图像优化策略

## 1. 概述 (Overview)

图像优化是提升网站性能的关键环节。未优化的图像会导致加载缓慢、带宽浪费、用户体验差等问题。本文将介绍在 Next.js 16 中实现图像优化的各种策略和最佳实践。

### 1.1 为什么需要图像优化

**性能影响**: 图像通常占网页总大小的 60-70%,是影响加载速度的主要因素。

**用户体验**: 慢速加载的图像会导致用户流失,研究表明页面加载时间每增加 1 秒,转化率下降 7%。

**SEO 影响**: 页面加载速度是 Google 搜索排名的重要因素。

**成本节约**: 优化图像可以减少带宽消耗,降低服务器成本。

### 1.2 优化目标

**减小文件体积**: 在保证质量的前提下尽可能减小文件大小。

**提升加载速度**: 使用懒加载、预加载等技术加快图像加载。

**改善用户体验**: 避免布局偏移,提供流畅的浏览体验。

**降低带宽成本**: 减少数据传输量,节约成本。

### 1.3 Next.js 16 的优化特性

🆕 **Next.js 16 新增**: 改进的图像缓存策略,更快的图像加载。

⚡ **Next.js 16 增强**: 更好的 AVIF 格式支持,更小的文件体积。

🆕 **Next.js 16 新增**: 自动检测图像格式,智能选择最优格式。

## 2. 图像格式选择 (Format Selection)

### 2.1 格式对比

| 格式 | 压缩类型  | 透明度 | 动画   | 浏览器支持 | 适用场景       |
| ---- | --------- | ------ | ------ | ---------- | -------------- |
| JPEG | 有损      | 不支持 | 不支持 | 100%       | 照片、复杂图像 |
| PNG  | 无损      | 支持   | 不支持 | 100%       | 图标、透明图像 |
| WebP | 有损/无损 | 支持   | 支持   | 96%        | 大部分场景     |
| AVIF | 有损/无损 | 支持   | 支持   | 80%        | 现代浏览器     |
| SVG  | 矢量      | 支持   | 支持   | 100%       | 图标、Logo     |
| GIF  | 无损      | 支持   | 支持   | 100%       | 简单动画       |

### 2.2 JPEG 优化

```bash
# 使用ImageMagick压缩JPEG
convert input.jpg -quality 85 -strip output.jpg

# 使用mozjpeg(更好的压缩)
cjpeg -quality 85 -optimize input.jpg > output.jpg
```

```tsx
// Next.js中使用JPEG
import Image from "next/image";

<Image src="/photo.jpg" alt="照片" width={800} height={600} quality={85} />;
```

### 2.3 PNG 优化

```bash
# 使用pngquant压缩PNG
pngquant --quality=65-80 input.png -o output.png

# 使用optipng优化
optipng -o7 input.png
```

```tsx
// Next.js中使用PNG

```

### 2.4 WebP 格式

```bash
# 转换为WebP
cwebp -q 80 input.jpg -o output.webp

# 批量转换
for file in *.jpg; do cwebp -q 80 "$file" -o "${file%.jpg}.webp"; done
```

```tsx

  src="/photo.jpg"
  alt="照片"
  width={800}
  height={600}

```

### 2.5 AVIF 格式

```bash

```


### 2.6 SVG 优化

```tsx
// 不同设备使用不同图像
"use client";

import Image from "next/image";
import { useState, useEffect } from "react";

export default function ResponsiveImage() {
  const [isMobile, setIsMobile] = useState(false);

  useEffect(() => {

      width={isMobile ? 400 : 1200}
      height={isMobile ? 300 : 800}
    />
  );
```

### 5.3 自定义懒加载

```tsx
"use client";

import { useEffect, useRef, useState } from "react";
import Image from "next/image";

export default function CustomLazyImage() {
  const [isVisible, setIsVisible] = useState(false);
  const imgRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          setIsVisible(true);
          observer.disconnect();
        }
      },
      { threshold: 0.1 }
    );

    if (imgRef.current) {
      observer.observe(imgRef.current);
    }

    return () => observer.disconnect();
  }, []);

  return (
    <div ref={imgRef}>
      {isVisible && (
        <Image src="/photo.jpg" alt="照片" width={800} height={600} />
      )}
    </div>
  );
}
```

### 6.1 使用 CDN

```ts
// app/utils/cdnLoader.ts
export default function cdnLoader({
  src,
  width,
  quality,
}: {
  src: string;
  width: number;
  quality?: number;
}) {
  const params = new URLSearchParams({
    url: src,
    w: width.toString(),
    q: (quality || 75).toString(),
  });
  return `https://cdn.example.com/image?${params}`;
}
```

## 12. 注意事项 (Precautions)

### 12.1 格式选择

**AVIF 优先**: 如果浏览器支持,优先使用 AVIF 格式。

**WebP 备选**: 作为 AVIF 的备选方案。

**JPEG/PNG 兜底**: 确保所有浏览器都能正常显示。

### 12.2 质量平衡

**不要过度压缩**: 质量太低会影响用户体验。

**不要过度优化**: 质量太高会增加文件大小。

**根据场景选择**: 重要图像使用高质量,缩略图使用低质量。

### 12.3 性能考虑

**首屏优先**: 优先加载首屏图像。

**懒加载**: 非首屏图像使用懒加载。

**预加载**: 预加载关键资源。

### 12.4 兼容性

**测试多浏览器**: 确保在不同浏览器中正常显示。

**提供备选方案**: 为不支持现代格式的浏览器提供备选方案。

**渐进增强**: 从基础功能开始,逐步增强。

## 14. 总结 (Summary)

### 14.1 核心要点

图像优化是提升网站性能的关键环节。通过合理选择图像格式、压缩图像、使用懒加载、CDN 等技术,我们可以大幅提升网站性能。

**格式选择**: 优先使用现代格式(AVIF、WebP)。

**压缩优化**: 在保证质量的前提下尽可能减小文件大小。

**懒加载**: 非首屏图像使用懒加载。

**CDN 加速**: 使用 CDN 加速图像加载。

### 14.2 优化策略对照表

| 策略      | 优点            | 缺点               | 适用场景   |
| --------- | --------------- | ------------------ | ---------- |
| AVIF 格式 | 体积最小        | 浏览器支持有限     | 现代浏览器 |
| WebP 格式 | 体积小,兼容性好 | 部分旧浏览器不支持 | 大部分场景 |
| 懒加载    | 减少初始加载    | 可能影响 SEO       | 非首屏图像 |
| CDN       | 加速加载        | 增加成本           | 大流量网站 |
| 压缩      | 减小体积        | 可能影响质量       | 所有场景   |
| 响应式    | 适配不同设备    | 增加复杂度         | 多设备访问 |

### 14.3 优化检查清单

- [ ] 使用现代图像格式(AVIF、WebP)
- [ ] 压缩图像(质量 75-85)
- [ ] 使用响应式图像
- [ ] 首屏图像使用 priority
- [ ] 非首屏图像使用懒加载
- [ ] 使用 blur 占位符
- [ ] 配置 CDN
- [ ] 监控性能指标
- [ ] 测试多浏览器兼容性
- [ ] 定期审查和优化

### 14.4 性能指标目标

| 指标     | 目标    | 说明           |
| -------- | ------- | -------------- |
| LCP      | < 2.5s  | 最大内容绘制   |
| FCP      | < 1.8s  | 首次内容绘制   |

图像优化是一个持续的过程。从格式选择、压缩优化、懒加载到 CDN 加速,每个环节都需要仔细考虑。通过合理使用 Next.js 提供的图像优化功能,我们可以大幅提升网站性能和用户体验。记住,优化不是一次性的工作,需要根据实际情况持续调整和改进。始终以用户体验为中心,用数据驱动优化决策,才能打造真正高性能的网站。

### 14.7 批量优化脚本

```bash
#!/bin/bash
# 批量优化图像脚本

# 创建输出目录
mkdir -p optimized

# 优化JPEG
for file in *.jpg; do
  echo "优化 $file"
  cjpeg -quality 85 -optimize "$file" > "optimized/$file"
done

# 优化PNG
for file in *.png; do
  echo "优化 $file"
  pngquant --quality=65-80 "$file" -o "optimized/$file"
done

# 转换为WebP
for file in *.{jpg,png}; do
  echo "转换 $file 为 WebP"
  cwebp -q 80 "$file" -o "optimized/${file%.*}.webp"
done

# 转换为AVIF
for file in *.{jpg,png}; do
  echo "转换 $file 为 AVIF"
  avifenc -s 4 "$file" "optimized/${file%.*}.avif"
done

echo "优化完成!"
```

### 14.10 最佳实践总结

**格式选择**:

1. 照片使用 AVIF/WebP
2. 图标使用 SVG
3. 透明图像使用 AVIF/WebP/PNG
4. 动画使用 WebP

**压缩策略**:

1. 重要图像质量 90+
2. 一般图像质量 75-85
3. 缩略图质量 50-60
4. 使用自动化工具

**加载策略**:

1. 首屏图像使用 priority
2. 非首屏图像使用懒加载
3. 使用 blur 占位符
4. 预加载关键资源

**CDN 策略**:

1. 使用 CDN 加速
2. 配置缓存策略
3. 使用多个 CDN 节点
4. 监控 CDN 性能

**监控策略**:

1. 监控 Core Web Vitals
2. 定期审查图像性能
3. 使用性能监控工具
4. 根据数据优化

通过系统化的图像优化策略,我们可以显著提升网站性能。从格式选择到压缩优化,从懒加载到 CDN 加速,每个环节都需要精心设计。记住,图像优化是一个持续的过程,需要根据实际情况不断调整和改进。
