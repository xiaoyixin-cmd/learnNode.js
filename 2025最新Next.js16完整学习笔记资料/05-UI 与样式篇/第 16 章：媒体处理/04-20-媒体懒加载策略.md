# 媒体懒加载策略

## 1. 概述 (Overview)

媒体懒加载是一种性能优化技术,通过延迟加载非首屏媒体资源来提升页面初始加载速度。在 Next.js 16 中,我们可以使用多种策略实现媒体懒加载,包括原生懒加载、Intersection Observer API、以及自定义懒加载方案。

### 1.1 为什么需要懒加载

**减少初始加载**: 只加载可见区域的媒体,减少初始页面体积。

**节省带宽**: 用户可能不会滚动到页面底部,避免加载不必要的资源。

**提升性能**: 减少网络请求,加快页面渲染速度。

**改善用户体验**: 更快的首屏加载时间。

### 1.2 懒加载的挑战

**SEO 影响**: 搜索引擎可能无法索引懒加载的内容。

**用户体验**: 加载指示器设计不当会影响体验。

**兼容性**: 需要考虑不支持懒加载的浏览器。

**性能权衡**: 过度懒加载可能导致滚动时卡顿。

### 1.3 Next.js 16 的懒加载特性

🆕 **Next.js 16 新增**: 改进的图像懒加载算法。

⚡ **Next.js 16 增强**: 更好的 Intersection Observer 支持。

🆕 **Next.js 16 新增**: 自动懒加载优化建议。

### 3.2 Intersection Observer 懒加载

```tsx
// app/components/IntersectionVideo.tsx
"use client";

import { useEffect, useRef, useState } from "react";

export default function IntersectionVideo() {
  const videoRef = useRef<HTMLVideoElement>(null);
  const [isLoaded, setIsLoaded] = useState(false);

  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting && !isLoaded) {
          setIsLoaded(true);
          observer.disconnect();
        }
      },
      { threshold: 0.25 }
    );

    if (videoRef.current) {
      observer.observe(videoRef.current);
    }

    return () => observer.disconnect();
  }, [isLoaded]);

  return (
    <video
      ref={videoRef}
      controls
      width="100%"
      height="auto"
      poster="/videos/poster.jpg"
      preload={isLoaded ? "metadata" : "none"}
    >
      {isLoaded && (
        <>
          <source src="/videos/sample.webm" type="video/webm" />
          <source src="/videos/sample.mp4" type="video/mp4" />
        </>
      )}
    </video>
  );
}
```

## 4. Intersection Observer API (Intersection Observer API)

### 4.1 基础用法

```tsx
// app/components/IntersectionObserverExample.tsx
"use client";

import { useEffect, useRef, useState } from "react";

export default function IntersectionObserverExample() {
  const elementRef = useRef<HTMLDivElement>(null);
  const [isVisible, setIsVisible] = useState(false);

  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => {
        setIsVisible(entry.isIntersecting);
      },
      {
        threshold: 0.1,
        rootMargin: "50px",
      }
    );

    if (elementRef.current) {
      observer.observe(elementRef.current);
    }

    return () => observer.disconnect();
  }, []);

  return <div ref={elementRef}>{isVisible && <p>元素可见!</p>}</div>;
}
```

### 4.2 自定义 Hook

```tsx
// app/hooks/useIntersectionObserver.ts
"use client";

import { useEffect, useRef, useState } from "react";

interface UseIntersectionObserverOptions {
  threshold?: number;
  rootMargin?: string;
  triggerOnce?: boolean;
}

export function useIntersectionObserver(
  options: UseIntersectionObserverOptions = {}
) {
  const { threshold = 0.1, rootMargin = "0px", triggerOnce = true } = options;

  const elementRef = useRef<HTMLElement>(null);
  const [isVisible, setIsVisible] = useState(false);

  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          setIsVisible(true);
          if (triggerOnce) {
            observer.disconnect();
          }
        } else if (!triggerOnce) {
          setIsVisible(false);
        }
      },
      { threshold, rootMargin }
    );

    if (elementRef.current) {
      observer.observe(elementRef.current);
    }

    return () => observer.disconnect();
  }, [threshold, rootMargin, triggerOnce]);

  return { elementRef, isVisible };
}
```

## 10. 注意事项 (Precautions)

### 10.1 SEO 影响

**搜索引擎索引**: 懒加载可能影响 SEO,重要内容不要懒加载。

**使用 noscript**: 为不支持 JavaScript 的环境提供备选方案。

```tsx
<noscript>
  <img src="/images/photo.jpg" alt="照片" />
</noscript>
```

### 10.2 性能权衡

**不要过度懒加载**: 首屏内容应该立即加载。

**合理设置阈值**: threshold 和 rootMargin 需要根据实际情况调整。

**避免布局偏移**: 使用占位符或固定尺寸避免 CLS。

### 10.3 用户体验

**提供加载指示**: 让用户知道内容正在加载。

**处理加载失败**: 提供错误处理和重试机制。

**支持无 JavaScript**: 提供降级方案。

### 10.4 兼容性

### 11.8 如何监控懒加载性能?

**问题**: 如何监控懒加载的效果?

**解决方案**: 使用 Performance API 监控。

```tsx
"use client";

import { useEffect } from "react";

export default function LazyLoadMonitor() {
  useEffect(() => {
    const observer = new PerformanceObserver((list) => {
      for (const entry of list.getEntries()) {
        if (entry.entryType === "resource") {
          const resourceEntry = entry as PerformanceResourceTiming;
          if (resourceEntry.name.match(/\.(jpg|jpeg|png|gif|webp)$/i)) {
            console.log("图像加载:", {
              url: resourceEntry.name,
              duration: resourceEntry.duration,
              size: resourceEntry.transferSize,
            });
          }
        }
      }
    });

    observer.observe({ entryTypes: ["resource"] });

    return () => observer.disconnect();
  }, []);

  return null;
}
```

// app/sw.ts
self.addEventListener("fetch", (event: any) => {
  if (event.request.destination === "image") {
    event.respondWith(
      caches.match(event.request).then((response) => {
        return (
          response ||
          fetch(event.request).then((fetchResponse) => {

### 11.10 如何实现预加载?

**问题**: 如何预加载即将进入视口的图像?

```tsx
"use client";

import { useIntersectionObserver } from "@/hooks/useIntersectionObserver";
import Image from "next/image";

export default function PreloadImage() {
  const { elementRef, isVisible } = useIntersectionObserver({
    threshold: 0,
    rootMargin: "200px", // 提前200px加载

  return (
    <div ref={elementRef as any}>

      )}
    </div>
  );
}
```

## 12. 总结 (Summary)

### 12.1 核心要点

媒体懒加载是提升网站性能的重要技术。通过合理使用原生懒加载、Intersection Observer、虚拟化列表等技术,我们可以显著减少初始加载时间和带宽消耗。

**原生懒加载**: 简单易用,适合大部分场景。

**Intersection Observer**: 灵活强大,适合复杂场景。

**虚拟化列表**: 处理大量数据的最佳方案。

**渐进式加载**: 提供更好的用户体验。

### 12.2 策略对比表

| 策略                  | 优点         | 缺点         | 适用场景     |
| --------------------- | ------------ | ------------ | ------------ |
| 原生 loading="lazy"   | 简单,无需 JS | 控制有限     | 简单场景     |
| Intersection Observer | 灵活,可控    | 需要编写代码 | 复杂场景     |
| 虚拟化列表            | 性能最佳     | 实现复杂     | 大量数据     |
| 按需加载              | 节省带宽     | 需要用户交互 | 视频等大文件 |
| 渐进式加载            | 体验好       | 增加复杂度   | 重要图像     |
