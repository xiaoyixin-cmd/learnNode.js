# 视频处理与优化

## 1. 概述 (Overview)

视频是现代网站中越来越重要的内容形式,但也是最消耗带宽和影响性能的资源之一。在 Next.js 16 中,我们需要采用合适的策略来处理和优化视频,确保在提供良好用户体验的同时不影响网站性能。

### 1.1 视频的挑战

**文件体积大**: 视频文件通常比图像大得多,一个 1 分钟的 1080p 视频可能超过 100MB。

**带宽消耗**: 视频会消耗大量带宽,特别是在移动网络环境下。

**加载时间**: 大文件需要更长的加载时间,影响用户体验。

**设备兼容性**: 不同设备和浏览器支持的视频格式不同。

### 1.2 优化目标

**减小文件体积**: 通过压缩和格式选择减小视频文件大小。

**提升加载速度**: 使用流式传输、懒加载等技术加快加载。

**改善用户体验**: 提供流畅的播放体验,避免卡顿。

**降低带宽成本**: 减少数据传输量,节约成本。

### 1.3 Next.js 16 的视频特性

🆕 **Next.js 16 新增**: 改进的视频缓存策略。

⚡ **Next.js 16 增强**: 更好的流式传输支持。

🆕 **Next.js 16 新增**: 自动视频优化建议。

## 3. 视频压缩 (Video Compression)

### 3.1 使用 FFmpeg 压缩

```bash
# 压缩MP4视频
ffmpeg -i input.mp4 -c:v libx264 -crf 23 -preset medium -c:a aac -b:a 128k output.mp4

# 转换为WebM (VP9)
ffmpeg -i input.mp4 -c:v libvpx-vp9 -crf 30 -b:v 0 -c:a libopus output.webm

# 转换为WebM (AV1)
ffmpeg -i input.mp4 -c:v libaom-av1 -crf 30 -b:v 0 -c:a libopus output.webm

# 生成多个分辨率
ffmpeg -i input.mp4 -vf scale=1920:1080 -c:v libx264 -crf 23 output_1080p.mp4
ffmpeg -i input.mp4 -vf scale=1280:720 -c:v libx264 -crf 23 output_720p.mp4
ffmpeg -i input.mp4 -vf scale=854:480 -c:v libx264 -crf 23 output_480p.mp4
```

### 3.2 压缩参数说明

**CRF (Constant Rate Factor)**: 控制质量,范围 0-51,值越小质量越高。

- 18-23: 高质量
- 23-28: 中等质量
- 28-35: 低质量

**Preset**: 控制编码速度和压缩率。

- ultrafast: 最快,压缩率最低
- medium: 平衡
- veryslow: 最慢,压缩率最高

### 3.3 批量压缩脚本

```bash
#!/bin/bash
# 批量压缩视频脚本

mkdir -p compressed

for file in *.mp4; do
  echo "压缩 $file"

  # 生成MP4
  ffmpeg -i "$file" -c:v libx264 -crf 23 -preset medium -c:a aac -b:a 128k "compressed/${file}"

  # 生成WebM
  ffmpeg -i "$file" -c:v libvpx-vp9 -crf 30 -b:v 0 -c:a libopus "compressed/${file%.mp4}.webm"
done

echo "压缩完成!"
```

## 5. 懒加载策略 (Lazy Loading)

### 5.1 基础懒加载

```tsx
// app/components/LazyVideo.tsx
"use client";

import { useEffect, useRef, useState } from "react";

export default function LazyVideo() {
  const videoRef = useRef<HTMLVideoElement>(null);
  const [isLoaded, setIsLoaded] = useState(false);

  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting && !isLoaded) {
          setIsLoaded(true);
          observer.disconnect();
        }
      },
      { threshold: 0.25 }
    );

    if (videoRef.current) {
      observer.observe(videoRef.current);
    }

    return () => observer.disconnect();
  }, [isLoaded]);

  return (
    <video
      ref={videoRef}
      controls
      width="100%"
      height="auto"
      poster="/videos/poster.jpg"
      preload={isLoaded ? "metadata" : "none"}
    >
      {isLoaded && (
        <>
          <source src="/videos/sample.webm" type="video/webm" />
          <source src="/videos/sample.mp4" type="video/mp4" />
        </>
      )}
    </video>
  );
}
```

## 6. 视频播放器 (Video Player)

### 6.1 自定义播放器

```tsx
// app/components/CustomPlayer.tsx
"use client";

import { useRef, useState } from "react";

export default function CustomPlayer() {
  const videoRef = useRef<HTMLVideoElement>(null);
  const [isPlaying, setIsPlaying] = useState(false);
  const [currentTime, setCurrentTime] = useState(0);
  const [duration, setDuration] = useState(0);
  const [volume, setVolume] = useState(1);

  const togglePlay = () => {
    if (videoRef.current) {
      if (isPlaying) {
        videoRef.current.pause();
      } else {
        videoRef.current.play();
      }
      setIsPlaying(!isPlaying);
    }
  };

  const handleTimeUpdate = () => {
    if (videoRef.current) {
      setCurrentTime(videoRef.current.currentTime);
    }
  };

  const handleLoadedMetadata = () => {
    if (videoRef.current) {
      setDuration(videoRef.current.duration);
    }
  };

  const handleSeek = (e: React.ChangeEvent<HTMLInputElement>) => {
    const time = parseFloat(e.target.value);
    if (videoRef.current) {
      videoRef.current.currentTime = time;
      setCurrentTime(time);
    }
  };

  const handleVolumeChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const vol = parseFloat(e.target.value);
    if (videoRef.current) {
      videoRef.current.volume = vol;
      setVolume(vol);
    }
  };

  const formatTime = (time: number) => {
    const minutes = Math.floor(time / 60);
    const seconds = Math.floor(time % 60);
    return `${minutes}:${seconds.toString().padStart(2, "0")}`;
  };

  return (
    <div className="relative">
      <video
        ref={videoRef}
        width="100%"
        height="auto"
        onTimeUpdate={handleTimeUpdate}
        onLoadedMetadata={handleLoadedMetadata}
        poster="/videos/poster.jpg"
      >
        <source src="/videos/sample.mp4" type="video/mp4" />
      </video>

      <div className="bg-gray-800 text-white p-4">
        <button onClick={togglePlay} className="px-4 py-2 bg-blue-500 rounded">
          {isPlaying ? "暂停" : "播放"}
        </button>

        <div className="mt-2">
          <input
            type="range"
            min="0"
            max={duration}
            value={currentTime}
            onChange={handleSeek}
            className="w-full"
          />
          <div className="flex justify-between text-sm">
            <span>{formatTime(currentTime)}</span>
            <span>{formatTime(duration)}</span>
          </div>
        </div>

        <div className="mt-2 flex items-center">
          <span className="mr-2">音量:</span>
          <input
            type="range"
            min="0"
            max="1"
            step="0.1"
            value={volume}
            onChange={handleVolumeChange}
            className="w-32"
          />
        </div>
      </div>
    </div>
  );
}
```

### 8.3 范围请求

```tsx
// app/api/video-range/route.ts
import { NextRequest } from "next/server";
import fs from "fs";
import path from "path";

export async function GET(request: NextRequest) {
  const videoPath = path.join(process.cwd(), "public", "videos", "sample.mp4");
  const stat = fs.statSync(videoPath);
  const fileSize = stat.size;
  const range = request.headers.get("range");

  if (range) {
    const parts = range.replace(/bytes=/, "").split("-");
    const start = parseInt(parts[0], 10);
    const end = parts[1] ? parseInt(parts[1], 10) : fileSize - 1;
    const chunksize = end - start + 1;
    const file = fs.createReadStream(videoPath, { start, end });

    return new Response(file as any, {
      status: 206,
      headers: {
        "Content-Range": `bytes ${start}-${end}/${fileSize}`,
        "Accept-Ranges": "bytes",
        "Content-Length": chunksize.toString(),
        "Content-Type": "video/mp4",
      },
    });
  } else {
    const file = fs.createReadStream(videoPath);
    return new Response(file as any, {
      headers: {
        "Content-Length": fileSize.toString(),
        "Content-Type": "video/mp4",
      },
    });
  }
}
```

## 11. 注意事项 (Precautions)

### 11.1 格式兼容性

**测试多浏览器**: 确保视频在不同浏览器中正常播放。

**提供备选格式**: 至少提供 MP4 和 WebM 两种格式。

**检查编解码器**: 确保使用广泛支持的编解码器。

### 11.2 文件大小

**控制视频时长**: 尽量将视频控制在合理的时长内。

**选择合适的分辨率**: 根据使用场景选择分辨率。

**使用压缩**: 在保证质量的前提下尽可能压缩。

### 11.3 性能影响

**避免自动播放**: 自动播放会消耗带宽和影响性能。

**使用懒加载**: 非首屏视频使用懒加载。

**限制同时播放**: 避免同时播放多个视频。

### 11.4 用户体验

**提供控制选项**: 让用户控制播放、暂停、音量等。

**显示加载状态**: 提供加载指示器。

**支持键盘操作**: 确保可以使用键盘控制。

## 12. 常见问题 (FAQ)

### 12.1 如何选择视频格式?

**问题**: 应该使用哪种视频格式?

**解决方案**:

| 场景       | 推荐格式    |
| ---------- | ----------- |
| 通用场景   | MP4 (H.264) |
| 现代浏览器 | WebM (VP9)  |
| 最新浏览器 | WebM (AV1)  |
| 流式传输   | HLS 或 DASH |
| 直播       | HLS         |

建议提供多种格式以确保兼容性:

```tsx
<video controls>
  <source src="/video.webm" type="video/webm" />
  <source src="/video.mp4" type="video/mp4" />
</video>
```

### 12.2 如何减小视频文件大小?

**问题**: 视频文件太大怎么办?

**解决方案**:

1. 降低分辨率
2. 降低比特率
3. 使用更高效的编解码器(VP9、AV1)
4. 调整 CRF 值
5. 缩短视频时长

```bash
# 压缩示例
ffmpeg -i input.mp4 -c:v libx264 -crf 28 -preset medium -c:a aac -b:a 96k output.mp4
```

### 12.3 如何实现自适应比特率?

**问题**: 如何根据网络状况自动调整视频质量?

**解决方案**: 使用 HLS 或 DASH 流式传输。

```bash
# 生成HLS流
ffmpeg -i input.mp4 \
  -c:v libx264 -b:v 5000k -s 1920x1080 -f hls -hls_time 10 -hls_list_size 0 1080p.m3u8 \
  -c:v libx264 -b:v 2800k -s 1280x720 -f hls -hls_time 10 -hls_list_size 0 720p.m3u8 \
  -c:v libx264 -b:v 1400k -s 854x480 -f hls -hls_time 10 -hls_list_size 0 480p.m3u8
```

### 12.4 如何优化视频加载速度?

**问题**: 视频加载太慢怎么办?

**解决方案**:

1. 使用 CDN
2. 使用流式传输
3. 压缩视频
4. 使用懒加载
5. 预加载元数据

```tsx
<video controls preload="metadata" poster="/poster.jpg">
  <source src="https://cdn.example.com/video.mp4" type="video/mp4" />
</video>
```

### 12.5 如何添加字幕?

**问题**: 如何为视频添加字幕?

**解决方案**: 使用 WebVTT 格式的字幕文件。

```tsx
<video controls>
  <source src="/video.mp4" type="video/mp4" />
  <track
    kind="subtitles"
    src="/subtitles-zh.vtt"
    srcLang="zh"
    label="中文"
    default
  />
</video>
```

### 12.6 如何处理视频在移动设备上的播放?

**问题**: 视频在移动设备上播放有问题怎么办?

**解决方案**:

1. 使用响应式设计
2. 提供多种分辨率
3. 避免自动播放(iOS 限制)
4. 使用 playsinline 属性

```tsx
<video controls playsInline width="100%" height="auto">
  <source src="/video.mp4" type="video/mp4" />
</video>
```
