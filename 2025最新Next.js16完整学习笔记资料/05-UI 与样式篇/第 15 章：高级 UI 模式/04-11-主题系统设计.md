# 主题系统设计

## 1. 概述 (Overview)

主题系统是现代 Web 应用中不可或缺的功能,它不仅仅是简单的颜色切换,而是一套完整的设计语言和视觉规范。一个好的主题系统能够让应用在保持品牌一致性的同时,为用户提供个性化的视觉体验。在 Next.js 16 中,我们可以构建从简单到复杂的各种主题系统。

### 1.1 主题系统的核心价值

**品牌一致性**: 通过统一的设计令牌(Design Tokens),确保整个应用的视觉风格保持一致。

**可维护性**: 集中管理颜色、字体、间距等设计元素,修改主题时只需要改动一处。

**可扩展性**: 支持多种主题变体,可以轻松添加新主题而不影响现有代码。

**用户体验**: 让用户可以根据自己的喜好选择主题,提升应用的个性化程度。

### 1.2 主题系统的组成部分

一个完整的主题系统通常包含以下几个部分:

**设计令牌(Design Tokens)**:

- 颜色系统(主色、辅助色、语义色)
- 字体系统(字体族、字号、行高、字重)
- 间距系统(padding、margin、gap)
- 圆角系统(border-radius)
- 阴影系统(box-shadow)
- 动画系统(transition、animation)

**主题变体**:

- 亮色主题
- 暗色主题
- 高对比度主题
- 自定义主题

**主题管理**:

- 主题切换逻辑
- 主题持久化
- 主题预览
- 主题导入导出

### 1.3 Next.js 16 中的主题系统特点

🆕 **Next.js 16 新增**: 服务端组件可以在渲染时读取主题配置,实现更好的首屏性能。

⚡ **Next.js 16 增强**: React Compiler 自动优化主题相关的组件,减少不必要的重渲染。

**核心优势**:

- 支持服务端渲染,避免主题闪烁
- 与 TypeScript 深度集成,提供完整的类型安全
- 支持 CSS 变量、CSS-in-JS、Tailwind 等多种方案
- 性能优化,主题切换流畅

### 2.3 主题上下文

#### 2.3.1 创建主题上下文

```typescript
// lib/theme/ThemeContext.tsx
"use client";

import {
  createContext,
  useContext,
  useState,
  useEffect,
  ReactNode,
} from "react";
import { Theme, ThemeMode } from "./types";
import { lightTheme } from "./themes/light";
import { darkTheme } from "./themes/dark";

interface ThemeContextType {
  theme: Theme;
  mode: ThemeMode;
  setMode: (mode: ThemeMode) => void;
  toggleMode: () => void;
}

const ThemeContext = createContext<ThemeContextType | undefined>(undefined);

export function ThemeProvider({ children }: { children: ReactNode }) {
  const [mode, setModeState] = useState<ThemeMode>("system");
  const [theme, setTheme] = useState<Theme>(lightTheme);

  // 获取系统主题偏好
  const getSystemTheme = (): "light" | "dark" => {
    if (typeof window === "undefined") return "light";
    return window.matchMedia("(prefers-color-scheme: dark)").matches
      ? "dark"
      : "light";
  };

  // 计算实际主题
  const getActualTheme = (mode: ThemeMode): Theme => {
    if (mode === "system") {
      return getSystemTheme() === "dark" ? darkTheme : lightTheme;
    }
    return mode === "dark" ? darkTheme : lightTheme;
  };

  // 设置主题模式
  const setMode = (newMode: ThemeMode) => {
    setModeState(newMode);
    localStorage.setItem("theme-mode", newMode);
    setTheme(getActualTheme(newMode));
  };

  // 切换主题
  const toggleMode = () => {
    const newMode = mode === "light" ? "dark" : "light";
    setMode(newMode);
  };

  // 初始化主题
  useEffect(() => {
    const savedMode = localStorage.getItem("theme-mode") as ThemeMode | null;
    const initialMode = savedMode || "system";
    setModeState(initialMode);
    setTheme(getActualTheme(initialMode));

    // 监听系统主题变化
    const mediaQuery = window.matchMedia("(prefers-color-scheme: dark)");
    const handleChange = () => {
      if (mode === "system") {
        setTheme(getActualTheme("system"));
      }
    };

    mediaQuery.addEventListener("change", handleChange);
    return () => mediaQuery.removeEventListener("change", handleChange);
  }, [mode]);

  return (
    <ThemeContext.Provider value={{ theme, mode, setMode, toggleMode }}>
      {children}
    </ThemeContext.Provider>
  );
}

export function useTheme() {
  const context = useContext(ThemeContext);
  if (!context) {
    throw new Error("useTheme must be used within ThemeProvider");
  }
  return context;
}
```

```typescript
// lib/theme/tenant.ts
interface TenantTheme {
  tenantId: string;
  theme: Theme;
  customCSS?: string;
}

export class TenantThemeManager {
  private themes: Map<string, TenantTheme> = new Map();

  async loadTenantTheme(tenantId: string): Promise<Theme> {
    // 从API加载租户主题
    const response = await fetch(`/api/tenants/${tenantId}/theme`);
    const themeData = await response.json();

    const theme = this.parseThemeData(themeData);
    this.themes.set(tenantId, { tenantId, theme });

    return theme;
  }

  getTenantTheme(tenantId: string): Theme | undefined {
    return this.themes.get(tenantId)?.theme;
  }

  private parseThemeData(data: any): Theme {
    // 解析主题数据
    return {
      ...lightTheme,
      colors: {
        ...lightTheme.colors,
        ...data.colors,
      },
    };
  }
}
```

### 3.2 不推荐的场景

#### 3.2.1 简单的静态网站

如果你的网站只是简单的展示页面,不需要复杂的主题系统。

**原因**:

- 增加不必要的复杂度
- 影响性能
- 维护成本高

**替代方案**:

- 使用简单的 CSS 变量
- 只提供亮色/暗色两种主题
- 使用 Tailwind 的暗色模式

#### 3.2.2 性能敏感的应用

**原因**:

- 主题系统会增加 JavaScript 包大小
- 主题切换可能导致重渲染
- 影响首屏加载时间

**替代方案**:

- 使用纯 CSS 方案
- 减少主题令牌数量
- 使用服务端渲染固定主题

```typescript
// lib/theme/utils/color.ts

/**
 * 将十六进制颜色转换为RGB
 */
export function hexToRgb(
  hex: string
): { r: number; g: number; b: number } | null {
  const result = /^#?([a-f\d]{2})([a-f\d]{2})([a-f\d]{2})$/i.exec(hex);
  return result
    ? {
        r: parseInt(result[1], 16),
        g: parseInt(result[2], 16),
        b: parseInt(result[3], 16),
      }
    : null;
}

/**
 * 将RGB转换为十六进制
 */
export function rgbToHex(r: number, g: number, b: number): string {
  return (
    "#" +
    [r, g, b]
      .map((x) => {
        const hex = x.toString(16);
        return hex.length === 1 ? "0" + hex : hex;
      })
      .join("")
  );
}

/**
 * 调整颜色亮度
 */
export function adjustBrightness(hex: string, percent: number): string {
  const rgb = hexToRgb(hex);
  if (!rgb) return hex;

  const adjust = (value: number) => {
    const adjusted = Math.round(value + (255 - value) * (percent / 100));
    return Math.max(0, Math.min(255, adjusted));
  };

  return rgbToHex(adjust(rgb.r), adjust(rgb.g), adjust(rgb.b));
}

/**
 * 获取颜色的对比色
 */
export function getContrastColor(hex: string): string {
  const rgb = hexToRgb(hex);
  if (!rgb) return "#000000";

  // 计算相对亮度
  const luminance = (0.299 * rgb.r + 0.587 * rgb.g + 0.114 * rgb.b) / 255;

  return luminance > 0.5 ? "#000000" : "#ffffff";
}

/**
 * 混合两个颜色
 */
export function mixColors(
  color1: string,
  color2: string,
  weight: number = 0.5
): string {
  const rgb1 = hexToRgb(color1);
  const rgb2 = hexToRgb(color2);

  if (!rgb1 || !rgb2) return color1;

  const mix = (v1: number, v2: number) =>
    Math.round(v1 * (1 - weight) + v2 * weight);

  return rgbToHex(
    mix(rgb1.r, rgb2.r),
    mix(rgb1.g, rgb2.g),
    mix(rgb1.b, rgb2.b)
  );
}
```

```typescript
/**
 * 导出主题为JSON

export function exportTheme(theme: Theme): string {
  return JSON.stringify(theme, null, 2);
}

/**
 * 从JSON导入主题
 */
export function importTheme(json: string): Theme {
  try {
    const parsed = JSON.parse(json);
    // 验证主题结构
    const { valid, errors } = validateTheme(parsed);
    if (!valid) {
      throw new Error(`主题验证失败: ${errors.join(", ")}`);
    }
    return parsed;
  } catch (error) {
    throw new Error(`导入主题失败: ${error.message}`);
  }
}

/**
 * 导出为CSS变量
 */
export function exportToCSSVariables(theme: Theme): string {
  const lines: string[] = [":root {"];

  // 导出颜色
  Object.entries(theme.colors).forEach(([category, colors]) => {
    Object.entries(colors).forEach(([name, value]) => {
      lines.push(`  --color-${category}-${name}: ${value};`);
    });
  });

  // 导出间距
  Object.entries(theme.spacing).forEach(([key, value]) => {
    lines.push(`  --spacing-${key}: ${value};`);
  });

  lines.push("}");
  return lines.join("\n");
}
```

## 6. 注意事项 (Precautions)

### 6.1 性能注意事项

**避免过度使用 CSS-in-JS**: CSS-in-JS 方案会增加运行时开销,对于大型应用可能影响性能。如果不需要动态主题,优先使用 CSS 变量或 Tailwind。

**主题令牌数量控制**: 不要定义过多的主题令牌,这会增加维护成本和包大小。只定义真正需要的令牌。

**避免主题闪烁**: 在服务端渲染时,确保主题脚本在 HTML 头部执行,避免页面加载时出现主题闪烁。

```tsx
// 正确做法
<html suppressHydrationWarning>
  <head>
    <script dangerouslySetInnerHTML={{ __html: themeScript }} />
  </head>
  <body>{children}</body>
</html>
```

**优化主题切换性能**: 使用 CSS 变量而不是重新渲染整个组件树。

```typescript
// 不好的做法 - 会导致整个应用重渲染
const [theme, setTheme] = useState(lightTheme);

// 好的做法 - 只改变CSS变量
document.documentElement.setAttribute("data-theme", "dark");
```

### 6.2 可访问性注意事项

**确保足够的对比度**: 主题颜色必须满足 WCAG 2.1 的对比度要求(AA 级别至少 4.5:1)。

```typescript
// lib/theme/utils/accessibility.ts
export function checkContrast(foreground: string, background: string): number {
  const getLuminance = (hex: string) => {
    const rgb = hexToRgb(hex);
    if (!rgb) return 0;

    const [r, g, b] = [rgb.r, rgb.g, rgb.b].map((val) => {
      val = val / 255;
      return val <= 0.03928
        ? val / 12.92
        : Math.pow((val + 0.055) / 1.055, 2.4);
    });

    return 0.2126 * r + 0.7152 * g + 0.0722 * b;
  };

  const l1 = getLuminance(foreground);
  const l2 = getLuminance(background);
  const lighter = Math.max(l1, l2);
  const darker = Math.min(l1, l2);

  return (lighter + 0.05) / (darker + 0.05);
}

// 使用
const contrast = checkContrast("#000000", "#ffffff"); // 21
if (contrast < 4.5) {
  console.warn("对比度不足,可能影响可读性");
}
```

```typescript
  if (hexRegex.test(color)) return true;

  // 验证RGB颜色
  const rgbRegex = /^rgb\(\s*\d{1,3}\s*,\s*\d{1,3}\s*,\s*\d{1,3}\s*\)$/;
  if (rgbRegex.test(color)) return true;

  return false;
}
```

**防止 XSS 攻击**: 不要直接将用户输入的主题数据插入到 DOM 中。

```typescript
// 危险 - 可能导致XSS
dangerouslySetInnerHTML={{ __html: userThemeCSS }}

// 安全 - 使用CSS变量
document.documentElement.style.setProperty('--user-color', sanitizedColor);
```

## 7. 常见问题 (FAQ)

### 7.1 主题切换时出现闪烁怎么办?

**问题**: 页面加载时,先显示默认主题,然后才切换到用户选择的主题,导致闪烁。

**解决方案**: 在 HTML 头部注入阻塞脚本,在页面渲染前设置主题。

```tsx
// app/layout.tsx
const themeScript = `
(function() {
  const theme = localStorage.getItem('theme') || 'system';
  const systemTheme = window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
  const actualTheme = theme === 'system' ? systemTheme : theme;
  document.documentElement.setAttribute('data-theme', actualTheme);
})();
`;

export default function RootLayout({ children }) {
  return (
    <html suppressHydrationWarning>
      <head>
        <script dangerouslySetInnerHTML={{ __html: themeScript }} />
      </head>
      <body>{children}</body>
    </html>
  );
}
```

### 7.2 如何在服务端组件中使用主题?

**问题**: 服务端组件无法访问客户端状态,如何获取当前主题?

**解决方案**: 使用 cookies 或 headers 传递主题信息。

```typescript
// app/components/ServerThemedComponent.tsx
import { cookies } from "next/headers";

export default async function ServerThemedComponent() {
  const cookieStore = await cookies();
  const theme = cookieStore.get("theme")?.value || "light";

  return <div data-theme={theme}>{/* 内容 */}</div>;
}

// middleware.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function middleware(request: NextRequest) {
  const theme = request.cookies.get("theme")?.value || "light";
  const response = NextResponse.next();

  // 确保cookie存在
  if (!request.cookies.has("theme")) {
    response.cookies.set("theme", theme);
  }

  return response;
}
```

### 7.3 如何实现主题预览功能?

**问题**: 用户在选择主题前想要预览效果。

**解决方案**: 使用临时状态和 Portal。

```tsx
// app/components/ThemePreview.tsx
"use client";

import { useState } from "react";
import { createPortal } from "react-dom";

export default function ThemePreview({ theme }) {
  const [isPreview, setIsPreview] = useState(false);

  const handlePreview = () => {
    setIsPreview(true);
    // 临时应用主题
    const originalTheme = document.documentElement.getAttribute("data-theme");
    document.documentElement.setAttribute("data-theme", theme);

    // 3秒后恢复
    setTimeout(() => {
      document.documentElement.setAttribute("data-theme", originalTheme);
      setIsPreview(false);
    }, 3000);
  };

  return (
    <>
      <button onClick={handlePreview}>预览主题</button>
      {isPreview &&
        createPortal(
          <div className="fixed top-4 right-4 bg-black text-white px-4 py-2 rounded">
            预览中...
          </div>,
          document.body
        )}
    </>
  );
}
```

### 7.4 如何处理第三方组件的主题?

**问题**: 第三方组件库有自己的主题系统,如何与应用主题保持一致?

**解决方案**: 使用 CSS 变量桥接或包装组件。

```tsx
// lib/theme/third-party-bridge.ts
import { useTheme } from "./ThemeContext";

export function useThirdPartyTheme() {
  const { theme } = useTheme();

  // 转换为第三方库的主题格式
  return {
    palette: {
      primary: {
        main: theme.colors.brand.primary,
      },
      background: {
        default: theme.colors.background.primary,
      },
    },
  };
}

// 使用
import { ThemeProvider as MuiThemeProvider } from "@mui/material";

function App() {
  const muiTheme = useThirdPartyTheme();

  return <MuiThemeProvider theme={muiTheme}>{/* 内容 */}</MuiThemeProvider>;
}
```

### 7.5 如何优化主题切换性能?

**问题**: 主题切换时整个应用重渲染,性能较差。

**解决方案**: 使用 CSS 变量而不是 React 状态。

```typescript
// 不好的做法
const [theme, setTheme] = useState(lightTheme);

// 好的做法
const setTheme = (themeName: string) => {
  document.documentElement.setAttribute("data-theme", themeName);
  localStorage.setItem("theme", themeName);
};
```

### 7.6 如何支持多个主题变体?

**问题**: 除了亮色/暗色,还需要支持多个品牌主题。

**解决方案**: 使用主题名称而不是模式。

```typescript
// lib/theme/themes/index.ts
export const themes = {
  light: lightTheme,
  dark: darkTheme,
  ocean: oceanTheme,
  forest: forestTheme,
  sunset: sunsetTheme,
};

export type ThemeName = keyof typeof themes;

// 使用
const [themeName, setThemeName] = useState<ThemeName>("light");
const theme = themes[themeName];
```

### 7.7 如何实现主题的渐变过渡?

**问题**: 主题切换太突兀,想要平滑过渡。

**解决方案**: 使用 CSS 过渡或 View Transitions API。

```typescript
// 方案1: CSS过渡
document.documentElement.style.transition =
  "background-color 300ms, color 300ms";
document.documentElement.setAttribute("data-theme", newTheme);
setTimeout(() => {
  document.documentElement.style.transition = "";
}, 300);

// 方案2: View Transitions API
if ("startViewTransition" in document) {
  (document as any).startViewTransition(() => {
    document.documentElement.setAttribute("data-theme", newTheme);
  });
} else {
  document.documentElement.setAttribute("data-theme", newTheme);
}
```

### 7.8 如何处理主题的 TypeScript 类型?

**问题**: 如何确保主题使用的类型安全?

**解决方案**: 使用严格的类型定义和类型守卫。

```typescript
// lib/theme/types.ts
export interface Theme {
  name: string;
  colors: {
    background: {
      primary: string;
      secondary: string;
    };
    // ...
  };
}

// 类型守卫
export function isValidTheme(obj: any): obj is Theme {
  return (
    typeof obj === "object" &&
    typeof obj.name === "string" &&
    typeof obj.colors === "object" &&
    typeof obj.colors.background === "object" &&
    typeof obj.colors.background.primary === "string"
  );
}

// 使用
const loadTheme = (data: unknown): Theme => {
  if (!isValidTheme(data)) {
    throw new Error("Invalid theme data");
  }
  return data;
};
```

## 8. 总结 (Summary)

### 8.1 核心要点

主题系统是现代 Web 应用的重要组成部分,一个好的主题系统应该具备以下特点:

**设计令牌化**: 将所有设计决策转化为可复用的令牌,包括颜色、字体、间距等。这样可以确保整个应用的视觉一致性,并且方便维护和扩展。

**类型安全**: 使用 TypeScript 定义严格的类型,确保主题使用的正确性。这可以在开发阶段就发现问题,避免运行时错误。

**性能优化**: 优先使用 CSS 变量而不是 JavaScript 状态,避免不必要的重渲染。在服务端渲染时注入主题脚本,防止闪烁。

**可访问性**: 确保所有主题都满足 WCAG 对比度要求,支持系统主题偏好,提供清晰的焦点指示器。

**可扩展性**: 设计灵活的主题架构,支持多个主题变体,方便添加新主题。提供主题导入导出功能,支持用户自定义。

### 8.2 最佳实践总结

| 方面     | 推荐做法                 | 避免做法               |
| -------- | ------------------------ | ---------------------- |
| 实现方案 | CSS 变量 + Tailwind      | 纯 JavaScript 状态管理 |
| 主题切换 | 在 HTML 头部注入脚本     | 在 useEffect 中切换    |
| 类型安全 | 使用 TypeScript 严格类型 | 使用 any 类型          |
| 性能优化 | 使用 CSS 变量            | 重渲染整个组件树       |
| 可访问性 | 检查对比度,支持系统偏好  | 忽略可访问性           |
| 维护性   | 文档化令牌,版本控制      | 硬编码颜色值           |

### 8.3 技术选型建议

**小型项目**: 使用 CSS 变量 + 简单的主题切换脚本,不需要复杂的主题系统。

**中型项目**: 使用 Tailwind CSS 的暗色模式 + CSS 变量,配合 next-themes 库。

**大型项目**: 构建完整的主题系统,包括设计令牌、主题管理、主题编辑器等。使用 TypeScript 确保类型安全。

**企业级项目**: 实现可配置的主题系统,支持白标定制、多租户、主题导入导出等高级功能。

### 8.4 未来展望

主题系统的发展趋势:

**View Transitions API**: 浏览器原生支持的页面过渡动画,可以实现更流畅的主题切换效果。

**CSS Color Module Level 4**: 新的 CSS 颜色函数,如 color-mix()、color-contrast(),可以简化主题颜色的生成和调整。

**Container Queries**: 容器查询可以让组件根据容器大小而不是视口大小调整样式,这对主题系统很有用。

**CSS Nesting**: 原生 CSS 嵌套支持,可以简化主题样式的编写。
