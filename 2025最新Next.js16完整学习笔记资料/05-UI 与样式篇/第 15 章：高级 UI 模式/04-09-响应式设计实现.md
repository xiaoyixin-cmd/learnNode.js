# 响应式设计实现

## 1. 概述 (Overview)

响应式设计是现代 Web 应用的基础能力,它让应用能够在不同设备和屏幕尺寸上提供一致且优质的用户体验。在 Next.js 16 中,响应式设计不仅仅是 CSS 媒体查询的应用,更涉及到服务端组件、客户端组件、图片优化、布局系统等多个层面的协同工作。

### 1.1 响应式设计的核心价值

**用户体验统一**: 无论用户使用手机、平板还是桌面设备,都能获得针对其设备优化的界面体验。

**性能优化**: 根据设备能力和屏幕尺寸,加载适当大小的资源,避免浪费带宽。

**维护成本降低**: 一套代码适配多端,相比为每个平台单独开发应用,大幅降低开发和维护成本。

**SEO 友好**: Google 等搜索引擎优先索引移动友好的网站,响应式设计是 SEO 的重要因素。

### 1.2 Next.js 16 中的响应式设计特点

🆕 **Next.js 16 新增**: 增强的服务端组件支持让响应式设计可以在服务端就做出决策,减少客户端的计算负担。

⚡ **Next.js 16 增强**: Turbopack 的快速构建能力让开发者可以更快地预览不同断点下的效果。

**核心特点**:

- **混合渲染策略**: 结合服务端和客户端组件,实现最优的响应式体验
- **智能图片优化**: next/image 组件自动根据设备提供最佳尺寸的图片
- **CSS 方案灵活**: 支持 Tailwind CSS、CSS Modules、CSS-in-JS 等多种响应式方案
- **性能优先**: 通过代码分割和懒加载,确保移动端性能

### 1.3 响应式设计的演进

**传统响应式设计** (2010-2015):

- 主要依赖 CSS 媒体查询
- 所有设备加载相同的 HTML 和资源
- 性能优化有限

**现代响应式设计** (2016-2020):

- 引入移动优先设计理念
- 使用响应式图片和懒加载
- 开始关注性能指标

**Next.js 时代的响应式设计** (2021-现在):

- 服务端渲染和静态生成
- 智能资源优化
- 组件级别的响应式控制
- 性能和用户体验并重

🆕 **Next.js 16 时代** (2025):

- 服务端组件让响应式决策前移
- React Compiler 自动优化响应式组件
- Turbopack 加速开发体验
- 更精细的性能控制

## 2. 核心概念与原理 (Concepts & Principles)

### 2.1 响应式设计的三大支柱

#### 2.1.1 流式布局 (Fluid Layouts)

流式布局使用相对单位(如百分比、vw、vh)而非固定像素值,让内容能够自适应容器大小。

**核心原则**:

- 使用相对单位定义宽度
- 避免固定宽度的容器
- 利用 CSS Grid 和 Flexbox 的弹性特性

```css
/* ❌ 固定布局 */
.container {
  width: 1200px;
  margin: 0 auto;
}

/* ✅ 流式布局 */
.container {
  width: 90%;
  max-width: 1200px;
  margin: 0 auto;
}
```

#### 2.1.2 弹性媒体 (Flexible Media)

媒体资源(图片、视频)需要能够根据容器大小自动缩放。

**核心原则**:

- 图片使用 max-width: 100%
- 视频使用 aspect-ratio 保持比例
- 使用 srcset 提供多尺寸图片

```css
/* 基础响应式图片 */
img {
  max-width: 100%;
  height: auto;
}

/* 响应式视频 */
.video-container {
  position: relative;
  aspect-ratio: 16 / 9;
  width: 100%;
}

.video-container iframe {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
}
```

#### 2.1.3 媒体查询 (Media Queries)

媒体查询允许根据设备特性应用不同的样式规则。

**核心原则**:

- 移动优先设计
- 使用语义化的断点
- 避免过多的断点

```css
/* 移动优先 */
.grid {
  display: grid;
  grid-template-columns: 1fr;
  gap: 1rem;
}

/* 平板 */
@media (min-width: 768px) {
  .grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

/* 桌面 */
@media (min-width: 1024px) {
  .grid {
    grid-template-columns: repeat(3, 1fr);
  }
}
```

### 2.2 Next.js 16 中的响应式工作机制

#### 2.2.1 服务端组件的响应式决策

🆕 **Next.js 16 新增**: 服务端组件可以在渲染时根据 User-Agent 或其他请求头信息,决定渲染哪个版本的组件。

```typescript
// app/components/ResponsiveHero.tsx
import { headers } from "next/headers";

export default async function ResponsiveHero() {
  const headersList = await headers();
  const userAgent = headersList.get("user-agent") || "";

  // 服务端判断设备类型
  const isMobile = /mobile/i.test(userAgent);

  if (isMobile) {
    return (
      <section className="hero-mobile">
        <h1>移动端优化的标题</h1>
        <p>简洁的描述文字</p>
      </section>
    );
  }

  return (
    <section className="hero-desktop">
      <h1>桌面端完整标题</h1>
      <p>详细的描述文字,包含更多信息</p>
      <div className="hero-features">{/* 桌面端额外的功能展示 */}</div>
    </section>
  );
}
```

**优势**:

- 减少客户端 JavaScript 执行
- 避免布局闪烁
- 更好的首屏性能

**注意事项**:

- User-Agent 检测不是 100%准确
- 需要考虑缓存策略
- 可能影响静态生成

#### 2.2.2 客户端组件的响应式状态

对于需要动态响应窗口大小变化的场景,使用客户端组件配合 React hooks。

```typescript
// app/components/ResponsiveLayout.tsx
"use client";

import { useState, useEffect } from "react";

function useMediaQuery(query: string): boolean {
  const [matches, setMatches] = useState(false);

  useEffect(() => {
    const media = window.matchMedia(query);

    // 初始化状态
    setMatches(media.matches);

    // 监听变化
    const listener = (e: MediaQueryListEvent) => {
      setMatches(e.matches);
    };

    media.addEventListener("change", listener);

    return () => {
      media.removeEventListener("change", listener);
    };
  }, [query]);

  return matches;
}

export default function ResponsiveLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  const isMobile = useMediaQuery("(max-width: 768px)");
  const isTablet = useMediaQuery("(min-width: 769px) and (max-width: 1024px)");
  const isDesktop = useMediaQuery("(min-width: 1025px)");

  return (
    <div
      className={`
      layout
      ${isMobile ? "layout-mobile" : ""}
      ${isTablet ? "layout-tablet" : ""}
      ${isDesktop ? "layout-desktop" : ""}
    `}
    >
      {children}
    </div>
  );
}
```

⚡ **Next.js 16 增强**: React Compiler 会自动优化这类响应式 hooks,减少不必要的重渲染。

#### 2.2.3 混合响应式策略

最佳实践是结合服务端和客户端的优势:

```typescript
// app/components/ProductGrid.tsx
import { headers } from "next/headers";
import ClientProductGrid from "./ClientProductGrid";

export default async function ProductGrid() {
  const headersList = await headers();
  const userAgent = headersList.get("user-agent") || "";
  const isMobile = /mobile/i.test(userAgent);

  // 服务端决定初始列数
  const initialColumns = isMobile ? 1 : 3;

  return (
    <ClientProductGrid
      initialColumns={initialColumns}
      // 传递其他必要的props
    />
  );
}
```

```typescript
// app/components/ClientProductGrid.tsx
"use client";

import { useState, useEffect } from "react";

interface Props {
  initialColumns: number;
}

export default function ClientProductGrid({ initialColumns }: Props) {
  const [columns, setColumns] = useState(initialColumns);

  useEffect(() => {
    const handleResize = () => {
      const width = window.innerWidth;
      if (width < 768) {
        setColumns(1);
      } else if (width < 1024) {
        setColumns(2);
      } else {
        setColumns(3);
      }
    };

    window.addEventListener("resize", handleResize);
    return () => window.removeEventListener("resize", handleResize);
  }, []);

  return (
    <div
      className="product-grid"
      style={{
        gridTemplateColumns: `repeat(${columns}, 1fr)`,
      }}
    >
      {/* 产品列表 */}
    </div>
  );
}
```

### 2.3 响应式设计的性能考量

#### 2.3.1 关键渲染路径优化

响应式设计需要特别注意关键渲染路径,避免因为响应式逻辑导致性能下降。

**优化策略**:

1. **CSS 优先**: 尽可能使用 CSS 媒体查询而非 JavaScript
2. **避免布局抖动**: 使用 CSS 而非 JS 控制布局变化
3. **懒加载非关键内容**: 移动端可以延迟加载某些桌面端特有的功能

```typescript
// app/components/FeatureSection.tsx
import dynamic from "next/dynamic";

// 桌面端特有的复杂组件,移动端不加载
const DesktopFeatures = dynamic(() => import("./DesktopFeatures"), {
  ssr: false,
  loading: () => <div>加载中...</div>,
});

export default function FeatureSection() {
  return (
    <section>
      {/* 所有设备都显示的基础内容 */}
      <div className="basic-features">
        <h2>核心功能</h2>
      </div>

      {/* 仅桌面端显示的高级功能 */}
      <div className="hidden md:block">
        <DesktopFeatures />
      </div>
    </section>
  );
}
```

#### 2.3.2 图片响应式优化

## 3. 适用场景 (Applicable Scenarios)

### 3.1 典型应用场景

#### 3.1.1 电商网站

**场景特点**:

- 产品列表需要在不同设备上展示不同的列数
- 移动端需要简化的购物流程
- 图片需要高质量且加载快速

**响应式策略**:

```typescript
// app/products/page.tsx
import ProductGrid from "@/components/ProductGrid";
import { headers } from "next/headers";

export default async function ProductsPage() {
  const headersList = await headers();
  const userAgent = headersList.get("user-agent") || "";
  const isMobile = /mobile/i.test(userAgent);

  return (
    <main>
      <ProductGrid
        columns={{
          mobile: 1,
          tablet: 2,
          desktop: 4,
        }}
        showFilters={!isMobile} // 移动端隐藏侧边栏筛选
      />
    </main>
  );
}
```

#### 3.1.2 内容网站/博客

**场景特点**:

- 大量文本内容需要良好的阅读体验
- 不同设备上的排版需求差异大
- 需要优化字体大小和行高

**响应式策略**:

```css
/* styles/article.css */
.article {
  /* 移动端 */
  font-size: 16px;
  line-height: 1.6;
  padding: 1rem;
  max-width: 100%;
}

@media (min-width: 768px) {
  .article {
    font-size: 18px;
    line-height: 1.8;
    padding: 2rem;
    max-width: 65ch; /* 最佳阅读宽度 */
    margin: 0 auto;
  }
}

@media (min-width: 1024px) {
  .article {
    font-size: 20px;
    line-height: 2;
  }
}
```

#### 3.1.3 SaaS 应用

**场景特点**:

- 复杂的数据表格和图表
- 移动端可能需要完全不同的交互方式
- 需要考虑触摸和鼠标两种交互模式

**响应式策略**:

```typescript
// app/dashboard/page.tsx
"use client";

import { useState, useEffect } from "react";
import DesktopDashboard from "@/components/DesktopDashboard";
import MobileDashboard from "@/components/MobileDashboard";

export default function DashboardPage() {
  const [isMobile, setIsMobile] = useState(false);

  useEffect(() => {
    const checkMobile = () => {
      setIsMobile(window.innerWidth < 768);
    };

    checkMobile();
    window.addEventListener("resize", checkMobile);

    return () => window.removeEventListener("resize", checkMobile);
  }, []);

  // 移动端和桌面端使用完全不同的组件
  return isMobile ? <MobileDashboard /> : <DesktopDashboard />;
}
```

### 3.2 不推荐的场景

#### 3.2.1 需要精确像素控制的设计工具

如果你的应用是设计工具、CAD 软件等需要精确像素控制的场景,响应式设计可能不是最佳选择。

**原因**:

- 设计工具需要固定的画布尺寸
- 响应式布局会导致元素位置不确定
- 用户期望的是固定的工作区域

**替代方案**:

- 提供固定尺寸的工作区
- 使用缩放而非响应式布局
- 为移动端提供简化版或只读版本

#### 3.2.2 游戏类应用

**原因**:

- 游戏通常需要固定的宽高比
- 响应式布局可能破坏游戏平衡
- 性能要求高,响应式逻辑可能影响帧率

**替代方案**:

- 使用固定宽高比的容器
- 提供不同设备的专用版本
- 使用 Canvas 或 WebGL 的缩放功能

## 4. 响应式断点系统 (Breakpoint System)

### 4.1 标准断点定义

在 Next.js 16 项目中,建议使用统一的断点系统,确保整个应用的响应式行为一致。

#### 4.1.1 Tailwind CSS 断点(推荐)

如果使用 Tailwind CSS,它提供了一套经过验证的断点系统:

| 断点前缀 | 最小宽度 | CSS 媒体查询                 | 目标设备   |
| :------- | :------- | :--------------------------- | :--------- |
| `sm`     | 640px    | `@media (min-width: 640px)`  | 大屏手机   |
| `md`     | 768px    | `@media (min-width: 768px)`  | 平板       |
| `lg`     | 1024px   | `@media (min-width: 1024px)` | 笔记本     |
| `xl`     | 1280px   | `@media (min-width: 1280px)` | 桌面显示器 |
| `2xl`    | 1536px   | `@media (min-width: 1536px)` | 大屏显示器 |

**使用示例**:

```tsx
// app/components/ResponsiveCard.tsx
export default function ResponsiveCard() {
  return (
    <div
      className="
      w-full          /* 移动端全宽 */
      sm:w-1/2        /* 小屏幕50%宽度 */
      md:w-1/3        /* 平板33%宽度 */
      lg:w-1/4        /* 桌面25%宽度 */
      p-4             /* 所有设备padding */
      sm:p-6          /* 小屏幕增加padding */
    "
    >
      <h3 className="text-lg md:text-xl lg:text-2xl">响应式标题</h3>
      <p className="text-sm md:text-base">响应式内容</p>
    </div>
  );
}
```

#### 4.1.2 自定义断点系统

如果不使用 Tailwind,可以定义自己的断点系统:

```typescript
// lib/breakpoints.ts
export const breakpoints = {
  mobile: 0,
  tablet: 768,
  desktop: 1024,
  wide: 1440,
} as const;

export type Breakpoint = keyof typeof breakpoints;

// 生成媒体查询字符串
export function mediaQuery(breakpoint: Breakpoint): string {
  return `(min-width: ${breakpoints[breakpoint]}px)`;
}

// 用于CSS-in-JS
export const media = {
  mobile: `@media ${mediaQuery("mobile")}`,
  tablet: `@media ${mediaQuery("tablet")}`,
  desktop: `@media ${mediaQuery("desktop")}`,
  wide: `@media ${mediaQuery("wide")}`,
};
```

**在 CSS Modules 中使用**:

```css
/* styles/component.module.css */
.container {
  padding: 1rem;
}

@media (min-width: 768px) {
  .container {
    padding: 2rem;
  }
}

@media (min-width: 1024px) {
  .container {
    padding: 3rem;
  }
}
```

**在 CSS-in-JS 中使用**:

```typescript
// app/components/StyledComponent.tsx
"use client";

import styled from "styled-components";
import { media } from "@/lib/breakpoints";

const Container = styled.div`
  padding: 1rem;

  ${media.tablet} {
    padding: 2rem;
  }

  ${media.desktop} {
    padding: 3rem;
  }
`;

export default function StyledComponent() {
  return <Container>响应式容器</Container>;
}
```

### 4.2 移动优先 vs 桌面优先

#### 4.2.1 移动优先策略(推荐)

移动优先意味着先为移动设备编写样式,然后使用`min-width`媒体查询为更大的屏幕添加样式。

**优势**:

- 符合渐进增强的理念
- 移动端性能更好(不需要覆盖桌面端样式)
- 强制开发者优先考虑移动体验
- 更容易维护

```css
/* ✅ 移动优先 */
.button {
  /* 移动端样式(基础样式) */
  padding: 0.5rem 1rem;
  font-size: 14px;
}

@media (min-width: 768px) {
  /* 平板样式(增强) */
  .button {
    padding: 0.75rem 1.5rem;
    font-size: 16px;
  }
}

@media (min-width: 1024px) {
  /* 桌面样式(进一步增强) */
  .button {
    padding: 1rem 2rem;
    font-size: 18px;
  }
}
```

#### 4.2.2 桌面优先策略

桌面优先使用`max-width`媒体查询,从大屏幕向小屏幕适配。

**劣势**:

- 移动端需要覆盖更多样式
- 性能略差
- 不符合现代 Web 开发趋势

```css
/* ❌ 桌面优先(不推荐) */
.button {
  /* 桌面端样式(基础样式) */
  padding: 1rem 2rem;
  font-size: 18px;
}

@media (max-width: 1023px) {
  /* 平板样式(覆盖) */
  .button {
    padding: 0.75rem 1.5rem;
    font-size: 16px;
  }
}

@media (max-width: 767px) {
  /* 移动端样式(再次覆盖) */
  .button {
    padding: 0.5rem 1rem;
    font-size: 14px;
  }
}
```

### 4.3 容器查询(Container Queries)

🆕 **Next.js 16 新增**: 现代浏览器已广泛支持容器查询,这是响应式设计的重大进步。

容器查询允许组件根据其父容器的大小而非视口大小来调整样式,这使得组件更加独立和可复用。

#### 4.3.1 基础容器查询

```css
/* styles/card.module.css */
.container {
  container-type: inline-size;
  container-name: card;
}

.card {
  padding: 1rem;
}

/* 当容器宽度大于400px时 */
@container card (min-width: 400px) {
  .card {
    padding: 2rem;
    display: grid;
    grid-template-columns: 1fr 2fr;
  }
}

/* 当容器宽度大于600px时 */
@container card (min-width: 600px) {
  .card {
    padding: 3rem;
  }
}
```

```tsx
// app/components/ResponsiveCard.tsx
import styles from "./card.module.css";

export default function ResponsiveCard() {
  return (
    <div className={styles.container}>
      <div className={styles.card}>
        <div className="image">图片</div>
        <div className="content">内容</div>
      </div>
    </div>
  );
}
```

#### 4.3.2 容器查询的优势

**场景对比**:

假设你有一个卡片组件,它可能出现在:

- 主内容区(宽度较大)
- 侧边栏(宽度较小)
- 网格布局中(宽度不确定)

使用传统媒体查询:

```css
/* ❌ 传统方式:依赖视口宽度,不够灵活 */
.card {
  display: block;
}

@media (min-width: 768px) {
  .card {
    display: grid;
    grid-template-columns: 1fr 2fr;
  }
}

/* 问题:在平板上,侧边栏的卡片也会变成网格布局,
   但侧边栏宽度可能不够,导致布局拥挤 */
```

使用容器查询:

```css
/* ✅ 容器查询:根据容器宽度,更加灵活 */
.cardContainer {
  container-type: inline-size;
}

.card {
  display: block;
}

@container (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: 1fr 2fr;
  }
}

/* 优势:无论卡片在哪里,只要容器宽度够,
   就会变成网格布局,更加智能 */
```

#### 4.3.3 容器查询单位

容器查询还提供了新的 CSS 单位:

| 单位    | 说明                  | 示例                |
| :------ | :-------------------- | :------------------ |
| `cqw`   | 容器宽度的 1%         | `width: 50cqw`      |
| `cqh`   | 容器高度的 1%         | `height: 50cqh`     |
| `cqi`   | 容器内联尺寸的 1%     | `padding: 2cqi`     |
| `cqb`   | 容器块尺寸的 1%       | `margin: 2cqb`      |
| `cqmin` | cqi 和 cqb 中较小的值 | `font-size: 3cqmin` |
| `cqmax` | cqi 和 cqb 中较大的值 | `gap: 2cqmax`       |

```css

  /* 字体大小随容器宽度变化 */

.content {
  /* padding随容器宽度变化 */
  padding: 2cqi;
```

## 5. 响应式布局技术 (Responsive Layout Techniques)

### 5.1 CSS Grid 响应式布局

CSS Grid 是实现复杂响应式布局的最佳选择。

#### 5.1.1 自动响应式网格

使用`auto-fit`和`minmax()`创建自动响应的网格:

```css
/* styles/grid.module.css */
.grid {
  display: grid;
  /* 自动创建列,每列最小250px,最大1fr */
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 1rem;
}

/* 不需要媒体查询,网格会自动响应! */
```

```tsx
// app/components/ProductGrid.tsx
import styles from "./grid.module.css";

export default function ProductGrid({ products }: { products: Product[] }) {
  return (
    <div className={styles.grid}>
      {products.map((product) => (
        <ProductCard key={product.id} product={product} />
      ))}
    </div>
  );
}
```

**工作原理**:

- `auto-fit`: 自动调整列数以填充可用空间
- `minmax(250px, 1fr)`: 每列最小 250px,如果空间足够则平均分配
- 当容器宽度小于 250px 时,自动变为单列
- 当容器宽度为 500px 时,自动变为两列
- 当容器宽度为 750px 时,自动变为三列

#### 5.1.2 命名网格区域

使用命名网格区域创建复杂的响应式布局:

## 6. 注意事项 (Precautions)

### 6.1 性能陷阱

#### 6.1.1 避免过度使用 JavaScript 检测

❌ **错误做法**:

```typescript
"use client";

import { useState, useEffect } from "react";

export default function BadExample() {
  const [windowWidth, setWindowWidth] = useState(0);

  useEffect(() => {
    const handleResize = () => {
      setWindowWidth(window.innerWidth);
    };

    // 每次resize都触发重渲染
    window.addEventListener("resize", handleResize);
    handleResize();

    return () => window.removeEventListener("resize", handleResize);
  }, []);

  return (
    <div>{windowWidth < 768 ? <MobileComponent /> : <DesktopComponent />}</div>
  );
}
```

✅ **正确做法**:

```css
/* 使用CSS隐藏/显示 */
.mobile-component {
  display: block;
}

.desktop-component {
  display: none;
}

@media (min-width: 768px) {
  .mobile-component {
    display: none;
  }

  .desktop-component {
    display: block;
  }
}
```

```tsx
export default function GoodExample() {
  return (
    <div>
      <div className="mobile-component">
        <MobileComponent />
      </div>
      <div className="desktop-component">
        <DesktopComponent />
      </div>
    </div>
  );
}
```

#### 6.1.2 防抖 resize 事件

如果必须使用 JavaScript 监听 resize,一定要防抖:

```typescript
"use client";

import { useState, useEffect, useCallback } from "react";

function debounce<T extends (...args: any[]) => any>(
  func: T,
  wait: number
): (...args: Parameters<T>) => void {
  let timeout: NodeJS.Timeout;

  return (...args: Parameters<T>) => {
    clearTimeout(timeout);
    timeout = setTimeout(() => func(...args), wait);
  };
}

export default function ResponsiveComponent() {
  const [width, setWidth] = useState(0);

  useEffect(() => {
    const handleResize = debounce(() => {
      setWidth(window.innerWidth);
    }, 150); // 150ms防抖

    handleResize();
    window.addEventListener("resize", handleResize);

    return () => window.removeEventListener("resize", handleResize);
  }, []);

  return <div>窗口宽度: {width}px</div>;
}
```

### 6.2 布局闪烁问题

#### 6.2.1 避免客户端检测导致的闪烁

❌ **问题代码**:

```typescript
"use client";

import { useState, useEffect } from "react";

export default function FlickerExample() {
  const [isMobile, setIsMobile] = useState(false);

  useEffect(() => {
    setIsMobile(window.innerWidth < 768);
  }, []);

  // 首次渲染显示桌面版,然后闪烁到移动版
  return isMobile ? <MobileView /> : <DesktopView />;
}
```

✅ **解决方案 1: 服务端检测**:

```typescript
// app/components/NoFlickerExample.tsx
import { headers } from "next/headers";

export default async function NoFlickerExample() {
  const headersList = await headers();
  const userAgent = headersList.get("user-agent") || "";
  const isMobile = /mobile/i.test(userAgent);

  // 服务端就决定渲染哪个版本,无闪烁
  return isMobile ? <MobileView /> : <DesktopView />;
}
```

✅ **解决方案 2: CSS 隐藏**:

```tsx
export default function NoFlickerExample() {
  return (
    <>
      <div className="block md:hidden">
        <MobileView />
      </div>
      <div className="hidden md:block">
        <DesktopView />
      </div>
    </>
  );
}
```

### 6.3 图片性能问题

#### 6.3.1 正确使用 sizes 属性

❌ **错误做法**:

```tsx
<Image

  alt="Hero"
```

## 7. 常见问题 (FAQ)

### 7.1 如何选择响应式方案?

**问题**: 应该使用 CSS 媒体查询还是 JavaScript 检测?

**回答**: 优先使用 CSS 媒体查询,只在必要时使用 JavaScript。

**详细说明**:

CSS 媒体查询的优势:

- 性能更好,不需要 JavaScript 执行
- 不会导致布局闪烁
- 更容易维护
- 支持服务端渲染

JavaScript 检测的适用场景:

- 需要根据窗口大小执行复杂逻辑
- 需要动态加载不同的组件
- 需要在 resize 时更新状态

**最佳实践**:

```tsx
// ✅ 推荐:CSS处理样式,JavaScript处理逻辑
export default function BestPractice() {
  return (
    <div
      className="
      grid
      grid-cols-1
      md:grid-cols-2
      lg:grid-cols-3
    "
    >
      {/* CSS处理布局 */}
    </div>
  );
}
```
