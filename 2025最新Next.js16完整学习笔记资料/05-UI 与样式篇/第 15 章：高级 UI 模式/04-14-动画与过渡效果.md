# 动画与过渡效果

## 1. 概述 (Overview)

动画和过渡效果是现代 Web 应用中不可或缺的一部分,它们不仅能提升用户体验,还能引导用户注意力、提供视觉反馈、增强品牌形象。在 Next.js 16 中,我们有多种方式实现动画效果,从简单的 CSS 过渡到复杂的 JavaScript 动画库。

### 1.1 为什么需要动画

**提升用户体验**: 平滑的过渡让界面感觉更流畅、更专业。

**视觉反馈**: 告诉用户操作已被接收和处理。

**引导注意力**: 将用户的注意力引导到重要元素上。

**品牌个性**: 独特的动画可以强化品牌形象。

**减少感知等待时间**: 加载动画可以让等待感觉更短。

### 1.2 动画的类型

**过渡(Transitions)**: 元素从一个状态平滑变化到另一个状态。

**关键帧动画(Keyframe Animations)**: 定义动画的多个关键点。

**物理动画(Physics-based)**: 基于物理规律的自然动画。

**手势动画(Gesture-based)**: 响应用户手势的动画。

**页面过渡(Page Transitions)**: 页面切换时的动画效果。

### 1.3 Next.js 16 中的动画方案

🆕 **Next.js 16 新增**: 改进的 View Transitions API 支持,实现原生页面过渡。

⚡ **Next.js 16 增强**: 更好的服务端组件与客户端动画的协作。

**主要方案**:

- CSS Transitions 和 Animations
- Tailwind CSS 动画工具类
- Framer Motion(最流行的 React 动画库)
- React Spring(基于物理的动画)
- GSAP(专业级动画库)
- View Transitions API(原生浏览器 API)

## 2. CSS 过渡与动画 (CSS Transitions & Animations)

### 2.1 CSS Transitions

CSS 过渡是最简单的动画方式,适合简单的状态变化。

#### 2.1.1 基础过渡

```tsx
// app/components/BasicTransition.tsx
"use client";

import { useState } from "react";

export default function BasicTransition() {
  const [isHovered, setIsHovered] = useState(false);

  return (
    <button
      onMouseEnter={() => setIsHovered(true)}
      onMouseLeave={() => setIsHovered(false)}
      className="px-6 py-3 bg-blue-500 text-white rounded transition-all duration-300 hover:bg-blue-600 hover:scale-105 hover:shadow-lg"
    >
      悬停我
    </button>
  );
}
```

#### 2.1.2 多属性过渡

```tsx
// app/components/MultiPropertyTransition.tsx
"use client";

import { useState } from "react";

export default function MultiPropertyTransition() {
  const [isExpanded, setIsExpanded] = useState(false);

  return (
    <div className="p-4">
      <button
        onClick={() => setIsExpanded(!isExpanded)}
        className="px-4 py-2 bg-blue-500 text-white rounded"
      >
        {isExpanded ? "收起" : "展开"}
      </button>

      <div
        className="mt-4 overflow-hidden transition-all duration-500 ease-in-out"
        style={{
          maxHeight: isExpanded ? "500px" : "0",
          opacity: isExpanded ? 1 : 0,
        }}
      >
        <div className="p-4 bg-gray-100 rounded">
          <p>这是可展开的内容。</p>
          <p>支持多个属性同时过渡。</p>
        </div>
      </div>
    </div>
  );
}
```

### 2.2 CSS Keyframe Animations

关键帧动画可以定义更复杂的动画序列。

#### 2.2.1 定义关键帧

```css
/* app/globals.css */
@keyframes fadeIn {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

@keyframes pulse {
  0%,
  100% {
    opacity: 1;
  }
  50% {
    opacity: 0.5;
  }
}

@keyframes spin {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

@keyframes bounce {
  0%,
  100% {
    transform: translateY(0);
  }
  50% {
    transform: translateY(-20px);
  }
}
```

#### 2.2.2 使用关键帧动画

```tsx
// app/components/KeyframeAnimation.tsx
export default function KeyframeAnimation() {

    <div className="space-y-4">

      <div className="animate-pulse p-4 bg-green-100 rounded">脉冲效果</div>

      {/* 旋转动画 */}
      <div className="animate-spin h-8 w-8 border-4 border-blue-500 border-t-transparent rounded-full"></div>
```

## 3. Framer Motion (Framer Motion)

Framer Motion 是 React 生态中最流行的动画库,提供了声明式的 API 和强大的功能。

### 3.1 安装和基础使用

```bash
npm install framer-motion
```

#### 3.1.1 基础动画

```tsx
// app/components/FramerBasic.tsx
"use client";

import { motion } from "framer-motion";

export default function FramerBasic() {
  return (
    <motion.div
      initial={{ opacity: 0, y: 20 }}
      animate={{ opacity: 1, y: 0 }}
      transition={{ duration: 0.5 }}
      className="p-4 bg-blue-100 rounded"

      这个元素会淡入并向上移动

  );
```

```tsx
const items = ["项目1", "项目2", "项目3", "项目4"];

          variants={item}

        </motion.li>
      ))}
    </motion.ul>
  );
}
```

### 3.3 页面过渡动画

```tsx
// app/components/PageTransition.tsx

import { motion, AnimatePresence } from "framer-motion";

export default function PageTransition({
  children,
}: {
  children: React.ReactNode;

  );
}
```

```tsx
export default function ScrollAnimation() {
```

## 6. 性能优化 (Performance Optimization)

### 6.1 使用 will-change

```css
/* 告诉浏览器哪些属性会变化 */
.animated-element {
  will-change: transform, opacity;
}

/* 动画结束后移除 */
.animated-element.animation-done {
  will-change: auto;
}
```

### 6.2 使用 transform 和 opacity

```tsx
// 好的做法: 使用transform
<motion.div
  animate={{ x: 100, opacity: 0.5 }}
/>

// 不好的做法: 使用left/top
<motion.div
  animate={{ left: 100, opacity: 0.5 }}
/>
```

### 6.3 减少重绘和回流

```tsx
// app/components/OptimizedAnimation.tsx
"use client";

import { motion } from "framer-motion";

export default function OptimizedAnimation() {
  return (
    <motion.div
      // 使用transform而不是width/height
      initial={{ scale: 0 }}
      animate={{ scale: 1 }}
      // 使用GPU加速
      style={{ transform: "translateZ(0)" }}
      className="p-4 bg-blue-100 rounded"
    >
      优化的动画
    </motion.div>
  );
}
```

### 6.4 懒加载动画

```tsx
// app/components/LazyAnimation.tsx
"use client";

import { motion } from "framer-motion";
import { useInView } from "framer-motion";

    </motion.div>
  );
}
```

## 8. 注意事项 (Precautions)

### 8.1 性能考虑

**避免过度动画**: 太多动画会影响性能和用户体验。

**使用 GPU 加速**: 优先使用 transform 和 opacity。

**减少动画时长**: 大多数动画应在 200-500ms 之间。

### 8.2 可访问性

**尊重用户偏好**: 检查 prefers-reduced-motion。

```tsx
// app/components/AccessibleAnimation.tsx
"use client";

import { motion } from "framer-motion";
import { useReducedMotion } from "framer-motion";

export default function AccessibleAnimation() {
  const shouldReduceMotion = useReducedMotion();

  return (
    <motion.div
      initial={{ opacity: 0, y: shouldReduceMotion ? 0 : 20 }}
      animate={{ opacity: 1, y: 0 }}
      transition={{ duration: shouldReduceMotion ? 0 : 0.5 }}
    >
      内容
    </motion.div>
  );
}
```

**提供禁用选项**: 允许用户关闭动画。

**不要仅依赖动画**: 确保没有动画时功能仍然可用。

### 8.3 用户体验

**有目的的动画**: 每个动画都应该有明确的目的。

**保持一致性**: 相似的交互使用相似的动画。

**不要阻塞交互**: 动画不应该阻止用户操作。

## 9. 常见问题 (FAQ)

### 9.1 如何选择动画库?

**问题**: CSS、Framer Motion、React Spring 该选哪个?

**解决方案**:

| 场景         | 推荐方案        |
| ------------ | --------------- |
| 简单过渡     | CSS Transitions |
| 复杂动画序列 | Framer Motion   |
| 物理动画     | React Spring    |
| 高性能要求   | CSS Animations  |
| 手势交互     | Framer Motion   |

### 9.2 动画卡顿怎么办?

**问题**: 动画不流畅,有卡顿。

**解决方案**:

1. **使用 transform 和 opacity**:

```tsx
// 好
<motion.div animate={{ x: 100, opacity: 0.5 }} />

// 不好
<motion.div animate={{ left: 100 }} />
```

2. **添加 will-change**:

```css
.animated {
  will-change: transform;
}
```

3. **减少动画元素数量**:

```tsx
// 不要同时动画太多元素
<motion.div
  variants={container}
  initial="hidden"
  animate="show"
>
  {items.slice(0, 20).map(...)} {/* 限制数量 */}
</motion.div>
```

### 9.5 如何实现滚动视差效果?

**问题**: 如何创建滚动时的视差动画?

**解决方案**:

```tsx
// app/components/Parallax.tsx
"use client";

import { motion, useScroll, useTransform } from "framer-motion";
import { useRef } from "react";

export default function Parallax() {
  const ref = useRef(null);
  const { scrollYProgress } = useScroll({
    target: ref,
    offset: ["start start", "end start"],
  });

  const y = useTransform(scrollYProgress, [0, 1], ["0%", "50%"]);
  const opacity = useTransform(scrollYProgress, [0, 1], [1, 0]);

  return (
    <div ref={ref} className="h-screen relative overflow-hidden">
      <motion.div
        style={{ y, opacity }}
        className="absolute inset-0 bg-gradient-to-b from-blue-500 to-purple-500"
      >
        <div className="flex items-center justify-center h-full">
          <h1 className="text-6xl font-bold text-white">视差效果</h1>
        </div>
      </motion.div>
    </div>
  );
}
```

## 10. 总结 (Summary)

### 10.1 核心要点

动画是提升用户体验的重要手段,但要适度使用。

**选择合适的工具**: 根据需求选择 CSS、Framer Motion 或其他库。

**性能优先**: 使用 transform 和 opacity,避免触发重排。

**可访问性**: 尊重用户的动画偏好设置。

**有目的性**: 每个动画都应该有明确的目的。

### 10.2 最佳实践

| 方面     | 推荐做法                    | 避免做法                      |
| -------- | --------------------------- | ----------------------------- |
| 性能     | 使用 transform 和 opacity   | 动画 width、height、left、top |
| 时长     | 200-500ms                   | 超过 1 秒的动画               |
| 缓动     | 使用自然的缓动函数          | 线性动画(除非特殊需求)        |
| 数量     | 适度使用                    | 过度动画                      |
| 可访问性 | 检查 prefers-reduced-motion | 忽略用户偏好                  |
| 目的     | 有明确目的                  | 为了动画而动画                |

### 10.3 动画类型选择

**CSS Transitions**: 简单的状态变化,如悬停效果。

**CSS Animations**: 循环动画,如加载指示器。

**Framer Motion**: 复杂的交互动画,页面过渡。

**React Spring**: 需要物理效果的动画。

**GSAP**: 复杂的时间线动画,专业级项目。

### 10.4 性能检查清单

- [ ] 使用 transform 和 opacity 而不是其他属性
- [ ] 添加 will-change 提示浏览器
- [ ] 限制同时动画的元素数量
- [ ] 使用 GPU 加速(translateZ(0))
- [ ] 在动画结束后清理 will-change
- [ ] 使用 requestAnimationFrame
- [ ] 检查移动设备性能

动画是一门艺术,需要在美观、性能和可访问性之间找到平衡。从简单开始,逐步添加复杂效果,始终以用户体验为中心。

```tsx
// app/components/AnimationPerformance.tsx
"use client";

import { useEffect, useRef } from "react";

export default function AnimationPerformance() {
  const fpsRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    let lastTime = performance.now();
    let frames = 0;

    const measureFPS = () => {
      frames++;
      const currentTime = performance.now();

      if (currentTime >= lastTime + 1000) {
        const fps = Math.round((frames * 1000) / (currentTime - lastTime));
        if (fpsRef.current) {
          fpsRef.current.textContent = `FPS: ${fps}`;
        }
        frames = 0;
        lastTime = currentTime;
      }

      requestAnimationFrame(measureFPS);
    };

    measureFPS();
  }, []);

  return (
    <div
      ref={fpsRef}
      className="fixed top-4 right-4 px-4 py-2 bg-black text-white rounded font-mono text-sm"
    >
      FPS: --
    </div>
  );
}
```

### 10.10 实战案例

#### 10.10.1 购物车动画

```tsx
// app/components/CartAnimation.tsx
"use client";

import { motion, AnimatePresence } from "framer-motion";
import { useState } from "react";

interface CartItem {
  id: number;
  name: string;
  price: number;
}

export default function CartAnimation() {
  const [items, setItems] = useState<CartItem[]>([]);
  const [showAdded, setShowAdded] = useState(false);

  const addItem = () => {
    const newItem = {
      id: Date.now(),
      name: `商品 ${items.length + 1}`,
      price: Math.floor(Math.random() * 100) + 10,
    };
    setItems([...items, newItem]);
    setShowAdded(true);
    setTimeout(() => setShowAdded(false), 2000);
  };

  const removeItem = (id: number) => {
    setItems(items.filter((item) => item.id !== id));
  };

  const total = items.reduce((sum, item) => sum + item.price, 0);

  return (
    <div className="max-w-md mx-auto p-4">
      <div className="flex justify-between items-center mb-4">
        <h2 className="text-2xl font-bold">购物车</h2>
        <motion.button
          onClick={addItem}
          whileTap={{ scale: 0.95 }}
          className="px-4 py-2 bg-blue-500 text-white rounded"
        >
          添加商品
        </motion.button>
      </div>

      <AnimatePresence>
        {showAdded && (
          <motion.div
            initial={{ opacity: 0, y: -20 }}
            animate={{ opacity: 1, y: 0 }}
            exit={{ opacity: 0 }}
            className="mb-4 p-3 bg-green-100 text-green-800 rounded"
          >
            商品已添加到购物车!
          </motion.div>
        )}
      </AnimatePresence>

      <div className="space-y-2 mb-4">
        <AnimatePresence>
          {items.map((item) => (
            <motion.div
              key={item.id}
              initial={{ opacity: 0, x: -20 }}
              animate={{ opacity: 1, x: 0 }}
              exit={{ opacity: 0, x: 20 }}
              className="flex justify-between items-center p-3 bg-white border rounded"
            >
              <div>
                <p className="font-medium">{item.name}</p>
                <p className="text-sm text-gray-600">¥{item.price}</p>
              </div>
              <button
                onClick={() => removeItem(item.id)}
                className="text-red-500 hover:text-red-700"
              >
                删除
              </button>
            </motion.div>
          ))}
        </AnimatePresence>
      </div>

      {items.length > 0 && (
        <motion.div
          initial={{ opacity: 0 }}
          animate={{ opacity: 1 }}
          className="p-4 bg-gray-100 rounded"
        >
          <div className="flex justify-between items-center">
            <span className="font-bold">总计:</span>
            <motion.span
              key={total}
              initial={{ scale: 1.2, color: "#3b82f6" }}
              animate={{ scale: 1, color: "#000" }}
              className="text-xl font-bold"
            >
              ¥{total}
            </motion.span>
          </div>
        </motion.div>
      )}
    </div>
  );
}
```
