# 无障碍(a11y)UI 设计

## 1. 概述 (Overview)

无障碍(Accessibility,简称 a11y)是指让所有人,包括残障人士,都能平等地访问和使用 Web 应用。这不仅是道德责任,在很多国家也是法律要求。在 Next.js 16 中实现无障碍 UI 设计,可以让你的应用服务更广泛的用户群体。

### 1.1 什么是 Web 无障碍

Web 无障碍是指让残障人士能够感知、理解、导航和与 Web 交互的实践。这包括:

**视觉障碍**: 盲人、低视力、色盲等用户。

**听觉障碍**: 聋人或听力受损的用户。

**运动障碍**: 无法使用鼠标或键盘的用户。

**认知障碍**: 学习障碍、记忆障碍等用户。

**临时性障碍**: 手臂骨折、在强光下看不清屏幕等临时情况。

### 1.2 为什么需要无障碍设计

**法律要求**: 许多国家和地区有无障碍法律,如美国的 ADA、欧盟的 EAA 等。

**扩大用户群**: 全球约 15%的人口有某种形式的残障,这是一个巨大的市场。

**提升用户体验**: 无障碍设计通常也能改善所有用户的体验。

**SEO 优势**: 搜索引擎爬虫和屏幕阅读器有相似的需求,无障碍设计有助于 SEO。

**品牌形象**: 展示企业的社会责任感。

### 1.3 WCAG 标准

WCAG(Web Content Accessibility Guidelines)是 Web 无障碍的国际标准,目前最新版本是 WCAG 2.2。

**等级**:

- A 级: 最基本的无障碍要求
- AA 级: 中等无障碍要求(大多数法律要求的级别)
- AAA 级: 最高无障碍要求

**四大原则**(POUR):

- **可感知(Perceivable)**: 信息和 UI 组件必须以用户能感知的方式呈现
- **可操作(Operable)**: UI 组件和导航必须是可操作的
- **可理解(Understandable)**: 信息和 UI 操作必须是可理解的
- **健壮(Robust)**: 内容必须足够健壮,能被各种用户代理(包括辅助技术)可靠地解释

### 1.4 Next.js 16 中的无障碍特性

🆕 **Next.js 16 新增**: 改进的服务端渲染支持,确保初始 HTML 就包含完整的语义结构。

⚡ **Next.js 16 增强**: 自动优化焦点管理,特别是在路由切换时。

**核心优势**:

- 服务端渲染确保内容对屏幕阅读器可用
- 自动处理路由切换时的焦点管理
- 支持语义化 HTML
- 与无障碍测试工具良好集成

## 2. 核心概念与原则 (Concepts & Principles)

### 2.1 语义化 HTML

使用正确的 HTML 元素是无障碍的基础。

#### 2.1.1 正确使用标题

```tsx
// app/page.tsx
export default function Page() {
  return (
    <main>
      <h1>页面主标题</h1>
      <section>
        <h2>第一部分</h2>
        <p>内容...</p>
        <h3>子标题</h3>
        <p>更多内容...</p>
      </section>
      <section>
        <h2>第二部分</h2>
        <p>内容...</p>
      </section>
    </main>
  );
}
```

**注意事项**:

- 每个页面只有一个 h1
- 标题层级不要跳跃(不要从 h1 直接跳到 h3)
- 不要仅为了样式而使用标题

常用 ARIA 属性:

| 属性             | 用途                 | 示例                                    |
| ---------------- | -------------------- | --------------------------------------- |
| aria-label       | 为元素提供标签       | `<button aria-label="关闭">×</button>`  |
| aria-labelledby  | 引用其他元素作为标签 | `<div aria-labelledby="title">`         |
| aria-describedby | 引用描述性文本       | `<input aria-describedby="help-text">`  |
| aria-hidden      | 对屏幕阅读器隐藏元素 | `<div aria-hidden="true">`              |
| aria-live        | 声明动态内容区域     | `<div aria-live="polite">`              |
| aria-expanded    | 元素是否展开         | `<button aria-expanded="false">`        |
| aria-selected    | 元素是否被选中       | `<div role="tab" aria-selected="true">` |
| aria-disabled    | 元素是否禁用         | `<button aria-disabled="true">`         |

### 2.3 键盘导航

#### 2.3.1 焦点管理

```tsx
// app/components/Modal.tsx
"use client";

import { useEffect, useRef } from "react";

interface ModalProps {
  isOpen: boolean;
  onClose: () => void;
  children: React.ReactNode;
}

export default function Modal({ isOpen, onClose, children }: ModalProps) {
  const modalRef = useRef<HTMLDivElement>(null);
  const previousFocusRef = useRef<HTMLElement | null>(null);

  useEffect(() => {
    if (isOpen) {
      // 保存当前焦点
      previousFocusRef.current = document.activeElement as HTMLElement;

      // 将焦点移到模态框
      modalRef.current?.focus();

      // 捕获焦点在模态框内
      const handleTab = (e: KeyboardEvent) => {
        if (e.key === "Tab") {
          const focusableElements = modalRef.current?.querySelectorAll(
            'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
          );

          if (!focusableElements || focusableElements.length === 0) return;

          const firstElement = focusableElements[0] as HTMLElement;
          const lastElement = focusableElements[
            focusableElements.length - 1
          ] as HTMLElement;

          if (e.shiftKey) {
            if (document.activeElement === firstElement) {
              lastElement.focus();
              e.preventDefault();
            }
          } else {
            if (document.activeElement === lastElement) {
              firstElement.focus();
              e.preventDefault();
            }
          }
        }

        if (e.key === "Escape") {
          onClose();
        }
      };

      document.addEventListener("keydown", handleTab);

      return () => {
        document.removeEventListener("keydown", handleTab);
        // 恢复之前的焦点
        previousFocusRef.current?.focus();
      };
    }
  }, [isOpen, onClose]);

  if (!isOpen) return null;

  return (
    <div
      className="fixed inset-0 bg-black/50 flex items-center justify-center"
      onClick={onClose}
    >
      <div
        ref={modalRef}
        role="dialog"
        aria-modal="true"
        aria-labelledby="modal-title"
        className="bg-white p-6 rounded-lg max-w-md"
        onClick={(e) => e.stopPropagation()}
        tabIndex={-1}
      >
        {children}
      </div>
    </div>
  );
}
```

### 2.4 颜色对比度

确保文本和背景有足够的对比度。

#### 2.4.1 对比度要求

| 级别 | 普通文本 | 大文本 |
| ---- | -------- | ------ |
| AA   | 4.5:1    | 3:1    |
| AAA  | 7:1      | 4.5:1  |

大文本定义: 18pt(24px)以上,或 14pt(18.66px)粗体以上。

#### 2.4.2 检查对比度

```typescript
// lib/utils/contrast.ts

/**
 * 计算相对亮度
 */
function getLuminance(hex: string): number {
  const rgb = parseInt(hex.slice(1), 16);
  const r = (rgb >> 16) & 0xff;
  const g = (rgb >> 8) & 0xff;
  const b = (rgb >> 0) & 0xff;

  const [rs, gs, bs] = [r, g, b].map((val) => {
    val = val / 255;
    return val <= 0.03928 ? val / 12.92 : Math.pow((val + 0.055) / 1.055, 2.4);
  });

  return 0.2126 * rs + 0.7152 * gs + 0.0722 * bs;
}

/**
 * 计算对比度
 */
export function getContrastRatio(
  foreground: string,
  background: string
): number {
  const l1 = getLuminance(foreground);
  const l2 = getLuminance(background);
  const lighter = Math.max(l1, l2);
  const darker = Math.min(l1, l2);

  return (lighter + 0.05) / (darker + 0.05);
}

/**
 * 检查是否符合WCAG标准
 */
export function meetsWCAG(
  foreground: string,
  background: string,
  level: "AA" | "AAA" = "AA",
  isLargeText: boolean = false
): boolean {
  const ratio = getContrastRatio(foreground, background);

  if (level === "AAA") {
    return isLargeText ? ratio >= 4.5 : ratio >= 7;
  }

  return isLargeText ? ratio >= 3 : ratio >= 4.5;
}

// 使用
const ratio = getContrastRatio("#000000", "#ffffff"); // 21
const meetsAA = meetsWCAG("#000000", "#ffffff", "AA"); // true
```

```tsx
import { useState } from "react";

export default function AccessibleForm() {
  const [formData, setFormData] = useState({
    name: "",
    email: "",
    password: "",
    agree: false,
  });

  const [errors, setErrors] = useState<Record<string, string>>({});

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();

    const newErrors: Record<string, string> = {};

    if (!formData.name) {
      newErrors.name = "请输入姓名";
    }

    if (!formData.email) {
      newErrors.email = "请输入邮箱";
    } else if (!/\S+@\S+\.\S+/.test(formData.email)) {
      newErrors.email = "邮箱格式不正确";

    if (!formData.password) {
      newErrors.password = "请输入密码";
    } else if (formData.password.length < 8) {
      newErrors.password = "密码至少8个字符";
    }

    if (!formData.agree) {
      newErrors.agree = "请同意服务条款";
    }

    setErrors(newErrors);

    if (Object.keys(newErrors).length === 0) {
      console.log("表单提交:", formData);

    <form onSubmit={handleSubmit} noValidate>
      {/* 姓名 */}
```

### 3.2 表单验证

```tsx
// app/components/FormValidation.tsx
"use client";

import { useState, useId } from "react";

interface InputProps {
  label: string;
  type?: string;
  required?: boolean;
  pattern?: string;
  minLength?: number;
  helpText?: string;
}

function AccessibleInput({
  label,
  type = "text",
  required = false,
  pattern,
  minLength,
  helpText,
}: InputProps) {
  const [value, setValue] = useState("");
  const [error, setError] = useState("");
  const [touched, setTouched] = useState(false);

  const id = useId();
  const helpId = `${id}-help`;
  const errorId = `${id}-error`;

  const validate = (val: string) => {
    if (required && !val) {
      return "此字段为必填项";
    }

    if (pattern && !new RegExp(pattern).test(val)) {
      return "格式不正确";
    }

    if (minLength && val.length < minLength) {
      return `至少${minLength}个字符`;
    }

    return "";
  };

  const handleBlur = () => {
    setTouched(true);
    setError(validate(value));
  };

  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const newValue = e.target.value;
    setValue(newValue);

    if (touched) {
      setError(validate(newValue));
    }
  };

  return (
    <div className="mb-4">
      <label htmlFor={id} className="block mb-2 font-medium">
        {label}
        {required && (
          <span aria-label="必填" className="text-red-500 ml-1">
            *
          </span>
        )}
      </label>

      <input
        type={type}
        id={id}
        value={value}
        onChange={handleChange}
        onBlur={handleBlur}
        aria-required={required}
        aria-invalid={!!error}
        aria-describedby={`${helpText ? helpId : ""} ${
          error ? errorId : ""
        }`.trim()}
        className={`w-full px-3 py-2 border rounded ${
          error ? "border-red-500" : "border-gray-300"
        }`}
      />

      {helpText && (
        <p id={helpId} className="mt-1 text-sm text-gray-500">
          {helpText}
        </p>
      )}

      {error && (
        <p id={errorId} role="alert" className="mt-1 text-sm text-red-500">
          {error}
        </p>
      )}
    </div>
  );
}

export default function FormValidation() {
  return (
    <form>
      <AccessibleInput
        label="用户名"
        required
        minLength={3}
        helpText="至少3个字符"
      />

      <AccessibleInput
        label="邮箱"
        type="email"
        required
        pattern="^[^\s@]+@[^\s@]+\.[^\s@]+$"
        helpText="请输入有效的邮箱地址"
      />

      <AccessibleInput
        label="电话"
        type="tel"
        pattern="^1[3-9]\d{9}$"
        helpText="请输入11位手机号"
      />
    </form>
  );
}
```

## 7. 注意事项 (Precautions)

### 7.1 不要过度依赖 ARIA

**原则**: 优先使用语义化 HTML,ARIA 是补充而非替代。

```tsx
// 好的做法
<button>点击我</button>

// 不好的做法
<div role="button" tabIndex={0} onClick={handleClick}>点击我</div>
```

### 7.2 不要隐藏焦点指示器

```css
/* 不好的做法 */
*:focus {
  outline: none;
}

/* 好的做法 */
*:focus-visible {
  outline: 2px solid blue;
  outline-offset: 2px;
}
```

### 7.3 颜色不是唯一的信息传达方式

```tsx
// 不好的做法
<p style={{ color: 'red' }}>错误</p>

// 好的做法
<p className="text-red-500">
  <span aria-label="错误">❌</span> 错误
</p>
```

### 7.4 动态内容要通知屏幕阅读器

```tsx
// 使用aria-live
<div aria-live="polite" aria-atomic="true">
  {statusMessage}
</div>
```

### 7.5 测试要覆盖真实用户场景

不要只依赖自动化工具,要进行真实的屏幕阅读器测试和键盘导航测试。

## 8. 常见问题 (FAQ)

### 8.1 什么时候使用 aria-label vs aria-labelledby?

**问题**: 这两个属性有什么区别?

**解决方案**:

| 属性            | 使用场景               | 示例                                   |
| --------------- | ---------------------- | -------------------------------------- |
| aria-label      | 直接提供标签文本       | `<button aria-label="关闭">×</button>` |
| aria-labelledby | 引用页面上已存在的元素 | `<div aria-labelledby="title">`        |

**选择原则**: 如果页面上已有合适的文本,使用 aria-labelledby;否则使用 aria-label。

### 8.2 如何处理图标按钮的无障碍?

**问题**: 只有图标没有文字的按钮如何让屏幕阅读器理解?

**解决方案**:

```tsx
// 方案1: 使用aria-label
<button aria-label="搜索">
  <SearchIcon />
</button>

// 方案2: 使用sr-only文本
<button>
  <SearchIcon aria-hidden="true" />
  <span className="sr-only">搜索</span>
</button>

// 方案3: 使用title(不推荐,屏幕阅读器支持不一致)
<button title="搜索">
  <SearchIcon />
</button>
```

### 8.3 单页应用路由切换时如何通知屏幕阅读器?

**问题**: Next.js 路由切换时,屏幕阅读器不知道页面已经变化。

**解决方案**:

```tsx
// app/components/RouteAnnouncer.tsx
"use client";

import { useEffect, useState } from "react";
import { usePathname } from "next/navigation";

export default function RouteAnnouncer() {
  const pathname = usePathname();
  const [announcement, setAnnouncement] = useState("");

  useEffect(() => {
    // 路由变化时更新通知
    setAnnouncement(`已导航到 ${document.title}`);
  }, [pathname]);

  return (
    <div
      role="status"
      aria-live="assertive"
      aria-atomic="true"
      className="sr-only"
    >
      {announcement}
    </div>
  );
}

// app/layout.tsx
export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <html>
      <body>
        <RouteAnnouncer />
        {children}
      </body>
    </html>
  );
}
```

### 8.4 如何让自定义下拉菜单无障碍?

**问题**: 使用 div 实现的下拉菜单如何支持键盘导航?

**解决方案**:

```tsx
// app/components/AccessibleDropdown.tsx
"use client";

import { useState, useRef, useEffect } from "react";

export default function AccessibleDropdown() {
  const [isOpen, setIsOpen] = useState(false);
  const [focusedIndex, setFocusedIndex] = useState(-1);
  const buttonRef = useRef<HTMLButtonElement>(null);
  const menuRef = useRef<HTMLDivElement>(null);

  const options = ["选项1", "选项2", "选项3"];

  useEffect(() => {
    if (isOpen && focusedIndex >= 0) {
      const items = menuRef.current?.querySelectorAll('[role="menuitem"]');
      (items?.[focusedIndex] as HTMLElement)?.focus();
    }
  }, [focusedIndex, isOpen]);

  const handleKeyDown = (e: React.KeyboardEvent) => {
    switch (e.key) {
      case "ArrowDown":
        e.preventDefault();
        setFocusedIndex((prev) => (prev + 1) % options.length);
        break;
      case "ArrowUp":
        e.preventDefault();
        setFocusedIndex((prev) => (prev - 1 + options.length) % options.length);
        break;
      case "Escape":
        setIsOpen(false);
        buttonRef.current?.focus();
        break;
      case "Enter":
      case " ":
        if (focusedIndex >= 0) {
          console.log("选择:", options[focusedIndex]);
          setIsOpen(false);
        }
        break;
    }
  };

  return (
    <div className="relative">
      <button
        ref={buttonRef}
        aria-haspopup="true"
        aria-expanded={isOpen}
        onClick={() => setIsOpen(!isOpen)}
        className="px-4 py-2 bg-blue-500 text-white rounded"
      >
        选择选项
      </button>

      {isOpen && (
        <div
          ref={menuRef}
          role="menu"
          onKeyDown={handleKeyDown}
          className="absolute mt-2 bg-white border rounded shadow-lg"
        >
          {options.map((option, index) => (
            <div
              key={option}
              role="menuitem"
              tabIndex={-1}
              onClick={() => {
                console.log("选择:", option);
                setIsOpen(false);
              }}
              className="px-4 py-2 hover:bg-gray-100 cursor-pointer"
            >
              {option}
            </div>
          ))}
        </div>
      )}
    </div>
  );
}
```

### 8.5 如何处理加载状态的无障碍?

**问题**: 异步加载数据时如何通知屏幕阅读器?

**解决方案**:

```tsx
// app/components/LoadingState.tsx
"use client";

import { useState, useEffect } from "react";

export default function LoadingState() {
  const [data, setData] = useState<string | null>(null);
  const [loading, setLoading] = useState(false);

  const loadData = async () => {
    setLoading(true);
    await new Promise((resolve) => setTimeout(resolve, 2000));
    setData("数据加载完成");
    setLoading(false);
  };

  return (
    <div>
      <button onClick={loadData} disabled={loading}>
        加载数据
      </button>

      {/* 加载状态通知 */}
      <div role="status" aria-live="polite" aria-atomic="true">
        {loading && <p>正在加载数据...</p>}
        {data && <p>{data}</p>}
      </div>

      {/* 视觉加载指示器 */}
      {loading && (
        <div className="mt-4" aria-hidden="true">
          <div className="animate-spin h-8 w-8 border-4 border-blue-500 border-t-transparent rounded-full"></div>
        </div>
      )}
    </div>
  );
}
```

## 9. 总结 (Summary)

### 9.1 核心要点

无障碍设计不是可选项,而是必需品。它不仅是法律要求,更是道德责任和商业机会。

**语义化 HTML**: 使用正确的 HTML 元素是无障碍的基础。

**ARIA 补充**: 在语义化 HTML 不足时使用 ARIA,但不要过度依赖。

**键盘导航**: 确保所有功能都能通过键盘访问。

**颜色对比度**: 确保文本和背景有足够的对比度。

**测试**: 使用自动化工具和手动测试相结合。

### 9.2 最佳实践总结

| 方面 | 推荐做法              | 避免做法                    |
| ---- | --------------------- | --------------------------- |
| HTML | 使用语义化元素        | 滥用 div 和 span            |
| ARIA | 作为补充使用          | 过度依赖或误用              |
| 键盘 | 所有功能可键盘访问    | 只支持鼠标操作              |
| 焦点 | 清晰的焦点指示器      | 隐藏焦点指示器              |
| 颜色 | 符合对比度标准        | 仅用颜色传达信息            |
| 图像 | 提供有意义的 alt 文本 | 空 alt 或无意义的 alt       |
| 表单 | 标签与输入框关联      | 使用 placeholder 代替 label |
| 测试 | 自动化+手动测试       | 只依赖自动化工具            |

### 9.3 WCAG 2.1 快速参考

**A 级**(最基本):

- 提供文本替代
- 提供时间媒体的替代
- 内容可以以不同方式呈现
- 内容易于看和听

**AA 级**(推荐目标):

- 对比度至少 4.5:1(普通文本)
- 文本可以缩放到 200%
- 不仅依赖颜色传达信息
- 所有功能可键盘访问

**AAA 级**(最高标准):

- 对比度至少 7:1(普通文本)
- 不使用纯文本图像
- 提供手语翻译

### 9.4 实施路线图

**第一阶段**: 基础无障碍

- 使用语义化 HTML
- 为图像添加 alt 文本
