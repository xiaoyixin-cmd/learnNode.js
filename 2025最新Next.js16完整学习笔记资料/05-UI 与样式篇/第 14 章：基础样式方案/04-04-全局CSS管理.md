# 全局 CSS 管理

## 1. 概述 (Overview)

全局 CSS 是指作用于整个应用的样式，包括重置样式、基础样式、字体定义、CSS 变量等。在 Next.js 16 中，全局 CSS 的管理方式有了新的改进，特别是在 App Router 中的使用更加灵活和强大。

### 1.1 全局 CSS 的作用

**基础样式重置**：

- 统一不同浏览器的默认样式
- 设置全局的盒模型
- 定义基础的排版样式
- 移除默认的边距和内边距

**全局变量定义**：

- CSS 自定义属性（CSS Variables）
- 主题颜色定义
- 间距系统
- 字体系统

**通用工具类**：

- 布局辅助类
- 文本对齐类
- 显示/隐藏类
- 响应式工具类

### 1.2 Next.js 16 中的全局 CSS

🆕 **Next.js 16 新特性**：

- 支持在任何布局或页面中导入全局 CSS
- 改进的 CSS 加载顺序
- 更好的 CSS 优化和压缩
- 支持 Lightning CSS（实验性）

⚡ **性能提升**：

- Turbopack 加速 CSS 编译
- 自动提取关键 CSS
- 优化的 CSS 代码分割
- 更小的 CSS 文件体积

### 1.3 全局 CSS vs 局部 CSS

| 特性     | 全局 CSS           | CSS Modules |
| :------- | :----------------- | :---------- |
| 作用域   | 全局               | 局部        |
| 命名冲突 | 可能冲突           | 自动避免    |
| 适用场景 | 基础样式、重置样式 | 组件样式    |
| 加载时机 | 应用启动时         | 按需加载    |
| 文件大小 | 较大               | 较小        |

## 2. 基础使用 (Basic Usage)

### 2.1 在 App Router 中使用全局 CSS

在 Next.js 16 的 App Router 中，可以在根布局中导入全局 CSS：

```tsx
// app/layout.tsx
import "./globals.css";

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="zh-CN">
      <body>{children}</body>
    </html>
  );
}
```

```css
/* app/globals.css */
* {
  box-sizing: border-box;
  padding: 0;
  margin: 0;
}

html,
body {
  max-width: 100vw;
  overflow-x: hidden;
}

body {
  color: rgb(var(--foreground-rgb));
  background: linear-gradient(
      to bottom,
      transparent,
      rgb(var(--background-end-rgb))
    ) rgb(var(--background-start-rgb));
}

a {
  color: inherit;
  text-decoration: none;
}
```

### 2.2 在 Pages Router 中使用全局 CSS

在 Pages Router 中，全局 CSS 只能在 `_app.tsx` 中导入：

```tsx
// pages/_app.tsx
import "@/styles/globals.css";
import type { AppProps } from "next/app";

export default function App({ Component, pageProps }: AppProps) {
  return <Component {...pageProps} />;
}
```

### 2.3 导入多个全局 CSS 文件

可以导入多个全局 CSS 文件：

```tsx
// app/layout.tsx
import "./globals.css";
import "./reset.css";
import "./typography.css";
import "./utilities.css";

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="zh-CN">
      <body>{children}</body>
    </html>
  );
}
```

### 2.4 从 node_modules 导入 CSS

可以直接导入第三方库的 CSS：

```tsx
// app/layout.tsx
import "normalize.css";
import "swiper/css";
import "react-toastify/dist/ReactToastify.css";
import "./globals.css";

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="zh-CN">
      <body>{children}</body>
    </html>
  );
}
```

```tsx
// components/ThemeToggle.tsx
"use client";

import { useEffect, useState } from "react";

export function ThemeToggle() {
  const [theme, setTheme] = useState<"light" | "dark">("light");

  useEffect(() => {
    const savedTheme =
      (localStorage.getItem("theme") as "light" | "dark") || "light";
    setTheme(savedTheme);
    document.documentElement.setAttribute("data-theme", savedTheme);
  }, []);

  const toggleTheme = () => {
    const newTheme = theme === "light" ? "dark" : "light";
    setTheme(newTheme);
    localStorage.setItem("theme", newTheme);
    document.documentElement.setAttribute("data-theme", newTheme);
  };

  return (
    <button onClick={toggleTheme}>{theme === "light" ? "🌙" : "☀️"}</button>
  );
}
```

## 5. 字体管理 (Font Management)

### 5.1 使用 next/font

Next.js 16 提供了内置的字体优化：

```tsx
// app/layout.tsx
import { Inter, Roboto_Mono } from "next/font/google";

const inter = Inter({
  subsets: ["latin"],
  display: "swap",
  variable: "--font-inter",
});

const robotoMono = Roboto_Mono({
  subsets: ["latin"],
  display: "swap",
  variable: "--font-roboto-mono",
});

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="zh-CN" className={`${inter.variable} ${robotoMono.variable}`}>
      <body>{children}</body>
    </html>
  );
}
```

```css
/* globals.css */
body {
  font-family: var(--font-inter), sans-serif;
}

code {
  font-family: var(--font-roboto-mono), monospace;
}
```

### 5.3 字体加载策略

```tsx
// app/layout.tsx
import { Inter } from "next/font/google";

const inter = Inter({
  subsets: ["latin"],
  display: "swap", // 字体加载策略
  preload: true, // 预加载
  fallback: ["system-ui", "arial"], // 后备字体
  adjustFontFallback: true, // 调整后备字体
});
```

**字体加载策略对比**：

| 策略     | 说明                             | 适用场景         |
| :------- | :------------------------------- | :--------------- |
| swap     | 立即显示后备字体，字体加载后替换 | 大多数情况       |
| block    | 短暂阻塞，然后显示后备字体       | 字体很重要的场景 |
| fallback | 极短阻塞，然后显示后备字体       | 性能优先         |
| optional | 仅在缓存时使用自定义字体         | 性能最优先       |

## 9. 注意事项 (Precautions)

### 9.1 避免过度使用全局样式

```css
/* ❌ 不好：过多的全局样式 */
.button {
}
.card {
}
.modal {
}
.dropdown {
}
/* 这些应该是组件样式，不是全局样式 */

/* ✅ 好：只定义真正全局的样式 */
:root {
  /* CSS 变量 */
}
* {
  /* 重置样式 */
}
body {
  /* 基础样式 */
}
h1,
h2,
h3 {
  /* 排版样式 */
}
```

### 9.2 CSS 变量命名规范

```css
/* ❌ 不好：命名不清晰 */
:root {
  --c1: #3b82f6;
  --s1: 0.5rem;
}

/* ✅ 好：语义化命名 */
:root {
  --color-primary: #3b82f6;
  --spacing-sm: 0.5rem;
}
```

### 9.3 注意 CSS 加载顺序

```tsx
// ❌ 不好：顺序错误
import "./globals.css";
import "normalize.css"; // 应该先导入

// ✅ 好：正确的顺序
import "normalize.css"; // 1. 重置样式
import "./globals.css"; // 2. 全局样式
```

### 9.4 避免样式冲突

```css
/* ❌ 不好：可能与组件样式冲突 */
.container {
  padding: 1rem;
  /* 太通用的类名 */
}

/* ✅ 好：使用更具体的类名或仅定义变量 */
:root {
  --container-padding: 1rem;
}
```

## 10. 常见问题 (FAQ)

### 10.1 全局 CSS 在哪里导入？

**问题**：应该在哪里导入全局 CSS？

**回答**：

在 App Router 中，在根布局导入：

```tsx
// app/layout.tsx
import "./globals.css";

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html>
      <body>{children}</body>
    </html>
  );
}
```

在 Pages Router 中，在 `_app.tsx` 导入：

```tsx
// pages/_app.tsx
import "@/styles/globals.css";

export default function App({ Component, pageProps }: AppProps) {
  return <Component {...pageProps} />;
}
```

### 10.2 如何实现主题切换？

**问题**：如何在 Next.js 16 中实现明暗主题切换？

**回答**：

使用 CSS 变量和数据属性：

```css
/* globals.css */
:root {
  --bg: white;
  --text: black;
}

[data-theme="dark"] {
  --bg: black;
  --text: white;
}

body {
  background-color: var(--bg);
  color: var(--text);
}
```

```tsx
// components/ThemeToggle.tsx
"use client";

import { useEffect, useState } from "react";

export function ThemeToggle() {
  const [theme, setTheme] = useState<"light" | "dark">("light");

  useEffect(() => {
    const saved =
      (localStorage.getItem("theme") as "light" | "dark") || "light";
    setTheme(saved);
    document.documentElement.setAttribute("data-theme", saved);
  }, []);

  const toggle = () => {
    const newTheme = theme === "light" ? "dark" : "light";
    setTheme(newTheme);
    localStorage.setItem("theme", newTheme);
    document.documentElement.setAttribute("data-theme", newTheme);
  };

  return <button onClick={toggle}>切换主题</button>;
}
```

### 10.3 全局 CSS 和 CSS Modules 可以一起使用吗？

**问题**：能否同时使用全局 CSS 和 CSS Modules？

**回答**：

可以，而且这是推荐的做法：

- **全局 CSS**：用于重置样式、CSS 变量、基础样式
- **CSS Modules**：用于组件特定的样式

```tsx
// app/layout.tsx
import "./globals.css"; // 全局样式

// components/Button.tsx
import styles from "./Button.module.css"; // 组件样式

export function Button() {
  return <button className={styles.button}>按钮</button>;
}
```

### 10.4 如何优化全局 CSS 的性能？

**问题**：全局 CSS 文件很大，如何优化？

**回答**：

1. **拆分文件**：

```tsx
import "./reset.css";
import "./variables.css";
import "./typography.css";
import "./utilities.css";
```

2. **使用 PurgeCSS**（如果使用 Tailwind）：

```javascript
// tailwind.config.js
module.exports = {
  content: ["./app/**/*.{js,ts,jsx,tsx}"],
  // ...
};
```

3. **启用 CSS 优化**：

```javascript
// next.config.js
module.exports = {
  experimental: {
    optimizeCss: true,
  },
};
```

### 10.5 如何处理第三方库的全局样式？

**问题**：第三方库的样式与我的全局样式冲突怎么办？

**回答**：

1. **调整导入顺序**：

```tsx
import "third-party-lib/styles.css"; // 先导入第三方
import "./globals.css"; // 后导入自己的，可以覆盖
```

2. **使用更具体的选择器**：

```css
/* globals.css */
body .my-component {
  /* 使用更具体的选择器覆盖第三方样式 */
}
```

3. **使用 CSS 层叠层（CSS Cascade Layers）**：

```css
@layer third-party, base, components;

@layer third-party {
  @import "third-party-lib/styles.css";
}

@layer base {
  /* 你的基础样式 */
}
```

## 11. 总结 (Summary)

### 11.1 核心要点

**全局 CSS 的作用**：

1. **样式重置**：统一浏览器默认样式
2. **变量系统**：定义全局 CSS 变量
3. **基础样式**：设置排版、颜色、间距
4. **工具类**：提供通用的辅助类
5. **主题系统**：支持多主题切换

**Next.js 16 的优势**：

1. **灵活导入**：可以在任何布局中导入
2. **自动优化**：CSS 压缩和代码分割
3. **字体优化**：next/font 自动优化字体加载
4. **Turbopack**：更快的编译速度

### 11.2 最佳实践总结

| 场景     | 推荐做法                          | 避免做法                  |
| :------- | :-------------------------------- | :------------------------ |
| 样式重置 | 使用现代 CSS 重置或 Normalize.css | 不重置样式                |
| CSS 变量 | 语义化命名，分类组织              | 随意命名                  |
| 工具类   | 只定义常用的工具类                | 定义过多工具类            |
| 主题切换 | 使用 CSS 变量                     | 动态生成样式              |
| 字体加载 | 使用 next/font                    | 直接使用 Google Fonts CDN |

### 11.3 关键收获

1. **全局 CSS 是项目基础**，每个项目都需要
2. **CSS 变量是主题系统的核心**，灵活且性能好
3. **next/font 自动优化字体**，提升性能
4. **合理组织全局样式**，避免过度使用
5. **与 CSS Modules 配合使用**，发挥各自优势

### 11.4 下一步建议

1. **建立设计系统**：定义完整的 CSS 变量系统
2. **优化字体加载**：使用 next/font 优化字体
3. **实现主题切换**：支持明暗主题
4. **性能优化**：启用 CSS 优化选项
5. **文档化**：记录全局样式的使用规范

全局 CSS 在 Next.js 16 中是构建应用的基础。通过合理使用 CSS 变量、字体优化和主题系统，可以创建一个灵活、高性能且易于维护的样式系统。
