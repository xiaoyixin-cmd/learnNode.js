## 2. React 19.2 核心特性 (React 19.2 Features)

### 2.1 React Server Components (RSC)

#### 2.1.1 Server Components 概念

定义:在服务器上渲染的 React 组件,可访问后端资源,零客户端 JavaScript。

工作原理:

```
客户端请求
  ↓
服务器执行 Server Component
  ↓
访问数据库/API/文件系统
  ↓
生成 RSC Payload
  ↓
发送到客户端
  ↓
React 在客户端重构组件树
```

基础示例:

```tsx
// app/products/page.tsx - Server Component
export default async function ProductsPage() {
  // 直接访问数据库
  const products = await db.product.findMany();

  return (
    <div>
      <h1>产品列表</h1>
      {products.map((product) => (
        <ProductCard key={product.id} product={product} />
      ))}
    </div>
  );
}

// ProductCard.tsx - Server Component
export default function ProductCard({ product }) {
  return (
    <div className="card">
      <h2>{product.name}</h2>
      <p>{product.description}</p>
      <p className="price">¥{product.price}</p>
      <AddToCartButton productId={product.id} />
    </div>
  );
}

// AddToCartButton.tsx - Client Component
("use client");

export default function AddToCartButton({ productId }) {
  const [isAdding, setIsAdding] = useState(false);

  const handleClick = async () => {
    setIsAdding(true);
    await addToCart(productId);
    setIsAdding(false);
  };

  return (
    <button onClick={handleClick} disabled={isAdding}>
      {isAdding ? "添加中..." : "加入购物车"}
    </button>
  );
}
```

#### 2.1.2 Server vs Client Components

对比表格:

| 特性        | Server Component   | Client Component   |
| ----------- | ------------------ | ------------------ |
| 声明方式    | 默认               | 'use client'       |
| 渲染位置    | 服务器             | 客户端             |
| 数据访问    | ✅ 数据库/文件系统 | ❌ 仅浏览器 API    |
| Hooks       | ❌ 不支持          | ✅ 支持            |
| 交互        | ❌ 无法监听事件    | ✅ 支持事件        |
| 浏览器 API  | ❌ 无法访问        | ✅ 可访问          |
| Bundle 大小 | 0 (不发送到客户端) | 占用客户端 Bundle  |
| 性能        | 更快 (服务器执行)  | 取决于 Bundle 大小 |

使用场景:

**Server Component 适用于**:

- 数据获取
- 访问后端资源
- 保护敏感信息 (API keys)
- 减少客户端 JavaScript
- SEO 优化

**Client Component 适用于**:

- 交互功能 (onClick, onChange)
- 状态管理 (useState, useReducer)
- 浏览器 API (localStorage, window)
- React Hooks
- 第三方客户端库

### 2.2 Server Actions

#### 2.2.1 Server Actions 概念

定义:可以从客户端直接调用的服务器函数,用于表单处理和数据变更。

特点:

- 类型安全
- 自动序列化
- CSRF 保护
- 渐进增强
- 错误处理

基础示例:

```tsx
// app/actions.ts
"use server";

import { revalidatePath } from "next/cache";
import { redirect } from "next/navigation";

export async function createPost(formData: FormData) {
  const title = formData.get("title") as string;
  const content = formData.get("content") as string;

  // 验证
  if (!title || !content) {
    return { success: false, error: "标题和内容不能为空" };
  }

  // 数据库操作
  try {
    const post = await db.post.create({
      data: { title, content },
    });

    // 重新验证缓存
    revalidatePath("/blog");

    // 重定向
    redirect(`/blog/${post.id}`);
  } catch (error) {
    return { success: false, error: "创建失败" };
  }
}
```

```tsx
import { useState, useTransition } from "react";

}
```

### 7.1 版本相关

**Q1: Next.js 16 必须使用 React 19 吗?**

- 通过严格测试
- 被数千个 Next.js 应用使用

### 7.2 Server Components 相关

**Q4: 什么时候使用 Server Component?**

A: 默认使用 Server Component,除非需要:

- useState/useEffect 等 Hooks
- 浏览器 API (localStorage, window)
- 事件监听 (onClick, onChange)
- 第三方客户端库

**Q5: Server Component 可以调用 API 吗?**

A: 可以,而且推荐:

```tsx
export default async function Page() {
  // ✅ 直接调用 API
  const data = await fetch("https://api.example.com/data").then((r) =>
    r.json()
  );

  // ✅ 访问数据库
  const users = await db.user.findMany();

  // ✅ 读取文件
  const content = await fs.readFile("data.json", "utf-8");

  return <div>{data.title}</div>;
}
```

**Q6: Server Component 和 SSR 有什么区别?**

A: 核心差异:

传统 SSR:

- 服务器渲染 HTML
- 客户端水合所有组件
- 所有组件代码发送到客户端

Server Components:

- 服务器渲染组件
- 仅交互组件发送到客户端
- 零客户端 JavaScript (非交互组件)

### 7.3 Server Actions 相关

**Q7: Server Actions 和 API Routes 有什么区别?**

A: 对比:

Server Actions:

- 类型安全
- 自动序列化
- 表单集成
- 渐进增强
- 内部使用

API Routes:

- 需要手动类型定义
- 手动处理请求
- 公开 API
- Webhook
- 外部调用

**Q8: Server Actions 安全吗?**

A: 非常安全:

```tsx
"use server";

export async function deleteUser(userId: string) {
  // ✅ 权限检查
  const session = await getSession();
  if (!session) {
    throw new Error("Unauthorized");
  }

  // ✅ 验证所有权
  const user = await db.user.findUnique({ where: { id: userId } });
  if (user.id !== session.userId) {
    throw new Error("Forbidden");
  }

  // ✅ 输入验证
  if (!userId || typeof userId !== "string") {
    throw new Error("Invalid input");
  }

  // 执行操作
  await db.user.delete({ where: { id: userId } });
}
```

自动保护:

- CSRF 保护
- 仅 POST 请求
- 自动加密传输
- 服务器端执行

### 7.4 并发特性相关

**Q9: useTransition 什么时候使用?**

A: 当有低优先级更新时:

```tsx
// 用户输入 → 高优先级
// 搜索结果 → 低优先级

function SearchPage() {
  const [input, setInput] = useState("");
  const [results, setResults] = useState([]);
  const [isPending, startTransition] = useTransition();

  function handleSearch(value: string) {
    setInput(value); // 立即更新输入框

    startTransition(() => {
      setResults(filterResults(value)); // 延迟更新结果
    });
  }

  return (
    <div>
      <input value={input} onChange={(e) => handleSearch(e.target.value)} />
      {isPending && <Spinner />}
      <ResultsList results={results} />
    </div>
  );
}
```

A: Suspense 捕获 Promise:

