# 增量静态生成原理

## 1. 架构总览

### 1.1 ISR 简介

增量静态生成(Incremental Static Regeneration, ISR)是 Next.js 的核心特性,它允许在运行时更新静态页面,无需重新构建整个应用。

**核心概念**:

- 按需生成: 首次访问时生成页面
- 后台重新验证: 定期更新过期页面
- 静态优先: 优先返回缓存的静态页面
- 增量更新: 只更新变化的页面

### 1.2 工作原理

ISR 的基本流程:

```
请求 → 检查缓存 → 返回静态页面 →
后台重新验证 → 更新缓存 → 下次请求返回新页面
```

### 1.3 与其他渲染模式的对比

| 特性       | SSG      | ISR        | SSR      |
| :--------- | :------- | :--------- | :------- |
| 构建时间   | 长       | 短         | 无       |
| 响应速度   | 快       | 快         | 慢       |
| 内容新鲜度 | 低       | 中         | 高       |
| 服务器负载 | 低       | 低         | 高       |
| 适用场景   | 静态内容 | 半动态内容 | 动态内容 |

## 2. 核心流程解析

### 2.1 页面生成流程

ISR 的页面生成包括初始生成和增量生成两个阶段。

**页面生成器**:

```typescript
// ISR 页面生成器

interface ISRConfig {
  revalidate: number; // 重新验证时间(秒)
  fallback: boolean | "blocking";
}

interface PageData {
  props: any;
  revalidate: number;
}

class ISRPageGenerator {
  private cache = new Map<string, CachedPage>();

  async generatePage(
    path: string,
    getStaticProps: () => Promise<PageData>
  ): Promise<string> {
    // 1. 检查缓存
    const cached = this.cache.get(path);

    if (cached && !this.isStale(cached)) {
      console.log(`[ISR] Serving cached page: ${path}`);
      return cached.html;
    }

    // 2. 生成新页面
    console.log(`[ISR] Generating page: ${path}`);
    const { props, revalidate } = await getStaticProps();
    const html = await this.renderPage(props);

    // 3. 更新缓存
    this.cache.set(path, {
      html,
      timestamp: Date.now(),
      revalidate: revalidate * 1000,
    });

    // 4. 如果有旧缓存,触发后台重新验证
    if (cached && this.isStale(cached)) {
      this.revalidateInBackground(path, getStaticProps);
    }

    return html;
  }

  private isStale(page: CachedPage): boolean {
    return Date.now() - page.timestamp > page.revalidate;
  }

  private async renderPage(props: any): Promise<string> {
    // 简化的渲染实现
    return `<html><body>${JSON.stringify(props)}</body></html>`;
  }

  private async revalidateInBackground(
    path: string,
    getStaticProps: () => Promise<PageData>
  ): Promise<void> {
    console.log(`[ISR] Revalidating in background: ${path}`);

    try {
      const { props, revalidate } = await getStaticProps();
      const html = await this.renderPage(props);

      this.cache.set(path, {
        html,
        timestamp: Date.now(),
        revalidate: revalidate * 1000,
      });

      console.log(`[ISR] Revalidation complete: ${path}`);
    } catch (error) {
      console.error(`[ISR] Revalidation failed: ${path}`, error);
    }
  }
}

interface CachedPage {
  html: string;
  timestamp: number;
  revalidate: number;
}
```

### 2.2 路径生成

ISR 需要预先生成一些路径,其他路径按需生成。

**路径生成器**:

```typescript
// 动态路径生成

interface PathParams {
  params: Record<string, string>;
}

class PathGenerator {
  async generateStaticPaths(
    getStaticPaths: () => Promise<{
      paths: PathParams[];
      fallback: boolean | "blocking";
    }>
  ): Promise<Set<string>> {
    const { paths, fallback } = await getStaticPaths();
    const generatedPaths = new Set<string>();

    for (const { params } of paths) {
      const path = this.buildPath(params);
      generatedPaths.add(path);
      console.log(`[ISR] Pre-generated path: ${path}`);
    }

    console.log(`[ISR] Fallback mode: ${fallback}`);

    return generatedPaths;
  }

  private buildPath(params: Record<string, string>): string {
    // 简化的路径构建
    return "/" + Object.values(params).join("/");
  }

  shouldGenerateOnDemand(
    path: string,
    preGeneratedPaths: Set<string>,
    fallback: boolean | "blocking"
  ): boolean {
    // 如果路径已预生成,不需要按需生成
    if (preGeneratedPaths.has(path)) {
      return false;
    }

    // 根据 fallback 模式决定是否按需生成
    return fallback === true || fallback === "blocking";
  }
}
```

### 2.3 缓存管理

ISR 的缓存管理是核心功能之一。

**缓存管理器**:

```typescript
// ISR 缓存管理

interface CacheEntry {
  html: string;
  props: any;
  timestamp: number;
  revalidate: number;
  tags: string[];
}

class ISRCacheManager {
  private cache = new Map<string, CacheEntry>();
  private tagIndex = new Map<string, Set<string>>();

  set(path: string, entry: Omit<CacheEntry, "timestamp">): void {
    const cacheEntry: CacheEntry = {
      ...entry,
      timestamp: Date.now(),
    };

    this.cache.set(path, cacheEntry);

    // 更新标签索引
    for (const tag of entry.tags) {
      if (!this.tagIndex.has(tag)) {
        this.tagIndex.set(tag, new Set());
      }
      this.tagIndex.get(tag)!.add(path);
    }

    console.log(`[Cache] Cached page: ${path}`);
  }

  get(path: string): CacheEntry | null {
    return this.cache.get(path) || null;
  }

  isStale(path: string): boolean {
    const entry = this.cache.get(path);

    if (!entry) {
      return true;
    }

    return Date.now() - entry.timestamp > entry.revalidate;
  }

  invalidate(path: string): void {
    this.cache.delete(path);
    console.log(`[Cache] Invalidated: ${path}`);
  }

  invalidateByTag(tag: string): void {
    const paths = this.tagIndex.get(tag);

    if (!paths) {
      return;
    }

    for (const path of paths) {
      this.invalidate(path);
    }

    this.tagIndex.delete(tag);
    console.log(`[Cache] Invalidated by tag: ${tag}`);
  }

  clear(): void {
    this.cache.clear();
    this.tagIndex.clear();
    console.log("[Cache] Cleared all cache");
  }
}
```

## 3. 关键模块源码分析

### 3.1 按需生成

按需生成允许在首次访问时生成页面。

**按需生成器**:

```typescript
// 按需页面生成

class OnDemandGenerator {
  private generator = new ISRPageGenerator();
  private pathGenerator = new PathGenerator();
  private generating = new Map<string, Promise<string>>();

  async handleRequest(
    path: string,
    getStaticProps: () => Promise<PageData>,
    fallback: boolean | "blocking"
  ): Promise<string | null> {
    // 1. 检查是否正在生成
    if (this.generating.has(path)) {
      console.log(`[OnDemand] Waiting for generation: ${path}`);
      return await this.generating.get(path)!;
    }

    // 2. 开始生成
    const promise = this.generate(path, getStaticProps, fallback);
    this.generating.set(path, promise);

    try {
      const html = await promise;
      return html;
    } finally {
      this.generating.delete(path);
    }
  }

  private async generate(
    path: string,
    getStaticProps: () => Promise<PageData>,
    fallback: boolean | "blocking"
  ): Promise<string | null> {
    if (fallback === false) {
      // 不支持按需生成,返回 404
      return null;
    }

    if (fallback === true) {
      // 返回 fallback 页面,后台生成
      this.generateInBackground(path, getStaticProps);
      return this.renderFallback();
    }

    // fallback === 'blocking'
    // 阻塞等待生成完成
    return await this.generator.generatePage(path, getStaticProps);
  }

  private async generateInBackground(
    path: string,
    getStaticProps: () => Promise<PageData>
  ): Promise<void> {
    try {
      await this.generator.generatePage(path, getStaticProps);
    } catch (error) {
      console.error(`[OnDemand] Background generation failed: ${path}`, error);
    }
  }

  private renderFallback(): string {
    return "<html><body><div>Loading...</div></body></html>";
  }
}
```

### 3.2 重新验证策略

ISR 支持多种重新验证策略。

**重新验证管理器**:

```typescript
// 重新验证策略

interface RevalidateOptions {
  path?: string;
  tag?: string;
  purge?: boolean;
}

class RevalidationManager {
  private cacheManager = new ISRCacheManager();
  private queue: string[] = [];
  private processing = false;

  async revalidate(options: RevalidateOptions): Promise<void> {
    if (options.path) {
      await this.revalidatePath(options.path, options.purge);
    } else if (options.tag) {
      await this.revalidateTag(options.tag, options.purge);
    }
  }

  private async revalidatePath(path: string, purge = false): Promise<void> {
    if (purge) {
      // 立即清除缓存
      this.cacheManager.invalidate(path);
      console.log(`[Revalidate] Purged: ${path}`);
    } else {
      // 加入重新验证队列
      this.queue.push(path);
      console.log(`[Revalidate] Queued: ${path}`);

      if (!this.processing) {
        this.processQueue();
      }
    }
  }

  private async revalidateTag(tag: string, purge = false): Promise<void> {
    if (purge) {
      this.cacheManager.invalidateByTag(tag);
      console.log(`[Revalidate] Purged tag: ${tag}`);
    } else {
      // 标记为需要重新验证
      console.log(`[Revalidate] Marked tag for revalidation: ${tag}`);
    }
  }

  private async processQueue(): Promise<void> {
    this.processing = true;

    while (this.queue.length > 0) {
      const path = this.queue.shift();

      if (!path) {
        break;
      }

      try {
        // 触发重新生成
        console.log(`[Revalidate] Processing: ${path}`);
        // 实际的重新生成逻辑
        await new Promise((resolve) => setTimeout(resolve, 100));
      } catch (error) {
        console.error(`[Revalidate] Failed: ${path}`, error);
      }
    }

    this.processing = false;
  }
}
```

### 3.3 完整 ISR 实现

将所有组件组合起来实现完整的 ISR。

**完整实现**:

```typescript
// 完整 ISR 实现

class ISREngine {
  private generator = new ISRPageGenerator();
  private pathGenerator = new PathGenerator();
  private cacheManager = new ISRCacheManager();
  private onDemandGenerator = new OnDemandGenerator();
  private revalidationManager = new RevalidationManager();

  async initialize(
    getStaticPaths: () => Promise<{
      paths: PathParams[];
      fallback: boolean | "blocking";
    }>
  ): Promise<void> {
    // 生成静态路径
    const paths = await this.pathGenerator.generateStaticPaths(getStaticPaths);
    console.log(`[ISR] Initialized with ${paths.size} pre-generated paths`);
  }

  async handleRequest(
    path: string,
    getStaticProps: () => Promise<PageData>,
    config: ISRConfig
  ): Promise<string> {
    // 1. 检查缓存
    const cached = this.cacheManager.get(path);

    if (cached && !this.cacheManager.isStale(path)) {
      console.log(`[ISR] Cache hit: ${path}`);
      return cached.html;
    }

    // 2. 如果缓存过期但存在,返回旧内容并后台更新
    if (cached && this.cacheManager.isStale(path)) {
      console.log(`[ISR] Stale cache, revalidating: ${path}`);
      this.revalidateInBackground(path, getStaticProps, config);
      return cached.html;
    }

    // 3. 缓存不存在,按需生成
    console.log(`[ISR] Cache miss, generating: ${path}`);
    const html = await this.onDemandGenerator.handleRequest(
      path,
      getStaticProps,
      config.fallback
    );

    if (!html) {
      throw new Error("Page not found");
    }

    return html;
  }

  private async revalidateInBackground(
    path: string,
    getStaticProps: () => Promise<PageData>,
    config: ISRConfig
  ): Promise<void> {
    try {
      const { props, revalidate } = await getStaticProps();
      const html = await this.generator.generatePage(path, getStaticProps);

      this.cacheManager.set(path, {
        html,
        props,
        revalidate: revalidate * 1000,
        tags: [],
      });

      console.log(`[ISR] Background revalidation complete: ${path}`);
    } catch (error) {
      console.error(`[ISR] Background revalidation failed: ${path}`, error);
    }
  }

  async revalidate(options: RevalidateOptions): Promise<void> {
    await this.revalidationManager.revalidate(options);
  }
}

// 使用示例
const isr = new ISREngine();

// 初始化
await isr.initialize(async () => ({
  paths: [
    { params: { id: "1" } },
    { params: { id: "2" } },
    { params: { id: "3" } },
  ],
  fallback: "blocking",
}));

// 处理请求
const html = await isr.handleRequest(
  "/posts/1",
  async () => ({
    props: { title: "Post 1", content: "Content..." },
    revalidate: 60,
  }),
  {
    revalidate: 60,
    fallback: "blocking",
  }
);

// 手动重新验证
await isr.revalidate({ path: "/posts/1" });
await isr.revalidate({ tag: "posts", purge: true });
```

## 4. 适用场景

### 4.1 最佳使用场景

ISR 特别适合以下场景:

1. **内容管理系统**: 博客、新闻网站
2. **电商网站**: 商品列表、商品详情
3. **文档网站**: API 文档、技术文档
4. **数据展示**: 统计数据、报表

### 4.2 性能对比

| 指标       | SSG        | ISR      | SSR  |
| :--------- | :--------- | :------- | :--- |
| 首次加载   | 极快       | 极快     | 慢   |
| 后续加载   | 极快       | 极快     | 慢   |
| 内容更新   | 需重新构建 | 自动更新 | 实时 |
| 构建时间   | 长         | 短       | 无   |
| 服务器成本 | 低         | 低       | 高   |

### 4.3 不适用场景

1. **实时数据**: 股票价格、体育比分
2. **用户特定内容**: 个人仪表板、购物车
3. **频繁更新**: 每秒更新的内容

## 6. 常见问题

**问题一: ISR 和 SSG 有什么区别?**

SSG 在构建时生成所有页面,ISR 可以在运行时增量生成和更新页面。ISR 是 SSG 的增强版本。

**问题二: revalidate 时间如何选择?**

根据内容更新频率选择:

- 新闻网站: 60-300 秒
- 博客: 3600-86400 秒(1 小时-1 天)
- 商品页面: 300-1800 秒(5-30 分钟)

**问题三: fallback 的三种模式有什么区别?**

- `false`: 只返回预生成的页面,其他返回 404
- `true`: 首次访问返回 fallback 页面,后台生成真实页面
- `'blocking'`: 首次访问阻塞等待页面生成完成

**问题四: 如何手动触发重新验证?**

使用 `revalidatePath` 或 `revalidateTag`:

```typescript
import { revalidatePath, revalidateTag } from "next/cache";

// 重新验证特定路径
await revalidatePath("/posts/1");

// 重新验证标签
await revalidateTag("posts");
```

// lib/cache-monitor.ts
export class CacheMonitor {
  private hits = 0;
  private misses = 0;

  recordHit() {
    this.hits++;
  }

  recordMiss() {
    this.misses++;
  }

  getStats() {
    const total = this.hits + this.misses;
    const hitRate = total > 0 ? (this.hits / total) * 100 : 0;

    return {
      hits: this.hits,
      misses: this.misses,
      hitRate: hitRate.toFixed(2) + "%",
    };
  }
}
