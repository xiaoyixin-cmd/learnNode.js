# 缓存机制工作原理

## 1. 架构总览

Next.js 16 的缓存系统是其性能优化的核心。它采用了多层缓存架构,在不同层级缓存不同类型的数据,从而实现最佳性能。

### 1.1 缓存层级结构

Next.js 16 的缓存系统包含以下几个层级:

**四层缓存架构**:

1. **请求记忆化(Request Memoization)** - 组件级别
2. **数据缓存(Data Cache)** - 跨请求持久化
3. **完整路由缓存(Full Route Cache)** - 静态页面缓存
4. **路由器缓存(Router Cache)** - 客户端缓存

| 缓存层级     | 作用范围 | 生命周期 | 存储位置       | 主要用途         |
| :----------- | :------- | :------- | :------------- | :--------------- |
| 请求记忆化   | 单个请求 | 请求期间 | 服务器内存     | 避免重复数据获取 |
| 数据缓存     | 跨请求   | 持久化   | 服务器文件系统 | 缓存 fetch 结果  |
| 完整路由缓存 | 跨请求   | 持久化   | 服务器文件系统 | 缓存渲染结果     |
| 路由器缓存   | 用户会话 | 会话期间 | 浏览器内存     | 客户端导航优化   |

### 1.2 缓存工作流程


### 1.3 缓存策略对比

Next.js 16 支持多种缓存策略:

| 策略         | 适用场景     | 性能 | 新鲜度 | 实现复杂度 |
| :----------- | :----------- | :--- | :----- | :--------- |
| 静态生成     | 内容不常变化 | 最高 | 低     | 低         |
| ISR          | 内容定期更新 | 高   | 中     | 中         |
| 按需重新验证 | 内容按需更新 | 高   | 高     | 中         |
| 动态渲染     | 内容实时变化 | 低   | 最高   | 低         |
| 流式渲染     | 部分内容慢   | 中   | 高     | 高         |

### 2.1 请求记忆化机制

请求记忆化是最内层的缓存,它确保在单个请求的渲染过程中,相同的数据获取只执行一次。

**记忆化实现**:

```typescript
// Next.js 内部的请求记忆化实现(简化版)

class RequestMemoization {
  private cache = new Map<string, Promise<any>>();
  private requestId: string;

  constructor() {
    this.requestId = generateRequestId();
  }

  async fetch(url: string, options?: RequestInit): Promise<Response> {
    // 生成缓存键
    const cacheKey = this.generateCacheKey(url, options);

    // 检查缓存
    if (this.cache.has(cacheKey)) {
      console.log(`[Memoization] Cache hit: ${url}`);
      return this.cache.get(cacheKey)!;
    }

    console.log(`[Memoization] Cache miss: ${url}`);

    // 发起请求并缓存 Promise
    const promise = fetch(url, options);
    this.cache.set(cacheKey, promise);

    return promise;
  }

  private generateCacheKey(url: string, options?: RequestInit): string {
    // 基于 URL 和请求选项生成唯一键
    const optionsKey = options
      ? JSON.stringify({
          method: options.method,
          headers: options.headers,
          body: options.body,
        })
      : "";

    return `${url}:${optionsKey}`;
  }

  clear() {
    this.cache.clear();
  }
}

// 每个请求都有自己的记忆化实例
const requestMemoization = new RequestMemoization();

// 使用示例
async function Page() {
  // 第一次调用,发起请求
  const data1 = await requestMemoization.fetch("/api/data");

  // 第二次调用,使用缓存
  const data2 = await requestMemoization.fetch("/api/data");

  // data1 和 data2 是同一个 Promise
  console.log(data1 === data2); // true

  return <div>...</div>;
}
```

### 2.2 数据缓存机制

数据缓存是持久化的,可以跨多个请求共享。

**数据缓存实现**:

```typescript
// 数据缓存的核心实现

interface CacheEntry {
  data: any;
  timestamp: number;
  revalidateAfter: number;
  tags: string[];
}

class DataCache {
  private cache = new Map<string, CacheEntry>();
  private cacheDir = ".next/cache/fetch-cache";

  async get(key: string): Promise<CacheEntry | null> {
    // 先检查内存缓存
    if (this.cache.has(key)) {
      const entry = this.cache.get(key)!;

      // 检查是否过期
      if (this.isValid(entry)) {
        return entry;
      }

      // 过期则删除
      this.cache.delete(key);
    }

    // 检查磁盘缓存
    const diskEntry = await this.readFromDisk(key);

    if (diskEntry && this.isValid(diskEntry)) {
      // 加载到内存
      this.cache.set(key, diskEntry);
      return diskEntry;
    }

    return null;
  }

  async set(key: string, data: any, options: CacheOptions): Promise<void> {
    const entry: CacheEntry = {
      data,
      timestamp: Date.now(),
      revalidateAfter: options.revalidate || Infinity,
      tags: options.tags || [],
    };

    // 存入内存
    this.cache.set(key, entry);

    // 持久化到磁盘
    await this.writeToDisk(key, entry);
  }

  private isValid(entry: CacheEntry): boolean {
    const now = Date.now();
    const age = now - entry.timestamp;

    return age < entry.revalidateAfter * 1000;
  }

  private async readFromDisk(key: string): Promise<CacheEntry | null> {
    try {
      const filePath = path.join(this.cacheDir, `${key}.json`);
      const content = await fs.readFile(filePath, "utf-8");
      return JSON.parse(content);
    } catch {
      return null;
    }
  }

  private async writeToDisk(key: string, entry: CacheEntry): Promise<void> {
    const filePath = path.join(this.cacheDir, `${key}.json`);
    await fs.mkdir(path.dirname(filePath), { recursive: true });
    await fs.writeFile(filePath, JSON.stringify(entry));
  }

  async revalidateTag(tag: string): Promise<void> {
    // 使所有包含该标签的缓存失效
    for (const [key, entry] of this.cache.entries()) {
      if (entry.tags.includes(tag)) {
        this.cache.delete(key);
        await this.deleteFromDisk(key);
      }
    }
  }

  private async deleteFromDisk(key: string): Promise<void> {
    const filePath = path.join(this.cacheDir, `${key}.json`);
    await fs.unlink(filePath).catch(() => {});
  }
}
```

**缓存键生成**:

```typescript
// 缓存键的生成逻辑

function generateCacheKey(url: string, options?: RequestInit): string {
  const components = [url];

  if (options) {
    // 包含请求方法
    if (options.method && options.method !== "GET") {
      components.push(`method:${options.method}`);
    }

    // 包含请求头
    if (options.headers) {
      const headers = new Headers(options.headers);
      const sortedHeaders = Array.from(headers.entries())
        .sort(([a], [b]) => a.localeCompare(b))
        .map(([key, value]) => `${key}:${value}`)
        .join(",");

      components.push(`headers:${sortedHeaders}`);
    }

    // 包含请求体
    if (options.body) {
      const bodyHash = createHash("md5")
        .update(String(options.body))
        .digest("hex");

      components.push(`body:${bodyHash}`);
    }
  }

  // 生成最终的缓存键
  const key = components.join("|");

  return createHash("sha256").update(key).digest("hex");
}

// 使用示例
const key1 = generateCacheKey("https://api.example.com/users");
// => "a1b2c3d4..."

const key2 = generateCacheKey("https://api.example.com/users", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ name: "John" }),
});
// => "e5f6g7h8..."
```

### 2.3 完整路由缓存

完整路由缓存存储整个页面的渲染结果,包括 HTML 和 RSC Payload。

**路由缓存实现**:

```typescript
// 完整路由缓存的实现

interface RouteCache {
  html: string;
  rsc: RSCPayload;
  headers: Record<string, string>;
  status: number;
  revalidateAfter: number;
  timestamp: number;
}

class FullRouteCache {
  private cache = new Map<string, RouteCache>();
  private cacheDir = ".next/cache/route-cache";

  async get(
    pathname: string,
    searchParams?: URLSearchParams
  ): Promise<RouteCache | null> {
    const key = this.generateKey(pathname, searchParams);

    // 检查内存缓存
    if (this.cache.has(key)) {
      const entry = this.cache.get(key)!;

      if (this.isValid(entry)) {
        return entry;
      }

      this.cache.delete(key);
    }

    // 检查磁盘缓存
    const diskEntry = await this.readFromDisk(key);

    if (diskEntry && this.isValid(diskEntry)) {
      this.cache.set(key, diskEntry);
      return diskEntry;
    }

    return null;
  }

  async set(
    pathname: string,
    searchParams: URLSearchParams | undefined,
    entry: Omit<RouteCache, "timestamp">
  ): Promise<void> {
    const key = this.generateKey(pathname, searchParams);

    const cacheEntry: RouteCache = {
      ...entry,
      timestamp: Date.now(),
    };

    this.cache.set(key, cacheEntry);
    await this.writeToDisk(key, cacheEntry);
  }

  private generateKey(
    pathname: string,
    searchParams?: URLSearchParams
  ): string {
    let key = pathname;

    if (searchParams && searchParams.toString()) {
      // 对查询参数排序以确保一致性
      const sorted = Array.from(searchParams.entries())
        .sort(([a], [b]) => a.localeCompare(b))
        .map(([k, v]) => `${k}=${v}`)
        .join("&");

      key += `?${sorted}`;
    }

    return createHash("md5").update(key).digest("hex");
  }

  private isValid(entry: RouteCache): boolean {
    if (entry.revalidateAfter === Infinity) {
      return true;
    }

    const age = Date.now() - entry.timestamp;
    return age < entry.revalidateAfter * 1000;
  }

  private async readFromDisk(key: string): Promise<RouteCache | null> {
    try {
      const filePath = path.join(this.cacheDir, `${key}.json`);
      const content = await fs.readFile(filePath, "utf-8");
      return JSON.parse(content);
    } catch {
      return null;
    }
  }

  private async writeToDisk(key: string, entry: RouteCache): Promise<void> {
    const filePath = path.join(this.cacheDir, `${key}.json`);
    await fs.mkdir(path.dirname(filePath), { recursive: true });
    await fs.writeFile(filePath, JSON.stringify(entry));
  }

  async invalidate(pathname: string): Promise<void> {
    // 删除所有匹配该路径的缓存
    for (const [key, entry] of this.cache.entries()) {
      if (key.startsWith(pathname)) {
        this.cache.delete(key);
        await this.deleteFromDisk(key);
      }
    }
  }

  private async deleteFromDisk(key: string): Promise<void> {
    const filePath = path.join(this.cacheDir, `${key}.json`);
    await fs.unlink(filePath).catch(() => {});
  }
}
```

### 2.4 路由器缓存

路由器缓存是客户端缓存,用于优化客户端导航。

**客户端缓存实现**:

```typescript
// 客户端路由器缓存

class RouterCache {
  private cache = new Map<string, CachedRoute>();
  private maxSize = 50; // 最多缓存50个路由
  private lruQueue: string[] = [];

  get(pathname: string): CachedRoute | null {
    const entry = this.cache.get(pathname);

    if (!entry) {
      return null;
    }

    // 检查是否过期
    if (this.isExpired(entry)) {
      this.cache.delete(pathname);
      return null;
    }

    // 更新 LRU 队列
    this.updateLRU(pathname);

    return entry;
  }

  set(pathname: string, route: CachedRoute): void {
    // 检查缓存大小
    if (this.cache.size >= this.maxSize && !this.cache.has(pathname)) {
      // 删除最久未使用的条目
      const oldest = this.lruQueue.shift();
      if (oldest) {
        this.cache.delete(oldest);
      }
    }

    this.cache.set(pathname, {
      ...route,
      timestamp: Date.now(),
    });

    this.updateLRU(pathname);
  }

  private updateLRU(pathname: string): void {
    // 从队列中移除
    const index = this.lruQueue.indexOf(pathname);
    if (index > -1) {
      this.lruQueue.splice(index, 1);
    }

    // 添加到队列末尾
    this.lruQueue.push(pathname);
  }

  private isExpired(entry: CachedRoute): boolean {
    const age = Date.now() - entry.timestamp;
    const maxAge = 30 * 1000; // 30秒

    return age > maxAge;
  }

  prefetch(pathname: string): void {
    // 预取路由数据
    if (this.cache.has(pathname)) {
      return;
    }

    fetch(`/_next/data/${pathname}.json`)
      .then((res) => res.json())
      .then((data) => {
        this.set(pathname, {
          data,
          timestamp: Date.now(),
        });
      })
      .catch((err) => {
        console.error("Prefetch failed:", err);
      });
  }

  invalidate(pathname: string): void {
    this.cache.delete(pathname);

    const index = this.lruQueue.indexOf(pathname);
    if (index > -1) {
      this.lruQueue.splice(index, 1);
    }
  }

  clear(): void {
    this.cache.clear();
    this.lruQueue = [];
  }
}

// 全局路由器缓存实例
const routerCache = new RouterCache();

// 使用示例
function Link({ href, children }: { href: string; children: React.ReactNode }) {
  const handleMouseEnter = () => {
    // 鼠标悬停时预取
    routerCache.prefetch(href);
  };

  const handleClick = (e: React.MouseEvent) => {
    e.preventDefault();

    // 检查缓存
    const cached = routerCache.get(href);

    if (cached) {
      // 使用缓存数据导航
      navigateWithCache(href, cached);
    } else {
      // 正常导航
      navigate(href);
    }
  };

  return (
    <a href={href} onMouseEnter={handleMouseEnter} onClick={handleClick}>
      {children}
    </a>
  );
}
```

### 3.1 缓存标签系统

缓存标签允许按需重新验证相关的缓存条目。

**标签系统实现**:

```typescript
// 缓存标签管理器

class CacheTagManager {
  private tagToKeys = new Map<string, Set<string>>();
  private keyToTags = new Map<string, Set<string>>();

  addTag(cacheKey: string, tag: string): void {
    // 添加标签到缓存键的映射
    if (!this.keyToTags.has(cacheKey)) {
      this.keyToTags.set(cacheKey, new Set());
    }
    this.keyToTags.get(cacheKey)!.add(tag);

    // 添加缓存键到标签的映射
    if (!this.tagToKeys.has(tag)) {
      this.tagToKeys.set(tag, new Set());
    }
    this.tagToKeys.get(tag)!.add(cacheKey);
  }

  getKeysByTag(tag: string): Set<string> {
    return this.tagToKeys.get(tag) || new Set();
  }

  getTagsByKey(cacheKey: string): Set<string> {
    return this.keyToTags.get(cacheKey) || new Set();
  }

  removeKey(cacheKey: string): void {
    const tags = this.keyToTags.get(cacheKey);

    if (tags) {
      // 从所有标签中移除该键
      for (const tag of tags) {
        const keys = this.tagToKeys.get(tag);
        if (keys) {
          keys.delete(cacheKey);

          // 如果标签下没有键了,删除标签
          if (keys.size === 0) {
            this.tagToKeys.delete(tag);
          }
        }
      }

      this.keyToTags.delete(cacheKey);
    }
  }

  revalidateTag(tag: string): Set<string> {
    const keys = this.getKeysByTag(tag);

    // 移除所有相关的键
    for (const key of keys) {
      this.removeKey(key);
    }

    return keys;
  }
}
```

### 3.2 增量静态再生成(ISR)

ISR 允许在不重新构建整个站点的情况下更新静态页面。

**ISR 实现原理**:

```typescript
// ISR 的核心实现

class ISRManager {
  private regenerationQueue = new Map<string, Promise<void>>();
  private locks = new Map<string, boolean>();

  async handleRequest(pathname: string, revalidate: number): Promise<Response> {
    const cached = await fullRouteCache.get(pathname);

    if (cached) {
      const age = Date.now() - cached.timestamp;

      if (age < revalidate * 1000) {
        // 缓存仍然有效,直接返回
        return new Response(cached.html, {
          headers: cached.headers,
          status: cached.status,
        });
      }

      // 缓存过期,但先返回旧内容
      const response = new Response(cached.html, {
        headers: {
          ...cached.headers,
          "X-Nextjs-Cache": "STALE",
        },
        status: cached.status,
      });

      // 在后台重新生成
      this.regenerateInBackground(pathname);

      return response;
    }

    // 没有缓存,同步生成
    return await this.regenerate(pathname);
  }

  private async regenerateInBackground(pathname: string): Promise<void> {
    // 检查是否已经在重新生成
    if (this.locks.get(pathname)) {
      return;
    }

    // 检查是否已经在队列中
    if (this.regenerationQueue.has(pathname)) {
      return this.regenerationQueue.get(pathname)!;
    }

    // 标记为正在重新生成
    this.locks.set(pathname, true);

    const promise = this.regenerate(pathname).finally(() => {
      this.locks.delete(pathname);
      this.regenerationQueue.delete(pathname);
    });

    this.regenerationQueue.set(pathname, promise);

    return promise;
  }

  private async regenerate(pathname: string): Promise<Response> {
    console.log(`[ISR] Regenerating: ${pathname}`);

    const startTime = Date.now();

    // 渲染页面
    const { html, rsc, headers, status } = await renderPage(pathname);

    // 存入缓存
    await fullRouteCache.set(pathname, undefined, {
      html,
      rsc,
      headers,
      status,
      revalidateAfter: getRevalidateTime(pathname),
    });

    const duration = Date.now() - startTime;
    console.log(`[ISR] Regenerated ${pathname} in ${duration}ms`);

    return new Response(html, {
      headers: {
        ...headers,
        "X-Nextjs-Cache": "MISS",
      },
      status,
    });
  }
}
```

### 3.3 缓存预热机制

缓存预热可以在用户访问前就准备好缓存。

**预热实现**:

```typescript
// 缓存预热管理器

class CacheWarmer {
  private warmupQueue: string[] = [];
  private isWarming = false;
  private concurrency = 5;

  async warmup(paths: string[]): Promise<void> {
    this.warmupQueue.push(...paths);

    if (!this.isWarming) {
      await this.processQueue();
    }
  }

  private async processQueue(): Promise<void> {
    this.isWarming = true;

    while (this.warmupQueue.length > 0) {
      // 取出一批路径
      const batch = this.warmupQueue.splice(0, this.concurrency);

      // 并行预热
      await Promise.all(batch.map((path) => this.warmPath(path)));
    }

    this.isWarming = false;
  }

  private async warmPath(pathname: string): Promise<void> {
    try {
      console.log(`[Warmup] Warming: ${pathname}`);

      // 检查是否已缓存
      const cached = await fullRouteCache.get(pathname);

      if (cached && fullRouteCache.isValid(cached)) {
        console.log(`[Warmup] Already cached: ${pathname}`);
        return;
      }

      // 渲染并缓存
      const { html, rsc, headers, status } = await renderPage(pathname);

      await fullRouteCache.set(pathname, undefined, {
        html,
        rsc,
        headers,
        status,
        revalidateAfter: Infinity,
      });

      console.log(`[Warmup] Warmed: ${pathname}`);
    } catch (error) {
      console.error(`[Warmup] Failed to warm ${pathname}:`, error);
    }
  }

  async warmupByPattern(pattern: RegExp): Promise<void> {
    // 从路由表中找到匹配的路径
    const routes = await getRoutes();
    const matchingPaths = routes.filter((route) => pattern.test(route));

    await this.warmup(matchingPaths);
  }

  async warmupPopular(limit: number = 100): Promise<void> {
    // 从分析数据中获取热门页面
    const popularPaths = await analytics.getPopularPages(limit);

    await this.warmup(popularPaths);
  }
}

// 使用示例
const warmer = new CacheWarmer();

// 在部署后预热缓存
async function postDeploy() {
  // 预热首页和主要页面
  await warmer.warmup(["/", "/about", "/products", "/contact"]);

  // 预热所有产品页面
  await warmer.warmupByPattern(/^\/products\/\d+$/);

  // 预热热门页面
  await warmer.warmupPopular(50);
}
```

## 6. 常见问题

### 6.1 基础问题

**问题一: Next.js 16 的缓存默认行为是什么?**

**回答**: Next.js 16 的默认缓存行为取决于渲染类型:

- fetch 请求默认会被缓存
- 静态页面默认会被缓存
- 动态页面不会被缓存


## 7. 总结

### 7.1 核心要点回顾

Next.js 16 的缓存系统是一个多层次、高度优化的架构,理解其工作原理对于构建高性能应用至关重要。

**关键概念**:

1. **四层缓存**: 请求记忆化、数据缓存、完整路由缓存、路由器缓存
2. **缓存策略**: 静态生成、ISR、按需重新验证、动态渲染
3. **缓存标签**: 允许精确控制缓存失效
4. **缓存预热**: 提前准备缓存提升性能

**最佳实践**:

1. 根据数据更新频率选择合适的缓存策略
2. 使用缓存标签进行精确的缓存失效
3. 合理设置 revalidate 时间
4. 监控缓存命中率和性能
5. 在生产环境测试缓存行为

**性能优化**:

1. 使用 ISR 而不是完全静态生成大量页面
2. 预取关键路径提升导航速度
3. 使用 Suspense 分离缓存边界
4. 定期清理过期缓存
5. 监控缓存大小和命中率

通过深入理解缓存机制,你可以构建出既快速又灵活的 Next.js 应用,在性能和内容新鲜度之间找到最佳平衡点。
