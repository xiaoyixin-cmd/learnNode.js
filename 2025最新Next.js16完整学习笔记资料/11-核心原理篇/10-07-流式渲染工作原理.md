# 流式渲染工作原理

## 1. 架构总览

### 1.1 流式渲染简介

流式渲染(Streaming Rendering)是 Next.js 16 的核心特性之一,它允许服务器逐步发送 HTML 内容,而不是等待整个页面渲染完成。这样可以显著提升首屏加载速度和用户体验。

**核心概念**:

- 渐进式渲染: 边渲染边发送
- Suspense 边界: 定义流式单元
- 选择性水合: 优先水合可见内容
- 并行数据获取: 同时请求多个数据源

### 1.2 工作原理

流式渲染的基本流程:

```
客户端请求 → 服务器开始渲染 → 发送初始 HTML →
继续渲染 → 发送更多 HTML → 完成渲染 → 客户端水合
```

### 1.3 与传统 SSR 的对比

| 特性              | 传统 SSR   | 流式渲染   |
| :---------------- | :--------- | :--------- |
| 首字节时间(TTFB)  | 慢         | 快         |
| 首次内容绘制(FCP) | 慢         | 快         |
| 可交互时间(TTI)   | 慢         | 快         |
| 服务器负载        | 高         | 低         |
| 用户体验          | 等待时间长 | 渐进式加载 |

### 2.1 渲染流程

流式渲染的完整流程包括多个阶段。

**渲染管道**:

```typescript
// 流式渲染管道实现

import { Readable } from "stream";
import { renderToReadableStream } from "react-dom/server";

interface StreamingOptions {
  onShellReady?: () => void;
  onShellError?: (error: Error) => void;
  onAllReady?: () => void;
  onError?: (error: Error) => void;
}

class StreamingRenderer {
  async render(
    element: React.ReactElement,
    options: StreamingOptions = {}
  ): Promise<ReadableStream> {
    const { onShellReady, onShellError, onAllReady, onError } = options;

    // 创建流式渲染流
    const stream = await renderToReadableStream(element, {
      onShellReady() {
        console.log("[Streaming] Shell ready");
        onShellReady?.();
      },
      onShellError(error) {
        console.error("[Streaming] Shell error:", error);
        onShellError?.(error);
      },
      onAllReady() {
        console.log("[Streaming] All content ready");
        onAllReady?.();
      },
      onError(error) {
        console.error("[Streaming] Error:", error);
        onError?.(error);
      },
    });

    return stream;
  }

  async renderToString(element: React.ReactElement): Promise<string> {
    const stream = await this.render(element);
    const reader = stream.getReader();
    const decoder = new TextDecoder();
    let result = "";

    while (true) {
      const { done, value } = await reader.read();

      if (done) {
        break;
      }

      result += decoder.decode(value, { stream: true });
    }

    return result;
  }
}
```

```typescript
interface SuspenseBoundary {
  id: string;
  fallback: React.ReactNode;
  children: React.ReactNode;

class SuspenseManager {

    }
  }

  isPending(id: string): boolean {
    return this.pendingBoundaries.has(id);
  }

  getPendingCount(): number {
    return this.pendingBoundaries.size;
  }
}
```

### 2.3 数据流处理

流式渲染需要处理数据流的发送和接收。

**数据流管理器**:

```typescript
// 数据流管理

class DataStreamManager {
  private encoder = new TextEncoder();
  private decoder = new TextDecoder();

  createDataStream(): TransformStream<string, Uint8Array> {
    return new TransformStream({
      transform: (chunk, controller) => {
        const encoded = this.encoder.encode(chunk);
        controller.enqueue(encoded);
      },
    });
  }

  async sendChunk(
    writer: WritableStreamDefaultWriter,
    data: any
  ): Promise<void> {
    const chunk = JSON.stringify(data);
    const encoded = this.encoder.encode(chunk + "\n");

    await writer.write(encoded);
  }

  async *readChunks(
    reader: ReadableStreamDefaultReader<Uint8Array>
  ): AsyncGenerator<string> {
    let buffer = "";

    while (true) {
      const { done, value } = await reader.read();

      if (done) {
        if (buffer) {
          yield buffer;
        }
        break;
      }

      buffer += this.decoder.decode(value, { stream: true });

      const lines = buffer.split("\n");
      buffer = lines.pop() || "";

      for (const line of lines) {
        if (line) {
          yield line;
        }
      }
    }
  }
}
```

### 3.2 渐进式内容注入

服务器渲染完成的内容需要逐步注入到客户端。

**内容注入器**:

```typescript
// 渐进式内容注入

interface ContentChunk {
  id: string;
  html: string;
  priority: number;
}

class ProgressiveInjector {
  private chunks: ContentChunk[] = [];
  private injected = new Set<string>();

  addChunk(chunk: ContentChunk): void {
    this.chunks.push(chunk);
    this.chunks.sort((a, b) => b.priority - a.priority);
  }

  generateInjectionScript(chunk: ContentChunk): string {
    return `
<script>
  (function() {
    const html = ${JSON.stringify(chunk.html)};
    window.__NEXT_STREAMING__.resolveSuspense('${chunk.id}', html);
  })();
</script>
    `.trim();
  }

  async injectChunks(writer: WritableStreamDefaultWriter): Promise<void> {
    const encoder = new TextEncoder();

    for (const chunk of this.chunks) {
      if (this.injected.has(chunk.id)) {
        continue;
      }

      const script = this.generateInjectionScript(chunk);
      const encoded = encoder.encode(script);

      await writer.write(encoded);
      this.injected.add(chunk.id);

      console.log(`[Injector] Injected chunk ${chunk.id}`);
    }
  }

  hasMore(): boolean {
    return this.chunks.length > this.injected.size;
  }
}
```

### 3.3 并行数据获取

流式渲染支持并行获取多个数据源。

**并行数据获取器**:

```typescript
// 并行数据获取

interface DataSource {
  id: string;
  fetch: () => Promise<any>;
  priority: number;
}

class ParallelDataFetcher {
  private sources: DataSource[] = [];
  private results = new Map<string, any>();
  private errors = new Map<string, Error>();

  addSource(source: DataSource): void {
    this.sources.push(source);
  }

  async fetchAll(): Promise<Map<string, any>> {
    // 按优先级分组
    const groups = this.groupByPriority();

    // 依次执行每个优先级组
    for (const group of groups) {
      await this.fetchGroup(group);
    }

    return this.results;
  }

  private groupByPriority(): DataSource[][] {
    const groups = new Map<number, DataSource[]>();

    for (const source of this.sources) {
      const group = groups.get(source.priority) || [];
      group.push(source);
      groups.set(source.priority, group);
    }

    // 按优先级降序排序
    return Array.from(groups.entries())
      .sort((a, b) => b[0] - a[0])
      .map(([_, sources]) => sources);
  }

  private async fetchGroup(sources: DataSource[]): Promise<void> {
    const promises = sources.map(async (source) => {
      try {
        const startTime = Date.now();
        const data = await source.fetch();
        const duration = Date.now() - startTime;

        this.results.set(source.id, data);
        console.log(`[Fetcher] Fetched ${source.id} in ${duration}ms`);
      } catch (error) {
        this.errors.set(source.id, error as Error);
        console.error(`[Fetcher] Error fetching ${source.id}:`, error);
      }
    });

    await Promise.all(promises);
  }

  getResult(id: string): any {
    return this.results.get(id);
  }

  getError(id: string): Error | undefined {
    return this.errors.get(id);
  }

  hasError(id: string): boolean {
    return this.errors.has(id);
  }
}
```

### 3.4 完整流式渲染示例

将所有组件组合起来实现完整的流式渲染。

**完整实现**:

```typescript
// 完整流式渲染实现

import { renderToReadableStream } from "react-dom/server";

class StreamingSSR {
  private shellRenderer = new ShellRenderer();
  private injector = new ProgressiveInjector();
  private fetcher = new ParallelDataFetcher();

  async render(
    element: React.ReactElement,
    config: ShellConfig
  ): Promise<ReadableStream> {
    // 1. 渲染 Shell
    const shell = this.shellRenderer.renderShell(config);
    const streamingScript = this.shellRenderer.injectStreamingScript();

    // 2. 创建流
    const { readable, writable } = new TransformStream();
    const writer = writable.getWriter();
    const encoder = new TextEncoder();

    // 3. 发送 Shell
    await writer.write(
      encoder.encode(shell.replace("</head>", `${streamingScript}</head>`))
    );

    // 4. 开始流式渲染
    this.startStreaming(element, writer);

    return readable;
  }

  private async startStreaming(
    element: React.ReactElement,
    writer: WritableStreamDefaultWriter
  ): Promise<void> {
    const encoder = new TextEncoder();

    try {
      // 渲染 React 组件
      const stream = await renderToReadableStream(element, {
        onShellReady: () => {
          console.log("[SSR] Shell ready, starting to stream");
        },
        onAllReady: async () => {
          console.log("[SSR] All content ready");

          // 注入所有待处理的内容
          await this.injector.injectChunks(writer);

          // 关闭流
          await writer.close();
        },
        onError: (error) => {
          console.error("[SSR] Streaming error:", error);
        },
      });

      // 将 React 流传输到输出流
      const reader = stream.getReader();

      while (true) {
        const { done, value } = await reader.read();

        if (done) {
          break;
        }

        await writer.write(value);
      }
    } catch (error) {
      console.error("[SSR] Fatal error:", error);

      // 发送错误页面
      const errorHTML = this.renderErrorPage(error as Error);
      await writer.write(encoder.encode(errorHTML));
      await writer.close();
    }
  }

  private renderErrorPage(error: Error): string {
    return `
<div style="padding: 2rem; text-align: center;">
  <h1>渲染错误</h1>
  <p>${error.message}</p>
</div>
    `.trim();
  }
}

// 使用示例
async function handleRequest(req: Request): Promise<Response> {
  const ssr = new StreamingSSR();

  const element = (
    <html>
      <body>
        <Suspense fallback={<div>Loading header...</div>}>
          <Header />
        </Suspense>

        <Suspense fallback={<div>Loading content...</div>}>
          <MainContent />
        </Suspense>

        <Suspense fallback={<div>Loading footer...</div>}>
          <Footer />
        </Suspense>
      </body>
    </html>
  );

  const config: ShellConfig = {
    title: "My App",
    description: "Streaming SSR Demo",
    styles: ["/styles/main.css"],
    scripts: ["/scripts/main.js"],
    preloadLinks: ["/scripts/main.js"],
  };

  const stream = await ssr.render(element, config);

  return new Response(stream, {
    headers: {
      "Content-Type": "text/html; charset=utf-8",
      "Transfer-Encoding": "chunked",
    },
  });
}
```

### 4.1 优先级调度

根据内容的重要性调整渲染优先级。

**优先级调度器**:

```typescript
// 优先级调度实现

enum Priority {
  IMMEDIATE = 1,
  HIGH = 2,
  NORMAL = 3,
  LOW = 4,
  IDLE = 5,
}

interface RenderTask {
  id: string;
  component: React.ReactElement;
  priority: Priority;
  deadline?: number;
}

class PriorityScheduler {
  private tasks: RenderTask[] = [];
  private running = false;

  schedule(task: RenderTask): void {
    this.tasks.push(task);
    this.tasks.sort((a, b) => a.priority - b.priority);

    if (!this.running) {
      this.run();
    }
  }

  private async run(): Promise<void> {
    this.running = true;

    while (this.tasks.length > 0) {
      const task = this.tasks.shift();

      if (!task) {
        break;
      }

      // 检查是否超时
      if (task.deadline && Date.now() > task.deadline) {
        console.warn(`[Scheduler] Task ${task.id} exceeded deadline`);
        continue;
      }

      await this.executeTask(task);
    }

    this.running = false;
  }

  private async executeTask(task: RenderTask): Promise<void> {
    const startTime = Date.now();

    try {
      // 渲染组件
      console.log(
        `[Scheduler] Executing task ${task.id} with priority ${task.priority}`
      );

      // 实际渲染逻辑
      await this.renderComponent(task.component);

      const duration = Date.now() - startTime;
      console.log(`[Scheduler] Task ${task.id} completed in ${duration}ms`);
    } catch (error) {
      console.error(`[Scheduler] Task ${task.id} failed:`, error);
    }
  }

  private async renderComponent(component: React.ReactElement): Promise<void> {
    // 简化的渲染实现
    await new Promise((resolve) => setTimeout(resolve, 10));
  }
}
```

### 4.2 缓存策略

缓存已渲染的内容以提升性能。

**流式缓存管理器**:

```typescript
// 流式缓存

interface CacheEntry {
  html: string;
  timestamp: number;
  ttl: number;
}

class StreamingCache {
  private cache = new Map<string, CacheEntry>();

  set(key: string, html: string, ttl: number = 60000): void {
    this.cache.set(key, {
      html,
      timestamp: Date.now(),
      ttl,
    });
  }

  get(key: string): string | null {
    const entry = this.cache.get(key);

    if (!entry) {
      return null;
    }

    // 检查是否过期
    if (Date.now() - entry.timestamp > entry.ttl) {
      this.cache.delete(key);
      return null;
    }

    return entry.html;
  }

  has(key: string): boolean {
    return this.get(key) !== null;
  }

  clear(): void {
    this.cache.clear();
  }

  cleanup(): void {
    const now = Date.now();

    for (const [key, entry] of this.cache.entries()) {
      if (now - entry.timestamp > entry.ttl) {
        this.cache.delete(key);
      }
    }
  }
}
```

```typescript
interface ErrorBoundaryProps {
  fallback: (error: Error) => React.ReactNode;
  children: React.ReactNode;
}

class StreamingErrorBoundary extends React.Component<ErrorBoundaryProps> {
  state = { hasError: false, error: null };

  static getDerivedStateFromError(error: Error) {
    return { hasError: true, error };
  }

  componentDidCatch(error: Error, errorInfo: React.ErrorInfo) {
    console.error("[ErrorBoundary] Caught error:", error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      return this.props.fallback(this.state.error);
    }

    return this.props.children;
  }
}
```

## 5. 适用场景

### 5.1 最佳使用场景

流式渲染特别适合以下场景:

1. **内容密集型页面**: 博客、新闻网站、文档站点
2. **数据驱动页面**: 仪表板、报表页面
3. **长列表页面**: 商品列表、搜索结果
4. **多数据源页面**: 需要从多个 API 获取数据的页面

### 5.2 性能对比

| 指标 | 传统 SSR | 流式渲染 | 改善幅度 |
| :--- | :------- | :------- | :------- |
| TTFB | 800ms    | 200ms    | 75%      |
| FCP  | 1200ms   | 400ms    | 67%      |
| LCP  | 2000ms   | 800ms    | 60%      |
| TTI  | 3000ms   | 1500ms   | 50%      |

### 5.3 不适用场景

1. **高度交互的页面**: 需要大量客户端状态的页面
2. **实时更新页面**: WebSocket 驱动的页面
3. **简单静态页面**: 内容很少的页面

## 7. 常见问题

**问题一: 流式渲染会影响 SEO 吗?**

不会。搜索引擎爬虫会等待整个页面加载完成后再抓取内容,流式渲染不会影响 SEO。

**问题二: 如何处理流式渲染中的错误?**

使用 Error Boundary 和 Suspense 的 fallback 来处理错误:

```typescript
<Suspense fallback={<Loading />}>
  <ErrorBoundary fallback={<Error />}>
    <AsyncComponent />
  </ErrorBoundary>
</Suspense>
```

**问题三: 流式渲染支持哪些浏览器?**

所有现代浏览器都支持流式渲染。对于不支持的浏览器,Next.js 会自动降级到传统 SSR。

**问题四: 如何测试流式渲染?**

使用 Chrome DevTools 的 Network 面板,查看 Response 的 Timing 标签,可以看到内容是分块传输的。

## 8. 总结

### 1. 流式渲染和传统 SSR 有什么区别?

**主要区别**:

| 特性       | 传统 SSR             | 流式渲染     |
| ---------- | -------------------- | ------------ |
| 渲染方式   | 等待所有内容渲染完成 | 逐步发送内容 |
| 首字节时间 | 较慢                 | 快           |
| 用户体验   | 白屏等待             | 渐进式显示   |
| 数据获取   | 串行                 | 并行         |
| 错误处理   | 全局错误             | 局部错误边界 |

**传统 SSR 流程**:

```typescript
// 传统 SSR
async function renderPage() {
  // 1. 等待所有数据
  const data1 = await fetchData1();
  const data2 = await fetchData2();
  const data3 = await fetchData3();

  // 2. 渲染完整 HTML
  const html = renderToString(
    <App data1={data1} data2={data2} data3={data3} />
  );

  // 3. 一次性发送
  return html;
}
```

**流式渲染流程**:

```typescript
// 流式渲染
async function streamPage() {
  // 1. 立即发送 Shell
  stream.write('<html><body><div id="root">');

  // 2. 并行获取数据并逐步发送
  Promise.all([
    fetchData1().then((data) => stream.write(renderComponent1(data))),
    fetchData2().then((data) => stream.write(renderComponent2(data))),
    fetchData3().then((data) => stream.write(renderComponent3(data))),
  ]);

  // 3. 发送结束标记
  stream.write("</div></body></html>");
}
```

### 3. 流式渲染对 SEO 有影响吗?

**SEO 影响分析**:

流式渲染对 SEO 是友好的,因为:

1. **完整的 HTML**: 搜索引擎爬虫会等待完整的 HTML
2. **更快的响应**: TTFB 更快,有利于排名
3. **结构化数据**: 可以在 Shell 中包含结构化数据

**最佳实践**:

```typescript
// 在 Shell 中包含关键 SEO 信息
export const metadata = {
```

```typescript
import { useQuery, QueryClient, dehydrate } from "@tanstack/react-query";
```

**降级策略**:

```typescript
// Next.js 会自动处理降级
// 不支持流式渲染的浏览器会回退到传统 SSR

// 检测浏览器支持
function supportsStreaming() {
  return typeof ReadableStream !== "undefined";
}

// 条件渲染
export default function Page() {
  if (!supportsStreaming()) {
    return <LegacyPage />;
  }

  return (
    <Suspense fallback={<Loading />}>
      <ModernPage />
    </Suspense>
  );
}
```
