## 题目一: Next.js 16 的核心特性有哪些?

### 问题描述

请列举 Next.js 16 的主要新特性,并简要说明每个特性的作用。

### 参考答案

Next.js 16 的核心特性包括:

1. **App Router**: 基于 React Server Components 的新路由系统

   - 支持服务端组件和客户端组件
   - 嵌套路由和布局
   - 流式渲染和 Suspense

2. **Turbopack**: 新一代构建工具

   - 基于 Rust 开发
   - 比 Webpack 快 700 倍
   - 增量编译和持久化缓存

3. **React Compiler**: 自动优化组件性能

   - 自动记忆化
   - 无需手动使用 useMemo/useCallback
   - 编译时优化

4. **Server Actions**: 服务端函数

   - 直接在组件中调用服务端代码
   - 类型安全
   - 自动处理表单提交

5. **Partial Prerendering**: 部分预渲染

   - 静态和动态内容混合
   - 提升首屏加载速度
   - 保持内容新鲜度

6. **增强的缓存系统**: 多层缓存机制

   - Request Memoization
   - Data Cache
   - Full Route Cache
   - Router Cache

7. **流式渲染**: 渐进式页面加载

   - 更快的 TTFB
   - 更好的用户体验
   - Suspense 边界

8. **元数据 API**: 统一的元数据管理
   - 类型安全
   - 动态生成
   - SEO 优化

## 题目二: App Router 和 Pages Router 有什么区别?

### 问题描述

请对比 Next.js 的 App Router 和 Pages Router,说明它们的主要区别和各自的优缺点。

### 参考答案

**主要区别**:

1. **文件结构**:

   - Pages Router: `pages/` 目录,基于文件系统路由
   - App Router: `app/` 目录,支持嵌套路由和布局

2. **组件类型**:

   - Pages Router: 所有组件都是客户端组件
   - App Router: 默认是服务端组件,可选客户端组件

3. **数据获取**:

   - Pages Router: `getServerSideProps`, `getStaticProps`
   - App Router: 直接在组件中使用 `async/await`

4. **布局系统**:

   - Pages Router: 使用 `_app.tsx` 和 `_document.tsx`
   - App Router: 使用 `layout.tsx` 支持嵌套布局

5. **加载状态**:
   - Pages Router: 需要手动处理
   - App Router: 使用 `loading.tsx` 自动处理

**对比表格**:

| 特性         | Pages Router | App Router  |
| :----------- | :----------- | :---------- |
| 默认组件类型 | 客户端       | 服务端      |
| 数据获取     | 特殊函数     | async/await |
| 布局         | 全局布局     | 嵌套布局    |
| 流式渲染     | 不支持       | 支持        |
| 学习曲线     | 平缓         | 陡峭        |
| 性能         | 一般         | 优秀        |

**优缺点**:

Pages Router 优点:

- 简单易学
- 生态成熟
- 文档完善

Pages Router 缺点:

- 性能较差
- 功能受限
- 不支持新特性

App Router 优点:

- 性能优秀
- 功能强大
- 支持最新特性

App Router 缺点:

- 学习曲线陡峭
- 生态不够成熟
- 某些库不兼容

### 代码示例

```typescript
// Pages Router
// pages/posts/[id].tsx
export async function getServerSideProps({ params }) {
  const post = await fetchPost(params.id);
  return { props: { post } };

export default function Post({ post }) {
  return <div>{post.title}</div>;
}

// App Router
// app/posts/[id]/page.tsx
async function Post({ params }) {
  const post = await fetchPost(params.id);
  return <div>{post.title}</div>;
}

```

## 题目三: 什么是 React Server Components?

### 问题描述

请解释什么是 React Server Components(RSC),它解决了什么问题,以及如何使用。

### 参考答案

**定义**:

React Server Components 是一种新的组件类型,它在服务器上渲染,不会发送到客户端。

**解决的问题**:

1. **Bundle 大小**: 服务端组件不包含在客户端 bundle 中
2. **数据获取**: 可以直接访问后端资源(数据库、文件系统)
3. **安全性**: 敏感逻辑保留在服务器
4. **性能**: 减少客户端 JavaScript 执行

**特点**:

- 默认在服务器渲染
- 可以直接使用 async/await
- 不能使用客户端 API(useState, useEffect 等)
- 不能使用浏览器 API
- 可以导入服务端模块

**使用方式**:

```typescript
// 服务端组件(默认)
async function ServerComponent() {
  const data = await fetchData(); // 直接获取数据
  return <div>{data}</div>;
}

// 客户端组件(需要标记)
("use client");

import { useState } from "react";

function ClientComponent() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
```

**组合使用**:

```typescript
// 服务端组件可以导入客户端组件
import ClientComponent from "./ClientComponent";

async function ServerComponent() {
  const data = await fetchData();

  return (
    <div>
      <h1>{data.title}</h1>
      <ClientComponent initialData={data} />
    </div>
  );
}
```

## 题目四: Next.js 的渲染模式有哪些?

### 问题描述

请列举 Next.js 支持的所有渲染模式,并说明每种模式的特点和适用场景。

### 参考答案

Next.js 支持以下渲染模式:

1. **静态生成(SSG - Static Site Generation)**:

   - 构建时生成 HTML
   - 最快的响应速度
   - 适合内容不常变化的页面

2. **增量静态生成(ISR - Incremental Static Regeneration)**:

   - 静态生成 + 后台更新
   - 平衡性能和新鲜度
   - 适合半动态内容

3. **服务端渲染(SSR - Server-Side Rendering)**:

   - 每次请求时渲染
   - 内容始终最新
   - 适合动态内容

4. **客户端渲染(CSR - Client-Side Rendering)**:

   - 在浏览器中渲染
   - 减少服务器负载
   - 适合用户特定内容

5. **流式渲染(Streaming)**:
   - 渐进式发送 HTML
   - 更快的首字节时间
   - 适合复杂页面

**对比表格**:

| 模式      | 渲染时机      | 性能 | 新鲜度 | 适用场景   |
| :-------- | :------------ | :--- | :----- | :--------- |
| SSG       | 构建时        | 极快 | 低     | 博客、文档 |
| ISR       | 构建时+运行时 | 快   | 中     | 电商、新闻 |
| SSR       | 请求时        | 慢   | 高     | 仪表板     |
| CSR       | 客户端        | 中   | 高     | 个人中心   |
| Streaming | 请求时        | 快   | 高     | 复杂页面   |

**代码示例**:

```typescript
// SSG
export async function generateStaticParams() {
  return [{ id: "1" }, { id: "2" }];
}

async function Page({ params }) {
  const data = await fetchData(params.id);
  return <div>{data}</div>;
}

// ISR
export const revalidate = 60; // 60秒后重新验证

async function Page() {
  const data = await fetchData();
  return <div>{data}</div>;
}

// SSR
export const dynamic = "force-dynamic";

async function Page() {
  const data = await fetchData();
  return <div>{data}</div>;
}

// CSR
("use client");

function Page() {
  const [data, setData] = useState(null);

  useEffect(() => {
    fetchData().then(setData);
  }, []);

  return <div>{data}</div>;
}

// Streaming
import { Suspense } from "react";

function Page() {
  return (
    <Suspense fallback={<Loading />}>
      <AsyncComponent />
    </Suspense>
  );
}
```

## 题目五: Next.js 的缓存机制是如何工作的?

### 问题描述

请详细说明 Next.js 16 的多层缓存机制,包括每一层的作用和生命周期。

### 参考答案

Next.js 16 有四层缓存机制:

**1. Request Memoization(请求记忆化)**:

- 作用: 在单次渲染中缓存相同的请求
- 生命周期: 仅在服务器渲染期间
- 范围: 单个请求
- 自动启用: 是

```typescript
// 同一个请求会被自动缓存
async function Component() {
  const data1 = await fetch("/api/data"); // 发起请求
  const data2 = await fetch("/api/data"); // 使用缓存
  // 只会发起一次实际请求
}
```

**2. Data Cache(数据缓存)**:

- 作用: 跨请求和部署缓存数据
- 生命周期: 持久化,直到重新验证或重新部署
- 范围: 全局
- 自动启用: 是(可选退出)

```typescript
// 默认缓存
const data = await fetch("/api/data");

// 不缓存
const data = await fetch("/api/data", { cache: "no-store" });

// 定时重新验证
const data = await fetch("/api/data", { next: { revalidate: 60 } });
```

**3. Full Route Cache(完整路由缓存)**:

- 作用: 缓存整个路由的渲染结果
- 生命周期: 持久化,直到重新验证或重新部署
- 范围: 静态路由
- 自动启用: 仅静态路由

```typescript
// 静态路由会被缓存
async function Page() {
  return <div>Static content</div>;
}

// 动态路由不会被缓存
export const dynamic = "force-dynamic";

async function Page() {
  return <div>Dynamic content</div>;
}
```

**4. Router Cache(路由器缓存)**:

- 作用: 缓存客户端导航的路由
- 生命周期: 用户会话期间或时间限制
- 范围: 客户端
- 自动启用: 是

```typescript
// 客户端导航会使用缓存
<Link href="/about">About</Link>
```

**缓存层级关系**:

```
请求 → Request Memoization → Data Cache → Full Route Cache → Router Cache → 响应
```

**缓存控制**:

```typescript
// 1. 禁用所有缓存
export const dynamic = "force-dynamic";
export const revalidate = 0;

// 2. 按需重新验证
import { revalidatePath, revalidateTag } from "next/cache";

await revalidatePath("/posts");
await revalidateTag("posts");

// 3. 配置缓存时间
export const revalidate = 3600; // 1小时
```

## 题目六: 什么是 Server Actions?如何使用?

### 问题描述

请解释 Next.js 的 Server Actions 是什么,它解决了什么问题,以及如何在实际项目中使用。

### 参考答案

**定义**:

Server Actions 是可以在客户端组件中直接调用的服务端函数,无需创建 API 路由。

**解决的问题**:

1. **简化数据变更**: 不需要创建 API 端点
2. **类型安全**: 端到端的类型检查
3. **自动处理**: 自动处理表单提交和重新验证
4. **渐进增强**: 即使 JavaScript 未加载也能工作

**基本使用**:

```typescript
// app/actions.ts
"use server";

export async function createPost(formData: FormData) {
  const title = formData.get("title");
  const content = formData.get("content");

  // 直接访问数据库
  await db.post.create({
    data: { title, content },
  });

  // 重新验证缓存
  revalidatePath("/posts");
}

// app/page.tsx
import { createPost } from "./actions";

function Page() {
  return (
    <form action={createPost}>
      <input name="title" />
      <textarea name="content" />
      <button type="submit">Create</button>
    </form>
  );
}
```

**在客户端组件中使用**:

```typescript
"use client";

import { createPost } from "./actions";
import { useFormState, useFormStatus } from "react-dom";

function Form() {
  const [state, formAction] = useFormState(createPost, null);

  return (
    <form action={formAction}>
      <input name="title" />
      <textarea name="content" />
      <SubmitButton />
      {state?.error && <p>{state.error}</p>}
    </form>
  );
}

function SubmitButton() {
  const { pending } = useFormStatus();

  return (
    <button type="submit" disabled={pending}>
      {pending ? "Creating..." : "Create"}
    </button>
  );
}
```

**高级用法**:

```typescript
// 1. 返回数据
"use server";

export async function updatePost(id: string, formData: FormData) {
  const post = await db.post.update({
    where: { id },
    data: {
      title: formData.get("title"),
      content: formData.get("content"),
    },
  });

  return { success: true, post };
}

// 2. 错误处理
("use server");

export async function deletePost(id: string) {
  try {
    await db.post.delete({ where: { id } });
    revalidatePath("/posts");
    return { success: true };
  } catch (error) {
    return { success: false, error: "Failed to delete post" };
  }
}

// 3. 验证
("use server");

import { z } from "zod";

const schema = z.object({
  title: z.string().min(1),
  content: z.string().min(10),
});

export async function createPost(formData: FormData) {
  const data = {
    title: formData.get("title"),
    content: formData.get("content"),
  };

  const result = schema.safeParse(data);

  if (!result.success) {
    return { error: result.error.flatten() };
  }

  await db.post.create({ data: result.data });
  revalidatePath("/posts");

  return { success: true };
}
```

**注意事项**:

1. Server Actions 必须标记 `'use server'`
2. 只能接收可序列化的参数
3. 返回值也必须可序列化
4. 不能在 Server Actions 中使用客户端 API

## 题目七: Next.js 的路由系统是如何工作的?

### 问题描述

请详细说明 Next.js App Router 的路由系统,包括文件约定、动态路由、路由组等概念。

### 参考答案

**文件约定**:

Next.js 使用文件系统路由,特殊文件名有特定含义:

| 文件名          | 作用     | 必需 |
| :-------------- | :------- | :--- |
| `page.tsx`      | 页面组件 | 是   |
| `layout.tsx`    | 布局组件 | 否   |
| `loading.tsx`   | 加载状态 | 否   |
| `error.tsx`     | 错误处理 | 否   |
| `not-found.tsx` | 404 页面 | 否   |
| `route.ts`      | API 路由 | 否   |

**基本路由**:

```
app/
├── page.tsx           → /
├── about/
│   └── page.tsx       → /about
└── blog/
    ├── page.tsx       → /blog
    └── [slug]/
        └── page.tsx   → /blog/:slug
```

**动态路由**:

```typescript
// app/posts/[id]/page.tsx
function Post({ params }: { params: { id: string } }) {
  return <div>Post {params.id}</div>;
}

// app/posts/[...slug]/page.tsx
function Post({ params }: { params: { slug: string[] } }) {
  // /posts/a/b/c → params.slug = ['a', 'b', 'c']
  return <div>Post {params.slug.join("/")}</div>;
}

// app/posts/[[...slug]]/page.tsx
function Post({ params }: { params: { slug?: string[] } }) {
  // /posts → params.slug = undefined
  // /posts/a → params.slug = ['a']
  return <div>Post {params.slug?.join("/") || "index"}</div>;
}
```

**路由组**:

使用 `(folder)` 创建路由组,不影响 URL:

```
app/
├── (marketing)/
│   ├── about/
│   │   └── page.tsx   → /about
│   └── contact/
│       └── page.tsx   → /contact
└── (shop)/
    ├── products/
    │   └── page.tsx   → /products
    └── cart/
        └── page.tsx   → /cart
```

**并行路由**:

使用 `@folder` 创建插槽:

```
app/
├── layout.tsx
├── @team/
│   └── page.tsx
└── @analytics/
    └── page.tsx

// layout.tsx
function Layout({ children, team, analytics }) {
  return (
    <div>
      {children}
      {team}
      {analytics}
    </div>
  );
}
```

**拦截路由**:

使用 `(..)folder` 拦截路由:

```
app/
├── feed/
│   └── page.tsx
└── photo/
    ├── [id]/
    │   └── page.tsx
    └── (..)photo/
        └── [id]/
            └── page.tsx
```

**路由处理器**:

```typescript
// app/api/posts/route.ts
export async function GET(request: Request) {
  const posts = await db.post.findMany();
  return Response.json(posts);
}

export async function POST(request: Request) {
  const body = await request.json();
  const post = await db.post.create({ data: body });
  return Response.json(post);
}
```

## 题目八: 如何在 Next.js 中进行数据获取?

### 问题描述

请说明在 Next.js 16 中获取数据的各种方式,以及每种方式的适用场景。

### 参考答案

**1. 服务端组件中直接获取**:

```typescript
// 最推荐的方式
async function Page() {
  const data = await fetch("https://api.example.com/data");
  const json = await data.json();

  return <div>{json.title}</div>;
}
```

**2. 使用 fetch 配置缓存**:

```typescript
// 缓存数据
const data = await fetch("https://api.example.com/data", {
  cache: "force-cache", // 默认
});

// 不缓存
const data = await fetch("https://api.example.com/data", {
  cache: "no-store",
});

// 定时重新验证
const data = await fetch("https://api.example.com/data", {
  next: { revalidate: 60 },
});
```

**3. 并行数据获取**:

```typescript
async function Page() {
  // 并行获取
  const [posts, users] = await Promise.all([
    fetch("/api/posts").then((r) => r.json()),
    fetch("/api/users").then((r) => r.json()),
  ]);

  return (
    <div>
      <Posts data={posts} />
      <Users data={users} />
    </div>
  );
}
```

**4. 顺序数据获取**:

```typescript
async function Page() {

  const posts = await fetch(`/api/posts?userId=${user.id}`).then((r) =>

}
```

**5. 使用 Suspense 流式获取**:


**7. 使用 SWR 或 React Query**:

```typescript
"use client";

import useSWR from "swr";

function Page() {
  const { data, error, isLoading } = useSWR("/api/data", fetcher);

  if (isLoading) return <div>Loading...</div>;
  if (error) return <div>Error</div>;

  return <div>{data.title}</div>;
}
```

**数据获取模式对比**:

| 方式             | 位置   | 缓存   | 适用场景     |
| :--------------- | :----- | :----- | :----------- |
| 服务端直接获取   | 服务端 | 是     | 静态内容     |
| fetch with cache | 服务端 | 可配置 | 半动态内容   |
| Suspense         | 服务端 | 是     | 复杂页面     |
| useEffect        | 客户端 | 否     | 用户特定数据 |
| SWR/React Query  | 客户端 | 是     | 实时数据     |

### 考察点
