# 路由系统面试题(10 题)

## 题目一: App Router 的文件约定有哪些?

### 问题描述

请详细说明 Next.js App Router 中的特殊文件名及其作用。

### 参考答案

App Router 使用特殊文件名来定义路由的不同部分:

**核心文件**:

| 文件名          | 作用                 | 是否必需 | 渲染时机   |
| :-------------- | :------------------- | :------- | :--------- |
| `page.tsx`      | 定义页面 UI          | 是       | 每次访问   |
| `layout.tsx`    | 定义共享布局         | 否       | 每次访问   |
| `template.tsx`  | 定义可重新挂载的布局 | 否       | 每次导航   |
| `loading.tsx`   | 定义加载 UI          | 否       | 加载时     |
| `error.tsx`     | 定义错误 UI          | 否       | 错误时     |
| `not-found.tsx` | 定义 404 页面        | 否       | 404 时     |
| `route.ts`      | 定义 API 端点        | 否       | API 调用时 |

**详细说明**:

1. **page.tsx**:

```typescript
// app/dashboard/page.tsx
export default function DashboardPage() {
  return <div>Dashboard</div>;
}
```

2. **layout.tsx**:

```typescript
// app/dashboard/layout.tsx
export default function DashboardLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <div>
      <nav>Dashboard Nav</nav>
      {children}
    </div>
  );
}
```

3. **template.tsx**:

```typescript
// app/dashboard/template.tsx
export default function DashboardTemplate({
  children,
}: {
  children: React.ReactNode;
}) {
  return <div className="dashboard-wrapper">{children}</div>;
}
```

4. **loading.tsx**:

```typescript
// app/dashboard/loading.tsx
export default function DashboardLoading() {
  return <div>Loading dashboard...</div>;
}
```

5. **error.tsx**:

```typescript
// app/dashboard/error.tsx
"use client";

export default function DashboardError({
  error,
  reset,
}: {
  error: Error;
  reset: () => void;
}) {
  return (
    <div>
      <h2>Something went wrong!</h2>
      <button onClick={reset}>Try again</button>
    </div>
  );
}
```

6. **not-found.tsx**:

```typescript
// app/dashboard/not-found.tsx
export default function DashboardNotFound() {
  return <div>Dashboard page not found</div>;
}
```

7. **route.ts**:

```typescript
// app/api/users/route.ts
export async function GET(request: Request) {
  const users = await fetchUsers();
  return Response.json(users);
}

export async function POST(request: Request) {
  const body = await request.json();
  const user = await createUser(body);
  return Response.json(user);
}
```

**layout vs template 的区别**:

| 特性     | layout     | template     |
| :------- | :--------- | :----------- |
| 状态保持 | 是         | 否           |
| 重新挂载 | 否         | 是           |
| 适用场景 | 持久化导航 | 需要重置状态 |

**文件优先级**:

```
layout.tsx → template.tsx → error.tsx → loading.tsx → not-found.tsx → page.tsx
```

### 考察点

- 对文件约定的理解
- 能否正确使用各种特殊文件
- 是否了解文件之间的关系

## 题目二: 动态路由如何实现?

### 问题描述

请说明 Next.js 中动态路由的实现方式,包括单个参数、多个参数和可选参数。

### 参考答案

**1. 单个动态参数**:

```typescript
// app/posts/[id]/page.tsx
interface PageProps {
  params: { id: string };
  searchParams: { [key: string]: string | string[] | undefined };
}

export default function PostPage({ params, searchParams }: PageProps) {
  return <div>Post ID: {params.id}</div>;
}

// 访问 /posts/123 → params.id = "123"
```

**2. 多个动态参数**:

```typescript
// app/blog/[category]/[slug]/page.tsx
interface PageProps {
  params: { category: string; slug: string };
}

export default function BlogPost({ params }: PageProps) {
  return (
    <div>
      Category: {params.category}
      <br />
      Slug: {params.slug}
    </div>
  );
}

// 访问 /blog/tech/nextjs → params = { category: "tech", slug: "nextjs" }
```

**3. 捕获所有路由**:

```typescript
// app/docs/[...slug]/page.tsx
interface PageProps {
  params: { slug: string[] };
}

export default function DocsPage({ params }: PageProps) {
  return <div>Path: {params.slug.join("/")}</div>;
}

// 访问 /docs/a/b/c → params.slug = ["a", "b", "c"]
```

**4. 可选捕获所有路由**:

```typescript
// app/shop/[[...slug]]/page.tsx
interface PageProps {
  params: { slug?: string[] };
}

export default function ShopPage({ params }: PageProps) {
  if (!params.slug) {
    return <div>Shop Home</div>;
  }

  return <div>Path: {params.slug.join("/")}</div>;
}

// 访问 /shop → params.slug = undefined
// 访问 /shop/electronics → params.slug = ["electronics"]
// 访问 /shop/electronics/phones → params.slug = ["electronics", "phones"]
```

**5. 生成静态参数**:

```typescript
// app/posts/[id]/page.tsx
export async function generateStaticParams() {
  const posts = await fetchPosts();

  return posts.map((post) => ({
    id: post.id.toString(),
  }));
}

export default function PostPage({ params }: { params: { id: string } }) {
  return <div>Post {params.id}</div>;
}
```

**6. 动态参数验证**:

```typescript
// app/posts/[id]/page.tsx
import { notFound } from "next/navigation";

export default async function PostPage({ params }: { params: { id: string } }) {
  const post = await fetchPost(params.id);

  if (!post) {
    notFound();
  }

  return <div>{post.title}</div>;
}
```

**路由模式对比**:

| 路由模式 | 文件路径                       | 匹配 URL      | params                                          |
| :------- | :----------------------------- | :------------ | :---------------------------------------------- |
| 静态     | `app/about/page.tsx`           | `/about`      | -                                               |
| 动态     | `app/posts/[id]/page.tsx`      | `/posts/123`  | `{ id: "123" }`                                 |
| 嵌套动态 | `app/[category]/[id]/page.tsx` | `/tech/123`   | `{ category: "tech", id: "123" }`               |
| 捕获所有 | `app/docs/[...slug]/page.tsx`  | `/docs/a/b`   | `{ slug: ["a", "b"] }`                          |
| 可选捕获 | `app/[[...slug]]/page.tsx`     | `/` 或 `/a/b` | `{ slug: undefined }` 或 `{ slug: ["a", "b"] }` |

### 考察点

- 对动态路由的理解
- 能否处理复杂的路由场景
- 是否了解参数类型和验证

## 题目三: 路由组(Route Groups)的作用是什么?

### 问题描述

请解释路由组的概念、使用场景和实现方式。

### 参考答案

**定义**:

路由组使用 `(folderName)` 语法创建,可以组织路由而不影响 URL 路径。

**主要作用**:

1. **组织路由结构**
2. **创建多个根布局**
3. **分组相关路由**
4. **不影响 URL 路径**

**基本使用**:

```
app/
├── (marketing)/
│   ├── about/
│   │   └── page.tsx      → /about
│   ├── contact/
│   │   └── page.tsx      → /contact
│   └── layout.tsx        → 营销页面布局
├── (shop)/
│   ├── products/
│   │   └── page.tsx      → /products
│   ├── cart/
│   │   └── page.tsx      → /cart
│   └── layout.tsx        → 商店布局
└── (dashboard)/
    ├── analytics/
    │   └── page.tsx      → /analytics
    ├── settings/
    │   └── page.tsx      → /settings
    └── layout.tsx        → 仪表板布局
```

**示例一: 多个根布局**:

```typescript

```

**示例二: 按功能分组**:

```
app/
├── (auth)/
│   ├── login/

│   ├── register/
│   │   └── page.tsx      → /register

    ├── dashboard/
    │   └── page.tsx      → /dashboard
    └── layout.tsx        → 应用布局
```

| 场景     | 说明                 | 示例                 |
| :------- | :------------------- | :------------------- |

| 功能模块 | 按功能组织路由       | 认证、仪表板、设置   |

## 题目四: 并行路由(Parallel Routes)如何使用?

### 问题描述

请解释并行路由的概念、使用场景和实现方式。

### 参考答案

**定义**:

并行路由允许在同一个布局中同时渲染多个页面,使用 `@folder` 语法创建命名插槽。

**基本结构**:

```
app/
├── layout.tsx
├── page.tsx
├── @team/
│   └── page.tsx
├── @analytics/
│   └── page.tsx
└── @notifications/
    └── page.tsx
```

**布局实现**:

```typescript
// app/layout.tsx
export default function Layout({
  children,
  team,
  analytics,
  notifications,
}: {
  children: React.ReactNode;
  team: React.ReactNode;
  analytics: React.ReactNode;
  notifications: React.ReactNode;
}) {
  return (
    <div>
      <div>{children}</div>
      <div className="grid grid-cols-2">
        <div>{team}</div>
        <div>{analytics}</div>
      </div>
      <div>{notifications}</div>
    </div>
  );

```

## 题目五: 拦截路由(Intercepting Routes)的作用是什么?

### 问题描述

请解释拦截路由的概念、使用场景和实现方式。

### 参考答案

**定义**:

拦截路由允许在当前布局中加载其他路由的内容,常用于模态框等场景。

**语法规则**:

| 约定       | 说明       |
| :--------- | :--------- |
| `(.)`      | 匹配同级   |
| `(..)`     | 匹配上一级 |
| `(..)(..)` | 匹配上两级 |
| `(...)`    | 匹配根目录 |

**基本示例**:

```
app/
├── feed/
│   └── page.tsx
└── photo/
    ├── [id]/
    │   └── page.tsx
    └── (..)photo/
        └── [id]/
            └── page.tsx
```

**实现模态框**:

```typescript
// app/photo/(..)photo/[id]/page.tsx

    </Modal>
  );
}

// app/photo/[id]/page.tsx
import PhotoDetail from "@/components/PhotoDetail";

export default function PhotoPage({ params }: { params: { id: string } }) {
  return (
    <div className="page">
      <PhotoDetail id={params.id} />
    </div>
  );

```

**Modal 组件**:

```typescript
// components/Modal.tsx
"use client";

import { useRouter } from "next/navigation";
import { useEffect, useRef } from "react";

export default function Modal({ children }: { children: React.ReactNode }) {
  const router = useRouter();
  const dialogRef = useRef<HTMLDialogElement>(null);

  useEffect(() => {
    dialogRef.current?.showModal();
  }, []);

  const closeModal = () => {
    dialogRef.current?.close();
    router.back();
  };

  return (
    <dialog ref={dialogRef} onClose={closeModal}>
      <button onClick={closeModal}>Close</button>
      {children}
    </dialog>
  );
}
```

## 题目六: 路由处理器(Route Handlers)如何使用?

### 问题描述

请说明如何创建和使用路由处理器,包括各种 HTTP 方法的实现。

### 参考答案

**基本定义**:

路由处理器允许使用 Web Request 和 Response API 创建自定义请求处理器。

**文件位置**:

```
app/api/route.ts
```

**支持的 HTTP 方法**:

```typescript
// app/api/users/route.ts
export async function GET(request: Request) {
  const users = await db.user.findMany();
  return Response.json(users);
}

export async function POST(request: Request) {
  const body = await request.json();
  const user = await db.user.create({ data: body });
  return Response.json(user, { status: 201 });
}

export async function PUT(request: Request) {
  const body = await request.json();
  const user = await db.user.update({
    where: { id: body.id },
    data: body,
  });
  return Response.json(user);
}

export async function PATCH(request: Request) {
  const body = await request.json();
  const user = await db.user.update({
    where: { id: body.id },
    data: body,
  });
  return Response.json(user);
}

export async function DELETE(request: Request) {
  const { searchParams } = new URL(request.url);
  const id = searchParams.get("id");

  await db.user.delete({ where: { id } });
  return new Response(null, { status: 204 });
}

export async function HEAD(request: Request) {
  return new Response(null, { status: 200 });
}

export async function OPTIONS(request: Request) {
  return new Response(null, {
    status: 200,
    headers: {
      Allow: "GET, POST, PUT, PATCH, DELETE, HEAD, OPTIONS",
    },
  });
}
```

**动态路由处理器**:

```typescript
// app/api/users/[id]/route.ts
export async function GET(
  request: Request,
  { params }: { params: { id: string } }
) {
  const user = await db.user.findUnique({
    where: { id: params.id },
  });

  if (!user) {
    return Response.json({ error: "User not found" }, { status: 404 });
  }

  return Response.json(user);
}

export async function DELETE(
  request: Request,
  { params }: { params: { id: string } }
) {
  await db.user.delete({
    where: { id: params.id },
  });

  return new Response(null, { status: 204 });
}
```

**请求对象**:

```typescript
export async function GET(request: Request) {
  // URL 参数
  const { searchParams } = new URL(request.url);
  const query = searchParams.get("query");

  // Headers
  const authorization = request.headers.get("authorization");

  // Cookies
  const { cookies } = await import("next/headers");
  const token = cookies().get("token");

  return Response.json({ query, authorization, token });
}

export async function POST(request: Request) {
  // JSON body
  const body = await request.json();

  // FormData
  const formData = await request.formData();
  const name = formData.get("name");

  // Text
  const text = await request.text();

  return Response.json({ body, name, text });
```

```typescript
// middleware.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

const rateLimit = new Map<string, { count: number; resetTime: number }>();

export function middleware(request: NextRequest) {
  const ip = request.ip || "unknown";
  const now = Date.now();
  const limit = 100; // 每分钟100次
  const window = 60 * 1000; // 1分钟

  const record = rateLimit.get(ip);

  if (!record || now > record.resetTime) {
    rateLimit.set(ip, { count: 1, resetTime: now + window });
    return NextResponse.next();
  }

  if (record.count >= limit) {
    return new NextResponse("Too Many Requests", { status: 429 });
  }

  record.count++;
  return NextResponse.next();
}
```

## 题目八: 如何实现路由保护和权限控制?

**方法一: 使用中间件**:

```typescript
// middleware.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";
import { verifyToken } from "@/lib/auth";

export async function middleware(request: NextRequest) {
  const token = request.cookies.get("token")?.value;
  const { pathname } = request.nextUrl;

  // 公开路由
  const publicPaths = ["/login", "/register", "/"];
  if (publicPaths.includes(pathname)) {
    return NextResponse.next();
  }

  // 验证 token
  if (!token) {
    return NextResponse.redirect(new URL("/login", request.url));
  }

  try {
    const user = await verifyToken(token);

    // 管理员路由
    if (pathname.startsWith("/admin") && !user.isAdmin) {
      return NextResponse.redirect(new URL("/unauthorized", request.url));
    }

    // 添加用户信息到请求头
    const requestHeaders = new Headers(request.headers);
    requestHeaders.set("x-user-id", user.id);

    return NextResponse.next({
      request: {
        headers: requestHeaders,
      },
    });
  } catch (error) {
    return NextResponse.redirect(new URL("/login", request.url));
  }
}

export const config = {
  matcher: ["/((?!api|_next/static|_next/image|favicon.ico).*)"],
};
```

**问题八: 重定向循环**

原因: 中间件或布局中的重定向逻辑错误

解决方案:

```typescript
// middleware.ts
export function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl;

  // 避免重定向到当前路径
  if (pathname === "/login") {
    return NextResponse.next();
  }

  const token = request.cookies.get("token");

  if (!token) {
    // 只在非登录页重定向
    return NextResponse.redirect(new URL("/login", request.url));
  }

  return NextResponse.next();
}
```

```typescript
export async function GET(request: Request) {
```
