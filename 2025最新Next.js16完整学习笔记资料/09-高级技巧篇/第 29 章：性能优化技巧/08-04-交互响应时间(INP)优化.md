# 交互响应时间(INP)优化

## 核心概念

交互响应时间(Interaction to Next Paint, INP)是衡量页面响应用户交互速度的指标,记录从用户交互到下一次绘制的延迟时间。INP 替代了 FID(First Input Delay),更全面地评估页面的交互响应性。

### INP 评分标准

| 评分     | INP 时间  | 用户体验            |
| -------- | --------- | ------------------- |
| 优秀     | 0-200ms   | 响应迅速,交互流畅   |
| 需要改进 | 200-500ms | 有延迟感,但可接受   |
| 差       | >500ms    | 响应缓慢,用户体验差 |

### INP 测量的交互类型

1. **点击**: 鼠标点击、触摸点击
2. **按键**: 键盘输入
3. **拖拽**: 拖放操作

### INP 的三个阶段

1. **输入延迟**: 从用户交互到事件处理开始
2. **处理时间**: 事件处理器执行时间
3. **呈现延迟**: 从处理完成到下一次绘制

```typescript
// hooks/useDebounce.ts
import { useEffect, useState } from "react";

export function useDebounce<T>(value: T, delay: number): T {
  const [debouncedValue, setDebouncedValue] = useState<T>(value);

  useEffect(() => {
    const timer = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => clearTimeout(timer);
  }, [value, delay]);

  return debouncedValue;
}
```

```typescript
export function VirtualList({
  items,
  itemHeight,
  containerHeight,
}: VirtualListProps) {
```

```typescript
      {todos.map((todo) => (
        <div key={todo.id} data-todo-id={todo.id}>
          <span>{todo.text}</span>
```

```typescript
import { useEffect, useRef } from "react";

export function SmoothScroll({ children }: { children: React.ReactNode }) {
  const containerRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    let ticking = false;

    const handleScroll = () => {
      if (!ticking) {
        requestAnimationFrame(() => {
          // 处理滚动逻辑
          ticking = false;
        });
        ticking = true;
      }
    };

    const container = containerRef.current;
    container?.addEventListener("scroll", handleScroll, { passive: true });

    return () => container?.removeEventListener("scroll", handleScroll);
  }, []);

  return (
    <div ref={containerRef} style={{ overflowY: "auto", height: "100vh" }}>
      {children}
    </div>
  );
}
```

```typescript
import Image from "next/image";

export function LazyImage({ src, alt }: { src: string; alt: string }) {
  const [isVisible, setIsVisible] = useState(false);
  const imgRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          setIsVisible(true);
          observer.disconnect();
        }
      },
      { rootMargin: "50px" }

      observer.observe(imgRef.current);
    }

    return () => observer.disconnect();
  }, []);

    <div ref={imgRef}>
      {isVisible && <Image src={src} alt={alt} width={800} height={600} />}
    </div>
  );
```

```typescript
"use client";

import { useState, useRef } from "react";

export function DraggableItem() {
  const [position, setPosition] = useState({ x: 0, y: 0 });
  const [isDragging, setIsDragging] = useState(false);
  const dragRef = useRef({ startX: 0, startY: 0 });

  const handleMouseDown = (e: React.MouseEvent) => {
    setIsDragging(true);
    dragRef.current = {
      startX: e.clientX - position.x,
      startY: e.clientY - position.y,
    };

  const handleMouseMove = (e: React.MouseEvent) => {
    if (!isDragging) return;

    requestAnimationFrame(() => {
      setPosition({
        x: e.clientX - dragRef.current.startX,
        y: e.clientY - dragRef.current.startY,
      });
    });
  };

  return (
    <div
      onMouseDown={handleMouseDown}
      onMouseMove={handleMouseMove}
      onMouseUp={() => setIsDragging(false)}
      style={{
        transform: `translate(${position.x}px, ${position.y}px)`,
        cursor: isDragging ? "grabbing" : "grab",
```

## 常见问题

### 1. 什么是 INP?

INP 衡量页面响应用户交互的速度,200ms 以下为优秀。

### 2. INP 和 FID 有什么区别?

FID 只测量首次交互,INP 测量所有交互。

### 3. 如何测试 INP?

使用 Chrome DevTools、Lighthouse、Web Vitals 库。

### 4. 为什么我的 INP 很高?

常见原因:长任务阻塞主线程、事件处理器执行时间过长、大量 DOM 操作。

### 5. Web Workers 适用于什么场景?

适用于耗时计算、数据处理等不需要访问 DOM 的任务。

### 6. 防抖和节流如何选择?

防抖适用于输入,节流适用于滚动和 resize。

### 7. 虚拟滚动如何实现?

只渲染可见区域的元素,根据滚动位置动态更新。

### 8. React.memo 什么时候使用?

当组件 props 不变时不需要重渲染。

### 9. 事件委托有什么好处?

减少事件监听器数量,提高性能。

### 10. CSS containment 如何使用?

使用 contain 属性限制布局、样式和绘制的影响范围。

使用 transform 和 opacity。

## 总结

INP 优化的核心是减少主线程阻塞,快速响应用户交互。

关键要点:

1. 使用防抖和节流优化频繁交互
2. 使用 Web Workers 处理耗时计算
3. 代码分割减少主线程负担
4. 虚拟滚动优化长列表
5. 使用 requestIdleCallback 处理非关键任务
6. React.memo 避免不必要的重渲染
7. 事件委托减少监听器数量
8. CSS containment 限制布局范围
9. 优化动画使用 transform 和 opacity
10. 持续监控真实用户 INP 数据

记住,INP 优化要从用户交互的角度出发,确保每次交互都能快速响应。

### 额外建议

1. **优先优化高频交互**: 关注用户最常使用的交互,如搜索、滚动、点击
2. **测试真实设备**: 在低端设备上测试,确保所有用户都能获得良好体验
3. **监控长任务**: 使用 Performance Observer 监控超过 50ms 的长任务
4. **分解大任务**: 将长任务分解为多个小任务,避免阻塞主线程
5. **使用性能预算**: 为 INP 设置性能预算,确保不超过 200ms
6. **持续优化**: INP 优化是持续的过程,需要定期检查和改进
7. **用户反馈**: 收集用户反馈,了解实际使用中的性能问题
8. **A/B 测试**: 对优化方案进行 A/B 测试,验证效果
9. **文档记录**: 记录优化方案和效果,方便团队学习和复用
10. **团队培训**: 培训团队成员,提高整体性能意识
