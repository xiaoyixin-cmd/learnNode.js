# 08-08-第三方脚本优化

第三方脚本是现代 Web 应用的常见组成部分,但如果处理不当会严重影响性能。本文介绍如何在 Next.js 中优化第三方脚本加载。

## 核心概念

### 第三方脚本类型

| 类型     | 示例             | 影响 | 优化策略       |
| -------- | ---------------- | ---- | -------------- |
| 分析脚本 | Google Analytics | 中   | 延迟加载       |
| 广告脚本 | Google Ads       | 高   | 按需加载       |
| 社交媒体 | Facebook SDK     | 中   | 懒加载         |
| 聊天工具 | Intercom         | 高   | 用户交互后加载 |
| 地图服务 | Google Maps      | 高   | 按需加载       |
| 支付网关 | Stripe           | 中   | 页面级加载     |

### Next.js Script 组件策略

| 策略              | 说明              | 适用场景   |
| ----------------- | ----------------- | ---------- |
| beforeInteractive | 页面交互前加载    | 关键脚本   |
| afterInteractive  | 页面交互后加载    | 分析脚本   |
| lazyOnload        | 空闲时加载        | 非关键脚本 |
| worker            | Web Worker 中加载 | 计算密集型 |

import Script from "next/script";

export function LazyScript() {
  const [loadScript, setLoadScript] = useState(false);

  useEffect(() => {
    const timer = setTimeout(() => {
      setLoadScript(true);
    }, 3000);

    return () => clearTimeout(timer);

### 实战场景十六: 脚本加载状态管理

```typescript
// hooks/useScriptLoader.ts
import { useEffect, useState } from "react";

export function useScriptLoader(src: string) {
  const [loaded, setLoaded] = useState(false);
  const [error, setError] = useState(false);

  useEffect(() => {
    const script = document.createElement("script");
    script.src = src;
    script.async = true;

    script.onload = () => setLoaded(true);
    script.onerror = () => setError(true);

    document.body.appendChild(script);

    return () => {
      document.body.removeChild(script);
    };
  }, [src]);

  return { loaded, error };
}
```

## 注意事项

### 1. 选择合适的加载策略

根据脚本的重要性选择合适的 strategy。

### 2. 避免阻塞渲染

使用 afterInteractive 或 lazyOnload 避免阻塞首屏渲染。

### 3. 监控脚本性能

使用 Performance API 监控第三方脚本的加载时间。

### 4. 处理加载错误

添加 onError 回调处理脚本加载失败的情况。

### 5. 使用 CDN 加速

对第三方脚本使用 preconnect 和 dns-prefetch。

## 常见问题

### 1. 如何优化第三方脚本加载?

使用 Next.js Script 组件,选择合适的 strategy,延迟加载非关键脚本。

### 2. beforeInteractive 和 afterInteractive 的区别?

beforeInteractive 在页面交互前加载,afterInteractive 在页面交互后加载。

### 3. 如何延迟加载分析脚本?

使用 strategy="afterInteractive"或"lazyOnload"。

### 4. 如何按需加载第三方脚本?

使用状态控制,在用户交互时才加载脚本。

### 5. 如何处理脚本加载失败?

使用 onError 回调,提供 fallback 方案。

### 6. 如何监控脚本性能?

使用 Performance API 或 Chrome DevTools。

### 7. 如何优化 Google Analytics?

使用 afterInteractive 策略,延迟加载。

### 8. 如何优化广告脚本?

使用 lazyOnload 策略,延迟 2-3 秒加载。

### 9. 如何优化聊天工具?

在用户滚动到一定位置后再加载。

### 10. 如何优化地图脚本?

在用户点击显示地图时再加载。

### 11. Script 组件有什么优势?

自动优化,支持多种加载策略,提供加载状态回调。

### 12. 如何预连接第三方域名?

使用 link rel="preconnect"。

### 13. 如何测试脚本性能?

使用 Lighthouse,WebPageTest,Performance API。

### 14. 如何避免脚本冲突?

使用命名空间,避免全局变量污染。

### 15. 如何优化多个第三方脚本?

按优先级加载,使用不同的 strategy。

### 16. 如何处理脚本依赖?

使用 onLoad 回调确保依赖顺序。

### 17. 如何优化脚本缓存?

设置合适的 Cache-Control 头。

### 18. 如何减少脚本大小?

使用压缩版本,按需加载功能。

### 19. 如何优化脚本执行?

使用 Web Worker,避免阻塞主线程。

### 20. 如何调试脚本问题?

使用 Chrome DevTools,检查 Network 和 Console。

### 自动化优化

使用脚本自动检测第三方脚本:

```javascript
// scripts/analyzeScripts.js
const puppeteer = require("puppeteer");

async function analyzeScripts(url) {
  const browser = await puppeteer.launch();
  const page = await browser.newPage();

  const scripts = [];

  page.on("response", async (response) => {
    if (response.request().resourceType() === "script") {
      scripts.push({
        url: response.url(),
        size: (await response.buffer()).length,
        status: response.status(),
      });
    }
  });

  await page.goto(url);
  await browser.close();

  console.log("Scripts:", scripts);
  return scripts;
}

analyzeScripts("https://example.com");
```

// lib/monitorScripts.ts
export function monitorScripts() {
  const observer = new PerformanceObserver((list) => {
    for (const entry of list.getEntries()) {
      if (entry.entryType === "resource" && entry.initiatorType === "script") {
        console.log("Script performance:", {
          name: entry.name,
          duration: entry.duration,
          size: entry.transferSize,
          cached: entry.transferSize === 0,
          blocking: entry.renderBlockingStatus,
        });
      }
    }
  });

  observer.observe({ entryTypes: ["resource"] });
}
