# 08-05-代码分割与懒加载策略

代码分割和懒加载是优化 Next.js 应用性能的关键技术。通过将代码拆分成更小的块,只在需要时加载,可以显著减少初始加载时间,提升用户体验。

## 核心概念

### 代码分割类型

| 分割类型   | 实现方式  | 适用场景 |
| ---------- | --------- | -------- |
| 路由级分割 | 自动      | 所有页面 |
| 组件级分割 | dynamic() | 大型组件 |
| 库级分割   | import()  | 第三方库 |
| 条件分割   | 动态导入  | 条件渲染 |

### 懒加载策略

1. **路由懒加载**: Next.js 自动为每个页面创建独立的 bundle
2. **组件懒加载**: 使用 dynamic()延迟加载组件
3. **库懒加载**: 按需加载第三方库
4. **图片懒加载**: 使用 Image 组件自动懒加载
5. **脚本懒加载**: 使用 Script 组件控制加载时机

### 实战场景十五: 分块加载大型数据

```typescript
// app/data/page.tsx
"use client";

import { useState, useEffect } from "react";

export default function DataPage() {
  const [chunks, setChunks] = useState<any[]>([]);

  useEffect(() => {
    loadDataInChunks();
  }, []);

  const loadDataInChunks = async () => {
    for (let i = 0; i < 10; i++) {
      const chunk = await import(`@/data/chunk-${i}`);
      setChunks((prev) => [...prev, chunk.default]);
      await new Promise((resolve) => setTimeout(resolve, 100));
    }
  };

  return (
    <div>
      {chunks.map((chunk, index) => (
        <div key={index}>{JSON.stringify(chunk)}</div>
      ))}
    </div>
  );
}
```

## 注意事项

### 1. 避免过度分割

过度分割会增加 HTTP 请求数量,反而降低性能。

### 2. 合理使用 loading 状态

为懒加载组件提供 loading 状态,提升用户体验。

### 3. 预加载关键资源

对关键路由和组件进行预加载。

### 4. 监控 bundle 大小

使用 webpack-bundle-analyzer 监控 bundle 大小。

### 5. 测试懒加载效果

在真实网络环境下测试懒加载效果。

## 常见问题

### 1. 什么是代码分割?

将代码拆分成更小的块,按需加载。

### 2. Next.js 如何实现代码分割?

Next.js 自动为每个页面创建独立的 bundle。

### 3. dynamic()和 import()有什么区别?

dynamic()是 Next.js 提供的封装,支持 SSR 和 loading 状态。

### 4. 如何懒加载第三方库?

使用动态 import()按需加载。

### 5. 如何预加载组件?

使用 import()或 router.prefetch()。

### 6. 懒加载会影响 SEO 吗?

客户端懒加载不影响 SEO,服务端渲染的内容会被索引。

### 7. 如何监控 bundle 大小?

使用 webpack-bundle-analyzer 或 Next.js 内置的分析工具。

### 8. 懒加载组件如何处理错误?

使用 Error Boundary 捕获错误。

### 9. 如何优化懒加载性能?

合理分割,预加载关键资源,使用 loading 状态。

### 10. 懒加载适用于所有组件吗?

不适用,首屏关键组件不应懒加载。

### 11. 如何测试懒加载?

使用 Chrome DevTools Network 面板查看加载情况。

### 12. 懒加载会增加复杂度吗?

会增加一定复杂度,但性能提升值得。

### 13. 如何处理懒加载失败?

使用 Error Boundary 和重试机制。

### 14. 懒加载和 Suspense 如何配合?

Suspense 可以为懒加载组件提供 fallback。

### 15. 如何优化懒加载体验?

提供 loading 状态,预加载关键资源,优化网络请求。

### 16. 如何懒加载 CSS?

使用动态 import()加载 CSS 模块。

### 17. 如何懒加载字体?

使用 font-display: swap 和动态加载。

### 18. 如何懒加载图片?

使用 Next.js Image 组件的 loading="lazy"。

### 19. 如何懒加载脚本?

使用 Script 组件的 strategy="lazyOnload"。

### 20. 如何懒加载音频?

使用动态 import()和 audio 元素。

### 21. 如何优化懒加载顺序?

根据用户行为和优先级调整加载顺序。

### 22. 如何处理懒加载超时?

设置超时时间,提供重试机制。

### 23. 如何懒加载 WebAssembly?

使用动态 import()加载 wasm 模块。

### 24. 如何懒加载 Service Worker?

在页面加载完成后注册 Service Worker。

### 25. 如何监控懒加载性能?

使用 Performance API 和 RUM 工具。

### 错误处理

使用 Error Boundary 处理懒加载错误:

```typescript
// components/ErrorBoundary.tsx
"use client";

import { Component, ReactNode } from "react";

interface Props {
  children: ReactNode;
  fallback?: ReactNode;
}

interface State {
  hasError: boolean;
}

export class ErrorBoundary extends Component<Props, State> {
  constructor(props: Props) {
    super(props);
    this.state = { hasError: false };
  }

  static getDerivedStateFromError() {
    return { hasError: true };
  }

  render() {
    if (this.state.hasError) {
      return this.props.fallback || <div>加载失败</div>;
    }

    return this.props.children;
  }
}
```

### 重试机制

实现懒加载重试机制:

```typescript
// utils/retryImport.ts
export function retryImport<T>(
  importFn: () => Promise<T>,
  retries = 3,
  delay = 1000
): Promise<T> {
  return importFn().catch((error) => {
    if (retries === 0) throw error;

    return new Promise((resolve) => {
      setTimeout(() => {
        resolve(retryImport(importFn, retries - 1, delay));
      }, delay);
    });
  });
}
```

使用重试机制:

```typescript

```

### 预加载策略

```typescript
// hooks/usePreload.ts
import { useEffect } from "react";

export function usePreload(importFn: () => Promise<any>, delay = 2000) {
  useEffect(() => {
    const timer = setTimeout(() => {
      importFn();
    }, delay);

    return () => clearTimeout(timer);
  }, [importFn, delay]);
}
```

### 网络感知加载

根据网络状况调整加载策略:

```typescript
// hooks/useNetworkStatus.ts
import { useState, useEffect } from "react";

export function useNetworkStatus() {
  const [effectiveType, setEffectiveType] = useState("4g");

  useEffect(() => {
    const connection = (navigator as any).connection;
    if (connection) {
      setEffectiveType(connection.effectiveType);

      const handleChange = () => {
        setEffectiveType(connection.effectiveType);
      };

      connection.addEventListener("change", handleChange);
      return () => connection.removeEventListener("change", handleChange);
    }
  }, []);

  return effectiveType;
}
```

使用网络感知加载:

```typescript

import dynamic from "next/dynamic";

const HeavyComponent = dynamic(() => import("@/components/HeavyComponent"));
const LightComponent = dynamic(() => import("@/components/LightComponent"));

export function AdaptiveLoader() {
  const networkType = useNetworkStatus();

  if (networkType === "4g" || networkType === "wifi") {
    return <HeavyComponent />;
  }

  return <LightComponent />;
}
```
