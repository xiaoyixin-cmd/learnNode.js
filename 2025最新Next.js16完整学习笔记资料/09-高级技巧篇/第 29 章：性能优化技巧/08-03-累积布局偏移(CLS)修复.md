# 累积布局偏移(CLS)修复

## 核心概念

累积布局偏移(Cumulative Layout Shift, CLS)衡量页面视觉稳定性,记录页面生命周期内所有意外布局偏移的累积分数。CLS 是用户体验的关键指标,高 CLS 会导致用户误点击和阅读困难。

### CLS 评分标准

| 评分     | CLS 分数 | 用户体验            |
| -------- | -------- | ------------------- |
| 优秀     | 0-0.1    | 视觉稳定,无明显偏移 |
| 需要改进 | 0.1-0.25 | 有轻微偏移,影响体验 |
| 差       | >0.25    | 严重偏移,用户体验差 |

### 布局偏移的常见原因

1. **无尺寸的图片和视频**: 未设置宽高导致加载后撑开布局
2. **动态注入内容**: 广告、嵌入内容突然出现
3. **Web 字体加载**: FOIT/FOUT 导致文本重排
4. **动态添加 DOM**: JavaScript 动态插入元素
5. **CSS 动画**: 不当使用 transform 以外的属性

### CLS 计算公式

```
CLS = 影响分数 × 距离分数
```

- 影响分数: 元素移动前后占据视口的比例
- 距离分数: 元素移动的距离占视口的比例

// components/Modal.tsx
"use client";

import { useEffect } from "react";

interface ModalProps {
  isOpen: boolean;
  onClose: () => void;
  children: React.ReactNode;
}

export function Modal({ isOpen, onClose, children }: ModalProps) {
  useEffect(() => {
    if (isOpen) {
      const scrollbarWidth =
        window.innerWidth - document.documentElement.clientWidth;
      document.body.style.overflow = "hidden";
      document.body.style.paddingRight = `${scrollbarWidth}px`;
    } else {
      document.body.style.overflow = "";
      document.body.style.paddingRight = "";
    }

    return () => {
      document.body.style.overflow = "";
      document.body.style.paddingRight = "";
    };
  }, [isOpen]);

  if (!isOpen) return null;

  return (
    <div
      className="modal-overlay"
      style={{
        position: "fixed",
        top: 0,
        left: 0,
        right: 0,
        bottom: 0,
        backgroundColor: "rgba(0, 0, 0, 0.5)",
        display: "flex",
        alignItems: "center",
        justifyContent: "center",
        zIndex: 1000,
      }}
      onClick={onClose}
    >
      <div className="modal-content" onClick={(e) => e.stopPropagation()}>
        {children}
      </div>
    </div>
  );
}

#### 使用 Intersection Observer 预留空间

```typescript
// components/LazySection.tsx
"use client";

import { useEffect, useRef, useState } from "react";

export function LazySection() {
  const [isVisible, setIsVisible] = useState(false);
  const sectionRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          setIsVisible(true);
          observer.disconnect();
        }
      },
      { rootMargin: "100px" }
    );

    if (sectionRef.current) {
      observer.observe(sectionRef.current);
    }

    return () => observer.disconnect();
  }, []);

  return (
    <div
      ref={sectionRef}
      style={{
        minHeight: "400px",
      }}
    >
      {isVisible ? (
        <div>实际内容</div>
      ) : (
        <div className="placeholder">加载中...</div>
      )}
    </div>
  );
}
```

  children,
  text,
}: {
  children: React.ReactNode;
  text: string;
}) {
  const [isVisible, setIsVisible] = useState(false);
  const [position, setPosition] = useState({ x: 0, y: 0 });

        {children}

        <div

            position: "fixed",
            left: position.x,
            top: position.y,

            fontSize: "14px",

## 注意事项

### 1. 为所有图片设置尺寸

这是最基本也是最重要的 CLS 优化。

```typescript
// ✅ 正确
<Image src="/product.jpg" width={800} height={600} alt="Product" />

// ❌ 错误
<img src="/product.jpg" alt="Product" />
```

### 2. 避免在现有内容上方插入内容

除非是响应用户交互,否则不要在现有内容上方插入新内容。

```typescript
// ✅ 正确:在底部添加
<div>
  {existingContent}
  {newContent}
</div>

// ❌ 错误:在顶部插入
<div>
  {newContent}
  {existingContent}
</div>
```

### 3. 使用 transform 和 opacity 做动画

这两个属性不会触发布局重排。

```typescript
// ✅ 正确
style={{ transform: 'translateY(10px)', opacity: 0.5 }}

// ❌ 错误
style={{ top: '10px', display: 'none' }}
```

### 4. 预留动态内容空间

为可能出现的内容预留足够空间。

```typescript
<div style={{ minHeight: "200px" }}>{dynamicContent}</div>
```

### 5. 监控真实用户 CLS 数据

```typescript
import { onCLS } from "web-vitals";

onCLS((metric) => {
  console.log("CLS:", metric.value);
  // 发送到分析服务
});
```

## 常见问题

### 1. 什么是 CLS?

CLS 衡量页面视觉稳定性,记录页面生命周期内所有意外布局偏移的累积分数。分数越低越好,0.1 以下为优秀。

### 2. 为什么我的 CLS 分数很高?

常见原因:

- 图片未设置尺寸
- 动态注入广告或内容
- Web 字体加载导致文本重排
- 动画使用了会触发布局的 CSS 属性

### 3. 如何测试 CLS?

使用工具:

- Chrome DevTools Lighthouse
- Web Vitals 扩展
- PageSpeed Insights
- web-vitals 库

### 4. 响应式图片如何避免 CLS?

使用 aspect-ratio 或设置明确的宽高比。

```typescript
<div style={{ aspectRatio: "16/9" }}>
  <Image src="/image.jpg" fill alt="Image" />
</div>
```

### 5. 广告如何避免 CLS?

为广告位预留固定空间。

```typescript
<div style={{ minHeight: "250px" }}>
  <AdComponent />
</div>
```

### 6. 字体加载如何避免 CLS?

使用 font-display: swap 和 adjustFontFallback。

```typescript
const font = Inter({
  display: "swap",
  adjustFontFallback: true,
});
```

### 7. 动画会导致 CLS 吗?

只有改变布局的动画会导致 CLS。使用 transform 和 opacity 不会。

### 8. 如何修复 iframe 导致的 CLS?

为 iframe 设置固定的宽高比。

```typescript
<div style={{ paddingBottom: "56.25%", position: "relative" }}>
  <iframe style={{ position: "absolute", width: "100%", height: "100%" }} />
</div>
```

### 9. 骨架屏能避免 CLS 吗?

可以,前提是骨架屏的尺寸与实际内容完全匹配。

### 10. 模态框会导致 CLS 吗?

如果不处理滚动条消失,会导致 CLS。需要添加 padding 补偿。

### 11. 懒加载图片会导致 CLS 吗?

如果未设置尺寸,会导致 CLS。设置明确尺寸即可避免。

### 12. 如何监控生产环境 CLS?

使用 RUM 工具收集真实用户数据。

```typescript
import { onCLS } from "web-vitals";

onCLS((metric) => {
  fetch("/api/analytics", {
    method: "POST",
    body: JSON.stringify({ cls: metric.value }),
  });
});
```

### 13. CLS 和用户交互有关吗?

用户交互后 500ms 内的布局偏移不计入 CLS。

### 14. 如何调试 CLS 问题?

使用 Chrome DevTools 的 Performance 面板,查看 Layout Shift 事件。

### 15. 移动端 CLS 如何优化?

移动端优化重点与桌面端相同,但要特别注意响应式图片和字体加载。

## 总结

CLS 优化的核心是保持页面视觉稳定性。通过为所有元素设置明确尺寸,预留动态内容空间,使用合适的动画属性,可以显著改善 CLS 指标。

关键要点:

1. 为所有图片和视频设置明确尺寸
2. 为动态内容预留空间
3. 使用 transform 和 opacity 做动画
4. 优化字体加载策略
5. 为广告位预留固定空间
6. 使用骨架屏匹配实际内容尺寸
7. 为 iframe 设置固定宽高比
8. 处理模态框滚动条问题
9. 避免在现有内容上方插入新内容
10. 持续监控真实用户 CLS 数据

记住,CLS 优化要从设计阶段就开始考虑,确保所有动态内容都有预留空间。
