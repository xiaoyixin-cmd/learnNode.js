# SQL 注入防护

SQL 注入是数据库应用最严重的安全威胁之一。本文介绍如何在 Next.js 应用中有效防护 SQL 注入攻击,保护数据库安全。

## 核心概念

### SQL 注入

攻击者通过输入恶意 SQL 代码,操纵数据库查询,窃取或破坏数据。

### 参数化查询

使用占位符代替直接拼接 SQL 语句,防止注入攻击。

### ORM

对象关系映射工具,自动处理 SQL 查询,减少注入风险。

### 输入验证

验证和清理用户输入,拒绝包含恶意代码的数据。

```typescript
// lib/queryBuilder.ts
import { Prisma } from "@prisma/client";

interface SearchParams {
  name?: string;
  email?: string;
  minAge?: number;
  maxAge?: number;
}

export function buildUserSearchQuery(
  params: SearchParams
): Prisma.UserWhereInput {
  const where: Prisma.UserWhereInput = {};

  if (params.name) {
    where.name = { contains: params.name, mode: "insensitive" };
  }

  if (params.email) {
    where.email = { contains: params.email, mode: "insensitive" };
  }

  if (params.minAge !== undefined || params.maxAge !== undefined) {
    where.age = {};
    if (params.minAge !== undefined) {
      where.age.gte = params.minAge;
    }
    if (params.maxAge !== undefined) {
      where.age.lte = params.maxAge;
    }
  }

  return where;
}

// app/api/users/advanced-search/route.ts
```

### 2.2 预编译语句

```typescript
// lib/db.ts
import { Pool } from "pg";

const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
});

export class PreparedStatements {
  private static statements = new Map<string, string>();

  static async prepare(name: string, sql: string) {
    this.statements.set(name, sql);
  }

  static async execute(name: string, params: any[]) {
    const sql = this.statements.get(name);
    if (!sql) {
      throw new Error(`Prepared statement ${name} not found`);
    }

    const client = await pool.connect();
    try {
      const result = await client.query(sql, params);
      return result.rows;
    } finally {
      client.release();
    }
  }
}

// 初始化预编译语句
PreparedStatements.prepare(
  "getUserByEmail",
  "SELECT id, name, email FROM users WHERE email = $1"
);

PreparedStatements.prepare(
  "searchUsers",
  "SELECT id, name, email FROM users WHERE name ILIKE $1"
);
```

```typescript
// lib/sql-whitelist.ts
const ALLOWED_COLUMNS = ["id", "name", "email", "createdAt"];
const ALLOWED_TABLES = ["users", "posts", "comments"];
const ALLOWED_ORDER = ["ASC", "DESC"];

export function validateColumn(column: string): boolean {
  return ALLOWED_COLUMNS.includes(column);
}

export function validateTable(table: string): boolean {
  return ALLOWED_TABLES.includes(table);
}

export function validateOrder(order: string): boolean {
  return ALLOWED_ORDER.includes(order.toUpperCase());
}

// app/api/users/route.ts
import { NextRequest, NextResponse } from "next/server";
import { prisma } from "@/lib/prisma";
import { validateColumn, validateOrder } from "@/lib/sql-whitelist";

export async function GET(request: NextRequest) {
  const { searchParams } = new URL(request.url);
  const sortBy = searchParams.get("sortBy") || "id";
  const order = searchParams.get("order") || "ASC";

  // 验证排序字段
  if (!validateColumn(sortBy)) {
    return NextResponse.json({ error: "无效的排序字段" }, { status: 400 });
  }

  // 验证排序方向
  if (!validateOrder(order)) {

  }

    orderBy: {

    },
    select: {
      id: true,
      name: true,
      email: true,
      createdAt: true,
    },
  });

}
```

```typescript
  // 转义LIKE查询中的特殊字符
```

## 常见问题

### 1. Prisma 是否完全防止 SQL 注入?

Prisma 使用参数化查询,可以防止大多数 SQL 注入攻击。但在使用原始查询时仍需小心:

```typescript
// ❌ 危险:直接拼接
const users =
  await prisma.$queryRaw`SELECT * FROM users WHERE name = '${name}'`;

// ✅ 安全:使用参数
const users = await prisma.$queryRaw`SELECT * FROM users WHERE name = ${name}`;
```

**安全性对比**:

| 方法          | 安全性 | 灵活性 | 性能 |
| ------------- | ------ | ------ | ---- |
| Prisma Client | 最高   | 高     | 高   |
| 参数化查询    | 高     | 中     | 高   |
| 存储过程      | 高     | 低     | 最高 |
| 原始 SQL      | 最低   | 最高   | 中   |

```typescript
// app/api/posts/route.ts
import { NextRequest, NextResponse } from "next/server";
import { prisma } from "@/lib/prisma";
import { z } from "zod";

const postSchema = z.object({
  title: z.string().min(1).max(200),
  content: z.string().min(1),
  tags: z.array(z.string()).max(10),
  published: z.boolean().default(false),
});

export async function POST(request: NextRequest) {
  const body = await request.json();

  // 验证输入
  const result = postSchema.safeParse(body);
  if (!result.success) {
    return NextResponse.json({ error: result.error }, { status: 400 });
  }

  // 创建文章和标签
  const post = await prisma.post.create({
    data: {
      title: result.data.title,
      content: result.data.content,
      published: result.data.published,
      tags: {
        connectOrCreate: result.data.tags.map((tag) => ({
          where: { name: tag },
          create: { name: tag },
        })),
      },
    },
    include: {
      tags: true,
    },
  });

  return NextResponse.json(post);
}
```

### 4. 数据分析查询

```typescript
// app/api/analytics/route.ts
import { NextRequest, NextResponse } from "next/server";
import { prisma } from "@/lib/prisma";
import { z } from "zod";

const analyticsSchema = z.object({
  startDate: z.string().datetime(),
  endDate: z.string().datetime(),
  metric: z.enum(["views", "clicks", "conversions"]),
  groupBy: z.enum(["day", "week", "month"]).default("day"),
});

export async function GET(request: NextRequest) {
  const { searchParams } = new URL(request.url);

  const params = {
    startDate: searchParams.get("startDate") || "",
    endDate: searchParams.get("endDate") || "",
    metric: searchParams.get("metric") || "views",
    groupBy: searchParams.get("groupBy") || "day",
  };

  // 验证参数
  const result = analyticsSchema.safeParse(params);
  if (!result.success) {
    return NextResponse.json({ error: result.error }, { status: 400 });
  }

  // 使用Prisma的聚合查询
  const analytics = await prisma.analytics.groupBy({
    by: ["date"],
    where: {
      date: {
        gte: new Date(result.data.startDate),
        lte: new Date(result.data.endDate),
      },
    },
    _sum: {
      [result.data.metric]: true,
    },
  });

  return NextResponse.json(analytics);
}
```
