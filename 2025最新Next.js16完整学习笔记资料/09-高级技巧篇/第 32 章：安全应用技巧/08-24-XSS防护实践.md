# XSS 防护实践

跨站脚本攻击(XSS)是 Web 应用最常见的安全威胁之一。本文介绍如何在 Next.js 应用中有效防护 XSS 攻击,保护用户数据安全。

## 核心概念

### XSS 攻击

攻击者在网页中注入恶意脚本,窃取用户信息或执行恶意操作。

### 存储型 XSS

恶意脚本存储在服务器数据库中,每次访问都会执行。

### 反射型 XSS

恶意脚本通过 URL 参数等方式传递,立即执行。

### DOM 型 XSS

恶意脚本通过修改 DOM 结构执行,不经过服务器。

```typescript
interface Props {
  href: string;
  children: React.ReactNode;
}

export default function SafeLink({ href, children }: Props) {
  // 验证URL
  const isValidUrl = (url: string) => {
    try {
      const parsed = new URL(url, window.location.origin);
      // 只允许http和https协议
      return ["http:", "https:"].includes(parsed.protocol);
    } catch {
      return false;
    }
  };

  // 如果URL无效,返回普通文本
  if (!isValidUrl(href)) {
    return <span>{children}</span>;
  }

  return (
    <a href={href} rel="noopener noreferrer" target="_blank">
      {children}
    </a>
  );
}
```

## 3. 内容安全策略(CSP)

### 3.1 严格的 CSP 配置

```typescript
// middleware.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function middleware(request: NextRequest) {
  const nonce = Buffer.from(crypto.randomUUID()).toString("base64");

  const cspHeader = `
    default-src 'self';
    script-src 'self' 'nonce-${nonce}' 'strict-dynamic';
    style-src 'self' 'nonce-${nonce}';
    img-src 'self' blob: data: https:;
    font-src 'self';
    object-src 'none';
    base-uri 'self';
    form-action 'self';
    frame-ancestors 'none';
    upgrade-insecure-requests;
  `
    .replace(/\s{2,}/g, " ")
    .trim();

  const response = NextResponse.next();
  response.headers.set("Content-Security-Policy", cspHeader);
  response.headers.set("X-Content-Type-Options", "nosniff");
  response.headers.set("X-Frame-Options", "DENY");
  response.headers.set("X-XSS-Protection", "1; mode=block");
  response.headers.set("Referrer-Policy", "strict-origin-when-cross-origin");

  return response;
}
```

```typescript
// components/SafeHTML.tsx
import DOMPurify from "isomorphic-dompurify";

interface Props {
  html: string;
  allowedTags?: string[];
}

export default function SafeHTML({ html, allowedTags }: Props) {
  const defaultTags = ["p", "br", "strong", "em", "a"];

  const sanitized = DOMPurify.sanitize(html, {
    ALLOWED_TAGS: allowedTags || defaultTags,
    ALLOWED_ATTR: ["href", "title"],
    ALLOW_DATA_ATTR: false,

}
```

```typescript
export function safeSetInnerHTML(element: HTMLElement, html: string) {
  // 创建临时元素
  const temp = document.createElement("div");
  temp.textContent = html;

  // 设置为文本内容,不解析HTML
  element.textContent = temp.textContent;
}

export function safeSetAttribute(
  element: HTMLElement,
  attr: string,
  value: string
) {
  // 验证属性名
  const allowedAttrs = ["class", "id", "title", "alt"];

  if (!allowedAttrs.includes(attr)) {
    console.warn(`Attribute ${attr} is not allowed`);
    return;
  }

  // 验证属性值
  if (attr === "href" || attr === "src") {
    try {
      const url = new URL(value, window.location.origin);
      if (!["http:", "https:"].includes(url.protocol)) {
        console.warn(`Protocol ${url.protocol} is not allowed`);
        return;
      }
    } catch {
      console.warn(`Invalid URL: ${value}`);
      return;
    }
  }

  element.setAttribute(attr, value);
}
```

### 4. 如何验证文件上传?

```typescript
// app/api/upload/route.ts
import { NextRequest, NextResponse } from "next/server";
import { writeFile } from "fs/promises";
import path from "path";

const ALLOWED_TYPES = ["image/jpeg", "image/png", "image/gif"];
const MAX_SIZE = 5 * 1024 * 1024; // 5MB

export async function POST(request: NextRequest) {
  const formData = await request.formData();
  const file = formData.get("file") as File;

  if (!file) {
    return NextResponse.json({ error: "没有文件" }, { status: 400 });
  }

  // 验证文件类型
  if (!ALLOWED_TYPES.includes(file.type)) {
    return NextResponse.json({ error: "不支持的文件类型" }, { status: 400 });
  }

  // 验证文件大小
  if (file.size > MAX_SIZE) {
    return NextResponse.json({ error: "文件太大" }, { status: 400 });
  }

  // 生成安全的文件名
  const ext = path.extname(file.name);
  const filename = `${Date.now()}-${Math.random()
    .toString(36)
    .substring(7)}${ext}`;

  // 保存文件
  const bytes = await file.arrayBuffer();
  const buffer = Buffer.from(bytes);

  await writeFile(path.join(process.cwd(), "public/uploads", filename), buffer);

  return NextResponse.json({ filename });
}
```

### 5. 如何防护 JSON 注入?

```typescript
// app/api/data/route.ts
import { NextRequest, NextResponse } from "next/server";

export async function POST(request: NextRequest) {
  try {
    const data = await request.json();

    // 验证JSON结构
    if (typeof data !== "object" || Array.isArray(data)) {
      return NextResponse.json({ error: "无效的数据格式" }, { status: 400 });
    }

    // 验证必需字段
    const requiredFields = ["name", "email"];
    for (const field of requiredFields) {
      if (!(field in data)) {
        return NextResponse.json(
          { error: `缺少字段: ${field}` },
          { status: 400 }
        );
      }
    }

    // 移除未知字段
    const allowedFields = ["name", "email", "age"];
    const sanitized = Object.keys(data)
      .filter((key) => allowedFields.includes(key))
      .reduce((obj, key) => {
        obj[key] = data[key];
        return obj;
      }, {} as Record<string, any>);

    return NextResponse.json({ success: true, data: sanitized });
  } catch (error) {
    return NextResponse.json({ error: "无效的JSON" }, { status: 400 });
  }
}
```

## 注意事项

### 1. 输入验证

- 在客户端和服务端都进行验证
- 使用白名单而非黑名单
- 限制输入长度和格式
- 拒绝包含恶意代码的输入

### 2. 输出转义

- 使用 React 的自动转义
- 谨慎使用 dangerouslySetInnerHTML
- 使用 DOMPurify 清理 HTML
- 验证 URL 和属性值

### 3. CSP 配置

- 使用严格的 CSP 策略
- 避免使用 unsafe-inline 和 unsafe-eval
- 使用 nonce 或 hash
- 配置 CSP 报告

### 4. 第三方内容

- 验证第三方脚本来源
- 使用 SRI (Subresource Integrity)
- 隔离第三方内容
- 获取用户同意

### 5. Cookie 安全

- 设置 HttpOnly 标志
- 设置 Secure 标志
- 设置 SameSite 属性
- 使用短期有效期

### 6. 错误处理

- 不要泄露敏感信息
- 记录可疑活动
- 实施速率限制
- 提供友好的错误消息

### 7. 定期审计

- 定期检查依赖更新
- 进行安全审计
- 测试 XSS 防护
- 监控安全事件

## 总结

XSS 防护是 Web 应用安全的基础。本文介绍了:

1. **输入验证**: 使用 Zod 验证用户输入
2. **输出转义**: React 自动转义和 DOMPurify 清理
3. **CSP 配置**: 严格的内容安全策略
4. **实战场景**: 评论系统、富文本编辑器、用户资料、搜索功能

关键要点:

- 永远不要信任用户输入
- 在客户端和服务端都进行验证
- 使用 DOMPurify 清理 HTML 内容
- 配置严格的 CSP 策略
- 避免使用 dangerouslySetInnerHTML
- 验证 URL 和属性值
- 使用 HttpOnly Cookie
- 定期更新依赖和审计安全

通过多层防护策略,可以有效防止 XSS 攻击,保护用户数据和应用安全。
