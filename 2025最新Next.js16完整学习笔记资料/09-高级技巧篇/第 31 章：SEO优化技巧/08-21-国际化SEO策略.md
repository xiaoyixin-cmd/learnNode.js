# 国际化 SEO 策略

国际化 SEO 是针对不同语言和地区优化网站的过程。Next.js 16 提供了强大的国际化支持,让多语言 SEO 变得简单高效。

## 核心概念

### hreflang 标签

告诉搜索引擎页面的语言和地区版本。

### 语言代码

ISO 639-1 标准的两字母语言代码,如 zh、en、ja。

### 地区代码

ISO 3166-1 Alpha 2 标准的两字母国家代码,如 CN、US、JP。

### 内容本地化

根据目标市场调整内容、格式和文化元素。

## 2. 高级国际化配置

### 2.1 自动语言检测

```typescript
// middleware.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

const locales = ["zh-CN", "en-US", "ja-JP"];
const defaultLocale = "zh-CN";

function getLocale(request: NextRequest): string {
  // 1. 检查URL路径
  const pathname = request.nextUrl.pathname;
  const pathnameLocale = locales.find(
    (locale) => pathname.startsWith(`/${locale}/`) || pathname === `/${locale}`
  );

  if (pathnameLocale) return pathnameLocale;

  // 2. 检查Cookie
  const cookieLocale = request.cookies.get("NEXT_LOCALE")?.value;
  if (cookieLocale && locales.includes(cookieLocale)) {
    return cookieLocale;
  }

  // 3. 检查Accept-Language头
  const acceptLanguage = request.headers.get("accept-language");
  if (acceptLanguage) {
    const preferredLocale = acceptLanguage
      .split(",")
      .map((lang) => lang.split(";")[0].trim())
      .find((lang) => locales.includes(lang));

    if (preferredLocale) return preferredLocale;
  }

  return defaultLocale;
}

export function middleware(request: NextRequest) {
  const pathname = request.nextUrl.pathname;

  // 检查路径是否已包含语言代码
  const pathnameHasLocale = locales.some(
    (locale) => pathname.startsWith(`/${locale}/`) || pathname === `/${locale}`
  );

  if (pathnameHasLocale) return;

  // 重定向到检测到的语言
  const locale = getLocale(request);
  request.nextUrl.pathname = `/${locale}${pathname}`;

  const response = NextResponse.redirect(request.nextUrl);
  response.cookies.set("NEXT_LOCALE", locale);

  return response;
}

export const config = {
  matcher: ["/((?!api|_next/static|_next/image|favicon.ico).*)"],
};
```

## 常见问题

### 1. 如何选择 URL 结构?

有三种主要的多语言 URL 结构:

| 结构类型 | 示例               | 优点              | 缺点              |
| -------- | ------------------ | ----------------- | ----------------- |
| 子目录   | example.com/zh-CN/ | 易于管理,单一域名 | URL 较长          |
| 子域名   | zh.example.com     | 独立管理,灵活     | 需要多个 SSL 证书 |
| 顶级域名 | example.cn         | 最佳本地化        | 成本高,管理复杂   |

推荐使用子目录结构,因为它在 SEO 和管理上都有优势。

export function middleware(request: NextRequest) {
  // 检查Accept-Language头
  const acceptLanguage = request.headers.get("accept-language");

  if (acceptLanguage) {
    // 解析首选语言
    const preferredLang = acceptLanguage.split(",")[0].split(";")[0];

    const langMap: Record<string, string> = {

    return NextResponse.redirect(new URL(`/${lang}`, request.url));
  }

  return NextResponse.next();
}

## 注意事项

### 1. URL 结构

- 使用子目录结构(/zh-CN/, /en-US/)
- 保持 URL 简洁和语义化
- 避免使用查询参数(?lang=zh)
- 确保 URL 在所有语言版本中一致

### 2. hreflang 标签

- 为每个页面添加所有语言版本的 hreflang
- 包含 x-default 标签
- 确保 hreflang 值与 HTML lang 属性一致
- 使用完整的绝对 URL

### 3. 内容本地化

- 不仅翻译文字,还要本地化格式
- 使用 Intl API 处理日期、数字、货币
- 考虑文化差异和习惯
- 本地化图片和媒体内容

### 4. 性能优化

- 使用 CDN 加速不同地区的访问
- 实施适当的缓存策略
- 预加载关键资源
- 优化字体加载

### 5. SEO 最佳实践

- 为每种语言创建独立的 sitemap
- 使用正确的语言代码(ISO 639-1)
- 避免自动重定向
- 提供语言切换器

### 6. 用户体验

- 记住用户的语言偏好
- 提供明显的语言切换选项
- 保持导航在切换语言时的一致性
- 处理不存在的翻译内容

### 7. 技术实现

- 使用 Next.js 的国际化路由
- 实施服务端渲染
- 使用中间件处理语言检测
- 管理翻译文件
