# 08-12-生产构建优化策略

生产构建优化直接影响应用性能和用户体验。本文介绍 Next.js 生产构建的优化策略。

## 核心概念

### 构建优化维度

| 维度     | 目标         | 影响     | 优先级 |
| -------- | ------------ | -------- | ------ |
| 包体积   | 减小文件大小 | 加载速度 | 高     |
| 编译速度 | 加快构建     | 开发效率 | 中     |
| 运行性能 | 优化执行     | 用户体验 | 高     |
| 缓存策略 | 提升复用     | 加载速度 | 高     |

### 实战场景二: 配置代码分割

```javascript
// next.config.js
module.exports = {
  webpack: (config, { isServer }) => {
    if (!isServer) {
      config.optimization.splitChunks = {
        chunks: "all",
        cacheGroups: {
          default: false,
          vendors: false,
          commons: {
            name: "commons",
            chunks: "all",
            minChunks: 2,
          },
          lib: {
            test: /[\\/]node_modules[\\/]/,
            name(module) {
              const packageName = module.context.match(
                /[\\/]node_modules[\\/](.*?)([\\/]|$)/
              )[1];
              return `npm.${packageName.replace("@", "")}`;
            },
          },
        },
      };
    }
    return config;
  },
};
```

## 注意事项

### 1. 平衡优化与可维护性

不要过度优化导致代码难以维护。

### 2. 测试优化效果

使用工具测量优化前后的性能差异。

### 3. 监控构建时间

优化不应显著增加构建时间。

### 4. 考虑浏览器兼容性

某些优化可能影响旧浏览器。

### 5. 定期审查配置

随着项目发展调整优化策略。

## 常见问题

### 1. 如何减小包体积?

使用 Tree Shaking、代码分割、压缩优化。

### 2. 构建时间过长怎么办?

启用缓存、并行编译、减少优化级别。

### 3. 如何优化首屏加载?

使用代码分割、预渲染、资源预加载。

### 4. Source Map 如何配置?

生产环境建议禁用或使用 hidden-source-map。

### 5. 如何优化第三方库?

使用 optimizePackageImports、动态导入。

### 6. CSS 文件过大怎么办?

启用 CSS 优化、使用 CSS Modules、移除未使用样式。

### 7. 如何优化图片?

使用 next/image、WebP/AVIF 格式、CDN。

### 8. 字体加载慢如何解决?

使用 next/font、字体子集化、预加载。

### 9. 如何分析构建产物?

使用@next/bundle-analyzer。

### 10. 缓存策略如何配置?

根据资源类型设置不同的缓存时间。

### 11. 如何优化服务端渲染?

使用流式渲染、Suspense、缓存。

### 12. 动态导入何时使用?

大型组件、条件加载、路由级分割。

### 13. 如何处理 Polyfill?

按需加载、使用 browserslist 配置。

### 14. 环境变量如何优化?

只暴露必要的变量、使用构建时替换。

### 15. 如何优化 API 路由?

使用边缘函数、缓存响应、压缩数据。

### 16. 静态导出如何优化?

使用 output: 'export'、优化静态资源。

### 17. 如何优化 Monorepo 构建?

使用 Turborepo、共享缓存、并行构建。

### 18. Docker 构建如何优化?

使用多阶段构建、缓存层、精简镜像。

### 19. CI/CD 如何优化?

使用缓存、并行任务、增量构建。

### 20. 如何监控构建性能?

记录构建时间、分析产物大小、性能指标。

```javascript
// next.config.js
```

```javascript
  webpack: (config) => {
    config.optimization = {
      ...config.optimization,
      splitChunks: {
        chunks: "all",
        cacheGroups: {
          default: false,
          vendors: false,
          // 框架代码单独打包
          framework: {
            name: "framework",
            chunks: "all",
            test: /[\\/]node_modules[\\/](react|react-dom|scheduler)[\\/]/,
            priority: 40,
            enforce: true,
          },
          // 公共库单独打包
          lib: {
            test: /[\\/]node_modules[\\/]/,
            name(module) {
              const packageName = module.context.match(
                /[\\/]node_modules[\\/](.*?)([\\/]|$)/
              )[1];
              return `npm.${packageName.replace("@", "")}`;
            },
            priority: 30,
          },
        },
      },
    };
    return config;
  },
};
```
