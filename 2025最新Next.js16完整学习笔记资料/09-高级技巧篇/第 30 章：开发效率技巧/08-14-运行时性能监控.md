# 运行时性能监控

运行时性能监控是确保应用在生产环境中保持良好性能的关键手段。通过实时监控各项性能指标,可以及时发现和解决性能问题,提升用户体验。

## 核心概念

### Performance API

浏览器提供的性能监控接口,可以获取各种性能指标。

### Web Vitals

Google 提出的核心性能指标,包括 LCP、FID、CLS 等。

### 用户体验监控

监控真实用户的使用体验,包括加载时间、交互响应等。

### 资源监控

监控各类资源的加载情况,包括脚本、样式、图片等。

## 实战场景一: 基础性能监控

```typescript
// lib/performance-monitor.ts
export class PerformanceMonitor {
  private metrics: Map<string, number> = new Map();

  constructor() {
    if (typeof window !== "undefined") {
      this.initMonitoring();
    }
  }

  private initMonitoring() {
    // 监听页面加载完成
    window.addEventListener("load", () => {
      this.collectLoadMetrics();
    });

    // 监听性能条目
    const observer = new PerformanceObserver((list) => {
      for (const entry of list.getEntries()) {
        this.processEntry(entry);
      }
    });

    observer.observe({
      entryTypes: ["navigation", "resource", "paint", "measure"],
    });
  }

  private collectLoadMetrics() {
    const navigation = performance.getEntriesByType(
      "navigation"
    )[0] as PerformanceNavigationTiming;

    if (navigation) {
      this.metrics.set(
        "dns",
        navigation.domainLookupEnd - navigation.domainLookupStart
      );
      this.metrics.set("tcp", navigation.connectEnd - navigation.connectStart);
      this.metrics.set(
        "ttfb",
        navigation.responseStart - navigation.requestStart
      );
      this.metrics.set(
        "download",
        navigation.responseEnd - navigation.responseStart
      );
      this.metrics.set(
        "domParse",
        navigation.domInteractive - navigation.responseEnd
      );
      this.metrics.set(
        "domReady",
        navigation.domContentLoadedEventEnd -
          navigation.domContentLoadedEventStart
      );
      this.metrics.set(
        "load",
        navigation.loadEventEnd - navigation.loadEventStart
      );
    }
  }

  private processEntry(entry: PerformanceEntry) {
    console.log(
      `Performance entry: ${entry.name} (${entry.entryType}) - ${entry.duration}ms`
    );
  }

  getMetrics() {
    return Object.fromEntries(this.metrics);
  }

  reportMetrics() {
    const metrics = this.getMetrics();

    // 发送到分析服务
    fetch("/api/analytics/performance", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        metrics,
        timestamp: Date.now(),
        url: window.location.href,
      }),
    });
  }
}

// 使用
const monitor = new PerformanceMonitor();
setTimeout(() => monitor.reportMetrics(), 5000);
```

### 实战场景三: 资源加载监控

```typescript
// lib/resource-monitor.ts
export class ResourceMonitor {
  private resources: PerformanceResourceTiming[] = [];

  constructor() {
    this.collectResources();
  }

  private collectResources() {
    const observer = new PerformanceObserver((list) => {
      for (const entry of list.getEntries()) {
        if (entry.entryType === "resource") {
          this.resources.push(entry as PerformanceResourceTiming);
          this.analyzeResource(entry as PerformanceResourceTiming);
        }
      }
    });

    observer.observe({ entryTypes: ["resource"] });
  }

  private analyzeResource(resource: PerformanceResourceTiming) {
    const duration = resource.responseEnd - resource.startTime;
    const size = resource.transferSize;

    // 检测慢资源
    if (duration > 1000) {
      console.warn(`Slow resource: ${resource.name} (${duration}ms)`);
      this.reportSlowResource(resource);
    }

    // 检测大文件
    if (size > 500000) {
      console.warn(
        `Large resource: ${resource.name} (${(size / 1024).toFixed(2)}KB)`
      );
      this.reportLargeResource(resource);
    }
  }

  private reportSlowResource(resource: PerformanceResourceTiming) {
    fetch("/api/analytics/slow-resource", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        name: resource.name,
        duration: resource.responseEnd - resource.startTime,
        type: resource.initiatorType,
        timestamp: Date.now(),
      }),
    });
  }

  private reportLargeResource(resource: PerformanceResourceTiming) {
    fetch("/api/analytics/large-resource", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        name: resource.name,
        size: resource.transferSize,
        type: resource.initiatorType,
        timestamp: Date.now(),
      }),
    });
  }

  getResourcesByType(type: string) {
    return this.resources.filter((r) => r.initiatorType === type);
  }

  getTotalSize() {
    return this.resources.reduce((sum, r) => sum + r.transferSize, 0);
  }
}
```

```typescript
// lib/long-task-monitor.ts
export class LongTaskMonitor {
  private longTasks: PerformanceEntry[] = [];

  constructor() {
    this.initMonitoring();
  }

  private initMonitoring() {
    const observer = new PerformanceObserver((list) => {
      for (const entry of list.getEntries()) {
        this.longTasks.push(entry);
        this.analyzeLongTask(entry);
      }
    });

    observer.observe({ entryTypes: ["longtask"] });
  }

  private analyzeLongTask(task: PerformanceEntry) {
    console.warn(`Long task detected: ${task.duration}ms`);

    // 报告长任务
    fetch("/api/analytics/long-task", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        duration: task.duration,
        startTime: task.startTime,
        timestamp: Date.now(),
        url: window.location.href,
      }),
    });
  }

  getLongTasks() {
    return this.longTasks;
  }

  getTotalBlockingTime() {
    return this.longTasks.reduce((sum, task) => {
      // 超过50ms的部分算作阻塞时间
      return sum + Math.max(0, task.duration - 50);
    }, 0);
  }
}
```

```typescript
export class MemoryMonitor {
  private samples: number[] = [];

  constructor() {
    this.startMonitoring();
  }

  private startMonitoring() {
    setInterval(() => {
      this.collectMemorySample();
    }, 10000); // 每10秒采样一次

    if ("memory" in performance) {
      const memory = (performance as any).memory;
      const usedMB = memory.usedJSHeapSize / 1024 / 1024;

      this.samples.push(usedMB);

      // 检测内存泄漏
      if (this.samples.length > 10) {
        const trend = this.calculateTrend();
        if (trend > 5) {
          // 内存持续增长超过5MB
          console.warn("Potential memory leak detected");
          this.reportMemoryLeak();
        }
      }

      // 只保留最近100个样本
      if (this.samples.length > 100) {
        this.samples.shift();
      }
    }
  }
}
```

## 常见问题

### 1. 性能监控会影响应用性能吗?

会有轻微影响,但可以通过采样和节流来优化。

| 监控类型   | 性能影响 | 优化方案                 |
| ---------- | -------- | ------------------------ |
| Web Vitals | 很小     | 使用 requestIdleCallback |
| 资源监控   | 小       | 批量上报                 |
| 错误监控   | 很小     | 异步上报                 |
| 内存监控   | 中等     | 降低采样频率             |

```typescript
// lib/batch-reporter.ts
class BatchReporter {
  private queue: Metric[] = [];
  private batchSize = 10;
  private flushInterval = 5000;

  constructor() {
    setInterval(() => this.flush(), this.flushInterval);
  }

  add(metric: Metric) {
    this.queue.push(metric);

    if (this.queue.length >= this.batchSize) {
      this.flush();
    }
  }

  flush() {
    if (this.queue.length === 0) return;

    fetch("/api/analytics/batch", {
      method: "POST",
      body: JSON.stringify(this.queue),
      keepalive: true,
    });

    this.queue = [];
  }
}
```

```typescript
export class CustomMetrics {
  static mark(name: string) {
    if (typeof window !== "undefined" && window.performance) {
      performance.mark(name);
    }
  }

  static measure(name: string, startMark: string, endMark: string) {

      const measure = performance.getEntriesByName(name)[0];

      return measure.duration;

    return 0;
  }
}
```

```typescript
// scripts/performance-regression.js
const fs = require("fs");

function detectRegression() {
  const current = JSON.parse(
    fs.readFileSync("performance-current.json", "utf8")
  );
  const baseline = JSON.parse(
    fs.readFileSync("performance-baseline.json", "utf8")
  );

  const regressions = [];

  Object.keys(current).forEach((metric) => {
    const currentValue = current[metric];
    const baselineValue = baseline[metric];

    const change = ((currentValue - baselineValue) / baselineValue) * 100;

    if (change > 10) {
      // 超过10%视为回归
      regressions.push({
        metric,
        baseline: baselineValue,
        current: currentValue,
        change: `+${change.toFixed(2)}%`,
      });
    }
  });

  if (regressions.length > 0) {
    console.error("检测到性能回归:");
    console.table(regressions);
    process.exit(1);
  }
}

detectRegression();
```

## 总结

运行时性能监控是优化应用性能的关键。本文介绍了:

1. **核心概念**: Web Vitals、Performance API、PerformanceObserver
2. **实战场景**: Web Vitals 监控、资源加载、错误追踪、长任务、内存监控

- 使用 Web Vitals 监控核心性能指标
- 使用 PerformanceObserver 监控资源加载

- 设置性能预算并监控超标情况
- 保护用户隐私,不收集敏感信息
- 监控代码不应影响应用正常运行

通过正确实施性能监控,可以及时发现和解决性能问题,提升用户体验。
