# 结构化数据实现

结构化数据使用 Schema.org 标准描述网页内容,帮助搜索引擎更好地理解页面,提升搜索结果展示效果。Next.js 16 提供了便捷的方式来实现结构化数据。

## 核心概念

### Schema.org

定义结构化数据标准的组织,提供各种类型的数据模式。

### JSON-LD

一种基于 JSON 的链接数据格式,是实现结构化数据的推荐方式。

### Rich Snippets

搜索结果中的富媒体展示,如星级评分、价格等。

### Knowledge Graph

Google 的知识图谱,使用结构化数据构建。

### 3.2 Schema.org Validator

```typescript
// 验证 JSON-LD 格式
function validateJsonLd(jsonLd: object): boolean {
  try {
    // 检查必需字段
    if (!jsonLd["@context"] || !jsonLd["@type"]) {
      return false;
    }

    // 检查 @context
    if (jsonLd["@context"] !== "https://schema.org") {
      return false;
    }

    return true;
  } catch (error) {
    return false;
  }
}
```

### 1. 如何选择合适的结构化数据类型?

**常见类型对照**:

| 页面类型 | Schema.org 类型 | 用途           |
| -------- | --------------- | -------------- |
| 博客文章 | Article         | 新闻、博客文章 |
| 产品页   | Product         | 电商产品       |
| 本地商家 | LocalBusiness   | 实体店铺       |
| 事件     | Event           | 活动、会议     |
| FAQ      | FAQPage         | 常见问题       |
| 视频     | VideoObject     | 视频内容       |
| 课程     | Course          | 在线课程       |
| 评论     | Review          | 用户评论       |
| 食谱     | Recipe          | 烹饪食谱       |
| 职位     | JobPosting      | 招聘信息       |

### 5. 如何避免结构化数据错误?

**常见错误**:

| 错误类型     | 原因                  | 解决方案             |
| ------------ | --------------------- | -------------------- |
| 缺少必需字段 | 未提供必需属性        | 查看 Schema.org 文档 |
| 日期格式错误 | 使用错误的日期格式    | 使用 ISO 8601 格式   |
| URL 格式错误 | 相对 URL 而非绝对 URL | 使用完整 URL         |
| 类型不匹配   | 属性值类型错误        | 检查属性类型要求     |
| 重复数据     | 多次定义相同数据      | 合并或移除重复       |

```typescript
// 使用 TypeScript 类型检查
interface ArticleJsonLd {
  "@context": "https://schema.org";
  "@type": "Article";
  headline: string;
  author: {
    "@type": "Person";
    name: string;
  };
  datePublished: string; // ISO 8601
  image: string; // 绝对 URL
}

function createArticleJsonLd(data: ArticleJsonLd) {
  // 验证日期格式
  if (!/^\d{4}-\d{2}-\d{2}/.test(data.datePublished)) {
    throw new Error("Invalid date format");
  }

  // 验证 URL
  if (!/^https?:\/\//.test(data.image)) {
    throw new Error("Image must be absolute URL");
  }

  return data;
}
```

## 注意事项

### 1. 使用绝对 URL

```typescript
// ❌ 错误 - 相对 URL
const jsonLd = {
  "@type": "Article",
  image: "/images/article.jpg",
};

// ✅ 正确 - 绝对 URL
const jsonLd = {
  "@type": "Article",
  image: "https://example.com/images/article.jpg",
};
```

### 2. 日期格式

```typescript
// ✅ 正确 - ISO 8601 格式
const jsonLd = {
  "@type": "Article",
  datePublished: "2024-01-15T10:30:00+08:00",
  dateModified: "2024-01-16T14:20:00+08:00",
};

// ❌ 错误 - 其他格式
const jsonLd = {
  "@type": "Article",
  datePublished: "01/15/2024",
};
```

### 3. 必需字段

每种类型都有必需字段,务必提供:

- **Article**: headline, author, datePublished, image
- **Product**: name, image, offers
- **Event**: name, startDate, location
- **Organization**: name, url

### 4. 避免重复

不要在同一页面重复定义相同的结构化数据。

## 总结

结构化数据是提升 SEO 的重要手段。本文介绍了:

1. **基础类型**: Article、Product、Breadcrumb 等常见类型
2. **高级类型**: Organization、LocalBusiness、Event、FAQ、Video、Course
3. **工具验证**: Google Rich Results Test、Schema Validator、自动化测试
4. **最佳实践**: 选择合适类型、处理多个数据、避免常见错误

关键要点:

- 使用 JSON-LD 格式
- 提供所有必需字段
- 使用绝对 URL
- 使用 ISO 8601 日期格式
- 定期测试和验证
- 避免重复数据
- 根据页面类型选择合适的 Schema

通过正确实施结构化数据,可以显著提升搜索结果的展示效果和点击率。
