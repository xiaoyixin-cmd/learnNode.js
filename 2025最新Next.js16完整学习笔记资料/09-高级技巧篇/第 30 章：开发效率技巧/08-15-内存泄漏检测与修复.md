# 内存泄漏检测与修复

## 1. 内存泄漏基础

### 1.1 什么是内存泄漏

内存泄漏指程序中已分配的内存无法被垃圾回收器回收,导致内存占用持续增长,最终可能导致应用崩溃或性能严重下降。

**常见表现**:

| 症状         | 描述                          | 影响                        |
| ------------ | ----------------------------- | --------------------------- |
| 内存持续增长 | 应用运行时间越长,内存占用越高 | 最终导致 OOM(Out of Memory) |
| 页面卡顿     | 垃圾回收频繁触发              | 用户体验下降                |
| 响应变慢     | 可用内存减少                  | 操作延迟增加                |
| 浏览器崩溃   | 内存耗尽                      | 应用不可用                  |

### 1.2 JavaScript 垃圾回收机制

**标记清除算法**:

```typescript
// 垃圾回收示例
function example() {
  let obj1 = { name: "Object 1" }; // 创建对象
  let obj2 = { name: "Object 2" };

  obj1.ref = obj2; // obj1 引用 obj2
  obj2.ref = obj1; // obj2 引用 obj1

  // 函数结束后,obj1 和 obj2 都不可达
  // 即使存在循环引用,也会被回收
}

example();
// obj1 和 obj2 会被垃圾回收
```

**引用计数算法**:

```typescript
// 引用计数问题示例
function createCircularReference() {
  let obj1 = {};
  let obj2 = {};

  obj1.ref = obj2;
  obj2.ref = obj1;

  // 在旧的引用计数算法中,这会导致内存泄漏
  // 因为两个对象的引用计数永远不会为 0
}
```

### 1.3 内存泄漏的常见原因

| 原因               | 示例                                            | 解决方案                  |
| ------------------ | ----------------------------------------------- | ------------------------- |
| 未清理的事件监听器 | `addEventListener` 未配对 `removeEventListener` | 在组件卸载时清理          |
| 定时器未清除       | `setInterval` 未调用 `clearInterval`            | 使用 `useEffect` 清理函数 |
| 闭包引用           | 闭包持有大对象引用                              | 及时释放引用              |
| 全局变量           | 意外创建全局变量                                | 使用严格模式              |
| DOM 引用           | 保存已删除 DOM 的引用                           | 清理 DOM 引用             |
| 缓存未限制         | 无限增长的缓存                                  | 实现 LRU 缓存             |

export class MemoryMonitor {
  private interval: NodeJS.Timeout | null = null;

  start(intervalMs: number = 5000) {
    this.interval = setInterval(() => {
      const usage = process.memoryUsage();

      console.log({
        rss: `${Math.round(usage.rss / 1024 / 1024)} MB`,
        heapTotal: `${Math.round(usage.heapTotal / 1024 / 1024)} MB`,
        heapUsed: `${Math.round(usage.heapUsed / 1024 / 1024)} MB`,

      clearInterval(this.interval);
      this.interval = null;
    }
  }
}

### 3.2 缓存泄漏

```typescript
// lib/lru-cache.ts
export class LRUCache<K, V> {
  private cache: Map<K, V>;
  private maxSize: number;

  constructor(maxSize: number = 100) {
    this.cache = new Map();
    this.maxSize = maxSize;
  }

  get(key: K): V | undefined {
    const value = this.cache.get(key);

    if (value !== undefined) {
      // 移到最后(最近使用)
      this.cache.delete(key);
      this.cache.set(key, value);
    }

    return value;
  }

  set(key: K, value: V): void {
    // 如果已存在,先删除
    if (this.cache.has(key)) {
      this.cache.delete(key);
    }

    // 添加到最后
    this.cache.set(key, value);

    // 超过大小限制,删除最旧的
    if (this.cache.size > this.maxSize) {
      const firstKey = this.cache.keys().next().value;
      this.cache.delete(firstKey);
    }
  }

  clear(): void {
    this.cache.clear();
  }
}

// 使用示例
const cache = new LRUCache<string, any>(100);

cache.set("key1", { data: "value1" });
cache.set("key2", { data: "value2" });

const value = cache.get("key1");
```

// lib/websocket-manager.ts
export class WebSocketManager {
  private ws: WebSocket | null = null;
  private reconnectTimer: NodeJS.Timeout | null = null;

  connect(url: string) {
    this.ws = new WebSocket(url);

    this.ws.onopen = () => {
      console.log("Connected");
    };

    this.ws.onclose = () => {
      console.log("Disconnected");
      // 自动重连
      this.reconnectTimer = setTimeout(() => {
        this.connect(url);
      }, 5000);
    };

    this.ws.onerror = (error) => {
      console.error("WebSocket error:", error);
    };
  }

  disconnect() {
    // 清理定时器
    if (this.reconnectTimer) {
      clearTimeout(this.reconnectTimer);
      this.reconnectTimer = null;
    }

    // 关闭连接
    if (this.ws) {
      this.ws.close();
      this.ws = null;
    }
  }

  send(data: any) {
    if (this.ws && this.ws.readyState === WebSocket.OPEN) {
      this.ws.send(JSON.stringify(data));
    }
  }
}

### 3. 如何在生产环境监控内存泄漏?

```typescript
// lib/production-memory-monitor.ts
export class ProductionMemoryMonitor {
  private checkInterval: NodeJS.Timeout | null = null;
  private memoryHistory: number[] = [];
  private maxHistorySize = 10;

  start() {
    this.checkInterval = setInterval(() => {
      const usage = process.memoryUsage();
      const heapUsedMB = usage.heapUsed / 1024 / 1024;

      this.memoryHistory.push(heapUsedMB);

      if (this.memoryHistory.length > this.maxHistorySize) {
        this.memoryHistory.shift();
      }

      // 检测持续增长
      if (this.isMemoryGrowing()) {
        this.reportLeak(heapUsedMB);
      }
    }, 60000); // 每分钟检查一次
  }

  private isMemoryGrowing(): boolean {
    if (this.memoryHistory.length < this.maxHistorySize) {
      return false;
    }

    // 检查是否持续增长
    for (let i = 1; i < this.memoryHistory.length; i++) {
      if (this.memoryHistory[i] <= this.memoryHistory[i - 1]) {
        return false;
      }
    }

    return true;
  }

  private reportLeak(currentMemory: number) {
    // 发送告警
    console.error("Potential memory leak detected!", {
      currentMemory: `${currentMemory.toFixed(2)} MB`,
      history: this.memoryHistory,
    });

    // 发送到监控服务
    fetch("/api/monitoring/memory-leak", {
      method: "POST",
      body: JSON.stringify({
        timestamp: new Date().toISOString(),
        memory: currentMemory,
        history: this.memoryHistory,
      }),
    });
  }

  stop() {
    if (this.checkInterval) {
      clearInterval(this.checkInterval);
      this.checkInterval = null;
    }
  }
}
```

### 1. 开发环境 vs 生产环境

- 开发环境可能不会暴露内存泄漏
- 使用生产构建进行测试
- 进行长时间运行测试

### 2. 浏览器差异

- 不同浏览器的垃圾回收策略不同
- 在多个浏览器中测试
- 关注目标用户的主要浏览器

### 3. 性能影响

- 过度的内存检查会影响性能
- 在生产环境使用采样监控
- 平衡监控成本和收益

内存泄漏是 Next.js 应用中需要重点关注的问题。本文介绍了:
