# 站点地图生成

站点地图(Sitemap)是告诉搜索引擎网站结构的 XML 文件,帮助搜索引擎更好地抓取和索引网站内容。Next.js 16 提供了便捷的方式来生成站点地图。

## 核心概念

### Sitemap

XML 格式的文件,列出网站所有页面的 URL。

### 优先级

页面的相对重要性,范围 0.0 到 1.0。

### 更新频率

页面内容更新的频率,如 daily、weekly 等。

### 最后修改时间

页面最后一次修改的时间戳。

### 实战场景三: 多语言站点地图

```typescript
// app/sitemap.ts
import { MetadataRoute } from "next";

const languages = ["zh", "en", "ja"];

export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  const posts = await getAllPosts();

  const sitemapEntries: MetadataRoute.Sitemap = [];

  // 为每种语言生成URL
  for (const lang of languages) {
    // 首页
    sitemapEntries.push({
      url: `https://example.com/${lang}`,
      lastModified: new Date(),
      changeFrequency: "yearly",
      priority: 1,
      alternates: {
        languages: Object.fromEntries(
          languages.map((l) => [l, `https://example.com/${l}`])
        ),
      },
    });

    // 文章页
    posts.forEach((post) => {
      sitemapEntries.push({
        url: `https://example.com/${lang}/blog/${post.slug}`,
        lastModified: new Date(post.updatedAt),
        changeFrequency: "weekly",
        priority: 0.7,
        alternates: {
          languages: Object.fromEntries(
            languages.map((l) => [
              l,
              `https://example.com/${l}/blog/${post.slug}`,
            ])
          ),
        },
      });
    });
  }

  return sitemapEntries;
}

async function getAllPosts() {
  return [
    { slug: "post-1", updatedAt: "2024-01-01" },
    { slug: "post-2", updatedAt: "2024-01-02" },
  ];
}
```

```typescript
// app/sitemap.ts
import { MetadataRoute } from "next";
import { db } from "@/lib/db";

export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  const baseUrl = "https://example.com";

  // 获取所有已发布的文章
  const posts = await db.post.findMany({
    where: { published: true },
    select: {
      slug: true,
      updatedAt: true,
      createdAt: true,
    },
    orderBy: { updatedAt: "desc" },
  });

  // 获取所有分类
  const categories = await db.category.findMany({
    select: {
      slug: true,
      updatedAt: true,
    },
  });

  // 获取所有标签
  const tags = await db.tag.findMany({
    select: {
      slug: true,
      updatedAt: true,
    },
  });

  return [
    // 首页
    {
      url: baseUrl,
      lastModified: new Date(),
      changeFrequency: "daily",
      priority: 1,
    },
    // 关于页面
    {
      url: `${baseUrl}/about`,
      lastModified: new Date(),
      changeFrequency: "monthly",
      priority: 0.8,
    },
    // 博客列表页
    {
      url: `${baseUrl}/blog`,
      lastModified: posts[0]?.updatedAt || new Date(),
      changeFrequency: "daily",
      priority: 0.9,
    },
    // 所有文章
    ...posts.map((post) => ({
      url: `${baseUrl}/blog/${post.slug}`,
      lastModified: post.updatedAt,
      changeFrequency: "weekly" as const,
      priority: 0.7,
    })),
    // 所有分类
    ...categories.map((category) => ({
      url: `${baseUrl}/category/${category.slug}`,
      lastModified: category.updatedAt,
      changeFrequency: "weekly" as const,
      priority: 0.6,
    })),
    // 所有标签
    ...tags.map((tag) => ({
      url: `${baseUrl}/tag/${tag.slug}`,
      lastModified: tag.updatedAt,
      changeFrequency: "weekly" as const,
      priority: 0.5,
    })),
  ];
}
```

```typescript
export async function GET() {

    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;")
    .replace(/'/g, "&apos;");
```

## 注意事项

### 1. URL 格式

- 使用完整的绝对 URL
- 包含协议 (https://)
- 确保 URL 可访问

### 2. 更新频率

- 设置合理的 changeFrequency
- 不要过于频繁
- 与实际更新频率匹配

### 3. 优先级

- 合理设置 priority (0.0-1.0)
- 首页通常为 1.0
- 内容页面 0.5-0.8

### 4. 性能优化

- 使用缓存减少数据库查询
- 大型网站使用分段站点地图
- 设置合理的 Cache-Control

### 5. 定期更新

- 内容更新时重新生成站点地图
- 使用 revalidate 自动更新
- 提交到搜索引擎

## 总结

站点地图是 SEO 的重要组成部分。本文介绍了:

1. **基础配置**: 静态、动态、多语言、分段站点地图
2. **高级配置**: 数据库驱动、缓存优化、图片/新闻/视频站点地图
3. **最佳实践**: 优先级设置、更新频率、大型网站分段策略

关键要点:

- 使用 Next.js 的 sitemap.ts 文件
- 包含所有重要页面
- 设置合理的优先级和更新频率
- 大型网站使用分段策略
- 定期提交到搜索引擎
- 使用缓存优化性能
- 排除不需要索引的页面

通过正确配置站点地图,可以帮助搜索引擎更好地抓取和索引网站内容,提升 SEO 效果。
