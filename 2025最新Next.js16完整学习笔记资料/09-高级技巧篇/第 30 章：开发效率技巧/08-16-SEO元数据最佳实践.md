# SEO 元数据最佳实践

## 1. 元数据基础

### 1.1 什么是 SEO 元数据

SEO 元数据是嵌入在 HTML 页面中的结构化信息,用于向搜索引擎和社交媒体平台描述页面内容。正确配置元数据可以:

- 提高搜索引擎排名
- 增加点击率(CTR)
- 改善社交媒体分享效果
- 提升用户体验

### 1.2 元数据类型对比

| 类型          | 用途                             | 示例                                   |
| ------------- | -------------------------------- | -------------------------------------- |
| 基础元数据    | 页面标题、描述、关键词           | `<title>`, `<meta name="description">` |
| Open Graph    | 社交媒体分享(Facebook, LinkedIn) | `<meta property="og:title">`           |
| Twitter Cards | Twitter 分享优化                 | `<meta name="twitter:card">`           |
| 结构化数据    | 搜索引擎理解内容                 | JSON-LD Schema.org                     |
| Robots 元标签 | 控制爬虫行为                     | `<meta name="robots">`                 |

### 1.3 Next.js 16 元数据 API

Next.js 16 提供了两种方式配置元数据:

**静态元数据**:

```typescript
// app/layout.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: "我的网站",
  description: "网站描述",
};
```

**动态元数据**:

```typescript
// app/blog/[slug]/page.tsx

export async function generateMetadata({ params }): Promise<Metadata> {
  const post = await getPost(params.slug);

  return {
    title: post.title,
    description: post.excerpt,
  };
}
```

// app/blog/[slug]/page.tsx
export default async function BlogPost({ params }) {
  const post = await getPost(params.slug);

  const jsonLd = {
    "@context": "https://schema.org",
    "@type": "BlogPosting",
    headline: post.title,
    image: post.image,
    datePublished: post.publishedAt,
    dateModified: post.updatedAt,
    author: {
      "@type": "Person",
      name: post.author.name,
      url: post.author.url,
    },
    publisher: {
      "@type": "Organization",
      name: "我的网站",
      logo: {
        "@type": "ImageObject",
        url: "https://example.com/logo.png",
      },
    },
    description: post.excerpt,
  };

  return (
    <>
      <script
        type="application/ld+json"
        dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }}
      />
      <article>
        <h1>{post.title}</h1>
        <div dangerouslySetInnerHTML={{ __html: post.content }} />
      </article>
    </>
  );
}

### 5. 如何处理多语言网站的元数据?

```typescript
// app/[lang]/layout.tsx
import type { Metadata } from "next";

const dictionaries = {
  en: {
    title: "My Website",
    description: "Welcome to my website",
  },
  zh: {
    title: "我的网站",
    description: "欢迎访问我的网站",
  },
  ja: {
    title: "私のウェブサイト",
    description: "私のウェブサイトへようこそ",
  },
};

export async function generateMetadata({ params }): Promise<Metadata> {
  const dict = dictionaries[params.lang] || dictionaries.en;

  return {
    title: dict.title,
    description: dict.description,
    alternates: {
      canonical: `https://example.com/${params.lang}`,
      languages: {
        "en-US": "https://example.com/en",
        "zh-CN": "https://example.com/zh",
        "ja-JP": "https://example.com/ja",
      },
    },
    openGraph: {
      locale:
        params.lang === "zh"
          ? "zh_CN"
          : params.lang === "ja"
          ? "ja_JP"
          : "en_US",
      alternateLocale: ["en_US", "zh_CN", "ja_JP"].filter(
        (l) =>
          l !==
          (params.lang === "zh"
            ? "zh_CN"
            : params.lang === "ja"
            ? "ja_JP"
            : "en_US")
      ),
    },
  };
}
```

## 注意事项

### 1. 元数据长度限制

- **标题**: 50-60 个字符(Google 显示约 60 个字符)
- **描述**: 150-160 个字符(Google 显示约 160 个字符)
- **关键词**: 不要超过 10 个

### 2. 避免重复内容

```typescript
// 使用 canonical 标签
export const metadata: Metadata = {
  alternates: {
    canonical: "https://example.com/original-page",
  },
};

// 或使用 robots 阻止索引
export const metadata: Metadata = {
  robots: {
    index: false,
    follow: true,
  },
};
```

### 3. 图片要求

- **Open Graph**: 1200x630 像素(比例 1.91:1)
- **Twitter Card**: 1200x675 像素(比例 16:9)
- **格式**: JPG 或 PNG
- **大小**: 小于 5MB

### 4. 性能考虑

// app/[lang]/page.tsx
import { Metadata } from "next";

const translations = {
  en: {
    title: "Welcome to Our Site",
    description: "Discover amazing content",

    title: "欢迎来到我们的网站",

export async function generateMetadata({
  params,
}: {
  params: { lang: string };

        "en-US": "/en",

      },
    },
  };
}

### 3. 面包屑导航结构化数据

```typescript
// components/Breadcrumbs.tsx
export function Breadcrumbs({
  items,
}: {
  items: Array<{ name: string; url: string }>;
}) {
  const breadcrumbSchema = {
    "@context": "https://schema.org",
    "@type": "BreadcrumbList",
    itemListElement: items.map((item, index) => ({
      "@type": "ListItem",
      position: index + 1,
      name: item.name,
      item: `https://example.com${item.url}`,
    })),
  };

  return (
    <>
      <script
        type="application/ld+json"
        dangerouslySetInnerHTML={{ __html: JSON.stringify(breadcrumbSchema) }}
      />
      <nav aria-label="Breadcrumb">
        {items.map((item, index) => (
          <span key={index}>
            <a href={item.url}>{item.name}</a>
            {index < items.length - 1 && " > "}
          </span>
        ))}
      </nav>
    </>
  );
}
```

export function FAQ({
  items,
}: {
  items: Array<{ question: string; answer: string }>;

    <>
      <script
        type="application/ld+json"
        dangerouslySetInnerHTML={{ __html: JSON.stringify(faqSchema) }}
      />
      <div>
        {items.map((item, index) => (
          <div key={index}>
            <h3>{item.question}</h3>
            <p>{item.answer}</p>
          </div>
        ))}
      </div>
    </>
  );
}
