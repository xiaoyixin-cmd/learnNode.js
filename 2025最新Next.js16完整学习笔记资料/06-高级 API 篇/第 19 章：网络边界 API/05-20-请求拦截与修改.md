# 05-20-请求拦截与修改

## 概述

在 Next.js 16 中,请求拦截与修改是 Middleware 的核心功能之一。通过拦截请求,我们可以在请求到达目标路由之前对其进行检查、修改或重定向。本文将详细介绍如何在 Next.js 16 中实现各种请求拦截和修改场景。

### 核心特性

- **请求拦截**: 在请求到达路由前拦截
- **请求修改**: 修改请求头、Cookie、URL 等
- **条件路由**: 根据条件决定请求走向
- **请求验证**: 验证请求的合法性
- **请求转换**: 转换请求参数和格式
- **请求日志**: 记录请求信息
- **性能监控**: 监控请求性能
- **边缘运行**: 在 Edge Runtime 运行

### 拦截流程

```
┌─────────────┐
│  客户端请求  │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  Middleware │
│  - 拦截     │
│  - 验证     │
│  - 修改     │
└──────┬──────┘
       │
       ├─ 重定向
       ├─ 重写
       ├─ 继续
       └─ 拒绝
       │
       ▼
┌─────────────┐
│  目标路由   │
└─────────────┘
```

### 拦截场景对比

| 场景     | 用途             | 复杂度 | 性能影响 |
| -------- | ---------------- | ------ | -------- |
| 身份验证 | 检查用户登录状态 | 中     | 低       |
| 权限控制 | 检查用户权限     | 高     | 中       |
| 请求日志 | 记录请求信息     | 低     | 低       |
| 限流保护 | 防止滥用         | 中     | 中       |
| 数据转换 | 转换请求数据     | 中     | 低       |

### 5.3 请求签名验证

```ts
// middleware.ts
import { NextRequest, NextResponse } from "next/server";
import crypto from "crypto";

export function middleware(request: NextRequest) {
  const signature = request.headers.get("X-Signature");
  const timestamp = request.headers.get("X-Timestamp");

  if (!signature || !timestamp) {
    return NextResponse.json(
      { error: "Missing signature or timestamp" },
      { status: 401 }
    );
  }

  // 检查时间戳是否过期(5分钟)
  const now = Date.now();
  const requestTime = parseInt(timestamp);

  if (now - requestTime > 5 * 60 * 1000) {
    return NextResponse.json({ error: "Request expired" }, { status: 401 });
  }

  // 验证签名
  const secret = process.env.API_SECRET!;
  const expectedSignature = crypto
    .createHmac("sha256", secret)
    .update(`${request.url}${timestamp}`)
    .digest("hex");

  if (signature !== expectedSignature) {
    return NextResponse.json({ error: "Invalid signature" }, { status: 403 });
  }

  return NextResponse.next();
}

export const config = {
  matcher: "/api/secure/:path*",
};
```

## 7. 实战案例

### 7.1 完整的身份验证中间件

```ts
// middleware.ts
import { NextRequest, NextResponse } from "next/server";

export function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl;

  // 公开路径
  const publicPaths = ["/login", "/register", "/forgot-password", "/"];
  if (publicPaths.includes(pathname)) {
    return NextResponse.next();
  }

  // 静态资源
  if (pathname.startsWith("/_next") || pathname.startsWith("/static")) {
    return NextResponse.next();
  }

  // 检查认证
  const token = request.cookies.get("auth-token")?.value;

  if (!token) {
    const url = new URL("/login", request.url);
    url.searchParams.set("redirect", pathname);
    return NextResponse.redirect(url);
  }

  try {
    const payload = verifyToken(token);

    // 检查权限
    if (pathname.startsWith("/admin") && payload.role !== "admin") {
      return NextResponse.redirect(new URL("/403", request.url));
    }

    // 添加用户信息到请求头
    const requestHeaders = new Headers(request.headers);
    requestHeaders.set("X-User-ID", payload.userId);
    requestHeaders.set("X-User-Role", payload.role);
    requestHeaders.set("X-User-Email", payload.email);

    return NextResponse.next({
      request: {
        headers: requestHeaders,
      },
    });
  } catch (error) {
    // Token无效,清除并重定向
    const response = NextResponse.redirect(new URL("/login", request.url));
    response.cookies.delete("auth-token");
    return response;
  }
}

function verifyToken(token: string): {
  userId: string;
  role: string;
  email: string;
} {
  // 实际的token验证逻辑
  return {
    userId: "123",
    role: "user",
    email: "user@example.com",
  };
}

export const config = {
  matcher: ["/((?!api|_next/static|_next/image|favicon.ico).*)"],
};
```

### 7.2 API 限流中间件

```ts
// middleware.ts
import { NextRequest, NextResponse } from "next/server";

const rateLimitMap = new Map<string, number[]>();

export function middleware(request: NextRequest) {
  const ip = request.ip || "unknown";
  const now = Date.now();
  const windowMs = 60 * 1000; // 1分钟
  const maxRequests = 100;

  // 获取该IP的请求记录
  const requests = rateLimitMap.get(ip) || [];

  // 清理过期的请求记录
  const validRequests = requests.filter((time) => now - time < windowMs);

  if (validRequests.length >= maxRequests) {
    return NextResponse.json(
      {
        error: "Too many requests",
        retryAfter: Math.ceil((validRequests[0] + windowMs - now) / 1000),
      },
      {
        status: 429,
        headers: {
          "Retry-After": "60",
          "X-RateLimit-Limit": maxRequests.toString(),
          "X-RateLimit-Remaining": "0",
        },
      }
    );
  }

  // 记录本次请求
  validRequests.push(now);
  rateLimitMap.set(ip, validRequests);

  // 添加限流信息到响应头
  const response = NextResponse.next();
  response.headers.set("X-RateLimit-Limit", maxRequests.toString());
  response.headers.set(
    "X-RateLimit-Remaining",
    (maxRequests - validRequests.length).toString()
  );

  return response;
}

export const config = {
  matcher: "/api/:path*",
};
```

### 7.3 国际化路由中间件

```ts
// middleware.ts
import { NextRequest, NextResponse } from "next/server";

const locales = ["en", "zh", "ja", "ko"];
const defaultLocale = "en";

export function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl;

  // 检查路径是否已包含语言前缀
  const pathnameHasLocale = locales.some(
    (locale) => pathname.startsWith(`/${locale}/`) || pathname === `/${locale}`
  );

  if (pathnameHasLocale) {
    return NextResponse.next();
  }

  // 获取用户首选语言
  const locale = getLocale(request);

  // 重定向到带语言前缀的路径
  const url = request.nextUrl.clone();
  url.pathname = `/${locale}${pathname}`;

  return NextResponse.redirect(url);
}

function getLocale(request: NextRequest): string {
  // 1. 检查Cookie
  const cookieLocale = request.cookies.get("locale")?.value;
  if (cookieLocale && locales.includes(cookieLocale)) {
    return cookieLocale;
  }

  // 2. 检查Accept-Language头
  const acceptLanguage = request.headers.get("Accept-Language");
  if (acceptLanguage) {
    const preferredLocale = acceptLanguage.split(",")[0].split("-")[0];

    if (locales.includes(preferredLocale)) {
      return preferredLocale;
    }
  }

  // 3. 返回默认语言
  return defaultLocale;
}

export const config = {
  matcher: ["/((?!api|_next|static|favicon.ico).*)"],
};
```

### 10.6 如何处理多个中间件?

**问题**: 需要组合多个中间件逻辑。

**解决方案**:

```ts
export function middleware(request: NextRequest) {
  // 1. 身份验证
  const authResult = handleAuth(request);
  if (authResult) return authResult;

  // 2. 限流
  const rateLimitResult = handleRateLimit(request);
  if (rateLimitResult) return rateLimitResult;
```

### 10.7 如何实现请求重试?

**问题**: 请求失败时自动重试。

**解决方案**:

```ts
export async function middleware(request: NextRequest) {
  const maxRetries = 3;
  let lastError;

  for (let i = 0; i < maxRetries; i++) {
    try {
      const response = await fetch(request.url);
      if (response.ok) {
        return NextResponse.next();
      }
    } catch (error) {
      lastError = error;
      await new Promise((resolve) => setTimeout(resolve, 1000 * (i + 1)));
    }
  }

  return NextResponse.json({ error: "Max retries exceeded" }, { status: 503 });
}
```

```ts
const cache = new Map();

export async function middleware(request: NextRequest) {
  if (request.method === "GET") {
    const cached = cache.get(request.url);

    if (cached && Date.now() - cached.timestamp < 60000) {
      return new NextResponse(cached.data);
    }
  }

  return NextResponse.next();
}
```
