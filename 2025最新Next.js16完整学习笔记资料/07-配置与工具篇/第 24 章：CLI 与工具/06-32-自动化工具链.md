# 自动化工具链

自动化工具链可以减少重复性工作,提高开发效率和代码质量。本文将介绍如何在 Next.js 16 项目中构建完整的自动化工具链,包括构建自动化、测试自动化、部署自动化等。

## 概述

现代前端项目的自动化工具链通常包括:

- **构建自动化**: 代码编译、打包、优化
- **测试自动化**: 单元测试、集成测试、E2E 测试
- **代码质量自动化**: Lint、格式化、类型检查
- **部署自动化**: CI/CD、版本发布、回滚
- **监控自动化**: 性能监控、错误追踪、日志收集

### 工具链对比

| 工具类型 | 推荐工具               | 优势         | 适用场景           |
| -------- | ---------------------- | ------------ | ------------------ |
| 构建工具 | Turbopack              | 速度快       | Next.js 16 默认    |
| 测试框架 | Jest + Testing Library | 生态成熟     | 单元测试、集成测试 |
| E2E 测试 | Playwright             | 功能强大     | 端到端测试         |
| CI/CD    | GitHub Actions         | 免费、集成好 | 开源项目           |
| 部署平台 | Vercel                 | 零配置       | Next.js 项目       |
| 监控工具 | Sentry                 | 功能全面     | 错误追踪           |

```javascript
const { execSync } = require("child_process");
const fs = require("fs");
const path = require("path");

console.log("🚀 开始构建...");

// 1. 清理旧文件

  execSync("tsc --noEmit", { stdio: "inherit" });
} catch (error) {
  console.error("❌ 类型检查失败");
  process.exit(1);
}

// 3. Lint 检查
console.log("🔍 Lint 检查...");
try {
  execSync("next lint", { stdio: "inherit" });
} catch (error) {
  console.error("❌ Lint 检查失败");
  process.exit(1);
}

// 4. 运行测试
console.log("🧪 运行测试...");
try {
  execSync("npm run test", { stdio: "inherit" });
} catch (error) {
  console.error("❌ 测试失败");
  process.exit(1);
}

// 5. 构建
console.log("🏗️  构建应用...");
try {
  execSync("next build", { stdio: "inherit" });
} catch (error) {
  console.error("❌ 构建失败");
  process.exit(1);
}

// 6. 生成构建信息
console.log("📝 生成构建信息...");
const buildInfo = {
  buildTime: new Date().toISOString(),
  version: require("../package.json").version,
  nodeVersion: process.version,
  env: process.env.NODE_ENV || "production",
};

fs.writeFileSync(
  path.join(__dirname, "../public/build-info.json"),
  JSON.stringify(buildInfo, null, 2)
);

console.log("✅ 构建完成!");
```

## CI/CD 自动化

### GitHub Actions 完整配置

```yaml
# .github/workflows/ci.yml
name: CI/CD Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main, develop]

env:
  NODE_VERSION: "20"

jobs:
  # 代码质量检查
  quality:
    name: Code Quality
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: "npm"

      - name: Install dependencies
        run: npm ci

      - name: Type check
        run: npm run type-check

      - name: Lint
        run: npm run lint

      - name: Format check
        run: npm run format:check

  # 单元测试
  test:
    name: Unit Tests
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: "npm"

      - name: Install dependencies
        run: npm ci

      - name: Run tests
        run: npm run test:coverage

      - name: Upload coverage
        uses: codecov/codecov-action@v3
        with:
          files: ./coverage/lcov.info

  # E2E 测试
  e2e:
    name: E2E Tests
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: "npm"

      - name: Install dependencies
        run: npm ci

      - name: Install Playwright
        run: npx playwright install --with-deps

      - name: Run E2E tests
        run: npm run test:e2e

      - name: Upload test results
        if: always()
        uses: actions/upload-artifact@v3
        with:
          name: playwright-report
          path: playwright-report/

  # 构建
  build:
    name: Build
    runs-on: ubuntu-latest
    needs: [quality, test]

    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: "npm"

      - name: Install dependencies
        run: npm ci

      - name: Build
        run: npm run build
        env:
          NEXT_PUBLIC_API_URL: ${{ secrets.API_URL }}

      - name: Upload build
        uses: actions/upload-artifact@v3
        with:
          name: build
          path: .next/

  # 部署到 Vercel
  deploy:
    name: Deploy to Vercel
    runs-on: ubuntu-latest
    needs: [build, e2e]
    if: github.ref == 'refs/heads/main'

    steps:
      - uses: actions/checkout@v4

      - name: Deploy to Vercel
        uses: amondnet/vercel-action@v25
        with:
          vercel-token: ${{ secrets.VERCEL_TOKEN }}
          vercel-org-id: ${{ secrets.VERCEL_ORG_ID }}
          vercel-project-id: ${{ secrets.VERCEL_PROJECT_ID }}
          vercel-args: "--prod"
```

```yaml
        with:
          username: ${{ secrets.DOCKER_USERNAME }}
          password: ${{ secrets.DOCKER_PASSWORD }}

      - name: Extract metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: myapp/nextjs
          tags: |
            type=ref,event=branch
            type=semver,pattern={{version}}
            type=semver,pattern={{major}}.{{minor}}

      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
```

```javascript
  stdio: "inherit",

  `cd ${config.path}`,
  "tar -xzf deploy.tar.gz",
  "npm ci --production",
  "pm2 restart nextjs-app",
].join(" && ");

execSync(`ssh ${config.user}@${config.server} "${remoteCommands}"`, {
  stdio: "inherit",
});

// 5. 清理
console.log("🧹 清理临时文件...");
fs.unlinkSync("deploy.tar.gz");

console.log("✅ 部署成功!");
```

### 性能监控脚本

```javascript
// scripts/monitor.js
const { execSync } = require("child_process");
const https = require("https");

const config = {
  url: process.env.MONITOR_URL || "https://example.com",
  interval: 60000, // 1 分钟
  slackWebhook: process.env.SLACK_WEBHOOK,
};

function checkHealth() {
  return new Promise((resolve, reject) => {
    const startTime = Date.now();

    https
      .get(config.url, (res) => {
        const duration = Date.now() - startTime;

        if (res.statusCode === 200) {
          console.log(`✅ 健康检查通过 (${duration}ms)`);
          resolve({ status: "ok", duration });
        } else {
          console.error(`❌ 健康检查失败: ${res.statusCode}`);
          reject(new Error(`Status code: ${res.statusCode}`));
        }
      })
      .on("error", (error) => {
        console.error(`❌ 健康检查失败: ${error.message}`);
        reject(error);
      });
  });
}

function sendAlert(message) {
  if (!config.slackWebhook) return;

  const payload = JSON.stringify({
    text: message,
  });

  execSync(
    `curl -X POST -H 'Content-type: application/json' --data '${payload}' ${config.slackWebhook}`,
    { stdio: "inherit" }
  );
}

async function monitor() {
  try {
    await checkHealth();
  } catch (error) {
    sendAlert(`🚨 网站异常: ${error.message}`);
  }
}

// 定期检查
setInterval(monitor, config.interval);
monitor();
```

```javascript
  execSync(commands[action], { stdio: "inherit" });
  console.log("✅ 迁移成功!");
} catch (error) {
  console.error("❌ 迁移失败");
  process.exit(1);
}
```

```javascript
// 2. 使用 Git 标签管理版本

async function rollback(version) {
  // 切换版本
  execSync(`git checkout v${version}`);

  // 重新构建
```

## 总结

自动化工具链是现代前端开发的基础设施。通过合理配置,可以:

1. **提高开发效率**: 自动化重复性工作
2. **保证代码质量**: 自动化测试和检查
3. **加快交付速度**: 持续集成和部署
4. **降低出错风险**: 标准化流程
5. **改善团队协作**: 统一的工作流程

关键要点:

- 从简单开始,逐步完善工具链
- 优先自动化最耗时的任务
- 使用 GitHub Actions 实现 CI/CD
- 配置完整的测试流程(单元 + E2E)
- 实现自动化部署和回滚
- 集成监控和告警系统
- 使用缓存优化构建速度
- 安全管理密钥和环境变量
- 为不同规模项目选择合适的工具
- 定期 review 和优化工具链

记住:自动化的目的是让开发者专注于编写代码,而不是处理重复性工作。选择适合团队的工具,不要过度工程化。
