# 测试环境配置

## 概述

测试环境配置是保证代码质量的关键环节。Next.js 16 支持多种测试框架和工具，包括 Jest、Vitest、Playwright、Cypress 等。合理的测试环境配置可以提高测试效率，确保代码质量。

### 测试环境的特点

1. **隔离性**：测试环境独立于开发和生产环境
2. **可重复性**：测试结果可重复验证
3. **快速反馈**：快速执行测试并返回结果
4. **完整覆盖**：单元测试、集成测试、E2E 测试
5. **自动化**：CI/CD 集成，自动执行测试

```typescript
// lib/test-db.ts
import { PrismaClient } from "@prisma/client";

const prisma = new PrismaClient({
  datasources: {
    db: {
      url: process.env.DATABASE_URL,
    },
  },
});

export async function setupTestDatabase() {
  await prisma.$executeRaw`DROP SCHEMA IF EXISTS test CASCADE`;
  await prisma.$executeRaw`CREATE SCHEMA test`;
  await prisma.$migrate.deploy();
}

export async function cleanupTestDatabase() {
  const tables = await prisma.$queryRaw`
    SELECT tablename FROM pg_tables WHERE schemaname = 'test'
  `;

  for (const { tablename } of tables) {
    await prisma.$executeRawUnsafe(`TRUNCATE TABLE test.${tablename} CASCADE`);
  }
}

export default prisma;
```

```typescript
describe("Button", () => {
```

### 案例 2：API 路由测试

```typescript
// app/api/users/route.test.ts
import { GET, POST } from "./route";
import { prisma } from "@/lib/db";

jest.mock("@/lib/db", () => ({
  prisma: {
    user: {
      findMany: jest.fn(),
      create: jest.fn(),
    },
  },
}));

describe("/api/users", () => {
  beforeEach(() => {
    jest.clearAllMocks();
  });

  describe("GET", () => {
    it("returns list of users", async () => {
      const mockUsers = [
        { id: 1, name: "User 1" },
        { id: 2, name: "User 2" },
      ];

      (prisma.user.findMany as jest.Mock).mockResolvedValue(mockUsers);

      const response = await GET();
      const data = await response.json();

      expect(data).toEqual(mockUsers);
      expect(prisma.user.findMany).toHaveBeenCalled();
    });
  });

  describe("POST", () => {
    it("creates new user", async () => {
      const newUser = { name: "New User", email: "new@example.com" };
      const createdUser = { id: 1, ...newUser };

      (prisma.user.create as jest.Mock).mockResolvedValue(createdUser);

      const request = new Request("http://localhost:3000/api/users", {
        method: "POST",
        body: JSON.stringify(newUser),
      });

      const response = await POST(request);
      const data = await response.json();

      expect(data).toEqual(createdUser);
      expect(prisma.user.create).toHaveBeenCalledWith({
        data: newUser,
      });
    });
  });
});
```

```typescript
// __tests__/integration/auth.test.ts
import { setupTestDatabase, cleanupTestDatabase } from "@/lib/test-db";

describe("Authentication Integration", () => {

    expect(user.email).toBe("test@example.com");

    // Login
    const foundUser = await prisma.user.findUnique({
      where: { email: "test@example.com" },
    });

    expect(foundUser).toBeDefined();
    expect(foundUser?.email).toBe("test@example.com");
  });
});
```

## 常见问题

### 1. Jest 配置不生效？

**问题**：Jest 配置修改后不生效

**解决方案**：

```bash
# 清理Jest缓存
jest --clearCache

# 重新运行测试
npm test
```

### 2. Mock 不工作？

**问题**：Mock 函数没有被调用

**解决方案**：

```typescript
// 确保Mock在导入前设置
jest.mock("@/lib/db");
import { prisma } from "@/lib/db";

// 检查Mock是否正确
expect(prisma.user.findMany).toHaveBeenCalled();
```

### 3. 测试超时？

**问题**：测试执行超时

**解决方案**：

```typescript
// 增加超时时间
jest.setTimeout(10000);

// 或在单个测试中
it("should complete", async () => {
  // test code
}, 10000);
```

### 4. 数据库连接失败？

**问题**：测试时无法连接数据库

**解决方案**：

```bash
# 确保测试数据库运行
docker-compose up -d postgres

# 检查连接字符串
DATABASE_URL=postgresql://localhost:5432/test
```

### 5. E2E 测试不稳定？

**问题**：E2E 测试时而通过时而失败

**解决方案**：

```typescript
// 使用waitFor等待元素
await page.waitForSelector(".element");

// 增加重试次数
test.describe.configure({ retries: 2 });
```

### 6. 组件测试找不到元素？

**问题**：Testing Library 找不到元素

**解决方案**：

```typescript
// 使用正确的查询方法

```
