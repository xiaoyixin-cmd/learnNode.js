# 生产环境配置

## 概述

生产环境配置是 Next.js 应用部署的关键，直接影响应用的性能、安全性和稳定性。Next.js 16 提供了完善的生产环境配置选项，包括构建优化、缓存策略、安全配置等。

### 生产环境的特点

1. **性能优化**：代码压缩、资源优化、缓存策略
2. **安全加固**：安全头、HTTPS、环境变量保护
3. **稳定性**：错误处理、日志记录、监控告警
4. **可扩展性**：负载均衡、CDN、容器化部署
5. **可维护性**：版本管理、回滚机制、文档完善

### Docker 部署

```dockerfile
# Dockerfile
FROM node:18-alpine AS base

# 依赖阶段
FROM base AS deps
RUN apk add --no-cache libc6-compat
WORKDIR /app

COPY package*.json ./
RUN npm ci

# 构建阶段
FROM base AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .

ENV NEXT_TELEMETRY_DISABLED 1
RUN npm run build

# 运行阶段
FROM base AS runner
WORKDIR /app

ENV NODE_ENV production
ENV NEXT_TELEMETRY_DISABLED 1

RUN addgroup --system --gid 1001 nodejs
RUN adduser --system --uid 1001 nextjs

COPY --from=builder /app/public ./public
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static

USER nextjs

EXPOSE 3000

ENV PORT 3000
ENV HOSTNAME "0.0.0.0"

CMD ["node", "server.js"]
```

    config.optimization.splitChunks = {
      chunks: "all",

## 总结

生产环境配置是 Next.js 应用成功部署的关键。合理的配置可以：

1. **提升性能**：代码分割、资源压缩、CDN 加速
2. **保障安全**：安全头、HTTPS、环境变量保护
3. **确保稳定**：健康检查、错误监控、日志记录
4. **支持扩展**：负载均衡、容器化、自动扩缩容
5. **便于维护**：监控告警、回滚机制、文档完善

关键要点：

- 使用 `output: 'standalone'` 优化部署包大小
- 配置完善的安全头
- 启用图片优化和 CDN
- 实施完整的监控和日志系统
- 配置健康检查和自动重启
- 使用容器化部署提高可移植性
- 实施零停机部署策略
- 定期备份和测试回滚流程

记住：生产环境配置需要持续优化和监控，根据实际情况调整配置参数，确保应用稳定高效运行。
