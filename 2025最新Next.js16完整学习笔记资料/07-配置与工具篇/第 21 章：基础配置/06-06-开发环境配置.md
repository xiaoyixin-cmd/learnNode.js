# 开发环境配置

## 概述

开发环境配置是 Next.js 项目开发的基础，合理的配置可以提升开发效率、改善开发体验。Next.js 16 提供了丰富的开发环境配置选项，包括热更新、错误提示、性能监控等功能。

### 开发环境的特点

1. **快速反馈**：代码修改后立即看到效果
2. **详细错误**：提供完整的错误堆栈和提示
3. **开发工具**：集成调试工具和性能分析
4. **模拟环境**：模拟生产环境的行为
5. **开发服务器**：提供本地开发服务器

### HTTPS 配置

```javascript
// server.js
const { createServer } = require("https");
const { parse } = require("url");
const next = require("next");
const fs = require("fs");

const dev = process.env.NODE_ENV !== "production";
const app = next({ dev });
const handle = app.getRequestHandler();

const httpsOptions = {
  key: fs.readFileSync("./certificates/localhost-key.pem"),
  cert: fs.readFileSync("./certificates/localhost.pem"),
};

app.prepare().then(() => {
  createServer(httpsOptions, (req, res) => {
    const parsedUrl = parse(req.url, true);
    handle(req, res, parsedUrl);
  }).listen(3000, (err) => {
    if (err) throw err;
    console.log("> Ready on https://localhost:3000");
  });
});
```

## 实战案例

### 案例 1：完整的开发环境配置

```javascript
// next.config.js
module.exports = {
  reactStrictMode: true,

  // 开发指示器
  devIndicators: {
    buildActivity: true,
    buildActivityPosition: "bottom-right",
  },

  // 压缩
  compress: true,

  // 按需编译
  onDemandEntries: {
    maxInactiveAge: 60 * 1000,
    pagesBufferLength: 5,
  },

  // 环境变量
  env: {
    CUSTOM_KEY: process.env.CUSTOM_KEY,
  },

  // 代理配置
  async rewrites() {
    return [
      {
        source: "/api/:path*",
        destination: "http://localhost:8000/api/:path*",
      },
    ];
  },

  // Webpack 配置
  webpack: (config, { dev, isServer }) => {
    if (dev && !isServer) {
      config.watchOptions = {
        poll: 1000,
        aggregateTimeout: 300,
      };
    }
    return config;
  },

  // 实验性功能
  experimental: {
    profiling: true,
    logging: {
      level: "verbose",
      fullUrl: true,
    },
  },
};
```

### 案例 2：多环境配置

```javascript
// next.config.js
const isDev = process.env.NODE_ENV === "development";

module.exports = {
  reactStrictMode: true,

  // 开发环境特定配置
  ...(isDev && {
    devIndicators: {
      buildActivity: true,
    },
    onDemandEntries: {
      maxInactiveAge: 60 * 1000,
      pagesBufferLength: 5,
    },
  }),

  // 代理配置（仅开发环境）
  async rewrites() {
    if (!isDev) return [];

    return [
      {
        source: "/api/:path*",
        destination: "http://localhost:8000/api/:path*",
      },
    ];
  },
};
```

## 性能优化

### 构建性能优化

```javascript
// next.config.js
module.exports = {
  // 开发时的性能优化
```

开发时注意端口占用：

```bash
# 检查端口占用
lsof -i :3000

# 使用其他端口
PORT=3001 npm run dev
```

确保环境变量正确加载：

```bash
# .env.development
NODE_ENV=development
NEXT_PUBLIC_API_URL=http://localhost:3000/api
```

遇到缓存问题时清理：

```bash
# 清理 Next.js 缓存
rm -rf .next

# 清理 node_modules
rm -rf node_modules
```

热更新不工作时检查：

## 常见问题

### 1. 开发服务器启动慢？

**问题**：`npm run dev` 启动时间过长

**解决方案**：

```bash
# 使用 Turbopack
npm run dev --turbo

# 增加 Node.js 内存
NODE_OPTIONS='--max-old-space-size=4096' npm run dev

# 清理缓存
rm -rf .next node_modules/.cache
```

### 2. 热更新不工作？

**问题**：修改代码后页面不自动刷新

**解决方案**：

```javascript
// next.config.js
module.exports = {
  webpack: (config, { dev, isServer }) => {
    if (dev && !isServer) {
      config.watchOptions = {
        poll: 1000,
        aggregateTimeout: 300,
      };
    }
    return config;
  },
};
```

### 3. 端口被占用？

**问题**：Error: listen EADDRINUSE: address already in use :::3000

**解决方案**：

```bash
# 查找占用端口的进程
lsof -i :3000

# 杀死进程
kill -9 <PID>

# 或使用其他端口
PORT=3001 npm run dev
```

### 4. 环境变量不生效？

**问题**：process.env.XXX 返回 undefined

**解决方案**：

```bash
# 1. 检查文件名
.env.development  # 正确
.env.dev          # 错误

# 2. 重启开发服务器
npm run dev

# 3. 客户端变量需要 NEXT_PUBLIC_ 前缀
NEXT_PUBLIC_API_URL=http://localhost:3000/api
```

### 5. TypeScript 类型错误？

**问题**：TypeScript 报错但代码正确

**解决方案**：

```bash
# 重启 TypeScript 服务器
# VSCode: Cmd+Shift+P -> TypeScript: Restart TS Server

# 清理类型缓存
rm -rf .next/types
npm run dev
```

### 6. 模块找不到？

**问题**：Cannot find module '@/components/...'

**解决方案**：

```json
// tsconfig.json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

## 总结

开发环境配置是 Next.js 项目的基础。合理的配置可以：

1. **提升效率**：快速启动、热更新、自动化工具
2. **改善体验**：详细错误提示、调试工具、代码提示
3. **保证质量**：代码规范、类型检查、自动测试
4. **团队协作**：统一配置、版本控制、文档化

关键要点：

- 使用 `.env.development` 管理开发环境变量
- 配置 VSCode 实现保存时自动格式化
- 使用 Docker Compose 管理本地服务
- 启用 TypeScript 严格模式
- 配置 ESLint 和 Prettier
- 使用 Turbopack 加快开发速度
- 合理配置热更新和缓存
- 集成调试工具

记住：好的开发环境配置可以让开发过程更顺畅，减少问题，提高生产力。
