# 数据获取错误处理

## 1. 概述与背景

### 1.1 什么是数据获取错误处理

数据获取错误处理是指在 Next.js 16 应用中,当数据获取失败时如何优雅地处理错误、提供用户反馈、实现重试机制和降级方案的完整策略。

错误处理的核心特点:

- **错误捕获**: 捕获各种类型的错误
- **用户反馈**: 提供清晰的错误信息
- **重试机制**: 自动或手动重试
- **降级方案**: 提供备用数据或 UI
- **错误日志**: 记录错误用于分析

### 1.2 为什么需要错误处理

在数据获取过程中,错误是不可避免的:

- **网络问题**: 网络不稳定或断开
- **服务器错误**: API 返回 500 错误
- **超时**: 请求超时
- **数据格式**: 返回数据格式错误
- **权限问题**: 认证或授权失败

### 1.3 错误处理策略对比

**对比表格**:

| 策略           | 实现难度 | 用户体验 | 适用场景   | 性能影响 |
| :------------- | :------- | :------- | :--------- | :------- |
| Error Boundary | 简单     | 好       | 组件级错误 | 低       |
| Try-Catch      | 简单     | 中等     | 函数级错误 | 低       |
| 重试机制       | 中等     | 好       | 临时性错误 | 中等     |
| 降级方案       | 复杂     | 很好     | 关键功能   | 低       |

## 2. 核心概念

### 2.1 Error Boundary

#### 基本实现

```typescript
"use client";

import { Component, ReactNode } from "react";

interface Props {
  children: ReactNode;
  fallback?: ReactNode;
}

interface State {
  hasError: boolean;
  error: Error | null;
}

export class ErrorBoundary extends Component<Props, State> {
  constructor(props: Props) {
    super(props);
    this.state = { hasError: false, error: null };
  }

  static getDerivedStateFromError(error: Error): State {
    return { hasError: true, error };
  }

  componentDidCatch(error: Error, errorInfo: any) {
    console.error("Error caught by boundary:", error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      return (
        this.props.fallback || (
          <div>
            <h2>Something went wrong</h2>
            <p>{this.state.error?.message}</p>
            <button
              onClick={() => this.setState({ hasError: false, error: null })}
            >
              Try again
            </button>
          </div>
        )
      );
    }

    return this.props.children;
  }
}
```

### 3.3 重试机制

#### 自动重试

```typescript
"use client";

import { useState } from "react";

interface RetryOptions {
  maxRetries: number;
  retryDelay: number;
  backoff?: boolean;
}

async function fetchWithRetry(
  url: string,
  options: RetryOptions = { maxRetries: 3, retryDelay: 1000 }
): Promise<Response> {
  let lastError: Error | null = null;

  for (let i = 0; i <= options.maxRetries; i++) {
    try {
      const response = await fetch(url);

      if (response.ok) {
        return response;
      }

      // 对于 5xx 错误重试,4xx 错误不重试
      if (response.status >= 500 && i < options.maxRetries) {
        const delay = options.backoff
          ? options.retryDelay * Math.pow(2, i)
          : options.retryDelay;

        await new Promise((resolve) => setTimeout(resolve, delay));
        continue;
      }

      throw new Error(`HTTP error! status: ${response.status}`);
    } catch (error) {
      lastError = error instanceof Error ? error : new Error("Unknown error");

      if (i < options.maxRetries) {
        const delay = options.backoff
          ? options.retryDelay * Math.pow(2, i)
          : options.retryDelay;

        await new Promise((resolve) => setTimeout(resolve, delay));
      }
    }
  }

  throw lastError || new Error("Max retries exceeded");
}

export default function RetryExample() {
  const [data, setData] = useState(null);
  const [error, setError] = useState<string | null>(null);
  const [loading, setLoading] = useState(false);
  const [retryCount, setRetryCount] = useState(0);

  const fetchData = async () => {
    try {
      setLoading(true);
      setError(null);
      setRetryCount(0);

      const response = await fetchWithRetry("/api/unreliable", {
        maxRetries: 3,
        retryDelay: 1000,
        backoff: true,
      });

      const json = await response.json();
      setData(json);
    } catch (err) {
      setError(err instanceof Error ? err.message : "Unknown error");
    } finally {
      setLoading(false);
    }
  };

  return (
    <div>
      <button onClick={fetchData} disabled={loading}>
        {loading ? "Loading..." : "Fetch Data"}
      </button>
      {retryCount > 0 && <div>Retry attempt: {retryCount}</div>}
      {error && <div style={{ color: "red" }}>Error: {error}</div>}
      {data && <pre>{JSON.stringify(data, null, 2)}</pre>}
    </div>
  );
}
```

### 4.2 错误处理配置

#### 全局错误处理器

```typescript
interface ErrorHandlerConfig {
  onError?: (error: Error) => void;
  retry?: {
    maxRetries: number;
    retryDelay: number;
    backoff: boolean;
  };
  fallback?: any;
  timeout?: number;
}

class ErrorHandler {
  private config: ErrorHandlerConfig;

  constructor(config: ErrorHandlerConfig = {}) {
    this.config = {
      retry: {
        maxRetries: 3,
        retryDelay: 1000,
        backoff: true,
      },
      timeout: 30000,
      ...config,
    };
  }

  async handle<T>(
    fn: () => Promise<T>,
    options?: Partial<ErrorHandlerConfig>
  ): Promise<T> {
    const mergedConfig = { ...this.config, ...options };

    try {
      return await this.executeWithRetry(fn, mergedConfig);
    } catch (error) {
      this.config.onError?.(
        error instanceof Error ? error : new Error("Unknown error")
      );

      if (mergedConfig.fallback !== undefined) {
        return mergedConfig.fallback;
      }

      throw error;
    }
  }

  private async executeWithRetry<T>(
    fn: () => Promise<T>,
    config: ErrorHandlerConfig
  ): Promise<T> {
    let lastError: Error | null = null;
    const maxRetries = config.retry?.maxRetries || 0;

    for (let i = 0; i <= maxRetries; i++) {
      try {
        return await fn();
      } catch (error) {
        lastError = error instanceof Error ? error : new Error("Unknown error");

        if (i < maxRetries) {
          const delay = config.retry?.backoff
            ? (config.retry.retryDelay || 1000) * Math.pow(2, i)
            : config.retry?.retryDelay || 1000;

          await new Promise((resolve) => setTimeout(resolve, delay));
        }
      }
    }

    throw lastError;
  }
}

export const errorHandler = new ErrorHandler({
  onError: (error) => {
    console.error("Global error:", error);
    // 发送到错误追踪服务
  },
});
```

```typescript
"use client";

import { useState, useCallback } from "react";

interface UseAsyncOptions<T> {
  onSuccess?: (data: T) => void;
  onError?: (error: Error) => void;
  retry?: number;
  retryDelay?: number;
}

export function useAsync<T>(
  asyncFunction: () => Promise<T>,
  options: UseAsyncOptions<T> = {}
) {
  const [data, setData] = useState<T | null>(null);
  const [error, setError] = useState<Error | null>(null);
  const [loading, setLoading] = useState(false);

  const execute = useCallback(async () => {
    setLoading(true);
    setError(null);

    let lastError: Error | null = null;
    const maxRetries = options.retry || 0;

    for (let i = 0; i <= maxRetries; i++) {
      try {
        const result = await asyncFunction();
        setData(result);
        options.onSuccess?.(result);
        setLoading(false);
        return result;
      } catch (err) {
        lastError = err instanceof Error ? err : new Error("Unknown error");

        if (i < maxRetries) {
          await new Promise((resolve) =>
            setTimeout(resolve, options.retryDelay || 1000)
          );
        }
      }
    }

    setError(lastError);
    options.onError?.(lastError!);
    setLoading(false);
    throw lastError;
  }, [asyncFunction, options]);

  return { data, error, loading, execute };
}

// 使用示例
export default function AsyncExample() {
  const { data, error, loading, execute } = useAsync(
    async () => {
      const response = await fetch("/api/data");
      if (!response.ok) throw new Error("Failed to fetch");
      return response.json();
    },
    {
      retry: 3,
      retryDelay: 1000,
      onSuccess: (data) => console.log("Success:", data),
      onError: (error) => console.error("Error:", error),
    }
  );

  return (
    <div>
      <button onClick={execute} disabled={loading}>
        {loading ? "Loading..." : "Fetch Data"}
      </button>
      {error && <div style={{ color: "red" }}>{error.message}</div>}
```

```typescript
interface ErrorLog {
  timestamp: number;
  error: Error;
  context?: any;
  userAgent: string;
  url: string;
}

class ErrorLogger {
  private logs: ErrorLog[] = [];
  private maxLogs = 100;

  log(error: Error, context?: any): void {
    const log: ErrorLog = {
      timestamp: Date.now(),
      error,
      context,
      userAgent: navigator.userAgent,
      url: window.location.href,
    };

    this.logs.push(log);

    if (this.logs.length > this.maxLogs) {
      this.logs.shift();
    }

    // 发送到服务器
    this.sendToServer(log);
  }

  private async sendToServer(log: ErrorLog): Promise<void> {
    try {
      await fetch("/api/error-log", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          message: log.error.message,
          stack: log.error.stack,
          timestamp: log.timestamp,
          context: log.context,
          userAgent: log.userAgent,
          url: log.url,
        }),
      });
    } catch (err) {
      console.error("Failed to send error log:", err);
    }
  }

  getLogs(): ErrorLog[] {
    return this.logs;
  }

  clearLogs(): void {
    this.logs = [];
  }
}
```

```typescript
"use client";

export function classifyError(error: unknown): {
  type: "network" | "timeout" | "server" | "client" | "unknown";
  message: string;
  retryable: boolean;
} {
  if (error instanceof TypeError && error.message.includes("fetch")) {
    return {
      type: "network",
      message: "Network error. Please check your connection.",
      retryable: true,
    };
  }

  if (error instanceof Error && error.name === "AbortError") {
    return {
      type: "timeout",
      message: "Request timeout. Please try again.",
      retryable: true,
    };
  }

  if (error instanceof Response) {
    if (error.status >= 500) {
      return {
        type: "server",
        message: "Server error. Please try again later.",
        retryable: true,
      };
    }

    if (error.status >= 400) {
      return {
        type: "client",
        message: "Invalid request. Please check your input.",
        retryable: false,
      };
    }
  }

  return {
    type: "unknown",
    message: "An unexpected error occurred.",
    retryable: false,
  };
}
```

```typescript
"use client";

class ErrorMonitor {
  private errorCount = 0;
  private requestCount = 0;
  private startTime = Date.now();

  recordRequest(): void {
    this.requestCount++;
  }

  recordError(): void {
    this.errorCount++;
  }

  getErrorRate(): number {
    if (this.requestCount === 0) return 0;
    return (this.errorCount / this.requestCount) * 100;
  }

  getStats() {
    const uptime = Date.now() - this.startTime;
    return {
      errorCount: this.errorCount,
      requestCount: this.requestCount,
      errorRate: this.getErrorRate(),
      uptime,
    };
  }

  reset(): void {
    this.errorCount = 0;
    this.requestCount = 0;
    this.startTime = Date.now();
  }
}

export const errorMonitor = new ErrorMonitor();

// 使用示例
export async function monitoredFetch(url: string) {
  errorMonitor.recordRequest();

  try {
    const response = await fetch(url);
    if (!response.ok) {
      errorMonitor.recordError();
      throw new Error(`HTTP ${response.status}`);
    }
    return response;
  } catch (error) {
    errorMonitor.recordError();
    throw error;
  }
}
```

## 7. 常见问题

### 7.1 基础问题

#### 问题一: 如何区分不同类型的错误?

**问题**: 网络错误、服务器错误、客户端错误如何区分?

**简短回答**: 检查错误类型和 HTTP 状态码。

**详细解释**:

网络错误通常是 TypeError,服务器错误是 5xx 状态码,客户端错误是 4xx 状态码。根据不同类型采取不同处理策略。

**代码示例**:

```typescript
if (error instanceof TypeError) {
  // 网络错误
} else if (response.status >= 500) {
  // 服务器错误
} else if (response.status >= 400) {
  // 客户端错误
}
```

#### 问题二: 何时应该重试请求?

**问题**: 哪些错误应该自动重试?

**简短回答**: 临时性错误(网络错误、5xx 错误)应该重试,永久性错误(4xx 错误)不应该重试。

**详细解释**:

网络错误和服务器错误通常是临时的,可以重试。客户端错误(如 400、401、404)通常是永久的,重试也不会成功。

**代码示例**:

```typescript
if (response.status >= 500 || error instanceof TypeError) {
  // 可以重试
  await retry();
} else if (response.status >= 400) {
  // 不应该重试
  throw error;
}
```

#### 问题三: 如何提供用户友好的错误消息?

**问题**: 如何将技术错误转换为用户能理解的消息?

**简短回答**: 创建错误消息映射表,将技术错误转换为友好消息。

**详细解释**:

维护一个错误代码到用户消息的映射表,根据错误类型返回相应的友好消息。

**代码示例**:

```typescript
const messages = {
  NETWORK_ERROR: "网络连接失败",
  SERVER_ERROR: "服务器错误",
};

const message = messages[errorCode] || "发生错误";
```

### 7.2 进阶问题

#### 问题四: 如何实现全局错误处理?

**问题**: 如何统一处理应用中的所有错误?

**简短回答**: 使用 Error Boundary 和全局错误处理器。

**详细解释**:

在应用顶层使用 Error Boundary 捕获组件错误,创建全局错误处理器处理异步错误,统一错误日志和用户反馈。

**代码示例**:

```typescript
// 顶层组件
<ErrorBoundary>
  <App />
</ErrorBoundary>;

// 全局错误处理
window.addEventListener("unhandledrejection", (event) => {
  errorHandler.handle(event.reason);
});
```

#### 问题五: 如何监控错误率?

**问题**: 如何追踪应用的错误率和趋势?

**简短回答**: 记录每次请求和错误,计算错误率并发送到监控服务。

**详细解释**:

维护请求计数和错误计数,定期计算错误率。将错误信息发送到监控服务(如 Sentry)进行分析。

**代码示例**:

```typescript
const errorRate = (errorCount / requestCount) * 100;

if (errorRate > 5) {
  // 错误率过高,发送告警
  sendAlert({ errorRate, timestamp: Date.now() });
}
```
