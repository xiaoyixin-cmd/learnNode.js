# SWR 数据获取库详解

## 1. 概述与背景

### 1.1 什么是 SWR

SWR (stale-while-revalidate) 是 Vercel 开发的 React Hooks 数据获取库。它的名字来自 HTTP 缓存失效策略,先返回缓存数据,然后在后台重新验证,最后更新为最新数据。

SWR 的核心特点:

- **轻量简洁**: API 简单易用
- **自动重新验证**: 智能刷新数据
- **缓存优先**: 立即显示缓存数据
- **实时体验**: 自动保持数据最新
- **TypeScript 支持**: 完整的类型定义

### 1.2 为什么使用 SWR

SWR 专为 Next.js 设计,与 Next.js 配合使用体验最佳:

- **简单**: API 设计简洁直观
- **快速**: 缓存优先策略提升性能
- **实时**: 自动重新验证保持数据新鲜
- **离线**: 支持离线模式
- **分页**: 内置分页和无限加载

### 1.3 SWR vs React Query

**对比表格**:

| 特性       | SWR      | React Query |
| :--------- | :------- | :---------- |
| 包大小     | 小 (4KB) | 大 (13KB)   |
| API 复杂度 | 简单     | 复杂        |
| 功能丰富度 | 基础     | 丰富        |
| DevTools   | 无       | 有          |
| 学习曲线   | 平缓     | 陡峭        |
| 适用场景   | 简单项目 | 复杂项目    |

### 5.2 进阶使用

#### 无限加载

```typescript
"use client";

import useSWRInfinite from "swr/infinite";

interface Post {
  id: string;
  title: string;
}

const fetcher = (url: string) => fetch(url).then((res) => res.json());

const getKey = (pageIndex: number, previousPageData: Post[] | null) => {
  if (previousPageData && !previousPageData.length) return null;
  return `/api/posts?page=${pageIndex + 1}&limit=10`;
};

export default function InfinitePosts() {
  const { data, size, setSize, isLoading } = useSWRInfinite<Post[]>(
    getKey,
    fetcher
  );

  const posts = data ? data.flat() : [];
  const isLoadingMore =
    isLoading || (size > 0 && data && typeof data[size - 1] === "undefined");
  const isEmpty = data?.[0]?.length === 0;
  const isReachingEnd = isEmpty || (data && data[data.length - 1]?.length < 10);

  return (
    <div>
      {posts.map((post) => (
        <div key={post.id}>{post.title}</div>
      ))}

      {!isReachingEnd && (
        <button onClick={() => setSize(size + 1)} disabled={isLoadingMore}>
          {isLoadingMore ? "Loading..." : "Load More"}
        </button>
      )}
    </div>
  );
}
```

#### 乐观更新

```typescript
"use client";

import useSWR, { useSWRConfig } from "swr";

interface Post {
  id: string;
  title: string;
  likes: number;
}

const fetcher = (url: string) => fetch(url).then((res) => res.json());

export default function PostWithLike({ postId }: { postId: string }) {
  const { data: post } = useSWR<Post>(`/api/posts/${postId}`, fetcher);
  const { mutate } = useSWRConfig();

  const handleLike = async () => {
    if (!post) return;

    // 乐观更新
    mutate(`/api/posts/${postId}`, { ...post, likes: post.likes + 1 }, false);

    try {
      await fetch(`/api/posts/${postId}/like`, { method: "POST" });
      // 重新验证
      mutate(`/api/posts/${postId}`);
    } catch (error) {
      // 失败时回滚
      mutate(`/api/posts/${postId}`);
    }
  };

  return (
    <div>
      <h2>{post?.title}</h2>
      <button onClick={handleLike}>Like ({post?.likes || 0})</button>
    </div>
  );
}
```

#### 组件级错误处理

```typescript
"use client";

import useSWR from "swr";

const fetcher = (url: string) =>
  fetch(url).then((res) => {
    if (!res.ok) {
      const error = new Error("An error occurred while fetching the data.");
      error.info = res.statusText;
      error.status = res.status;
      throw error;
    }
    return res.json();
  });

export default function PostWithError({ id }: { id: string }) {
  const { data, error } = useSWR(`/api/posts/${id}`, fetcher, {
    onErrorRetry: (error, key, config, revalidate, { retryCount }) => {
      // 404 不重试
      if (error.status === 404) return;

      // 最多重试 3 次
      if (retryCount >= 3) return;

      // 1 秒后重试
      setTimeout(() => revalidate({ retryCount }), 1000);
    },
  });

  if (error) return <div>Error: {error.message}</div>;
  if (!data) return <div>Loading...</div>;

  return <div>{data.title}</div>;
}
```

## 7. 常见问题

### 7.1 基础问题

#### 问题一: SWR 和 React Query 应该选哪个?

**问题**: 什么时候用 SWR,什么时候用 React Query?

**简短回答**: 简单项目用 SWR,复杂项目用 React Query。

**详细解释**:

SWR 更轻量简单,适合简单的数据获取场景。React Query 功能更强大,适合复杂的数据管理需求。

**代码示例**:

```typescript
// SWR - 简单直接
import useSWR from "swr";

const { data } = useSWR("/api/posts", fetcher);

// React Query - 功能丰富
import { useQuery } from "@tanstack/react-query";

const { data } = useQuery({
  queryKey: ["posts"],
  queryFn: fetchPosts,
  staleTime: 5000,
});
```

#### 问题二: 如何禁用自动重新验证?

**问题**: 如何阻止 SWR 自动刷新数据?

**简短回答**: 设置 revalidate 相关选项为 false。

**详细解释**:

SWR 默认在窗口聚焦、重新连接时刷新数据。可以通过配置禁用这些行为。

**代码示例**:

```typescript
const { data } = useSWR("/api/data", fetcher, {
  revalidateOnFocus: false,
  revalidateOnReconnect: false,
  revalidateOnMount: false,
});
```

#### 问题三: 如何设置轮询?

**问题**: 如何让 SWR 定期刷新数据?

**简短回答**: 使用 refreshInterval 选项。

**详细解释**:

refreshInterval 设置轮询间隔,单位是毫秒。设置为 0 禁用轮询。

**代码示例**:

```typescript
const { data } = useSWR("/api/stats", fetcher, {
  refreshInterval: 5000, // 每 5 秒刷新一次
  refreshWhenHidden: false, // 隐藏时不刷新
  refreshWhenOffline: false, // 离线时不刷新
});
```

### 7.2 进阶问题

#### 问题四: 如何实现乐观更新?

**问题**: 如何在请求完成前更新 UI?

**简短回答**: 使用 mutate 函数的第二个参数。

**详细解释**:

mutate 的第二个参数设置为 false 可以立即更新本地数据,不触发重新验证。

**代码示例**:

```typescript
const { data, mutate } = useSWR("/api/user", fetcher);

const updateName = async (newName: string) => {
  // 乐观更新
  mutate({ ...data, name: newName }, false);

  try {
    await fetch("/api/user", {
      method: "PUT",
      body: JSON.stringify({ name: newName }),
    });
    // 成功后重新验证
    mutate();
  } catch (error) {
    // 失败时回滚
    mutate();
  }
};
```

#### 问题五: 如何处理分页数据?

**问题**: 如何实现分页或无限滚动?

**简短回答**: 普通分页用 useSWR,无限滚动用 useSWRInfinite。

**详细解释**:

普通分页每页独立缓存,无限滚动使用 useSWRInfinite 累积数据。

**代码示例**:

```typescript
// 普通分页
const [page, setPage] = useState(1);
const { data } = useSWR(`/api/posts?page=${page}`, fetcher);

// 无限滚动
import useSWRInfinite from "swr/infinite";

const { data, size, setSize } = useSWRInfinite(
  (index) => `/api/posts?page=${index + 1}`,
  fetcher
);
```

#### 问题六: 如何预取数据?

**问题**: 如何在用户操作前预加载数据?

**简短回答**: 使用 preload 函数。

**详细解释**:

preload 函数可以在用户操作前预加载数据到缓存。

**代码示例**:

```typescript
import { preload } from "swr";

const prefetchPost = (id: string) => {
  preload(`/api/posts/${id}`, fetcher);
};

<li onMouseEnter={() => prefetchPost(post.id)}>{post.title}</li>;
```

#### 问题七: 如何处理依赖请求?

**问题**: 如何在一个请求完成后再发起另一个请求?

**简短回答**: 使用条件 key。

**详细解释**:

将 key 设置为 null 可以阻止请求,等依赖数据准备好后再发起请求。

**代码示例**:

```typescript
const { data: user } = useSWR("/api/user", fetcher);

const { data: posts } = useSWR(
  user ? `/api/users/${user.id}/posts` : null,
  fetcher
);
```
