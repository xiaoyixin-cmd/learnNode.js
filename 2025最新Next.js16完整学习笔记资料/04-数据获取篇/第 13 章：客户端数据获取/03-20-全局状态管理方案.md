# 全局状态管理方案

## 1. 概述与背景

### 1.1 什么是全局状态管理

全局状态管理是在 React 应用中跨组件共享和管理状态的方案。在 Next.js 16 中,由于服务端组件和客户端组件的分离,状态管理变得更加重要。

全局状态管理的核心特点:

- **跨组件共享**: 多个组件访问同一状态
- **集中管理**: 统一管理应用状态
- **可预测性**: 状态变化可追踪
- **性能优化**: 避免不必要的重新渲染
- **开发体验**: 提供调试工具

### 1.2 为什么需要全局状态管理

在 Next.js 应用中,某些状态需要在多个组件间共享:

- **用户信息**: 登录状态、用户资料
- **主题设置**: 深色/浅色模式
- **购物车**: 商品列表、总价
- **通知**: 全局提示消息
- **UI 状态**: 侧边栏展开/收起

### 1.3 状态管理方案对比

**对比表格**:

| 方案          | 包大小    | 学习曲线 | 性能 | DevTools | 适用场景   |

### 3.2 主题切换

#### Context API 实现

```typescript
// providers/ThemeProvider.tsx
"use client";

import {
  createContext,
  useContext,
  useState,
  useEffect,
  ReactNode,
} from "react";

type Theme = "light" | "dark";

interface ThemeContextType {
  theme: Theme;
  toggleTheme: () => void;
}

const ThemeContext = createContext<ThemeContextType | undefined>(undefined);

export function ThemeProvider({ children }: { children: ReactNode }) {
  const [theme, setTheme] = useState<Theme>("light");

  useEffect(() => {
    const savedTheme = localStorage.getItem("theme") as Theme;
    if (savedTheme) {
      setTheme(savedTheme);
    }
  }, []);

  useEffect(() => {
    document.documentElement.classList.toggle("dark", theme === "dark");
    localStorage.setItem("theme", theme);
  }, [theme]);

  const toggleTheme = () => {
    setTheme((prev) => (prev === "light" ? "dark" : "light"));
  };

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

export function useTheme() {
  const context = useContext(ThemeContext);
  if (context === undefined) {
    throw new Error("useTheme must be used within a ThemeProvider");
  }
  return context;
}
```

```typescript
"use client";

import { useTheme } from "@/providers/ThemeProvider";

  const { theme, toggleTheme } = useTheme();

  return (

  );
}
```

### 3.3 购物车管理

#### Zustand 实现

```typescript
// stores/cartStore.ts
import { create } from "zustand";

interface CartItem {
  id: string;
  name: string;
  price: number;
  quantity: number;
}

interface CartStore {
  items: CartItem[];
  total: number;
  addItem: (item: Omit<CartItem, "quantity">) => void;
  removeItem: (id: string) => void;
  updateQuantity: (id: string, quantity: number) => void;
  clearCart: () => void;
}

export const useCartStore = create<CartStore>((set, get) => ({
  items: [],
  total: 0,

  addItem: (item) =>
    set((state) => {
      const existingItem = state.items.find((i) => i.id === item.id);

      if (existingItem) {
        return {
          items: state.items.map((i) =>
            i.id === item.id ? { ...i, quantity: i.quantity + 1 } : i
          ),
          total: state.total + item.price,
        };
      }

      return {
        items: [...state.items, { ...item, quantity: 1 }],
        total: state.total + item.price,
      };
    }),

  removeItem: (id) =>
    set((state) => {
      const item = state.items.find((i) => i.id === id);
      if (!item) return state;

      return {
        items: state.items.filter((i) => i.id !== id),
        total: state.total - item.price * item.quantity,
      };
    }),

  updateQuantity: (id, quantity) =>
    set((state) => {
      const item = state.items.find((i) => i.id === id);
      if (!item) return state;

      const diff = quantity - item.quantity;

      return {
        items: state.items.map((i) => (i.id === id ? { ...i, quantity } : i)),
        total: state.total + item.price * diff,
      };
    }),

  clearCart: () => set({ items: [], total: 0 }),
}));
```

#### 使用示例

```typescript
"use client";

import { useCartStore } from "@/stores/cartStore";

export default function Cart() {
  const { items, total, removeItem, updateQuantity } = useCartStore();

  return (
    <div>
      <h2>Shopping Cart</h2>
      {items.map((item) => (
        <div key={item.id}>
          <h3>{item.name}</h3>
          <p>${item.price}</p>
          <input
            type="number"
            value={item.quantity}
            onChange={(e) => updateQuantity(item.id, parseInt(e.target.value))}
          />
          <button onClick={() => removeItem(item.id)}>Remove</button>
        </div>
      ))}
      <p>Total: ${total.toFixed(2)}</p>
    </div>
  );
}
```

### 3.4 通知系统

#### Zustand 实现

```typescript
// stores/notificationStore.ts
import { create } from "zustand";

interface Notification {
  id: string;
  message: string;
  type: "success" | "error" | "info" | "warning";
  duration?: number;
}

interface NotificationStore {
  notifications: Notification[];
  addNotification: (notification: Omit<Notification, "id">) => void;
  removeNotification: (id: string) => void;
}

export const useNotificationStore = create<NotificationStore>((set) => ({
  notifications: [],

  addNotification: (notification) => {
    const id = Math.random().toString(36).substr(2, 9);
    const newNotification = { ...notification, id };

    set((state) => ({
      notifications: [...state.notifications, newNotification],
    }));

    if (notification.duration !== 0) {
      setTimeout(() => {
        set((state) => ({
          notifications: state.notifications.filter((n) => n.id !== id),
        }));
      }, notification.duration || 3000);
    }
  },

  removeNotification: (id) =>
    set((state) => ({
      notifications: state.notifications.filter((n) => n.id !== id),
    })),
}));
```

## 7. 常见问题

### 7.1 基础问题

#### 问题一: 应该选择哪个状态管理方案?

**问题**: Context API、Zustand、Redux Toolkit 应该选哪个?

**简短回答**: 简单状态用 Context API,中小型应用用 Zustand,大型应用用 Redux Toolkit。

**详细解释**:

Context API 适合简单的全局状态,如主题、语言。Zustand 轻量且易用,适合大多数场景。Redux Toolkit 功能强大,适合复杂的大型应用。

**代码示例**:

```typescript
// Context API - 简单主题切换
const ThemeContext = createContext();

// Zustand - 用户状态管理
const useUserStore = create((set) => ({
  user: null,
  setUser: (user) => set({ user }),
}));

// Redux Toolkit - 复杂应用状态
const store = configureStore({
  reducer: {
    user: userReducer,
    posts: postsReducer,
    comments: commentsReducer,
  },
});
```

#### 问题二: 如何在服务端组件中使用状态?

**问题**: Next.js 16 的服务端组件能用状态管理吗?

**简短回答**: 不能,状态管理只能在客户端组件中使用。

**详细解释**:

服务端组件不支持状态和副作用。需要状态管理的组件必须标记为 'use client'。

**代码示例**:

```typescript
// 服务端组件 - 不能使用状态
export default async function ServerPage() {
  const data = await fetchData();
  return <div>{data}</div>;
}

// 客户端组件 - 可以使用状态
("use client");

import { useUserStore } from "@/stores/userStore";

export default function ClientPage() {
  const user = useUserStore((state) => state.user);
  return <div>{user?.name}</div>;
}
```

#### 问题三: 如何持久化状态?

**问题**: 如何让状态在页面刷新后保留?

**简短回答**: 使用 persist 中间件。

**详细解释**:

Zustand 提供 persist 中间件,可以将状态保存到 localStorage 或 sessionStorage。

**代码示例**:

```typescript
import { create } from "zustand";
import { persist } from "zustand/middleware";

const useStore = create(
  persist(
    (set) => ({
      user: null,
      setUser: (user) => set({ user }),
    }),
    {
      name: "user-storage",
    }
  )
);
```

### 7.2 进阶问题

#### 问题四: 如何处理异步操作?

**问题**: 如何在状态管理中处理异步请求?

**简短回答**: 在 action 中使用 async/await。

**详细解释**:

Zustand 支持异步 action,可以直接使用 async/await。Redux Toolkit 使用 createAsyncThunk。

**代码示例**:

```typescript
// Zustand
const useStore = create((set) => ({
  data: null,
  loading: false,
  fetchData: async () => {
    set({ loading: true });
    const data = await fetch("/api/data").then((r) => r.json());
    set({ data, loading: false });
  },
}));

// Redux Toolkit
const fetchData = createAsyncThunk("data/fetch", async () => {
  const response = await fetch("/api/data");
  return response.json();
});
```

#### 问题五: 如何优化性能?

**问题**: 如何避免不必要的重新渲染?

**简短回答**: 使用选择器只订阅需要的数据。

**详细解释**:

不要订阅整个 store,只订阅组件需要的数据。使用 shallow 比较避免引用变化导致的重新渲染。

**代码示例**:

```typescript
// 不好 - 订阅整个 store
const store = useStore();

// 好 - 只订阅需要的数据
const userName = useStore((state) => state.user.name);

// 更好 - 使用 shallow 比较
const { name, email } = useStore(
  (state) => ({ name: state.user.name, email: state.user.email }),
  shallow
);
```

**状态管理的主要目的**:

4. **避免过度使用**: 不是所有状态都需要全局管理
