# 缓存生命周期管理

## 1. 概述与背景

### 1.1 什么是缓存生命周期

缓存生命周期指的是缓存数据从创建、使用、更新到失效的整个过程。在 Next.js 16 中,缓存生命周期管理变得更加精细和可控,开发者可以通过多种方式控制缓存的生命周期。

缓存生命周期包含以下几个阶段:

1. **创建阶段**: 数据首次被获取并存入缓存
2. **有效阶段**: 缓存数据可以被直接使用
3. **过期阶段**: 缓存数据超过有效期,需要重新验证
4. **重新验证阶段**: 检查数据是否有更新
5. **失效阶段**: 缓存被清除或替换

### 1.2 为什么需要管理缓存生命周期

在实际应用中,不同类型的数据有不同的更新频率和重要性。如果不合理管理缓存生命周期,会导致:

- **数据过期**: 用户看到过时的信息
- **性能浪费**: 频繁请求不变的数据
- **内存占用**: 大量无用缓存占用内存
- **一致性问题**: 相关数据不同步

合理的缓存生命周期管理可以:

- 提升应用性能
- 减少服务器负载
- 保证数据新鲜度
- 优化用户体验

### 1.3 Next.js 16 的缓存生命周期特性

Next.js 16 提供了多种控制缓存生命周期的方式:

| 方式             | 作用               | 适用场景       |
| :--------------- | :----------------- | :------------- |
| `revalidate`     | 设置缓存过期时间   | 定期更新的数据 |
| `tags`           | 标签化缓存管理     | 按需更新的数据 |
| `revalidateTag`  | 按标签重新验证     | 内容管理系统   |
| `revalidatePath` | 按路径重新验证     | 页面级别更新   |
| `cacheLife`      | 精细化生命周期控制 | 复杂缓存策略   |
| `unstable_cache` | 自定义缓存函数     | 非 fetch 数据  |

## 2. 核心概念

### 2.1 缓存过期时间

缓存过期时间决定了缓存数据的有效期。Next.js 使用 `revalidate` 参数来设置过期时间。

#### 基本用法

```typescript
// app/page.tsx

// 路由级别的过期时间
export const revalidate = 3600; // 1小时

export default async function Page() {
  // 请求级别的过期时间
  const res = await fetch("https://api.example.com/data", {
    next: { revalidate: 60 }, // 60秒
  });

  return <div>{await res.text()}</div>;
}
```

#### 过期时间的优先级

1. fetch 请求的 `next.revalidate`
2. 路由段的 `revalidate` 配置
3. 默认值(Next.js 16 中是不缓存)

```typescript
// app/page.tsx

export const revalidate = 3600; // 路由级别: 1小时

export default async function Page() {
  // 这个请求使用 60 秒(优先级更高)
  const res1 = await fetch("https://api.example.com/data1", {
    next: { revalidate: 60 },
  });

  // 这个请求使用路由级别的 3600 秒
  const res2 = await fetch("https://api.example.com/data2");

  // 这个请求不缓存(优先级最高)
  const res3 = await fetch("https://api.example.com/data3", {
    cache: "no-store",
  });

  return <div>...</div>;
}
```

### 2.2 缓存标签

缓存标签允许你为缓存数据打上标记,然后可以按标签批量重新验证缓存。

#### 基本用法

```typescript
// app/posts/page.tsx

export default async function PostsPage() {
  const res = await fetch("https://api.example.com/posts", {
    next: {
      tags: ["posts", "content"],
      revalidate: 3600,
    },
  });

  const posts = await res.json();

  return (
    <div>
      {posts.map((post: any) => (
        <article key={post.id}>
          <h2>{post.title}</h2>
        </article>
      ))}
    </div>
  );
}
```

#### 按标签重新验证

```typescript
// app/actions.ts
```

### 2.4 缓存生命周期状态

缓存在其生命周期中会经历不同的状态:

| 状态         | 说明       | 行为                      |
| :----------- | :--------- | :------------------------ |
| MISS         | 缓存未命中 | 发起新请求,创建缓存       |
| HIT          | 缓存命中   | 直接返回缓存数据          |
| STALE        | 缓存过期   | 返回过期数据,后台重新验证 |
| REVALIDATING | 重新验证中 | 检查数据是否有更新        |
| UPDATING     | 更新中     | 获取新数据并更新缓存      |

#### 缓存状态转换

```
MISS → 发起请求 → 创建缓存 → HIT
  ↓
HIT → 时间到期 → STALE → 后台重新验证 → REVALIDATING
  ↓
REVALIDATING → 数据未变 → HIT
  ↓
REVALIDATING → 数据已变 → UPDATING → HIT
```

### 2.5 Stale-While-Revalidate 策略

Next.js 使用 Stale-While-Revalidate (SWR) 策略来平衡性能和数据新鲜度。

**工作原理**:

1. 缓存过期后,先返回过期的数据(Stale)
2. 同时在后台发起请求重新验证(Revalidate)
3. 下次请求时返回更新后的数据

```typescript
// app/page.tsx

export const revalidate = 60; // 60秒后过期

export default async function Page() {
  const res = await fetch("https://api.example.com/data", {
    next: { revalidate: 60 },
  });

  // 第一次请求: MISS,发起请求,创建缓存
  // 60秒内的请求: HIT,直接返回缓存
  // 60秒后的请求: STALE,返回过期数据,后台重新验证
  // 重新验证后的请求: HIT,返回新数据

  return <div>{await res.text()}</div>;
}
```

```typescript
  const authHeader = request.headers.get("authorization");
  if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return Response.json({ error: "Unauthorized" }, { status: 401 });
  }

  // 重新验证统计数据
  revalidateTag("stats");

  // 重新验证热门内容
  revalidateTag("trending");

  // 重新验证首页
  revalidatePath("/");

  return Response.json({
    revalidated: true,
    timestamp: new Date().toISOString(),
  });
}
```

## 6. 注意事项

### 6.1 性能影响

#### 过度重新验证

频繁的重新验证会导致性能下降:

```typescript
// ❌ 错误: 重新验证太频繁
export const revalidate = 1; // 每秒重新验证

export default async function Page() {
  const res = await fetch("https://api.example.com/data", {
    next: { revalidate: 1 },
  });

  // 服务器压力过大
  return <div>{await res.text()}</div>;
}

// ✅ 正确: 合理的重新验证时间
export const revalidate = 60; // 60秒

export default async function Page() {
  const res = await fetch("https://api.example.com/data", {
    next: { revalidate: 60 },
  });

  return <div>{await res.text()}</div>;
}
```

#### 缓存雪崩

大量缓存同时过期会导致缓存雪崩:

```typescript
// ❌ 错误: 所有缓存同时过期
export const revalidate = 3600; // 所有页面都是1小时

// ✅ 正确: 错开过期时间
export const revalidate = 3600 + Math.floor(Math.random() * 600); // 1小时 + 0-10分钟随机值
```

### 6.2 数据一致性

#### 相关数据不同步

更新数据时要确保相关缓存一起更新:

```typescript
// ❌ 错误: 只更新部分缓存
"use server";

import { revalidateTag } from "next/cache";

  await fetch(`https://api.example.com/products/${productId}`, {
    method: "PUT",
    body: JSON.stringify(data),
  });

  // 只更新产品缓存,忘记更新列表
  revalidateTag(`product-${productId}`);
}

// ✅ 正确: 更新所有相关缓存
("use server");

import { revalidateTag } from "next/cache";

export async function updateProduct(productId: string, data: any) {
  await fetch(`https://api.example.com/products/${productId}`, {
    method: "PUT",
    body: JSON.stringify(data),
  });

  // 更新所有相关缓存
  revalidateTag(`product-${productId}`);
  revalidateTag("products");
  revalidateTag(`category-${data.categoryId}`);
}
```

#### 标签命名冲突

避免标签命名冲突:

```typescript
// ❌ 错误: 标签命名太通用
const res = await fetch("https://api.example.com/posts/1", {
  next: { tags: ["data"] }, // 太通用,可能影响其他数据

// ✅ 正确: 使用具体的标签名
```

### 6.3 错误处理

#### 重新验证失败

重新验证可能失败,需要处理错误:

```typescript
// app/actions.ts
"use server";

import { revalidateTag } from "next/cache";

export async function updatePost(postId: string, data: any) {
  try {
    const res = await fetch(`https://api.example.com/posts/${postId}`, {
      method: "PUT",
      body: JSON.stringify(data),
    });

    if (!res.ok) {
      throw new Error("Failed to update post");
    }

    const post = await res.json();

    // 更新成功后才重新验证
    revalidateTag(`post-${postId}`);
    revalidateTag("posts");

    return { success: true, post };
  } catch (error) {
    console.error("Update failed:", error);
    return { success: false, error: error.message };
  }
}
```

#### 部分更新失败

处理部分更新失败的情况:

```typescript
// app/actions.ts
"use server";

import { revalidateTag } from "next/cache";

export async function batchUpdateProducts(products: any[]) {
  const results = [];
  const updatedIds = [];

  for (const product of products) {
    try {
      const res = await fetch(
        `https://api.example.com/products/${product.id}`,
        {
          method: "PUT",
          body: JSON.stringify(product),
        }
      );

      if (res.ok) {
        updatedIds.push(product.id);
        results.push({ id: product.id, success: true });
      } else {
        results.push({
          id: product.id,
          success: false,
          error: "Update failed",
        });
      }
    } catch (error) {
      results.push({ id: product.id, success: false, error: error.message });
    }
  }

  // 只重新验证成功更新的产品
  updatedIds.forEach((id) => revalidateTag(`product-${id}`));

  // 如果有任何更新成功,重新验证列表
  if (updatedIds.length > 0) {
    revalidateTag("products");
  }

  return results;
}
```

## 7. 常见问题

### 7.1 基础问题

#### 问题一: revalidate 和 cache 有什么区别?

**问题**: `revalidate` 和 `cache` 参数有什么区别?

**简短回答**: `cache` 控制是否缓存,`revalidate` 控制缓存多久后重新验证。

**详细解释**:

- `cache: 'no-store'`: 完全不缓存,每次都请求
- `cache: 'force-cache'`: 缓存数据
- `revalidate`: 在缓存的基础上,设置过期时间

**代码示例**:

```typescript
// 不缓存
const res1 = await fetch("/api/data", {
  cache: "no-store",
});

// 永久缓存
const res2 = await fetch("/api/data", {
  cache: "force-cache",
});

// 定时重新验证
const res3 = await fetch("/api/data", {
  next: { revalidate: 60 },
});
```

#### 问题二: 如何立即更新缓存?

**问题**: 数据更新后,如何立即更新缓存而不是等待过期?

**简短回答**: 使用 `revalidateTag` 或 `revalidatePath`。

**详细解释**:

使用标签化缓存,在数据更新时调用 `revalidateTag` 立即重新验证缓存。

**代码示例**:

```typescript
// 页面中使用标签
const res = await fetch("https://api.example.com/posts", {
  next: {
    tags: ["posts"],
    revalidate: 3600,
  },
});

// 更新数据时重新验证
("use server");

import { revalidateTag } from "next/cache";

export async function createPost(data: any) {
  await fetch("https://api.example.com/posts", {
    method: "POST",
    body: JSON.stringify(data),
  });

  // 立即重新验证
  revalidateTag("posts");
}
```

#### 问题三: 如何为不同环境设置不同的缓存时间?

**问题**: 开发环境和生产环境需要不同的缓存时间,如何配置?

**简短回答**: 使用环境变量。

**详细解释**:

通过环境变量控制缓存时间,在不同环境使用不同的值。

**代码示例**:

```typescript
// app/page.tsx

const REVALIDATE_TIME =
  process.env.NODE_ENV === "production"
    ? 3600 // 生产环境: 1小时
    : 10; // 开发环境: 10秒

export const revalidate = REVALIDATE_TIME;

export default async function Page() {
  const res = await fetch("https://api.example.com/data", {
    next: { revalidate: REVALIDATE_TIME },
  });

  return <div>{await res.text()}</div>;
}
```

#### 问题四: 如何实现缓存预热?

**问题**: 如何在部署时预先生成缓存?

**简短回答**: 使用 `generateStaticParams` 预生成页面。

**详细解释**:

在构建时使用 `generateStaticParams` 预生成静态页面,这些页面会被缓存。

#### 问题五: 如何处理缓存依赖?

**问题**: A 数据依赖 B 数据,B 更新时如何同时更新 A?

**简短回答**: 使用相同的标签或级联重新验证。

**详细解释**:

为相关数据使用相同的标签,或在更新时级联重新验证所有依赖的缓存。

**代码示例**:

```typescript
// 方案一: 使用相同标签
const categoryRes = await fetch(`https://api.example.com/categories/${id}`, {
  next: { tags: ["categories", `category-${id}`] },
});

const productsRes = await fetch(
  `https://api.example.com/categories/${id}/products`,
  {
    next: { tags: ["categories", `category-${id}`] }, // 使用相同标签
  }
);

// 更新分类时,两个缓存都会被重新验证
revalidateTag(`category-${id}`);

// 方案二: 级联重新验证
("use server");

import { revalidateTag } from "next/cache";

export async function updateCategory(id: string, data: any) {
  await fetch(`https://api.example.com/categories/${id}`, {
    method: "PUT",
    body: JSON.stringify(data),
  });

  // 重新验证分类
  revalidateTag(`category-${id}`);

  // 重新验证该分类的产品
  revalidateTag(`category-${id}-products`);
}
```

## 8. 总结

### 8.1 核心要点回顾

**缓存生命周期阶段**:

1. **创建**: 数据首次获取并缓存
2. **有效**: 缓存数据可直接使用
3. **过期**: 缓存超过有效期
4. **重新验证**: 检查数据是否更新
5. **失效**: 缓存被清除或替换

**生命周期管理方式**:

| 方式             | 作用       | 适用场景       |
| :--------------- | :--------- | :------------- |
| `revalidate`     | 定时过期   | 定期更新的数据 |
| `revalidateTag`  | 按标签更新 | 按需更新的数据 |
| `revalidatePath` | 按路径更新 | 页面级别更新   |
| `unstable_cache` | 自定义缓存 | 非 fetch 数据  |

### 8.2 关键收获

1. **理解生命周期**: 掌握缓存从创建到失效的完整过程
2. **选择合适方式**: 根据数据特性选择最优的管理方式
3. **标签化管理**: 使用标签实现精细化的缓存控制
4. **级联更新**: 确保相关数据的缓存一起更新
5. **错误处理**: 处理重新验证失败的情况

### 8.3 最佳实践

1. **合理设置过期时间**: 根据数据更新频率设置合适的 `revalidate` 值
2. **使用有意义的标签**: 标签命名要清晰,避免冲突
3. **级联重新验证**: 更新数据时重新验证所有相关缓存
4. **监控缓存状态**: 在开发环境中监控缓存命中率
5. **处理错误**: 重新验证失败时要有降级方案
