# 编译错误处理与调试

## 概述

React Compiler 在编译过程中可能会遇到各种错误和警告,理解这些错误信息并掌握调试技巧对于成功使用编译器至关重要。本文将详细介绍 React Compiler 的错误类型、错误处理机制、调试方法和最佳实践,帮助开发者快速定位和解决编译问题,确保应用能够正确编译和优化。

**编译错误类型**:

- 语法错误(Syntax Errors)

- 优化失败(Optimization Failures)

## 第一部分:常见错误

### 1.1 规则违反错误

**直接修改 Props**

```typescript
// ❌ 错误:直接修改 props
function Component({ user }) {
  // 编译器错误:不能修改 props
  user.name = "New Name";

  return <div>{user.name}</div>;
}

// 编译器错误信息:
// Error: Cannot mutate props directly
// Component: Component
// Location: user.name = 'New Name'
// Suggestion: Create a copy or use state

// ✅ 正确:创建副本
function Component({ user }) {
  const [userData, setUserData] = useState(user);

  const handleUpdate = () => {
    setUserData({ ...userData, name: "New Name" });
  };

  return <div>{userData.name}</div>;
}
```

**在渲染中使用副作用**

```typescript
// ❌ 错误:渲染中修改外部变量
let globalCounter = 0;

function Component() {
  // 编译器错误:渲染函数不应有副作用
  globalCounter++;

  return <div>Count: {globalCounter}</div>;
}

// 编译器错误信息:
// Error: Side effects in render
// Component: Component
// Issue: Modifying external variable 'globalCounter'
// Suggestion: Use useEffect for side effects

// ✅ 正确:使用 effect
function Component() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    // 副作用在 effect 中执行
    globalCounter++;
  }, []);

  return <div>Count: {count}</div>;
}
```

**不稳定的 Hook 调用**

```typescript
// ❌ 错误:条件调用 Hook
function Component({ condition }) {
  if (condition) {
    // 编译器错误:Hook 必须在顶层调用
    const [value, setValue] = useState(0);
  }

  return <div>Component</div>;
}

// 编译器错误信息:
// Error: Hooks called conditionally
// Component: Component
// Issue: useState called inside if statement
// Rule: Hooks must be called at the top level

// ✅ 正确:顶层调用
function Component({ condition }) {
  const [value, setValue] = useState(0);

  if (condition) {
    // 使用 value
  }

  return <div>Component</div>;
}
```

### 1.2 优化失败

**复杂闭包**

```typescript
// ⚠️ 警告:无法优化复杂闭包
function Component({ items }) {
  let accumulator = 0;

  // 编译器警告:复杂闭包难以优化
  const processedItems = items.map((item) => {
    accumulator += item.value;
    return {
      ...item,
      cumulative: accumulator,
    };
  });

  return <List items={processedItems} />;
}

// 编译器警告信息:
// Warning: Complex closure detected
// Component: Component
// Issue: Accumulator variable captured in closure
// Impact: May reduce optimization effectiveness

// ✅ 优化建议:使用 reduce
function Component({ items }) {
  const processedItems = items.reduce((acc, item) => {
    const cumulative = (acc[acc.length - 1]?.cumulative || 0) + item.value;
    return [...acc, { ...item, cumulative }];
  }, []);

  return <List items={processedItems} />;
}
```

**循环依赖**

```typescript
// ⚠️ 警告:检测到循环依赖
function Component() {
  const [a, setA] = useState(0);
  const [b, setB] = useState(0);

  // a 依赖 b
  useEffect(() => {
    setA(b + 1);
  }, [b]);

  // b 依赖 a
  useEffect(() => {
    setB(a + 1);
  }, [a]);

  return (
    <div>
      {a} - {b}
    </div>
  );
}

// 编译器警告:
// Warning: Circular dependency detected
// Components: a -> b -> a
// Impact: May cause infinite update loop
// Suggestion: Review dependency chain

// ✅ 修复:打破循环
function Component() {
  const [value, setValue] = useState(0);
  const a = value;
  const b = value + 1;

  return (
    <div>
      {a} - {b}
    </div>
  );
}
```

## 第七部分:故障排查指南

### 7.1 常见故障模式

**内存泄漏检测**

```typescript
class MemoryLeakDetector {
  private snapshots: any[] = [];

  takeSnapshot() {
    if (typeof window !== "undefined" && (window as any).gc) {
      (window as any).gc();
    }

    const snapshot = {
      timestamp: Date.now(),
      memory: performance.memory?.usedJSHeapSize || 0,
    };

    this.snapshots.push(snapshot);
    return snapshot;
  }

  analyze() {
    if (this.snapshots.length < 2) {
      return { leak: false };
    }

    const first = this.snapshots[0];
    const last = this.snapshots[this.snapshots.length - 1];
    const growth = last.memory - first.memory;
    const growthRate = growth / (last.timestamp - first.timestamp);

    return {
      leak: growthRate > 1000, // 1KB/ms
      growth,
      growthRate,
      snapshots: this.snapshots,
    };
  }
}
```

```typescript
        `⚠️ Possible infinite loop detected in ${component.name}`,
        `Rendered ${renderCount} times`
      );
      throw new Error("Infinite render loop detected");
    }

    return component(props);
  };

  return wrapped;
}
```

### 7.3 错误恢复机制

```typescript
class EnhancedErrorBoundary extends React.Component<
  { fallback: React.ReactNode },
  { hasError: boolean; error: Error | null }
> {
  state = { hasError: false, error: null };

  static getDerivedStateFromError(error: Error) {
    return { hasError: true, error };
  }

  componentDidCatch(error: Error, errorInfo: React.ErrorInfo) {
    // 记录错误
    console.error("Component Error:", error);
    console.error("Error Info:", errorInfo);

    // 发送到错误追踪服务
    errorTracker.report({
      error,
      errorInfo,
      componentStack: errorInfo.componentStack,
      timestamp: Date.now(),
    });

    // 尝试恢复
    setTimeout(() => {
      this.setState({ hasError: false, error: null });
    }, 5000);
  }

  render() {
    if (this.state.hasError) {
      return this.props.fallback;
    }

    return this.props.children;
  }
}
```
