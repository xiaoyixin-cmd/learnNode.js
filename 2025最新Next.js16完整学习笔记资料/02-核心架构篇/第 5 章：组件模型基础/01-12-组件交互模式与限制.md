# 组件交互模式与限制

## 概述

在 Next.js 16 中,服务端组件和客户端组件之间的交互是构建复杂应用的核心。理解不同组件之间的交互模式、通信机制以及各种限制,对于开发高效、可维护的应用至关重要。本文档将深入探讨组件之间的各种交互模式、最佳实践和常见陷阱。

### 什么是组件交互

组件交互是指不同组件之间的通信和数据交换。在 Next.js 16 中,这包括:

- **服务端组件之间的交互**:通过 props 和组合模式
- **客户端组件之间的交互**:通过 props、Context、状态提升
- **跨边界交互**:服务端和客户端组件之间的通信
- **Server Actions**:客户端到服务端的 RPC 调用

### 核心价值

1. **清晰的数据流**:单向数据流确保应用的可预测性
2. **职责分离**:不同组件负责不同的关注点
3. **可复用性**:通过标准化的交互模式提高代码复用
4. **类型安全**:TypeScript 确保交互的正确性
5. **性能优化**:避免不必要的重渲染和数据传递

// app/components/ProductFilter.tsx
"use client";

import { useState } from "react";
import FilterControls from "./FilterControls";
import ProductList from "./ProductList";

interface Product {
  id: string;
  name: string;
  category: string;
  price: number;
}

interface Props {
  products: Product[];
}

export default function ProductFilter({ products }: Props) {
  // 状态提升到共同的父组件
  const [category, setCategory] = useState("all");
  const [priceRange, setPriceRange] = useState([0, 1000]);

  const filteredProducts = products.filter((product) => {
    const matchCategory = category === "all" || product.category === category;
    const matchPrice =
      product.price >= priceRange[0] && product.price <= priceRange[1];
    return matchCategory && matchPrice;
  });

  return (
    <div className="product-filter">
      {/* 两个子组件通过父组件状态通信 */}
      <FilterControls
        category={category}
        priceRange={priceRange}
        onCategoryChange={setCategory}
        onPriceRangeChange={setPriceRange}
      />
      <ProductList products={filteredProducts} />
    </div>
  );
}

// app/components/providers/AppContext.tsx
"use client";

import { createContext, useContext, useState, ReactNode } from "react";

interface AppContextType {
  user: {
    id: string;
    name: string;
    role: "admin" | "user";
  } | null;
  theme: "light" | "dark";
  setUser: (user: AppContextType["user"]) => void;
  setTheme: (theme: "light" | "dark") => void;
}

const AppContext = createContext<AppContextType | undefined>(undefined);

export function AppProvider({ children }: { children: ReactNode }) {
  const [user, setUser] = useState<AppContextType["user"]>(null);
  const [theme, setTheme] = useState<"light" | "dark">("light");

  return (
    <AppContext.Provider value={{ user, theme, setUser, setTheme }}>
      {children}
    </AppContext.Provider>
  );
}

export function useApp() {
  const context = useContext(AppContext);
  if (!context) {
    throw new Error("useApp 必须在 AppProvider 内使用");
  }
  return context;
}

### 2.3 自定义 Hooks 实现逻辑复用

**创建自定义 Hook**

```typescript
// app/hooks/useLocalStorage.ts
"use client";

import { useState, useEffect } from "react";

export function useLocalStorage<T>(key: string, initialValue: T) {
  const [storedValue, setStoredValue] = useState<T>(() => {
    if (typeof window === "undefined") {
      return initialValue;
    }

    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      console.error(error);
      return initialValue;
    }
  });

  const setValue = (value: T | ((val: T) => T)) => {
    try {
      const valueToStore =
        value instanceof Function ? value(storedValue) : value;
      setStoredValue(valueToStore);

      if (typeof window !== "undefined") {
        window.localStorage.setItem(key, JSON.stringify(valueToStore));
      }
    } catch (error) {
      console.error(error);
    }
  };

  return [storedValue, setValue] as const;
}
```

export default function FocusInput() {

export interface VideoPlayerHandle {
  play: () => void;
  pause: () => void;
  seek: (time: number) => void;
}

interface Props {
  src: string;
}

const VideoPlayer = forwardRef<VideoPlayerHandle, Props>(({ src }, ref) => {
  const videoRef = useRef<HTMLVideoElement>(null);

  useImperativeHandle(ref, () => ({
    play() {
      videoRef.current?.play();
    },
    pause() {
      videoRef.current?.pause();
    },
    seek(time: number) {
      if (videoRef.current) {
        videoRef.current.currentTime = time;
      }
    },
  }));

  return <video ref={videoRef} src={src} />;
});

// app/actions/todos.ts
"use server";

import { revalidatePath } from "next/cache";

export async function createTodo(formData: FormData) {
  const title = formData.get("title") as string;

  const newTodo = await fetch("https://api.example.com/todos", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ title, completed: false }),
  }).then((res) => res.json());

  revalidatePath("/todos");

  return { success: true, todo: newTodo };
}

export async function updateTodo(id: string, completed: boolean) {
  await fetch(`https://api.example.com/todos/${id}`, {
    method: "PATCH",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ completed }),
  });

  revalidatePath("/todos");

  return { success: true };
}

**使用 useMemo 缓存计算结果**

```typescript
// app/components/DataGrid.tsx
"use client";

import { useMemo } from "react";

interface Props {
  data: Array<{ id: string; value: number }>;
  filterThreshold: number;
}

export default function DataGrid({ data, filterThreshold }: Props) {
  const filteredData = useMemo(() => {
    console.log("重新计算过滤数据");
    return data.filter((item) => item.value > filterThreshold);
  }, [data, filterThreshold]);

  const total = useMemo(() => {
    console.log("重新计算总和");
    return filteredData.reduce((sum, item) => sum + item.value, 0);
  }, [filteredData]);

  return (
    <div>
      <p>总和: {total}</p>
      {filteredData.map((item) => (
        <div key={item.id}>{item.value}</div>
      ))}
    </div>
  );
}
```

## 常见问题

### Q1: 服务端组件和客户端组件可以互相嵌套吗?

**A**: 可以,但有一些规则:

1. **服务端组件可以直接包含客户端组件**
2. **客户端组件不能直接导入服务端组件**
3. **客户端组件可以通过 children 包含服务端组件**

### Q2: 如何在客户端组件中访问服务端数据?

**A**: 有几种方法:

1. **通过 props 传递初始数据**
2. **使用 Server Actions**
3. **使用 Route Handlers**

### Q3: Context 可以跨越服务端和客户端边界吗?

**A**: Context Provider 必须是客户端组件,但可以在服务端组件中使用。

### Q4: 如何优化大量组件的性能?

**A**: 使用以下策略:

1. **使用 React.memo 避免不必要的重渲染**
2. **使用 useMemo 缓存计算结果**
3. **使用 useCallback 缓存回调函数**
4. **使用动态导入延迟加载组件**
5. **使用虚拟滚动处理大列表**

### Q5: Server Actions 和 Route Handlers 的区别是什么?

**A**: 主要区别:

- **Server Actions**: 表单提交、数据变更、直接调用函数、完全类型安全
- **Route Handlers**: RESTful API、HTTP 请求、需要手动定义类型
