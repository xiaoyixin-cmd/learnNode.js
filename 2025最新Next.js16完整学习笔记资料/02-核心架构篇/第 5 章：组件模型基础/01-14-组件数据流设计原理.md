# 组件数据流设计原理

## 概述

在 Next.js 16 中,组件数据流设计是构建可预测、可维护应用的核心。合理的数据流设计能够确保数据在组件之间正确传递,避免状态管理混乱,提高应用性能。本文档将深入探讨 Next.js 16 中组件数据流的设计原理、模式和最佳实践。

### 什么是数据流

数据流是指应用中数据的流动方向和传递机制。在 React 和 Next.js 中,数据流主要包括:

- **Props 传递**:父组件向子组件传递数据
- **状态提升**:将共享状态提升到共同父组件
- **Context 传递**:跨层级传递数据
- **Server Actions**:客户端到服务端的数据传递
- **缓存策略**:服务端数据的缓存和共享

### 核心价值

1. **可预测性**:明确的数据流向使应用行为可预测
2. **可维护性**:清晰的数据流便于理解和修改
3. **性能优化**:避免不必要的数据传递和重渲染
4. **调试友好**:便于追踪数据变化和定位问题
5. **类型安全**:TypeScript 确保数据流的类型正确性

## 第一部分:单向数据流原理

### 1.1 React 单向数据流基础

**基本概念**

```typescript
// 父组件
function ParentComponent() {
  const [data, setData] = useState("Hello");

  // 数据向下流动
  return <ChildComponent data={data} onDataChange={setData} />;
}

// 子组件
function ChildComponent({ data, onDataChange }) {
  // 接收数据
  return (
    <div>
      <p>{data}</p>
      {/* 事件向上传递 */}
      <button onClick={() => onDataChange("World")}>Change</button>
    </div>
  );
}
```

**数据流示意图**

```
┌─────────────────┐
│ ParentComponent │
│    (状态持有)    │
└────────┬────────┘
         │ Props ↓
         │
┌────────▼────────┐
│ ChildComponent  │
│   (数据消费)    │
└────────┬────────┘
         │ Event ↑
         │
┌────────▼────────┐
│   Update State  │
└─────────────────┘
```

```typescript
import { useState } from "react";

function BoilingVerdict({ celsius }: { celsius: number }) {
  if (celsius >= 100) {
    return <p>水会沸腾</p>;
  }
  return <p>水不会沸腾</p>;
}

function TemperatureInput({
  scale,
  temperature,
  onTemperatureChange,
}: {
  scale: "c" | "f";
  temperature: string;
  onTemperatureChange: (value: string) => void;
}) {
  const scaleNames = {
    c: "摄氏度",
    f: "华氏度",
  };

  return (
    <fieldset>
      <legend>输入{scaleNames[scale]}温度:</legend>
      <input
        value={temperature}
        onChange={(e) => onTemperatureChange(e.target.value)}
      />
    </fieldset>
  );
}

export default function TemperatureCalculator() {
  // 状态提升到共同父组件
  const [temperature, setTemperature] = useState("");
  const [scale, setScale] = useState<"c" | "f">("c");

  const handleCelsiusChange = (temp: string) => {
    setScale("c");
    setTemperature(temp);
  };

  const handleFahrenheitChange = (temp: string) => {
    setScale("f");
    setTemperature(temp);
  };

  const celsius =
    scale === "f"
      ? ((parseFloat(temperature) - 32) * 5) / 9
      : parseFloat(temperature);

  return (
    <div>
      <TemperatureInput
        scale="c"
        temperature={scale === "c" ? temperature : ""}
        onTemperatureChange={handleCelsiusChange}
      />
      <TemperatureInput
        scale="f"
        temperature={scale === "f" ? temperature : ""}
        onTemperatureChange={handleFahrenheitChange}
      />
      <BoilingVerdict celsius={celsius} />
    </div>
  );
}
```

**何时提升状态**

- 多个组件需要访问相同的数据
- 兄弟组件之间需要通信
- 需要同步多个组件的状态
- 需要统一管理相关状态

### 2.3 Context 状态管理

**创建 Context**

```typescript
// contexts/CartContext.tsx
"use client";

import { createContext, useContext, useState, ReactNode } from "react";

interface CartItem {
  id: string;
  name: string;
  price: number;
  quantity: number;
}

interface CartContextType {
  items: CartItem[];
  addItem: (item: CartItem) => void;
  removeItem: (id: string) => void;
  updateQuantity: (id: string, quantity: number) => void;
  clearCart: () => void;
  totalPrice: number;
  totalItems: number;
}

const CartContext = createContext<CartContextType | undefined>(undefined);

export function CartProvider({ children }: { children: ReactNode }) {
  const [items, setItems] = useState<CartItem[]>([]);

  const addItem = (item: CartItem) => {
    setItems((prev) => {
      const existing = prev.find((i) => i.id === item.id);
      if (existing) {
        return prev.map((i) =>
          i.id === item.id ? { ...i, quantity: i.quantity + item.quantity } : i
        );
      }
      return [...prev, item];
    });
  };

  const removeItem = (id: string) => {
    setItems((prev) => prev.filter((i) => i.id !== id));
  };

  const updateQuantity = (id: string, quantity: number) => {
    if (quantity <= 0) {
      removeItem(id);
      return;
    }
    setItems((prev) => prev.map((i) => (i.id === id ? { ...i, quantity } : i)));
  };

  const clearCart = () => {
    setItems([]);
  };

  const totalPrice = items.reduce(
    (sum, item) => sum + item.price * item.quantity,
    0
  );

  const totalItems = items.reduce((sum, item) => sum + item.quantity, 0);

  return (
    <CartContext.Provider
      value={{
        items,
        addItem,
        removeItem,
        updateQuantity,
        clearCart,
        totalPrice,
        totalItems,
      }}
    >
      {children}
    </CartContext.Provider>
  );
}

export function useCart() {
  const context = useContext(CartContext);
  if (!context) {
    throw new Error("useCart 必须在 CartProvider 内使用");
  }
  return context;
}
```

```typescript
// stores/useUserStore.ts
import { create } from "zustand";
import { persist } from "zustand/middleware";

interface User {
  id: string;
  name: string;
  email: string;
  role: "admin" | "user";
}

interface UserState {
  user: User | null;
  isAuthenticated: boolean;
  login: (user: User) => void;
  logout: () => void;
  updateUser: (updates: Partial<User>) => void;
}

export const useUserStore = create<UserState>()(
  persist(
    (set) => ({
      user: null,
      isAuthenticated: false,

      login: (user) =>
        set({
          user,
          isAuthenticated: true,
        }),

      logout: () =>
        set({
          user: null,
          isAuthenticated: false,
        }),

      updateUser: (updates) =>
        set((state) => ({
          user: state.user ? { ...state.user, ...updates } : null,
        })),
    }),
    {
      name: "user-storage",
    }
  )
);
```

## 常见问题

### Q1: 什么时候应该使用 Context?

**A**: Context 适用于以下场景:

- 需要在多个层级的组件间共享数据
- 避免 prop drilling
- 全局主题、语言设置等配置
- 用户认证状态

**不适合使用 Context 的场景**:

- 频繁变化的数据(会导致不必要的重渲染)
- 局部状态
- 简单的父子组件通信

### Q2: 如何避免不必要的重渲染?

**A**: 使用以下策略:

1. **React.memo**: 缓存组件渲染结果
2. **useMemo**: 缓存计算结果
3. **useCallback**: 缓存回调函数
4. **Context 拆分**: 将频繁变化的数据分离到独立 Context

### Q3: Server Actions 和 API Routes 的区别?

**A**: 主要区别:

| 特性         | Server Actions     | API Routes   |
| ------------ | ------------------ | ------------ |
| 用途         | 表单提交、数据变更 | RESTful API  |
| 调用方式     | 直接函数调用       | HTTP 请求    |
| 类型安全     | 完全类型安全       | 需要手动定义 |
| 缓存重新验证 | 内置支持           | 需要手动处理 |

### Q4: 如何在客户端和服务端共享数据?

**A**: 主要方式:

1. **Props 传递**: 服务端组件通过 props 传递数据给客户端组件
2. **Server Actions**: 客户端通过 Server Actions 获取服务端数据
3. **API Routes**: 客户端通过 fetch 调用 API
4. **Route Handlers**: 客户端通过 HTTP 请求获取数据

## 适用场景

### 1. 简单应用

- **局部状态**: useState + props
- **简单共享**: 状态提升
- **数据获取**: 直接在服务端组件中获取

### 2. 中等复杂度应用

- **全局配置**: Context API
- **表单管理**: React Hook Form
- **数据缓存**: React cache
- **状态管理**: useState + Context
