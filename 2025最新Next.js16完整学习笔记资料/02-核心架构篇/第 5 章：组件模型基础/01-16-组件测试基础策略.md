# 组件测试基础策略

## 概述

在 Next.js 16 开发中,组件测试是确保代码质量、防止回归错误、提高代码可维护性的关键实践。良好的测试策略不仅能提高代码质量,还能增强团队对代码修改的信心,加快开发速度。本文档将深入探讨 Next.js 16 中组件测试的基础策略、工具选择和最佳实践。

### 为什么需要组件测试

组件测试的价值体现在:

- **质量保证**:确保组件按预期工作
- **防止回归**:避免修改代码时破坏已有功能
- **文档作用**:测试代码是组件使用的最好文档
- **重构信心**:有测试覆盖的代码更容易重构
- **团队协作**:新成员通过测试了解组件行为

### 核心原则

1. **测试行为而非实现**:测试用户可见的行为
2. **保持测试简单**:每个测试只验证一个概念
3. **快速反馈**:测试应该快速运行
4. **可维护性**:测试代码应该易于理解和维护
5. **覆盖关键路径**:优先测试关键业务逻辑

```json
}
```

```typescript
// components/Timer.tsx
"use client";

import { useState, useEffect } from "react";

export default function Timer() {
  const [seconds, setSeconds] = useState(0);
  const [isRunning, setIsRunning] = useState(false);

  useEffect(() => {
    if (!isRunning) return;

    const interval = setInterval(() => {
      setSeconds((s) => s + 1);
    }, 1000);

    return () => clearInterval(interval);
  }, [isRunning]);

  return (
    <div>
      <p>时间: {seconds}秒</p>
      <button onClick={() => setIsRunning(!isRunning)}>
        {isRunning ? "暂停" : "开始"}
      </button>
      <button
        onClick={() => {
          setSeconds(0);
          setIsRunning(false);
        }}
      >
        重置
      </button>
    </div>
  );
}
```

**SearchBox 组件**

```typescript
// components/SearchBox.tsx
"use client";

import { useState } from "react";

interface Props {
  onSearch: (query: string) => Promise<string[]>;
}

export default function SearchBox({ onSearch }: Props) {
  const [query, setQuery] = useState("");
  const [results, setResults] = useState<string[]>([]);
  const [loading, setLoading] = useState(false);

  const handleSearch = async () => {
    setLoading(true);
    try {
      const data = await onSearch(query);
      setResults(data);
    } finally {
      setLoading(false);
    }
  };

  return (
    <div>
      <input
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="搜索..."
      />
      <button onClick={handleSearch} disabled={loading}>
        {loading ? "搜索中..." : "搜索"}
      </button>
      {results.length > 0 && (
        <ul>
          {results.map((result, index) => (
            <li key={index}>{result}</li>
          ))}
        </ul>
      )}
    </div>
  );
}
```

```typescript
// components/LoginForm.tsx
"use client";

import { useState } from "react";
import { login } from "@/actions/auth";

export default function LoginForm() {
  const [email, setEmail] = useState("");
  const [password, setPassword] = useState("");
  const [error, setError] = useState("");
  const [loading, setLoading] = useState(false);

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    setLoading(true);
    setError("");

    try {
      await login(email, password);
    } catch (err) {
      setError("登录失败");
    } finally {
      setLoading(false);
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        placeholder="邮箱"
      />
      <input
        type="password"
        value={password}
        onChange={(e) => setPassword(e.target.value)}
        placeholder="密码"
      />
      {error && <p role="alert">{error}</p>}
      <button type="submit" disabled={loading}>
        {loading ? "登录中..." : "登录"}
      </button>
    </form>
  );
}
```

```typescript
// components/UserList.test.tsx
import { fetchUsers } from "@/api";

  (fetchUsers as jest.Mock).mockResolvedValueOnce([{ id: "1", name: "王五" }]);

    expect(screen.getByText("王五")).toBeInTheDocument();
  });
});
```

## 常见问题

### Q1: 应该测试什么?

**A**: 优先级排序:

1. **关键业务逻辑**: 核心功能和用户流程
2. **边缘情况**: 错误处理、空状态、极端值
3. **用户交互**: 点击、输入、提交等
4. **集成点**: 组件之间的交互

### Q2: 如何组织测试文件?

**A**: 推荐的组织方式:

```
src/
├── components/
│   ├── Button/
│   │   ├── Button.tsx
│   │   ├── Button.test.tsx
│   │   └── index.ts
│   └── UserList/
│       ├── UserList.tsx
│       ├── UserList.test.tsx
│       └── index.ts
```

### Q3: 如何提高测试覆盖率?

**A**: 策略:

1. **识别未测试代码**: 使用覆盖率报告
2. **补充边缘情况**: 测试异常路径
3. **测试条件分支**: 确保所有分支都被覆盖
4. **集成测试**: 测试组件协作

### Q4: 如何处理异步测试?

**A**: 使用 waitFor 和 async/await:

```typescript
await waitFor(() => {
  expect(screen.getByText("加载完成")).toBeInTheDocument();
});
```
