# 资源优化与代码分割

## 概述

资源优化和代码分割是提升 Web 应用性能的关键技术。在 Next.js 16 中,Turbopack 通过智能的代码分割策略和全面的资源优化能力,帮助开发者构建高性能的应用程序。合理的代码分割可以将首屏加载时间缩短 50% 以上,资源优化可以减少 30-60% 的传输体积。本文将深入探讨 Turbopack 的资源优化和代码分割机制,包括自动分割、手动优化、资源压缩和加载策略。

### 为什么需要优化

1. **首屏性能**:减少初始加载时间,提升用户体验
2. **带宽节省**:减小传输体积,降低流量成本
3. **缓存效率**:优化缓存策略,提高命中率
4. **运行性能**:减少解析和执行时间

### 优化目标

- **代码体积**: 减小 JavaScript bundle 大小
- **资源大小**: 优化图片、字体等静态资源
- **加载策略**: 按需加载,延迟加载
- **缓存利用**: 最大化缓存命中率

**智能分割算法**

```typescript
// 智能代码分割算法

class SmartCodeSplitter {
  // 分析最优分割方案
  analyzeSplitStrategy(
    modules: Module[],
    constraints: SplitConstraints
  ): SplitStrategy {
    // 1. 计算模块依赖关系
    const dependencyGraph = this.buildDependencyGraph(modules);

    // 2. 识别关键路径
    const criticalPaths = this.findCriticalPaths(dependencyGraph);

    // 3. 分组模块
    const groups = this.groupModules(modules, {
      maxSize: constraints.maxChunkSize,
      minSize: constraints.minChunkSize,
      maxParallelRequests: constraints.maxParallelRequests,
    });

    // 4. 优化分组
    const optimizedGroups = this.optimizeGroups(groups);

    return {
      chunks: optimizedGroups,
      loadingStrategy: this.determineLoadingStrategy(optimizedGroups),
    };
  }

  // 构建依赖图
  private buildDependencyGraph(modules: Module[]): DependencyGraph {
    const graph = new Map<string, Set<string>>();

    modules.forEach((module) => {
      const deps = new Set<string>();

      // 解析 import 语句
      const importRegex = /import\s+.*?\s+from\s+['"]([^'"]+)['"]/g;
      let match;

      while ((match = importRegex.exec(module.source)) !== null) {
        deps.add(match[1]);
      }

      graph.set(module.id, deps);
    });

    return graph;
  }

  // 查找关键路径
  private findCriticalPaths(graph: DependencyGraph): string[][] {
    const paths: string[][] = [];

    // 从入口点开始 DFS
    graph.forEach((deps, moduleId) => {
      if (this.isEntryPoint(moduleId)) {
        const path = this.tracePath(moduleId, graph);
        paths.push(path);
      }
    });

    return paths;
  }

  private tracePath(
    start: string,
    graph: DependencyGraph,
    visited = new Set<string>()
  ): string[] {
    if (visited.has(start)) return [];

    visited.add(start);
    const path = [start];

    const deps = graph.get(start);
    if (deps && deps.size > 0) {
      // 递归最长路径
      let longestSubPath: string[] = [];

      deps.forEach((dep) => {
        const subPath = this.tracePath(dep, graph, visited);
        if (subPath.length > longestSubPath.length) {
          longestSubPath = subPath;
        }
      });

      path.push(...longestSubPath);
    }

    return path;
  }

  // 分组模块
  private groupModules(
    modules: Module[],
    constraints: {
      maxSize: number;
      minSize: number;
      maxParallelRequests: number;
    }
  ): ModuleGroup[] {
    const groups: ModuleGroup[] = [];
    let currentGroup: Module[] = [];
    let currentSize = 0;

    // 按依赖关系排序
    const sorted = this.topologicalSort(modules);

    sorted.forEach((module) => {
      const moduleSize = module.source.length;

      // 检查是否超过最大大小
      if (currentSize + moduleSize > constraints.maxSize) {
        if (currentGroup.length > 0) {
          groups.push({
            modules: currentGroup,
            size: currentSize,
          });
        }

        currentGroup = [module];
        currentSize = moduleSize;
      } else {
        currentGroup.push(module);
        currentSize += moduleSize;
      }
    });

    // 添加最后一组
    if (currentGroup.length > 0) {
      groups.push({
        modules: currentGroup,
        size: currentSize,
      });
    }

    return groups;
  }

  // 优化分组
  private optimizeGroups(groups: ModuleGroup[]): ModuleGroup[] {
    // 1. 合并小分组
    const merged = this.mergeSmallGroups(groups);

    // 2. 分割大分组
    const split = this.splitLargeGroups(merged);

    // 3. 平衡分组大小
    const balanced = this.balanceGroups(split);

    return balanced;
  }

  private mergeSmallGroups(groups: ModuleGroup[]): ModuleGroup[] {
    // 合并小于最小大小的分组
    return groups;
  }

  private splitLargeGroups(groups: ModuleGroup[]): ModuleGroup[] {
    // 分割超过最大大小的分组
    return groups;
  }

  private balanceGroups(groups: ModuleGroup[]): ModuleGroup[] {
    // 平衡分组大小
    return groups;
  }

  private topologicalSort(modules: Module[]): Module[] {
    // 拓扑排序
    return modules;
  }

  private isEntryPoint(moduleId: string): boolean {
    // 判断是否为入口点
    return false;
  }

  private determineLoadingStrategy(groups: ModuleGroup[]): LoadingStrategy {
    // 确定加载策略
    return {
      type: "progressive",
      priority: groups.map((g, i) => ({
        chunk: g,
        priority: i,
      })),
    };
  }
}
```

```typescript
// 基于视口的懒加载

"use client";

import { useEffect, useRef, useState } from "react";

export function LazySection({ children }: { children: React.ReactNode }) {
  const [isVisible, setIsVisible] = useState(false);
  const ref = useRef<HTMLDivElement>(null);

  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          setIsVisible(true);
          observer.disconnect();
        }
      },
      {
        threshold: 0.1,
        rootMargin: "100px",
      }
    );

    if (ref.current) {
      observer.observe(ref.current);
    }

    return () => observer.disconnect();
  }, []);

  return (
    <div ref={ref}>
      {isVisible ? children : <div className="placeholder">加载中...</div>}
    </div>
  );
}

// 使用
export default function Page() {
  return (
    <div>
      <section>首屏内容</section>

      <LazySection>
        <HeavyComponent />
      </LazySection>

      <LazySection>
        <AnotherHeavyComponent />
      </LazySection>
    </div>
  );
}
```

## 常见问题

### Q1: 如何减小 JavaScript bundle 体积?

**A**: 多种优化方法:

1. **Tree Shaking**: 移除未使用代码
2. **代码分割**: 按路由/组件分割
3. **动态导入**: 懒加载非关键代码
4. **依赖优化**: 移除不必要的依赖
5. **压缩**: 启用代码压缩

### Q2: 图片优化的最佳实践是什么?

**A**: 优化建议:

1. **使用 next/image**: 自动优化和懒加载
2. **现代格式**: WebP/AVIF
3. **响应式**: 根据设备加载适当尺寸
4. **优先级**: LCP 图片设置 priority
5. **压缩**: 适当压缩质量

### Q3: 如何优化首屏加载时间?

**A**: 优化策略:

1. **关键资源预加载**: preload 关键 CSS/字体
2. **代码分割**: 减小初始 bundle
3. **懒加载**: 延迟非关键组件
4. **缓存**: 利用浏览器缓存
5. **CDN**: 使用 CDN 加速

### Q4: 动态导入什么时候使用?

**A**: 适用场景:

1. **大组件**: 体积大的第三方库
2. **条件渲染**: 根据条件显示的组件
3. **非首屏**: 用户交互后才显示
4. **客户端专用**: 仅客户端运行的代码
