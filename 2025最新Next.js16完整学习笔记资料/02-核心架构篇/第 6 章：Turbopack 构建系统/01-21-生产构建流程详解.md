# 生产构建流程详解

## 概述

生产构建是将 Next.js 应用程序准备好部署到生产环境的关键过程。Turbopack 的生产构建经过精心设计,不仅追求极致的构建速度,更注重输出质量和运行时性能。在 Next.js 16 中,Turbopack 的生产构建相比 Webpack 快 2-5 倍,同时生成的代码体积更小、性能更优。本文将深入解析 Turbopack 生产构建的完整流程,包括优化策略、输出结构和最佳实践。

### 生产构建的目标

1. **最小化体积**:通过 Tree Shaking、代码分割等技术减小输出体积
2. **最大化性能**:优化加载速度和运行时性能
3. **最优化缓存**:生成利于缓存的文件结构
4. **确保稳定**:生成稳定、可预测的构建输出

### Turbopack vs Webpack 构建对比

```bash
# 中型应用(1500模块)生产构建对比

Webpack:
- 构建时间: 45s

- 构建时间: 18s (2.5x 更快)
```

```typescript
  async generateClientBundles() {
    console.log("   生成客户端 bundles...");

    // 1. 创建主 bundle
    const mainBundle = await this.createMainBundle();
    await this.writeBundle("main", mainBundle);

    // 2. 创建框架 bundle
    const frameworkBundle = await this.createFrameworkBundle();
    await this.writeBundle("framework", frameworkBundle);

    // 3. 创建页面 bundles
    const pageChunks = await this.createPageBundles();
    for (const [name, chunk] of pageChunks) {
      await this.writeBundle(`pages/${name}`, chunk);
    }

    // 4. 创建共享 bundles
    const sharedChunks = await this.createSharedBundles();
    for (const [name, chunk] of sharedChunks) {
      await this.writeBundle(`shared/${name}`, chunk);
    }
  }

  // 创建主 bundle
  private async createMainBundle(): Promise<string> {
    const modules = [
      // Next.js 客户端运行时
      "_app",
      "_document",
      "webpack-runtime",
    ];

    return this.bundleModules(modules);

  private async createFrameworkBundle(): Promise<string> {

    for (const page of pages) {
      const modules = await this.getPageModules(page);
```

### 2.1 Tree Shaking

**死代码消除**

```typescript
// Tree Shaking 实现

class TreeShaker {
  private usedExports = new Map<string, Set<string>>();

  // 分析使用的导出
  analyzeUsedExports(graph: DependencyGraph) {
    console.log("🌳 Tree Shaking...");

    // 从入口点开始分析
    graph.entryPoints.forEach((entry) => {
      this.traceUsedExports(entry, graph);
    });

    // 统计结果
    const totalExports = this.countTotalExports(graph);
    const usedCount = this.countUsedExports();
    const removed = totalExports - usedCount;

    console.log(`   移除 ${removed}/${totalExports} 个未使用导出`);
  }

  // 追踪使用的导出
  private traceUsedExports(
    moduleId: string,
    graph: DependencyGraph,
    importedNames?: Set<string>
  ) {
    const module = graph.getModule(moduleId);
    if (!module) return;

    // 记录使用的导出
    if (importedNames) {
      this.usedExports.set(moduleId, importedNames);
    } else {
      // 入口点:所有导出都使用
      this.usedExports.set(moduleId, new Set(module.exports));
    }

    // 分析依赖
    module.dependencies.forEach((dep) => {
      const importedFromDep = this.getImportedNames(module, dep.moduleId);
      this.traceUsedExports(dep.moduleId, graph, importedFromDep);
    });
  }

  // 获取从模块导入的名称
  private getImportedNames(module: Module, depId: string): Set<string> {
    const imports = new Set<string>();

    // 分析 import 语句
    const importRegex = /import\s+{([^}]+)}\s+from\s+['"]([^'"]+)['"]/g;
    let match;

    while ((match = importRegex.exec(module.source)) !== null) {
      const [, names, source] = match;

      if (this.resolveModule(source) === depId) {
        names.split(",").forEach((name) => {
          imports.add(name.trim());
        });
      }
    }

    // import * as xxx
    const namespaceImportRegex =
      /import\s+\*\s+as\s+\w+\s+from\s+['"]([^'"]+)['"]/g;
    if (namespaceImportRegex.test(module.source)) {
      // 命名空间导入:所有导出都使用
      const depModule = graph.getModule(depId);
      if (depModule) {
        depModule.exports.forEach((exp) => imports.add(exp));
      }
    }

    return imports;
  }

  // 移除未使用的导出
  removeUnusedExports(module: Module): Module {
    const usedNames = this.usedExports.get(module.id) || new Set();

    // 过滤导出
    const newSource = this.filterExports(module.source, usedNames);

    return {
      ...module,
      source: newSource,
    };
  }

  private filterExports(source: string, usedNames: Set<string>): string {
    let newSource = source;

    // 移除未使用的 export
    const exportRegex = /export\s+(const|function|class)\s+(\w+)/g;
    let match;

    while ((match = exportRegex.exec(source)) !== null) {
      const [fullMatch, , name] = match;

      if (!usedNames.has(name)) {
        // 移除此导出
        newSource = newSource.replace(fullMatch, `const ${name}`);
      }
    }

    return newSource;
  }

  private resolveModule(source: string): string {
    // 解析模块路径
    return source;
  }

  private countTotalExports(graph: DependencyGraph): number {
    let count = 0;

    graph.getAllModules().forEach((module) => {
      count += module.exports.length;
    });

    return count;
  }

  private countUsedExports(): number {
    let count = 0;

    this.usedExports.forEach((names) => {
      count += names.size;
    });

    return count;
  }
}
```

```typescript
// 代码分割策略

class CodeSplitter {
  // 分析分割点
  async analyzeSplitPoints(graph: DependencyGraph): Promise<SplitPoint[]> {
    const splitPoints: SplitPoint[] = [];

    // 1. 路由级别分割
    const routeSplits = this.findRouteSplitPoints(graph);
    splitPoints.push(...routeSplits);

    // 2. 动态导入分割
    const dynamicSplits = this.findDynamicImportSplitPoints(graph);
    splitPoints.push(...dynamicSplits);

    // 3. 共享模块分割
    const sharedSplits = this.findSharedModuleSplitPoints(graph);
    splitPoints.push(...sharedSplits);

    // 4. 第三方库分割
    const vendorSplits = this.findVendorSplitPoints(graph);
    splitPoints.push(...vendorSplits);

    console.log(`找到 ${splitPoints.length} 个分割点`);

    return splitPoints;
  }

  // 路由分割
  private findRouteSplitPoints(graph: DependencyGraph): SplitPoint[] {
    const splits: SplitPoint[] = [];

    // 每个路由页面一个 chunk
    graph.pages.forEach((page) => {
      splits.push({
        type: "route",
        name: page.route,
        modules: [page.moduleId],
      });
    });

    return splits;
  }

  // 动态导入分割
  private findDynamicImportSplitPoints(graph: DependencyGraph): SplitPoint[] {
    const splits: SplitPoint[] = [];

    graph.getAllModules().forEach((module) => {
      // 查找 dynamic import
      const dynamicImportRegex = /import\(['"]([^'"]+)['"]\)/g;
      let match;

      while ((match = dynamicImportRegex.exec(module.source)) !== null) {
        const [, importPath] = match;
        const moduleId = this.resolveModule(importPath);

        splits.push({
          type: "dynamic",
          name: importPath,
          modules: [moduleId],
        });
      }
    });

    return splits;
  }

  // 共享模块分割
  private findSharedModuleSplitPoints(graph: DependencyGraph): SplitPoint[] {
    const splits: SplitPoint[] = [];

    // 找到被多个页面使用的模块
    const usage = new Map<string, number>();

    graph.pages.forEach((page) => {
      const deps = graph.getDependencies(page.moduleId);
      deps.forEach((dep) => {
        usage.set(dep, (usage.get(dep) || 0) + 1);
      });
    });

    // 被多于 2 个页面使用的模块
    const sharedModules: string[] = [];
    usage.forEach((count, moduleId) => {
      if (count > 2) {
        sharedModules.push(moduleId);
      }
    });

    if (sharedModules.length > 0) {
      splits.push({
        type: "shared",
        name: "shared",
        modules: sharedModules,
      });
    }

    return splits;
  }

  // 第三方库分割
  private findVendorSplitPoints(graph: DependencyGraph): SplitPoint[] {
    const splits: SplitPoint[] = [];

    // 提取 node_modules 中的模块
    const vendorModules = graph
      .getAllModules()
      .filter((module) => module.id.includes("node_modules"));

    // 按包名分组
    const packages = new Map<string, string[]>();

    vendorModules.forEach((module) => {
      const packageName = this.extractPackageName(module.id);

      if (!packages.has(packageName)) {
        packages.set(packageName, []);
      }

      packages.get(packageName)!.push(module.id);
    });

    // 大包单独分割
    packages.forEach((modules, packageName) => {
      const size = this.calculateSize(modules);

      if (size > 50 * 1024) {
        // > 50KB
        splits.push({
          type: "vendor",
          name: `vendor-${packageName}`,
          modules,
        });
      }
    });

    return splits;
  }

  private resolveModule(path: string): string {
    // 解析模块路径
    return path;
  }

  private extractPackageName(moduleId: string): string {
    const match = moduleId.match(/node_modules\/(@[^\/]+\/[^\/]+|[^\/]+)/);
    return match ? match[1] : "unknown";
  }

  private calculateSize(modules: string[]): number {
    // 计算模块总大小
    return 0;
  }
}
```

## 常见问题

### Q1: 如何减小构建输出体积?

**A**: 多种优化方法:

1. **启用压缩**: `compress: true`
2. **Tree Shaking**: 自动移除未使用代码
3. **代码分割**: 合理拆分 chunks
4. **图片优化**: 使用 WebP/AVIF 格式
5. **依赖优化**: 移除不必要的依赖

### Q2: 构建速度慢怎么优化?

**A**: 优化策略:

1. **持久化缓存**: 启用 `turbopackPersistentCaching`
2. **并行构建**: Turbopack 自动并行
3. **减少文件**: 排除不必要的文件
4. **增量构建**: 只构建变化的部分

### Q3: 如何调试构建问题?

**A**: 调试方法:

1. **启用详细日志**: `NEXT_TURBOPACK_TRACING=1`
2. **分析 bundle**: 使用 bundle 分析工具
3. **检查清单文件**: 查看生成的清单
4. **验证输出**: 检查 .next 目录

### Q4: 生产环境如何部署?

**A**: 部署选项:

1. **Standalone 模式**: 独立部署包
2. **Docker**: 容器化部署
3. **Vercel**: 一键部署
4. **自定义服务器**: Node.js 服务器
