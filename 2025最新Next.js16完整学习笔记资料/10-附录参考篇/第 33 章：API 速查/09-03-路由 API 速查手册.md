# 路由 API 速查手册

## 1. 概述 (Overview)

Next.js 16 的路由系统基于文件系统,提供了丰富的路由 API 用于导航、参数获取和路由控制。本速查手册汇总了所有与路由相关的 API,帮助开发者快速查找和使用路由功能。

### 1.1 路由系统概览

Next.js 16 的路由系统包含以下几个方面:

- **导航 API**: useRouter, Link, redirect 等
- **参数获取 API**: useParams, useSearchParams, usePathname 等
- **路由配置**: generateStaticParams, Route Segment Config 等
- **特殊路由**: 错误处理、加载状态、未找到页面等

### 1.2 App Router vs Pages Router

本手册专注于 **App Router** (app 目录),这是 Next.js 13+ 推荐的路由方式。

| 特性     | App Router     | Pages Router       |
| :------- | :------------- | :----------------- |
| 目录     | app/           | pages/             |
| 组件类型 | 服务端组件优先 | 客户端组件         |
| 布局     | 支持嵌套布局   | 有限支持           |
| 数据获取 | async/await    | getServerSideProps |
| 推荐使用 | ✅ 是          | ❌ 旧项目          |

### 1.3 手册使用说明

**快速查找**: 使用 Ctrl+F 搜索 API 名称

**分类浏览**: 按照功能分类查看

**版本标识**: 🆕 表示 Next.js 16 新增,⚠️ 表示有破坏性变更

## 2. 客户端导航 API (Client-Side Navigation)

### 2.1 useRouter Hook

**功能**: 在客户端组件中进行编程式导航

**类型签名**:

```typescript
"use client";

import { useRouter } from "next/navigation";

function useRouter(): {
  push: (href: string, options?: NavigateOptions) => void;
  replace: (href: string, options?: NavigateOptions) => void;
  refresh: () => void;
  prefetch: (href: string) => void;
  back: () => void;
  forward: () => void;
};

interface NavigateOptions {
  scroll?: boolean;
}
```

**使用示例**:

```typescript
"use client";

import { useRouter } from "next/navigation";

export default function Page() {
  const router = useRouter();

  return (
    <div>
      {/* 导航到新页面 */}
      <button onClick={() => router.push("/dashboard")}>Go to Dashboard</button>

      {/* 替换当前历史记录 */}
      <button onClick={() => router.replace("/login")}>
        Replace with Login
      </button>

      {/* 刷新当前页面 */}
      <button onClick={() => router.refresh()}>Refresh</button>

      {/* 预取页面 */}
      <button onClick={() => router.prefetch("/products")}>
        Prefetch Products
      </button>

      {/* 后退 */}
      <button onClick={() => router.back()}>Back</button>

      {/* 前进 */}
      <button onClick={() => router.forward()}>Forward</button>
    </div>
  );
}
```

**方法说明**:

| 方法     | 说明         | 历史记录     | 使用场景     |
| :------- | :----------- | :----------- | :----------- |
| push     | 导航到新页面 | 添加新记录   | 一般导航     |
| replace  | 替换当前页面 | 替换当前记录 | 登录后跳转   |
| refresh  | 刷新当前页面 | 不变         | 重新获取数据 |
| prefetch | 预取页面     | 不变         | 性能优化     |
| back     | 后退         | 后退一步     | 返回上一页   |
| forward  | 前进         | 前进一步     | 前进到下一页 |

**scroll 选项**:

```typescript
// 导航后滚动到顶部(默认)
router.push("/page", { scroll: true });

// 导航后保持滚动位置
router.push("/page", { scroll: false });
```

**注意事项**:

- 只能在客户端组件中使用
- 需要 `'use client'` 指令
- 不能在服务端组件中使用

**适用场景**:

- 表单提交后导航
- 条件性导航
- 需要编程控制的导航

### 2.2 Link 组件

**功能**: 声明式客户端导航

**类型签名**:

```typescript
import Link from "next/link";

interface LinkProps {
  href: string | UrlObject;
  as?: string;
  replace?: boolean;
  scroll?: boolean;
  shallow?: boolean;
  prefetch?: boolean;
  locale?: string | false;
  // ... 标准 <a> 属性
}

interface UrlObject {
  pathname?: string;
  query?: Record<string, string | string[]>;
  hash?: string;
}
```

**使用示例**:

```typescript
import Link from "next/link";

export default function Page() {
  return (
    <div>
      {/* 基础用法 */}
      <Link href="/about">About</Link>

      {/* 动态路由 */}
      <Link href={`/blog/${post.slug}`}>{post.title}</Link>

      {/* 对象形式 */}
      <Link
        href={{
          pathname: "/search",
          query: { q: "nextjs", category: "docs" },
        }}
      >
        Search
      </Link>

      {/* 禁用预取 */}
      <Link href="/heavy-page" prefetch={false}>
        Heavy Page
      </Link>

      {/* 替换历史记录 */}
      <Link href="/login" replace>
        Login
      </Link>

      {/* 禁用滚动 */}
      <Link href="/page" scroll={false}>
        No Scroll
      </Link>

      {/* 外部链接 */}
      <Link
        href="https://example.com"
        target="_blank"
        rel="noopener noreferrer"
      >
        External
      </Link>
    </div>
  );
}
```

**属性说明**:

| 属性     | 类型                | 默认值 | 说明             |
| :------- | :------------------ | :----- | :--------------- |
| href     | string \| UrlObject | 必填   | 目标路径         |
| replace  | boolean             | false  | 是否替换历史记录 |
| scroll   | boolean             | true   | 是否滚动到顶部   |
| prefetch | boolean             | true   | 是否预取页面     |

**预取行为**:

```typescript
// 生产环境自动预取(默认)
<Link href="/page">Page</Link>

// 禁用预取
<Link href="/page" prefetch={false}>Page</Link>

// 开发环境不预取
// 生产环境在视口中时预取
```

**注意事项**:

- Link 组件会自动处理客户端导航
- 外部链接不会被预取
- 动态路由会自动编码

**适用场景**:

- 页面间导航
- 菜单和导航栏
- 列表项链接

**相关章节**: 02-02-Link 组件详解.md

### 2.3 usePathname Hook

**功能**: 获取当前路径名

**类型签名**:

```typescript
"use client";

import { usePathname } from "next/navigation";

function usePathname(): string;
```

**使用示例**:

```typescript
"use client";

import { usePathname } from "next/navigation";

export default function Navigation() {
  const pathname = usePathname();

  return (
    <nav>
      <Link href="/" className={pathname === "/" ? "active" : ""}>
        Home
      </Link>
      <Link href="/about" className={pathname === "/about" ? "active" : ""}>
        About
      </Link>
    </nav>
  );
}

// 动态路由
export default function Page() {
  const pathname = usePathname();
  // pathname: '/blog/hello-world'

  return <div>Current path: {pathname}</div>;
}
```

**返回值说明**:

| 路由            | pathname 返回值        |
| :-------------- | :--------------------- |
| /               | '/'                    |
| /about          | '/about'               |
| /blog/[slug]    | '/blog/hello-world'    |
| /shop/[...slug] | '/shop/clothes/shirts' |

**注意事项**:

- 只能在客户端组件中使用
- 返回的是实际路径,不是路由模式
- 不包含查询参数和哈希

**适用场景**:

- 高亮当前导航项
- 根据路径显示不同内容
- 面包屑导航

### 2.4 useSearchParams Hook

**功能**: 获取 URL 查询参数

**类型签名**:

```typescript
"use client";

import { useSearchParams } from "next/navigation";

function useSearchParams(): ReadonlyURLSearchParams;

interface ReadonlyURLSearchParams extends URLSearchParams {
  get(name: string): string | null;
  getAll(name: string): string[];
  has(name: string): boolean;
  keys(): IterableIterator<string>;
  values(): IterableIterator<string>;
  entries(): IterableIterator<[string, string]>;
  forEach(callback: (value: string, key: string) => void): void;
}
```

**使用示例**:

```typescript
"use client";

import { useSearchParams } from "next/navigation";

export default function SearchPage() {
  const searchParams = useSearchParams();

  // 获取单个参数
  const query = searchParams.get("q");
  // URL: /search?q=nextjs -> 'nextjs'

  // 获取多个同名参数
  const tags = searchParams.getAll("tag");
  // URL: /search?tag=react&tag=nextjs -> ['react', 'nextjs']

  // 检查参数是否存在
  const hasFilter = searchParams.has("filter");

  // 遍历所有参数
  const params = {};
  searchParams.forEach((value, key) => {
    params[key] = value;
  });

  return (
    <div>
      <p>Search query: {query}</p>
      <p>Tags: {tags.join(", ")}</p>
      <p>Has filter: {hasFilter ? "Yes" : "No"}</p>
    </div>
  );
}

// 更新查询参数
export default function FilterPage() {
  const router = useRouter();
  const pathname = usePathname();
  const searchParams = useSearchParams();

  function updateFilter(key: string, value: string) {
    const params = new URLSearchParams(searchParams.toString());
    params.set(key, value);

    router.push(`${pathname}?${params.toString()}`);
  }

  return (
    <button onClick={() => updateFilter("sort", "price")}>Sort by Price</button>
  );
}
```

**常用方法**:

| 方法         | 说明             | 示例                       |
| :----------- | :--------------- | :------------------------- |
| get(name)    | 获取参数值       | searchParams.get('q')      |
| getAll(name) | 获取所有同名参数 | searchParams.getAll('tag') |
| has(name)    | 检查参数是否存在 | searchParams.has('filter') |
| toString()   | 转换为字符串     | searchParams.toString()    |

**注意事项**:

- 只能在客户端组件中使用
- 返回的是只读对象
- 需要配合 useRouter 更新参数

**适用场景**:

- 搜索页面
- 筛选和排序
- 分页

**相关章节**: 02-05-useSearchParams Hook 详解.md

### 2.5 useParams Hook

**功能**: 获取动态路由参数

**类型签名**:

```typescript
"use client";

import { useParams } from "next/navigation";

function useParams<T = {}>(): T;
```

**使用示例**:

```typescript
"use client";

import { useParams } from "next/navigation";

// app/blog/[slug]/page.tsx
export default function BlogPost() {
  const params = useParams<{ slug: string }>();

  return <div>Post: {params.slug}</div>;
}

// app/shop/[category]/[product]/page.tsx
export default function Product() {
  const params = useParams<{ category: string; product: string }>();

  return (
    <div>
      <p>Category: {params.category}</p>
      <p>Product: {params.product}</p>
    </div>
  );
}

// app/blog/[...slug]/page.tsx
export default function CatchAll() {
  const params = useParams<{ slug: string[] }>();

  return <div>Path: {params.slug.join("/")}</div>;
}
```

**参数类型**:

| 路由模式    | params 类型                  | 示例                      |
| :---------- | :--------------------------- | :------------------------ |
| [slug]      | { slug: string }             | { slug: 'hello' }         |
| [id]/[name] | { id: string; name: string } | { id: '1', name: 'john' } |
| [...slug]   | { slug: string[] }           | { slug: ['a', 'b'] }      |
| [[...slug]] | { slug?: string[] }          | { slug: ['a'] } 或 {}     |

**注意事项**:

- 只能在客户端组件中使用
- 参数值总是字符串或字符串数组
- 需要手动类型转换

**适用场景**:

- 动态路由页面
- 获取路由参数
- 客户端组件中访问参数

## 3. 服务端导航 API (Server-Side Navigation)

### 3.1 redirect 函数

**功能**: 在服务端组件或 Server Actions 中重定向

**类型签名**:

```typescript
import { redirect } from "next/navigation";

function redirect(path: string, type?: "replace" | "push"): never;
```

**使用示例**:

```typescript
import { redirect } from "next/navigation";

// 在服务端组件中
export default async function Page() {
  const session = await getSession();

  if (!session) {
    redirect("/login");
  }

  return <Dashboard />;
}

// 在 Server Actions 中
async function createPost(formData: FormData) {
  "use server";

  const post = await db.post.create({
    data: {
      title: formData.get("title"),
      content: formData.get("content"),
    },
  });

  redirect(`/blog/${post.slug}`);
}

// 替换历史记录
export default async function Page() {
  const user = await getUser();

  if (!user.onboarded) {
    redirect("/onboarding", "replace");
  }

  return <Dashboard />;
}
```

**type 参数**:

| 值            | 行为             | 使用场景     |
| :------------ | :--------------- | :----------- |
| 'push' (默认) | 添加新历史记录   | 一般重定向   |
| 'replace'     | 替换当前历史记录 | 登录后重定向 |

**注意事项**:

- 只能在服务端组件或 Server Actions 中使用
- 会抛出 NEXT_REDIRECT 错误来中断执行
- 不需要 return 语句

**适用场景**:

- 权限检查后重定向
- 表单提交后重定向
- 条件性重定向

**相关章节**: 02-07-redirect 函数详解.md

### 3.2 permanentRedirect 函数

**功能**: 永久重定向 (HTTP 308)

**类型签名**:

```typescript
import { permanentRedirect } from "next/navigation";

function permanentRedirect(path: string): never;
```

**使用示例**:

```typescript
import { permanentRedirect } from "next/navigation";

// 旧 URL 永久重定向到新 URL
export default async function OldPage() {
  permanentRedirect("/new-page");
}

// 在 Server Actions 中
async function migrateUser(userId: string) {
  "use server";

  await db.user.update({
    where: { id: userId },
    data: { migrated: true },
  });

  permanentRedirect("/dashboard");
}
```

**与 redirect 的区别**:

| 特性        | redirect   | permanentRedirect |
| :---------- | :--------- | :---------------- |
| HTTP 状态码 | 307 (临时) | 308 (永久)        |
| 浏览器缓存  | 不缓存     | 缓存重定向        |
| 使用场景    | 临时重定向 | URL 迁移          |

**注意事项**:

- 浏览器会缓存永久重定向
- 谨慎使用,难以撤销
- 适用于 URL 结构变更

**适用场景**:

- URL 结构迁移
- 旧页面废弃
- SEO 优化

## 4. 路由配置 API (Route Configuration)

### 4.1 generateStaticParams 函数

**功能**: 在构建时生成静态路由参数

**类型签名**:

```typescript
export async function generateStaticParams(): Promise<Params[]>;

type Params = Record<string, string | string[]>;
```

**使用示例**:

```typescript
// app/blog/[slug]/page.tsx
export async function generateStaticParams() {
  const posts = await db.post.findMany();

  return posts.map((post) => ({
    slug: post.slug,
  }));
}

export default async function Post({ params }: { params: { slug: string } }) {
  const post = await getPost(params.slug);

  return <article>{post.content}</article>;
}

// 多个动态参数
// app/shop/[category]/[product]/page.tsx
export async function generateStaticParams() {
  const products = await db.product.findMany({
    include: { category: true },
  });

  return products.map((product) => ({
    category: product.category.slug,
    product: product.slug,
  }));
}

// 嵌套路由
// app/blog/[category]/[slug]/page.tsx
export async function generateStaticParams() {
  const categories = await db.category.findMany({
    include: { posts: true },
  });

  return categories.flatMap((category) =>
    category.posts.map((post) => ({
      category: category.slug,
      slug: post.slug,
    }))
  );
}
```

**配合 dynamicParams 使用**:

```typescript
// 只允许预生成的参数
export const dynamicParams = false;

export async function generateStaticParams() {
  return [{ slug: "post-1" }, { slug: "post-2" }];
}

// 访问 /blog/post-3 会返回 404
```

**注意事项**:

- 在构建时执行
- 返回的参数会被静态生成
- 可以配合 dynamicParams 控制行为

**适用场景**:

- 博客文章列表
- 产品目录
- 文档页面

**相关章节**: 02-09-generateStaticParams 函数详解.md

### 4.2 Route Segment Config

**功能**: 配置路由段的行为

**类型签名**:

```typescript
// 动态渲染模式
export const dynamic = 'auto' | 'force-dynamic' | 'error' | 'force-static'

// 动态参数处理
export const dynamicParams = boolean

// 重新验证时间
export const revalidate = false | 0 | number

// 运行时
export const runtime = 'nodejs' | 'edge'

// 首选区域
export const preferredRegion = 'auto' | string | string[]

// 最大持续时间
export const maxDuration = number
```

**使用示例**:

```typescript
// app/dashboard/page.tsx
// 强制动态渲染
export const dynamic = "force-dynamic";

export default async function Dashboard() {
  const data = await getRealtimeData();

  return <div>{data}</div>;
}

// app/blog/page.tsx
// 每小时重新验证
export const revalidate = 3600;

export default async function Blog() {
  const posts = await getPosts();

  return <PostList posts={posts} />;
}

// app/api/route.ts
// 使用 Edge Runtime
export const runtime = "edge";

export async function GET() {
  return Response.json({ message: "Hello from Edge" });
}

// app/page.tsx
// 设置首选区域
export const preferredRegion = ["iad1", "sfo1"];

export default function Home() {
  return <div>Home</div>;
}

// app/api/long-task/route.ts
// 设置最大执行时间(秒)
export const maxDuration = 300; // 5 分钟

export async function POST() {
  await longRunningTask();
  return Response.json({ success: true });
}
```

**dynamic 选项详解**:

| 值              | 行为         | 使用场景   |
| :-------------- | :----------- | :--------- |
| 'auto'          | 自动选择     | 默认值     |
| 'force-dynamic' | 强制动态     | 实时数据   |
| 'force-static'  | 强制静态     | 纯静态页面 |
| 'error'         | 静态失败报错 | 确保静态   |

**revalidate 选项详解**:

| 值     | 行为             | 使用场景   |
| :----- | :--------------- | :--------- |
| false  | 永久缓存         | 静态内容   |
| 0      | 不缓存           | 实时数据   |
| number | 定时重新验证(秒) | 准静态内容 |

**runtime 选项详解**:

| 值       | 特点             | 限制       |
| :------- | :--------------- | :--------- |
| 'nodejs' | 完整 Node.js API | 冷启动较慢 |
| 'edge'   | 快速启动         | API 受限   |

**注意事项**:

- 这些配置只能在页面或路由处理器中导出
- 不能在客户端组件中使用
- 会影响整个路由段

**适用场景**:

- 控制渲染模式
- 配置缓存策略
- 选择运行时环境

### 5.2 page.tsx

**功能**: 定义页面内容

**类型签名**:

### 9.3 如何更新 URL 查询参数?

使用 `useRouter` 和 `useSearchParams`:

```typescript
"use client";

import { useRouter, usePathname, useSearchParams } from "next/navigation";

export default function FilterComponent() {
  const router = useRouter();
  const pathname = usePathname();
  const searchParams = useSearchParams();

  function updateFilter(key: string, value: string) {
    const params = new URLSearchParams(searchParams.toString());
    params.set(key, value);

    router.push(`${pathname}?${params.toString()}`);
  }

  return (
    <button onClick={() => updateFilter("sort", "price")}>Sort by Price</button>
  );
}
```

### 9.4 如何实现面包屑导航?

使用 `usePathname`:

```typescript
"use client";

import { usePathname } from "next/navigation";
import Link from "next/link";

export default function Breadcrumbs() {
  const pathname = usePathname();
  const paths = pathname.split("/").filter(Boolean);

  return (
    <nav>
      <Link href="/">Home</Link>
      {paths.map((path, index) => {
        const href = "/" + paths.slice(0, index + 1).join("/");
        return (
          <span key={href}>
            {" / "}
            <Link href={href}>{path}</Link>
          </span>
        );
      })}
    </nav>
  );
}
```

### 9.7 动态路由和静态路由如何选择?
