# 09-05-元数据 API 速查手册

## 1. 概述 (Overview)

### 1.1 什么是元数据 API

元数据 API 是 Next.js 提供的用于定义页面元数据(如标题、描述、Open Graph 标签等)的接口。良好的元数据对 SEO 和社交媒体分享至关重要。

### 1.2 Next.js 16 元数据变更

**⚠️ 重要变更**:

- generateMetadata 中的 params 现在是 Promise
- 需要使用 await 解析 params
- 支持更多元数据类型

### 1.3 元数据定义方式

| 方式                       | 使用场景      | 优先级 |
| :------------------------- | :------------ | :----- |
| 静态 metadata 对象         | 固定元数据    | 低     |
| 动态 generateMetadata 函数 | 动态元数据    | 高     |
| 文件约定                   | 图标、OG 图片 | 中     |

## 2. 静态元数据 (Static Metadata)

### 2.1 基本元数据

**功能**: 定义页面的基本信息

**类型签名**:

```typescript
export const metadata: Metadata = {
  title: string | TemplateString
  description: string
  keywords: string | string[]
  authors: Author | Author[]
  creator: string
  publisher: string
  robots: string | Robots
  alternates: Alternates
  icons: Icons
  openGraph: OpenGraph
  twitter: Twitter
  verification: Verification
}
```

**使用示例**:

```typescript
// app/layout.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: "My App",
  description: "This is my awesome app",
  keywords: ["Next.js", "React", "TypeScript"],
  authors: [{ name: "John Doe", url: "https://example.com" }],
  creator: "John Doe",
  publisher: "My Company",
};

// app/about/page.tsx
export const metadata: Metadata = {
  title: "About Us",
  description: "Learn more about our company",
};
```

**注意事项**:

- 子页面会继承父布局的元数据
- 子页面可以覆盖父布局的元数据
- title 会自动合并模板

**适用场景**:

- 静态页面
- 固定内容
- 全局配置

**相关章节**: 04-01-静态元数据详解.md

### 2.2 标题模板

**功能**: 定义标题模板

**使用示例**:

```typescript
// app/layout.tsx
export const metadata: Metadata = {
  title: {
    template: "%s | My App",
    default: "My App",
  },
};

// app/about/page.tsx
export const metadata: Metadata = {
  title: "About",
};
// 最终标题: "About | My App"

// app/blog/[slug]/page.tsx
export const metadata: Metadata = {
  title: "Blog Post",
};
// 最终标题: "Blog Post | My App"

// 绝对标题(不使用模板)
export const metadata: Metadata = {
  title: {
    absolute: "Standalone Page",
  },
};
// 最终标题: "Standalone Page"
```

**注意事项**:

- template 中的 %s 会被替换为页面标题
- default 用于没有定义 title 的页面
- absolute 会忽略模板

**适用场景**:

- 统一标题格式
- 品牌一致性
- SEO 优化

### 2.3 Open Graph 元数据

```typescript
      },

    type: "website",
  },

    type: "article",
```

### 2.6 Robots 元数据

**功能**: 控制搜索引擎爬虫

**使用示例**:

```typescript
export const metadata: Metadata = {
  robots: {
    index: true,
    follow: true,
    nocache: false,
    googleBot: {
      index: true,
      follow: true,
      noimageindex: false,
      "max-video-preview": -1,
      "max-image-preview": "large",
      "max-snippet": -1,
    },
  },
};

// 简写形式
export const metadata: Metadata = {
  robots: "index, follow",
};

// 禁止索引
export const metadata: Metadata = {
  robots: {
    index: false,
    follow: false,
  },
};

// 特定爬虫
export const metadata: Metadata = {
  robots: {
    index: true,
    follow: true,
    googleBot: {
      index: false,
    },
  },
};
```

**常用选项**:

| 选项         | 说明         | 默认值 |
| :----------- | :----------- | :----- |
| index        | 允许索引     | true   |
| follow       | 允许跟踪链接 | true   |
| noarchive    | 禁止缓存     | false  |
| nosnippet    | 禁止摘要     | false  |
| noimageindex | 禁止图片索引 | false  |

**注意事项**:

- 影响 SEO
- 谨慎使用 noindex
- 可以针对不同爬虫设置

**适用场景**:

- SEO 控制
- 隐私页面
- 测试环境

## 3. 动态元数据 (Dynamic Metadata)

### 3.1 generateMetadata 函数

**功能**: 根据路由参数生成元数据

**⚠️ Next.js 16 变更**:

```typescript
// ❌ Next.js 15
export async function generateMetadata({ params }) {
  const post = await getPost(params.id);
  return { title: post.title };
}

// ✅ Next.js 16
export async function generateMetadata({ params }) {
  const { id } = await params;
  const post = await getPost(id);
  return { title: post.title };
}
```

**类型签名**:

```typescript
export async function generateMetadata(
  {
    params,
    searchParams,
  }: {
    params: Promise<Record<string, string | string[]>>;
    searchParams: Promise<Record<string, string | string[] | undefined>>;
  },
  parent: ResolvingMetadata
): Promise<Metadata>;
```

**使用示例**:

```typescript
// app/blog/[slug]/page.tsx
import type { Metadata } from "next";

export async function generateMetadata({
  params,
}: {
  params: Promise<{ slug: string }>;
}): Promise<Metadata> {
  const { slug } = await params;
  const post = await fetch(`https://api.example.com/posts/${slug}`).then(
    (res) => res.json()
  );

  return {
    title: post.title,
    description: post.excerpt,
    openGraph: {
      title: post.title,
      description: post.excerpt,
      images: [post.coverImage],
    },
  };
}

// 使用 searchParams
export async function generateMetadata({
  searchParams,
}: {
  searchParams: Promise<{ category?: string }>;
}): Promise<Metadata> {
  const { category } = await searchParams;

  return {
    title: category ? `${category} Posts` : "All Posts",
  };
}

// 使用父元数据
export async function generateMetadata(
  { params }: { params: Promise<{ id: string }> },
  parent: ResolvingMetadata
): Promise<Metadata> {
  const { id } = await params;
  const product = await getProduct(id);

  // 读取父元数据
  const previousImages = (await parent).openGraph?.images || [];

  return {
    title: product.name,
    openGraph: {
      images: [product.image, ...previousImages],
    },
  };
}
```

**注意事项**:

- params 和 searchParams 现在是 Promise
- 必须使用 await 解析
- 可以访问父元数据

**适用场景**:

- 动态页面
- 博客文章
- 产品详情

**相关章节**: 04-08-generateMetadata 详解.md

### 4.2 sitemap.xml

**功能**: 生成网站地图

**文件约定**:

```typescript
// app/sitemap.ts
import { MetadataRoute } from "next";

export default function sitemap(): MetadataRoute.Sitemap {
  return [
    {
      url: "https://example.com",
      lastModified: new Date(),
      changeFrequency: "yearly",
      priority: 1,
    },
    {
      url: "https://example.com/about",
      lastModified: new Date(),
      changeFrequency: "monthly",
      priority: 0.8,
    },
    {
      url: "https://example.com/blog",
      lastModified: new Date(),
      changeFrequency: "weekly",
      priority: 0.5,
    },
  ];
}

// 动态生成
export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  const posts = await fetch("https://api.example.com/posts").then((res) =>
    res.json()
  );

  const postUrls = posts.map((post) => ({
    url: `https://example.com/blog/${post.slug}`,
    lastModified: new Date(post.updatedAt),
    changeFrequency: "weekly" as const,
    priority: 0.7,
  }));

  return [
    {
      url: "https://example.com",
      lastModified: new Date(),
      changeFrequency: "yearly",
      priority: 1,
    },
    ...postUrls,
  ];
}

// 多个 sitemap
// app/sitemap.ts - 主 sitemap
export default function sitemap() {
  return [
    {
      url: "https://example.com/sitemap/posts.xml",
      lastModified: new Date(),
    },
    {
      url: "https://example.com/sitemap/products.xml",
      lastModified: new Date(),
    },
  ];
}

// app/sitemap/posts.xml/route.ts
export async function GET() {
  const posts = await getPosts();

  const sitemap = `<?xml version="1.0" encoding="UTF-8"?>
    <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
      ${posts
        .map(
          (post) => `
        <url>
          <loc>https://example.com/blog/${post.slug}</loc>
          <lastmod>${post.updatedAt}</lastmod>
        </url>
      `
        )
        .join("")}
    </urlset>`;

  return new Response(sitemap, {
    headers: {
      "Content-Type": "application/xml",
    },
  });
}
```

**changeFrequency 选项**:

| 值      | 说明           |
| :------ | :------------- |
| always  | 每次访问都变化 |
| hourly  | 每小时         |
| daily   | 每天           |
| weekly  | 每周           |
| monthly | 每月           |
| yearly  | 每年           |
| never   | 从不变化       |

**注意事项**:

- 帮助搜索引擎爬取
- priority 范围 0.0-1.0
- 大型网站考虑分割

**适用场景**:

- SEO 优化
- 搜索引擎提交
- 网站索引

**相关章节**: 04-11-Sitemap 生成详解.md

### 4.3 robots.txt

```typescript
// app/robots.ts
import { MetadataRoute } from "next";

export default function robots(): MetadataRoute.Robots {
  return {
    rules: {
      userAgent: "*",
      allow: "/",
      disallow: "/private/",
    },
    sitemap: "https://example.com/sitemap.xml",
  };
}

// 多个规则
export default function robots(): MetadataRoute.Robots {
  return {
    rules: [
      {
        userAgent: "Googlebot",
        allow: "/",
        disallow: "/admin/",
        crawlDelay: 2,
      },
      {
        userAgent: "*",
        allow: "/",
        disallow: ["/private/", "/temp/"],
      },
    ],
    sitemap: "https://example.com/sitemap.xml",
  };
}

// 环境相关
export default function robots(): MetadataRoute.Robots {
  const baseUrl = process.env.NEXT_PUBLIC_BASE_URL || "https://example.com";

  if (process.env.NODE_ENV === "production") {
    return {
      rules: {
        userAgent: "*",
        allow: "/",
      },
      sitemap: `${baseUrl}/sitemap.xml`,
    };
  }

  // 开发环境禁止爬取
  return {
    rules: {
      userAgent: "*",
      disallow: "/",
    },
  };
}
```

### 8.5 类型安全注意事项

**使用 TypeScript 类型**:

```typescript
import type { Metadata, ResolvingMetadata } from "next";

export const metadata: Metadata = {
  title: "My Page",
  description: "Page description",
};

export async function generateMetadata(
  {
    params,
    searchParams,
  }: {
    params: Promise<{ id: string }>;
    searchParams: Promise<{ q?: string }>;
  },
  parent: ResolvingMetadata
): Promise<Metadata> {
  const { id } = await params;
  const { q } = await searchParams;

  return {
    title: `Post ${id}`,
    description: q ? `Search: ${q}` : "Default description",
  };
}
```

**自定义元数据类型**:

```typescript
interface CustomMetadata extends Metadata {
  customField?: string;
}

// 注意: Next.js 只会处理标准 Metadata 字段
export const metadata: CustomMetadata = {
  title: "My Page",
  customField: "custom value", // 这个字段不会被 Next.js 处理
};
```

### 9.2 动态元数据如何缓存?

**问题**: generateMetadata 每次都重新执行影响性能

**解决方案**:

```typescript
"use cache";

import { cacheLife, cacheTag } from "next/cache";

export async function generateMetadata({ params }) {
  cacheLife("hours");
  cacheTag("metadata");

  const { slug } = await params;
  const post = await getPost(slug);

  return {
    title: post.title,
    description: post.excerpt,
  };
}

// 需要时清除缓存
import { revalidateTag } from "next/cache";

export async function updatePost(slug: string) {
  await updatePostInDB(slug);
  revalidateTag("metadata");
}
```

### 9.5 如何为不同语言设置不同元数据?

**问题**: 多语言网站需要不同的元数据

**解决方案**:

```typescript
// app/[lang]/layout.tsx
const translations = {
  en: {
    title: "My App",
    description: "Welcome to my app",
  },
  zh: {
    title: "我的应用",
    description: "欢迎使用我的应用",
  },
  ja: {
    title: "マイアプリ",
    description: "マイアプリへようこそ",
  },
};

export async function generateMetadata({ params }) {
  const { lang } = await params;
  const t = translations[lang] || translations.en;

  return {
    title: t.title,
    description: t.description,
    alternates: {
      canonical: `https://example.com/${lang}`,
      languages: {
        "en-US": "https://example.com/en",
        "zh-CN": "https://example.com/zh",
        "ja-JP": "https://example.com/ja",
      },
    },
    openGraph: {
      locale: lang === "zh" ? "zh_CN" : lang === "ja" ? "ja_JP" : "en_US",
    },
  };
}
```

### 9.6 元数据合并规则是什么?

**问题**: 不清楚子页面和父布局的元数据如何合并

**解决方案**:

元数据合并遵循以下规则:

1. 子页面覆盖父布局的同名字段
2. 对象字段会深度合并
3. 数组字段会被完全替换

```typescript
// app/layout.tsx
export const metadata: Metadata = {
  title: {
    template: "%s | My App",
    default: "My App",
  },
  description: "Default description",
  openGraph: {
    siteName: "My App",
    locale: "en_US",
    type: "website",
  },
};

// app/blog/[slug]/page.tsx
export async function generateMetadata({ params }) {
  const { slug } = await params;
  const post = await getPost(slug);

  return {
    title: post.title, // 使用模板: "Post Title | My App"
    description: post.excerpt, // 覆盖父布局的 description
    openGraph: {
      type: "article", // 覆盖父布局的 type
      images: [post.image], // 新增字段
      // siteName 和 locale 继承自父布局
    },
  };
}

// 最终元数据:
// {
//   title: 'Post Title | My App',
//   description: 'Post excerpt',
//   openGraph: {
//     siteName: 'My App',
//     locale: 'en_US',
//     type: 'article',
//     images: ['post-image.jpg']
//   }
// }
```

### 9.7 如何动态生成 OG 图片?

```typescript
// app/blog/[slug]/opengraph-image.tsx
import { ImageResponse } from "next/og";

export const runtime = "edge";
export const alt = "Blog Post";
export const size = {
  width: 1200,
  height: 630,
};
export const contentType = "image/png";

export default async function Image({
  params,
}: {
  params: Promise<{ slug: string }>;

  const { slug } = await params;
  const post = await fetch(`https://api.example.com/posts/${slug}`).then(
    (res) => res.json()

          width: "100%",

          color: "white",
          padding: "40px",
        }}
      >
        <h1 style={{ margin: 0, textAlign: "center" }}>{post.title}</h1>
        <p style={{ fontSize: 30, marginTop: 20, opacity: 0.8 }}>
          {post.author.name}
        </p>
      </div>
    ),
    {
      ...size,
    }
  );
}
```

## 10. 总结

### 10.1 核心要点

元数据 API 是 Next.js 提供的强大功能,用于优化 SEO 和社交媒体分享:

1. **静态元数据**: 使用 metadata 对象定义固定元数据
2. **动态元数据**: 使用 generateMetadata 函数根据路由参数生成元数据
3. **文件约定**: 使用特殊文件(opengraph-image, sitemap.ts 等)生成元数据
4. **元数据合并**: 子页面元数据会与父布局合并
5. **Next.js 16 变更**: params 和 searchParams 现在是 Promise

### 10.2 最佳实践总结

1. **SEO 优化**:

   - 每个页面独特的 title 和 description
   - 使用标题模板保持一致性
   - 合理使用关键词
   - 设置规范 URL

2. **社交媒体优化**:

   - 提供高质量的 OG 图片(1200x630)
   - 设置完整的 Open Graph 和 Twitter 元数据
   - 使用绝对 URL
   - 测试分享效果

3. **性能优化**:

   - 缓存动态元数据生成
   - 使用静态生成
   - 避免重复请求
   - 优化图片大小

4. **多语言支持**:

   - 使用 alternates 定义语言版本
   - 设置正确的 locale
   - 提供语言切换链接

5. **类型安全**:
   - 使用 TypeScript 类型
   - 正确定义 params 和 searchParams 类型
   - 利用 IDE 自动补全

### 10.3 迁移到 Next.js 16

如果从 Next.js 15 迁移到 16,需要注意:

1. 所有 generateMetadata 中的 params 和 searchParams 都需要 await
2. 更新类型定义为 Promise
3. 测试所有动态路由的元数据生成
4. 检查是否有依赖同步访问 params 的代码
