# 09-08-工具函数 API 速查手册

## 1. 概述 (Overview)

### 1.1 什么是工具函数

工具函数是 Next.js 提供的一组辅助函数,用于处理常见的服务器端任务,如访问请求头、cookies、重定向、错误处理等。这些函数简化了服务器端开发,提供了类型安全的 API。

### 1.2 Next.js 16 工具函数特性

**核心特性**:

- cookies(): 访问和设置 cookies (🆕 异步)
- headers(): 访问请求头 (🆕 异步)
- redirect(): 服务器端重定向
- notFound(): 触发 404 页面
- unstable_noStore(): 禁用缓存
- draftMode(): 草稿模式

### 1.3 工具函数分类

| 类别     | 函数                             | 用途         | 执行环境 |
| :------- | :------------------------------- | :----------- | :------- |
| 请求处理 | cookies, headers                 | 访问请求信息 | 服务器   |
| 导航     | redirect, permanentRedirect      | 重定向       | 服务器   |
| 错误处理 | notFound, unstable_after         | 错误和后处理 | 服务器   |
| 缓存控制 | unstable_noStore, unstable_cache | 缓存管理     | 服务器   |
| 内容管理 | draftMode                        | 草稿预览     | 服务器   |

## 2. cookies() 函数

### 2.1 基本用法

**功能**: 读取和设置 HTTP cookies

**类型签名**:

```typescript
// Next.js 16 - 异步函数 🆕
function cookies(): Promise<ReadonlyRequestCookies>;

interface ReadonlyRequestCookies {
  get(name: string): RequestCookie | undefined;
  getAll(): RequestCookie[];
  has(name: string): boolean;
  set(name: string, value: string, options?: CookieOptions): void;
  delete(name: string): void;
}

interface RequestCookie {
  name: string;
  value: string;
}

interface CookieOptions {
  domain?: string;
  expires?: Date;
  httpOnly?: boolean;
  maxAge?: number;
  path?: string;
  sameSite?: "strict" | "lax" | "none";
  secure?: boolean;
}
```

**使用示例**:

```typescript
// Server Component
import { cookies } from "next/headers";

export default async function Page() {
  const cookieStore = await cookies();
  const theme = cookieStore.get("theme");

  return <div>Current theme: {theme?.value}</div>;
}

// Server Action
("use server");

import { cookies } from "next/headers";

export async function setTheme(theme: string) {
  const cookieStore = await cookies();

  cookieStore.set("theme", theme, {
    httpOnly: true,
    secure: process.env.NODE_ENV === "production",
    sameSite: "lax",
    maxAge: 60 * 60 * 24 * 365, // 1 year
  });
}

// Route Handler
import { cookies } from "next/headers";

export async function GET() {
  const cookieStore = await cookies();
  const token = cookieStore.get("token");

  if (!token) {
    return new Response("Unauthorized", { status: 401 });
  }

  return Response.json({ authenticated: true });
}
```

**注意事项**:

- Next.js 16 中 cookies() 是异步函数,必须 await
- 只能在 Server Components、Server Actions 和 Route Handlers 中使用
- 设置 cookie 后,当前请求中无法立即读取新值
- httpOnly cookies 无法在客户端访问

**适用场景**:

- 用户认证
- 主题设置
- 语言偏好
- 会话管理

### 2.2 读取 Cookies

**功能**: 获取 cookie 值

**使用示例**:

```typescript
import { cookies } from "next/headers";

export default async function Page() {
  const cookieStore = await cookies();

  // 获取单个 cookie
  const sessionCookie = cookieStore.get("session");
  console.log(sessionCookie?.value);

  // 获取所有 cookies
  const allCookies = cookieStore.getAll();
  console.log(allCookies);

  // 检查 cookie 是否存在
  const hasSession = cookieStore.has("session");

  return <div>{hasSession ? "Logged in" : "Not logged in"}</div>;
}

// 使用默认值
export async function getTheme() {
  const cookieStore = await cookies();
  const theme = cookieStore.get("theme");
  return theme?.value || "light"; // 默认值
}

// 解析 JSON cookie
export async function getUserPreferences() {
  const cookieStore = await cookies();
  const prefsCookie = cookieStore.get("preferences");

  if (!prefsCookie) {
    return null;
  }

  try {
    return JSON.parse(prefsCookie.value);
  } catch {
    return null;
  }
}
```

```typescript
export async function clearExpiredCookies() {
  const cookieStore = await cookies();
  const allCookies = cookieStore.getAll();

      cookieStore.delete(cookie.name);
    }
  });
}
```

## 3. headers() 函数

### 3.1 基本用法

**功能**: 读取 HTTP 请求头

**类型签名**:

```typescript
// Next.js 16 - 异步函数 🆕
function headers(): Promise<ReadonlyHeaders>;

interface ReadonlyHeaders {
  get(name: string): string | null;
  has(name: string): boolean;
  forEach(callback: (value: string, key: string) => void): void;
  entries(): IterableIterator<[string, string]>;
  keys(): IterableIterator<string>;
  values(): IterableIterator<string>;
}
```

**使用示例**:

```typescript
// Server Component
import { headers } from "next/headers";

export default async function Page() {
  const headersList = await headers();
  const userAgent = headersList.get("user-agent");

  return <div>User Agent: {userAgent}</div>;
}

// Server Action
("use server");

import { headers } from "next/headers";

export async function logRequest() {
  const headersList = await headers();
  const referer = headersList.get("referer");
  const ip = headersList.get("x-forwarded-for");

  console.log(`Request from ${ip}, referer: ${referer}`);
}

// Route Handler
import { headers } from "next/headers";

export async function GET() {
  const headersList = await headers();
  const authorization = headersList.get("authorization");

  if (!authorization) {
    return new Response("Unauthorized", { status: 401 });
  }

  return Response.json({ authenticated: true });
}
```

**注意事项**:

- Next.js 16 中 headers() 是异步函数,必须 await
- 只能读取,不能修改请求头
- 只能在服务器端使用
- 请求头名称不区分大小写

**适用场景**:

- 用户代理检测
- IP 地址获取
- 认证令牌验证
- 语言检测

### 3.2 常用请求头

**功能**: 访问常见的请求头

**使用示例**:

```typescript
import { headers } from "next/headers";

export default async function Page() {
  const headersList = await headers();

  // User Agent
  const userAgent = headersList.get("user-agent");
  const isMobile = /mobile/i.test(userAgent || "");

  // IP Address
  const ip =
    headersList.get("x-forwarded-for") ||
    headersList.get("x-real-ip") ||
    "unknown";

  // Referer
  const referer = headersList.get("referer");

  // Accept Language
  const acceptLanguage = headersList.get("accept-language");
  const preferredLanguage = acceptLanguage?.split(",")[0];

  // Authorization
  const authorization = headersList.get("authorization");
  const token = authorization?.replace("Bearer ", "");

  // Content Type
  const contentType = headersList.get("content-type");

  // Host
  const host = headersList.get("host");

  return (
    <div>
      <p>Mobile: {isMobile ? "Yes" : "No"}</p>
      <p>IP: {ip}</p>
      <p>Language: {preferredLanguage}</p>
      <p>Host: {host}</p>
    </div>
  );
}

// 检测设备类型
export async function getDeviceType() {
  const headersList = await headers();
  const userAgent = headersList.get("user-agent") || "";

  if (/mobile/i.test(userAgent)) return "mobile";
  if (/tablet/i.test(userAgent)) return "tablet";
  return "desktop";
}

// 获取客户端 IP
export async function getClientIP() {
  const headersList = await headers();

  return (
    headersList.get("x-forwarded-for")?.split(",")[0].trim() ||
    headersList.get("x-real-ip") ||
    headersList.get("cf-connecting-ip") || // Cloudflare
    "unknown"
  );
}
```

## 4. redirect() 函数

### 4.1 基本用法

**功能**: 服务器端重定向

**类型签名**:

```typescript
function redirect(url: string, type?: "replace" | "push"): never;

// 永久重定向
function permanentRedirect(url: string, type?: "replace" | "push"): never;
```

**使用示例**:

```typescript
// Server Component
import { redirect } from "next/navigation";

export default async function Page() {
  const user = await getUser();

  if (!user) {
    redirect("/login");
  }

  return <div>Welcome {user.name}</div>;
}

// Server Action
("use server");

import { redirect } from "next/navigation";

export async function createPost(formData: FormData) {
  const post = await db.post.create({
    data: {
      title: formData.get("title") as string,
    },
  });

  redirect(`/posts/${post.id}`);
}

// Route Handler
import { redirect } from "next/navigation";

export async function GET(request: Request) {
  const { searchParams } = new URL(request.url);
  const token = searchParams.get("token");

  if (!token) {
    redirect("/login");
  }

  return Response.json({ success: true });
}
```

**注意事项**:

- redirect() 会抛出错误,终止当前执行
- 默认返回 307 (临时重定向)
- permanentRedirect() 返回 308 (永久重定向)
- 可以重定向到相对或绝对 URL

**适用场景**:

- 认证检查
- 表单提交后跳转
- 权限验证
- URL 规范化

```typescript
// 规范化 URL
export default async function Page({ params }: { params: { slug: string } }) {
  const { slug } = await params;

  // 强制小写

    permanentRedirect(`/posts/${slug.toLowerCase()}`);
  }

  const post = await getPost(slug);
  return <div>{post.title}</div>;

// 移除尾部斜杠
export async function GET(request: Request) {
  const url = new URL(request.url);

  if (url.pathname.endsWith("/") && url.pathname !== "/") {
    permanentRedirect(url.pathname.slice(0, -1));
  }

  return Response.json({ success: true });
}
```

## 7. draftMode() 函数

### 7.1 基本用法

**功能**: 启用草稿模式预览

**类型签名**:

```typescript
function draftMode(): Promise<DraftMode>;

interface DraftMode {
  isEnabled: boolean;
  enable(): void;
  disable(): void;
}
```

**使用示例**:

```typescript
// 启用草稿模式
// app/api/draft/route.ts
import { draftMode } from "next/headers";

export async function GET(request: Request) {
  const { searchParams } = new URL(request.url);
  const secret = searchParams.get("secret");

  if (secret !== process.env.DRAFT_SECRET) {
    return new Response("Invalid token", { status: 401 });

  const draft = await draftMode();
  draft.enable();

  return new Response("Draft mode enabled");

// 禁用草稿模式
// app/api/draft/disable/route.ts
import { draftMode } from "next/headers";

export async function GET() {
  const draft = await draftMode();
  draft.disable();

  return new Response("Draft mode disabled");
}

// 使用草稿模式
// app/posts/[slug]/page.tsx
import { draftMode } from "next/headers";

export default async function Page({ params }: { params: { slug: string } }) {
  const { slug } = await params;
  const draft = await draftMode();

  const post = await getPost(slug, draft.isEnabled);

  return (
    <div>
      {draft.isEnabled && (
        <div className="draft-banner">Draft Mode Enabled</div>
      )}
      <h1>{post.title}</h1>
      <div>{post.content}</div>
    </div>
  );
}

// 获取草稿内容
async function getPost(slug: string, isDraft: boolean) {
  if (isDraft) {
    // 获取草稿版本
    return await cms.getPost(slug, { draft: true });
  }

  // 获取发布版本
  return await cms.getPost(slug, { draft: false });
}
```

**注意事项**:

- 草稿模式使用 cookie 存储状态
- 禁用页面缓存
- 适合 CMS 预览
- 需要保护启用端点

**适用场景**:

- CMS 内容预览
- 草稿文章预览
- 未发布内容查看
- 编辑器预览

### 8.2 多语言支持

**场景描述**: 基于请求头的语言检测

**实现方案**:

```typescript
import { headers, cookies } from "next/headers";

export async function getLocale() {
  // 优先使用 cookie 中的语言设置
  const cookieStore = await cookies();
  const localeCookie = cookieStore.get("locale");

  if (localeCookie) {
    return localeCookie.value;
  }

  // 从 Accept-Language 头检测
  const headersList = await headers();
  const acceptLanguage = headersList.get("accept-language");

  if (acceptLanguage) {
    const languages = acceptLanguage.split(",");
    const preferred = languages[0].split(";")[0].trim();

    // 支持的语言列表
    const supported = ["en", "zh", "ja", "ko"];
    const locale = preferred.split("-")[0];

    if (supported.includes(locale)) {
      return locale;
    }
  }

  // 默认语言
  return "en";
}

// 设置语言
("use server");

export async function setLocale(locale: string) {
  const cookieStore = await cookies();

  cookieStore.set("locale", locale, {
    maxAge: 60 * 60 * 24 * 365, // 1 year
  });
}

// 使用
export default async function Page() {
  const locale = await getLocale();
  const translations = await getTranslations(locale);

  return <div>{translations.welcome}</div>;
}
```

### 9.2 错误处理

```typescript
  if (!user) {
    redirect("/login");
  }

}

// cookies 和 headers 的错误处理
export default async function Page() {

    console.error("Failed to read cookies:", error);
```

## 10. 常见问题

### 10.1 为什么 cookies() 和 headers() 变成异步了?

**问题**: Next.js 16 中的破坏性变更

**解答**:

Next.js 16 将 `cookies()` 和 `headers()` 改为异步函数是为了支持异步请求上下文(Async Request Context)。这个变更带来以下好处:

1. **更好的性能**: 异步上下文可以更好地利用 React 的并发特性
2. **一致性**: 与其他异步 API 保持一致
3. **未来兼容**: 为未来的 React 特性做准备

迁移方法:

```typescript
// 旧代码
const cookieStore = cookies();

// 新代码
const cookieStore = await cookies();

// 组件也需要改为 async
export default async function Page() {
  const cookieStore = await cookies();
  return <div>...</div>;
}
```

### 10.2 如何在客户端组件中使用 cookies?

**问题**: 客户端组件无法使用 cookies() 函数

**解决方案**:

```typescript
// 方法1: 通过 props 传递
// app/page.tsx (Server Component)
import { cookies } from "next/headers";

export default async function Page() {
  const cookieStore = await cookies();
  const theme = cookieStore.get("theme")?.value || "light";

  return <ClientComponent theme={theme} />;
}

// app/ClientComponent.tsx (Client Component)
("use client");

export function ClientComponent({ theme }: { theme: string }) {
  return <div className={theme}>Content</div>;
}

// 方法2: 使用 document.cookie (客户端)
("use client");

export function ClientComponent() {
  const theme =
    document.cookie
      .split("; ")
      .find((row) => row.startsWith("theme="))
      ?.split("=")[1] || "light";

  return <div className={theme}>Content</div>;
}

// 方法3: 使用 js-cookie 库
("use client");

import Cookies from "js-cookie";

export function ClientComponent() {
  const theme = Cookies.get("theme") || "light";

  function setTheme(newTheme: string) {
    Cookies.set("theme", newTheme, { expires: 365 });
  }

  return (
    <div className={theme}>
      <button onClick={() => setTheme("dark")}>Dark</button>
      <button onClick={() => setTheme("light")}>Light</button>
    </div>
  );
}
```

### 10.3 redirect() 和 router.push() 有什么区别?

**问题**: 两种导航方式的选择

**解答**:

| 特性     | redirect()                        | router.push()        |
| :------- | :-------------------------------- | :------------------- |
| 执行环境 | 服务器端                          | 客户端               |
| 使用位置 | Server Components, Server Actions | Client Components    |
| 行为     | 抛出错误,终止执行                 | 返回 Promise         |
| 状态码   | 307/308                           | 无(客户端导航)       |
| SEO      | 友好                              | 友好                 |
| 适用场景 | 认证检查、表单提交                | 用户交互、客户端导航 |

使用示例:

```typescript
// redirect() - 服务器端
import { redirect } from "next/navigation";

export default async function Page() {
  const user = await getUser();

  if (!user) {
    redirect("/login"); // 服务器端重定向
  }

  return <div>Welcome</div>;
}

// router.push() - 客户端
("use client");

import { useRouter } from "next/navigation";

export function LoginButton() {
  const router = useRouter();

  return <button onClick={() => router.push("/login")}>Login</button>;
}
```

### 10.5 notFound() 和 redirect('/404') 有什么区别?

**问题**: 404 处理的最佳实践

**解答**:

```typescript
// ✅ 推荐 - 使用 notFound()
import { notFound } from "next/navigation";

export default async function Page({ params }: { params: { id: string } }) {
  const { id } = await params;
  const post = await getPost(id);

  if (!post) {
    notFound(); // 渲染 not-found.tsx,返回 404 状态码
  }

  return <div>{post.title}</div>;
}

// ❌ 不推荐 - 重定向到 404 页面
import { redirect } from "next/navigation";

export default async function Page({ params }: { params: { id: string } }) {
  const { id } = await params;
  const post = await getPost(id);

  if (!post) {
    redirect("/404"); // 返回 307 状态码,不是 404
  }

  return <div>{post.title}</div>;
}
```

**区别**:

| 特性   | notFound()    | redirect('/404') |
| :----- | :------------ | :--------------- |
| 状态码 | 404           | 307              |
| SEO    | 正确          | 不正确           |
| 渲染   | not-found.tsx | /404 页面        |
| URL    | 保持不变      | 变为 /404        |
| 推荐   | ✅            | ❌               |

## 11. 总结

### 11.1 核心要点

工具函数是 Next.js 16 服务器端开发的基础:

1. **cookies()**: 读写 HTTP cookies (🆕 异步)
2. **headers()**: 读取请求头 (🆕 异步)
3. **redirect()**: 服务器端重定向
4. **notFound()**: 触发 404 页面
5. **unstable_noStore()**: 禁用缓存
6. **draftMode()**: 草稿模式预览

### 11.2 最佳实践

1. **异步处理**:

   - 始终 await cookies() 和 headers()
   - 组件必须是 async 函数
   - 注意 Next.js 16 的破坏性变更

2. **错误处理**:

   - 不要捕获 redirect() 和 notFound() 的错误
   - 这些函数会抛出特殊错误供 Next.js 处理
   - 正常的错误处理仍然需要

3. **性能优化**:

   - 避免重复调用工具函数
   - 在组件顶部调用一次
   - 合理使用缓存

4. **安全性**:
   - httpOnly cookies 防止 XSS
   - secure cookies 仅在 HTTPS
   - 验证用户输入
   - 保护敏感端点
