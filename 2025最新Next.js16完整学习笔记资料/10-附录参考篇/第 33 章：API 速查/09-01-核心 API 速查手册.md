## 2. 路由与导航 API (Routing & Navigation)

### 2.1 useRouter Hook

**功能**: 在客户端组件中访问路由对象,执行导航操作

**类型签名**:

```typescript
function useRouter(): AppRouterInstance;

interface AppRouterInstance {
  push(href: string, options?: NavigateOptions): void;
  replace(href: string, options?: NavigateOptions): void;
  refresh(): void;
  prefetch(href: string): void;
  back(): void;
  forward(): void;
}

interface NavigateOptions {
  scroll?: boolean;
}
```

**使用示例**:

```typescript
"use client";

import { useRouter } from "next/navigation";

export default function Page() {
  const router = useRouter();

  const handleClick = () => {
    router.push("/dashboard");
  };

  return <button onClick={handleClick}>Go to Dashboard</button>;
}
```

**注意事项**:

- 只能在客户端组件中使用
- 需要从 `next/navigation` 导入,不是 `next/router`
- ⚠️ Next.js 16 变更: 移除了 `pathname`、`query` 等属性,需使用专用 Hook

**功能**: Next.js 扩展的 fetch API,支持缓存和重新验证

**类型签名**:

```typescript
function fetch(
  input: RequestInfo | URL,
  init?: RequestInit & {
    next?: {
      revalidate?: number | false;
      tags?: string[];
    };
  }

```

**使用示例**:

```typescript
// 缓存 1 小时

  next: { revalidate: 3600 },

  next: { tags: ["products"] },

// ⚠️ Next.js 16 变更: 默认不缓存,需要显式指定

});
```

**注意事项**:

- ⚠️ Next.js 16 变更: 默认缓存策略从 `force-cache` 改为 `no-store`
- `revalidate: false` 表示永久缓存
- `tags` 用于按需重新验证

### 3.3 cookies 函数

**功能**: 在服务端读取和设置 Cookie

**类型签名**:

```typescript
function cookies(): Promise<ReadonlyRequestCookies>;

interface ReadonlyRequestCookies {
  get(name: string): { name: string; value: string } | undefined;
  getAll(name?: string): { name: string; value: string }[];
  has(name: string): boolean;
  set(name: string, value: string, options?: CookieOptions): void;
  delete(name: string): void;
}

interface CookieOptions {
  maxAge?: number;
  expires?: Date;
  path?: string;
  domain?: string;
  secure?: boolean;
  httpOnly?: boolean;
  sameSite?: "strict" | "lax" | "none";
}
```

**使用示例**:

```typescript
import { cookies } from "next/headers";

export default async function Page() {
  const cookieStore = await cookies();
  const theme = cookieStore.get("theme");

  return <div>Current theme: {theme?.value}</div>;
}

// 在 Server Action 中设置 Cookie
async function updateTheme(formData: FormData) {
  "use server";

  const cookieStore = await cookies();
  cookieStore.set("theme", formData.get("theme") as string, {
    maxAge: 60 * 60 * 24 * 365, // 1 年
    path: "/",
    httpOnly: true,
    secure: process.env.NODE_ENV === "production",
  });
}
```

**注意事项**:

- ⚠️ Next.js 16 变更: 现在是异步函数,需要 await
- 在服务端组件中只能读取,不能设置
- 在 Server Actions 和 Route Handlers 中可以设置

```typescript
function cacheLife(profile: CacheLifeProfile): void;
```

```typescript
    expire: 3600, // 1 小时后过期
  });

}
```

### 6.2 useActionState Hook

**功能**: 管理 Server Action 的状态

**类型签名**:

```typescript
function useActionState<State, Payload>(
  action: (state: State, payload: Payload) => Promise<State>,
  initialState: State,
  permalink?: string
): [state: State, dispatch: (payload: Payload) => void, isPending: boolean];
```

**使用示例**:

```typescript
"use client";

import { useActionState } from "react";

async function updateUser(prevState: any, formData: FormData) {
  "use server";

  try {
    await db.user.update({
      where: { id: formData.get("id") },
      data: { name: formData.get("name") },
    });

    return { success: true, message: "User updated" };
  } catch (error) {
    return { success: false, message: "Update failed" };

export default function UserForm() {
  const [state, formAction, isPending] = useActionState(updateUser, {
    success: false,
    message: "",
  });

  return (
    <form action={formAction}>
      <input name="name" />
      <button disabled={isPending}>{isPending ? "Saving..." : "Save"}</button>
      {state.message && <p>{state.message}</p>}
    </form>
  );
}
```

**注意事项**:

- 替代了旧的 `useFormState` Hook
- 可以在客户端组件中使用
- 提供了 `isPending` 状态

### 6.3 useOptimistic Hook

**功能**: 实现乐观更新

**类型签名**:

```typescript
function useOptimistic<State, UpdateType = State>(
  passthrough: State,
  reducer?: (state: State, update: UpdateType) => State
): [State, (update: UpdateType) => void];
```

**使用示例**:

```typescript
"use client";

import { useOptimistic } from "react";

export default function TodoList({ todos }: { todos: Todo[] }) {
  const [optimisticTodos, addOptimisticTodo] = useOptimistic(
    todos,
    (state, newTodo: Todo) => [...state, newTodo]
  );

  async function handleAdd(formData: FormData) {
    const newTodo = {
      id: Date.now(),
      text: formData.get("text") as string,
      completed: false,
    };

    // 立即更新 UI
    addOptimisticTodo(newTodo);

    // 发送到服务器
    await createTodo(newTodo);
  }

  return (
    <div>
      <ul>
        {optimisticTodos.map((todo) => (
          <li key={todo.id}>{todo.text}</li>
        ))}
      </ul>
      <form action={handleAdd}>
        <input name="text" />
        <button>Add</button>
      </form>
    </div>
  );
}
```

**注意事项**:

- 用于提升用户体验
- 需要处理失败回滚
- 适用于低延迟操作

## 11. 常见问题 (FAQ)

### 11.1 为什么某些 API 需要 await?

**问题**: Next.js 16 中 `headers()`、`cookies()` 等函数变为异步

**原因**: 为了支持动态渲染和更好的性能优化

**解决**: 在调用这些函数时添加 `await` 关键字

```typescript
// ❌ Next.js 15
const headersList = headers();

// ✅ Next.js 16
const headersList = await headers();
```

### 11.2 fetch 默认不缓存了怎么办?

**问题**: ⚠️ Next.js 16 中 fetch 默认策略从 `force-cache` 改为 `no-store`

**影响**: 可能导致性能下降

**解决**: 显式指定缓存策略

```typescript
// 需要缓存的请求
const res = await fetch("https://api.example.com/data", {
  cache: "force-cache",
});

// 或使用 next.revalidate
const res = await fetch("https://api.example.com/data", {
  next: { revalidate: 3600 },
});
```

### 11.3 如何选择合适的缓存 API?

**场景一: 缓存整个页面**

- 使用静态生成或 ISR
- 配置 `revalidate` 选项

**场景二: 缓存特定数据**

- 使用 `unstable_cache` 包装数据获取函数
- 或使用 `'use cache'` 指令配合 `cacheLife`

**场景三: 按需重新验证**

- 使用 `cacheTag` 标记缓存
- 在数据更新时调用 `revalidateTag`

### 11.4 useRouter 和 usePathname 有什么区别?

**useRouter**: 提供导航方法(push、replace 等)

**usePathname**: 只返回当前路径字符串

**使用建议**:

- 需要导航时使用 `useRouter`
- 只需要读取路径时使用 `usePathname`(性能更好)

### 11.5 Server Actions 和 Route Handlers 如何选择?

**Server Actions**:

- 适用于表单提交
- 与 React 表单集成更好
- 自动处理 CSRF 保护

**Route Handlers**:

- 适用于 RESTful API
- 支持所有 HTTP 方法
- 更灵活的请求/响应处理
