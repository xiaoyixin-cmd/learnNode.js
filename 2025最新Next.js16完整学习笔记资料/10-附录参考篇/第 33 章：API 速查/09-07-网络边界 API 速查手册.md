# 09-07-网络边界 API 速查手册

## 1. 概述 (Overview)

### 1.1 什么是网络边界

网络边界(Network Boundary)是 Next.js 中区分服务器端代码和客户端代码的概念。通过明确的边界标识,Next.js 可以优化代码分割、数据获取和渲染策略。

### 1.2 Next.js 16 网络边界特性

**核心概念**:

- Server Components: 默认在服务器端渲染
- Client Components: 在客户端渲染和交互
- 'use client': 客户端边界标识
- 'use server': 服务器端边界标识
- 混合渲染: 服务器和客户端组件可以组合使用

### 1.3 组件类型对比

| 特性        | Server Components | Client Components          |
| :---------- | :---------------- | :------------------------- |
| 默认类型    | 是                | 否,需要 'use client'       |
| 数据获取    | 直接访问数据库    | 通过 API 或 Server Actions |
| 交互性      | 无                | 支持 hooks 和事件          |
| Bundle 大小 | 不计入客户端      | 计入客户端                 |
| 执行环境    | 服务器            | 客户端                     |
| 适用场景    | 数据展示、SEO     | 用户交互、状态管理         |

## 2. 'use client' 指令

### 2.1 基本用法

**功能**: 标记组件为客户端组件

**语法**:

```typescript
"use client";

import { useState } from "react";

export default function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}
```

**使用规则**:

- 必须在文件顶部
- 在任何 import 之前
- 只需在边界组件中声明
- 子组件自动继承

**示例**:

```typescript
// app/components/InteractiveButton.tsx
"use client";

import { useState } from "react";

export function InteractiveButton() {
  const [clicked, setClicked] = useState(false);

  return (
    <button onClick={() => setClicked(true)}>
      {clicked ? "Clicked!" : "Click me"}
    </button>
  );
}

// app/page.tsx (Server Component)
import { InteractiveButton } from "./components/InteractiveButton";

export default function Page() {
  // 这是服务器组件
  const data = await fetchData();

  return (
    <div>
      <h1>{data.title}</h1>
      <InteractiveButton /> {/* 客户端组件 */}
    </div>
  );
}
```

**注意事项**:

- 客户端组件会增加 bundle 大小
- 不能直接访问服务器端资源
- 可以使用 React hooks
- 支持浏览器 API

**适用场景**:

- 需要交互的组件
- 使用 useState、useEffect 等 hooks
- 需要访问浏览器 API
- 事件处理

**相关章节**: 02-01-Server Components 与 Client Components.md

### 2.2 客户端组件限制

**功能**: 了解客户端组件的限制

**限制列表**:

```typescript
// ❌ 不能在客户端组件中直接访问服务器资源
"use client";

import { db } from "@/lib/db"; // 错误!

export function UserList() {
  // ❌ 不能直接查询数据库
  const users = await db.user.findMany();

  return <div>{users.map((u) => u.name)}</div>;
}

// ✅ 正确做法 - 通过 props 传递数据
// app/page.tsx (Server Component)
import { UserList } from "./components/UserList";

export default async function Page() {
  const users = await db.user.findMany();

  return <UserList users={users} />;
}

// app/components/UserList.tsx (Client Component)
("use client");

export function UserList({ users }: { users: User[] }) {
  return <div>{users.map((u) => u.name)}</div>;
}
```

**不能使用的功能**:

- 直接数据库访问
- 服务器端环境变量(非 NEXT*PUBLIC*)
- Node.js 特定 API
- 服务器端文件系统操作

**可以使用的功能**:

- React hooks (useState, useEffect, etc.)
- 浏览器 API (window, document, etc.)
- 事件处理
- 客户端状态管理

### 2.3 客户端组件数据获取

**功能**: 在客户端组件中获取数据

**使用示例**:

```typescript
// 方法1: 使用 useEffect
"use client";

import { useState, useEffect } from "react";

export function Posts() {
  const [posts, setPosts] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch("/api/posts")
      .then((res) => res.json())
      .then((data) => {
        setPosts(data);
        setLoading(false);
      });
  }, []);

  if (loading) return <div>Loading...</div>;

  return (
    <div>
      {posts.map((post) => (
        <div key={post.id}>{post.title}</div>
      ))}
    </div>
  );
}

// 方法2: 使用 SWR
("use client");

import useSWR from "swr";

const fetcher = (url: string) => fetch(url).then((r) => r.json());

export function Posts() {
  const { data, error, isLoading } = useSWR("/api/posts", fetcher);

  if (error) return <div>Failed to load</div>;
  if (isLoading) return <div>Loading...</div>;

  return (
    <div>
      {data.map((post) => (
        <div key={post.id}>{post.title}</div>
      ))}
    </div>
  );
}

// 方法3: 使用 React Query
("use client");

import { useQuery } from "@tanstack/react-query";

export function Posts() {
  const { data, isLoading, error } = useQuery({
    queryKey: ["posts"],
    queryFn: () => fetch("/api/posts").then((r) => r.json()),
  });

  if (isLoading) return <div>Loading...</div>;
  if (error) return <div>Error loading posts</div>;

  return (
    <div>
      {data.map((post) => (
        <div key={post.id}>{post.title}</div>
      ))}
    </div>
  );
}
```

**注意事项**:

- 客户端数据获取会增加加载时间
- 考虑使用 Server Components 预加载数据
- 使用数据获取库(SWR, React Query)优化体验
- 处理加载和错误状态

## 3. 'use server' 指令

### 3.1 Server Actions

**功能**: 定义服务器端函数

**基本语法**:

```typescript
// app/actions/post.ts
"use server";

export async function createPost(formData: FormData) {
  const title = formData.get("title") as string;
  const content = formData.get("content") as string;

  await db.post.create({
    data: { title, content },
  });

  revalidatePath("/posts");
}

// 使用
// app/components/PostForm.tsx
("use client");

import { createPost } from "@/app/actions/post";

export function PostForm() {
  return (
    <form action={createPost}>
      <input type="text" name="title" />
      <textarea name="content" />
      <button type="submit">Create</button>
    </form>
  );
}
```

**内联 Server Action**:

```typescript
// app/page.tsx
export default function Page() {
  async function handleSubmit(formData: FormData) {
    "use server";

    const data = formData.get("data");
    await saveToDatabase(data);
  }

  return (
    <form action={handleSubmit}>
      <input type="text" name="data" />
      <button type="submit">Submit</button>
    </form>
  );
}
```

**注意事项**:

- 只能在服务器端执行
- 可以访问数据库和服务器资源
- 自动处理序列化
- 支持 FormData 和普通参数

**适用场景**:

- 表单提交
- 数据修改
- 服务器端操作
- API 调用

**相关章节**: 05-01-Server Actions 详解.md

## 5. 混合渲染策略

### 5.1 选择合适的组件类型

**决策流程**:

```typescript
// 需要交互? → Client Component
"use client";

export function InteractiveForm() {
  const [value, setValue] = useState("");
  return <input value={value} onChange={(e) => setValue(e.target.value)} />;
}

// 需要 hooks? → Client Component
("use client");

export function Timer() {
  const [time, setTime] = useState(Date.now());

  useEffect(() => {
    const interval = setInterval(() => setTime(Date.now()), 1000);
    return () => clearInterval(interval);
  }, []);

  return <div>{new Date(time).toLocaleTimeString()}</div>;
}

// 需要浏览器 API? → Client Component
("use client");

export function GeolocationDisplay() {
  const [location, setLocation] = useState<GeolocationPosition | null>(null);

  useEffect(() => {
    navigator.geolocation.getCurrentPosition(setLocation);
  }, []);

  return <div>{location?.coords.latitude}</div>;
}

// 需要访问数据库? → Server Component
export async function UserList() {
  const users = await db.user.findMany();
  return (
    <ul>
      {users.map((u) => (
        <li key={u.id}>{u.name}</li>
      ))}
    </ul>
  );
}

// 需要 SEO? → Server Component
export async function BlogPost({ params }: { params: { slug: string } }) {
  const { slug } = await params;
  const post = await db.post.findUnique({ where: { slug } });

  return (
    <article>
      <h1>{post.title}</h1>
      <div>{post.content}</div>
    </article>
  );
}

// 静态内容? → Server Component
export function Footer() {
  return (
    <footer>
      <p>&copy; 2024 My Company</p>
    </footer>
  );
}
```

**选择指南**:

| 需求     | 组件类型 | 原因             |
| :------- | :------- | :--------------- |
| 用户交互 | Client   | 需要事件处理     |
| 状态管理 | Client   | 需要 useState    |
| 副作用   | Client   | 需要 useEffect   |
| 数据获取 | Server   | 直接访问数据库   |
| SEO 优化 | Server   | 服务器端渲染     |
| 静态内容 | Server   | 减少 bundle 大小 |

## 7. 注意事项

### 7.1 组件边界规划

**规划原则**:

```typescript
// ❌ 错误 - 过度使用客户端组件
"use client";

export default function Page() {
  // 整个页面都是客户端组件,即使大部分是静态内容
  const data = await fetch("/api/data").then((r) => r.json());

  return (
    <div>
      <h1>{data.title}</h1>
      <p>{data.description}</p>
      <button onClick={() => alert("Click")}>Click</button>
    </div>
  );
}

// ✅ 正确 - 只在需要的地方使用客户端组件
// app/page.tsx (Server Component)
export default async function Page() {
  const data = await fetchData();

  return (
    <div>
      <h1>{data.title}</h1>
      <p>{data.description}</p>
      <ClickButton /> {/* 只有按钮是客户端组件 */}
    </div>
  );
}

// app/ClickButton.tsx (Client Component)
("use client");

export function ClickButton() {
  return <button onClick={() => alert("Click")}>Click</button>;
}
```

**边界设计建议**:

- 尽可能推迟客户端边界
- 将交互逻辑隔离到小组件
- 避免整个页面使用 'use client'
- 考虑组件的可复用性

### 7.2 数据传递限制

**序列化限制**:

```typescript
// ❌ 不能传递的数据类型
"use client";

// 函数
export function Component({ onClick }: { onClick: () => void }) {
  // 错误: 不能从服务器组件传递函数到客户端组件
}

// 类实例
export function Component({ user }: { user: UserClass }) {
  // 错误: 不能传递类实例
}

// Symbol
export function Component({ key }: { key: symbol }) {
  // 错误: 不能传递 Symbol
}

// ✅ 可以传递的数据类型
("use client");

// 基本类型
export function Component({
  name,
  age,
  isActive,
}: {
  name: string;
  age: number;
  isActive: boolean;
}) {
  // 正确
}

// 普通对象
export function Component({
  user,
}: {
  user: { id: string; name: string; email: string };
}) {
  // 正确
}

// 数组
export function Component({
  items,
}: {
  items: Array<{ id: string; name: string }>;
}) {
  // 正确
}

// Date (会转为字符串)
export function Component({ createdAt }: { createdAt: Date }) {
  // 正确,但 Date 会被序列化为字符串
  // 需要在客户端重新转换: new Date(createdAt)
}
```

**解决方案**:

```typescript
// 使用 Server Actions 代替传递函数

import { DeleteButton } from "./DeleteButton";

        </div>
      ))}
    </div>
  );
}

// app/DeleteButton.tsx (Client Component)
("use client");

  postId,
  deleteAction,
}: {
  postId: string;
  deleteAction: (id: string) => Promise<void>;
}) {

}

// app/actions/post.ts
("use server");

  await db.post.delete({ where: { id } });

}
```

### 8.2 'use client' 必须放在文件顶部吗?

**问题**: 'use client' 的位置要求

**解答**:

```typescript
// ✅ 正确 - 在文件顶部
"use client";

import { useState } from "react";

export function Component() {
  const [count, setCount] = useState(0);
  return <div>{count}</div>;
}

// ❌ 错误 - 在 import 之后
import { useState } from "react";

("use client"); // 错误位置

export function Component() {
  const [count, setCount] = useState(0);
  return <div>{count}</div>;
}

// ❌ 错误 - 在注释之后
// This is a comment

("use client"); // 错误位置

export function Component() {
  return <div>Hello</div>;
}

// ✅ 正确 - 注释可以在 'use client' 之后
("use client");

// This is a comment

import { useState } from "react";

export function Component() {
  return <div>Hello</div>;
```

**规则**:

- 必须在文件的第一行(或第一个非空白行)
- 必须在所有 import 之前
- 必须在所有代码之前
- 注释可以在 'use client' 之后

### 8.3 服务器组件可以导入客户端组件吗?

**问题**: 组件导入的方向性

**解答**:

```typescript
// ✅ 正确 - 服务器组件导入客户端组件
// app/page.tsx (Server Component)
import { ClientButton } from "./ClientButton"; // 客户端组件

// ❌ 错误 - 客户端组件导入服务器组件(异步)
// app/ClientWrapper.tsx (Client Component)
("use client");

import { ServerContent } from "./ServerContent"; // 服务器组件

export function ClientWrapper() {
  return (
    <div>
      <ServerContent /> {/* 错误: 不能直接导入异步服务器组件 */}
    </div>
  );
}

// ✅ 正确 - 通过 children 传递
// app/page.tsx (Server Component)
import { ClientWrapper } from "./ClientWrapper";
import { ServerContent } from "./ServerContent";

export default function Page() {
  return (
    <ClientWrapper>
      <ServerContent /> {/* 作为 children 传递 */}
    </ClientWrapper>

}
```

- 客户端组件不能导入异步服务器组件 ❌
- 客户端组件可以通过 props/children 接收服务器组件 ✅

### 8.4 如何在服务器组件中使用 Context?

**问题**: 服务器组件不支持 Context

**解决方案**:

### 8.5 Server Actions 可以在客户端组件中定义吗?

**问题**: Server Actions 的定义位置

**解答**:

```typescript
// ❌ 错误 - 在客户端组件文件中定义
// app/components/Form.tsx
"use client";

// 这个 Server Action 不会工作
async function submitForm(formData: FormData) {
  "use server";
  await db.post.create({ data: { title: formData.get("title") } });
}

export function Form() {
  return <form action={submitForm}>...</form>;
}

// ✅ 正确 - 在单独的文件中定义
// app/actions/post.ts
("use server");

export async function submitForm(formData: FormData) {
  await db.post.create({
    data: { title: formData.get("title") as string },
  });
}

// app/components/Form.tsx
("use client");

import { submitForm } from "@/app/actions/post";

export function Form() {
  return <form action={submitForm}>...</form>;
}

// ✅ 正确 - 内联 Server Action (仅在服务器组件中)
// app/page.tsx (Server Component)
export default function Page() {
  async function submitForm(formData: FormData) {
    "use server";
    await db.post.create({
      data: { title: formData.get("title") as string },
    });
  }

  return <form action={submitForm}>...</form>;
}
```

**规则**:

- Server Actions 应该在单独的文件中定义(推荐)
- 或在服务器组件中内联定义
- 不能在客户端组件文件中定义

## 9. 总结

### 9.1 核心要点

网络边界是 Next.js 16 的核心概念:

1. **'use client'**: 标记客户端组件边界
2. **'use server'**: 定义服务器端函数
3. **Server Components**: 默认的服务器端组件
4. **混合渲染**: 灵活组合服务器和客户端组件
5. **性能优化**: 合理划分边界减少 bundle 大小

### 9.2 最佳实践

1. **组件选择**:

   - 默认使用 Server Components
   - 只在需要交互时使用 Client Components
   - 推迟客户端边界
   - 隔离交互逻辑

2. **数据传递**:

   - 通过 props 传递数据
   - 使用 Server Actions 处理操作
   - 避免传递不可序列化的数据
   - 使用 children 模式组合组件

3. **性能优化**:

   - 减少客户端 bundle 大小
   - 使用动态导入
   - 预加载关键数据
   - 合理使用缓存

4. **代码组织**:
   - Server Actions 放在单独文件
   - 客户端组件按功能分组
   - 使用清晰的命名约定
   - 文档化组件类型
