# 09-04-数据获取 API 速查手册

## 1. 概述 (Overview)

### 1.1 什么是数据获取 API

数据获取 API 是 Next.js 提供的一系列用于在服务端和客户端获取数据的接口和函数。Next.js 16 对数据获取进行了重大改进,特别是在缓存策略和性能优化方面。

### 1.2 Next.js 16 数据获取变更

**⚠️ 重要变更**:

- fetch 默认缓存从 `force-cache` 改为 `no-store`
- 引入新的 `'use cache'` 指令
- `cacheLife` 和 `cacheTag` 函数用于精细化缓存控制
- `after` 函数用于响应后任务

### 1.3 数据获取方式对比

| 方式              | 执行位置 | 缓存       | 使用场景 |
| :---------------- | :------- | :--------- | :------- |
| Server Components | 服务端   | 可配置     | 默认方式 |
| Route Handlers    | 服务端   | 可配置     | API 路由 |
| Server Actions    | 服务端   | 不缓存     | 表单提交 |
| Client Components | 客户端   | 浏览器缓存 | 交互数据 |

## 2. 服务端数据获取 API (Server-Side Data Fetching)

### 2.1 fetch API

**功能**: 在服务端组件中获取数据

**⚠️ Next.js 16 变更**:

```typescript
// ❌ Next.js 15 - 默认缓存
const res = await fetch("https://api.example.com/data");
// 默认: cache: 'force-cache'

// ✅ Next.js 16 - 默认不缓存
const res = await fetch("https://api.example.com/data");
// 默认: cache: 'no-store'
```

**类型签名**:

```typescript
fetch(url: string, options?: {
  cache?: 'force-cache' | 'no-store'
  next?: {
    revalidate?: number | false
    tags?: string[]
  }
}): Promise<Response>
```

**使用示例**:

```typescript
// 基本用法 - 不缓存
export default async function Page() {
  const res = await fetch("https://api.example.com/posts");
  const posts = await res.json();

  return <PostList posts={posts} />;
}

// 启用缓存
export default async function Page() {
  const res = await fetch("https://api.example.com/posts", {
    cache: "force-cache",
  });
  const posts = await res.json();

  return <PostList posts={posts} />;
}

// 定时重新验证
export default async function Page() {
  const res = await fetch("https://api.example.com/posts", {
    next: { revalidate: 3600 }, // 1小时
  });
  const posts = await res.json();

  return <PostList posts={posts} />;
}

// 使用缓存标签
export default async function Page() {
  const res = await fetch("https://api.example.com/posts", {
    next: { tags: ["posts"] },
  });
  const posts = await res.json();

  return <PostList posts={posts} />;
}

// 禁用缓存
export default async function Page() {
  const res = await fetch("https://api.example.com/posts", {
    cache: "no-store",
  });
  const posts = await res.json();

  return <PostList posts={posts} />;
}
```

**缓存选项详解**:

| 选项                 | 行为            | 使用场景   |
| :------------------- | :-------------- | :--------- |
| cache: 'force-cache' | 永久缓存        | 静态数据   |
| cache: 'no-store'    | 不缓存          | 实时数据   |
| next.revalidate: 60  | 60 秒后重新验证 | 准静态数据 |
| next.tags: ['tag']   | 标签化缓存      | 按需清除   |

**错误处理**:

```typescript
export default async function Page() {
  try {
    const res = await fetch("https://api.example.com/posts");

    if (!res.ok) {
      throw new Error("Failed to fetch data");
    }

    const posts = await res.json();
    return <PostList posts={posts} />;
  } catch (error) {
    return <div>Error loading posts</div>;
  }
}
```

**注意事项**:

- Next.js 16 默认不缓存,需要显式启用
- 只在服务端组件中可用
- 自动去重相同请求

**适用场景**:

- 从 REST API 获取数据
- 第三方 API 调用
- 需要缓存控制的场景

### 2.3 🆕 'use cache' 指令

**功能**: Next.js 16 新增的缓存指令

**类型签名**:

```typescript
"use cache";

export async function getData() {
  // 函数体
}
```

**使用示例**:

```typescript
// 缓存整个函数
"use cache";

export async function getPosts() {
  const res = await fetch("https://api.example.com/posts");
  return res.json();
}

// 在组件中使用
export default async function Page() {
  const posts = await getPosts();

  return <PostList posts={posts} />;
}

// 配合 cacheLife 使用
("use cache");

import { cacheLife } from "next/cache";

export async function getUser(id: string) {
  cacheLife("hours");

  const res = await fetch(`https://api.example.com/users/${id}`);
  return res.json();
}

// 配合 cacheTag 使用
("use cache");

import { cacheTag } from "next/cache";

export async function getProducts() {
  cacheTag("products");

  const res = await fetch("https://api.example.com/products");
  return res.json();
}
```

**注意事项**:

- 只能在文件顶部使用
- 影响整个函数或模块
- 与 fetch 缓存独立

**适用场景**:

- 函数级缓存
- 复杂计算结果缓存
- 多个数据源聚合

### 2.4 🆕 cacheLife 函数

**功能**: 设置缓存生命周期

**类型签名**:

```typescript
cacheLife(profile: 'seconds' | 'minutes' | 'hours' | 'days' | 'weeks' | 'max'): void
```

**预设配置**:

| Profile | stale | revalidate | expire |
| :------ | :---- | :--------- | :----- |
| seconds | 0s    | 1s         | 1m     |
| minutes | 5m    | 1m         | 1h     |
| hours   | 5m    | 1h         | 1d     |
| days    | 5m    | 1d         | 1w     |
| weeks   | 5m    | 1w         | 1M     |
| max     | 5m    | 1M         | ∞      |

**使用示例**:

```typescript
"use cache";

import { cacheLife } from "next/cache";

export async function getStaticData() {
  cacheLife("max");

  const res = await fetch("https://api.example.com/static");
  return res.json();
}

export async function getRealtimeData() {
  cacheLife("seconds");

  const res = await fetch("https://api.example.com/realtime");
  return res.json();
}

export async function getBlogPosts() {
  cacheLife("hours");

  const res = await fetch("https://api.example.com/posts");
  return res.json();
}

// 自定义配置
import { cacheLife } from "next/cache";

export async function getCustomData() {
  cacheLife({
    stale: 60, // 60秒内返回缓存
    revalidate: 300, // 5分钟后重新验证
    expire: 3600, // 1小时后过期
  });

  const res = await fetch("https://api.example.com/data");
  return res.json();
}
```

**注意事项**:

- 必须在 'use cache' 函数中使用
- 预设配置覆盖大多数场景
- 自定义配置提供更精细控制

**适用场景**:

- 不同数据的缓存策略
- 性能优化
- 缓存时间控制

**相关章节**: 03-04-cacheLife 函数详解.md

### 2.5 🆕 cacheTag 函数

**功能**: 为缓存添加标签,支持按需清除

**类型签名**:

```typescript
cacheTag(...tags: string[]): void
```

**使用示例**:

```typescript
"use cache";

import { cacheTag } from "next/cache";

export async function getPosts() {
  cacheTag("posts", "blog");

  const res = await fetch("https://api.example.com/posts");
  return res.json();
}

export async function getPost(id: string) {
  cacheTag("posts", `post-${id}`);

  const res = await fetch(`https://api.example.com/posts/${id}`);
  return res.json();
}

// 多个标签
export async function getUserPosts(userId: string) {
  cacheTag("posts", `user-${userId}`, "user-content");

  const res = await fetch(`https://api.example.com/users/${userId}/posts`);
  return res.json();
}
```

**清除缓存**:

```typescript
import { revalidateTag } from "next/cache";

// 在 Server Action 中清除
export async function createPost(data: FormData) {
  await db.post.create({ data });

  // 清除所有带 'posts' 标签的缓存
  revalidateTag("posts");
}

// 清除特定文章缓存
export async function updatePost(id: string, data: FormData) {
  await db.post.update({ where: { id }, data });

  revalidateTag(`post-${id}`);
  revalidateTag("posts");
}
```

**注意事项**:

- 必须在 'use cache' 函数中使用
- 标签名称区分大小写
- 可以添加多个标签

**适用场景**:

- 按需清除缓存
- 相关数据联动更新
- 精细化缓存控制

### 2.6 🆕 after 函数

**功能**: 在响应发送后执行任务

**类型签名**:

```typescript
after(callback: () => void | Promise<void>): void
```

**使用示例**:

```typescript
import { after } from "next/server";

export default async function Page() {
  const data = await getData();

  // 响应后记录日志
  after(() => {
    console.log("Page rendered");
  });

  return <div>{data}</div>;
}

// 异步任务
export default async function Page() {
  const user = await getUser();

  after(async () => {
    await logUserActivity(user.id);
    await updateAnalytics(user.id);
  });

  return <UserProfile user={user} />;
}

// 在 Server Action 中使用
export async function createOrder(data: FormData) {
  const order = await db.order.create({ data });

  after(async () => {
    await sendConfirmationEmail(order.email);
    await updateInventory(order.items);
    await notifyWarehouse(order.id);
  });

  return { success: true, orderId: order.id };
}

// 错误处理
export default async function Page() {
  const data = await getData();

  after(async () => {
    try {
      await trackPageView();
    } catch (error) {
      console.error("Failed to track page view:", error);
    }
  });

  return <div>{data}</div>;
}
```

**注意事项**:

- 不会阻塞响应
- 错误不会影响主流程
- 有执行时间限制

**适用场景**:

- 日志记录
- 分析统计
- 发送通知
- 后台任务

## 3. 客户端数据获取 API (Client-Side Data Fetching)

### 3.1 useEffect + fetch

**功能**: 在客户端组件中获取数据

**使用示例**:

```typescript
"use client";

import { useState, useEffect } from "react";

export default function ClientComponent() {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetch("/api/data")
      .then((res) => res.json())
      .then((data) => {
        setData(data);
        setLoading(false);
      })
      .catch((error) => {
        setError(error);
        setLoading(false);
      });
  }, []);

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error.message}</div>;

  return <div>{JSON.stringify(data)}</div>;
}

// 带依赖的数据获取
export default function UserPosts({ userId }: { userId: string }) {
  const [posts, setPosts] = useState([]);

  useEffect(() => {
    fetch(`/api/users/${userId}/posts`)
      .then((res) => res.json())
      .then(setPosts);
  }, [userId]);

  return <PostList posts={posts} />;
}
```

## 5. Server Actions 数据获取 (Server Actions Data Fetching)

### 5.1 基本 Server Action

**功能**: 在服务端执行函数

**使用示例**:

```typescript
// app/actions.ts
"use server";

export async function createPost(formData: FormData) {
  const title = formData.get("title") as string;
  const content = formData.get("content") as string;

  const post = await db.post.create({
    data: { title, content },
  });

  revalidatePath("/posts");

  return { success: true, post };
}

// app/create-post/page.tsx
import { createPost } from "../actions";

export default function CreatePost() {
  return (
    <form action={createPost}>
      <input name="title" required />
      <textarea name="content" required />
      <button type="submit">Create</button>
    </form>
  );
}

// 在客户端组件中使用
("use client");

import { createPost } from "../actions";
import { useFormState } from "react-dom";

export default function CreatePostForm() {
  const [state, formAction] = useFormState(createPost, null);

  return (
    <form action={formAction}>
      <input name="title" required />
      <textarea name="content" required />
      <button type="submit">Create</button>
      {state?.success && <p>Post created!</p>}
    </form>
  );
}
```

**注意事项**:

- 必须标记 'use server'
- 自动处理序列化
- 支持渐进式增强

**适用场景**:

- 表单提交
- 数据变更
- 服务端逻辑

### 5.2 数据重新验证

**功能**: 更新缓存数据

**使用示例**:

```typescript
"use server";

import { revalidatePath, revalidateTag } from "next/cache";

export async function updatePost(id: string, formData: FormData) {
  await db.post.update({
    where: { id },
    data: {
      title: formData.get("title"),
      content: formData.get("content"),
    },
  });

  // 重新验证路径
  revalidatePath("/posts");
  revalidatePath(`/posts/${id}`);

  // 重新验证标签
  revalidateTag("posts");
  revalidateTag(`post-${id}`);
}

// 重新验证整个布局
export async function updateSettings(formData: FormData) {
  await db.settings.update({ data: formData });

  revalidatePath("/", "layout");
}

// 重新验证所有页面
export async function clearAllCache() {
  revalidatePath("/", "page");
}
```

**注意事项**:

- revalidatePath 重新验证路径
- revalidateTag 重新验证标签
- 可以指定类型(page/layout)

**适用场景**:

- 数据更新后刷新
- 缓存失效
- 内容发布

## 6. 数据获取模式 (Data Fetching Patterns)

### 6.1 并行数据获取

**功能**: 同时获取多个数据源

**使用示例**:

```typescript
export default async function Page() {
  // 并行获取
  const [posts, users, comments] = await Promise.all([
    fetch("https://api.example.com/posts").then((res) => res.json()),
    fetch("https://api.example.com/users").then((res) => res.json()),
    fetch("https://api.example.com/comments").then((res) => res.json()),
  ]);

  return (
    <div>
      <PostList posts={posts} />
      <UserList users={users} />
      <CommentList comments={comments} />
    </div>
  );
}

// 使用 Promise.allSettled 处理错误
export default async function Page() {
  const results = await Promise.allSettled([
    fetch("https://api.example.com/posts").then((res) => res.json()),
    fetch("https://api.example.com/users").then((res) => res.json()),
  ]);

  const posts = results[0].status === "fulfilled" ? results[0].value : [];
  const users = results[1].status === "fulfilled" ? results[1].value : [];

  return <Dashboard posts={posts} users={users} />;
}
```

**注意事项**:

- 减少总加载时间
- 注意错误处理
- 避免过多并行请求

**适用场景**:

- 独立数据源
- 仪表板页面
- 数据聚合

---

### 6.2 串行数据获取

**功能**: 依次获取数据

**使用示例**:

```typescript
export default async function Page({ params }: { params: { id: string } }) {
  // 先获取用户
  const user = await fetch(`https://api.example.com/users/${params.id}`).then(
    (res) => res.json()
  );

  // 再获取用户的文章
  const posts = await fetch(
    `https://api.example.com/users/${user.id}/posts`
  ).then((res) => res.json());

  // 最后获取文章的评论
  const comments = await fetch(
    `https://api.example.com/posts/${posts[0].id}/comments`
  ).then((res) => res.json());

  return <UserProfile user={user} posts={posts} comments={comments} />;
}
```

**注意事项**:

- 数据有依赖关系时使用
- 会增加总加载时间
- 考虑是否可以并行

**适用场景**:

- 数据依赖
- 条件获取
- 分步加载

### 6.3 流式数据获取

**功能**: 逐步渲染数据

**使用示例**:

```typescript
import { Suspense } from "react";

export default function Page() {
  return (
    <div>
      <h1>Dashboard</h1>

      <Suspense fallback={<PostsSkeleton />}>
        <Posts />
      </Suspense>

      <Suspense fallback={<UsersSkeleton />}>
        <Users />
      </Suspense>
    </div>
  );
}

async function Posts() {
  const posts = await fetch("https://api.example.com/posts").then((res) =>
    res.json()
  );

  return <PostList posts={posts} />;
}

async function Users() {
  const users = await fetch("https://api.example.com/users").then((res) =>
    res.json()
  );

  return <UserList users={users} />;
}
```

**注意事项**:

- 提升感知性能
- 需要 Suspense 边界
- 适合独立组件

**适用场景**:

- 大型页面
- 不同加载速度的数据
- 改善用户体验

## 8. 注意事项 (Precautions)

### 8.1 Next.js 16 缓存变更

**默认行为变化**:

```typescript
// ❌ Next.js 15 - 默认缓存
const res = await fetch("https://api.example.com/data");
// 等同于: cache: 'force-cache'

// ✅ Next.js 16 - 默认不缓存
const res = await fetch("https://api.example.com/data");
// 等同于: cache: 'no-store'

// 需要显式启用缓存
const res = await fetch("https://api.example.com/data", {
  cache: "force-cache",
});
```

**迁移建议**:

- 检查所有 fetch 调用
- 根据需求添加缓存配置
- 使用新的 'use cache' 指令

### 9.6 如何实现无限滚动?

使用客户端数据获取:

```typescript
"use client";

import { useState, useEffect } from "react";
import { useInView } from "react-intersection-observer";

export default function InfiniteScroll() {
  const [posts, setPosts] = useState([]);
  const [page, setPage] = useState(1);
  const { ref, inView } = useInView();

  useEffect(() => {
    if (inView) {
      fetch(`/api/posts?page=${page}`)
        .then((res) => res.json())
        .then((newPosts) => {
          setPosts((prev) => [...prev, ...newPosts]);
          setPage((prev) => prev + 1);
        });
    }
  }, [inView]);

  return (
    <div>
      {posts.map((post) => (
        <Post key={post.id} post={post} />
      ))}
      <div ref={ref}>Loading...</div>
    </div>
  );
}
```

// app/api/export/route.ts
export async function GET() {
  const encoder = new TextEncoder();

  const stream = new ReadableStream({
    async start(controller) {
      const data = await db.post.findMany();

      for (const item of data) {
        controller.enqueue(encoder.encode(JSON.stringify(item) + "\n"));

      controller.close();
