# 09-15-代码质量保障策略

## 1. 概述

### 1.1 什么是代码质量

代码质量是指代码的可读性、可维护性、可测试性、性能和安全性。高质量的代码能减少 bug,提高开发效率,降低维护成本。

### 1.2 代码质量的重要性

**价值**:

- 减少 bug 和故障
- 提高开发效率
- 降低维护成本
- 改善团队协作
- 提升用户体验

### 1.3 质量保障体系

| 层次     | 工具/方法        | 目的         |
| :------- | :--------------- | :----------- |
| 编码规范 | ESLint, Prettier | 统一代码风格 |
| 类型检查 | TypeScript       | 类型安全     |
| 单元测试 | Jest, Vitest     | 功能正确性   |
| 集成测试 | Testing Library  | 组件交互     |
| E2E 测试 | Playwright       | 端到端流程   |
| 代码审查 | GitHub PR        | 人工检查     |
| 持续集成 | GitHub Actions   | 自动化检查   |

## 14. 常见问题

### 14.1 如何提高测试覆盖率?

**问题**: 测试覆盖率低

**解答**:

1. 识别未覆盖的代码
2. 优先测试关键路径
3. 添加边界条件测试
4. 测试错误处理

**示例**:

```typescript
// ✅ 完整的测试覆盖
describe("divide", () => {
  it("divides two numbers", () => {
    expect(divide(10, 2)).toBe(5);
  });

  it("throws error when dividing by zero", () => {
    expect(() => divide(10, 0)).toThrow("Cannot divide by zero");
  });

  it("handles negative numbers", () => {
    expect(divide(-10, 2)).toBe(-5);
  });

  it("handles decimal numbers", () => {
    expect(divide(10, 3)).toBeCloseTo(3.33, 2);
  });
});
```

### 14.2 如何处理 E2E 测试的不稳定性?

**问题**: E2E 测试经常失败

**解答**:

1. 使用明确的等待
2. 避免硬编码延迟
3. 使用测试数据隔离
4. 重试失败的测试

**示例**:

```typescript
// ✅ 使用明确的等待
test("should load data", async ({ page }) => {
  await page.goto("/dashboard");

  // 等待特定元素出现
  await page.waitForSelector('[data-testid="user-list"]');

  // 等待网络请求完成
  await page.waitForResponse(
    (response) =>
      response.url().includes("/api/users") && response.status() === 200
  );

  expect(
    await page.locator('[data-testid="user-item"]').count()
  ).toBeGreaterThan(0);
});

// ❌ 使用硬编码延迟
test("should load data", async ({ page }) => {
  await page.goto("/dashboard");
  await page.waitForTimeout(3000); // 不推荐
  expect(
    await page.locator('[data-testid="user-item"]').count()
  ).toBeGreaterThan(0);
});
```

### 4.2 组件测试

**测试 React 组件**:

```typescript
// components/Button.test.tsx
import { render, screen, fireEvent } from "@testing-library/react";
import { Button } from "./Button";

describe("Button", () => {
  it("renders button with text", () => {
    render(<Button>Click me</Button>);
    expect(screen.getByText("Click me")).toBeInTheDocument();
  });

  it("calls onClick when clicked", () => {
    const handleClick = jest.fn();
    render(<Button onClick={handleClick}>Click me</Button>);

    fireEvent.click(screen.getByText("Click me"));
    expect(handleClick).toHaveBeenCalledTimes(1);
  });

  it("is disabled when disabled prop is true", () => {
    render(<Button disabled>Click me</Button>);
    expect(screen.getByText("Click me")).toBeDisabled();
  });
});
```
