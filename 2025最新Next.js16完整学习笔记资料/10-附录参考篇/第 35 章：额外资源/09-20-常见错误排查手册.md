## 2. 开发环境错误

### 2.1 端口被占用

**错误信息**:

```
Error: listen EADDRINUSE: address already in use :::3000
```

**排查步骤**:

1. 查看占用端口的进程
2. 终止进程或更换端口

**解决方案**:

```bash
# Windows
netstat -ano | findstr :3000
taskkill /PID <PID> /F

# macOS/Linux
lsof -i :3000
kill -9 <PID>

# 或更换端口
PORT=3001 npm run dev
```

---

### 2.2 模块未找到

**错误信息**:

```
Module not found: Can't resolve '@/components/Button'
```

**排查步骤**:

1. 检查文件是否存在
2. 检查路径是否正确
3. 检查 tsconfig.json 配置
4. 重启开发服务器

**解决方案**:

```json
// tsconfig.json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./*"]
    }
  }
}
```

```bash
# 重启开发服务器
npm run dev
```

## 3. 编译错误

### 3.1 TypeScript 类型错误

**错误信息**:

```
Type 'string | undefined' is not assignable to type 'string'
```

**排查步骤**:

1. 查看错误位置
2. 理解类型要求
3. 添加类型守卫或类型断言

**解决方案**:

```typescript
// ❌ 错误
const name: string = formData.get("name");

// ✅ 方案 1: 类型断言
const name = formData.get("name") as string;

// ✅ 方案 2: 类型守卫
const nameValue = formData.get("name");
if (typeof nameValue !== "string") {
  throw new Error("Name is required");
}
const name: string = nameValue;

// ✅ 方案 3: 可选类型
const name: string | null = formData.get("name") as string | null;
```

### 14.5 使用错误边界

**示例**:

```typescript
// components/ErrorBoundary.tsx
"use client";

import { Component, ReactNode } from "react";

interface Props {
  children: ReactNode;
  fallback?: ReactNode;
}

interface State {
  hasError: boolean;
  error?: Error;
}

export class ErrorBoundary extends Component<Props, State> {
  constructor(props: Props) {
    super(props);
    this.state = { hasError: false };
  }

  static getDerivedStateFromError(error: Error) {
    return { hasError: true, error };
  }

  componentDidCatch(error: Error, errorInfo: any) {
    console.error("Error caught by boundary:", error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      return this.props.fallback || <div>Something went wrong</div>;
    }

    return this.props.children;
  }
}
```

## 4. 运行时错误

### 4.1 Hydration 错误

**错误信息**:

```
Error: Hydration failed because the initial UI does not match what was rendered on the server
```

**排查步骤**:

1. 检查服务端和客户端渲染是否一致
2. 检查是否使用了浏览器 API
3. 检查是否使用了随机数或时间戳

**解决方案**:

```typescript
// ❌ 错误: 服务端和客户端不一致
export default function Component() {
  return <div>{Math.random()}</div>;
}

// ✅ 正确: 使用 useEffect
("use client");

import { useState, useEffect } from "react";

export default function Component() {
  const [random, setRandom] = useState(0);

  useEffect(() => {
    setRandom(Math.random());
  }, []);

  return <div>{random || "Loading..."}</div>;
}
```

### 4.2 Cannot read properties of undefined

**错误信息**:

```
TypeError: Cannot read properties of undefined (reading 'map')
```

**排查步骤**:

1. 检查数据是否已加载
2. 检查数据结构
3. 添加空值检查

**解决方案**:

```typescript
// ❌ 错误: 未检查数据
export default function Posts({ posts }) {
  return posts.map((post) => <div key={post.id}>{post.title}</div>);
}

// ✅ 正确: 添加检查
export default function Posts({ posts }) {
  if (!posts || posts.length === 0) {
    return <div>No posts</div>;
  }

  return posts.map((post) => <div key={post.id}>{post.title}</div>);
}

// ✅ 更好: 使用可选链
export default function Posts({ posts }) {
  return (
    <div>
      {posts?.map((post) => <div key={post.id}>{post.title}</div>) ||
        "No posts"}
    </div>
  );
}
```

### 4.3 Maximum update depth exceeded

**错误信息**:

```
Error: Maximum update depth exceeded
```

**排查步骤**:

1. 检查是否在渲染时调用 setState
2. 检查 useEffect 依赖
3. 检查事件处理函数

**解决方案**:

```typescript
// ❌ 错误: 在渲染时调用 setState
export default function Component() {
  const [count, setCount] = useState(0);

  setCount(count + 1); // 无限循环

  return <div>{count}</div>;
}

// ✅ 正确: 在 useEffect 中调用
export default function Component() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    setCount(count + 1);
  }, []); // 只执行一次

  return <div>{count}</div>;
}

// ✅ 更好: 在事件处理函数中调用
export default function Component() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>{count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}
```

## 5. 数据获取错误

### 5.1 fetch 失败

**错误信息**:

```
TypeError: Failed to fetch
```

**排查步骤**:

1. 检查 URL 是否正确
2. 检查网络连接
3. 检查 CORS 配置
4. 检查 API 是否可用

**解决方案**:

```typescript
// ❌ 错误: 未处理错误
async function getData() {
  const res = await fetch("https://api.example.com/data");
  return res.json();
}

// ✅ 正确: 处理错误
async function getData() {
  try {
    const res = await fetch("https://api.example.com/data");

    if (!res.ok) {
      throw new Error(`HTTP error! status: ${res.status}`);
    }

    return await res.json();
  } catch (error) {
    console.error("Fetch error:", error);
    throw error;
  }
}

// ✅ 更好: 添加重试
async function getData(retries = 3) {
  for (let i = 0; i < retries; i++) {
    try {
      const res = await fetch("https://api.example.com/data");

      if (!res.ok) {
        throw new Error(`HTTP error! status: ${res.status}`);
      }

      return await res.json();
    } catch (error) {
      if (i === retries - 1) {
        throw error;
      }

      await new Promise((resolve) => setTimeout(resolve, 1000 * (i + 1)));
    }
  }
}
```

## 8. 性能问题

### 8.1 首次加载慢

**问题**: 首次访问加载时间长

### 8.2 数据库查询慢

**问题**: 页面加载慢,数据库查询耗时

**排查步骤**:

1. 检查查询语句
2. 检查索引
3. 检查 N+1 查询
4. 使用查询分析工具

**解决方案**:

```typescript
// ❌ 错误: N+1 查询
const posts = await db.post.findMany();
for (const post of posts) {
  const author = await db.user.findUnique({ where: { id: post.authorId } });
}

// ✅ 正确: 使用 include
const posts = await db.post.findMany({
  include: {
    author: true,
  },
});
```
