# 09-22-版本差异速查表

## 1. 概述

### 1.1 关于本文档

本文档详细对比了 Next.js 15 和 Next.js 16 之间的差异,包括破坏性变更、新增特性、废弃特性等。

### 1.2 版本信息

| 版本 | 发布日期 | 主要变更 |
|:-----|:---------|:---------|
| Next.js 15 | 2024年10月 | Turbopack, React 19 |
| Next.js 16 | 2025年 | 异步 API, 新缓存系统 |

---

## 2. 破坏性变更

### 2.1 headers() 和 cookies() 变为异步

**Next.js 15**:

```typescript
import { headers, cookies } from 'next/headers'

export function Component() {
  const headersList = headers()
  const cookieStore = cookies()
  
  const userAgent = headersList.get('user-agent')
  const token = cookieStore.get('token')
}
```

**Next.js 16**:

```typescript
import { headers, cookies } from 'next/headers'

export async function Component() {
  const headersList = await headers()
  const cookieStore = await cookies()
  
  const userAgent = headersList.get('user-agent')
  const token = cookieStore.get('token')
}
```

**影响**: 所有使用 headers() 和 cookies() 的代码都需要添加 await

---

### 2.2 params 和 searchParams 变为 Promise

**Next.js 15**:

```typescript
export default function Page({ 
  params,
  searchParams 
}: { 
  params: { id: string }
  searchParams: { q: string }
}) {
  return <div>ID: {params.id}, Query: {searchParams.q}</div>
}
```

**Next.js 16**:

```typescript
export default async function Page({
  params,
  searchParams
}: {
  params: Promise<{ id: string }>
  searchParams: Promise<{ q: string }>
}) {
  const { id } = await params
  const { q } = await searchParams
  
  return <div>ID: {id}, Query: {q}</div>
}
```

**影响**: 所有页面组件和 generateMetadata 函数都需要更新

---

### 2.3 fetch 默认缓存策略改变

**Next.js 15**:

```typescript
// 默认: force-cache (缓存)
fetch('https://api.example.com/data')
```

**Next.js 16**:

```typescript
// 默认: no-store (不缓存)
fetch('https://api.example.com/data')

// 需要缓存时明确指定
fetch('https://api.example.com/data', { cache: 'force-cache' })
```

**影响**: 可能影响性能,需要明确指定缓存策略

---

## 3. 新增特性

### 3.1 'use cache' 指令

**Next.js 15**: 不支持

**Next.js 16**:

```typescript
'use cache'

export async function getProducts() {
  const products = await db.product.findMany()
  return products
}
```

**好处**: 更细粒度的缓存控制

---

### 3.2 cacheLife() 函数

**Next.js 15**: 不支持

**Next.js 16**:

```typescript
import { unstable_cacheLife as cacheLife } from 'next/cache'

export async function getUser() {
  'use cache'
  cacheLife('hours')
  
  const user = await db.user.findFirst()
  return user
}
```

**预设配置**:

| 配置 | 时间 |
|:-----|:-----|
| seconds | 1秒 |
| minutes | 1分钟 |
| hours | 1小时 |
| days | 1天 |
| weeks | 1周 |
| max | 最大值 |

---

### 3.3 cacheTag() 函数

**Next.js 15**: 不支持

**Next.js 16**:

```typescript
import { unstable_cacheTag as cacheTag } from 'next/cache'

export async function getPosts() {
  'use cache'
  cacheTag('posts')
  
  const posts = await db.post.findMany()
  return posts
}
```

**重新验证**:

```typescript
import { revalidateTag } from 'next/cache'

export async function createPost() {
  await db.post.create(...)
  revalidateTag('posts')
}
```

---

### 3.4 after() 函数

**Next.js 15**: 不支持

**Next.js 16**:

```typescript
import { after } from 'next/server'

export async function POST(request: Request) {
  const data = await request.json()
  
  // 立即返回响应
  const response = NextResponse.json({ success: true })
  
  // 响应后执行
  after(async () => {
    await sendEmail(data.email)
    await logActivity(data)
  })
  
  return response
}
```

**好处**: 不阻塞响应,提升性能

**示例**:

```typescript
// 替换
// 将 function 改为 async function
// 将 headers() 改为 await headers()
// 将 cookies() 改为 await cookies()
```

---

- 新的特性
- 更好的开发体验
- 长期支持
