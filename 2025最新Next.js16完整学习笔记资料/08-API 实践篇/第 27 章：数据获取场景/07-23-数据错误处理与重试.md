# 数据错误处理与重试

在实际应用中,网络请求可能因为各种原因失败。合理的错误处理和重试机制可以提升应用的稳定性和用户体验。

## 核心概念

### 错误类型

| 错误类型   | 原因       | 处理方式 |
| ---------- | ---------- | -------- |
| 网络错误   | 断网、超时 | 自动重试 |
| 服务器错误 | 500、502   | 延迟重试 |
| 客户端错误 | 400、401   | 提示用户 |
| 业务错误   | 自定义错误 | 特殊处理 |

## 实战场景二: 自动重试

### 简单重试机制

```typescript
"use client";

import { useState } from "react";

async function fetchWithRetry(
  url: string,
  options: RequestInit = {},
  maxRetries = 3
): Promise<Response> {
  let lastError: Error;

  for (let i = 0; i < maxRetries; i++) {
    try {
      const response = await fetch(url, options);

      if (response.ok) {
        return response;
      }

      // 5xx 错误才重试
      if (response.status >= 500) {
        lastError = new Error(`HTTP ${response.status}`);
        continue;
      }

      // 4xx 错误不重试
      throw new Error(`HTTP ${response.status}`);
    } catch (error) {
      lastError = error as Error;

      // 最后一次重试失败
      if (i === maxRetries - 1) {
        throw lastError;
      }

      // 等待后重试
      await new Promise((resolve) => setTimeout(resolve, 1000 * (i + 1)));
    }
  }

  throw lastError!;
}

export default function RetryExample() {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(false);

  async function loadData() {
    setLoading(true);

    try {
      const response = await fetchWithRetry("/api/data");
      const result = await response.json();
      setData(result);
    } catch (error) {
      console.error("重试失败:", error);
    } finally {
      setLoading(false);
    }
  }

  return (
    <div>
      <button onClick={loadData} disabled={loading}>
        {loading ? "加载中..." : "加载数据"}
      </button>
      {data && <pre>{JSON.stringify(data, null, 2)}</pre>}
    </div>
  );
}
```

### 指数退避重试

```typescript
"use client";

async function fetchWithExponentialBackoff(
  url: string,
  maxRetries = 5,
  baseDelay = 1000
): Promise<Response> {
  for (let i = 0; i < maxRetries; i++) {
    try {
      const response = await fetch(url);

      if (response.ok) {
        return response;
      }

      if (response.status >= 500 && i < maxRetries - 1) {
        // 指数退避: 1s, 2s, 4s, 8s, 16s
        const delay = baseDelay * Math.pow(2, i);
        // 添加随机抖动,避免惊群效应
        const jitter = Math.random() * 1000;
        await new Promise((resolve) => setTimeout(resolve, delay + jitter));
        continue;
      }

      throw new Error(`HTTP ${response.status}`);
    } catch (error) {
      if (i === maxRetries - 1) throw error;
    }
  }

  throw new Error("Max retries exceeded");
}

export default function ExponentialBackoffExample() {
  async function loadData() {
    try {
      const response = await fetchWithExponentialBackoff("/api/data");
      const data = await response.json();
      console.log(data);
    } catch (error) {
      console.error("加载失败:", error);
    }
  }

  return <button onClick={loadData}>加载数据</button>;
}
```

## 实战场景三: 错误边界

### React 错误边界

```typescript
"use client";

import { Component, ReactNode } from "react";

interface Props {
  children: ReactNode;
  fallback?: ReactNode;
}

interface State {
  hasError: boolean;
  error: Error | null;
}

export class ErrorBoundary extends Component<Props, State> {
  constructor(props: Props) {
    super(props);
    this.state = { hasError: false, error: null };
  }

  static getDerivedStateFromError(error: Error): State {
    return { hasError: true, error };
  }

  componentDidCatch(error: Error, errorInfo: any) {
    console.error("错误边界捕获:", error, errorInfo);

    // 上报错误
    if (window.gtag) {
      window.gtag("event", "exception", {
        description: error.message,
        fatal: false,
      });
    }
  }

  render() {
    if (this.state.hasError) {
      return (
        this.props.fallback || (
          <div className="error-boundary">
            <h2>出错了</h2>
            <p>{this.state.error?.message}</p>
            <button
              onClick={() => this.setState({ hasError: false, error: null })}
            >
              重试
            </button>
          </div>
        )
      );
    }

    return this.props.children;
  }
}
```

async function fetchWithTimeout(
  url: string,
  timeout = 5000
): Promise<Response> {
  const controller = new AbortController();

"use client";

async function fetchWithTimeoutAndRetry(
  url: string,
  timeout = 5000,
  maxRetries = 3
): Promise<Response> {
  for (let i = 0; i < maxRetries; i++) {
    try {
      const controller = new AbortController();
      const timeoutId = setTimeout(() => controller.abort(), timeout);

      const response = await fetch(url, {
        signal: controller.signal,
      });

      clearTimeout(timeoutId);

      if (response.ok) {
        return response;
      }

      throw new Error(`HTTP ${response.status}`);
    } catch (error) {
      if (i === maxRetries - 1) throw error;

      // 超时或服务器错误才重试
      if (error.name === "AbortError" || error.message.includes("5")) {
        await new Promise((resolve) => setTimeout(resolve, 1000 * (i + 1)));
        continue;
      }

      throw error;
    }
  }

  throw new Error("Max retries exceeded");
}

class AuthError extends Error {
  constructor(message: string) {
    super(message);
    this.name = "AuthError";
  }
}

async function fetchData(url: string) {
  try {
    const response = await fetch(url);

    if (response.status === 401) {
      throw new AuthError("未授权");
    }

    if (response.status === 400) {
      const error = await response.json();
      throw new ValidationError(error.message, error.field);
    }

    if (!response.ok) {
      throw new NetworkError(`HTTP ${response.status}`);
    }

    return response.json();
  } catch (error) {
    if (error instanceof AuthError) {
      // 跳转登录
      window.location.href = "/login";
    } else if (error instanceof ValidationError) {
      // 显示字段错误
      console.error(`字段 ${error.field} 验证失败: ${error.message}`);
    } else if (error instanceof NetworkError) {
      // 网络错误,可以重试
      console.error("网络错误:", error.message);
    }

    throw error;
  }
}

## 实战场景十一: 完整错误处理方案

### 统一错误处理器

```typescript
"use client";

interface ErrorHandlerOptions {
  retry?: boolean;
  maxRetries?: number;
  onError?: (error: Error) => void;
  fallback?: any;
}

async function handleRequest<T>(
  fn: () => Promise<T>,
  options: ErrorHandlerOptions = {}
): Promise<T> {
  const { retry = true, maxRetries = 3, onError, fallback } = options;

  let lastError: Error;

  for (let i = 0; i < (retry ? maxRetries : 1); i++) {
    try {
      return await fn();
    } catch (error) {
      lastError = error as Error;

      // 调用错误回调
      onError?.(lastError);

      // 最后一次重试
      if (i === maxRetries - 1) {
        if (fallback !== undefined) {
          return fallback;
        }
        throw lastError;
      }

      // 等待后重试
      await new Promise((resolve) => setTimeout(resolve, 1000 * (i + 1)));
    }
  }

  throw lastError!;
}

export default function UnifiedErrorHandling() {
  async function loadData() {
    const data = await handleRequest(
      () => fetch("/api/data").then((r) => r.json()),
      {
        retry: true,
        maxRetries: 3,
        onError: (error) => {
          console.error("请求失败:", error);
        },
        fallback: { message: "使用默认数据" },
      }
    );

    console.log(data);
  }

  return <button onClick={loadData}>加载数据</button>;
}
```

### 错误恢复队列

```typescript
"use client";

import { useState, useEffect } from "react";

interface FailedRequest {
  id: string;
  fn: () => Promise<any>;
  retries: number;
  maxRetries: number;
}

export default function ErrorRecoveryQueue() {
  const [queue, setQueue] = useState<FailedRequest[]>([]);
  const [processing, setProcessing] = useState(false);

  async function addToQueue(fn: () => Promise<any>, maxRetries = 3) {
    const request: FailedRequest = {
      id: Math.random().toString(36),
      fn,
      retries: 0,
      maxRetries,
    };

    setQueue((prev) => [...prev, request]);
  }

  async function processQueue() {
    if (processing || queue.length === 0) return;

    setProcessing(true);

    const request = queue[0];

    try {
      await request.fn();
      // 成功,移除队列
      setQueue((prev) => prev.filter((r) => r.id !== request.id));
    } catch (error) {
      // 失败,增加重试次数
      if (request.retries < request.maxRetries) {
        setQueue((prev) =>
          prev.map((r) =>
            r.id === request.id ? { ...r, retries: r.retries + 1 } : r
          )
        );
      } else {
        // 达到最大重试次数,移除队列
        setQueue((prev) => prev.filter((r) => r.id !== request.id));
      }
    }

    setProcessing(false);
  }

  useEffect(() => {
    const interval = setInterval(processQueue, 5000);
    return () => clearInterval(interval);
  }, [queue, processing]);

  return (
    <div>
      <div>队列中的请求: {queue.length}</div>
      <button onClick={() => addToQueue(() => fetch("/api/data"))}>
        添加请求
      </button>
    </div>
  );
}
```

## 常见问题

### 1. 如何判断是否应该重试?

根据 HTTP 状态码判断:

- 2xx: 成功,不重试
- 4xx: 客户端错误,通常不重试(除了 429)
- 5xx: 服务器错误,应该重试
- 网络错误: 应该重试

### 2. 重试间隔如何设置?

推荐使用指数退避:

- 第 1 次: 1 秒
- 第 2 次: 2 秒
- 第 3 次: 4 秒
- 第 4 次: 8 秒
- 第 5 次: 16 秒

### 3. 如何避免重试风暴?

### 5. 如何实现断点续传?

```typescript
async function uploadWithResume(file: File) {
  const chunkSize = 1024 * 1024; // 1MB
  let uploaded = 0;

  while (uploaded < file.size) {
    const chunk = file.slice(uploaded, uploaded + chunkSize);

    try {
      await fetch("/api/upload", {
        method: "POST",
        headers: {
          "Content-Range": `bytes ${uploaded}-${uploaded + chunk.size}/${
            file.size
          }`,
        },
        body: chunk,
      });

      uploaded += chunk.size;
    } catch (error) {
      // 重试当前分片
      continue;
    }
  }
}
```

错误处理和重试机制是构建稳定应用的基础。通过本文的学习,我们掌握了:

### 核心技术

1. **错误处理**: try-catch、错误边界、SWR 错误处理

1. **区分错误类型**: 根据不同错误采取不同策略
