# 路由组组织大型应用

路由组(Route Groups)是 Next.js 16 中用于组织路由结构的强大功能。它允许你在不影响 URL 路径的情况下,对路由进行逻辑分组,使大型应用的文件结构更加清晰和易于维护。

## 路由组基础

### 什么是路由组

路由组使用括号 `(folder)` 命名文件夹,这个文件夹名称不会出现在 URL 中:

```
app/
├── (marketing)/
│   ├── about/
│   │   └── page.tsx        # /about
│   └── contact/
│       └── page.tsx        # /contact
├── (shop)/
│   ├── products/
│   │   └── page.tsx        # /products
│   └── cart/
│       └── page.tsx        # /cart
└── page.tsx                # /
```

### 基本用法

```typescript
// app/(marketing)/about/page.tsx
export default function AboutPage() {
  return <h1>About Us</h1>;
}
// URL: /about (不包含 marketing)

// app/(shop)/products/page.tsx
export default function ProductsPage() {
  return <h1>Products</h1>;
}
// URL: /products (不包含 shop)
```

## 路由组的作用

### 1. 逻辑分组

将相关路由组织在一起:

```
app/
├── (auth)/
│   ├── login/
│   ├── register/
│   └── forgot-password/
├── (dashboard)/
│   ├── overview/
│   ├── analytics/
│   └── settings/
└── (public)/
    ├── home/
    ├── about/
    └── contact/
```

### 2. 不同布局

为不同路由组应用不同布局:

```typescript
// app/(marketing)/layout.tsx
export default function MarketingLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <div>
      <MarketingHeader />
      {children}
      <MarketingFooter />
    </div>
  );
}

// app/(shop)/layout.tsx
export default function ShopLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <div>
      <ShopHeader />
      <ShopSidebar />
      {children}
    </div>
  );
}
```

### 3. 不同配置

为不同路由组应用不同配置:

```typescript
// app/(marketing)/layout.tsx
export const metadata = {
  title: {
    template: "%s | Marketing",
    default: "Marketing",
  },
};

// app/(shop)/layout.tsx
export const metadata = {
  title: {
    template: "%s | Shop",
    default: "Shop",
  },
};

// app/(dashboard)/layout.tsx
export const dynamic = "force-dynamic"; // 强制动态渲染
export const revalidate = 0; // 禁用缓存
```

export async function generateMetadata({
  params,
}: {
  params: { tenant: string };
}) {
  const tenant = await getTenant(params.tenant);

  if (!tenant) {
    return {
      title: "Tenant Not Found",
    };
  }

  return {
    title: {
      template: `%s | ${tenant.name}`,
      default: tenant.name,
    },
  };
}

export default async function TenantLayout({
  children,
  params,
}: {
  children: React.ReactNode;
  params: { tenant: string };
}) {
  const tenant = await getTenant(params.tenant);

  if (!tenant) {
    notFound();
  }

  return (
    <div
      className="min-h-screen"
      style={{
        "--primary-color": tenant.primaryColor,
      }}
    >
      <TenantNav tenant={tenant} />
      <main>{children}</main>
    </div>
  );
}

### 4. 路由组与中间件

```typescript
// middleware.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function middleware(request: NextRequest) {
  const path = request.nextUrl.pathname;

  // 管理后台需要认证
  if (path.startsWith("/admin") || path.startsWith("/dashboard")) {
    const token = request.cookies.get("auth-token");

    if (!token) {
      return NextResponse.redirect(new URL("/login", request.url));
    }
  }

  // 已登录用户访问认证页面重定向
  if (path.startsWith("/login") || path.startsWith("/register")) {
    const token = request.cookies.get("auth-token");

    if (token) {
      return NextResponse.redirect(new URL("/dashboard", request.url));
    }
  }

  return NextResponse.next();
}

export const config = {
  matcher: ["/admin/:path*", "/dashboard/:path*", "/login", "/register"],
};
```

## 适用场景

### 何时使用路由组

| 场景     | 说明                   | 示例                 |
| -------- | ---------------------- | -------------------- |
| 不同布局 | 不同页面需要不同的布局 | 营销页面 vs 应用页面 |
| 权限控制 | 不同路由组需要不同权限 | 公开页面 vs 管理后台 |
| 逻辑分组 | 相关路由组织在一起     | 认证相关页面         |
| 不同配置 | 不同路由组需要不同配置 | 静态页面 vs 动态页面 |
| 团队协作 | 不同团队负责不同模块   | 前端团队 vs 后端团队 |

### 路由组 vs 普通文件夹

| 特性     | 路由组 (folder)  | 普通文件夹 folder |
| -------- | ---------------- | ----------------- |
| URL 路径 | 不包含在 URL 中  | 包含在 URL 中     |
| 布局隔离 | 可以有独立布局   | 继承父级布局      |
| 元数据   | 可以有独立元数据 | 继承父级元数据    |
| 配置     | 可以有独立配置   | 继承父级配置      |
| 使用场景 | 逻辑分组         | URL 路径分段      |

## 注意事项

### 1. 路由组不影响 URL

```
app/
├── (marketing)/
│   └── about/
│       └── page.tsx
└── (shop)/
    └── about/
        └── page.tsx
```

这会导致冲突,因为两个页面的 URL 都是 `/about`。需要确保不同路由组中的路由不会产生相同的 URL。

### 2. 布局嵌套

```typescript
// app/layout.tsx (根布局)
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html>
      <body>{children}</body>
    </html>
  );
}

// app/(marketing)/layout.tsx (路由组布局)
export default function MarketingLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return <div className="marketing-layout">{children}</div>;
}

// 最终渲染:
// <html>
//   <body>
//     <div className="marketing-layout">
//       {page content}
//     </div>
//   </body>
// </html>
```

### 3. 元数据合并

```typescript
// app/layout.tsx
export const metadata = {
  title: "My App",
  description: "My app description",
};

// app/(marketing)/layout.tsx
export const metadata = {
  title: {
    template: "%s | Marketing",
    default: "Marketing",
  },
};

// app/(marketing)/about/page.tsx
export const metadata = {
  title: "About Us",
};

// 最终元数据:
// title: 'About Us | Marketing'
// description: 'My app description'
```

### 4. 路由组命名规范

```
推荐命名:
(marketing)  - 营销页面
(shop)       - 商店页面
(auth)       - 认证页面
(app)        - 应用页面
(admin)      - 管理后台
(public)     - 公开页面
(dashboard)  - 仪表板

避免命名:
(group1)     - 不清晰
(temp)       - 临时性
(test)       - 测试性
```

### 5. 性能考虑

```typescript
// 避免在路由组布局中进行重复的数据获取

// 不好的做法
// app/(marketing)/layout.tsx
export default async function MarketingLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  const settings = await getSettings(); // 每个页面都会执行

  return <div>{children}</div>;
}

// 好的做法
// app/(marketing)/layout.tsx
import { unstable_cache } from "next/cache";

const getCachedSettings = unstable_cache(
  async () => getSettings(),
  ["settings"],
  { revalidate: 3600 }
);

export default async function MarketingLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  const settings = await getCachedSettings();

  return <div>{children}</div>;
}
```

## 常见问题

### 1. 路由组可以嵌套吗?

可以。路由组可以任意嵌套:

```
app/
└── (marketing)/
    ├── (landing)/
    │   └── page.tsx
    └── (content)/
        └── blog/
            └── page.tsx
```

### 2. 路由组可以有多个根页面吗?

不可以。每个应用只能有一个根页面 (`app/page.tsx`),即使使用路由组也不能有多个:

```
app/
├── (marketing)/
│   └── page.tsx        # 错误: 不能有多个根页面
└── (shop)/
    └── page.tsx        # 错误: 不能有多个根页面
```

正确做法:

```
app/
├── page.tsx            # 唯一的根页面
├── (marketing)/
│   └── about/
│       └── page.tsx
└── (shop)/
    └── products/
        └── page.tsx
```

### 3. 如何在路由组之间共享组件?

将共享组件放在 `components` 目录:

```
project/
├── app/
│   ├── (marketing)/
│   │   └── page.tsx
│   └── (shop)/
│       └── page.tsx
└── components/
    └── shared/
        └── Button.tsx
```

```typescript
// app/(marketing)/page.tsx
import { Button } from "@/components/shared/Button";

// app/(shop)/page.tsx
import { Button } from "@/components/shared/Button";
```

### 4. 路由组会影响 API 路由吗?

不会。API 路由不受路由组影响:

```
app/
├── (marketing)/
│   └── api/
│       └── newsletter/
│           └── route.ts    # /api/newsletter
└── (shop)/
    └── api/
        └── products/
            └── route.ts    # /api/products
```

### 5. 如何在路由组中使用中间件?

中间件基于 URL 路径,不受路由组影响:

```typescript
// middleware.ts
export function middleware(request: NextRequest) {
  // 检查 URL 路径,不是文件夹名称
  if (request.nextUrl.pathname.startsWith("/dashboard")) {
    // 处理逻辑
  }
}
```

### 6. 路由组可以有自己的 loading.tsx 吗?

可以。每个路由组可以有自己的 loading 状态:

```
app/
├── (marketing)/
│   ├── loading.tsx
│   └── about/
│       └── page.tsx
└── (shop)/
    ├── loading.tsx
    └── products/
        └── page.tsx
```

### 7. 路由组可以有自己的 error.tsx 吗?

可以。每个路由组可以有自己的错误处理:

```
app/
├── (marketing)/
│   ├── error.tsx
│   └── about/
│       └── page.tsx
└── (shop)/
    ├── error.tsx
    └── products/
        └── page.tsx
```

### 8. 如何在路由组之间导航?

使用 `Link` 或 `router.push`,基于 URL 路径:

```typescript
import Link from 'next/link'

// 从 (marketing) 导航到 (shop)
<Link href="/products">Shop</Link>

// 从 (shop) 导航到 (marketing)
<Link href="/about">About</Link>
```

### 9. 路由组可以有自己的 not-found.tsx 吗?

可以。每个路由组可以有自己的 404 页面:

```
app/
├── (marketing)/
│   ├── not-found.tsx
│   └── about/
│       └── page.tsx
└── (shop)/
    ├── not-found.tsx
    └── products/
        └── page.tsx
```

### 10. 如何组织大型应用的路由组?

建议按功能模块划分:

```
app/
├── (public)/          # 公开页面
├── (auth)/            # 认证页面
├── (app)/             # 应用主体
├── (admin)/           # 管理后台
└── (api)/             # API 路由
```

每个路由组内部再按业务逻辑细分:

```
app/
└── (app)/
    ├── dashboard/     # 仪表板
    ├── projects/      # 项目管理
    ├── team/          # 团队管理
    └── settings/      # 设置
```

## 总结

路由组是 Next.js 16 中组织大型应用的强大工具。通过本文的学习,我们掌握了:

### 核心概念

1. **路由组语法**: 使用 `(folder)` 创建路由组,不影响 URL
2. **独立布局**: 每个路由组可以有自己的布局
3. **独立配置**: 每个路由组可以有自己的元数据和配置
4. **逻辑分组**: 将相关路由组织在一起

### 实战应用

我们通过四个实战场景学习了路由组的应用:

1. **电商应用**: 营销、商店、账户、管理四个路由组
2. **SaaS 应用**: 公开、认证、应用三个路由组
3. **多租户应用**: 租户、平台、共享三个路由组
4. **内容管理系统**: 网站、编辑器、管理三个路由组

### 高级用法

1. **嵌套路由组**: 路由组可以任意嵌套
2. **并行路由**: 路由组与并行路由结合使用
3. **拦截路由**: 路由组与拦截路由结合使用
4. **中间件集成**: 路由组与中间件配合使用

### 最佳实践

1. **清晰命名**: 使用描述性的路由组名称
2. **避免冲突**: 确保不同路由组的路由不会产生相同 URL
3. **合理分组**: 按功能模块或权限级别分组
4. **性能优化**: 在路由组布局中使用缓存
5. **团队协作**: 不同团队负责不同路由组

### 注意事项

1. 路由组不影响 URL 路径
2. 布局会嵌套渲染
3. 元数据会合并
4. 每个应用只能有一个根页面
5. 避免在路由组布局中进行重复的数据获取

通过合理使用路由组,可以让大型应用的文件结构更加清晰,代码更易维护,团队协作更顺畅。
