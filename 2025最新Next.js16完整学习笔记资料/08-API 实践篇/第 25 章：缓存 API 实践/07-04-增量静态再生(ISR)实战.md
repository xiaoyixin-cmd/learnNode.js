# 增量静态再生(ISR)实战

## 概述

增量静态再生(Incremental Static Regeneration, ISR)是 Next.js 的核心特性之一,允许在构建后更新静态页面,无需重新构建整个网站。ISR 结合了静态生成的性能优势和动态内容的灵活性。

### ISR 核心特性

1. **按需重新验证**: 在指定时间后重新生成页面
2. **即时更新**: 后台重新生成,用户看到旧内容直到新内容准备好
3. **无需重新构建**: 更新内容不需要重新部署
4. **自动缓存**: 生成的页面自动缓存
5. **回退支持**: 未生成的页面可以按需生成

### ISR 工作原理

```
用户请求 → 检查缓存 → 返回缓存页面 → 后台重新验证 → 更新缓存
```

```typescript
// lib/posts.ts
import { db } from "./db";

export async function getPosts() {
  const posts = await db.post.findMany({
    where: { published: true },
    include: {
      author: {
        select: {
          name: true,
          avatar: true,
        },
      },
      tags: true,
    },
    orderBy: {
      publishedAt: "desc",
    },
  });

  return posts;
}

export async function getPost(slug: string) {
  const post = await db.post.findUnique({
    where: { slug },
    include: {
      author: {
        select: {
          name: true,
          avatar: true,
          bio: true,
        },
      },
      tags: true,
      comments: {
        where: { approved: true },
        include: {
          user: {
            select: {
              name: true,
              avatar: true,
            },
          },
        },
        orderBy: {
          createdAt: "desc",
        },
      },
    },
  });

  return post;
}
```

```typescript
// app/admin/posts/[slug]/edit/actions.ts
"use server";

import { revalidatePath, revalidateTag } from "next/cache";
import { db } from "@/lib/db";

export async function updatePost(
  slug: string,
  data: {
    title?: string;
    content?: string;
    excerpt?: string;
    published?: boolean;
  }
) {
  const post = await db.post.update({
    where: { slug },
    data,
  });

  // 重新验证文章详情页
  revalidatePath(`/blog/${slug}`);

  // 重新验证文章列表页
  revalidatePath("/blog");

  // 使用标签重新验证
  revalidateTag("posts");
  revalidateTag(`post-${slug}`);

  return { success: true, post };
}

export async function publishPost(slug: string) {
  await db.post.update({
    where: { slug },
    data: {
      published: true,
      publishedAt: new Date(),
    },
  });

  revalidatePath("/blog");
  revalidatePath(`/blog/${slug}`);

  return { success: true };
}

export async function deletePost(slug: string) {
  await db.post.delete({
    where: { slug },
  });

  revalidatePath("/blog");

  return { success: true };
}
```

```typescript
// app/admin/products/[slug]/actions.ts
"use server";

import { revalidatePath, revalidateTag } from "next/cache";
import { db } from "@/lib/db";

export async function updateProduct(
  slug: string,
  data: {
    name?: string;
    price?: number;
    stock?: number;
    description?: string;
  }
) {
  const product = await db.product.update({
    where: { slug },
    data,
  });

  // 重新验证产品详情页
  revalidatePath(`/products/${slug}`);

  // 重新验证产品列表页
  revalidatePath("/products");

  // 使用标签重新验证
  revalidateTag("products");
  revalidateTag(`product-${slug}`);

  return { success: true, product };
}

export async function updateStock(slug: string, quantity: number) {
  await db.product.update({
    where: { slug },
    data: {
      stock: {
        increment: quantity,
      },
    },
  });

  // 只重新验证产品详情页
  revalidatePath(`/products/${slug}`);

  return { success: true };
}
```

```typescript
    </div>
  );
}
```

## ISR 配置策略

### 不同内容类型的重新验证时间

| 内容类型 | 重新验证时间   | 说明               |
| -------- | -------------- | ------------------ |
| 新闻文章 | 3-5 分钟       | 新闻更新频繁       |
| 博客文章 | 30 分钟-1 小时 | 文章发布后很少修改 |
| 产品页面 | 5-10 分钟      | 价格和库存可能变化 |
| 文档页面 | 30 分钟-1 小时 | 文档更新不频繁     |
| 用户资料 | 10-30 分钟     | 用户信息偶尔更新   |
| 配置页面 | 1-24 小时      | 配置很少变化       |

### 混合策略

```typescript
// 根据内容类型设置不同的重新验证时间
export function getRevalidateTime(contentType: string): number {
  const revalidateMap: Record<string, number> = {
    news: 180, // 3分钟
    blog: 1800, // 30分钟
    product: 600, // 10分钟
    doc: 3600, // 1小时
    profile: 1800, // 30分钟
    config: 86400, // 24小时
  };

  return revalidateMap[contentType] || 3600;
}

// 使用
export const revalidate = getRevalidateTime("blog");
```

### 适合使用 ISR 的场景

| 场景     | 说明             | 示例             |
| -------- | ---------------- | ---------------- |
| 内容网站 | 内容更新不太频繁 | 博客、新闻、文档 |
| 电商网站 | 产品信息相对稳定 | 产品列表、详情页 |
| 营销页面 | 页面内容定期更新 | 活动页、促销页   |

### 不适合使用 ISR 的场景

| 场景         | 说明                 | 替代方案           |
| ------------ | -------------------- | ------------------ |
| 实时数据     | 需要实时更新         | 客户端渲染 + API   |
| 用户特定内容 | 每个用户看到不同内容 | 动态渲染           |
| 高频更新     | 内容每秒都在变化     | WebSocket + 客户端 |
| 交互式应用   | 需要用户交互         | 客户端组件         |
| 个性化推荐   | 基于用户行为         | 服务端渲染 + 缓存  |

## 注意事项

### 1. 重新验证时间的选择

选择合适的重新验证时间很重要:

```typescript
// ❌ 过短的重新验证时间
export const revalidate = 1; // 1秒 - 失去了静态生成的优势

// ❌ 过长的重新验证时间
export const revalidate = 604800; // 7天 - 内容可能过时

// ✅ 合理的重新验证时间
export const revalidate = 600; // 10分钟 - 平衡性能和新鲜度
```

### 2. generateStaticParams 的数量

不要预生成太多页面:

```typescript
// ❌ 预生成所有页面
export async function generateStaticParams() {
  const posts = await getPosts(); // 可能有数千篇
  return posts.map((post) => ({ slug: post.slug }));
}

// ✅ 只预生成热门页面
export async function generateStaticParams() {
  const posts = await getPosts({ limit: 100 }); // 只预生成前100篇
  return posts.map((post) => ({ slug: post.slug }));
}
```

### 3. 按需重新验证的时机

合理使用 revalidatePath 和 revalidateTag:

```typescript
// ✅ 在数据更新后立即重新验证
export async function updatePost(slug: string, data: any) {
  await db.post.update({ where: { slug }, data });
  revalidatePath(`/posts/${slug}`);
  return { success: true };
}

// ❌ 过度使用重新验证
export async function viewPost(slug: string) {
  await db.post.update({
    where: { slug },
    data: { views: { increment: 1 } },
  });
  revalidatePath(`/posts/${slug}`); // 每次浏览都重新验证 - 不必要
}
```

### 4. 缓存层级

理解 Next.js 的多层缓存:

```
请求 → CDN 缓存 → Next.js 缓存 → 数据源
```

### 5. 部署平台的支持

确保部署平台支持 ISR:

- Vercel: 完全支持
- Netlify: 支持(需要配置)
- 自托管: 需要配置缓存存储

## 常见问题

### 1. ISR 和 SSR 有什么区别?

**问题**: 这两种渲染方式有什么不同?

**解答**:

| 特性       | ISR           | SSR        |
| ---------- | ------------- | ---------- |
| 渲染时机   | 构建时 + 按需 | 每次请求时 |
| 性能       | 快(静态)      | 慢(动态)   |
| 内容新鲜度 | 定期更新      | 实时       |
| 服务器负载 | 低            | 高         |
| 适用场景   | 内容网站      | 个性化内容 |

### 2. 如何调试 ISR?

**问题**: 如何确认页面是否被重新验证?

**解答**:

添加时间戳:

```typescript
export default async function Page() {
  const data = await getData();
  const buildTime = new Date().toISOString();

  return (
    <div>
      <p>构建时间: {buildTime}</p>
      <div>{data}</div>
    </div>
  );
}
```

### 3. 重新验证失败怎么办?

**问题**: 如果重新验证时数据获取失败?

**解答**:

Next.js 会继续提供旧的缓存内容:

```typescript
export default async function Page() {
  try {
    const data = await getData();
    return <div>{data}</div>;
  } catch (error) {
    // 返回降级内容
    return <div>内容暂时不可用</div>;
  }
}
```

### 4. 可以手动触发重新验证吗?

**问题**: 如何在不等待时间的情况下更新页面?

**解答**:

使用 revalidatePath 或 revalidateTag:

```typescript
"use server";

import { revalidatePath } from "next/cache";

export async function refreshPage(path: string) {
  revalidatePath(path);
  return { success: true };
}
```

### 5. ISR 会影响 SEO 吗?

**问题**: 使用 ISR 对 SEO 有影响吗?

**解答**:

不会。ISR 生成的是完整的 HTML,对 SEO 友好:

- 搜索引擎看到完整的 HTML
- 页面加载速度快
- 内容定期更新

### 6. 如何处理大量动态路由?

**问题**: 有数千个动态页面怎么办?

**解答**:

使用 fallback 策略:

```typescript
export async function generateStaticParams() {
  // 只预生成热门页面
  const popularPosts = await getPopularPosts(100);
  return popularPosts.map((post) => ({ slug: post.slug }));
}

// 其他页面按需生成
export const dynamicParams = true;
```

### 7. 重新验证会影响性能吗?

**问题**: 后台重新验证会影响用户体验吗?

**解答**:

不会。重新验证在后台进行:

1. 用户请求 → 返回缓存内容(快)
2. 后台重新验证 → 更新缓存
3. 下次请求 → 返回新内容

### 8. 如何监控 ISR 的效果?

**问题**: 如何知道 ISR 是否正常工作?

**解答**:

添加监控:

```typescript
export default async function Page() {
  const startTime = Date.now();
  const data = await getData();
  const duration = Date.now() - startTime;

  console.log(`Page generated in ${duration}ms`);

  return <div>{data}</div>;
}
```

### 9. 可以为不同用户显示不同内容吗?

**问题**: ISR 页面可以个性化吗?

**解答**:

可以结合客户端渲染:

```typescript
export default async function Page() {
  // 静态内容
  const staticData = await getStaticData();

  return (
    <div>
      <StaticContent data={staticData} />
      <ClientPersonalizedContent /> {/* 客户端渲染 */}
    </div>
  );
}
```

### 10. ISR 和 CDN 缓存有什么关系?

**问题**: ISR 和 CDN 如何配合?

**解答**:

ISR 生成的页面会被 CDN 缓存:

```
用户 → CDN(缓存) → Next.js(ISR) → 数据源
```

CDN 缓存提供更快的全球访问速度。

## 总结

### 核心概念

1. **按需重新验证**: 在指定时间后自动更新页面
2. **即时更新**: 后台重新生成,用户体验不受影响

4. **自动缓存**: 生成的页面自动缓存到 CDN
5. **回退支持**: 未预生成的页面可以按需生成
