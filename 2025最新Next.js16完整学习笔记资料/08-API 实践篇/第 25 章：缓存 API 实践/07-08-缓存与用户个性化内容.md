# 缓存与用户个性化内容

## 概述

在现代 Web 应用中,用户个性化内容和缓存策略往往存在矛盾。缓存追求的是复用,而个性化要求的是差异化。如何在保证个性化体验的同时,充分利用缓存提升性能,是一个需要仔细权衡的问题。

### 核心挑战

1. **缓存粒度**: 如何划分可缓存和不可缓存的内容
2. **缓存键设计**: 如何为不同用户生成合适的缓存键
3. **部分缓存**: 如何缓存页面的公共部分,动态加载个性化部分
4. **缓存失效**: 用户数据变化时如何精确失效相关缓存
5. **性能平衡**: 在个性化和性能之间找到最佳平衡点

### 常见场景

```typescript
// 场景1: 完全个性化(不可缓存)
// 用户仪表盘、个人设置页面

// 场景2: 部分个性化(混合缓存)
// 电商首页(推荐商品个性化,其他内容可缓存)

// 场景3: 基于角色的个性化(可缓存)
// 不同用户角色看到不同内容,但同角色用户看到相同内容

// 场景4: 基于地理位置的个性化(可缓存)
// 根据用户地理位置显示不同内容
```

### 缓存键设计

```typescript
// lib/cache/personalized-cache.ts

// 策略1: 基于用户ID的缓存
export function getUserCacheKey(userId: string, resource: string) {
  return `user:${userId}:${resource}`;
}

// 策略2: 基于用户角色的缓存
export function getRoleCacheKey(role: string, resource: string) {
  return `role:${role}:${resource}`;
}

// 策略3: 基于用户属性的缓存
export function getAttributeCacheKey(
  attributes: Record<string, any>,
  resource: string
) {
  // 将属性排序后生成稳定的键
  const sortedAttrs = Object.keys(attributes)
    .sort()
    .map((key) => `${key}:${attributes[key]}`)
    .join("|");

  return `attrs:${sortedAttrs}:${resource}`;
}

// 策略4: 基于用户分组的缓存
export function getSegmentCacheKey(segment: string, resource: string) {
  return `segment:${segment}:${resource}`;
}
```

interface CacheStrategy {
  level: "public" | "private" | "hybrid";
  ttl: number;
  keyGenerator: (context: any) => string;
}

// 公共内容缓存(所有用户共享)
export const publicCache: CacheStrategy = {
  level: "public",
  ttl: 3600, // 1小时
  keyGenerator: (context) => `public:${context.resource}`,
};

// 私有内容缓存(每个用户独立)
export const privateCache: CacheStrategy = {
  level: "private",
  ttl: 300, // 5分钟
  keyGenerator: (context) => `user:${context.userId}:${context.resource}`,
};

// 混合内容缓存(部分共享,部分个性化)
export const hybridCache: CacheStrategy = {
  level: "hybrid",
  ttl: 600, // 10分钟
  keyGenerator: (context) => {
    // 根据用户属性生成缓存键
    const segment = getUserSegment(context.user);
    return `segment:${segment}:${context.resource}`;
  },
};

// 根据用户属性确定用户分组
function getUserSegment(user: any): string {
  // 示例: 根据用户等级和地区分组
  return `${user.tier}-${user.region}`;
}

import { unstable_cache } from "next/cache";

export async function getDashboardData(userId: string) {
  // 并行获取数据
  const [user, stats, orders, announcements] = await Promise.all([
    getUserProfile(userId),
    getUserStats(userId),
    getRecentOrders(userId),
    getAnnouncements(), // 公共内容
  ]);

  return { user, stats, orders, announcements };
}

// 用户资料(个性化缓存)
const getUserProfile = unstable_cache(
  async (userId: string) => {
    return await db.user.findUnique({
      where: { id: userId },
      include: {
        profile: true,
        preferences: true,
      },
    });
  },
  ["user-profile"],
  {
    tags: (userId: string) => [`user-${userId}`, "profile"],
    revalidate: 300, // 5分钟
  }
);

// 用户统计(个性化缓存)
const getUserStats = unstable_cache(
  async (userId: string) => {
    const [orderCount, totalSpent, points] = await Promise.all([
      db.order.count({ where: { userId } }),
      db.order.aggregate({
        where: { userId, status: "completed" },
        _sum: { total: true },
      }),
      db.userPoints.findUnique({ where: { userId } }),
    ]);

    return {
      orderCount,
      totalSpent: totalSpent._sum.total || 0,
      points: points?.balance || 0,
    };
  },
  ["user-stats"],
  {
    tags: (userId: string) => [`user-${userId}`, "stats"],
    revalidate: 600, // 10分钟
  }
);

// 最近订单(个性化缓存)
const getRecentOrders = unstable_cache(
  async (userId: string) => {
    return await db.order.findMany({
      where: { userId },
      take: 5,
      orderBy: { createdAt: "desc" },
      include: {
        items: {
          include: {
            product: true,
          },
        },
      },
    });
  },
  ["user-recent-orders"],
  {
    tags: (userId: string) => [`user-${userId}`, "orders"],
    revalidate: 60, // 1分钟
  }
);

// 系统公告(公共缓存)
const getAnnouncements = unstable_cache(
  async () => {
    return await db.announcement.findMany({
      where: { active: true },
      orderBy: { createdAt: "desc" },
      take: 5,
    });
  },
  ["announcements"],
  {
    tags: ["announcements"],
    revalidate: 3600, // 1小时
  }

// middleware.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function middleware(request: NextRequest) {
  // 从请求头获取地理位置信息
  const country = request.geo?.country || "US";
  const region = request.geo?.region || "";
  const city = request.geo?.city || "";

  // 将地理位置信息添加到请求头
  const requestHeaders = new Headers(request.headers);
  requestHeaders.set("x-user-country", country);
  requestHeaders.set("x-user-region", region);
  requestHeaders.set("x-user-city", city);

  return NextResponse.next({
    request: {
      headers: requestHeaders,
    },
  });
}

export const config = {
  matcher: ["/((?!api|_next/static|_next/image|favicon.ico).*)"],
};

## 缓存策略对比

### 不同个性化程度的缓存策略

| 个性化程度 | 缓存策略 | TTL                | 缓存键示例                | 适用场景             |
| ---------- | -------- | ------------------ | ------------------------- | -------------------- |
| 完全公共   | 全局缓存 | 长(1-24 小时)      | `public:content`          | 静态页面、公告       |
| 基于角色   | 角色缓存 | 中(30 分钟-2 小时) | `role:vip:content`        | 会员专区、权限内容   |
| 基于地区   | 地区缓存 | 中(30 分钟-2 小时) | `country:US:content`      | 地区促销、本地化内容 |
| 基于分组   | 分组缓存 | 短(5-30 分钟)      | `segment:premium:content` | 用户分组推荐         |
| 完全个性化 | 用户缓存 | 短(1-10 分钟)      | `user:123:content`        | 个人仪表盘、购物车   |
| 实时数据   | 不缓存   | 无                 | -                         | 实时聊天、库存       |

## 适用场景

### 何时使用个性化缓存

| 场景       | 是否使用 | 缓存策略 | 说明                    |
| ---------- | -------- | -------- | ----------------------- |
| 电商首页   | 是       | 混合缓存 | 公共内容缓存,推荐个性化 |
| 用户仪表盘 | 是       | 用户缓存 | 短 TTL,精确失效         |
| 搜索结果   | 否       | 不缓存   | 查询参数太多,命中率低   |
| 产品详情   | 是       | 公共缓存 | 所有用户看到相同内容    |
| 购物车     | 是       | 用户缓存 | 实时性要求高,短 TTL     |
| 订单历史   | 是       | 用户缓存 | 变化不频繁,可缓存       |
| 实时聊天   | 否       | 不缓存   | 实时性要求极高          |
| 文章列表   | 是       | 分组缓存 | 基于用户兴趣分组        |

## 注意事项

### 1. 隐私和安全

个性化缓存可能泄露用户隐私:

```typescript
// 错误示例: 缓存键包含敏感信息
const badKey = `user:${email}:${password}:data`; // 不要这样做!

// 正确示例: 使用用户ID
const goodKey = `user:${userId}:data`;

// 敏感数据不要缓存
export async function getSensitiveData(userId: string) {
  // 直接从数据库获取,不缓存
  return await db.sensitiveData.findUnique({
    where: { userId },
  });
}
```

### 2. 缓存一致性

用户数据变化时及时失效缓存:

```typescript
// app/actions/user-actions.ts
"use server";

export async function updateUserPreferences(userId: string, preferences: any) {
  // 1. 更新数据库
  await db.user.update({
    where: { id: userId },
    data: { preferences },
  });

  // 2. 清除相关缓存
  revalidateTag(`user-${userId}`);

// 缓存监控
class PersonalizedCacheMonitor {
  private stats = {
    public: { hits: 0, misses: 0 },
    segment: { hits: 0, misses: 0 },
    user: { hits: 0, misses: 0 },
  };

  record(level: "public" | "segment" | "user", hit: boolean) {
    if (hit) {
      this.stats[level].hits++;
    } else {
      this.stats[level].misses++;
    }
  }

  getReport() {
    return Object.entries(this.stats).map(([level, data]) => ({
      level,
      hitRate: (data.hits / (data.hits + data.misses)) * 100,
      total: data.hits + data.misses,
    }));
  }
}

export const cacheMonitor = new PersonalizedCacheMonitor();

## 总结

缓存与用户个性化内容的结合是现代 Web 应用的重要课题。通过本文的学习,我们掌握了:

### 核心概念

1. **缓存粒度**: 公共、分组、用户三个层次
2. **缓存键设计**: 基于用户 ID、角色、属性、地理位置
3. **混合策略**: 公共内容缓存,个性化内容动态加载
4. **精确失效**: 用户数据变化时及时清除相关缓存

### 实战应用

我们通过四个实战场景学习了个性化缓存:

1. **电商首页**: 公共内容+个性化推荐的混合策略
2. **用户仪表盘**: 个性化数据的短 TTL 缓存
3. **基于角色**: 同角色用户共享缓存
4. **地理位置**: 基于地区的内容个性化

### 最佳实践

1. **分层缓存**: 公共 → 分组 → 用户,逐层细化
2. **合理 TTL**: 根据数据变化频率设置 TTL
3. **精确失效**: 数据变化时及时清除相关缓存
4. **监控优化**: 跟踪命中率,持续优化策略
5. **成本控制**: 只缓存活跃用户,使用压缩

### 注意事项

1. 保护用户隐私,不缓存敏感数据
2. 保持缓存一致性,及时失效
3. 防止缓存穿透,验证输入
4. 控制内存使用,限制缓存大小
5. 简化测试,提供测试辅助函数

通过合理设计个性化缓存策略,可以在保证用户体验的同时,充分利用缓存提升性能,降低服务器负载。

// lib/cache/smart-keys.ts

// 基于用户属性生成智能缓存键
export function generateSmartKey(user: any, resource: string) {
  // 提取关键属性
  const keyAttributes = {
    tier: user.tier,
    region: user.region,
    language: user.language,
  };

  // 生成稳定的键
  const attributeHash = hashObject(keyAttributes);

  return `smart:${resource}:${attributeHash}`;
}

function hashObject(obj: any): string {
  const str = JSON.stringify(obj, Object.keys(obj).sort());
  return createHash("md5").update(str).digest("hex").substring(0, 8);
}

// 使用示例
export async function getSmartContent(user: any) {
  const cacheKey = generateSmartKey(user, "content");

  return unstable_cache(
    async () => {
      return await fetchContentForUser(user);
    },
    [cacheKey],
    { revalidate: 600 }
  )();
}

// lib/cache/sharding.ts

// 将大数据集分片缓存
export async function getShardedData(userId: string, page: number) {
  const shardSize = 100;
  const shardIndex = Math.floor(page / shardSize);

  return await getDataShard(userId, shardIndex, page % shardSize);
}

const getDataShard = unstable_cache(
  async (userId: string, shardIndex: number, offset: number) => {
    const data = await db.data.findMany({
      where: { userId },
      skip: shardIndex * 100 * 20,
      take: 100 * 20, // 缓存100页的数据
    });

    // 返回当前页的数据
    return data.slice(offset * 20, (offset + 1) * 20);
  },
  ["data-shard"],
  {
    tags: (userId: string, shardIndex: number) => [
      `user-${userId}`,
      `shard-${shardIndex}`,
    ],
    revalidate: 600,
  }
);
```

// lib/cache/fallback.ts

// 缓存失败时的降级策略
export async function getDataWithFallback(userId: string) {
  try {
    // 尝试从缓存获取
    return await getCachedData(userId);
  } catch (error) {
    console.error("Cache error:", error);

    // 降级到数据库
    return await getDataFromDB(userId);
  }
}

// 多级降级
export async function getDataWithMultipleFallbacks(userId: string) {
  // 1. 尝试用户专属缓存
  try {
    return await getUserCache(userId);
  } catch (e1) {
    // 2. 尝试分组缓存
    try {
      const segment = await getUserSegment(userId);
      return await getSegmentCache(segment);
    } catch (e2) {
      // 3. 使用默认数据
      return await getDefaultData();
    }
  }
}

### 5. 缓存监控和告警

```typescript
// lib/cache/monitoring.ts

class CacheMonitor {
  private metrics = {
    hits: 0,
    misses: 0,
    errors: 0,
    latency: [] as number[],
  };

  recordHit() {
    this.metrics.hits++;
  }

  recordMiss() {
    this.metrics.misses++;
  }

  recordError() {
    this.metrics.errors++;
  }

  recordLatency(ms: number) {
    this.metrics.latency.push(ms);

    // 只保留最近1000条
    if (this.metrics.latency.length > 1000) {
      this.metrics.latency.shift();
    }
  }

  getMetrics() {
    const hitRate =
      this.metrics.hits / (this.metrics.hits + this.metrics.misses);
    const avgLatency =
      this.metrics.latency.reduce((a, b) => a + b, 0) /
      this.metrics.latency.length;

    return {
      hitRate: (hitRate * 100).toFixed(2) + "%",
      hits: this.metrics.hits,
      misses: this.metrics.misses,
      errors: this.metrics.errors,
      avgLatency: avgLatency.toFixed(2) + "ms",
    };
  }

  // 检查是否需要告警
  checkAlerts() {
    const hitRate =
      this.metrics.hits / (this.metrics.hits + this.metrics.misses);

    if (hitRate < 0.5) {
      console.warn("Cache hit rate is below 50%:", hitRate);
    }

    if (this.metrics.errors > 100) {
      console.error("Too many cache errors:", this.metrics.errors);
    }
  }
}

export const cacheMonitor = new CacheMonitor();

// 使用示例
export async function getMonitoredData(userId: string) {
  const start = Date.now();

  try {
    const data = await getCachedData(userId);

    if (data) {
      cacheMonitor.recordHit();
    } else {
      cacheMonitor.recordMiss();
    }

    return data;
  } catch (error) {
    cacheMonitor.recordError();
    throw error;
  } finally {
    cacheMonitor.recordLatency(Date.now() - start);
  }
}
```
