# 条件路由实现策略

## 1. 概述

条件路由是指根据特定条件(如用户权限、设备类型、地理位置、A/B 测试等)动态决定渲染哪个页面或组件的路由策略。在 Next.js 16 中,虽然没有专门的"条件路由"API,但通过组合使用并行路由、中间件、服务端组件等特性,可以实现强大而灵活的条件路由系统。

### 1.1 概念定义

条件路由不是一个独立的路由类型,而是一种路由设计模式。它的核心思想是:同一个 URL 路径,根据不同的条件,可以渲染完全不同的内容。

**关键特征**:

- **URL 一致性**: 不同条件下的用户访问相同的 URL
- **内容差异化**: 根据条件渲染不同的页面或组件
- **动态判断**: 条件判断可以在服务端或客户端进行
- **无缝切换**: 条件变化时,内容可以平滑过渡

**与传统路由的区别**:

传统路由是静态的,一个 URL 对应一个固定的页面。条件路由是动态的,一个 URL 可以对应多个可能的页面,具体渲染哪个取决于运行时的条件。

### 1.2 核心价值

**个性化用户体验**:

根据用户的角色、偏好、历史行为等,展示定制化的内容。例如,同一个首页,普通用户看到的是营销内容,已登录用户看到的是个性化推荐,管理员看到的是管理面板。

**A/B 测试与灰度发布**:

在不改变 URL 的情况下,为不同用户群体展示不同版本的页面,收集数据进行对比分析。这对于产品迭代和优化至关重要。

**多端适配**:

根据设备类型(桌面端、移动端、平板)或浏览器特性,渲染优化过的页面版本,提供最佳的用户体验。

**权限控制**:

根据用户权限动态展示不同的页面内容,避免未授权访问,同时保持 URL 结构的简洁性。

**国际化与本地化**:

根据用户的地理位置或语言偏好,展示本地化的内容,而不需要在 URL 中显式包含语言代码。

### 1.3 实现方式对比

Next.js 16 提供了多种实现条件路由的方式,每种方式都有其适用场景:

| 实现方式           | 判断时机     | 性能影响 | 适用场景               | 复杂度 |
| :----------------- | :----------- | :------- | :--------------------- | :----- |
| 中间件             | 请求到达时   | 最小     | 重定向、重写、简单条件 | 低     |
| 并行路由           | 渲染时       | 较小     | 多版本内容并存         | 中     |
| 服务端组件条件渲染 | 服务端渲染时 | 中等     | 复杂业务逻辑           | 中     |
| 客户端条件渲染     | 客户端渲染时 | 较大     | 交互式条件切换         | 低     |
| 路由处理器         | API 请求时   | 最小     | API 路由条件处理       | 低     |

## 2. 核心概念与原理

### 2.1 条件判断的时机

条件路由的判断可以发生在请求处理的不同阶段,每个阶段都有其特点:

**请求拦截阶段(中间件)**:

这是最早的判断时机,发生在请求到达应用之前。中间件可以检查请求头、Cookie、URL 参数等,然后决定:

- 重定向到另一个 URL
- 重写请求到不同的路由
- 修改请求头或响应头
- 直接返回响应

**优势**: 性能最好,可以避免不必要的页面渲染
**劣势**: 无法访问数据库或复杂的业务逻辑

**服务端渲染阶段**:

在服务端组件中进行条件判断,可以访问数据库、调用 API、执行复杂的业务逻辑。

**优势**: 功能最强大,可以进行复杂的条件判断
**劣势**: 性能开销较大,每次请求都需要执行

**客户端渲染阶段**:

在客户端组件中进行条件判断,可以响应用户交互、访问浏览器 API。

**优势**: 可以实现动态的条件切换,无需重新加载页面
**劣势**: 不利于 SEO,首屏渲染可能较慢

条件路由的判断依据可以来自多个来源:

**用户身份与权限**:

### 2.3 条件路由的实现模式

**模式 1: 中间件重定向**:

适用于需要完全跳转到不同 URL 的场景。

```tsx
// middleware.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function middleware(request: NextRequest) {
  const userRole = request.cookies.get("user_role")?.value;

  if (request.nextUrl.pathname === "/dashboard") {
    if (userRole === "admin") {
      return NextResponse.redirect(new URL("/admin/dashboard", request.url));
    } else if (userRole === "user") {
      return NextResponse.redirect(new URL("/user/dashboard", request.url));
    } else {
      return NextResponse.redirect(new URL("/login", request.url));
    }
  }

  return NextResponse.next();
}
```

**模式 2: 中间件重写**:

适用于保持 URL 不变,但渲染不同页面的场景。

```tsx
// middleware.ts
export function middleware(request: NextRequest) {
  const isMobile = /mobile/i.test(request.headers.get("user-agent") || "");

  if (request.nextUrl.pathname === "/") {
    if (isMobile) {
      return NextResponse.rewrite(new URL("/mobile-home", request.url));
    }
  }

  return NextResponse.next();
}
```

**模式 3: 并行路由条件渲染**:

适用于需要在同一布局中展示不同内容的场景。

```tsx
// app/layout.tsx
export default async function Layout({
  children,
  admin,
  user,
}: {
  children: React.ReactNode;
  admin: React.ReactNode;
  user: React.ReactNode;
}) {
  const session = await auth();
  const role = session?.user?.role;

  return (
    <div>
      {children}
      {role === "admin" ? admin : user}
    </div>
  );
}
```

**模式 4: 组件级条件渲染**:

适用于页面内部的条件逻辑。

```tsx
// app/page.tsx
export default async function Page() {
  const session = await auth();

  if (!session) {
    return <LoginPrompt />;
  }

  if (session.user.role === "admin") {
    return <AdminView />;
  }

  return <UserView />;
}
```

## 3. 适用场景

### 3.1 权限控制与角色管理

**场景描述**:

企业应用中,不同角色的用户需要访问不同的功能。例如,管理员可以访问所有功能,普通用户只能访问部分功能,访客只能浏览公开内容。

**实现方案**:

```tsx
// app/dashboard/page.tsx
import { auth } from "@/lib/auth";
import { redirect } from "next/navigation";

export default async function DashboardPage() {
  const session = await auth();

  if (!session) {
    redirect("/login");
  }

  const role = session.user.role;

  switch (role) {
    case "admin":
      return <AdminDashboard />;
    case "manager":
      return <ManagerDashboard />;
    case "employee":
      return <EmployeeDashboard />;
    default:
      return <UnauthorizedPage />;
  }
}
```

**使用中间件保护路由**:

```tsx
// middleware.ts
import { auth } from "@/lib/auth";
import { NextResponse } from "next/server";

export async function middleware(request: NextRequest) {
  const session = await auth();
  const path = request.nextUrl.pathname;

  // 管理员路由保护
  if (path.startsWith("/admin")) {
    if (!session || session.user.role !== "admin") {
      return NextResponse.redirect(new URL("/unauthorized", request.url));
    }
  }

  // 用户路由保护
  if (path.startsWith("/dashboard")) {
    if (!session) {
      return NextResponse.redirect(new URL("/login", request.url));
    }
  }

  return NextResponse.next();
}

export const config = {
  matcher: ["/admin/:path*", "/dashboard/:path*"],
};
```

### 3.2 多端适配

**场景描述**:

同一个应用需要在桌面端、移动端、平板上提供不同的用户体验。移动端需要简化的界面,桌面端可以展示更多信息。

**实现方案**:

```tsx
// middleware.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function middleware(request: NextRequest) {
  const userAgent = request.headers.get("user-agent") || "";
  const url = request.nextUrl.clone();

  // 检测设备类型
  const isMobile = /mobile/i.test(userAgent);
  const isTablet = /tablet|ipad/i.test(userAgent);

  // 重写到对应的设备版本
  if (url.pathname === "/") {
    if (isMobile) {
      url.pathname = "/mobile";
      return NextResponse.rewrite(url);
    } else if (isTablet) {
      url.pathname = "/tablet";
      return NextResponse.rewrite(url);
    }
  }

  return NextResponse.next();
}
```

**使用并行路由实现**:

```

├── @desktop/
│   └── page.tsx
├── @mobile/
│   └── page.tsx
└── @tablet/
    └── page.tsx
```

```tsx
export default async function Layout({
```

### 3.3 A/B 测试

**场景描述**:

产品团队想要测试两个不同的页面设计,看哪个转化率更高。需要将用户随机分配到 A 组或 B 组,并记录他们的行为数据。

**实现方案**:

```tsx
// middleware.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function middleware(request: NextRequest) {
  const response = NextResponse.next();

  // 检查是否已经分配了测试组
  let variant = request.cookies.get("ab_test_variant")?.value;

  if (!variant) {
    // 随机分配到 A 或 B 组
    variant = Math.random() < 0.5 ? "A" : "B";
    response.cookies.set("ab_test_variant", variant, {
      maxAge: 60 * 60 * 24 * 30, // 30 天
    });
  }

  // 重写到对应的版本
  if (request.nextUrl.pathname === "/landing") {
    const url = request.nextUrl.clone();
    url.pathname = `/landing-${variant.toLowerCase()}`;
    return NextResponse.rewrite(url);
  }

  return response;
}
```

**在页面中使用**:

```tsx
// app/landing/page.tsx
import { cookies } from "next/headers";

export default async function LandingPage() {
  const cookieStore = await cookies();
  const variant = cookieStore.get("ab_test_variant")?.value || "A";

  return <div>{variant === "A" ? <VersionA /> : <VersionB />}</div>;
}
```

### 3.4 国际化与本地化

**场景描述**:

应用需要支持多个国家和地区,根据用户的地理位置或语言偏好,展示本地化的内容。

**实现方案**:

```tsx
// middleware.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

const locales = ["en", "zh", "ja", "ko"];
const defaultLocale = "en";

export function middleware(request: NextRequest) {
  // 从 Cookie 或 Accept-Language 头获取语言偏好
  const cookieLocale = request.cookies.get("locale")?.value;
  const acceptLanguage = request.headers.get("accept-language");

  let locale = cookieLocale || defaultLocale;

  if (!cookieLocale && acceptLanguage) {
    // 解析 Accept-Language 头
    const preferredLocale = acceptLanguage.split(",")[0].split("-")[0];

    if (locales.includes(preferredLocale)) {
      locale = preferredLocale;
    }
  }

  // 重写到对应的语言版本
  const url = request.nextUrl.clone();
  url.pathname = `/${locale}${url.pathname}`;

  const response = NextResponse.rewrite(url);
  response.cookies.set("locale", locale);

  return response;
}
```

### 3.5 功能开关与灰度发布

**场景描述**:

新功能开发完成后,不想立即对所有用户开放,而是先对一小部分用户进行灰度测试,收集反馈后再逐步扩大范围。

**实现方案**:

```tsx
// lib/feature-flags.ts
export async function isFeatureEnabled(
  featureName: string,
  userId: string
): Promise<boolean> {
  // 从数据库或配置服务获取功能开关状态
  const feature = await db.featureFlag.findUnique({
    where: { name: featureName },
  });

  if (!feature || !feature.enabled) {
    return false;
  }

  // 检查用户是否在白名单中
  if (feature.whitelist.includes(userId)) {
    return true;
  }

  // 根据百分比随机开启
  const hash = hashUserId(userId);
  return hash % 100 < feature.percentage;
}
```

### 3.6 时间敏感内容

**场景描述**:

根据当前时间展示不同的内容,例如节日促销、限时活动、营业时间等。

**实现方案**:

```tsx
// app/page.tsx
export default function Page() {
  const now = new Date();
  const isHoliday = checkIfHoliday(now);
  const isBusinessHours = checkBusinessHours(now);

  if (isHoliday) {
    return <HolidayPromotion />;
  }

  if (!isBusinessHours) {
    return <ClosedNotice />;
  }

  return <NormalPage />;
}

function checkIfHoliday(date: Date): boolean {
  const month = date.getMonth() + 1;
  const day = date.getDate();

  // 检查是否是特定节日
  if (month === 12 && day === 25) return true; // 圣诞节
  if (month === 1 && day === 1) return true; // 元旦

  return false;
}

function checkBusinessHours(date: Date): boolean {
  const hour = date.getHours();
  const day = date.getDay();

  // 周末不营业
  if (day === 0 || day === 6) return false;

  // 营业时间 9:00 - 18:00
  return hour >= 9 && hour < 18;
}
```

### 5.4 使用中间件实现全局条件路由

**场景**: 在中间件层面实现设备检测和重定向

```tsx
// middleware.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl;
  const userAgent = request.headers.get("user-agent") || "";

  // 检测设备类型
  const isMobile = /mobile/i.test(userAgent);
  const isTablet = /tablet|ipad/i.test(userAgent);
  const isBot = /bot|crawler|spider/i.test(userAgent);

  // 对于爬虫,总是返回桌面版
  if (isBot) {
    return NextResponse.next();
  }

  // 移动端重定向
  if (isMobile && !pathname.startsWith("/m")) {
    const url = request.nextUrl.clone();
    url.pathname = `/m${pathname}`;
    return NextResponse.redirect(url);
  }

  // 平板重定向
  if (isTablet && !pathname.startsWith("/tablet")) {
    const url = request.nextUrl.clone();
    url.pathname = `/tablet${pathname}`;
    return NextResponse.redirect(url);
  }

  // 桌面端访问移动端路径,重定向回桌面版
  if (!isMobile && !isTablet && pathname.startsWith("/m")) {
    const url = request.nextUrl.clone();
    url.pathname = pathname.replace(/^\/m/, "") || "/";
    return NextResponse.redirect(url);
  }

  return NextResponse.next();
}

export const config = {
  matcher: ["/((?!api|_next/static|_next/image|favicon.ico).*)"],
};
```

### 5.6 A/B 测试完整实现

**场景**: 实现一个完整的 A/B 测试系统,包括分组、追踪和数据收集

```tsx
// lib/ab-test.ts
import { cookies } from "next/headers";

export type Variant = "A" | "B";

export async function getVariant(testName: string): Promise<Variant> {
  const cookieStore = await cookies();
  const cookieName = `ab_test_${testName}`;

  // 检查是否已经分配了变体
  let variant = cookieStore.get(cookieName)?.value as Variant;

  if (!variant) {
    // 随机分配变体
    variant = Math.random() < 0.5 ? "A" : "B";

    // 保存到 Cookie(30 天)
    cookieStore.set(cookieName, variant, {
      maxAge: 60 * 60 * 24 * 30,
      httpOnly: true,
      sameSite: "lax",
    });

    // 记录分配事件
    await trackEvent({
      event: "ab_test_assigned",
      testName,
      variant,
    });
  }

  return variant;
}

export async function trackEvent(data: {
  event: string;
  testName: string;
  variant: Variant;
  [key: string]: any;
}) {
  // 发送到分析服务
  await fetch("/api/analytics", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(data),
  });
}
```

## 6. 注意事项

### 6.1 性能考虑

**避免在中间件中执行耗时操作**:

中间件在每个请求上都会执行,应该保持轻量和快速。避免在中间件中:

- 查询数据库
- 调用外部 API
- 执行复杂计算

如果需要这些操作,应该在页面组件中进行。

**缓存条件判断结果**:

如果条件判断涉及数据库查询或 API 调用,应该使用缓存来减少重复计算:

```tsx
import { cache } from "react";

const getUserRole = cache(async (userId: string) => {
  const user = await db.user.findUnique({
    where: { id: userId },
    select: { role: true },
  });
  return user?.role;
});

export default async function Page() {
  const session = await auth();
  const role = await getUserRole(session.user.id);

  // 使用 role 进行条件渲染
}
```

### 6.2 SEO 影响

**确保爬虫能访问正确的内容**:

搜索引擎爬虫通常不会执行 JavaScript 或保存 Cookie,所以基于客户端条件的路由可能导致 SEO 问题。

**解决方案**:

- 在服务端进行条件判断
- 为爬虫提供默认的内容版本
- 使用 User-Agent 检测爬虫并返回优化的内容

```tsx
// middleware.ts
export function middleware(request: NextRequest) {
  const userAgent = request.headers.get("user-agent") || "";
  const isBot = /bot|crawler|spider|googlebot/i.test(userAgent);

  if (isBot) {
    // 为爬虫返回 SEO 优化的版本
    return NextResponse.rewrite(new URL("/seo-optimized", request.url));
  }

  // 为普通用户返回正常版本
  return NextResponse.next();
}
```

### 6.3 用户体验

**避免闪烁**:

如果在客户端进行条件判断,可能会导致页面闪烁(先显示默认内容,然后切换到条件内容)。

**解决方案**:

- 尽量在服务端进行条件判断
- 使用 loading 状态平滑过渡
- 使用 Suspense 边界

```tsx
// app/page.tsx
import { Suspense } from "react";

export default function Page() {
  return (
    <Suspense fallback={<LoadingSpinner />}>
      <ConditionalContent />
    </Suspense>
  );
}

async function ConditionalContent() {
  const session = await auth();

  if (!session) {
    return <GuestContent />;
  }

  return <UserContent />;
}
```

### 6.4 安全性

**不要在客户端暴露敏感信息**:

条件判断的逻辑如果在客户端执行,可能会暴露敏感信息。例如,不要在客户端代码中硬编码管理员 ID 或权限规则。

**错误示例**:

```tsx
"use client";

export default function Page() {
  const userId = getUserId();

  // ❌ 不安全:权限判断在客户端
  if (userId === "admin-123") {
    return <AdminPanel />;
  }

  return <UserPanel />;
}
```

**正确示例**:

```tsx
// ✅ 安全:权限判断在服务端
export default async function Page() {
  const session = await auth();

  if (session?.user?.role === "admin") {
    return <AdminPanel />;
  }

  return <UserPanel />;
}
```

**使用中间件保护敏感路由**:

```tsx
// middleware.ts
export async function middleware(request: NextRequest) {
  const session = await auth();

  // 保护管理员路由
  if (request.nextUrl.pathname.startsWith("/admin")) {
    if (!session || session.user.role !== "admin") {
      return NextResponse.redirect(new URL("/unauthorized", request.url));
    }
  }

  return NextResponse.next();
}
```

## 7. 常见问题

### 7.1 中间件和页面组件中的条件判断有什么区别?

**中间件**:

- 在请求到达应用之前执行
- 性能最好,可以避免不必要的页面渲染
- 无法访问数据库或复杂的业务逻辑
- 适合简单的条件判断,如设备检测、权限检查

**页面组件**:

- 在服务端渲染时执行
- 可以访问数据库、调用 API、执行复杂逻辑
- 性能开销较大
- 适合复杂的业务逻辑判断

**选择建议**:

- 如果条件判断只需要请求头、Cookie 等信息,使用中间件
- 如果需要查询数据库或执行复杂逻辑,使用页面组件
- 可以组合使用:中间件做初步筛选,页面组件做详细判断

### 7.2 如何在条件路由中保持 URL 不变?

使用 `NextResponse.rewrite()` 而不是 `NextResponse.redirect()`:

```tsx
// middleware.ts
export function middleware(request: NextRequest) {
  const isMobile = /mobile/i.test(request.headers.get("user-agent") || "");

  if (isMobile) {
    // ✅ 使用 rewrite,URL 保持不变
    return NextResponse.rewrite(new URL("/mobile", request.url));

    // ❌ 使用 redirect,URL 会改变
    // return NextResponse.redirect(new URL('/mobile', request.url));
  }

  return NextResponse.next();
}
```

### 7.3 条件路由会影响 SEO 吗?

**可能会影响**,取决于实现方式:

**不影响 SEO 的方式**:

- 在服务端进行条件判断
- 使用 `NextResponse.rewrite()` 保持 URL 不变
- 为搜索引擎爬虫提供一致的内容

**可能影响 SEO 的方式**:

- 在客户端进行条件判断
- 使用 JavaScript 动态切换内容
- 不同用户看到完全不同的内容(可能被视为隐藏内容)

**最佳实践**:

```tsx
// middleware.ts
export function middleware(request: NextRequest) {
  const userAgent = request.headers.get("user-agent") || "";
  const isBot = /bot|crawler|spider/i.test(userAgent);

  // 为爬虫提供标准版本
  if (isBot) {
    return NextResponse.next();
  }

  // 为普通用户提供个性化版本
  const isMobile = /mobile/i.test(userAgent);
  if (isMobile) {
    return NextResponse.rewrite(new URL("/mobile", request.url));
  }

  return NextResponse.next();
}
```

### 7.5 条件路由可以嵌套吗?

可以,但要注意避免过度复杂:

```tsx
// app/page.tsx
export default async function Page() {
  const session = await auth();

  // 第一层条件:登录状态
  if (!session) {
    return <GuestPage />;
  }

  const role = session.user.role;

  // 第二层条件:用户角色
  if (role === "admin") {

    if (subscription?.plan === "enterprise") {
      return <EnterpriseAdminPage />;
    }

    return <StandardAdminPage />;
  }

  return <UserPage />;
}
```

**建议**:

- 嵌套不要超过 3 层
- 考虑提取条件判断逻辑到单独的函数
- 使用早期返回(early return)保持代码清晰

### 7.7 条件路由的性能开销有多大?

**中间件条件路由**:

- 性能开销极小,通常在 1-5ms
- 不会增加页面渲染时间
- 适合高流量应用

**服务端组件条件路由**:

- 性能开销取决于条件判断的复杂度
- 简单判断(检查 Cookie):1-10ms
- 数据库查询:10-100ms
- 外部 API 调用:100-1000ms

**优化建议**:

- 使用缓存减少重复计算
- 将耗时操作移到后台任务
- 使用 Streaming 和 Suspense 优化用户体验

```tsx
// app/page.tsx
```

## 8. 总结

条件路由是 Next.js 应用中实现个性化用户体验的关键技术。通过合理使用中间件、并行路由、服务端组件等特性,可以构建灵活而强大的条件路由系统。

**核心要点**:

1. **选择合适的实现方式**: 根据条件判断的复杂度和性能要求,选择中间件、并行路由或服务端组件
2. **保持简洁**: 避免过度复杂的嵌套条件,使用早期返回和函数提取保持代码清晰
3. **注重性能**: 使用缓存、避免重复计算、合理使用 Streaming 和 Suspense
4. **考虑 SEO**: 在服务端进行条件判断,为搜索引擎提供一致的内容
5. **确保安全**: 不要在客户端暴露敏感信息,使用中间件保护敏感路由
6. **充分测试**: 为条件路由编写测试,确保各种情况下的正确性

**适用场景**:

- 权限控制与角色管理
- 多端适配(桌面端、移动端、平板)
