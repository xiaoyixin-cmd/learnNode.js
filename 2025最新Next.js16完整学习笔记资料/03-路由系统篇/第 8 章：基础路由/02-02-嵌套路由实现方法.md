# 嵌套路由实现方法

## 概述

嵌套路由是 Next.js App Router 的核心特性之一，允许你创建具有层级结构的页面布局。通过文件夹嵌套，可以实现复杂的页面结构，同时保持代码的组织性和可维护性。

在 Next.js 16 中，嵌套路由与布局系统紧密结合，支持共享布局、并行路由、拦截路由等高级功能。理解嵌套路由的实现方法对于构建大型应用至关重要。

本章将详细介绍嵌套路由的实现方法、布局系统、最佳实践以及常见应用场景。

---

## 基础概念

### 文件夹嵌套

Next.js 使用文件夹结构定义路由层级：

```
app/
├── page.tsx                    # /
├── about/
│   └── page.tsx                # /about
├── blog/
│   ├── page.tsx                # /blog
│   └── [slug]/
│       └── page.tsx            # /blog/[slug]
└── dashboard/
    ├── page.tsx                # /dashboard
    ├── settings/
    │   └── page.tsx            # /dashboard/settings
    └── analytics/
        └── page.tsx            # /dashboard/analytics
```

### 路由段

每个文件夹代表一个路由段，对应 URL 中的一部分：

| 文件路径                        | URL                 | 说明         |
| ------------------------------- | ------------------- | ------------ |
| app/page.tsx                    | /                   | 根路由       |
| app/blog/page.tsx               | /blog               | 一级路由     |
| app/blog/[slug]/page.tsx        | /blog/hello         | 二级动态路由 |
| app/dashboard/settings/page.tsx | /dashboard/settings | 二级静态路由 |

## 布局系统

### 根布局

根布局是必需的，应用于所有页面：

```typescript
// app/layout.tsx
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="zh-CN">
      <body>
        <header>
          <nav>全局导航</nav>
        </header>
        <main>{children}</main>
        <footer>全局页脚</footer>
      </body>
    </html>
  );
}
```

### 嵌套布局

在子路由中创建布局：

```typescript
// app/dashboard/layout.tsx
export default function DashboardLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <div className="dashboard">
      <aside>
        <nav>
          <a href="/dashboard">概览</a>
          <a href="/dashboard/settings">设置</a>
          <a href="/dashboard/analytics">分析</a>
        </nav>
      </aside>
      <div className="content">{children}</div>
    </div>
  );
}
```

```typescript
// app/dashboard/page.tsx
export default function DashboardPage() {
  return <h1>Dashboard Overview</h1>;
}
```

```typescript
// app/dashboard/settings/page.tsx
export default function SettingsPage() {
  return <h1>Settings</h1>;
}
```

访问 `/dashboard/settings` 时的渲染层级：

```
RootLayout
  └── DashboardLayout
      └── SettingsPage
```

### 布局嵌套规则

布局会自动嵌套：

```
app/
├── layout.tsx              # 根布局
├── page.tsx
└── dashboard/
    ├── layout.tsx          # Dashboard 布局
    ├── page.tsx
    └── settings/
        ├── layout.tsx      # Settings 布局
        └── page.tsx
```

渲染结构：

```
RootLayout
  └── DashboardLayout
      └── SettingsLayout
          └── SettingsPage
```

## 共享布局

### 持久化布局

布局在路由切换时不会重新渲染：

```typescript
// app/blog/layout.tsx
"use client";

import { useState } from "react";

export default function BlogLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  const [count, setCount] = useState(0);

  return (
    <div>
      <div>
        <button onClick={() => setCount(count + 1)}>Count: {count}</button>
      </div>
      {children}
    </div>
  );
}
```

在 `/blog/post-1` 和 `/blog/post-2` 之间切换时，count 状态会保持。

### 部分渲染

只有页面内容会重新渲染，布局保持不变：

```
app/
├── layout.tsx              # 不重新渲染
└── blog/
    ├── layout.tsx          # 不重新渲染
    ├── [slug]/
    │   └── page.tsx        # 重新渲染
    └── page.tsx            # 重新渲染
```

## 模板

### Template vs Layout

模板在每次导航时都会重新渲染：

```typescript
// app/blog/template.tsx
"use client";

import { useState } from "react";

export default function BlogTemplate({
  children,
}: {
  children: React.ReactNode;
}) {
  const [count, setCount] = useState(0);

  console.log("Template rendered");

  return (
    <div>
      <div>Count: {count}</div>
      {children}
    </div>
  );
}
```

| 特性     | Layout  | Template            |
| -------- | ------- | ------------------- |
| 重新渲染 | 否      | 是                  |
| 状态保持 | 是      | 否                  |
| 使用场景 | 共享 UI | 每次都需要重置的 UI |

### 使用场景

Template 适用于：

1. 需要在每次导航时重置状态
2. 需要在每次导航时触发动画
3. 需要在每次导航时执行副作用

```typescript
// app/blog/template.tsx
"use client";

import { useEffect } from "react";
import { motion } from "framer-motion";

export default function BlogTemplate({
  children,
}: {
  children: React.ReactNode;
}) {
  useEffect(() => {
    console.log("Page changed");
  }, []);

  return (
    <motion.div
      initial={{ opacity: 0 }}
      animate={{ opacity: 1 }}
      exit={{ opacity: 0 }}
    >
      {children}
    </motion.div>
  );
}
```

## 5. API 签名与配置

### 5.1 Layout 组件签名

```tsx
export default function Layout({
  children,
  params,
}: {
  children: React.ReactNode;
  params?: Promise<{ [key: string]: string | string[] }>;
}) {
  return <>{children}</>;
}
```

**参数说明**:

| 参数       | 类型              | 说明               |
| :--------- | :---------------- | :----------------- |
| `children` | `React.ReactNode` | 子路由的内容       |
| `params`   | `Promise<object>` | 动态路由参数(可选) |

### 5.2 Template 组件签名

```tsx
export default function Template({ children }: { children: React.ReactNode }) {
  return <>{children}</>;
}
```

**参数说明**:

| 参数       | 类型              | 说明         |
| :--------- | :---------------- | :----------- |
| `children` | `React.ReactNode` | 子路由的内容 |

### 5.3 generateMetadata 函数

为嵌套路由生成元数据:

```tsx
import { Metadata } from "next";

interface PageProps {
  params: Promise<{ slug: string }>;
}

export async function generateMetadata({
  params,
}: PageProps): Promise<Metadata> {
  const { slug } = await params;

  return {
    title: `${slug} - My Site`,
    description: `Page for ${slug}`,
  };
}

export default async function Page({ params }: PageProps) {
  const { slug } = await params;
  return <div>{slug}</div>;
}
```

### 6.4 进阶:条件布局

根据条件渲染不同的布局:

```tsx
// app/blog/[slug]/layout.tsx
import { getPostType } from "@/lib/posts";

interface LayoutProps {
  children: React.ReactNode;
  params: Promise<{ slug: string }>;
}

export default async function PostLayout({ children, params }: LayoutProps) {
  const { slug } = await params;
  const postType = await getPostType(slug);

  // 根据文章类型使用不同布局
  if (postType === "video") {
    return (
      <div className="video-post-layout">
        <div className="video-sidebar">视频相关内容</div>
        {children}
      </div>
    );
  }

  if (postType === "tutorial") {
    return (
      <div className="tutorial-layout">
        <div className="tutorial-toc">目录</div>
        {children}
        <div className="tutorial-nav">上一篇/下一篇</div>
      </div>
    );
  }

  // 默认布局
  return <div className="default-layout">{children}</div>;
}
```

### 6.7 进阶:客户端布局状态

在布局中管理客户端状态:

```tsx
// app/dashboard/layout.tsx

import { useState } from "react";
import { SidebarContext } from "@/contexts/SidebarContext";

export default function DashboardLayout({

          {children}
        </div>
      </div>
    </SidebarContext.Provider>
  );
}

// app/dashboard/page.tsx
("use client");

import { SidebarContext } from "@/contexts/SidebarContext";

}
```

## 7. 注意事项

### 7.1 布局不会重新渲染

在同一层级的页面之间导航时,布局不会重新渲染。这意味着:

- 布局中的状态会保持
- 布局中的副作用不会重新执行
- 布局中的数据获取不会重新触发

**示例**:

```tsx
// app/blog/layout.tsx
"use client";

import { useEffect } from "react";

export default function BlogLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  useEffect(() => {
    console.log("布局挂载"); // 只在首次访问时执行
  }, []);

  return <div>{children}</div>;
}
```

从 `/blog/post-1` 导航到 `/blog/post-2` 时,`useEffect` 不会再次执行。

### 7.2 布局必须返回单个根元素

布局组件必须返回单个根元素:

```tsx
// 错误
export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <header>头部</header>
    {children}
    <footer>底部</footer>
  );
}

// 正确
export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <div>
      <header>头部</header>
      {children}
      <footer>底部</footer>
    </div>
  );
}
```

### 7.3 根布局必须包含 html 和 body 标签

根布局(app/layout.tsx)必须包含 `<html>` 和 `<body>` 标签:

```tsx
// app/layout.tsx - 正确
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="zh">
      <body>{children}</body>
    </html>
  );
}

// app/blog/layout.tsx - 错误,不能包含 html/body
export default function BlogLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html>
      {" "}
      {/* 错误! */}
      <body>{children}</body>
    </html>
  );
}
```

### 7.4 布局不能访问当前路径

布局组件不能直接访问当前路径,需要使用客户端组件:

```tsx
// 错误 - 布局中无法获取路径
export default function Layout({ children }: { children: React.ReactNode }) {
  // 无法在服务端布局中获取当前路径
  return <div>{children}</div>;
}

// 正确 - 使用客户端组件
("use client");

import { usePathname } from "next/navigation";

export default function Layout({ children }: { children: React.ReactNode }) {
  const pathname = usePathname();

  return (
    <div>
      <div>当前路径: {pathname}</div>
      {children}
    </div>
  );
}
```

### 7.5 避免在布局中使用 searchParams

布局不应该使用 `searchParams`,因为这会导致整个布局在每次查询参数变化时重新渲染:

```tsx
// 不推荐
export default async function Layout({
  children,
  searchParams,
}: {
  children: React.ReactNode;
  searchParams: Promise<{ [key: string]: string }>;
}) {
  const params = await searchParams;
  // 每次查询参数变化都会重新渲染整个布局
  return <div>{children}</div>;
}

// 推荐 - 在页面中使用
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string }>;
}) {
  const params = await searchParams;
  // 只有页面重新渲染
  return <div>搜索: {params.q}</div>;
}
```

### 7.6 布局中的数据获取会被缓存

布局中的数据获取默认会被缓存:

```tsx
// app/blog/layout.tsx
async function getCategories() {
  const res = await fetch("https://api.example.com/categories", {
    cache: "force-cache", // 默认行为
  });
  return res.json();
}

export default async function BlogLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  const categories = await getCategories();

  return (
    <div>
      <aside>{/* 分类列表 */}</aside>
      {children}
    </div>
  );
}
```

如果需要动态数据,使用 `cache: 'no-store'` 或 `revalidate`:

```tsx
async function getCategories() {
  const res = await fetch("https://api.example.com/categories", {
    next: { revalidate: 60 }, // 每60秒重新验证
  });
  return res.json();
}
```

### 7.7 Template 会在每次导航时重新挂载

使用 Template 时要注意性能影响:

```tsx
// app/blog/template.tsx
"use client";

import { useEffect } from "react";

export default function BlogTemplate({
  children,
}: {
  children: React.ReactNode;
}) {
  useEffect(() => {
    console.log("Template 挂载"); // 每次导航都会执行

    return () => {
      console.log("Template 卸载"); // 每次导航都会执行
    };
  }, []);

  return <div>{children}</div>;
}
```

### 7.8 嵌套布局的性能考虑

过多的嵌套层级可能影响性能:

```tsx
// 不推荐 - 过多嵌套
app/
├── layout.tsx          # 第1层
└── a/
    ├── layout.tsx      # 第2层
    └── b/
        ├── layout.tsx  # 第3层
        └── c/
            ├── layout.tsx  # 第4层
            └── d/
                ├── layout.tsx  # 第5层
                └── page.tsx

// 推荐 - 合理的嵌套层级(2-3层)
app/
├── layout.tsx          # 第1层
└── dashboard/
    ├── layout.tsx      # 第2层
    └── analytics/
        └── page.tsx
```

## 8. 常见问题

### 8.1 布局中的状态为什么不更新?

**问题**: 在页面之间导航时,布局中的状态没有更新。

**原因**: 布局在同一层级的页面之间导航时不会重新渲染。

**解决方案**: 使用 Template 或将状态提升到页面级别:

```tsx
// 方案1: 使用 Template
// app/blog/template.tsx
"use client";

import { useState } from "react";

export default function BlogTemplate({
  children,
}: {
  children: React.ReactNode;
}) {
  const [count, setCount] = useState(0);

  return (
    <div>
      <div>Count: {count}</div>
      {children}
    </div>
  );
}

// 方案2: 在页面中管理状态
// app/blog/[slug]/page.tsx
("use client");

import { useState } from "react";

export default function PostPage() {
  const [count, setCount] = useState(0);

  return <div>Count: {count}</div>;
}
```

### 8.2 如何在布局中获取当前路由?

**问题**: 需要在布局中根据当前路由显示不同内容。

**解决方案**: 使用客户端组件和 `usePathname`:

```tsx
// app/dashboard/layout.tsx
"use client";

import { usePathname } from "next/navigation";

export default function DashboardLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  const pathname = usePathname();

  return (
    <div>
      <nav>
        <a
          href="/dashboard"
          className={pathname === "/dashboard" ? "active" : ""}
        >
          首页
        </a>
        <a
          href="/dashboard/analytics"
          className={pathname === "/dashboard/analytics" ? "active" : ""}
        >
          分析
        </a>
      </nav>
      {children}
    </div>
  );
}
```

### 8.3 布局和页面都需要相同的数据怎么办?

**问题**: 布局和页面都需要获取相同的数据,如何避免重复请求?

**解决方案**: Next.js 会自动去重相同的 fetch 请求:

```tsx
// lib/data.ts
export async function getUser(id: string) {
  const res = await fetch(`https://api.example.com/users/${id}`);
  return res.json();
}

// app/profile/[id]/layout.tsx
import { getUser } from "@/lib/data";

export default async function ProfileLayout({
  children,
  params,
}: {
  children: React.ReactNode;
  params: Promise<{ id: string }>;
}) {
  const { id } = await params;
  const user = await getUser(id); // 第一次请求

  return (
    <div>
      <h1>{user.name}</h1>
      {children}
    </div>
  );
}

// app/profile/[id]/page.tsx
import { getUser } from "@/lib/data";

export default async function ProfilePage({
  params,
}: {
  params: Promise<{ id: string }>;
}) {
  const { id } = await params;
  const user = await getUser(id); // 自动去重,不会发起第二次请求

  return <div>{user.bio}</div>;
}
```

### 8.4 如何在嵌套路由中共享状态?

**问题**: 需要在父布局和子页面之间共享状态。

**解决方案**: 使用 React Context:

```tsx
// contexts/AppContext.tsx
"use client";

import { createContext, useState, useContext } from "react";

const AppContext = createContext<any>(null);

export function AppProvider({ children }: { children: React.ReactNode }) {
  const [user, setUser] = useState(null);

  return (
    <AppContext.Provider value={{ user, setUser }}>
      {children}
    </AppContext.Provider>
  );
}

export function useApp() {
  return useContext(AppContext);
}

// app/layout.tsx
import { AppProvider } from "@/contexts/AppContext";

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html>
      <body>
        <AppProvider>{children}</AppProvider>
      </body>
    </html>
  );
}

// app/dashboard/page.tsx
("use client");

import { useApp } from "@/contexts/AppContext";

export default function DashboardPage() {
  const { user, setUser } = useApp();

  return <div>用户: {user?.name}</div>;
}
```

### 8.5 嵌套布局的加载顺序是什么?

**问题**: 多层嵌套布局的加载和渲染顺序是怎样的?

**回答**: 布局从外到内依次加载和渲染:

```
1. RootLayout 开始渲染
2. RootLayout 中的数据获取完成
3. DashboardLayout 开始渲染
4. DashboardLayout 中的数据获取完成
5. AnalyticsLayout 开始渲染
6. AnalyticsLayout 中的数据获取完成
7. Page 开始渲染
8. Page 中的数据获取完成
9. 完整页面渲染完成
```

可以使用 Suspense 优化加载体验:

```tsx
// app/dashboard/layout.tsx
import { Suspense } from "react";

export default function DashboardLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <div>
      <Suspense fallback={<div>加载中...</div>}>
        <AsyncSidebar />
      </Suspense>
      {children}
    </div>
  );
}
```

## 9. 总结

嵌套路由是 Next.js App Router 的核心特性,通过本章学习,你应该掌握了:

**核心概念**:

1. **布局嵌套**: 通过文件系统自动创建嵌套布局
2. **状态保持**: 布局在导航时保持状态和 DOM
3. **部分渲染**: 只有变化的部分重新渲染
4. **Template**: 每次导航都重新渲染的特殊布局

**实际应用**:

- 内容管理系统的多层级结构
- 电商网站的分类和产品页面
- 文档网站的层级导航
- 仪表板应用的功能模块

**最佳实践**:

- 合理规划路由层级,避免过深嵌套
- 在布局中放置共享的 UI 和数据
- 使用 Template 处理需要重置的状态
- 利用 React Context 共享状态
- 使用 Suspense 优化加载体验

**注意事项**:

- 布局不会在同级页面导航时重新渲染
- 根布局必须包含 html 和 body 标签
- 避免在布局中使用 searchParams
- 布局中的数据获取会被缓存
- Template 会在每次导航时重新挂载

通过合理使用嵌套路由,可以构建结构清晰、性能优秀的 Next.js 应用。嵌套布局让代码复用变得简单,同时保持了良好的用户体验。
