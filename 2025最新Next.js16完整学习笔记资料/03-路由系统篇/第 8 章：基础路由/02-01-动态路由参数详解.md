# 动态路由参数详解

## 概述

动态路由是 Next.js 路由系统的核心功能之一，允许你创建可以匹配多个 URL 路径的页面。通过在文件夹名称中使用方括号，可以定义动态路由段，从而实现灵活的路由匹配。

在 Next.js 16 中，动态路由参数的处理方式有了重要变化。params 和 searchParams 现在是异步的，需要使用 await 来访问。这是为了支持部分预渲染（PPR）和更好的性能优化。

本章将详细介绍动态路由参数的使用方法、最佳实践以及在 Next.js 16 中的新特性。

---

## 基础用法

### 单个动态参数

创建一个动态路由段，使用方括号包裹参数名：

```
app/
├── blog/
│   └── [slug]/
│       └── page.tsx
```

这个路由可以匹配：

- /blog/hello-world
- /blog/nextjs-tutorial
- /blog/any-slug-here

在 Next.js 16 中访问参数：

```typescript
// app/blog/[slug]/page.tsx
interface PageProps {
  params: Promise<{ slug: string }>;
}

export default async function BlogPost({ params }: PageProps) {
  const { slug } = await params;

  return (
    <div>
      <h1>Blog Post: {slug}</h1>
    </div>
  );
}
```

### 多个动态参数

可以在路径中使用多个动态段：

```
app/
├── shop/
│   └── [category]/
│       └── [product]/
│           └── page.tsx
```

匹配路径：

- /shop/electronics/laptop
- /shop/clothing/shirt
- /shop/books/nextjs-guide

访问多个参数：

```typescript
// app/shop/[category]/[product]/page.tsx
interface PageProps {
  params: Promise<{
    category: string;
    product: string;
  }>;
}

export default async function ProductPage({ params }: PageProps) {
  const { category, product } = await params;

  return (
    <div>
      <h1>Category: {category}</h1>
      <h2>Product: {product}</h2>
    </div>
  );
}
```

## Next.js 16 新特性：异步 params

### 为什么 params 变成异步

Next.js 16 将 params 改为异步是为了：

1. 支持部分预渲染（PPR）
2. 允许在渲染过程中动态获取参数
3. 更好的流式渲染支持
4. 提升性能和用户体验

### 迁移指南

从 Next.js 15 迁移到 16：

```typescript
// Next.js 15（旧方式）
export default function Page({ params }: { params: { slug: string } }) {
  return <div>{params.slug}</div>;
}

// Next.js 16（新方式）
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>;
}) {
  const { slug } = await params;
  return <div>{slug}</div>;
}
```

### 在客户端组件中使用

客户端组件不能直接使用 async/await，需要通过 props 传递：

```typescript
// app/blog/[slug]/page.tsx（服务端组件）
import ClientComponent from "./ClientComponent";

interface PageProps {
  params: Promise<{ slug: string }>;
}

export default async function Page({ params }: PageProps) {
  const { slug } = await params;

  return <ClientComponent slug={slug} />;
}
```

```typescript
// app/blog/[slug]/ClientComponent.tsx（客户端组件）
"use client";

interface ClientComponentProps {
  slug: string;
}

export default function ClientComponent({ slug }: ClientComponentProps) {
  return <div>Client: {slug}</div>;
}
```

## 捕获所有路由段

### 基础捕获所有段

使用 `[...slug]` 捕获所有后续路径段：

```
app/
├── docs/
│   └── [...slug]/
│       └── page.tsx
```

匹配路径：

- /docs/a
- /docs/a/b
- /docs/a/b/c
- /docs/a/b/c/d

访问捕获的段：

```typescript
// app/docs/[...slug]/page.tsx
interface PageProps {
  params: Promise<{ slug: string[] }>;
}

export default async function DocsPage({ params }: PageProps) {
  const { slug } = await params;

  return (
    <div>
      <h1>Docs Path: {slug.join("/")}</h1>
      <ul>
        {slug.map((segment, index) => (
          <li key={index}>{segment}</li>
        ))}
      </ul>
    </div>
  );
}
```

### 可选捕获所有段

使用 `[[...slug]]` 使捕获段变为可选：

```
app/
├── shop/
│   └── [[...slug]]/
│       └── page.tsx
```

匹配路径：

- /shop（slug 为空数组）
- /shop/electronics
- /shop/electronics/laptop
- /shop/electronics/laptop/macbook

处理可选段：

```typescript
// app/shop/[[...slug]]/page.tsx
interface PageProps {
  params: Promise<{ slug?: string[] }>;
}

export default async function ShopPage({ params }: PageProps) {
  const { slug = [] } = await params;

  if (slug.length === 0) {
    return <div>Shop Home</div>;
  }

  return (
    <div>
      <h1>Category: {slug.join(" > ")}</h1>
    </div>
  );
}
```

## 路由匹配规则

### 匹配优先级

Next.js 按照以下优先级匹配路由：

| 优先级 | 路由类型     | 示例              | 匹配路径           |
| ------ | ------------ | ----------------- | ------------------ |
| 1      | 静态路由     | /blog/about       | /blog/about        |
| 2      | 动态路由     | /blog/[slug]      | /blog/hello        |
| 3      | 捕获所有路由 | /blog/[...slug]   | /blog/a/b/c        |
| 4      | 可选捕获所有 | /blog/[[...slug]] | /blog 或 /blog/a/b |

示例目录结构：

```
app/
├── blog/
│   ├── about/
│   │   └── page.tsx          # 优先级 1
│   ├── [slug]/
│   │   └── page.tsx          # 优先级 2
│   └── [...all]/
│       └── page.tsx          # 优先级 3
```

访问 `/blog/about` 会匹配 `about/page.tsx`，而不是 `[slug]/page.tsx`。

### 路由冲突处理

避免创建冲突的路由：

```
app/
├── posts/
│   ├── [id]/
│   │   └── page.tsx
│   └── [slug]/              # 错误：与 [id] 冲突
│       └── page.tsx
```

正确的做法是使用不同的路径层级：

```
app/
├── posts/
│   ├── by-id/
│   │   └── [id]/
│   │       └── page.tsx
│   └── by-slug/
│       └── [slug]/
│           └── page.tsx
```

## 生成静态参数

### generateStaticParams 函数

在构建时生成静态路由：

```typescript
// app/blog/[slug]/page.tsx
interface PageProps {
  params: Promise<{ slug: string }>;
}

export async function generateStaticParams() {
  const posts = await fetch("https://api.example.com/posts").then((res) =>
    res.json()
  );

  return posts.map((post: { slug: string }) => ({
    slug: post.slug,
  }));
}

export default async function BlogPost({ params }: PageProps) {
  const { slug } = await params;
  const post = await fetch(`https://api.example.com/posts/${slug}`).then(
    (res) => res.json()
  );

  return (
    <article>
      <h1>{post.title}</h1>
      <div>{post.content}</div>
    </article>
  );
}
```

### 多层动态路由

为嵌套的动态路由生成参数：

```typescript
// app/shop/[category]/[product]/page.tsx
interface PageProps {
  params: Promise<{
    category: string;
    product: string;
  }>;
}

export async function generateStaticParams() {
  const categories = await fetch("https://api.example.com/categories").then(
    (res) => res.json()
  );

  const params = [];

  for (const category of categories) {
    const products = await fetch(
      `https://api.example.com/categories/${category.id}/products`
    ).then((res) => res.json());

    for (const product of products) {
      params.push({
        category: category.slug,
        product: product.slug,
      });
    }
  }

  return params;
}

export default async function ProductPage({ params }: PageProps) {
  const { category, product } = await params;

  return (
    <div>
      <h1>
        {category} - {product}
      </h1>
    </div>
  );
}
```

### 动态参数验证

验证参数并返回 404：

```typescript
// app/blog/[slug]/page.tsx
import { notFound } from "next/navigation";

interface PageProps {
  params: Promise<{ slug: string }>;
}

const validSlugs = ["hello-world", "nextjs-tutorial", "react-guide"];

export async function generateStaticParams() {
  return validSlugs.map((slug) => ({ slug }));
}

export default async function BlogPost({ params }: PageProps) {
  const { slug } = await params;

  if (!validSlugs.includes(slug)) {
    notFound();
  }

  return <div>Post: {slug}</div>;
}
```

```typescript
// app/blog/[slug]/page.tsx
import { z } from "zod";
import { notFound } from "next/navigation";

const paramsSchema = z.object({
  slug: z
    .string()
    .min(1)
    .max(100)
    .regex(/^[a-z0-9-]+$/),
});

interface PageProps {
  params: Promise<{ slug: string }>;
}

export default async function BlogPost({ params }: PageProps) {
  const rawParams = await params;
  const result = paramsSchema.safeParse(rawParams);

  if (!result.success) {
    notFound();
  }

  const { slug } = result.data;

  return <div>Valid slug: {slug}</div>;
}
```

```typescript
import { Metadata } from "next";

interface Post {
  slug: string;
  title: string;
  content: string;
  author: string;
  publishedAt: string;
}

interface PageProps {
  params: Promise<{ slug: string }>;
}

async function getPost(slug: string): Promise<Post | null> {
  try {
    const res = await fetch(`https://api.example.com/posts/${slug}`, {
      next: { revalidate: 3600 },
    });

    if (!res.ok) return null;

    return res.json();
  } catch {
    return null;
  }
}

export async function generateMetadata({
  params,
}: PageProps): Promise<Metadata> {
  const { slug } = await params;
  const post = await getPost(slug);

  if (!post) {
    return {
      title: "Post Not Found",
    };
  }

  return {
    title: post.title,
    description: post.content.substring(0, 160),
  };
}

export async function generateStaticParams() {
  const posts = await fetch("https://api.example.com/posts").then((res) =>
    res.json()
  );

  return posts.map((post: Post) => ({
    slug: post.slug,
  }));
}

export default async function BlogPost({ params }: PageProps) {
  const { slug } = await params;
  const post = await getPost(slug);

  if (!post) {
    notFound();
  }

  return (
    <article>
      <h1>{post.title}</h1>
      <p>
        By {post.author} on {new Date(post.publishedAt).toLocaleDateString()}
      </p>
      <div dangerouslySetInnerHTML={{ __html: post.content }} />
    </article>
  );
}
```

## 常见问题

### 1. params 未定义

问题描述：访问 params 时得到 undefined。

原因分析：

- 忘记使用 await
- 不在动态路由段中
- 文件夹命名错误

解决方案：

```typescript
// 错误
export default async function Page({ params }: PageProps) {
  console.log(params.slug); // 错误：params 是 Promise
}

// 正确
export default async function Page({ params }: PageProps) {
  const { slug } = await params;
  console.log(slug);
}
```

### 2. 类型错误

问题描述：TypeScript 报告类型错误。

解决方案：

```typescript
  params: Promise<{ slug: string }>;
  searchParams: Promise<{ q?: string }>;
```

### 9. 参数验证失败后的处理

问题描述：参数验证失败后如何优雅处理。

解决方案：

```typescript
import { notFound } from "next/navigation";
import { z } from "zod";

const paramsSchema = z.object({
  id: z.string().regex(/^\d+$/),
});

export default async function Page({ params }: PageProps) {
  const resolvedParams = await params;

  // 验证参数
  const result = paramsSchema.safeParse(resolvedParams);

  if (!result.success) {
    notFound(); // 返回 404 页面
  }

  const { id } = result.data;
  return <div>ID: {id}</div>;
}
```

### 10. 嵌套动态路由的参数传递

问题描述：如何在嵌套布局中访问动态参数。

解决方案：

```typescript
// app/blog/[category]/layout.tsx
interface LayoutProps {
  children: React.ReactNode;
  params: Promise<{ category: string }>;
}

export default async function CategoryLayout({
  children,
  params,
}: LayoutProps) {
  const { category } = await params;

  return (
    <div>
      <h1>分类: {category}</h1>
      {children}
    </div>
  );
}

// app/blog/[category]/[slug]/page.tsx
interface PageProps {
  params: Promise<{ category: string; slug: string }>;
}

export default async function PostPage({ params }: PageProps) {
  const { category, slug } = await params;

  return (
    <article>
      <p>分类: {category}</p>
      <p>文章: {slug}</p>
    </article>
  );
}
```

## 注意事项

### 1. 异步参数

Next.js 16 中 params 和 searchParams 都是异步的，必须使用 await。

### 2. 路由优先级

静态路由优先于动态路由，动态路由优先于捕获所有路由。

### 3. 参数验证

始终验证动态参数，防止无效输入。

### 4. SEO 考虑

使用 generateMetadata 为动态页面生成元数据。

### 5. 性能优化

使用 generateStaticParams 在构建时生成静态页面。
