# 路由加载状态管理

## 1. 概述

路由加载状态管理是 Next.js 16 中用于优化用户体验的重要机制。通过 `loading.tsx` 文件和 React Suspense,你可以在页面或组件加载时显示加载指示器,避免用户看到空白页面,提供流畅的导航体验。

### 1.1 概念定义

路由加载状态管理指的是在路由切换或数据加载过程中,向用户展示加载状态的机制。Next.js 提供了多种方式来实现这一功能:

**关键特征**:

- **自动触发**: 路由切换时自动显示加载状态
- **即时反馈**: 用户点击链接后立即看到加载指示器
- **流式渲染**: 支持部分内容先显示,其余内容逐步加载
- **嵌套支持**: 不同层级的路由可以有独立的加载状态

**实现方式**:

1. **loading.tsx 文件**: 在路由段中创建 `loading.tsx`,自动包裹页面
2. **React Suspense**: 手动控制加载边界
3. **useRouter 状态**: 在客户端组件中监听路由状态
4. **Streaming SSR**: 服务端流式渲染

### 1.2 核心价值

**提升用户体验**:

加载状态让用户知道应用正在响应他们的操作,而不是卡住或无响应。这对于数据加载较慢的页面尤其重要。

**减少感知延迟**:

即使实际加载时间没有改变,显示加载指示器也能让用户感觉等待时间更短。研究表明,有反馈的等待比无反馈的等待感觉更短。

**支持渐进式加载**:

通过 Suspense 边界,可以让页面的关键内容先显示,次要内容后加载,用户可以更快地开始交互。

**优化 SEO**:

流式渲染允许搜索引擎爬虫更快地获取页面的初始内容,而不需要等待所有数据加载完成。

### 1.3 加载状态的类型

Next.js 16 支持多种类型的加载状态:

| 类型       | 实现方式         | 触发时机     | 适用场景       |
| :--------- | :--------------- | :----------- | :------------- |
| 路由级加载 | `loading.tsx`    | 路由切换时   | 整个页面加载   |
| 组件级加载 | `<Suspense>`     | 组件渲染时   | 部分内容加载   |
| 客户端加载 | `useRouter` 状态 | 客户端导航时 | 自定义加载逻辑 |
| 流式加载   | Streaming SSR    | 服务端渲染时 | 大型页面优化   |

## 2. 核心概念与原理

### 2.1 loading.tsx 文件

`loading.tsx` 是 Next.js App Router 中的特殊文件,用于定义路由段的加载状态。

**文件位置**:

```
app/
├── loading.tsx          # 根加载状态
├── page.tsx
├── dashboard/
│   ├── loading.tsx      # dashboard 加载状态
│   └── page.tsx
└── blog/
    ├── [slug]/
    │   ├── loading.tsx  # 博客文章加载状态
    │   └── page.tsx
    └── loading.tsx      # 博客列表加载状态
```

**工作原理**:

当用户导航到某个路由时,Next.js 会:

1. 立即显示 `loading.tsx` 的内容
2. 在后台加载 `page.tsx` 和数据
3. 加载完成后,用页面内容替换加载状态

**自动 Suspense 包裹**:

`loading.tsx` 实际上是 React Suspense 的语法糖:

```tsx
// app/dashboard/loading.tsx
export default function Loading() {
  return <div>Loading dashboard...</div>;
}

// 等价于:
// app/dashboard/layout.tsx
import { Suspense } from "react";

export default function Layout({ children }) {
  return (
    <Suspense fallback={<div>Loading dashboard...</div>}>{children}</Suspense>
  );
}
```

### 2.2 React Suspense

Suspense 是 React 提供的用于处理异步操作的组件,Next.js 在此基础上增强了功能。

**基本用法**:

```tsx
import { Suspense } from "react";

export default function Page() {
  return (
    <div>
      <h1>我的页面</h1>
      <Suspense fallback={<div>加载中...</div>}>
        <AsyncComponent />
      </Suspense>
    </div>
  );
}

async function AsyncComponent() {
  const data = await fetchData();
  return <div>{data}</div>;
}
```

**嵌套 Suspense**:

可以在页面的不同部分使用多个 Suspense 边界:

```tsx
export default function Page() {
  return (
    <div>
      <Suspense fallback={<HeaderSkeleton />}>
        <Header />
      </Suspense>

      <Suspense fallback={<ContentSkeleton />}>
        <MainContent />
      </Suspense>

      <Suspense fallback={<SidebarSkeleton />}>
        <Sidebar />
      </Suspense>
    </div>
  );
}
```

### 2.3 流式渲染(Streaming SSR)

流式渲染允许服务器逐步发送 HTML,而不是等待所有内容准备好。

**工作流程**:

1. 服务器开始渲染页面
2. 遇到 Suspense 边界时,先发送 fallback 内容
3. 继续渲染其他部分
4. 异步内容准备好后,通过流发送更新
5. 客户端接收更新并替换 fallback

**优势**:

- **更快的首字节时间(TTFB)**: 不需要等待所有数据
- **更好的用户体验**: 用户更快看到内容
- **SEO 友好**: 搜索引擎可以更快索引页面

**示例**:

```tsx
// app/page.tsx
import { Suspense } from "react";

export default function Page() {
  return (
    <div>
      {/* 这部分立即渲染 */}
      <h1>欢迎</h1>

      {/* 这部分流式加载 */}
      <Suspense fallback={<div>加载推荐内容...</div>}>
        <Recommendations />
      </Suspense>
    </div>
  );
}

async function Recommendations() {
  // 这个数据获取不会阻塞页面的初始渲染
  const recommendations = await fetchRecommendations();
  return <div>{/* 渲染推荐内容 */}</div>;
}
```

### 2.4 加载状态的层级

Next.js 的加载状态是分层的,每个路由段可以有自己的加载状态:

```
app/
├── loading.tsx              # Level 1: 根加载
├── dashboard/
│   ├── loading.tsx          # Level 2: dashboard 加载
│   └── analytics/
│       ├── loading.tsx      # Level 3: analytics 加载
│       └── page.tsx
```

**层级规则**:

- 子路由的加载状态会覆盖父路由的加载状态
- 如果子路由没有 `loading.tsx`,会使用最近的父级加载状态
- 每个层级的加载状态只影响该层级及其子级

**示例**:

访问 `/dashboard/analytics` 时:

1. 如果 `analytics/loading.tsx` 存在,显示它
2. 否则,如果 `dashboard/loading.tsx` 存在,显示它
3. 否则,如果根 `loading.tsx` 存在,显示它
4. 否则,不显示加载状态

## 4. API 签名与配置

### 4.1 loading.tsx 文件

**文件签名**:

```tsx
// app/[segment]/loading.tsx
export default function Loading() {
  return React.ReactNode;
}
```

**特点**:

- 必须是默认导出
- 返回 React 组件
- 不接收任何 props
- 可以是客户端组件或服务端组件

**示例**:

```tsx
// 简单的加载指示器
export default function Loading() {
  return <div>Loading...</div>;
}

// 使用 Tailwind CSS 的骨架屏
export default function Loading() {
  return (
    <div className="animate-pulse">
      <div className="h-8 bg-gray-200 rounded mb-4"></div>
      <div className="h-64 bg-gray-200 rounded"></div>
    </div>
  );
}

// 使用自定义加载组件
import Spinner from "@/components/Spinner";

export default function Loading() {
  return (
    <div className="flex justify-center items-center h-screen">
      <Spinner />
    </div>
  );
}
```

### 4.2 Suspense 组件

**API 签名**:

```tsx
import { Suspense } from "react";

<Suspense fallback={React.ReactNode}>{children}</Suspense>;
```

**Props**:

| 属性       | 类型              | 必需 | 说明             |
| :--------- | :---------------- | :--- | :--------------- |
| `fallback` | `React.ReactNode` | 是   | 加载时显示的内容 |
| `children` | `React.ReactNode` | 是   | 异步内容         |

**示例**:

```tsx
import { Suspense } from "react";

export default function Page() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <AsyncComponent />
    </Suspense>
  );
}
```

### 4.3 useRouter 状态(客户端)

在客户端组件中,可以使用 `useRouter` 监听路由状态:

```tsx
"use client";

import { useRouter } from "next/navigation";
import { useEffect, useState } from "react";

export default function ClientComponent() {
  const router = useRouter();
  const [isNavigating, setIsNavigating] = useState(false);

  function handleNavigate() {
    setIsNavigating(true);
    router.push("/dashboard");
  }

  return (
    <button onClick={handleNavigate} disabled={isNavigating}>
      {isNavigating ? "跳转中..." : "前往仪表板"}
    </button>
  );
}
```

**注意**: Next.js 16 的 App Router 不提供全局路由事件,需要手动管理状态。

"use client";

import { useState, useEffect } from "react";

export function ProgressLoader({ duration = 3000 }: { duration?: number }) {
  const [progress, setProgress] = useState(0);

  useEffect(() => {
    const interval = setInterval(() => {
      setProgress((prev) => {
        if (prev >= 90) return prev;
        return prev + 10;
      });

    return () => clearInterval(interval);

      </div>
      <div className="w-full bg-gray-200 rounded-full h-2">
        <div
          className="bg-blue-600 h-2 rounded-full transition-all duration-300"
          style={{ width: `${progress}%` }}
        ></div>
      </div>
    </div>
  );
}

## 6. 注意事项

### 6.1 loading.tsx 的作用域

`loading.tsx` 只影响同级和子级的 `page.tsx`,不影响 `layout.tsx`:

```
app/
├── layout.tsx           # 不受 loading.tsx 影响
├── loading.tsx          # 只包裹 page.tsx
└── page.tsx             # 被 loading.tsx 包裹
```

**错误示例**:

```tsx
// ❌ 这不会显示加载状态,因为 layout 不受 loading.tsx 影响
// app/layout.tsx
export default async function RootLayout({ children }) {
  const data = await fetchData(); // 这会阻塞整个应用
  return (
    <html>
      <body>{children}</body>
    </html>
  );
}
```

**正确做法**:

```tsx
// ✅ 在 layout 中使用 Suspense
import { Suspense } from "react";

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        <Suspense fallback={<div>Loading...</div>}>
          <AsyncHeader />
        </Suspense>
        {children}
      </body>
    </html>
  );
}
```

### 6.2 避免过度使用加载状态

不是所有内容都需要加载状态:

```tsx
// ❌ 过度使用
export default function Page() {
  return (
    <div>
      <Suspense fallback={<div>Loading...</div>}>
        <StaticContent /> {/* 静态内容不需要 Suspense */}
      </Suspense>
    </div>
  );
}

// ✅ 只对异步内容使用
export default function Page() {
  return (
    <div>
      <StaticContent /> {/* 静态内容直接渲染 */}
      <Suspense fallback={<div>Loading...</div>}>
        <AsyncContent /> {/* 只对异步内容使用 Suspense */}
      </Suspense>
    </div>
  );
}
```

### 6.3 加载状态的性能影响

过多的 Suspense 边界可能影响性能:

```tsx
// ❌ 太多 Suspense 边界
export default function Page() {
  return (
    <div>
      {items.map((item) => (
        <Suspense key={item.id} fallback={<div>Loading...</div>}>
          <Item id={item.id} />
        </Suspense>
      ))}
    </div>
  );
}

// ✅ 合理分组
export default function Page() {
  return (
    <Suspense fallback={<div>Loading items...</div>}>
      <ItemList items={items} />
    </Suspense>
  );
}
```

### 6.4 客户端组件的限制

客户端组件不能直接使用 async/await:

```tsx
// ❌ 错误:客户端组件不能是 async
"use client";

export default async function ClientComponent() {
  const data = await fetchData();
  return <div>{data}</div>;
}

// ✅ 正确:使用 useEffect 或 SWR
("use client");

import { useState, useEffect } from "react";

export default function ClientComponent() {
  const [data, setData] = useState(null);
  const [isLoading, setIsLoading] = useState(true);

  useEffect(() => {
    fetchData().then((result) => {
      setData(result);
      setIsLoading(false);
    });
  }, []);

  if (isLoading) return <div>Loading...</div>;
  return <div>{data}</div>;
}
```

### 6.5 SEO 考虑

加载状态不应该影响 SEO:

```tsx
// ✅ 关键内容不使用 Suspense
export default function BlogPost({ params }) {
  return (
    <article>
      {/* SEO 关键内容立即渲染 */}
      <h1>{post.title}</h1>
      <meta name="description" content={post.description} />

      {/* 次要内容可以延迟加载 */}
      <Suspense fallback={<div>Loading comments...</div>}>
        <Comments postId={post.id} />
      </Suspense>
    </article>
  );
}
```

## 7. 常见问题

### 7.1 loading.tsx 不显示怎么办?

**可能原因**:

1. 页面加载太快,加载状态一闪而过
2. 使用了客户端导航,loading.tsx 只在服务端导航时显示
3. 页面是静态生成的,没有加载过程

**解决方案**:

```tsx
// 添加最小显示时间
export default function Loading() {
  return <div>Loading...</div>;
}

// 或者在开发时添加延迟测试
// app/page.tsx
export default async function Page() {
  if (process.env.NODE_ENV === "development") {
    await new Promise((resolve) => setTimeout(resolve, 2000));
  }

  const data = await fetchData();
  return <div>{data}</div>;
}
```

### 7.2 如何在客户端导航时显示加载状态?

使用 `useRouter` 和状态管理:

```tsx
"use client";

import { useRouter } from "next/navigation";
import { useState, useTransition } from "react";

export function NavigationButton() {
  const router = useRouter();
  const [isPending, startTransition] = useTransition();

  function handleClick() {
    startTransition(() => {
      router.push("/dashboard");
    });
  }

  return (
    <button onClick={handleClick} disabled={isPending}>
      {isPending ? "跳转中..." : "前往仪表板"}
    </button>
  );
}
```

### 7.3 如何为不同的路由使用不同的加载样式?

在每个路由段创建独立的 `loading.tsx`:

```tsx
// app/products/loading.tsx
export default function ProductsLoading() {
  return <ProductGridSkeleton />;
}

// app/blog/loading.tsx
export default function BlogLoading() {
  return <BlogListSkeleton />;
}

// app/dashboard/loading.tsx
export default function DashboardLoading() {
  return <DashboardSkeleton />;
}
```

### 7.4 如何测试加载状态?

**方法 1: 添加人工延迟**:

```tsx
export default async function Page() {
  // 仅在开发环境添加延迟
  if (process.env.NODE_ENV === "development") {
    await new Promise((resolve) => setTimeout(resolve, 3000));
  }

  const data = await fetchData();
  return <div>{data}</div>;
}
```

**方法 2: 使用网络节流**:

在浏览器开发者工具中启用网络节流(Slow 3G)。

**方法 3: 使用 React DevTools**:

React DevTools 可以手动触发 Suspense 状态。

### 7.5 加载状态会影响 SEO 吗?

**不会**,如果使用得当:

- 服务端渲染的内容会包含在初始 HTML 中
- Suspense 边界内的内容会在流式渲染中发送
- 搜索引擎可以等待流式渲染完成

**注意**: 不要将 SEO 关键内容放在 Suspense 边界内。

## 8. 总结

路由加载状态管理是提升 Next.js 应用用户体验的关键功能。通过合理使用 `loading.tsx` 和 Suspense,可以创建流畅、响应迅速的应用。

**核心要点**:

1. 使用 `loading.tsx` 为路由段添加加载状态
2. 使用 Suspense 实现细粒度的加载控制
3. 使用骨架屏提供更好的视觉反馈
4. 合理分组 Suspense 边界,避免过度使用
5. 注意 SEO 和性能影响

**最佳实践**:

- 为数据密集型页面添加加载状态
- 使用骨架屏而不是简单的加载指示器
- 优先加载关键内容,延迟加载次要内容
- 测试加载状态的显示效果
- 避免将 SEO 关键内容放在 Suspense 边界内

掌握加载状态管理,可以显著提升应用的用户体验和感知性能。
