# 路由预加载策略

## 1. 概述

路由预加载(Route Prefetching)是 Next.js 16 的核心性能优化特性,通过提前加载用户可能访问的页面资源,实现近乎瞬时的页面导航。Next.js 提供了多种预加载策略,开发者可以根据实际需求选择合适的方案。

### 1.1 概念定义

路由预加载指的是在用户实际访问页面之前,提前加载页面所需的 JavaScript、CSS 和数据,当用户点击链接时,页面可以立即显示。

**关键特征**:

- **自动预加载**: `<Link>` 组件默认启用预加载
- **智能策略**: 根据可见性和网络状况调整
- **手动控制**: 可以禁用或手动触发预加载
- **性能优化**: 显著提升导航速度

**工作流程**:

```
链接进入视口
  ↓
检查预加载条件
  ↓
加载页面资源
  ↓
缓存到客户端
  ↓
用户点击链接
  ↓
瞬间显示页面
```

### 1.2 核心价值

**极速导航**:

预加载后的页面导航几乎是瞬时的,用户体验接近原生应用。

**智能优化**:

Next.js 会根据网络状况、设备性能等因素智能调整预加载策略,在性能和资源消耗之间找到平衡。

**开发者友好**:

默认配置已经很好,大多数情况下无需额外配置,但也提供了灵活的控制选项。

**SEO 友好**:

预加载不影响 SEO,搜索引擎爬虫仍然能正常抓取页面内容。

### 1.3 预加载策略对比

| 策略       | 触发时机   | 预加载内容      | 适用场景         |
| :--------- | :--------- | :-------------- | :--------------- |
| 自动预加载 | 链接可见时 | 页面组件 + 数据 | 大多数场景(默认) |
| 手动预加载 | 开发者控制 | 指定页面        | 特殊优化需求     |
| 禁用预加载 | 不预加载   | 无              | 极端带宽限制     |
| 悬停预加载 | 鼠标悬停时 | 页面组件 + 数据 | 桌面端优化       |

## 2. 核心概念与原理

### 2.1 Link 组件的默认预加载

`<Link>` 组件默认启用预加载:

```tsx
import Link from "next/link";

export default function Page() {
  return (
    <div>
      {/* 默认启用预加载 */}
      <Link href="/about">关于我们</Link>

      {/* 禁用预加载 */}
      <Link href="/contact" prefetch={false}>
        联系我们
      </Link>
    </div>
  );
}
```

**预加载行为**:

- 生产环境:链接进入视口时自动预加载
- 开发环境:不预加载(避免影响开发体验)

### 2.2 prefetch 属性

`<Link>` 组件的 `prefetch` 属性控制预加载行为:

```tsx
import Link from "next/link";

export default function Page() {
  return (
    <div>
      {/* 默认:自动预加载 */}
      <Link href="/page1">页面 1</Link>

      {/* 显式启用预加载 */}
      <Link href="/page2" prefetch={true}>
        页面 2
      </Link>

      {/* 禁用预加载 */}
      <Link href="/page3" prefetch={false}>
        页面 3
      </Link>

      {/* null:使用默认行为 */}
      <Link href="/page4" prefetch={null}>
        页面 4
      </Link>
    </div>
  );
}
```

**prefetch 值说明**:

| 值              | 行为       | 使用场景                |
| :-------------- | :--------- | :---------------------- |
| `true`          | 强制预加载 | 重要页面,确保快速加载   |
| `false`         | 禁用预加载 | 不常访问的页面,节省带宽 |
| `null` 或未设置 | 默认行为   | 大多数场景              |

### 2.3 手动预加载

使用 `router.prefetch()` 手动触发预加载:

```tsx
"use client";

import { useRouter } from "next/navigation";
import { useEffect } from "react";

export default function Page() {
  const router = useRouter();

  useEffect(() => {
    // 预加载重要页面
    router.prefetch("/dashboard");
    router.prefetch("/settings");
    router.prefetch("/profile");
  }, [router]);

  return <div>首页</div>;
}
```

**手动预加载的高级用法**:

```tsx
"use client";

import { useRouter } from "next/navigation";
import { useEffect, useState } from "react";

export default function SmartPrefetch() {
  const router = useRouter();
  const [userRole, setUserRole] = useState<string>("");

  useEffect(() => {
    // 根据用户角色预加载不同页面
    const fetchUserRole = async () => {
      const role = await getUserRole();
      setUserRole(role);

      if (role === "admin") {
        router.prefetch("/admin/dashboard");
        router.prefetch("/admin/users");
        router.prefetch("/admin/settings");
      } else if (role === "user") {
        router.prefetch("/user/profile");
        router.prefetch("/user/orders");
      }
    };

    fetchUserRole();
  }, [router]);

  return <div>加载中...</div>;
}
```

**条件预加载**:

```tsx
"use client";

import { useRouter } from "next/navigation";
import { useEffect } from "react";

export default function ConditionalPrefetch() {
  const router = useRouter();

  useEffect(() => {
    // 检查网络状况
    if ("connection" in navigator) {
      const connection = (navigator as any).connection;

      // 只在4G或WiFi下预加载
      if (connection.effectiveType === "4g" || connection.type === "wifi") {
        router.prefetch("/heavy-page");
      }
    }

    // 检查设备内存
    if ("deviceMemory" in navigator) {
      const memory = (navigator as any).deviceMemory;

      // 只在内存充足时预加载
      if (memory >= 4) {
        router.prefetch("/memory-intensive-page");
      }
    }
  }, [router]);

  return <div>智能预加载</div>;
}
```

**使用场景**:

- 预加载用户可能访问的页面
- 在特定条件下触发预加载
- 优化关键用户路径

### 2.4 预加载的内容

Next.js 预加载以下内容:

**1. 页面组件代码**:

```tsx
// app/about/page.tsx 的 JavaScript bundle
export default function AboutPage() {
  return <div>关于页面</div>;
}
```

**2. 共享布局代码**:

```tsx
// app/about/layout.tsx (如果存在)
export default function AboutLayout({ children }) {
  return <div>{children}</div>;
}
```

**3. RSC Payload**:

服务端组件的数据(React Server Component Payload)。

**4. 静态资源**:

CSS 文件、字体等静态资源。

### 2.5 预加载缓存

Next.js 会缓存预加载的资源:

**缓存时长**:

- 静态页面:缓存 5 分钟
- 动态页面:缓存 30 秒

**缓存失效**:

- 超过缓存时长
- 页面重新部署
- 手动刷新(`router.refresh()`)

```tsx
"use client";

import { useRouter } from "next/navigation";

export default function Page() {
  const router = useRouter();

  function handleRefresh() {
    // 清除缓存并刷新
    router.refresh();
  }

  return <button onClick={handleRefresh}>刷新</button>;
}
```

## 3. 适用场景

### 3.1 导航菜单优化

**场景**: 预加载导航菜单中的所有页面。

```tsx
// components/Navigation.tsx
"use client";

import Link from "next/link";
import { useRouter } from "next/navigation";
import { useEffect } from "react";

const menuItems = [
  { href: "/", label: "首页" },
  { href: "/about", label: "关于" },
  { href: "/services", label: "服务" },
  { href: "/contact", label: "联系" },
];

export default function Navigation() {
  const router = useRouter();

  useEffect(() => {
    // 预加载所有菜单页面
    menuItems.forEach((item) => {
      router.prefetch(item.href);
    });
  }, [router]);

  return (
    <nav>
      {menuItems.map((item) => (
        <Link key={item.href} href={item.href}>
          {item.label}
        </Link>
      ))}
    </nav>
  );
}
```

### 3.4 悬停预加载

**场景**: 鼠标悬停时预加载,节省带宽。

```tsx
"use client";

import Link from "next/link";
import { useRouter } from "next/navigation";
import { useState } from "react";

export default function HoverPrefetch({ href, children }) {
  const router = useRouter();
  const [prefetched, setPrefetched] = useState(false);

  const handleMouseEnter = () => {
    if (!prefetched) {
      router.prefetch(href);
      setPrefetched(true);
    }
  };

  return (
    <Link href={href} prefetch={false} onMouseEnter={handleMouseEnter}>
      {children}
    </Link>
  );
}
```

### 3.5 滚动预加载

**场景**: 用户滚动到底部时预加载下一页。

```tsx
"use client";

import { useRouter } from "next/navigation";
import { useEffect, useState } from "react";

export default function ScrollPrefetch({ nextPage }) {
  const router = useRouter();
  const [hasPreloaded, setHasPreloaded] = useState(false);

  useEffect(() => {
    const handleScroll = () => {
      const scrollPercentage =
        (window.scrollY + window.innerHeight) / document.body.scrollHeight;

      if (scrollPercentage > 0.8 && !hasPreloaded) {
        router.prefetch(nextPage);
        setHasPreloaded(true);
      }
    };

    window.addEventListener("scroll", handleScroll);
    return () => window.removeEventListener("scroll", handleScroll);
  }, [router, nextPage, hasPreloaded]);

  return null;
}
```

### 3.6 网络感知预加载

**场景**: 根据网络状况调整预加载策略。

```tsx
"use client";

import { useRouter } from "next/navigation";
import { useEffect } from "react";

export default function NetworkAwarePrefetch() {
  const router = useRouter();

  useEffect(() => {
    if ("connection" in navigator) {
      const connection = (navigator as any).connection;

      // 4G 网络:预加载所有关键页面
      if (connection.effectiveType === "4g") {
        router.prefetch("/dashboard");
        router.prefetch("/profile");
        router.prefetch("/settings");
      }
      // 3G 网络:只预加载最重要的页面
      else if (connection.effectiveType === "3g") {
        router.prefetch("/dashboard");
      }
      // 慢速网络:不预加载
    }
  }, [router]);

  return null;
}
```

## 4. 常见问题

### 1. 预加载会影响首屏加载吗?

不会。Next.js 的预加载是低优先级的,不会阻塞首屏渲染。

```tsx
// 预加载在浏览器空闲时执行
if ("requestIdleCallback" in window) {
  requestIdleCallback(() => {
    router.prefetch("/page");
  });
}
```

### 2. 如何禁用所有预加载?

可以在 `next.config.js` 中配置:

```js
// next.config.js
module.exports = {
  experimental: {
    optimisticClientCache: false,
  },
};
```

或者在每个 Link 上设置 `prefetch={false}`:

```tsx
<Link href="/page" prefetch={false}>
  页面
</Link>
```

### 3. 预加载的数据会过期吗?

会。Next.js 有缓存时间限制:

| 页面类型 | 缓存时间 |
| -------- | -------- |
| 静态页面 | 5 分钟   |
| 动态页面 | 30 秒    |

超过时间后会重新获取数据。

### 4. 如何监控预加载效果?

使用 Performance API:

```tsx
"use client";

import { useRouter } from "next/navigation";
import { useEffect } from "react";

export default function MonitorPrefetch() {
  const router = useRouter();

  useEffect(() => {
    const startTime = performance.now();

    router.prefetch("/dashboard").then(() => {
      const endTime = performance.now();
      console.log(`预加载耗时: ${endTime - startTime}ms`);
    });
  }, [router]);

  return null;
}
```

### 5. 预加载会消耗多少流量?

取决于页面大小。一般来说:

| 页面类型 | 大小     |
| -------- | -------- |
| 简单页面 | 10-50KB  |
| 中等页面 | 50-200KB |
| 复杂页面 | 200KB+   |

建议只预加载关键页面,避免浪费流量。

### 5.4 避免重复预加载

```tsx
"use client";

import { useRouter } from "next/navigation";
import { useEffect, useRef } from "react";

export default function NoDuplicatePrefetch() {
  const router = useRouter();
  const prefetched = useRef(new Set<string>());

  const prefetchPage = (path: string) => {
    if (!prefetched.current.has(path)) {
      router.prefetch(path);
      prefetched.current.add(path);
    }

  useEffect(() => {
    prefetchPage("/dashboard");
    prefetchPage("/profile");
  }, []);

  return null;
}
```

## 6. 注意事项

### 6.1 开发环境不预加载

在开发环境中,Next.js 不会自动预加载,避免影响开发体验。

### 6.2 预加载不保证成功

网络问题可能导致预加载失败,应该有降级方案。

### 6.3 预加载会消耗资源

过度预加载会消耗带宽和内存,需要权衡。

### 6.4 缓存时间有限

预加载的内容有缓存时间限制,过期后会重新获取。

### 6.5 服务端组件的预加载

服务端组件的预加载包括 RSC Payload,可能比客户端组件大。

### 6.6 动态路由的预加载

动态路由需要明确指定参数:

```tsx
router.prefetch("/products/123"); // ✅ 正确
router.prefetch("/products/[id]"); // ❌ 错误
```

### 6.7 预加载与 SEO

预加载不影响 SEO,搜索引擎爬虫不会触发预加载。

### 5. 预加载会消耗多少流量?

**问题**: 担心预加载消耗用户流量

| 中型页面 | 200KB      | 500ms    |
| 大型页面 | 500KB      | 1000ms   |

```tsx
import { useRouter } from "next/navigation";

export default function DataSaverPrefetch() {
  const router = useRouter();

  useEffect(() => {
    if ("connection" in navigator) {
      const connection = (navigator as any).connection;
      if (connection.saveData) return;
    }
    router.prefetch("/dashboard");
  }, [router]);

  return null;
}
```

### 7. 预加载失败怎么办?

**问题**: 预加载可能因网络问题失败

**解决方案**: 实现重试机制

```tsx
"use client";
import { useRouter } from "next/navigation";
import { useEffect } from "react";

export default function RobustPrefetch() {
  const router = useRouter();

  useEffect(() => {
    const prefetchWithRetry = async (path: string, retries = 3) => {
      for (let i = 0; i < retries; i++) {
        try {
          await router.prefetch(path);
          break;
        } catch (error) {
          if (i === retries - 1) {
            console.error(`预加载失败: ${path}`, error);
          }
          await new Promise((resolve) => setTimeout(resolve, 1000 * (i + 1)));
        }
      }
    };

    prefetchWithRetry("/dashboard");
  }, [router]);

  return null;
}
```

### 10. 如何根据用户行为预加载?

**问题**: 想要根据用户行为智能预加载

**解决方案**: 追踪用户行为模式

```tsx
"use client";
import { useRouter } from "next/navigation";
import { useEffect, useState } from "react";

export default function BehaviorBasedPrefetch() {
  const router = useRouter();
  const [visitHistory, setVisitHistory] = useState<string[]>([]);

  useEffect(() => {
    const history = JSON.parse(localStorage.getItem("visitHistory") || "[]");
    setVisitHistory(history);

    const frequentPages = getFrequentPages(history);
    frequentPages.forEach((page) => router.prefetch(page));
  }, [router]);

  const getFrequentPages = (history: string[]) => {
    const counts: Record<string, number> = {};
    history.forEach((page) => {
      counts[page] = (counts[page] || 0) + 1;
    });

    return Object.entries(counts)
      .sort(([, a], [, b]) => b - a)
      .slice(0, 3)
      .map(([page]) => page);
  };

  return null;
}
```

### 13. 如何处理预加载缓存?

**问题**: 预加载的内容如何缓存和失效

**解决方案**: 缓存策略

```tsx
"use client";
import { useRouter } from "next/navigation";
import { useEffect } from "react";

export default function CachePrefetch() {
  const router = useRouter();

  useEffect(() => {
    const prefetchWithCache = (path: string, ttl: number = 60000) => {
      const cacheKey = `prefetch-${path}`;
      const cached = sessionStorage.getItem(cacheKey);

      if (cached) {
        const { timestamp } = JSON.parse(cached);
        if (Date.now() - timestamp < ttl) {
          return;
        }
      }

      router.prefetch(path);
      sessionStorage.setItem(
        cacheKey,
        JSON.stringify({ timestamp: Date.now() })
      );
    };

    prefetchWithCache("/dashboard", 60000);
  }, [router]);

  return null;
}
```

## 8. 最佳实践

### 1. 预加载优先级策略

**高优先级页面**(立即预加载):

- 用户登录后的首页
- 导航菜单中的主要页面
- 业务流程的下一步

**中优先级页面**(延迟 1-2 秒预加载):

- 用户资料页面
- 设置页面
- 常用功能页面

**低优先级页面**(延迟 3-5 秒或不预加载):

- 帮助文档
- 关于页面
- 很少访问的页面

**实现示例**:

```tsx
"use client";
import { useRouter } from "next/navigation";
import { useEffect } from "react";

export default function PriorityPrefetch() {
  const router = useRouter();

  useEffect(() => {
    // 高优先级:立即预加载
    router.prefetch("/dashboard");

    // 中优先级:延迟1秒
    setTimeout(() => {
      router.prefetch("/profile");
    }, 1000);

    // 低优先级:延迟3秒
    setTimeout(() => {
      router.prefetch("/settings");
    }, 3000);
  }, [router]);

  return null;
}
```

### 4. 智能预加载管理器

创建统一的预加载管理器:

```tsx
"use client";
import { useRouter } from "next/navigation";
import { useEffect, useRef } from "react";

class PrefetchManager {
  private prefetched = new Set<string>();
  private queue: Array<{ path: string; priority: number }> = [];
  private processing = false;

  constructor(private router: any) {}

  add(path: string, priority: number = 5) {
    if (this.prefetched.has(path)) return;

    this.queue.push({ path, priority });
    this.queue.sort((a, b) => a.priority - b.priority);

    this.process();
  }

  private async process() {
    if (this.processing || this.queue.length === 0) return;

    this.processing = true;

    while (this.queue.length > 0) {
      const item = this.queue.shift()!;

      if (!this.prefetched.has(item.path)) {
        await this.router.prefetch(item.path);
        this.prefetched.add(item.path);
        await new Promise((resolve) => setTimeout(resolve, 100));
      }
    }

    this.processing = false;
  }

  clear() {
    this.queue = [];
    this.prefetched.clear();
  }
}

export default function SmartPrefetchManager() {
  const router = useRouter();
  const manager = useRef<PrefetchManager>();

  useEffect(() => {
    manager.current = new PrefetchManager(router);

    manager.current.add("/dashboard", 1);
    manager.current.add("/profile", 2);
    manager.current.add("/settings", 3);
  }, [router]);

  return null;
}
```

### 5. 用户行为预测

根据用户行为预测下一步操作:

```tsx
"use client";
import { useRouter } from "next/navigation";
import { useEffect } from "react";

export default function PredictivePrefetch() {
  const router = useRouter();

  useEffect(() => {
    const trackUserBehavior = () => {
      const currentPath = window.location.pathname;
      const history = JSON.parse(
        localStorage.getItem("navigationHistory") || "[]"
      );

      history.push({
        path: currentPath,
        timestamp: Date.now(),
      });

      if (history.length > 50) {
        history.shift();
      }

      localStorage.setItem("navigationHistory", JSON.stringify(history));

      const nextPage = predictNextPage(history, currentPath);
      if (nextPage) {
        router.prefetch(nextPage);
      }
    };

    const predictNextPage = (
      history: Array<{ path: string; timestamp: number }>,
      currentPath: string
    ) => {
      const patterns: Record<string, string[]> = {};

      for (let i = 0; i < history.length - 1; i++) {
        const current = history[i].path;
        const next = history[i + 1].path;

        if (!patterns[current]) {
          patterns[current] = [];
        }
        patterns[current].push(next);
      }

      if (patterns[currentPath]) {
        const nextPages = patterns[currentPath];
        const counts: Record<string, number> = {};

        nextPages.forEach((page) => {
          counts[page] = (counts[page] || 0) + 1;
        });

        const mostCommon = Object.entries(counts).sort(
          ([, a], [, b]) => b - a
        )[0];

        return mostCommon ? mostCommon[0] : null;
      }

      return null;
    };

    trackUserBehavior();
  }, [router]);

  return null;
}
```
