# 路由事件与生命周期钩子

## 1. 概述

在 Next.js 16 App Router 中,传统的路由事件系统已被移除,取而代之的是基于 React 生命周期和自定义 Hook 的新模式。本文档详细介绍如何在 App Router 中监听和响应路由变化。

### 1.1 概念定义

路由事件指的是在路由导航过程中触发的各种事件,如导航开始、导航完成、路由变化等。在 App Router 中,这些事件通过 React 的生命周期钩子和自定义实现来处理。

**关键特征**:

- **无内置事件**: App Router 不提供 `router.events`
- **React 生命周期**: 使用 `useEffect` 等 Hook 监听变化
- **自定义实现**: 需要自己实现事件监听逻辑
- **更灵活**: 可以根据需求定制事件处理

**Pages Router vs App Router**:

```tsx
// Pages Router (旧)
import { useRouter } from "next/router";
import { useEffect } from "react";

function Component() {
  const router = useRouter();

  useEffect(() => {
    const handleRouteChange = (url) => {
      console.log("路由变化:", url);
    };

    router.events.on("routeChangeStart", handleRouteChange);

    return () => {
      router.events.off("routeChangeStart", handleRouteChange);
    };
  }, [router]);
}

// App Router (新)
("use client");

import { usePathname, useSearchParams } from "next/navigation";
import { useEffect } from "react";

function Component() {
  const pathname = usePathname();
  const searchParams = useSearchParams();

  useEffect(() => {
    console.log("路由变化:", pathname);
  }, [pathname, searchParams]);
}
```

### 1.2 核心价值

**更好的 React 集成**:

使用标准的 React 模式,与 React 生态系统更好地集成,代码更易理解和维护。

**类型安全**:

TypeScript 支持更好,编译时就能发现类型错误。

**性能优化**:

避免全局事件监听器,只在需要的组件中监听路由变化,减少不必要的渲染。

**灵活性**:

可以根据具体需求定制事件处理逻辑,不受框架限制。

### 1.3 常见路由事件对比

| Pages Router 事件     | App Router 替代方案         | 说明             |
| :-------------------- | :-------------------------- | :--------------- |
| `routeChangeStart`    | `useEffect` + `usePathname` | 监听路由开始变化 |
| `routeChangeComplete` | `useEffect` + `usePathname` | 监听路由完成变化 |
| `routeChangeError`    | 自定义错误边界              | 处理路由错误     |
| `beforeHistoryChange` | `useEffect` cleanup         | 路由变化前执行   |
| `hashChangeStart`     | `useEffect` + hash 监听     | 监听 hash 变化   |

## 2. 核心概念与原理

### 2.1 使用 usePathname 监听路由变化

`usePathname` Hook 返回当前路径,当路径变化时组件会重新渲染。

```tsx
"use client";

import { usePathname } from "next/navigation";
import { useEffect } from "react";

export default function RouteListener() {
  const pathname = usePathname();

  useEffect(() => {
    console.log("路由变化:", pathname);

    // 在这里执行路由变化后的逻辑
    // 例如:发送分析事件、更新状态等
  }, [pathname]);

  return null; // 或者返回 UI
}
```

### 2.2 使用 useSearchParams 监听查询参数变化

`useSearchParams` Hook 返回当前查询参数,参数变化时组件会重新渲染。

```tsx
"use client";

import { useSearchParams } from "next/navigation";
import { useEffect } from "react";

export default function SearchParamsListener() {
  const searchParams = useSearchParams();

  useEffect(() => {
    const query = searchParams.get("q");
    console.log("搜索参数变化:", query);

    // 执行搜索逻辑
  }, [searchParams]);

  return null;
}
```

### 2.3 组合监听路径和参数

同时监听路径和查询参数的变化:

```tsx
"use client";

import { usePathname, useSearchParams } from "next/navigation";

export default function FullRouteListener() {

  const searchParams = useSearchParams();

  }, [pathname, searchParams]);

  return null;
}
```

### 2.4 自定义路由事件 Hook

创建一个自定义 Hook 来模拟 Pages Router 的事件系统:

```tsx
// hooks/useRouteChange.ts
"use client";

import { usePathname, useSearchParams } from "next/navigation";
import { useEffect, useRef } from "react";

export function useRouteChange(onRouteChange: (url: string) => void) {
  const pathname = usePathname();
  const searchParams = useSearchParams();
  const isFirstRender = useRef(true);

  useEffect(() => {
    // 跳过首次渲染
    if (isFirstRender.current) {
      isFirstRender.current = false;
      return;
    }

    const url = `${pathname}?${searchParams}`;
    onRouteChange(url);
  }, [pathname, searchParams, onRouteChange]);
}

// 使用示例
export default function MyComponent() {
  useRouteChange((url) => {
    console.log("路由变化:", url);
  });

  return <div>内容</div>;
}
```

## 4. 常见问题

### 1. 为什么 App Router 没有 router.events?

App Router 采用了新的架构,使用 React 的生命周期和 Hook 来处理路由事件,这样更符合 React 的设计理念。

| Pages Router       | App Router              |
| ------------------ | ----------------------- |
| router.events.on() | useEffect + usePathname |
| 全局事件监听       | 组件级监听              |
| 命令式 API         | 声明式 API              |

### 2. 如何监听路由开始变化?

使用 useEffect 的 cleanup 函数:

```tsx
"use client";

import { usePathname } from "next/navigation";
import { useEffect } from "react";

export default function Component() {
  const pathname = usePathname();

  useEffect(() => {
    console.log("路由变化开始");

    return () => {
      console.log("路由变化结束");
    };
  }, [pathname]);

  return null;
}
```

### 3. 如何处理路由错误?

使用 Error Boundary:

```tsx
"use client";

import { Component, ReactNode } from "react";

interface Props {
  children: ReactNode;
}

interface State {
  hasError: boolean;
}

export class RouteErrorBoundary extends Component<Props, State> {
  constructor(props: Props) {
    super(props);
    this.state = { hasError: false };
  }

  static getDerivedStateFromError() {
    return { hasError: true };
  }

  componentDidCatch(error: Error) {
    console.error("路由错误:", error);
  }

  render() {
    if (this.state.hasError) {
      return <div>路由加载失败</div>;
    }

    return this.props.children;
  }
}
```

## 6. 注意事项

### 6.1 服务端组件不能使用路由 Hook

路由 Hook 只能在客户端组件中使用:

```tsx
// ❌ 错误:服务端组件
export default function ServerComponent() {
  const pathname = usePathname(); // 错误!
  return <div>{pathname}</div>;
}

// ✅ 正确:客户端组件
("use client");

export default function ClientComponent() {
  const pathname = usePathname(); // 正确
  return <div>{pathname}</div>;
}
```

### 6.2 避免在 useEffect 中执行耗时操作

```tsx
// ❌ 不好
useEffect(() => {
  // 耗时操作会阻塞渲染
  heavyComputation();
}, [pathname]);

// ✅ 好
useEffect(() => {
  // 使用 setTimeout 异步执行
  setTimeout(() => {
    heavyComputation();
  }, 0);
}, [pathname]);
```

### 6.3 注意内存泄漏

```tsx
"use client";

import { usePathname } from "next/navigation";
import { useEffect } from "react";

export default function MemoryLeakExample() {
  const pathname = usePathname();

  useEffect(() => {
    const subscription = someObservable.subscribe();

    // 必须清理订阅
    return () => subscription.unsubscribe();
  }, [pathname]);

  return null;
}
```

```tsx
"use client";

import { usePathname, useRouter } from "next/navigation";
import { useEffect, useState } from "react";

export function RouteInterceptor({ children }: { children: React.ReactNode }) {
```

```tsx
"use client";

import { usePathname } from "next/navigation";
import { useEffect } from "react";

const routeCache = new Map<string, any>();

export function useRouteCache<T>(key: string, fetcher: () => Promise<T>) {
  const pathname = usePathname();
  const cacheKey = `${pathname}:${key}`;

  useEffect(() => {
    if (!routeCache.has(cacheKey)) {
      fetcher().then((data) => {
        routeCache.set(cacheKey, data);
      });
    }
  }, [cacheKey, fetcher]);

  return routeCache.get(cacheKey);
}
```

### 8.4 如何实现路由回退确认?

**问题**: 用户点击后退时弹出确认框

**解决方案**:

```tsx
"use client";

import { useEffect } from "react";

export function BackConfirmation({ enabled }: { enabled: boolean }) {
  useEffect(() => {
    if (!enabled) return;

    const handlePopState = (e: PopStateEvent) => {
      const confirmed = window.confirm("确定要离开此页面吗?");
      if (!confirmed) {
        window.history.pushState(null, "", window.location.href);
      }

    window.addEventListener("popstate", handlePopState);

    return () => {

    };
  }, [enabled]);

  return null;
}
```

```tsx
export function RouteErrorBoundary({
  children,
}: {
  children: React.ReactNode;
}) {
  const pathname = usePathname();
  const [error, setError] = useState<Error | null>(null);

  useEffect(() => {
    setError(null);
  }, [pathname]);

  if (error) {
    return (
      <div>
        <h2>路由加载失败</h2>
        <p>{error.message}</p>
        <button onClick={() => setError(null)}>重试</button>
      </div>
    );
  }

  return <>{children}</>;
}
```

## 9. 总结

App Router 中的路由事件处理采用了新的模式。本文介绍了:

1. **核心概念**: usePathname、useSearchParams、useEffect 监听路由变化
2. **实战场景**: 页面浏览统计、面包屑导航、全局分析、加载状态、历史记录、权限检查
3. **常见问题**: 为什么没有 router.events、如何监听路由开始、错误处理、滚动控制、查询参数监听
4. **最佳实践**: 自定义 Hook、避免重渲染、Suspense、清理副作用、性能优化
5. **注意事项**: 服务端组件限制、避免耗时操作、内存泄漏、Suspense 要求、渲染机制

关键要点:

- 使用 usePathname 和 useSearchParams 监听路由变化
- 使用 useEffect 处理路由变化的副作用
- 封装自定义 Hook 提高代码复用性
- 注意清理副作用避免内存泄漏
- 只在客户端组件中使用路由 Hook

通过正确使用这些模式,可以实现 Pages Router 中 router.events 的所有功能,并且代码更加清晰和可维护。
